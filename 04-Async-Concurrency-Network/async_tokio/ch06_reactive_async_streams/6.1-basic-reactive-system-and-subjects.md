# 6.1 Building a Basic Reactive System & Defining Our Subjects

## 响应式编程在做什么

代码对**数据值或事件**的变化做出**动态响应**，而不是一次性算完就结束。本章从手写加热器系统入手，不依赖庞大外部框架。

## 观察者模式 (Observer Pattern)

| 角色 | 职责 |
|------|------|
| **Subject（主题）** | 持有状态，变化时通知订阅方 |
| **Observer（观察者）** | 订阅变化并执行副作用（控温、刷新 UI 等） |

**优点**：主体与观察者解耦，增删观察者不必改主体逻辑。

## 加热器案例中的主题

书中将以下状态建模为全局可共享的**主题**：

- **当前温度** `TEMP`（`AtomicI16`）
- **目标温度** `DESIRED_TEMP`（`AtomicI16`）
- **加热器开关** `HEATER_ON`（`AtomicBool`）

## 实现原则：避免过度设计

- 仅用标准并发原语：`AtomicI16` / `AtomicBool` + `LazyLock`（或 `OnceLock`）提供**线程安全、低开销**的全局状态。
- 先让「主题」清晰，再为每个观察者写独立 **Future**（见 §6.2–6.3）。
