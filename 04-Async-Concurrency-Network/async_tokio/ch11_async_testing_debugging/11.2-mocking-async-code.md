# 11.2 Mocking Async Code（模拟异步代码）

> Demo：[11.2-mocking-async-code/11.2-mocking-async-code-demo.rs](./11.2-mocking-async-code/11.2-mocking-async-code-demo.rs)

## 纯异步 Trait 的 Mock

与 §11.1 测「同步壳」不同，本节直接 mock **返回 Future 的 trait 方法**。`mockall` 对 `async fn`（经 `async_trait` 展开）可生成 `expect_*().returning(|_| async { ... })`。

## 为何每测单独 `current_thread` 运行时

```rust
let rt = tokio::runtime::Builder::new_current_thread()
    .enable_all()
    .build()
    .unwrap();
rt.block_on(async { /* 被测逻辑 */ });
```

| 问题 | 单测独立 RT 的解法 |
|------|-------------------|
| 全局 `Runtime` 泄漏任务 | 每用例新建 RT，drop 即清理 |
| 测试间顺序依赖 | 无共享队列 |
| 多线程 RT 放大竞态 | current_thread 更可复现 |

生产可用 `#[tokio::test]`（默认 often multi_thread）；**教学与脆弱用例**优先 current_thread。

## AFIT vs `async_trait`

- **AFIT**：`fn fetch(&self) -> impl Future<Output = T>` — mock 生成器支持有限；
- **`async_trait`**：`async fn fetch(...) -> T` — 书中与本书仓库推荐路径。

## Demo 说明

`11.2-mocking-async-code-demo.rs` 演示：**无 mockall 依赖** 的手写 `trait` + 假实现 + `current_thread` `block_on`，理解「替换依赖」与「隔离运行时」两件事；真实项目换 `#[automock]` 即可。
