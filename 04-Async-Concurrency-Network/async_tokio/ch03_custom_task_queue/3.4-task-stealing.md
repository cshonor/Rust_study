# 3.4 Task Stealing（任务窃取）

## 核心梳理

- **问题**：高优队列空转、低优队列堆积 → 浪费 CPU。
- **窃取**：某 worker 主队列空时，尝试从**其他**通道（常为低优）拉取任务执行。
- **流程**：
  1. `recv` 自己的主通道；
  2. 若空，尝试低优（或其它）通道；
  3. 仍空则 **park** / sleep 短间隔后重试。

## 风险

过度窃取可能让轻量 I/O 任务与重计算任务混跑，延迟抖动；可按业务关闭部分队列的窃取目标。
