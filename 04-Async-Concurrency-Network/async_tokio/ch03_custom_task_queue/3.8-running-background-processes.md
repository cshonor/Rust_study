# 3.8 Running Background Processes（运行后台进程）

## 核心梳理

- **场景**：监控、清理、心跳等需**整个进程生命周期**持续运行的任务。
- **问题**：若 `spawn` 后主线程既不 `await` 也不托管，Task 可能在完成前被 **drop** 取消。
- **`detach()`**：将任务所有权交给运行时后台循环，**无需**主函数显式等待（API 名以 `async-task` / 书中为准）。

## 与第 2 章衔接

注意 **cancel safety** 与 **detach** 后无人等待时的错误处理；关键状态应自带超时/监督。
