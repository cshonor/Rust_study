# 4.5 Implementing the Tokio AsyncWrite Trait（实现 AsyncWrite）

## 核心梳理

- **`poll_write`**：缓冲区数据写入流；阻塞则 `Pending`。
- **`poll_flush`**：确保内核/协议栈已接收缓冲数据。
- **`poll_shutdown`**：半关闭写侧，完成优雅收尾。
- 三者都须在**不可阻塞**前提下把控制权还给 Executor。

## 与 AsyncRead 配对

自定义 `CustomStream` 需同时实现 Read + Write，`hyper` 才能双向传 HTTP body。
