## 一、本节在学什么

网络代码里两个高频小工具：

- **`bitflags`**：类型安全的 **位标志**（TCP 头、socket 选项、自定义协议）
- **`url`**：解析/构造 URL（scheme、host、path、query）

---

## 二、`bitflags`

协议常出现 **C 风格位标志**（2 的幂）：

```rust
use bitflags::bitflags;

bitflags! {
    struct TcpFlags: u8 {
        const FIN = 0b001;
        const SYN = 0b010;
        const ACK = 0b100;
    }
}

let flags = TcpFlags::SYN | TcpFlags::ACK;
println!("bits: {:b}, has ACK? {}", flags.bits(), flags.contains(TcpFlags::ACK));
```

| 能力 | 说明 |
|------|------|
| `\|` `&` | 组合、掩码 |
| `contains()` | 是否含某位 |
| `bits()` | 原始整数，可写入报文 |

对照 TCP 首部：FIN/SYN/RST/PSH/ACK/URG 即典型 bitflags 场景。

---

## 三、`url`

```rust
use url::Url;

let u = Url::parse("https://example.com:8443/path?q=1#frag")?;
println!("scheme={} host={:?} path={}", u.scheme(), u.host(), u.path());
```

| 字段 | 示例 |
|------|------|
| scheme | `https` |
| host | `example.com` |
| port | `8443`（可选，默认随 scheme） |
| path | `/path` |
| query | `q=1` |
| fragment | `#frag`（通常不发 HTTP 请求） |
| username / password | 可选（避免明文存密码） |

与 **reqwest**：可先 `Url::parse` 再 `join` 路径、改 query，比字符串拼接安全。

---

## 四、典型网络用途

| crate | 用途 |
|-------|------|
| `bitflags` | TCP 标志、IP 头、setsockopt、自定义协议头 |
| `url` | 爬虫起始 URL、重定向解析、API 客户端基址 |

---

## 五、新手易错

1. **字符串拼 URL**：特殊字符未 encode → 用 `Url::parse_with_params` 等 API。
2. **bitflags 宽度**：与协议字段一致（`u8` vs `u32`）。
3. **把 fragment 当服务端路由**：`#` 段不发送到服务器。
4. **URL 里明文密码**：日志/Referer 可能泄露。
