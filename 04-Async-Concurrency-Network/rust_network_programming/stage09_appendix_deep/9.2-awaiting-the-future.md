## 一、本节在学什么

`futures` 0.1 时代手写 **`and_then`** 链 → **回调地狱** + 复杂关联类型。

书中 **`futures-await`** 用宏预研 **`async/await`**；Rust **1.39+** 语言内置，**`futures-await` 已被取代**。

读附录时：把本节当作 **async/await 的历史与动机**。

---

## 二、问题：`and_then` 时代

```text
step1().and_then(|a| {
    step2(a).and_then(|b| {
        step3(b).and_then(|c| {
            ...
        })
    })
})
```

| 痛点 | async/await 解法 |
|------|------------------|
| 嵌套深 | 线性代码 |
| 错误传播麻烦 | 函数内直接用 `?` |
| 类型推断困难 | `async fn` 返回 `impl Future` |

---

## 三、现代：语言内置 `async/await`

| 书中 `futures-await` | 现代 Rust |
|----------------------|-----------|
| `#[async]` | `async fn` |
| `await!(future)` | `future.await` |
| crate 宏 | 编译器内置 |

```rust
use tokio::time::{sleep, Duration};

async fn step1() -> u32 {
    sleep(Duration::from_millis(50)).await;
    1
}

async fn step2(v: u32) -> u32 {
    sleep(Duration::from_millis(50)).await;
    v + 2
}

#[tokio::main]
async fn main() {
    let result = step2(step1().await).await;
    println!("linear async result: {result}");
}
```

等价于顺序 `and_then`，但可读性接近同步代码。

仓库对应：[9.2-awaiting-future-linear-demo.rs](./9.2-awaiting-the-future/9.2-awaiting-future-linear-demo.rs)

## 四、谁 poll 你的 Future？

| 机制 | 说明 |
|------|------|
| **`#[tokio::main]`** | 创建 runtime，跑 `main` 的 Future |
| **`.await`** | 当前 task 注册 waker，Pending 时 yield |
| **`block_on`** | 当前线程阻塞直到 Future 完成 |

深入理解 executor：[async_tokio/ch10_noop_waker_block_on/](../../async_tokio/ch10_dependency_free_async_server/) — 无依赖 server + noop waker。

## 五、新手易错

1. **在文档里找 `futures-await`**：已过时，用标准 `async/await`。
2. **`.await` 在 non-async 上下文**：须 `block_on` 或包在 `async fn` 里。
3. **忘记 `?` 需要 `async fn` 返回 `Result`**：错误类型要一致。
4. **把 `.await` 当成阻塞 OS 线程**：只阻塞**当前 task**，不阻塞其它 task。
