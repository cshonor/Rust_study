## 一、本节在学什么

在 [§6.1 Hyper](./6.1-introducing-hyper.md) 的底层 HTTP 之上，认识 **Rocket** —— 高层 **Web 框架**，用**属性宏**声明路由、用**提取器**解析请求，快速搭建 REST API 与动态页面。

**一句话**：Hyper 管「HTTP 线与连接」；Rocket 管「URL → 函数 → 响应」的应用结构。

---

## 二、Rocket 简介

**Rocket**：强调声明式路由与易用性（体验上常被比作 Python **Flask**）。

| 特点 | 说明 |
|------|------|
| 路由 | **`#[get("/path")]`、`#[post(...)]`** 等属性宏 |
| 启动 | **`#[launch]`** 生成 `main`，配置端口等 |
| 编译器 | 书中时代常依赖 **nightly**；**Rocket 0.5** 起以 stable 为主，以官方文档为准 |
| 定位 | 快速做网站 / REST API，少写底层 HTTP 解析 |

### 最小路由（仓库 demo）

```rust
#[get("/")]
fn index() -> &'static str {
    "Hello from 6.2 Rocket demo"
}

#[get("/api/ping")]
fn ping() -> Json<Ping> { ... }

#[launch]
fn rocket() -> _ {
    let config = rocket::Config { port: 18082, ..Default::default() };
    rocket::custom(config).mount("/", routes![index, ping])
}
```

| 元素 | 作用 |
|------|------|
| `#[get("/api/ping")]` | 路径 + HTTP 方法绑定到函数 |
| `Json<Ping>` | 响应体自动 `Serialize` 为 JSON |
| `#[launch]` | 入口与 `Rocket` 实例装配 |
| `mount("/", routes![...])` | 注册路由表 |

## 三、REST API + 数据库：`diesel` + `r2d2`

书中 **博客 CRUD** 典型栈（完整示例建议单独项目搭建，避免与本章其它 demo 依赖纠缠）：

| 组件 | 作用 |
|------|------|
| **`diesel`** | ORM：类型安全 SQL、迁移、对接 SQLite/Postgres 等 |
| **`r2d2`** | **连接池**：复用 DB 连接，避免每请求 `open` 一次 |
| **Rocket 路由** | `GET/POST/PUT/DELETE` 映射到 handler |

```text
HTTP 请求 → Rocket 路由 → handler → diesel 查询 → JSON/HTML 响应
```

### Data Guards + Serde

入参在边界自动 **反序列化**（与 [Ch04](../stage04_serde_parse/) 的 `Deserialize` 同一套）：

```rust
// 概念
#[post("/posts", data = "<post>")]
fn create(post: Json<NewPost>) -> Json<Post> { ... }
```

| 行为 | 说明 |
|------|------|
| 合法 JSON | 填入 `NewPost` 字段 |
| 非法 JSON / 缺字段 | 框架返回 **400**，无需手写校验样板 |
| 响应 `Json<Post>` | 自动 `Content-Type: application/json` |

## 四、动态 HTML：Tera + Managed State

书中动态页面路径：

| 组件 | 作用 |
|------|------|
| **`Tera`** | 模板引擎（类似 Jinja2），在 HTML 里填变量 |
| **`rocket_contrib`**（书中） | 集成模板；新版本可能拆到独立 crate，以文档为准 |

### Managed State + `AtomicUsize`

Rocket **托管状态**：应用级单例（如访问计数器）。

```rust
// 概念
struct Counter(AtomicUsize);

#[get("/")]
fn index(counter: &State<Counter>) -> Template { ... }
```

- 用 **`AtomicUsize`** 实现**线程安全**自增，高并发下统计 PV 无需 `Mutex`（仅演示足够；生产注意溢出与语义）。
- 与 [atomic §1](../../atomic/Chapter-01-Rust-Concurrency-Basics/1.1-threads-in-rust/1.1-threads-in-rust.md) 的原子类型一致。

```text
REST JSON API  ←→  diesel + r2d2
动态 HTML      ←→  Tera 模板 + Managed State
```

Rocket 依赖树较重，与本章 **`reqwest` / 阻塞 demo** 分开，避免 feature 与编译时间缠在一起。

| 位置 | 说明 |
|------|------|

## 六、框架选型（不必绑死 Rocket）

| 框架 | 特点 |
|------|------|
| **Rocket** | 宏路由、提取器、上手快；本书主线 |
| **axum** | Tower 生态、与 Hyper 衔接紧 |
| **actix-web** | 高性能、自有 actor 模型 |

学习 Rocket 重在掌握 **「路由宏 + 提取器 + 托管状态」** 这一套 Web 边界模式，迁移到其它框架时概念可复用。

---

## 七、新手易错

1. **端口冲突**：demo 固定 `18082`，若占用需改 `rocket::Config::port`。
2. **`#[launch]` 与 `main`**：不要手写第二个 `main`；路由需写进 `routes![...]` 才会生效。
3. **JSON 提取器**：请求体字段名须与 Serde 结构体一致，否则 400。
4. **书中 nightly 叙述**：以当前 Rocket 版本文档为准，勿假设仍要 nightly。
5. **diesel 博客示例**：迁移、连接串、池大小宜在独立仓库练，勿强行 merge 进本章 workspace。
