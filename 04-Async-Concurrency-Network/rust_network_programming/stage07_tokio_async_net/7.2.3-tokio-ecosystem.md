## 一、本节在学什么

Tokio 不仅是 `TcpListener`：**周边 crate** 组成完整异步网络栈。书中部分 crate 已更名或并入主库；对照**现代常用组合**选型。

---

## 二、书中 crate → 现代对照

| 书中 crate | 作用 | 现代常见替代 |
|------------|------|--------------|
| **`tokio-io`** | 异步 Read/Write 工具 | `tokio::io`、`AsyncReadExt` / `AsyncWriteExt` |
| **`tokio-curl`** | 异步 HTTP 客户端 | **`reqwest`**（内置 Tokio，`rustls-tls`） |
| **`tokio-tls`** | 非阻塞 TLS | **`tokio-rustls`**、`native-tls` + tokio |
| **`tokio-core`** | Reactor 核心 | **`tokio::runtime`**、`#[tokio::main]` |

## 五、新手易错

1. **crate 版本/feature 对齐**：`reqwest` 默认 TLS 与 `tokio` feature 需与项目一致（见 Ch8 `rustls-tls`）。
2. **在 Tokio 外 poll async I/O**：hyper/reqwest 假设在 Tokio runtime 内；CLI 用 `#[tokio::main]` 或 `Runtime::block_on`。
3. **重复造轮子**：分帧优先 `tokio_util::codec`；HTTP 优先 hyper/axum，不必手写 HTTP 解析。
4. **忽视 `tower` 抽象**：限流、超时、日志可叠在 Service 层，与业务 handler 分离。
