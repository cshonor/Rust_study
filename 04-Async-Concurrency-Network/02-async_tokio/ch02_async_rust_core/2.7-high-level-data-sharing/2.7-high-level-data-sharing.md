# 2.7 High-Level Data Sharing Between Futures（高级数据共享）

> Demo：[2.7-high-level-data-sharing/2.7-high-level-data-sharing-demo.rs](./code/2.7-high-level-data-sharing-demo.rs)

## 核心梳理

- **对比 §2.6**：不再手写 `poll` + `try_lock`，而用 **`tokio::sync::Mutex`** + **`async/await`**，`lock().await` 在争用时返回 `Pending` 并正确让出。
- **利弊**：
  - **高级抽象**：代码短、不易在 `poll` 里误阻塞。
  - **底层控制**：手写 `poll` 可极致优化或对接非 Tokio 执行器，但复杂且易错。

## 实践建议

日常业务优先 `async fn` + Tokio 同步原语；仅在学习/定制执行器时深入 §2.6 路径。
