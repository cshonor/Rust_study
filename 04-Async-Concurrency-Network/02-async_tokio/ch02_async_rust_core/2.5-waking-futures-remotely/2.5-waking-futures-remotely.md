# 2.5 Waking Futures Remotely（远程唤醒 Future）

> Demo：[2.5-waking-futures-remotely/2.5-waking-futures-remotely-demo.rs](./code/2.5-waking-futures-remotely-demo.rs)

## 核心梳理

- **问题**：对外部事件（网络、通道、另一线程）若**盲目轮询** Future，浪费 CPU。
- **做法**：在 `poll` 返回 `Pending` 前，把 `cx.waker().clone()` 存到共享槽位；**外部**在数据就绪时调用 `wake()`。
- **本仓库**：`mpsc` 收到消息后由另一任务取出保存的 `Waker` 并唤醒；Future 再次 `poll` 时 `try_recv` 成功 → `Ready`。

## 与真实栈的对应

生产里由 epoll/IOCP、Tokio driver、定时器 wheel 等在就绪点调用 `wake`，而不是手写通道槽位；本节建立「**谁保存 Waker、谁在何时 wake**」直觉。
