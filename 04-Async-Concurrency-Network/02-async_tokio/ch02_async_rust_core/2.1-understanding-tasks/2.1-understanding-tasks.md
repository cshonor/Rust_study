# 2.1 Understanding Tasks（理解任务）

> Demo：[2.1-understanding-tasks/2.1-understanding-tasks-join-demo.rs](./code/2.1-understanding-tasks-join-demo.rs)

## 核心梳理

- **Task（任务）**：由**执行器**驱动、直至完成的异步工作单元；`async fn` 被调用时往往先得到 Future，再被包装成 Task 入队调度。
- **协作式多任务（cooperative multitasking）**：任务在 `await` / 返回 `Pending` 时**主动让出**线程，同一线程上可切换去 `poll` 其它任务——不是 OS 抢占式「硬切」每个 async 块。
- **`tokio::join!`**：在**同一任务**内并发等待多个子 Future（例如多个 `tokio::time::sleep`），总耗时接近**最慢**那个，而不是顺序相加。

## 与线程的区别（直觉）

| | OS 线程 | async Task |
|---|---------|------------|
| 调度 | 内核抢占 | 运行时协作式 `poll` |
| 阻塞 sleep | 占满线程 | `tokio::time::sleep` 不占线程干等 |
| 数量级 | 过多线程贵 | 大量 I/O 等待任务更划算 |

## 本仓库示例

`join!` 并发三个不同延迟的 `work`；打印完成顺序与返回值。
