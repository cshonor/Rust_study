# 11.4 Testing for Race Conditions（测试竞态）

> Demo：[11.4-testing-for-race-conditions/11.4-testing-for-race-conditions-demo.rs](./code/11.4-testing-for-race-conditions-demo.rs)

## 什么是数据竞争

多个任务**并发**读—改—写同一内存，且至少一次写，且无同步 → **未定义行为**（Rust 安全代码里常表现为「偶发错数」）。

异步里还多一层：**在两次 `.await` 之间** 状态可能被其它任务改掉（过期快照），与第 6 章原子主题呼应。

## 单线程测试的盲区

`#[tokio::test(flavor = "current_thread")]` 下，任务交替是**协作式**的，很多交错不会出现。书中要求：

```rust
#[tokio::test(flavor = "multi_thread", worker_threads = 4)]
async fn stress_counter() { /* ... */ }
```

并在 CI 中 **重复运行**（`cargo test -- --test-threads=8` + 循环）提高撞车概率。

## 策略矩阵

| 手段 | 适用 |
|------|------|
| `AtomicUsize` / 通道 | 从设计上消灭共享 `mut` |
| 多线程 stress test | 集成层撞竞态 |
| **`loom`** | 探索小状态空间（进阶） |
| Miri | 部分 UB 检测 |
