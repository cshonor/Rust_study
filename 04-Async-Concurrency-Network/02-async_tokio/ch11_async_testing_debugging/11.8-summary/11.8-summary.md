# 11.8 Summary（总结）

> Demo：[11.8-summary/11.8-summary-demo.rs](./code/11.8-summary-demo.rs)

## 全书测试观

异步系统测试要覆盖 **功能 + 时序 + 并发 + I/O**：

| § | 工具/手法 |
|---|-----------|
| 11.1 | `mockall` + AAA，测隔离模块同步壳 |
| 11.2 | mock 异步 trait；**每测 `current_thread` RT** |
| 11.3 | `timeout` 判定卡死 |
| 11.4 | **multi_thread** stress，防竞态盲区 |
| 11.5 | 有界通道 + timeout 验证背压 |
| 11.6 | `mockito` / 本地 mock HTTP |
| 11.7 | `tokio-test` 逐步 poll |

## 工程建议

- **测试与实现同步推进**（TDD 或同 PR），避免「上线前一周补测试」；
- 超时失败时 **缩小用例** 二分定位，而非盲目加大 timeout；
- 能消灭共享状态就消灭；必须共享时用 **Actor / 通道 / 原子**；
- 生产默认仍依赖成熟运行时与可观测性（tracing、metrics）。
