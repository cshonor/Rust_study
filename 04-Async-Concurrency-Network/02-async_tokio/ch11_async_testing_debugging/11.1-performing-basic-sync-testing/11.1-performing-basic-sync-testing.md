# 11.1 Performing Basic Sync Testing（执行基础的同步测试）

> Demo：[11.1-performing-basic-sync-testing/code/](./code/)（占位 + 见下方集成测试）

## 测什么

第九章的 **隔离异步模块**：对外是**同步 Facade**（`submit(id)` / `blocking_get_result(id)`），对内是 Tokio `Runtime` + `HashMap<TaskId, JoinHandle>`。测试目标是在**不启动完整业务进程**的前提下，验证 Facade 与依赖协作是否正确。

## `mockall` 与 AAA

| 阶段 | 做什么 |
|------|--------|
| **Arrange** | `#[automock]` 生成 trait mock；`expect_foo().times(1).returning(...)` |
| **Act** | 调用同步 API |
| **Assert** | 返回值、`mock.verify()` 或 drop 时自动校验 |

```rust
// 示意（完整见 tests/ch11_database_mock.rs）
#[automock]
#[async_trait::async_trait]
trait Database {
    async fn fetch(&self, key: &str) -> Option<String>;
}
```

若 trait 使用 **AFIT**（`impl Future`），稳定工具链上常改为 **`async_trait`**，语义等价且 `mockall` 支持成熟。

## 与「整库集成测试」对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| 单元 + mock | 快、定位准 | 需维护 mock 与真实实现一致 |
| 端到端 | 真实 | 慢、难复现边界 |

## 实践建议

- 同步壳只测 **ID 映射、错误传播、取消语义** 的契约；
- 异步逻辑用 `#[tokio::test]` 单测或 §11.2 的 current_thread 运行时；
- 避免在单元测试里 `sleep` 过久，用毫秒级或 `tokio::time::pause`。

## 运行

- 占位：`rustc 11.1-performing-basic-sync-testing-stdlib-demo.rs`（需 `#![crate_name]`）
- 推荐：在仓库根添加 `tests/ch11_database_mock.rs` 后 `cargo test --test ch11_database_mock`
