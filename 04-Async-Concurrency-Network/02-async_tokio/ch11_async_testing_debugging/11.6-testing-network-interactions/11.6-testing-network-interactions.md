## 真实外服的问题

| 问题 | 后果 |
|------|------|
| 慢 | CI 超时、反馈差 |
| 不稳定 | 偶发红构建 |
| 共享状态 | 并行测试互相污染（服务端竞态） |

## `mockito` 思路

在测试进程内启动 **本地 HTTP mock server**：

```toml
# Cargo.toml [dev-dependencies]
# mockito = "1"
```

```rust
let mut server = mockito::Server::new_async().await;
let mock = server
    .mock("POST", "/items")
    .with_status(201)
    .with_body(r#"{"id":1}"#)
    .expect(1)
    .create_async()
    .await;
let url = server.url();
// 被测客户端请求 url ...
mock.assert_async().await;
```

- **可注入 base URL**，避免硬编码生产地址；
- **`expect(n)`** 断言调用次数；
- 与 **`wiremock`** 同类，按项目选型。

## 与 `reqwest` / hyper 客户端

被测代码应接受 `base_url: &str` 或 trait 抽象，测试里指向 `server.url()`。

## Demo

本仓库 demo 为 **std**：说明 mockito 流程 + 用 `TcpListener` 回一行固定 HTTP（无 mockito 依赖，便于 `rustc`）。集成测试请在主 crate 加 dev-deps 后按书补全。
