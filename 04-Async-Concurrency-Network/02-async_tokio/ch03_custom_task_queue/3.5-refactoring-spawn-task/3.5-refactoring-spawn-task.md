# 3.5 Refactoring Our spawn_task Function（重构 spawn_task）

## 核心梳理

- **痛点**：早期要求 Future 实现「优先级」特型，调用繁琐。
- **改进**：用 **`macro_rules!`** 包装 `spawn_task`：
  - 默认走**低优先级**；
  - 可选传入 `High` / `Low` 等优先级参数。
- **收益**：API 接近日常 `spawn`，仍保留路由到不同队列的能力。

## 宏思路（示意）

```rust
spawn_task!(my_future);           // 默认 Low
spawn_task!(High, my_future);     // 显式高优
```
