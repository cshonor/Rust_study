# 3.6 Creating Our Own Join Macro（自定义 Join 宏）

> Demo：[3.6-creating-join-macro/3.6-creating-join-macro-demo.rs](./code/3.6-creating-join-macro-demo.rs)

## 核心梳理

- **痛点**：主函数里多次 `block_on` 等待多个任务，样板代码多。
- **做法**：自写 **`join!` / `try_join!`**，一次传入多个 Future 并收集结果。

## 本仓库宏语义（重要）

```rust
macro_rules! join {
    ($($future:expr),*) => {{
        let mut results = Vec::new();
        $( results.push(futures_lite::future::block_on($future)); )*
        results
    }};
}
```

这是对每个 Future **依次** `block_on`，**不是** Tokio `join!` 在同一执行器上的真并发。对照书中「自建队列 + worker」时，应让多个任务先入队再由线程池推进。

## 依赖

`futures-lite`、`flume`（demo 末尾通道收发演示）。
