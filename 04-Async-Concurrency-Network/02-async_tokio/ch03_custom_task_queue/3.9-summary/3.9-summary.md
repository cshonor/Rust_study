# 3.9 Summary（总结）

## 核心梳理

- 本章从 **`spawn_task` + flume + worker 循环** 出发，逐步扩展到**多线程、多队列、优先级路由、任务窃取、`spawn` 宏、`join!` 宏、`Runtime` 配置、`detach` 后台任务**。
- 掌握队列机制后，可按业务**禁用某些窃取**、调整 worker 比例、隔离后台任务。
- **下一章**：在同一运行时骨架上**接入网络 I/O（Networking）**。

## 常见陷阱（回顾）

| 陷阱 | 说明 |
|------|------|
| `std::thread::sleep` in worker | 阻塞整条 worker，拖垮队列 |
| 顺序 `join!` 宏 | ≠ 真并发；勿与 Tokio `join!` 混淆 |
| 未 await / 未 detach 的 spawn | 任务可能被提前丢弃 |
| 日志顺序 | 异步完成顺序非确定性 |
