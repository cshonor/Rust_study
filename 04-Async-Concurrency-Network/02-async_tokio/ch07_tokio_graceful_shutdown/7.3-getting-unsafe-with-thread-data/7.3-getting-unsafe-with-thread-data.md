# 7.3 Getting Unsafe with Thread Data（不安全地访问线程数据）

## 动机

在单线程本地池中，若仍用 **`RefCell`**，运行时要做借用检查。为榨取性能，书中用 **`UnsafeCell`** **无锁**访问线程内数据。

## 安全边界（必须同时满足）

1. 任务跑在 **单线程本地池**，数据**不跨线程** `Send`。
2. 持有裸指针访问期间 **不 `await`**——避免任务挂起后同线程跑别的任务破坏别名规则。
3. 自行证明无数据竞争；否则为 **UB**。

## 与 §7.2 的关系

`spawn_pinned` 提供「固定线程」语义；`UnsafeCell` 提供「同步块内无借用检查」语义——二者组合才是书中完整优化路径。

## 常见陷阱

- 在 `unsafe` 块中 **`await`** → 可能切换任务并破坏独占访问。
- 把 `UnsafeCell` 包装的类型 **`Send` 到别的线程** → 竞争。
