# 7.4 Graceful Shutdowns（优雅停机）

## 覆盖默认强制退出

用户 **Ctrl+C** 或 **`SIGHUP`** 等信号不应立刻 `exit`，而应触发**收尾流水线**：

1. 停止接受新请求；
2. **`await`** 未完成句柄（各本地池上的任务）；
3. 汇总状态（如书中 `get_complete_count` 一类接口）并**持久化**；
4. 再退出进程。

## 信号监听

```rust
use tokio::signal;

signal::ctrl_c().await.expect("ctrl_c");
// 向各 LocalPool 发 shutdown 任务，刷盘…
```

## 与本地池结合

向**各独立本地线程** `spawn` 专用异步任务，提取线程局部数据并汇总——避免主线程一把抓导致遗漏 pinned 状态。
