# 8.2 Working with Actors Versus Mutexes（Actor 与 Mutex 对比）

## 基准测试结论（书中）

高并发下 **Actor（消息传递）** 往往优于 **Mutex（共享锁）**：

| 维度 | Actor | Mutex |
|------|-------|-------|
| 锁竞争 | 无（串行处理邮箱） | 多任务争用，可能挂起等待 |
| 状态封装 | 完全在 Actor 内 | 锁外可见性难控 |
| 死锁 | 消息图清晰时较少 | 锁顺序错误易死锁 |
| 吞吐 | 常更高（复杂场景） | 上下文切换 + 等锁开销 |

## 何时仍用 Mutex

极小临界区、只读多写少、或已有同步数据结构时，Mutex 更简单；**不必全栈 Actor**。
