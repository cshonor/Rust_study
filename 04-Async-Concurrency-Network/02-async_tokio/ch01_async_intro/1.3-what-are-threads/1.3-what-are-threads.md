# 1.3 What Are Threads?（什么是线程？）

## 核心梳理

- **调度单元**：线程是 OS 调度的常见**最小执行序列**；`thread::spawn` + `JoinHandle` 分发计算任务。
- **四种生命周期（API 视角）**：可连接 / 分离 / 终止 / `main` 退出强杀 — 精读见 [atomic §1.1.7](../../01-atomic/Chapter-01-Rust-Concurrency-Basics/1.1-threads-in-rust/1.1.7-thread-lifecycle-four-states.md)（Rust **无**与内核一一对应的线程状态枚举）。
- **共享内存**：同进程线程共享地址空间；传数据可用 **`Arc<T>`**（比跨进程 IPC 轻）。
- **同步原语**：**`Mutex`** 互斥、**`Condvar`** 在条件不满足时**挂起**线程，状态就绪后**唤醒**——这是理解 async 里 `park`/`Waker` 的 OS 层对照。

## 上下文切换

线程过多时，保存/恢复寄存器、缓存失效带来开销。async 用**少量 OS 线程** multiplex 大量「等 I/O」任务，减少「为等而开线程」。

## 数据竞争

多线程改同一数据需纪律；Rust 编译期拦截部分 **Data Race**，逻辑设计仍须 `Mutex`/消息传递。
