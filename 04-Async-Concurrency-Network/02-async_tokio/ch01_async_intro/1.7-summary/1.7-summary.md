# 1.7 Summary（总结）

## 核心梳理

- 异步在**不额外占满 CPU 核心**的前提下，高效处理**多个同时等待**的任务（I/O-bound）。
- 全章脉络：**进程隔离 vs 线程共享** → **I/O 延迟尺度** → **文件/HTTP 实战** → 为第 2 章 **Future / Task / poll** 铺路。

## 不要阻塞执行器（全章最重要守则）

| 错误 | 后果 |
|------|------|
| `async` 里 `std::thread::sleep` | 阻塞跑 `poll` 的 OS 线程，拖垮同池其它任务 |
| 同步大文件读、死锁式 `Mutex` | 同上 |

**正确**：`tokio::time::sleep`、`spawn_blocking`、异步锁；重 CPU 用独立线程池。
