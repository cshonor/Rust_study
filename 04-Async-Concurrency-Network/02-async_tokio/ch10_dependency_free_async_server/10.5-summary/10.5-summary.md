# 10.5 Summary（总结）

## 核心梳理

- 自研运行时 **≠** Tokio 性能/生态，但理解 **poll / Waker / 非阻塞 I/O** 的底气无价；
- 适用：定制调度、小众协议、与成熟框架**边界协作**。

## 陷阱

- 忙轮询教学版 ≠ epoll/IOCP 生产性能；
- 生产默认仍选 **Tokio** 等成熟栈。
