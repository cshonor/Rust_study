# 4.4 Implementing the Tokio AsyncRead Trait（实现 AsyncRead）

## 核心梳理

- 第三方库仍需要 **`tokio::io::AsyncRead`**（或 hyper 版本要求的 IO 抽象）。
- **`poll_read`**：底层未就绪 → `Poll::Pending` + 注册 **Waker**；有字节 → 拷入 `buf` → `Poll::Ready(n)`。
- **非阻塞**：底层 `read` 遇 `WouldBlock` 时必须返回 `Pending`，不能阻塞 worker 线程。

## 与 Future 一致

AsyncRead 本质是「可轮询的读 Future」；与 §2.2 `poll` 模型相同。
