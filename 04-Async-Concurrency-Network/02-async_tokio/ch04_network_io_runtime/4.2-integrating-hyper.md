# 4.2 Integrating hyper into Our Async Runtime（将 hyper 集成到自定义运行时）

## 核心梳理

- **`hyper` 默认依赖 Tokio**；在无 Tokio 的自定义运行时会崩溃或无法调度。
- **做法**：为手写运行时实现 **`hyper::rt::Executor`**（或对应版本的执行器 trait），让 `hyper` 的 spawn 走你的任务队列。
- **目标**：复用 `hyper` 的 HTTP 解析/客户端，而不被锁定在 Tokio 全家桶。

## 实现要点

```text
hyper Client::builder().executor(CustomExecutor).build(...)
```

`CustomExecutor` 内部调用第 3 章 `spawn_task` / Runnable。
