# 6.5 Enabling Broadcasting with an Event Bus & Struct

> Demo：[6.5-event-bus-broadcasting/code/](./code/)

## 为何从原子变量升级

§6.1–6.4 用全局 **Subject（原子变量）** 足够简单；当需要把**同一条事件**投递给**多个独立订阅者**（各有过滤逻辑、各自队列）时，观察者直接轮询原子会导致：

- 订阅者之间隐式耦合；
- 广播语义不清晰；
- 难以做 per-subscriber 背压。

**事件总线**把「发布」与「消费」显式化。

## `EventBus` 结构（书中）

```text
EventBus
├── next_id: AtomicUsize / 互斥递增
└── subscribers: AsyncMutex<HashMap<SubscriberId, VecDeque<Event>>>
```

| 设计点 | 原因 |
|--------|------|
| **每订阅者一个 `VecDeque`** | 独立背压与消费速度 |
| **`Event: Clone`** | 一条消息 fan-out 多份 |
| **`AsyncMutex`** | 异步上下文安全修改 map |
| **`publish` 持锁尽量短** | 减少阻塞其它任务 |

## 发布流程（概念）

1. `publish(event)` 克隆事件；
2. 遍历 map，对每个队列 `push_back`；
3. （可选）唤醒各订阅者 waker。

## 与观察者模式对比

| 模式 | 数据路径 |
|------|----------|
| 观察者 + 原子 | 所有人读同一 `TEMP` |
| 事件总线 | `publish(Event::TempChanged { .. })` |
