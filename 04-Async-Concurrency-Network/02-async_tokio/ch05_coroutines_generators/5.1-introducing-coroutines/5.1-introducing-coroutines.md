# 5.1 Introducing Coroutines（协程简介）

## 什么是协程

协程可在执行中**暂停**，之后从**断点恢复**——不同于跑到底的普通函数，也不同于 OS **抢占式**调度的线程。

| 对比 | 行为 |
|------|------|
| 子例程 | 一次跑完 |
| 线程 | OS 抢占 |
| 协程 | **协作式**主动让出 |

## 为什么使用协程

- **单线程追加写文件**：保留状态、多次写入，比反复 open/close 快（书中约 **6×** 量级，视 I/O 而定）。
- **无需 Mutex**：同一线程内协程交替执行，无数据竞争。
- **有状态 + 非阻塞**：暂停时不占满 CPU（由外部循环决定何时 `resume`）。

## Rust 现状

语言级 `#[coroutine]` / `Coroutine` 仍在 **nightly**；本仓库用 **状态机** 在 stable 上对照 Yield/Resume 语义（见 §5.2 demo）。
