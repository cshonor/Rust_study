# 5.5 Testing Coroutines（测试协程）

## 核心梳理

- 协程是**同步代码 ↔ 异步运行时**的桥梁，**单元测试**尤其强大。
- **Step-by-step**：在纯同步测试里手动 `resume` 两个争抢同一 **`Mutex`** 的协程，精确控制**轮询顺序**，覆盖边界情况。
- **无需复杂 async 测试框架**即可验证逻辑；通过后再包成 `Future` 接 **`#[tokio::test]`** 做集成验证。

## 与第 2 章对比

| | 直接测 Future | 测协程 + 适配器 |
|---|----------------|-----------------|
| 顺序控制 | 难 | 易（手动 resume） |
| 依赖 | 需运行时 | 可先零运行时 |

## nightly

语言内置协程用 `cargo +nightly` 在独立 example 实验；stable 用 §5.2 状态机测法。
