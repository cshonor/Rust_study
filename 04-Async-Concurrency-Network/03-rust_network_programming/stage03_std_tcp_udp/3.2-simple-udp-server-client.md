## 一、UDP 与 TCP 的心智差异

**一句话**：UDP **无连接**，不建立「会话」；每个报文（**datagram**）独立，带目标地址，**保留报文边界**。

| | **TCP** | **UDP** |
|---|---------|---------|
| 连接 | `listen` / `accept` / `connect` | 无；`bind` 即可收发包 |
| 语义 | **字节流**（需自己分帧） | **独立 datagram** |
| 可靠/有序 | 协议保证 | 不保证 |
| 典型 API | `read` / `write` | **`recv_from` / `send_to`** |
| 每包地址 | 连接建立后隐含 | 每包显式 **`SocketAddr`** |

## 二、`UdpSocket` 基本用法

### 服务端

```rust
use std::net::UdpSocket;

let socket = UdpSocket::bind("127.0.0.1:9000")?;
let mut buf = [0u8; 1500];
let (len, src) = socket.recv_from(&mut buf)?;
socket.send_to(&buf[..len], src)?;
```

### 客户端

客户端通常也 **`bind`** 本地地址；端口 **`0`** 表示由 OS 分配临时端口（ephemeral）：

```rust
let socket = UdpSocket::bind("127.0.0.1:0")?;
socket.send_to(b"hello", "127.0.0.1:9000")?;
let mut buf = [0u8; 1500];
let (len, src) = socket.recv_from(&mut buf)?;
```

### API 精读

| 方法 | 返回 | 含义 |
|------|------|------|
| **`recv_from(&mut buf)`** | `(usize, SocketAddr)` | 收到字节数 + **来源地址** |
| **`send_to(buf, addr)`** | `usize` | 向指定地址发一个 datagram |
| **`bind`** | — | 绑定本地 IP:端口，开始收包 |

缓冲区大小常取 **1500** 左右（以太网 MTU 量级）；更大报文会被 IP 层分片，应用层仍按一次 `recv_from` 收一整 UDP 载荷（在未超 MTU 时）。

## 四、何时选 UDP

| 适合 UDP | 适合 TCP |
|----------|----------|
| 可丢包、可乱序（实时音视频、游戏状态） | 必须可靠交付（HTTP、文件、RPC） |
| 请求—响应短报文（DNS、DHCP） | 长连接、流式传输 |
| 局域网广播/多播（见 [3.2.1](./3.2.1-udp-multicasting.md)） | 需要背压与流量控制 |

## 五、新手易错

1. **用 `read`/`write` 思维写 UDP**：必须用 **`recv_from`/`send_to`**（或 `connect` 后用 `send`/`recv`，本书先不展开）。
2. **忘记 bind 客户端**：未 bind 时部分平台行为不一致；`127.0.0.1:0` 最稳妥。
3. **假设 `send_to` 一次必达**：UDP 无重传；丢包要应用层处理。
4. **报文大于 MTU**：可能 IP 分片，增加丢包概率；大 payload 用 TCP 或应用层分片。
