## 一、为什么需要类型化的地址

`bind` / `connect` / `send_to` 都要 **IP + 端口**。`std::net` 用枚举区分 **IPv4 / IPv6**，避免字符串拼错、便于校验「是否回环、是否公网」等。

---

## 二、`IpAddr` 与 `SocketAddr`

| 类型 | 含义 |
|------|------|
| **`IpAddr`** | `V4(Ipv4Addr)` \| `V6(Ipv6Addr)` |
| **`SocketAddr`** | `SocketAddrV4` \| `SocketAddrV6`：**IP + 端口** |

### 常用方法

| 方法 | 含义 |
|------|------|
| `is_loopback()` | `127.0.0.1` / `::1` |
| `is_global()` | 是否全球单播（非公网/组播等特殊用途） |
| `is_ipv4()` / `is_ipv6()` | 协议族分支 |

### 解析字符串

```rust
use std::net::SocketAddr;

let addr: SocketAddr = "192.168.1.10:8080".parse()?;
let loopback: SocketAddr = "127.0.0.1:8080".parse()?;
println!("loopback? {}", loopback.ip().is_loopback());
```

```rust
use std::net::IpAddr;

let ip: IpAddr = "::1".parse()?;
println!("ipv6? {}", ip.is_ipv6());
```

## 三、DNS 与 `ToSocketAddrs`

### 稳定用法（推荐）

`TcpStream::connect("example.com:80")?` 接受任何实现 **`ToSocketAddrs`** 的类型：字面量、`SocketAddr`、**(主机名, 端口)** 等；标准库在底层解析 DNS（通常 **A / AAAA**）。

```rust
use std::net::TcpStream;

let _ = TcpStream::connect("example.com:443")?;
let _ = TcpStream::connect(("127.0.0.1", 8080))?;
```

### 书中 `lookup_host`

历史上 **`std::net::lookup_host`** 曾为 nightly API，底层类似 C 的 **`getaddrinfo`**。日常开发：

- **连上去就行** → `connect("host:port")`
- **要 MX / TXT / NS 或异步解析** → [3.4 hickory-dns / trust-dns](./3.4-related-crates.md)

---

## 四、IPv4 专用类型（多播等）

| 类型 | 用途 |
|------|------|
| **`Ipv4Addr`** | 四个 `u8`；`UNSPECIFIED` = `0.0.0.0` |
| **`SocketAddrV4`** | IPv4 + 端口；[3.2.1 多播](./3.2.1-udp-multicasting.md) 绑定 `0.0.0.0:PORT` |

---

## 五、对照：字面量 vs 解析 vs DNS

| 写法 | 何时用 |
|------|--------|
| `"127.0.0.1:8080".parse()` | 配置里已知 IP |
| `connect("domain:443")` | 生产客户端 |
| 第三方 DNS crate | 自定义解析逻辑、非 A 记录 |

---

## 六、新手易错

1. **忘记端口**：`SocketAddr` 必须带 `:port`。
2. **IPv6 地址要方括号**：如 `[::1]:8080`（`connect` 字符串形式）。
3. **把 `0.0.0.0` 当目标地址**：它是「本机所有网卡」绑定用，不是发给某台远程机器。
4. **DNS 每次 connect 都可能解析**：高频场景可缓存 `SocketAddr`。
