## 一、一句话回顾

用 **`std::net`** 写阻塞式 **TCP/UDP** 客户端与服务器；TCP 用 **`TcpListener`/`TcpStream` + 多线程**，UDP 用 **`UdpSocket` + recv_from/send_to**；地址与 DNS 工具在 **`IpAddr`/`SocketAddr`**；更强能力交给 **`mio`、ipnetwork、libpnet、hickory-dns** 等 crate，最终演进到 **[Tokio](../stage07_tokio_async_net/)**。

---

## 二、核心术语

| 术语 | 含义 |
|------|------|
| **`TcpListener`** | 监听端口，`accept` 得到已连接流 |
| **`TcpStream`** | TCP 字节流；`Read`/`Write` |
| **`UdpSocket`** | 无连接 datagram；`recv_from`/`send_to` |
| **Datagram** | UDP 报文边界；与 TCP 流式相对 |
| **`SocketAddr`** | IP + 端口 |
| **`ToSocketAddrs`** | `connect` 可接受主机名或地址 |
| **Multicast** | 一对多 UDP；`join_multicast_v4`（[3.2.1](../3.2.1-udp-multicasting/3.2.1-udp-multicasting.md)） |
| **`mio`** | 非阻塞 poll；Tokio 底层 |

## 四、TCP vs UDP（本章级）

| | **TCP** | **UDP** |
|---|---------|---------|
| 连接 | 有 | 无 |
| 数据单位 | 流 | Datagram |
| 可靠性 | 高 | 尽力而为 |
| 本书 demo 端口 | `38081` | `39090` / 多播 `39191` |
| 并发 demo | 每连接 `thread::spawn` | 单线程 loop 即可 |

## 七、新手易错（全章）

1. **TCP 一次 read ≠ 一条消息** → 分帧协议。
2. **UDP 无连接 ≠ 不用 bind** → 客户端也建议 `bind("0.0.0.0:0")`。
3. **阻塞 server 线程数 = 连接数** → 连接多时用 mio/Tokio。
4. **多播 demo 在 Windows 上可能需额外权限** → 以 API 笔记为主。
