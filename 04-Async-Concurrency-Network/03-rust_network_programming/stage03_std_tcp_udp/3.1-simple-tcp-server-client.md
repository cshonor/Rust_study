## 一、本章在学什么

把 [Ch1](../../stage01_network_basic/notes/Ch01-客户端服务器网络简介-学习笔记.md) 的 **TCP 客户端—服务器** 模型落到 Rust 标准库 **`std::net`**：

- 服务端：`TcpListener` 监听 → `accept` 得到 `TcpStream`
- 客户端：`TcpStream::connect` 建立连接
- 并发：**每连接一个 OS 线程**（阻塞 I/O 的最小模型）

异步版见 [stage07 Tokio](../stage07_tokio_async_net/)。

## 二、TCP 服务器：`TcpListener`

**一句话**：`bind` 本地地址 → 进入 **LISTEN** → `incoming()` / `accept()` 拿到已连接的 **`TcpStream`**。

| 步骤 | API / 做法 |
|------|------------|
| 监听 | `TcpListener::bind("127.0.0.1:PORT")?` |
| 接受连接 | `listener.incoming()` 迭代，或单次 `accept()` |
| 得到 | **`TcpStream`**（已 ESTABLISHED 的连接） |
| 并发 | 每接受一个客户端 → **`thread::spawn(move \|\| …)`** 处理 |

```rust
use std::io::{Read, Write};
use std::net::{TcpListener, TcpStream};
use std::thread;

fn handle_client(mut stream: TcpStream) -> std::io::Result<()> {
    let mut buf = [0u8; 1024];
    let n = stream.read(&mut buf)?;
    stream.write_all(&buf[..n])?;
    Ok(())
}

fn main() -> std::io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    for stream in listener.incoming() {
        let stream = stream?;
        thread::spawn(move || {
            let _ = handle_client(stream);
        });
    }
    Ok(())
}
```

### 要点

1. **`incoming()`** 等价于循环 `accept()`；每次成功返回一条新连接。
2. 工作线程里必须 **`move`** 拿走 `TcpStream` 所有权（同 [JoinHandle / spawn](../../01-atomic/Chapter-01-Rust-Concurrency-Basics/1.1-threads-in-rust/1.1-threads-in-rust.md)）。
3. 本书 demo 为**回显**：读多少写回多少；生产还需优雅退出、连接上限、日志、`join`/线程池等。

仓库对应：[3.1-simple-tcp-server-client-echo-server-demo.rs](./3.1-simple-tcp-server-client/3.1-simple-tcp-server-client-echo-server-demo.rs) · 监听 **`127.0.0.1:38081`**

## 三、TCP 客户端：`TcpStream`

**一句话**：`connect` 完成三次握手 → 对 `TcpStream`  impl **`Read` + `Write`**。

| 步骤 | API |
|------|-----|
| 连接 | `TcpStream::connect("127.0.0.1:8080")?`（域名也可，见 [3.3](./3.3-std-net-utilities.md)） |
| 写 | `write_all`（保证写满缓冲区） |
| 读 | `read`（可能短读；流式协议需自己组帧） |
| 缓冲 | **`BufReader` / `BufWriter`** 减少 syscall，按行读更方便 |

```rust
use std::io::{Read, Write};
use std::net::TcpStream;

fn main() -> std::io::Result<()> {
    let mut stream = TcpStream::connect("127.0.0.1:8080")?;
    stream.write_all(b"hello")?;
    let mut buf = [0u8; 1024];
    let n = stream.read(&mut buf)?;
    println!("{}", String::from_utf8_lossy(&buf[..n]));
    Ok(())
}
```

仓库对应：[3.1-simple-tcp-server-client-echo-client-demo.rs](./3.1-simple-tcp-server-client/3.1-simple-tcp-server-client-echo-client-demo.rs)

## 四、读超时：`set_read_timeout`

阻塞 `read` 在对端无响应时可能**永远卡住**。

```rust
use std::net::TcpStream;
use std::time::Duration;

let stream = TcpStream::connect("127.0.0.1:8080")?;
stream.set_read_timeout(Some(Duration::from_secs(5)))?;
// 超时后 read 返回 Err，kind 常为 TimedOut
```

| API | 作用 |
|-----|------|
| `set_read_timeout` | 单次 `read` 最长阻塞时间 |
| `set_write_timeout` | 单次 `write` 最长阻塞时间 |

> **连接阶段**超时：`connect` 本身无标准库超时参数，需非阻塞 `connect` + `poll` 或第三方 crate；书上以**已连接流上的读超时**为主。

## 五、对照：阻塞多线程 vs 异步

| | **本章（`std::net` + `thread::spawn`）** | **[stage07 Tokio](../stage07_tokio_async_net/)** |
|---|------------------------------------------|--------------------------------------------------|
| I/O | 阻塞；一线程等一条连接/读 | 非阻塞 + reactor；单线程可管大量连接 |
| 模型 | 简单直观 | 需 `async`/`.await` |
| 适用 | 学习、连接数少 | 高并发网络服务 |

---

## 六、新手易错

1. **TCP 是字节流**：一次 `read` 不一定等于一条「消息」；要换行、长度前缀或 HTTP 等分帧。
2. **先起 server 再起 client**；端口被占用时 `bind` 失败。
3. **`write` vs `write_all`**：前者可能只写出部分字节。
4. **spawn 后不 join**：demo 里 server 主线程卡在 `incoming()` 循环，子线程处理连接；生产要管线程生命周期。
