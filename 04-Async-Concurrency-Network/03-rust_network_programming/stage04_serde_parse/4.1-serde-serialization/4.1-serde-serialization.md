## 4.1 Serialization and deserialization using Serde（使用 Serde 序列化与反序列化）

## 一、本节在学什么

[Ch3](../stage03_std_tcp_udp/) 用 `TcpStream` 传的是**原始字节流**；本节解决「字节 ↔ 应用层结构体」：

- **序列化**：`Point3D` 等结构 → JSON / 二进制 `Vec<u8>`
- **反序列化**：收到的字节 → 还原为 Rust 类型
- **Serde** 是 Rust 生态里做这件事的**事实标准**；网络服务里 **JSON**（人机可读）和**二进制**（体积小）最常见

下一节 [4.1.1](../4.1.1-custom-serde/4.1.1-custom-serde.md) 讲字段名不一致、手写 `Visitor` 等进阶；[4.2](../4.2-parsing-textual/4.2-parsing-textual.md) / [4.3](../4.3-parsing-binary/4.3-parsing-binary.md) 讲不用 Serde 的协议解析。

## 二、Serde 是什么

**Serde** = **Ser**ialize / **De**serialize：一套 trait + 格式适配 crate，同一套 `#[derive]` 可换输出格式。

| 格式 | 常见 crate | 网络场景 |
|------|------------|----------|
| JSON | `serde_json` | REST API、调试、人机可读消息 |
| MessagePack / Bincode | `rmp-serde`、`bincode` | 服务间 RPC、游戏同步 |
| YAML / TOML / CSV | `serde_yaml`、`toml`、`csv` | 配置、批处理（较少直接上 wire） |

本 stage 的 `Cargo.toml` 已包含 `serde`（`derive`）与 `serde_json`。

---

## 三、基本用法：`#[derive(Serialize, Deserialize)]`

**一句话**：在结构体上派生两个 trait，即可 `to_vec` / `from_slice` 往返。

```toml
# Cargo.toml（节选）
[dependencies]
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Serialize, Deserialize, PartialEq)]
struct Point3D {
    x: f64,
    y: f64,
    z: f64,
}

fn main() -> serde_json::Result<()> {
    let p = Point3D { x: 1.0, y: 2.0, z: 2.0 };
    let json = serde_json::to_vec(&p)?;              // 序列化 → Vec<u8>
    let back: Point3D = serde_json::from_slice(&json)?; // 反序列化
    assert_eq!(p, back);
    Ok(())
}
```

### 常用 API

| 方向 | 函数 | 输出 / 输入 |
|------|------|-------------|
| 序列化 | `serde_json::to_vec` | `Vec<u8>`（直接 `write_all`） |
| 序列化 | `serde_json::to_string` | `String`（打印、日志） |
| 反序列化 | `serde_json::from_slice` | `&[u8]` |
| 反序列化 | `serde_json::from_str` | `&str` |

过程宏自动生成实现，**无需手写** `impl Serialize for Point3D`。

仓库对应：[4.1-serde-serialization-point3d-demo.rs](./code/4.1-serde-serialization-point3d-demo.rs) — 往返 JSON 并计算到原点距离 √(x²+y²+z²)。

## 四、配合 TCP：书中典型流程

与 [3.1 TcpStream](../stage03_std_tcp_udp/3.1-simple-tcp-server-client/3.1-simple-tcp-server-client.md) 组合时的消息流：

```text
客户端                          服务器
  读控制台输入 (x,y,z)
  构建 Point3D
  serde_json::to_vec → write_all
                              read → from_slice → Point3D
                              计算距离 √(x²+y²+z²)
                              序列化结果 → write_all
  read → 显示距离
```

Serde 只负责「**这一段**字节 ↔ 结构体」；**如何把多条消息从字节流里切出来**要在 I/O 层设计（见下一节）。

---

## 五、TCP 粘包与分帧

`TcpStream` 是**字节流**，一次 `read` **不一定**对应一条完整 JSON。

| 分帧方式 | 做法 | 适用 |
|----------|------|------|
| **长度前缀** | 先读 4 字节 `len`（常大端 u32），再读 `len` 字节 body | 二进制友好、通用 RPC |
| **换行分隔** | `read_line` / `\n` 结尾，每行一条 JSON | 简单 CLI、日志流 |
| **HTTP 式** | `Content-Length` 头 + body | 应用层协议（见 [stage06](../stage06_http_programming/)） |

```rust
use serde::Serialize;
use std::io::Write;
use std::net::TcpStream;

// 示例：长度前缀（u32 大端）+ JSON body
fn send_json(stream: &mut TcpStream, msg: &impl Serialize) -> std::io::Result<()> {
    let body = serde_json::to_vec(msg)?;
    let len = (body.len() as u32).to_be_bytes();
    stream.write_all(&len)?;
    stream.write_all(&body)?;
    Ok(())
}
```

完整 `nom` 解析长度前缀见 [4.3](../4.3-parsing-binary/4.3-parsing-binary.md) 的 `demo_4_3_binary_length_prefix`。

## 六、对照：Serde vs 手写解析

| | **本节（Serde + JSON）** | **[4.2 nom 文本](../4.2-parsing-textual/4.2-parsing-textual.md)** | **[4.3 nom 二进制](../4.3-parsing-binary/4.3-parsing-binary.md)** |
|---|--------------------------|-----------------------------------------------|------------------------------------------------|
| 输入 | 已知 schema 的一段字节 | HTTP 行等文本切分 | 定长/位域协议头 |
| 代码量 | `derive` 即可 | 组合子拼解析器 | `be_u32`、`take`、`bits!` |
| 跨语言 | JSON 易对接 | 文本协议通用 | 需文档约定布局 |

三者常**组合**：固定头用 `nom`，负载用 Serde。

## 七、新手易错

1. **只序列化不设计分帧**：`from_slice` 在粘包/半包时会失败或解析错对象；必须先定「一条消息」边界。
2. **`write` vs `write_all`**：JSON 体要用 `write_all` 保证写满（同 [Ch3](../stage03_std_tcp_udp/3.1-simple-tcp-server-client/3.1-simple-tcp-server-client.md)）。
3. **浮点与 JSON**：`f64` 序列化可能有精度展示差异；金融场景考虑 `Decimal` 或整数分。
4. **字段名与协议不一致**：优先 `#[serde(rename = "...")]`（见 [4.1.1](../4.1.1-custom-serde/4.1.1-custom-serde.md)），再考虑手写 trait。
5. **`Result` 传播**：`serde_json::Result` 与 `std::io::Result` 不同，可用 `?` 或 `map_err` 统一错误类型。
