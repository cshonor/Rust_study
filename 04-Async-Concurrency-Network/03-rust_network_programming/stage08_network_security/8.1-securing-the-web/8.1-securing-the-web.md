## 一、本节在学什么

在 TCP/HTTP 之上理解 **TLS/HTTPS** 要解决的两件事：

| 目标 | 威胁 | 手段 |
|------|------|------|
| **机密性** | 窃听、中间人读明文 | 对称加密（会话密钥加密数据） |
| **身份与完整性** | 冒充服务器、篡改 | **证书** + 签名 + 握手认证 |

```text
HTTP  →  明文，易被嗅探
HTTPS →  HTTP over TLS（仍从 Ch3 TCP 三次握手开始）
```

---

## 二、TLS 握手（简化）

```text
1. TCP connect
2. TLS ClientHello / ServerHello — 版本、密码套件、随机数
3. 服务器出示 X.509 证书链
4. 客户端用信任 CA 根验证链 → 确认域名/组织
5. 协商会话密钥（常含 ECDHE 前向保密）
6. HTTP 在加密通道内传输
```

| 术语 | 含义 |
|------|------|
| **CA** | 证书颁发机构，用私钥给站点证书签名 |
| **证书链** | 站点 ← 中间 CA ← 根 CA（根预装系统/浏览器） |
| **SAN** | Subject Alternative Name，须包含访问的域名 |
| **密码套件** | 协商的加密/ MAC / 密钥交换算法组合 |

**最关键一步**：验证服务器身份 — 依赖 **CA 签发的数字证书**，而非仅「连上某个 IP」。

## 三、Rust 侧：HTTPS 客户端

**`rustls`**：纯 Rust TLS，不链接 OpenSSL；配合 **`webpki-roots`** 校验证书。

**`reqwest`**：`rustls-tls` feature 一键 HTTPS GET（本 stage Cargo 已启用）。

```rust
fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = reqwest::blocking::Client::new();
    let resp = client.get("https://httpbin.org/get").send()?;
    println!("HTTPS status: {}", resp.status());
    Ok(())
}
```

| 组件 | 作用 |
|------|------|
| **`rustls`** | TLS 协议实现 |
| **`webpki`** | 证书路径校验、主机名匹配 |
| **`reqwest`** | HTTP 客户端，封装 TLS |

仓库对应：[8.1-securing-the-web-https-get-demo.rs](./code/8.1-securing-the-web-https-get-demo.rs)（需外网）

## 四、TLS 相关库速查（原书 + 现代）

| 库 | 角色 |
|----|------|
| **`rustls`** | 纯 Rust TLS；适合 HTTPS 客户端/服务端 |
| **`rust-openssl`** | OpenSSL 绑定；生成 RSA、X.509、自签名 |
| **`tokio-tls`**（书中） | Tokio 异步 TLS 包装 → 现代 **`tokio-rustls`** |
| **`native-tls`** | 系统 TLS；`tokio-native-tls` 底层 |

**现代常见组合**：`tokio-rustls` + `rustls` · 或 `tokio-native-tls` + `native-tls`（偏系统栈）。

## 五、新手易错

1. **开发环境自签名**：客户端默认**不信任**；仅 dev 可关闭校验或 mkcert，**生产禁止**。
2. **证书域名不匹配**：IP 访问但证书 SAN 只有域名 → 校验失败。
3. **过期证书**：Let's Encrypt 约 90 天，需 ACME 续期（见 [8.1.1](../8.1.1-letsencrypt-using-rust/8.1.1-letsencrypt-using-rust.md)）。
4. **HTTP 与 HTTPS 端口**：443 是惯例，非强制；本地测试常用 8443。
5. **明文 HTTP 传密码**：必须 HTTPS 或 TLS 隧道。
