## 5.3 Introduction to FTP and TFTP（FTP 与 TFTP 简介）

## 一、本节在学什么

文件传输有两类经典应用协议：

- **FTP**：**TCP**、**双连接**（控制 + 数据）、有登录认证，年代久但有助于理解「命令通道 vs 数据通道」
- **TFTP**：**UDP**、极简、**无认证**，用于嵌入式、**PXE** 网络启动

书中分别用 **`rust-ftp`**（客户端）与 **`tftp_server`**（本机 UDP 服务）演示；本章仓库 demo 为**协议步骤速查**（无 live 服务器），完整联调按书在独立项目练习。

---

## 二、FTP（文件传输协议）

**一句话**：**控制连接**上发文本命令，**数据连接**上传/下载文件；登录后执行 `LIST`、`RETR`、`STOR` 等。

| 项目 | 说明 |
|------|------|
| 模型 | **基于文本**的命令 / 响应 |
| 控制连接 | 常 **TCP 21**，全程保持 |
| 数据连接 | 传文件与目录列表；**主动（PORT）** 或 **被动（PASV）** 模式 |
| 能力 | 登录、列目录、上传/下载、改名、断点续传（扩展） |
| Rust（书中） | **`rust-ftp`** 客户端 |

### 双连接示意

```text
客户端                          FTP 服务器
   │──── TCP 21 控制连接 ────────────│  USER / PASS / PWD / CWD ...
   │                                 │
   │──── TCP 数据连接（另开端口）────│  LIST / RETR / STOR 的载荷
```

### 书中客户端流程（`rust-ftp`）

```text
连接 ftp.example.com:21
→ 登录（用户名 / 密码）
→ 打印当前目录（PWD / LIST）
→ put：用 std::io::Cursor 作内存缓冲区上传文件
```

```rust
// 概念流程（crate API 以文档为准）
// FtpStream::connect("ftp.example.com")?
// ftp.login("user", "pass")?;
// ftp.cwd("/remote/path")?;
// ftp.put_file("name.txt", &mut cursor)?;
```

### 主动 vs 被动

| 模式 | 谁监听数据端口 | NAT/防火墙 |
|------|----------------|------------|
| **主动 PORT** | 服务端连回客户端告知的 IP:端口 | 客户端后侧 NAT 常失败 |
| **被动 PASV** | 服务端告知 IP:端口，客户端去连 | **现代环境更常用** |

**特点**：协议年代久，防火墙/NAT 下 **PASV** 仍可能踩坑；新项目更常用 **SFTP**、**S3**、**HTTPS** 传文件，但读 FTP 有助于理解「控制通道 vs 数据通道」分离设计。

## 三、TFTP（简单文件传输协议）

**一句话**：在 **UDP** 上用固定块大小读写文件，无登录，适合引导与固件场景。

| 对比项 | FTP | TFTP |
|--------|-----|------|
| 传输 | **TCP** | **UDP** |
| 连接 | 控制 + 数据双 TCP | 无连接；请求 / 数据块 / ACK 报文 |
| 认证 | 用户名 / 密码 | **无** |
| 复杂度 | 命令集丰富 | 极简（RRQ/WRQ、DATA、ACK、ERROR） |
| 典型场景 | 传统文件交换 | **嵌入式**、**PXE/BOOTP** 启动、设备固件 |
| Rust（书中） | `rust-ftp` | **`tftp_server`** |

```text
TFTP 客户端  --UDP-->  tftp_server（Rust）
              读请求 RRQ / 数据块 DATA / 确认 ACK
```

书中用 **`tftp_server`** 在本机绑定 UDP 端口，向 TFTP 客户端提供文件下载。

**与 Ch3**：直接对应 [`UdpSocket`](../stage03_std_tcp_udp/3.2-simple-udp-server-client.md) 上的**无连接、有报文边界**模型；比 FTP 更接近「最小文件传输」。

## 四、本章 Demo：FTP vs TFTP 步骤速查

无 live FTP/TFTP 服务器；运行 bin 打印书中典型步骤与 crate 名，便于对照原书阅读。

```text
FTP (TCP):
  1. connect :21 control
  2. USER / PASS
  3. PASV or PORT → data channel
  4. STOR/RETR on data connection

TFTP (UDP):
  1. RRQ/WRQ datagram
  2. DATA blocks + ACK
  3. no auth — embedded / PXE

Crates: rust-ftp, tftp_server — use per book in separate projects.
```

## 五、协议形态对照（FTP / TFTP 在本章位置）

| 协议 | 传输 | 编码 | Rust 生态（书中） | 典型场景 |
|------|------|------|-------------------|----------|
| **FTP** | TCP（双连接） | 文本命令 | `rust-ftp` | 传统文件交换、理解双通道 |
| **TFTP** | **UDP** | 二进制块 | `tftp_server` | PXE、嵌入式固件 |

与 [5.1 RPC](./5.1-introduction-to-rpc.md)、[5.2 SMTP](./5.2-introduction-to-smtp.md) 并列：同属**应用层**，在 TCP/UDP 之上定义**业务语义**，而非自己发明字节偏移。

## 六、新手易错

1. **只开一个 TCP 连接就想传 FTP 文件**：文件内容走**数据连接**，不是控制连接上的 `DATA` 字符串。
2. **NAT 后 FTP 列表为空或卡住**：优先试 **PASV**；检查服务端返回的被动 IP 是否可达。
3. **把 TFTP 当 FTP 的「简化 TCP 版」**：TFTP 是 **UDP**，无连接，无登录，语义完全不同。
4. **在无隔离网络外开 TFTP**：无认证，仅适合受控内网或引导网段。
5. **本章 demo 不联网**：不代表已会 `rust-ftp` API；需按书在独立项目写几行 `put` / `get`。
