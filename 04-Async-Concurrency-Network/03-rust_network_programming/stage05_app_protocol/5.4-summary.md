## 一、本章在学什么（回顾）

[Ch03](../stage03_std_tcp_udp/) / [Ch04](../stage04_serde_parse/) 解决「字节怎么传、怎么序列化」；**第 5 章**把视角升到**应用层**：在 TCP/UDP 之上，用 Rust 与生态库实现常见**业务协议**：

| 书 § | 主题 | 核心 takeaway |
|------|------|----------------|
| [5.1](./5.1-introduction-to-rpc.md) | RPC / gRPC | IDL（`.proto`）→ 生成 stub/skeleton；默认 HTTP/2 + protobuf |
| [5.2](./5.2-introduction-to-smtp.md) | SMTP | 文本命令发信；`lettre` 组装 MIME + TLS 配置 |
| [5.3](./5.3-introduction-to-ftp-and-tftp.md) | FTP / TFTP | FTP 双 TCP 通道；TFTP 无认证 UDP 块传输 |
| 5.4 | 总结 | 协议对照、自检、后续 stage |

不再手写「第 3 字节是命令号」，而是**按协议或 IDL** 分工：你写业务 impl，库处理帧与状态机。

## 二、协议形态总表

| 协议 | 传输 | 编码 | Rust 生态（书中） | 现代常见替代 | 典型场景 |
|------|------|------|-------------------|--------------|----------|
| **gRPC** | TCP + HTTP/2 | protobuf | `protoc-rust-grpc` / **`tonic`** | 同左 | 微服务 API、流式数据 |
| **SMTP** | TCP | 文本 + MIME | **`lettre`** | 同左 + 事务邮件 API | 告警、通知邮件 |
| **FTP** | TCP（双连接） | 文本命令 | **`rust-ftp`** | SFTP、S3、HTTPS | 传统文件交换 |
| **TFTP** | **UDP** | 二进制块 | **`tftp_server`** | 同左（引导场景） | PXE、嵌入式固件 |

```text
传输层（Ch03）          应用层（Ch05）
────────────────────────────────────────
TcpStream / UdpSocket  →  RPC / SMTP / FTP / TFTP
Serde 字节 ↔ 结构体    →  协议库 + IDL / MIME / 命令集
```
