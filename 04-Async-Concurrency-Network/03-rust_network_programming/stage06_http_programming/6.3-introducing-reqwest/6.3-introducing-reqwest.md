## 一、本节在学什么

用 **reqwest** 从 Rust 程序发起 **HTTP 客户端**请求 —— 调第三方 API、微服务间 HTTP、脚本抓取等。API 设计接近 Python **`requests`**，并与 **Serde** 无缝衔接 JSON。

**一句话**：写服务看 Hyper/Rocket；**调 API 优先 reqwest**（本章最常用的一节）。

---

## 二、reqwest 简介

| 能力 | 说明 |
|------|------|
| 方法 | `get`、`post`、自定义头、超时、代理 |
| 体 | JSON、表单、multipart |
| TLS | feature：`rustls-tls` / `native-tls` |
| 模式 | **阻塞**（`blocking`）与 **异步**（`async` + Tokio） |

章根 [Cargo.toml](./Cargo.toml) 配置（摘录）：

```toml
reqwest = { version = "0.12", default-features = false, features = ["blocking", "rustls-tls"] }
```

| 选择 | 含义 |
|------|------|
| `default-features = false` | 按需启用 TLS，减小依赖 |
| `blocking` | CLI / 脚本用同步 API |
| `rustls-tls` | 纯 Rust TLS，免系统 OpenSSL |

---

## 三、同步客户端（阻塞）

适合 **CLI、一次性脚本、测试**；阻塞当前线程直到响应返回。

```rust
use reqwest::blocking::Client;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = Client::new();
    let resp = client.get("https://httpbin.org/get").send()?;
    println!("status: {}", resp.status());
    let body = resp.text()?;
    Ok(())
}
```

| 步骤 | API |
|------|-----|
| 建客户端 | `Client::new()`（可复用，连接池在内部） |
| 发请求 | `.get(url).send()?` |
| 读体 | `.text()?` / `.bytes()?` |
| 状态码 | `resp.status()` |

仓库 demo 打印状态码与 body 前 200 字符；需**可访问外网**。

## 四、JSON + Serde（与 Ch04 一致）

**POST 序列化 + 响应反序列化** 用同一套 `Serialize` / `Deserialize`：

```rust
use serde::{Deserialize, Serialize};

#[derive(Serialize)]
struct Payload { name: String }

#[derive(Deserialize)]
struct ApiResult { ok: bool }

let client = reqwest::blocking::Client::new();
let res: ApiResult = client
    .post("https://api.example.com/items")
    .json(&Payload { name: "a".into() })
    .send()?
    .json()?;
```

| 链式调用 | 作用 |
|----------|------|
| `.json(&payload)` | 序列化 body，设 `Content-Type: application/json` |
| `.send()?` | 发出请求 |
| `.json()?` | 将响应体反序列化为 `ApiResult` |

无需手写 header 拼接与 body 字符串 —— 与 [Ch04](../stage04_serde_parse/) 边界一致。

## 五、异步客户端（Tokio）

书中从 `tokio_core` + `Future` 链演进；现代写法：

```rust
#[tokio::main]
async fn main() -> Result<(), reqwest::Error> {
    let body = reqwest::get("https://httpbin.org/get")
        .await?
        .text()
        .await?;
    Ok(())
}
```

| | **阻塞 `blocking`** | **异步 `async`** |
|---|---------------------|------------------|
| 运行时 | 无需 Tokio（blocking feature 自带） | 需 `#[tokio::main]` 等 |
| 并发多 URL | 多线程或顺序阻塞 | `join!` / `try_join!` 单线程并发 I/O |
| 学习顺序 | 本章 demo 先用这个 | 与 [stage07](../stage07_tokio_async_net/) 一起练 |

### 仓库交叉

- 根 workspace / `async_tokio`：**`ch01_reqwest_join`** 等 — 并发发多个 HTTP 请求，可与本节 API 对照。

## 六、常用配置（概念）

```rust
// 概念
let client = reqwest::blocking::Client::builder()
    .timeout(std::time::Duration::from_secs(10))
    .user_agent("my-app/1.0")
    .build()?;
```

| 配置 | 用途 |
|------|------|
| `timeout` | 避免对端无响应时永久卡住（对照 Ch3 `set_read_timeout`） |
| `user_agent` | 部分 API 要求标识客户端 |
| 代理 | `Proxy::http(...)` 等，按环境变量或显式设置 |

HTTPS 证书校验由 TLS feature 处理；自签证书场景见 stage08 / reqwest 文档中的 danger 选项（仅开发环境）。

## 七、reqwest vs 其它客户端

| | **reqwest** | **ureq** | **hyper 低级 Client** |
|---|-------------|----------|------------------------|
| API 层级 | 高，接近 requests | 高，偏同步 | 低，需自己拼 |
| 异步 | 支持 | 基本同步 | 原生 async |
| 本书 | §6.3 主线 | 未重点讲 | 与 §6.1 服务端配套 |

## 八、新手易错

1. **忘记 `send()`**：构建 `get()` 链不会发请求，必须 `.send()?`。
2. **阻塞 demo 跑在 async runtime 里**：混用会卡住 executor；二选一或查文档 `block_in_place` 模式。
3. **外网 demo 失败**：`httpbin.org` 不可达时检查代理/防火墙，非代码问题。
4. **大响应一次 `.text()`**：超大 body 占内存；可用 `.bytes()` 流式处理或限制大小。
5. **TLS feature 未开**：`https://` URL 需启用 `rustls-tls` 或 `native-tls`。
