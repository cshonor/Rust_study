## 1.1.2.1 JoinHandle 是什么

> 书 §1.1（2.1）· 索引：[1.1.2](./1.1.2-join-handle.md) · 前：[1.1.1 spawn/join](./1.1.1-spawn-join-basics.md) · 后：[1.1.2.2 spawn/join 类型](./1.1.2.2-spawn-join-types.md)

---

**一句话结论**：`std::thread::spawn(闭包)` → 返回 **`JoinHandle<T>`**，`T` = 子线程闭包的**返回值类型**。

```rust
use std::thread;

// 闭包无 return、只有语句 → T = () → JoinHandle<()>
let h: thread::JoinHandle<()> = thread::spawn(|| {
    println!("hello from thread");
});
let _ = h.join(); // Result<(), Box<dyn Any + Send + 'static>>
```

| 要点 | 说明 |
|------|------|
| **`T`** | 闭包 **`return` 的类型**；不写 return 时默认为 **`()`** |
| **所有权** | **`JoinHandle` 不可 `Clone`**；全进程唯一一份 handle，**只能被一个所有者** `move` 后 **`join(self)` 一次**（与 OS「只能 join 一次」一致） |

```rust
use std::thread;

fn main() {
    let h = thread::spawn(|| {
        println!("子线程正在运行");
        100 // 子线程返回值 → JoinHandle<i32>
    });

    let res = h.join().unwrap();
    println!("子线程返回值：{res}");
    println!("主线程收尾");
}
```

```text
子线程正在运行
子线程返回值：100
主线程收尾
```

## 一、JoinHandle 是什么

1. **持有对应子线程的 join 权**（句柄，非线程本体）；另有 **`.thread()`** 可查 `ThreadId`，**同步等待靠 `join()`**。
2. **`.join()`**：阻塞**当前**线程，等待子线程执行完毕，取回 **`Result<T, Box<dyn Any + Send + 'static>>`**：
   - 线程正常结束 → `Ok(T)`
   - 子线程 panic → `Err(Box<dyn Any + Send + 'static>)`（见 [1.1.2.3 Err 三层](./1.1.2.3-panic-box-dyn-any.md#err-类型三层拆解boxdyn-any--send--static)）
3. **不调用 `join`、也不 drop 这个变量**：`main` 结束时进程退出，**子线程会被强制终止**（见 [1.1.2.5 §四](./1.1.2.5-join-os-drop-pitfalls.md#四不用-join-会怎样)）。

> 补充：若 **`drop` 了 `JoinHandle` 且未 `join`**，子线程变为 **detached**，与 caller 脱钩；`main` 退出时同样可能来不及跑完。
