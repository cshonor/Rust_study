## 1.1.5 `where F: FnOnce() -> T` 语法拆解

> 书 §1.1（续）· FnOnce **语义** vs move：[1.1.4](./1.1.4-closure-traits-fnonce.md) · `spawn` 签名：[1.1.3](./1.1.3-send-static-spawn.md) · scope `'env`：[1.2.4](../1.2-scoped-threads/1.2.4-scoped-join-handle.md)

**本节重点**：`FnOnce<()>` 尖括号里的 **`()` 是入参元组（无参）**，不是「返回空」；**trait 选 Fn/FnMut/FnOnce、括号里参数、`->` 返回值是三个独立维度** → [§先建立地图](#先建立地图三个独立维度)；与 `thread::scope` 的 **`'env`**（生命周期）写法相似、含义完全不同 → [§六](#六trait-尖括号类型参数-vs-生命周期参数)。

## 标准写法

```rust
fn demo<T, F>() where F: FnOnce() -> T {}

// 与尖括号内约束完全等价：
fn demo<T, F: FnOnce() -> T>() {}
```

**`where`**：后置泛型约束；约束多、行长时**可读性更好**。

---

## 先建立地图：三个独立维度

把 `FnOnce(String) -> i32` 这类约束**拆开看** — **调用次数**、**参数签名**、**返回值** 是三个互不替代的维度：

```text
FnOnce ( String )  ->  i32
  │        │            │
  │        │            └─ ③ 返回值约束：闭包体算出的类型
  │        └─ ② 参数签名：调用时要传什么（空括号 = 无参）
  └─ ① 选哪个 Callable trait：决定「怎么调用 F、能调几次」
```

| 维度 | 写在哪儿 | 管什么 | 典型取值 |
|------|----------|--------|----------|
| **① Trait 选择** | `Fn` / `FnMut` / **`FnOnce`**（箭头**左边、括号前**） | **调用次数 & `self` 语义** | 见 [§三](#三三-trait-对比调用--捕获习惯) |
| **② 参数签名** | 括号 **`()` / `(String)` / `(i32, &str)`** | 闭包**入参类型**（与调用次数无关） | `()` = 无参；`(String,)` = 一个 `String` |
| **③ 返回值** | 箭头 **`-> T`** | 闭包**返回类型**（与调用次数无关） | `i32`、`()`、`String` … |

**① 三个 trait 各管「能调几次」**（与参数个数、返回类型正交）：

| Trait | 调用方式 | 次数 | 对捕获变量的典型影响 |
|-------|----------|------|----------------------|
| **`FnOnce`** | `call_once(self)` — **消耗**闭包 | **最多一次** | 常伴随 move 走捕获物；也可无捕获 |
| **`FnMut`** | `call_mut(&mut self)` | **多次** | 调用间可**修改**捕获变量 |
| **`Fn`** | `call(&self)` | **多次** | 共享 `&` 读捕获，**不修改** |

**② + ③ 组合示例**（trait 与 `(…) -> …` 自由搭配）：

```rust
// 无参 · 只能调一次 · 返回 i32
fn once_no_arg<F: FnOnce() -> i32>(f: F) { f(); }

// 一个 String 入参 · 只能调一次 · 返回 i32
fn once_with_arg<F: FnOnce(String) -> i32>(f: F) {
    f(String::from("tick"));
}

// 无参 · 可多次调 · 无返回值（Output = ()）
fn many_no_ret<F: Fn() -> ()>(mut f: F) {
    f();
    f();
}
```

> 语法糖 **`FnOnce(String) -> i32`** 展开为 **`FnOnce<(String,), Output = i32>`**（单参写成元组 `(String,)`）。不要写成 `FnOnce<String>` — 尖括号在 Callable trait 里留给**入参元组**，不是「泛型参数名 String」。

**`spawn` 钉死的是哪几维？**

```rust
pub fn spawn<F, T>(f: F) -> JoinHandle<T>
where
    F: FnOnce() -> T + Send + 'static,
    T: Send + 'static,
```

| 片段 | 钉住的维度 |
|------|------------|
| **`FnOnce`** | ① 子线程**拥有**闭包，**只执行一次** |
| **`()`** | ② **无参**入口（线程体不接受调用方再传参） |
| **`-> T`** | ③ 返回值类型 → `JoinHandle<T>` / `join` 的 `Ok(T)` |
| **`+ Send + 'static`** | 跨线程 move 与生命周期（**第四层**，不在箭头语法里）→ [1.1.3](./1.1.3-send-static-spawn.md) |

调用 `spawn` 时仍只写 `|| { … }`；上面四维约束写在 **API 的 where** 上，由编译器对闭包做校验 → [§何时必须写箭头](#何时必须写-fnonce---t何时可省略)。

## 一、逐个拆解

### 1. `F` — 泛型参数

`F` 代表**可调用对象**（闭包、函数指针、自定义 `impl FnOnce` 的类型）。名字随意：`Func`、`Closure` 均可。

### 2. `where F: Trait` — 约束

限定：**类型 `F` 必须实现**后面这个 trait（及关联类型绑定）。

### 3. `FnOnce() -> T` — 三大 Callable trait 之一

Rust 函数/闭包分 **`Fn` / `FnMut` / `FnOnce`**（子类型关系见 [1.1.4](./1.1.4-closure-traits-fnonce.md)）。

#### ① `FnOnce`

- **Trait 规则**：`call_once(self, …)` — 调用**消耗 F 自身**，故**最多一次**。
- **常见场景**：`move` 捕获 **owned** 变量时，编译器往往**只**实现 `FnOnce`（≠ trait 定义写了 move，见 1.1.5）。

```rust
let s = String::from("hello");
let closure = move || s;
closure();
// closure(); // 错误：F 已被消耗 / 捕获资源已 move 出
```

#### ② `()` — 参数列表（**不是**返回值）

`()` = **无参**闭包 — 对应 trait 形 **`FnOnce<()>`** 里尖括号的 **空元组入参**（[§六](#六trait-尖括号类型参数-vs-生命周期参数)）。  
双参数示例：`FnOnce(i32, &str) -> T` ↔ `FnOnce<(i32, &str), Output = T>`。

#### ③ `-> T` — 返回值

闭包执行结果类型为 **`T`**（`T` 为外层泛型）。与 `spawn` 的 **`JoinHandle<T>`** 中 `T` 同一角色。

## 何时必须写 `FnOnce() -> T`，何时可省略

| 你在写什么 | `-> T` 要不要写 | 说明 |
|------------|----------------|------|
| **`spawn` / 自定义泛型函数的 `where`** | **必须写** | 边界上要绑定 **`Output = T`**，并和 `JoinHandle<T>`、`Result<T, _>` 对齐 |
| **调用 `spawn` 时的闭包字面量** | **不用写** | `thread::spawn(|| …)` 即可；`T` 由闭包体 + `spawn` 签名**双向推导** |
| **`let x: impl FnOnce()` 等局部约束** | **常可省略** | 无参且上下文能推出 `()` 时，只写 **`FnOnce()`** 足够 |

```rust
// 定义侧（库作者）— 必须 FnOnce() -> T
fn run_bg<F, T>(f: F) -> std::thread::JoinHandle<T>
where
    F: FnOnce() -> T + Send + 'static,
    T: Send + 'static,
{
    std::thread::spawn(f)
}

// 调用侧 — 只写闭包
run_bg(|| println!("hi")); // T = ()

// 局部 impl Trait — 可省略 ->
let f: impl FnOnce() = || println!("hi");
```

**记忆**：**箭头写在「约束闭包类型的边界」上**（where / trait object / 函数泛型），**不写在「传进去的闭包字面量」上**。跨线程场景在边界上还要叠 **`Send + 'static`**，让编译器校验捕获物与生命周期 → [1.1.3](./1.1.3-send-static-spawn.md)。

对照 [1.1.2 §有/无返回值](./1.1.2-join-handle.md#有--无返回值签名不变t-自动推导)：`T = ()` 仍是推导结果，不是调用者手写出来的。

---

## 二、一句话翻译

```rust
where F: FnOnce() -> T
```

> **`F` 必须是无参、返回 `T`、且按 `FnOnce` 规则最多调用一次的可调用对象。**

`thread::spawn` 里在此基础上还要 **`F: Send + 'static`**（[1.1.3](./1.1.3-send-static-spawn.md)）。

## 三、三 trait 对比（调用 + 捕获习惯）

| Trait | 调用时 `self` | 典型调用次数 | 常见捕获方式（习惯，非 trait 字面） |
|-------|---------------|--------------|--------------------------------------|
| **`Fn`** | `&self` | 多次 | 共享引用 `&T` |
| **`FnMut`** | `&mut self` | 多次 | 可变引用 `&mut T` |
| **`FnOnce`** | **`self`（消耗）** | **一次** | 常伴随 **move owned**；也可无捕获（见 1.1.4） |

**兼容（自动 impl）**：

- `Fn` ⇒ 也有 `FnMut` + `FnOnce`
- `FnMut` ⇒ 也有 `FnOnce`
- **反过来不成立**

## 四、实战：`run_once`

```rust
fn run_once<T, F>(f: F) -> T
where
    F: FnOnce() -> T,
{
    f() // 消耗 F
}

fn main() {
    let msg = String::from("test");
    let c = move || msg;
    let res = run_once(c);
    println!("{res}");
    // c(); // 错误：c 已 move 进 run_once
}
```

与 **`jh.join()`** 类似：闭包/句柄**只消费一次**。

## 五、语法糖：`-> T` 与 `Output = T`

`FnOnce() -> T` 是糖。Trait 真实形状（简化）：

```rust
trait FnOnce<Args> {
    type Output;
    fn call_once(self, args: Args) -> Self::Output;
}
```

| 写法 | 展开 |
|------|------|
| `FnOnce() -> T` | `FnOnce<(), Output = T>` |
| `where F: FnOnce() -> T` | `where F: FnOnce<(), Output = T>` |

**`Output = T`**：把关联类型 **`Output`** 绑定为 **`T`** — 即你看到的「等于」语法，**不是** `F = F` 这种无意义相等。

#### `Args` 元组：入参列表（易混点）

| 闭包 / 函数 | Trait 展开 | 尖括号 `()` 含义 |
|-------------|-----------|-----------------|
| `\|\| 10` | `FnOnce<(), Output = i32>` | **无入参**（空元组） |
| `\|x\| {}` | `FnOnce<(i32,), Output = ()>` | 入参 1 个 `i32`；**`-> ()` 才是无返回值** |
| `\|x, y\| …` | `FnOnce<(i32, String), …>` | 多参 = 多元组 |

> **不要混淆**：`FnOnce<()>` 的 `<()>` = **调用时要传的参数整体（空）**；**不是**「返回 unit」。

高阶约束偶尔写成：

```rust
where
    F: FnOnce(),
    F::Output = T,
// 与 F: FnOnce() -> T 等价
```

## 六、Trait 尖括号：类型参数 vs 生命周期参数

### 6.1 基础共识：trait 后 `<…>` 都能放什么？

所有 trait 都可在名字后写尖括号，**给本次实现传「配置信息」**——与 `Vec<T>`、`impl<T> …` 同一套泛型逻辑：

| 尖括号内容 | 类别 | 示例 |
|-----------|------|------|
| **类型 / 元组** | 类型参数 | `FnOnce<()>`、`Iterator<Item = i32>` |
| **`'a` 带撇号** | 生命周期参数 | `impl<'env> …`、`Trait + 'a` |

**实现语法统一**：

```rust
impl Display for MyStruct {}              // 无参 trait

impl FnOnce<()> for MyFunc {}             // 类型参数：无入参 Callable（stable 下闭包自动 impl；手写见 [1.1.4 §三](./1.1.4-closure-traits-fnonce.md#场景-2手动实现-fnonce-的类型逻辑对stable-不行)）

impl<'env> SomeTrait<'env> for S {}       // 生命周期参数
```

### 6.2 `FnOnce<Args>`：`<()>` 是**调用入参**，不是返回值

```rust
trait FnOnce<Args> {
    type Output;
    fn call_once(self, args: Args) -> Self::Output;
}
```

| `Args` | 含义 | 对应闭包 |
|--------|------|----------|
| **`()`** | 无入参 | `\|\| 666` |
| **`(i32,)`** | 1 个 `i32` | `\|x\| …` |
| **`(i32, String)`** | 两个参数 | `\|x, y\| …` |

**为何入参要放进 trait 泛型？** Callable 是**静态分发**：编译期须确定 (1) 调用传什么类型、(2) 返回什么（`Output`）、(3) `call_once` 的 `args` 容器类型。

**`impl FnOnce<()> for MyFunc`**（nightly 语义）：实现「**无入参**版 FnOnce」— `call_once(self, _args: ())`，`_args` 是空元组，不含数据。

完整对照（nightly 手写示例，逻辑演示）：

```rust
struct FuncObj;
impl FnOnce<()> for FuncObj {
    type Output = i32;
    fn call_once(self, _args: ()) -> i32 { 666 }
} // 对应 || 666

struct FuncObj2;
impl FnOnce<(i32,)> for FuncObj2 {
    type Output = String;
    fn call_once(self, args: (i32,)) -> String {
        args.0.to_string()
    }
} // 对应 |x| x.to_string()
```

### 6.3 对比 `FnOnce<()>` 与 `Scope<'env>`：类型参数 vs 结构体生命周期

| | **`FnOnce<()>` 的 `()`** | **`Scope<'scope, 'env>` 的 `'env`** |
|--|---------------------------|--------------------------------------|
| **挂在谁上** | **Trait** `FnOnce` 的**类型参数** | **`Scope` 结构体**的生命周期参数（**不是 trait**） |
| **管什么** | 函数**调用入参**（空元组 = 无参） | **本 scope .spawn 的全部子线程**对外层栈的借用 **≤ `'env`** |
| **与返回值** | 看 **`Output`** / `-> T` | 与 Callable 返回值无关 |

```rust
// 类型参数 ()：Callable 入参
fn take<F: FnOnce() -> i32>(f: F) { f(); }

// 结构体生命周期 'env：scope 外 msg 的存活区间（见 1.2.4 §2.0）
fn demo() {
    let msg = String::from("test");
    thread::scope(|s| { s.spawn(|| println!("{msg}")); });
}
```

**一句话**：

- **`<元组>`**（`FnOnce`）→ 管**调用传什么参**；  
- **`'env`**（`Scope` **结构体**）→ 管**scope 内线程能借的外层环境活多久**；  

与 **`Trait<'buf>`**（trait 方法里的引用）又是第三层区别 → [1.2.4 §2.0 `'env` 精准含义](../1.2-scoped-threads/1.2.4-scoped-join-handle.md#env-精准含义结构体参数不是-trait-约束)。

## 七、与 `spawn` 签名对照

```rust
pub fn spawn<F, T>(f: F) -> JoinHandle<T>
where
    F: FnOnce() -> T + Send + 'static,
    T: Send + 'static,
```

| 片段 | 含义 |
|------|------|
| `F: FnOnce() -> T` | 无参闭包，返回 `T`，执行一次 |
| `F: Send + 'static` | 闭包可跨线程 move，不借短生命周期 |
| `T: Send + 'static` | 返回值可经 `join` 移回 |

---

## 八、总结（背诵）

1. **三个维度**：**trait**（调几次）· **括号**（入参）· **`-> T`**（返回值）— 互不替代 → [§地图](#先建立地图三个独立维度)。  
2. **`where`**：后置约束；**trait 后 `<>`** 可放**类型参数**或**生命周期参数**（[§六](#六trait-尖括号类型参数-vs-生命周期参数)）。  
3. **`FnOnce() -> T`**：`()` = **`FnOnce<()>` 无入参**；`-> T` = **`Output = T`**（≠ 返回空）。  
4. **`'env` vs `()`**：前者是 **`Scope` 结构体**上管**外层借用时长**；后者是 **`FnOnce` 入参元组**；再别与 **`Trait<'a>`** 混（[1.2.4 §2.0](../1.2-scoped-threads/1.2.4-scoped-join-handle.md#env-精准含义结构体参数不是-trait-约束)）。  
5. **`spawn`**：`FnOnce() -> T + Send + 'static` = 无参 · 执行一次 · 返回 `T` · 跨线程安全。  
6. **整句**：`F` 是无参、返回 `T`、FnOnce 可调用对象。
