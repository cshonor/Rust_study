### 2.1 panic 捕获与 `downcast_ref`

#### 完整可运行示例

```rust
use std::thread;

fn main() {
    let h = thread::spawn(|| {
        panic!("线程崩溃信息");
    });

    match h.join() {
        Ok(v) => println!("线程正常返回: {v:?}"),
        Err(e) => {
            if let Some(s) = e.downcast_ref::<&str>() {
                // s: &&str — downcast_ref::<&str> 返回 Option<&&str>
                eprintln!("子线程 panic(&str): {s}");
            } else if let Some(msg) = e.downcast_ref::<String>() {
                eprintln!("子线程 panic(String): {msg}");
            } else {
                eprintln!("子线程 panic 未知类型");
            }
        }
    }
}
```

仓库对应：[1.1-threads-in-rust-join-panic-demo.rs](./code/1.1-threads-in-rust-join-panic-demo.rs)

#### `join()` 返回值再强调

```rust
Result<T, Box<dyn Any + Send + 'static>>
```

| 分支 | 含义 |
|------|------|
| **`Ok(T)`** | 子线程**正常**结束，返回闭包返回值 |
| **`Err(Box<dyn Any>)`** | 子线程 **panic**，盒子里是 panic **抛出的值**（类型擦除） |

`downcast_ref` 的语法、`<T>` 类型参数、以及 **`s: &&str`** 为何能直接打印 → [1.1.2.4 §2.2](./1.1.2.4-panic-capture-downcast.md#22-downcast_refstrturbofish-与类型层级精读)。

#### 边界：非字符串 panic

```rust
thread::spawn(|| std::panic::panic_any(999i32)); // 载荷 i32（`panic!` 宏只接受格式字符串）
```

此时 `&str`、`String` 都匹配失败 → 走兜底分支；可用 **`err.type_id()`** 调试（见 demo）。

| 写法 | 常见载荷 | 说明 |
|------|----------|------|
| `panic!("错误")` | `&'static str` | 字面量编译期进 `.rodata`，**堆上无字符串分配** |
| 标准库越界 / `unwrap` 等 | **`&'static str`** | 如 `"index out of bounds: the len is …"` 中固定片段常为 static |
| `panic!("{}", 123)` | `String` | 格式化在**运行时堆分配** |
| `panic_any(StrategyError { … })` | 自定义 struct（须 **`Send + 'static`**）→ [1.1.2.3 §设计动机](./1.1.2.3-panic-box-dyn-any.md#设计动机子线程-panic-载荷完全不可控) |
| `panic_any(999)` | `i32` 等任意 `Send + 'static` 类型 |

#### 工具函数封装（统一打印）

```rust
use std::any::Any;
use std::thread::JoinHandle;

fn join_and_print<T>(handle: JoinHandle<T>) {
    match handle.join() {
        Ok(val) => println!("线程返回: {:?}", val),
        Err(panic_err) => print_panic_msg(&panic_err),
    }
}

fn print_panic_msg(err: &Box<dyn Any + Send>) {
    // 顺序有意为之：见 [1.1.2.4 §downcast 顺序](./1.1.2.4-panic-capture-downcast.md#为何-downcast-顺序先-str-再-string)
    if let Some(s) = err.downcast_ref::<&str>() {
        // s: &&str — 含 &'static str（标准库原生 panic、panic!("字面量")）
        eprintln!("panic(&str): {s}");
    } else if let Some(s) = err.downcast_ref::<String>() {
        // 运行时格式化的动态字符串
        eprintln!("panic(String): {s}");
    } else {
        eprintln!("panic 未知类型，type_id: {:?}", err.type_id());
    }
}
```

#### 为何 downcast 顺序：先 `&str`，再 `String`？

`&str` 与 `String` 是**两种不同类型**，不会互相抢分支；顺序体现的是**优先覆盖哪类 panic**：

| 匹配顺序 | 覆盖场景 | 载荷从哪来 |
|----------|----------|------------|
| **① `downcast_ref::<&str>()`** | **标准库运行时 panic**（越界、`unwrap` 失败…）、**`panic!("静态字面量")`** | 多为 **`&'static str`** — 消息在编译期写进程序，**不为这条字面量单独堆分配 String** |
| **② `downcast_ref::<String>()`** | **`panic!("{}", var)`** 等需运行时拼接的消息 | **`String` 在堆上分配** |
| **③ 兜底** | `panic_any(StrategyError)`、`panic_any(999)` 等 | 自定义 struct / 其它 `Send + 'static` 类型 |

```rust
// 运行时自动 panic — 载荷通常是 &'static str（示意）
let v = vec![1, 2, 3];
let _ = v[99]; // → join 后常可 downcast_ref::<&str>() 读到 "index out of bounds" 类消息

// 主动格式化 panic — 载荷是 String
panic!("code={}", 4001); // → downcast_ref::<String>()
```

**`print_panic_msg` 先 `&str` 再 `String`**：先兜住**官方原生 / 静态字面量** panic（自动 + 简单 `panic!`），再处理**用户主动抛的动态字符串**。业务向的 **`on_strategy_panic`** 则通常在字符串分支**之前**加 **`StrategyError`** 分支 — 自定义 struct 与 `&str`/`String` 不冲突，全链路顺序常为：

```text
StrategyError（自定义）→ &str（static / 标准库）→ String（动态）→ type_id 兜底
```

#### 关键知识点

1. **`join()`** 阻塞当前线程，等待子线程结束；
2. **panic 不会跨线程自动传播**到主线程，须通过 **`join()` 的 `Err`** 捕获；
3. **分工**：**`panic!` / `panic_any` = 抛**（不可恢复）；**`match` + `downcast_ref` = 接** — **主动 panic 与运行时自动 panic 汇合为同一 `Err(Box<dyn Any>)`**，匹配链须同时覆盖 `&str`/`String` 与自定义 struct → [1.1.2.3 §主动 vs 自动](./1.1.2.3-panic-box-dyn-any.md#主动-panic-vs-运行时自动-panic来源不同汇合点相同)；
4. **`Box<dyn Any>`** = **统一容器**；`downcast_ref` 还原 `StrategyError` / 字符串等；
5. **`downcast` 顺序**：通用打印 **`&str` → `String` → 兜底**（先 static / 标准库，再动态字符串）；业务 handler 再加 **`StrategyError`** 等自定义分支 → [1.1.2.4 §downcast 顺序](./1.1.2.4-panic-capture-downcast.md#为何-downcast-顺序先-str-再-string)。

#### 可选：`resume_unwind` 重新抛出

```rust
Err(e) => {
    std::panic::resume_unwind(e); // 主线程也跟着 panic
}
```

适合需要 **panic 连锁传递**、不必在主线程里消化错误的场景。

### 2.2 `downcast_ref::<&str>()`：turbofish 与类型层级精读

先把 **turbofish、类型、层级** 拆清楚，再回头看 §2.1 里的 `e.downcast_ref::<&str>()`。

#### （1）`Err(e)` 里 `e` 的真实类型

`h.join()` 失败时：

```rust
e: Box<dyn Any + Send + 'static>
```

`e` 是**装箱的 trait 对象**，里面装的是：**子线程 panic 时扔出来的那个值**（已被类型擦除，运行时靠 `downcast_ref` 恢复）。

| 子线程写法 | panic 抛出的原始值 `V` |
|------------|------------------------|
| `panic!("abc")` | `&'static str` |
| `panic!("num: {}", 123)` | `String` |

#### （2）`downcast_ref::<T>()`：turbofish 语法与 `<T>` 含义

`e.downcast_ref::<&str>()` 里的 **`::<&str>`** 就是 Rust 的 **turbofish** — 给**泛型方法**显式指定类型参数：

```rust
// 方法调用（日常写法）
err.downcast_ref::<&str>()

// 等价完整路径（方法名 + turbofish 都在）
std::any::Any::downcast_ref::<&str>(&err)
//              ^^^^^^^^^^^^  ^^^^^^^^
//              方法名         目标类型 T = &str
```

| 片段 | 是什么 | 作用 |
|------|--------|------|
| **`downcast_ref`** | **`Any` trait 的方法名** | 不是类型参数的一部分；turbofish **只填泛型 `T`** |
| **`::<&str>`** | **turbofish** | 告诉编译器：**要把盒内 `dyn Any` 当成 `&str` 来取引用** |
| **返回值** | **`Option<&T>`** | 运行时做**类型检查**：内部值真是 `T` → `Some(&T)`；否则 **`None`**（安全，不 panic） |

**语义一句话**：`dyn Any` 里的真实类型编译期不可见 → 你**显式声明**想要的 `T` → 标准库在**运行时**比对类型 id；匹配才借出 **`&T`**。

**为何必须写 turbofish？`T` 无法从参数推导**

`downcast_ref` 的泛型 **`T` =「要转成的目标类型」**，但入参只有 **`&dyn Any`**（类型已擦除）— 编译器**看不出**你想还原成 `&str`、`String` 还是 `StrategyError`：

```rust
fn downcast_ref<T: Any>(&self) -> Option<&T>;
//              ^ 目标类型 — 不出现在 fn 参数列表里

let payload: &Box<dyn Any + Send> = /* join 的 Err */;
// 仅凭 payload，T 可以是 &str | String | StrategyError | … — 无法推断
payload.downcast_ref::<&str>(); // 必须「喂」类型参数：这次按 &str 做运行时检查
```

| 能否推断 | 原因 |
|----------|------|
| **不能省略 `::<T>`** | 输入侧只有 **`dyn Any`**，**输出** `Option<&T>` 里的 **`T` 与输入无唯一对应** |
| **必须手动指定** | 等于告诉方法：「请检查 Box 里存的**是不是这个 `T`**，是则 `Some(&T)`，否则 `None`」 |

可以把它理解成给方法**传入一个类型实参**（不是值实参）：运行时据此做 **`TypeId` 比对**，再决定返回哪种 `Option` 分支。

```rust
err.downcast_ref::<T>()
```

| 部分 | 含义 |
|------|------|
| **`<T>`** | **显式类型参数**（turbofish）：「我想把内部值当作 **`T`** 来取引用」 |
| **`downcast_ref`** | 返回 **`Option<&T>`** — 见上表 |

常见 turbofish 目标（与 [1.1.2.4 §downcast 顺序](./1.1.2.4-panic-capture-downcast.md#为何-downcast-顺序先-str-再-string) 对应）：

```rust
payload.downcast_ref::<&str>()           // T = &str  → Option<&&str>
payload.downcast_ref::<String>()         // T = String → Option<&String>
payload.downcast_ref::<StrategyError>()  // T = 自定义 struct → Option<&StrategyError>
```

> **重点**：`downcast_ref` **本身就返回引用**（`&T`），不是把内部值「变成」引用；它是在已装箱的值上再借一层。

#### （3）拆解 `downcast_ref::<&str>`：目标类型 `T = &str`

以 `panic!("hello")` 为例，**一层层剥开**：

1. 子线程 `panic!("hello")` — 字面量 `"hello"` 类型：**`&'static str`**
2. 这个 **`&str`** 被装进 **`Box<dyn Any>`** → 就是 `join()` 拿到的 **`e`**
3. 调用 **`e.downcast_ref::<&str>()`**：
   - 检查：内部值是不是 **`&str`**？**是**
   - 返回：**`Some( &(内部的 &str) )`**

完整类型链：

```
e: Box<dyn Any>
  ↓ 内部真实值
V = &'static str
  ↓ downcast_ref::<&str>()
Option< &V > = Option< &&str >
```

因此：

```rust
if let Some(s) = e.downcast_ref::<&str>() {
    // s 的类型是 &&str —— 「字符串引用的引用」
}
```

#### （4）为什么 `&&str` 能直接打印？

Rust 有 **自动解引用（Deref coercion）**：`&&str` 在调用方法、打印、传参时会逐层解引用到 `&str` / `str`，所以 **`println!("{s}")` 完全正常**，不必写 `(*s)`。

极简对照：

```rust
fn main() {
    let s: &str = "hello";   // &str
    let ss: &&str = &s;      // &&str — 引用的引用

    println!("{}", ss);      // 自动解引用 → hello
}
```

对应 panic 捕获：

- `downcast_ref::<&str>()` → `Some(&&str)`
- 绑定到 `s` → **`s: &&str`**
- 打印 / 比较 / 传参 → 自动解引用，日常**感知不到**双层引用

#### （5）为什么不能写 `downcast_ref::<str>`？

```rust
// 错误写法
e.downcast_ref::<str>()
```

1. **`str` 是 DST（不定长类型）**，不能作为 `downcast` 的目标类型参数；
2. panic 抛出的本来就是 **`&str`**，不是裸 **`str`**。

**必须写 `<&str>`**。

#### （6）对比：`panic` 抛出 `String`

```rust
panic!("num: {}", 123); // 生成并抛出 String
```

- 内部值 **`V = String`**
- 应写：**`downcast_ref::<String>()`**
- 返回 **`Option<&String>`**（单层引用，因为 `T = String`，`&T = &String`）

两场景对照：

```rust
// 场景 1：panic!("字面量") → 内部是 &str
if let Some(s) = e.downcast_ref::<&str>() {
    // s: &&str
}

// 场景 2：panic!(格式化) → 内部是 String
else if let Some(msg) = e.downcast_ref::<String>() {
    // msg: &String
}
```

#### （7）一句话总结

1. `panic!("xxx")` 抛出的原始值是 **`&str`**；
2. **`downcast_ref::<&str>`** = 把内部值当作 **`&str`** 来取引用；
3. 绑定得到的 **`s` 类型是 `&&str`**（引用的引用）；
4. Rust 自动解引用，**正常打印 / 使用即可**，不必手动解一层。
