## 一、一句话回顾

Rust 并发从**线程共享内存**出发，用**共享借 / 排他借**在编译期防数据竞争；跨线程用 **`'static` / `scope` / `Arc`** 解决所有权；可变共享靠 **内部可变性 + `Mutex`/`RwLock`/原子**；线程边界由 **`Send`/`Sync`** 标记；等条件用 **park / `Condvar`**，且必须 **`loop` 查条件** 防虚假唤醒。

---

## 二、核心术语

| 术语 | 含义 |
|------|------|
| **Shared / Exclusive Reference** | `&T` / `&mut T` 的准确叫法 |
| **Data Race** | 并发读写同内存 → UB |
| **JoinHandle\<T\>** | `spawn` 返回值；详见 [1.1.2 JoinHandle](../1.1-threads-in-rust/1.1.2-join-handle.md) |
| **Interior Mutability** | 通过 `&T` 改内部数据 |
| **Send / Sync** | 跨线程移所有权 / 跨线程共享 `&T` |
| **Lock Poisoning** | 持锁 panic 后锁标记中毒 |
| **Spurious Wake-up** | `park` / `wait` 可无因苏醒 → **必须 `loop`** |
| **Leakpocalypse** | 依赖 Drop 的 scoped API 因 leak 而 Unsound |
| **Writer Starvation** | `RwLock` 读者占满，写者长期等待 |

## 七、面试 / 背诵

**一段话**：Rust 并发用借用规则防数据竞争；跨线程用 `'static`/`Arc`/`scope`；可变共享用 `Mutex`/`Condvar`/原子；`Send`/`Sync` 划边界。

| # | 极简句 |
|---|--------|
| 1 | 同进程线程共享内存；`&T` 共享借、`&mut T` 排他借。 |
| 2 | `spawn` → `JoinHandle`；`.join()` 阻塞取 `Result<T, _>`。 |
| 3 | `scope` 可借局部；`Arc` 长期共享；`Mutex` 包数据。 |
| 4 | `park`/`Condvar` 等条件必 `loop`；早放锁。 |
