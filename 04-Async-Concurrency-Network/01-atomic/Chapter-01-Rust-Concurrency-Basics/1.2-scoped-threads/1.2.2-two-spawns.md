## 1.2.2 有界 scope vs 无界 spawn

> 书 §1.2 · 承接 [1.1.2 JoinHandle](../1.1-threads-in-rust/1.1.2-join-handle.md) · 索引：[1.2](./1.2-scoped-threads.md) · 前：[1.2.1 动机](./1.2.1-scope-motivation.md) · 后：[1.2.3 join](./1.2.3-join-auto-join.md)

---

Rust 标准库里创建 OS 线程主要有两条路：**有界作用域**的 `thread::scope` 与**无界**的 `thread::spawn`。二者 API 相似（都叫 `spawn`），但**返回值类型、生命周期约束、join 行为**完全不同。

### 0. 两个 `spawn` 完整区分（带类型标注）

#### ① 全局：`std::thread::spawn`（无 scope 普通线程）

```rust
use std::thread;

// 返回：JoinHandle<T>
let handle: thread::JoinHandle<i32> = thread::spawn(move || 123);
handle.join().unwrap();
```

| 项 | 说明 |
|----|------|
| **返回类型** | `JoinHandle<T>`（[1.1.2](../1.1-threads-in-rust/1.1.2-join-handle.md)） |
| **闭包约束** | `Send + 'static`；捕获局部变量须 **`move`**，**不能借用**栈上临时数据 |
| **线程寿命** | 不受当前函数 / scope 限制，可活得比创建者更久 |
| **等待** | **不会**自动 join；须手动 `.join()`，否则 drop → **detach**，`main` 提前退出则可能强杀子线程 |

#### ② scope 内：`s.spawn()`（有界作用域线程）

```rust
use std::thread;

thread::scope(|s| {
    // s.spawn 返回 ScopedJoinHandle<'scope, T>
    let scoped_handle: thread::ScopedJoinHandle<'_, i32> = s.spawn(|| 123);
    let _ = scoped_handle; // 也可手动 scoped_handle.join().unwrap()
});
// 离开 scope：Scope 收尾批量 join 尚未 join 的 handle
```

| 项 | 说明 |
|----|------|
| **返回类型** | `ScopedJoinHandle<'scope, T>`，带生命周期 **`'scope`** |
| **闭包约束** | `Send + 'scope`；**可借用** scope 外、仍在存活的栈变量，通常**不必 move** |
| **线程寿命** | **锁在 scope 内**；handle **不能逃出** scope 闭包 |
| **等待** | scope 收尾**自动 join** 未处理的 handle；也可在 scope 内**手动** `.join()` |

> 口语里常把 `ScopedJoinHandle` 简称 scope 句柄；标准库全名是 **`ScopedJoinHandle`**，不是 `ScopeJoinHandle`。

### 1. 有界作用域线程 `thread::scope`（稳定 API，1.63+）

```rust
use std::thread;

fn main() {
    thread::scope(|s| {
        // s.spawn 返回 ScopedJoinHandle<'scope, T>
        let h = s.spawn(|| 1 + 1);
        let _ = h; // 不显式 join 也行——见下文「自动 join」
    });
    // 离开 scope 闭包时，标准库会对尚未 join 的 handle 批量 join
}
```

**核心特性**：

1. 在 `scope` 闭包里用 **`s.spawn()`** 开子线程；
2. **scope 闭包执行完毕时，会自动隐式 join 所有尚未 join 的子线程**，不必像 `spawn` 那样逐个手动 `.join()`；
3. 可**安全借用** scope 外的局部变量，通常**不必 `move`**；
4. 仍可**手动提前 `.join()`** 取返回值：

```rust
thread::scope(|s| {
    let h = s.spawn(|| 2);
    let res = h.join().unwrap(); // 手动提前 join
    println!("{res}");
});
```

> **易错**：`ScopedJoinHandle` **不能逃出** scope（不能 `push` 进 scope 外的 `Vec` 再慢慢 join）——类型系统保证子线程寿命 ≤ scope。

### 2. 普通无界线程 `thread::spawn`

```rust
use std::thread;

let h = thread::spawn(move || 1 + 1);
// 没有自动 join；必须手动 join，否则 main 可能先退出、子线程被强杀
h.join().unwrap();
```

- 返回 **`JoinHandle<T>`**（详见 [1.1.2](../1.1-threads-in-rust/1.1.2-join-handle.md)），**不会自动 join**；
- 闭包须 **`Send + 'static`**，捕获栈上局部变量须 **`move`** 所有权，不能直接借用。

### 3. 借用局部变量：`scope` 典型用法

```rust
use std::thread;

fn main() {
    let num = 50;
    let vec = vec![1, 2, 3];

    thread::scope(|s| {
        s.spawn(|| {
            println!("{num}, {:?}", vec); // 借用 scope 外变量，无需 move
        });
    });
    println!("scope 结束，vec 仍可用: {:?}", vec);
}
```

**补充规则**：

1. **`thread::scope` 会阻塞**调用它的线程，直到 scope 内全部子线程结束；
2. scope 闭包**里面**新定义的变量，若要在 `s.spawn` 里用，通常仍要 **`move`**；
3. 多个 `s.spawn` 的**执行顺序由 OS 调度**，打印顺序不保证（见 Demo 示例 2）。
