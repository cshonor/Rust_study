1. **缓存行争用** — 多核抢同一 cache line；

## 假共享（False Sharing）

CPU 按 **缓存行**（常见 **64B**）缓存内存：

- 多个**无关**锁/原子挤在同一行 → 一核改锁 → 它核该行失效 → **性能暴跌**。

**对策**（高频多锁场景）：

```rust
#[repr(align(64))]
struct AlignedLock { /* … */ }
```

→ 与 [第 7 章处理器 / 缓存](../Chapter-07-Processors/本章学习笔记.md) 交叉。

**注意**：锁与**保护的数据**紧挨有时利于局部性 —— 视访问模式而定（[互斥锁贯通 §5.4](../互斥锁与锁体系-贯通笔记.md#54-优化-3假共享false-sharing)）。

## Benchmarks（书中）

对比维度通常包括：

| 变量 | 观察 |
|------|------|
| 两态 vs **三态** | syscall 次数 |
| 纯阻塞 vs **混合** | 短临界区吞吐 |
| 未对齐 vs **align(64)** | 多核扩展性 |

本仓库 demo 待补；阅读时关注**相对趋势**，非绝对纳秒数。
