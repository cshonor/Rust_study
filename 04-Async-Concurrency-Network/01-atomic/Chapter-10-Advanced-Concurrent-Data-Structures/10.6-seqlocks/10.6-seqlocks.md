## 10.6 Seqlocks（顺序锁）

---

## 核心：原子序列号 `seq`

| 角色 | 行为 |
|------|------|
| **写者** | 写前 **`seq += 1`**（变**奇数**=写入中）→ 写数据 → 再 **`seq += 1`**（变**偶数**=稳定） |
| **读者** | 读 `seq₁` → 读数据 → 读 `seq₂`；若 **`seq₁ == seq₂` 且为偶数**，则期间无并发写 |

---

## 特点

| ✅ | ❌ |
|----|-----|
| 写者**不必等**读者 | 读者可能**重试**（写入期间快照不一致） |
| 读者几乎无原子写，**读吞吐极高** | 不适合写多 |
| 内核时间戳、读极多写极少配置 | 需要强一致读的场景 |

---

## 与 RCU / RwLock

| | **SeqLock** | **RCU** | **RwLock** |
|---|-------------|---------|------------|
| 读者阻塞写者 | 否 | 否（旧版本） | 写者等读者 |
| 一致性 | 重试得有效快照 | 可能读旧 | 强一致 |
| 回收 | 无指针换版问题 | 需 epoch | Guard RAII |

---

## 读者伪代码

```rust
loop {
    let s1 = seq.load(Acquire);
    if s1 % 2 != 0 { continue; } // 写入中
    let snapshot = read_data();
    let s2 = seq.load(Acquire);
    if s1 == s2 { break snapshot; }
}
```

---

## 背诵

> **SeqLock：写时 seq 奇数；读者前后 seq 相等且偶数则快照有效。**
