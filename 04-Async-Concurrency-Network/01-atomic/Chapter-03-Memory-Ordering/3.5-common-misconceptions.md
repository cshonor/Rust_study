## 新手易错与正解

| 误区 | 正解 |
|------|------|
| **Relaxed = 更新「更慢」** | 模型规定**顺序**不规定**时间**；快慢看缓存一致性，与 Ordering 无简单对应 |
| **x86 不乱序就不用 Ordering** | **编译器仍可重排**；错 Ordering 会生成错误机器码；且 Rust 语义按**抽象模型**而非单架构 |
| **SeqCst 万能最安全** | 多数场景 **Release/Acquire** 足够；SeqCst 掩盖分析、损性能（ARM 上更明显） |
| **Relaxed 当「数据就绪」标志** | 必须 **Release**（写标志）+ **Acquire**（读标志） |
| **有原子就不需要 Ordering** | 每次原子 API **必传** `Ordering`；默认无「安全默认项」 |
| **Ordering 只管 CPU 不管编译器** | **编译器优化**同样受 Ordering 约束 |
