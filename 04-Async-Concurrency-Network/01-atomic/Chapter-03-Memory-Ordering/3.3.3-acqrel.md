## 3.3.3 `Ordering::AcqRel`（获取并释放）

> 书 §3.3 · 前：[3.3.2 Release/Acquire](./3.3.2-release-acquire.md) · 索引：[3.3-memory-order-options.md](./3.3-memory-order-options.md) · 后：[3.3.4 SeqCst](./3.3.4-seq-cst.md)

---

| 项 | 内容 |
|----|------|
| **用于** | **RMW**（Read-Modify-Write）：`fetch_add`、`compare_exchange`、`swap` 等 |
| **语义** | **读侧 Acquire** + **写侧 Release** 合一 |
| **场景** | 单次原子操作既要「看见之前发布」又要「向后续发布」 |

---

## 为什么需要 AcqRel？

对 **RMW**，操作同时**读旧值**又**写新值**：

- 读侧需要 **Acquire** — 看见此前其它线程 **Release** 发布的内容；
- 写侧需要 **Release** — 向后续 **Acquire** 订阅者发布本次更新。

分开写 Acquire / Release 无法表达「一次 RMW 两侧都要同步」。

```rust
use std::sync::atomic::{AtomicUsize, Ordering};

static STATE: AtomicUsize = AtomicUsize::new(0);

// RMW 当同步点：AcqRel
fn update() {
    STATE.fetch_add(1, Ordering::AcqRel);
}
```

---

## 常见 API

| API | 典型 Ordering |
|-----|---------------|
| `fetch_add` / `fetch_sub` / … | 仅统计 → **Relaxed**；同步点 → **AcqRel** |
| `compare_exchange` / `compare_exchange_weak` | 参与锁 / 状态机 → **AcqRel** |
| `swap` | 自旋锁加锁 → 常 **Acquire** 或 **AcqRel**（见第 4 章） |

---

## 与 Release/Acquire 的关系

| 操作类型 | 常用 Ordering |
|----------|---------------|
| 纯 **store** | **Release** |
| 纯 **load** | **Acquire** |
| **RMW** | **AcqRel**（或按语义选 Acquire-only / Release-only，较少见） |

---

## 记忆

> **AcqRel = RMW 专用；读 Acquire + 写 Release 一次做完。**
