## 3.3.2 `Release` / `Acquire`（释放 / 获取）

> 书 §3.3 · 前：[3.3.1 Relaxed](./3.3.1-relaxed.md) · 索引：[3.3-memory-order-options.md](./3.3-memory-order-options.md) · 后：[3.3.3 AcqRel](./3.3.3-acqrel.md)

---

**必须成对** — 并发通信的「工作马」。

| | **Release** | **Acquire** |
|---|-------------|-------------|
| **用于** | **Store**（及 RMW 的**写侧**） | **Load**（及 RMW 的**读侧**） |
| **屏障** | 前面的内存操作**不能**排到它**之后** | 后面的内存操作**不能**排到它**之前** |
| **HB** | A：`Release` store → B：`Acquire` load **读到该写入** → A 在 Release **之前**的写对 B 在 Acquire **之后**可见 |

---

## 典型模式：发布 / 订阅

```rust
// 线程 A（发布者）
data = 42;                              // 写普通内存
ready.store(true, Ordering::Release);    // Release：之前的写不能排到 store 后

// 线程 B（订阅者）
if ready.load(Ordering::Acquire) {       // Acquire：读到 Release 写入
    use(data);                           // 保证看见 data = 42
}
```

对比 [3.1](../3.1-reordering-and-optimizations/3.1-reordering-and-optimizations.md)：`Relaxed` 标志无法保证 `data` 可见。

---

## 工程场景

| 场景 | Release | Acquire |
|------|---------|---------|
| 消息传递 | 写完 payload 后 **Release** 标志 | **Acquire** 读标志后读 payload |
| **自旋锁 / Mutex** | **unlock**：`store(false, Release)` | **lock**：`swap(true, Acquire)` 或 load |
| 一次性初始化 | 写完全局状态后 Release | 读者 Acquire 见标志后读状态 |

## HB 链（背诵）

```text
  A：写普通内存 … → Release store(原子)
                          ║ happens-before
  B：Acquire load(原子) 读到该值 … → 读普通内存
```

---

## 记忆

> **Release 发布，Acquire 订阅；写标志用 Release，读标志用 Acquire。**
