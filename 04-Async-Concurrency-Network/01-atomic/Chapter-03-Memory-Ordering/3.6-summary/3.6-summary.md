## 一、一句话回顾

**重排**使多线程下可见顺序与源码不一致；**内存模型 + Happens-Before** 定义合法同步；**Ordering** 限制重排 —— **Relaxed** 只保单原子总序，**Release/Acquire** 同步普通内存，**AcqRel** 用于 RMW，**SeqCst** 全局最严、极少用。

## 三、选型原则（从弱到强）

1. **首选 Relaxed** — 只更新单变量、不传「别数据已就绪」。  
2. **默认 Acquire / Release** — 标志位、锁、信道式同步：写 **Release**，读 **Acquire**。  
3. **RMW 当同步点** — 考虑 **AcqRel**。  
4. **极度谨慎 SeqCst** — 先证明 RA 不够；ARM 上还有额外成本。

```text
  信使原子同步普通内存？ → 禁止 Relaxed，用 Release/Acquire（或 AcqRel 的 RMW）
  仅单变量统计？ → Relaxed
  多原子全局时间线？ → 才考虑 SeqCst
```

## 六、面试一句话

**Ordering 用 HB 约束重排：Relaxed 只保单原子总序；Release/Acquire 同步普通内存；AcqRel 用于 RMW；SeqCst 全局最严极少用；锁与标志用 Acquire 加锁、Release 解锁。**

## 七、极简背诵卡

| # | 背这句 |
|---|--------|
| 1 | 重排+内存模型；Ordering 定义 HB，每原子操作必传。 |
| 2 | Relaxed：无 HB，单原子总修改顺序；独立计数用。 |
| 3 | Release：前序写不能排到 store 后；用于发布/解锁。 |
| 4 | Acquire：后续读不能排到 load 前；读到 Release 见之前写。 |
| 5 | AcqRel：RMW 读 Acquire+写 Release。 |
| 6 | SeqCst：全局一致顺序；多原子先后决策才考虑。 |
| 7 | 选型弱→强；信使禁 Relaxed；x86 也要防编译器重排。 |
| 8 | 第 4 章自旋锁：lock=Acquire swap，unlock=Release store。 |
