## 1. 自旋锁是什么

> 与 **`Mutex` / `RwLock` 选型、syscall 优化** 对照：[互斥锁与锁体系-贯通笔记.md](../互斥锁与锁体系-贯通笔记.md)

| 对比 | 行为 |
|------|------|
| **`Mutex`（OS）** | 争不到锁时线程**睡眠**，由内核调度 |
| **自旋锁（Spin Lock）** | 争不到锁时 **忙等**（`while` + `spin_loop`），不陷入内核 |

适用：锁持有时间**极短**、线程数不多、不愿付出上下文切换成本。持有过久会浪费 CPU（书中后文会对比其他锁）。

## 2. 核心结构与内存序匹配

实现见 **`4.1-minimal-implementation/code/4.1-minimal-implementation-demo.rs`**（与书中逻辑一致，补全 `Guard` + `Deref`）。

```rust
pub struct SpinLock<T> {
    locked: AtomicBool,      // 锁状态
    value: UnsafeCell<T>,    // 被保护数据（需 unsafe 访问）
}
```

| 操作 | API | Ordering | 作用 |
|------|-----|----------|------|
| **加锁（swap）** | `locked.swap(true, Acquire)` | **Acquire** | 一旦成功，后续对 `value` 的读写**不能**重排到加锁**之前** |
| **加锁（CAS）** | `compare_exchange_weak(false, true, Acquire, Relaxed)` 循环 | **Acquire** | 与 swap 对照；见 `spin_lock.rs::lock_cas` |
| **解锁** | `locked.store(false, Release)` | **Release** | 此前对 `value` 的修改**不能**重排到解锁**之后** |
| **等待** | `while swap 仍为 true` + `spin_loop()` | — | 降低自旋功耗；书中强调自旋锁用 **`swap`** 而非在循环里狂 CAS |

### Happens-Before 链条

```text
  线程 1                          线程 2
  写 *value
  unlock: store(false, Release)
                                  lock: swap(true, Acquire) 成功
                                  读/写 *value  // 必见线程 1 的写入
```

**Release（解锁）→ Acquire（加锁）** 建立 HB，从而在**无系统调用**的情况下避免对 `value` 的数据竞争。
