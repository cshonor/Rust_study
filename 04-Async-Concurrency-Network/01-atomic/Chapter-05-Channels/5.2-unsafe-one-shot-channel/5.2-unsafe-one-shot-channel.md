### 2.1 避免多余分配：`MaybeUninit<T>`

- 通道槽位用 **`UnsafeCell<MaybeUninit<T>>`**，不必为「尚未写入」多包一层 `Option<T>` 的判别式开销（教学向；生产用标准库更安全）。  
- 读取方在确认就绪后用 **`assume_init_read()`**（unsafe，由协议保证已初始化）。

### 2.2 状态标志：`AtomicBool` + 内存序

- **`ready`**：发送方 `store(true, Release)` → 接收方 `load(Acquire)` 见消息写入。  
- **禁止** 用 Relaxed 当「数据已写好」的信使。

- **`ready`**：发送方 `store(true, Release)` → 接收方 `load(Acquire)` 见消息写入。  
- **禁止** 用 Relaxed 当「数据已写好」的信使。

### 2.3 类型安全：`Sender` / `Receiver` + 消耗 `self`

- `send(self, …)`、`receive(self)` **各只能调用一次**，编译期杜绝二次 send/receive 的数据竞争。  
- 比仅靠运行时检查更可靠。

### 2.4 阻塞：`park` / `unpark`

- 接收方未就绪时 **`thread::park()`** 交出 CPU（对比第 4 章忙等自旋）。  
- 发送方 `Release` 写消息后，对登记好的接收线程 **`unpark()`**。  
- 实现见 **`5.2-unsafe-one-shot-channel/5.2-unsafe-one-shot-channel-demo.rs`**：接收方在 `park` 前把 `thread::current()` 写入 `receiver_thread`。

**运行逻辑简述**
