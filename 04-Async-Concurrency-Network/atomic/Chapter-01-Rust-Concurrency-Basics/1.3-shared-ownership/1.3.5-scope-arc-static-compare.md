## 1.3.5 scope / Arc / `'static` 对照与易错

> 书 §1.3 · 可变共享见 [1.5](../1.5-interior-mutability/1.5-interior-mutability.md)、[1.7 Mutex](../1.7-mutex-rwlock/1.7-mutex-rwlock.md) · 索引：[1.3-shared-ownership](./1.3-shared-ownership.md) · 前：[1.3.4 克隆遮蔽](./1.3.4-arc-clone-shadow.md) · 后：[1.4 借用](../1.4-borrowing-data-races/1.4-borrowing-data-races.md)

---

| | **`thread::scope`** | **`Arc<T>`** | **`'static` / leak** |
|---|---------------------|--------------|----------------------|
| 数据位置 | 常在栈上 | 堆 + 计数 | 全局或泄漏堆 |
| 线程寿命 | 限于 scope | 任意 | 任意 |
| 可变共享 | 须额外同步类型 | 须 `Mutex` 等 | 须 `Mutex` 或原子 |
| 典型用途 | 短生命周期并行 | 长期多线程只读/加锁写 | 全局配置、一次性泄漏 |

---


1. **以为 `Arc::clone` 复制了数据**：只复制指针并 `+1` 计数。
2. **用 `Rc` 做跨线程共享**：编译期会拦；换 `Arc`。
3. **只有 `Arc` 就能改数据**：不行，默认只读；改要用 `Arc<Mutex<T>>`。
4. **滥用 `Box::leak`**：泄漏不可回收，测试/临时方案可以，生产要谨慎。
