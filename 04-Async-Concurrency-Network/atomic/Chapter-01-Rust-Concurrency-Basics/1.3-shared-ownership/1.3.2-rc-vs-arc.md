## 1.3.2 `Rc<T>` 与 `Arc<T>`

> 书 §1.3 · 索引：[1.3-shared-ownership](./1.3-shared-ownership.md) · 前：[1.3.1 动机](./1.3.1-shared-ownership-motivation.md) · 后：[1.3.3 static/leak](./1.3.3-static-and-leak.md)

---

**一句话**：`Rc::new(x)` / `Arc::new(x)` 在堆上创建 `x`，栈上只留**智能指针**；`clone()` **只增计数、不复制堆数据**。

```rust
use std::rc::Rc;

let rc1 = Rc::new(42);
let rc2 = Rc::clone(&rc1); // 计数 2，仍指向同一块堆内存
println!("{} {}", rc1, rc2);
```

### 1. 共同点：默认只能共享读

`Rc<T>` / `Arc<T>` 提供的是 **`&T` 级别的共享**，**不能直接 `*rc = …` 修改**。要改内部数据 → 配合 `RefCell` / `Mutex`（见 [1.5](../1.5-interior-mutability/1.5-interior-mutability.md)、[1.7](../1.7-mutex-rwlock/1.7-mutex-rwlock.md)）。

### 2. 关键区别

| | **`Rc<T>`** | **`Arc<T>`** |
|---|-------------|--------------|
| 模块 | `std::rc` | `std::sync` |
| 计数更新 | 普通整数加减 | **原子**操作 |
| 跨线程 | ❌ 不能 `Send` | ✅ 可 `spawn` + `clone` 分发 |
| 性能 | 略快（无原子开销） | 略慢 |
| 典型组合 | `Rc<RefCell<T>>` | `Arc<Mutex<T>>`、`Arc<RwLock<T>>` |

```rust
use std::sync::Arc;
use std::thread;

let arc = Arc::new(100);
let a = Arc::clone(&arc);
let h = thread::spawn(move || println!("{a}"));
h.join().unwrap();
println!("主线程仍可读: {arc}");
```

仓库对应：[1.3-shared-ownership-arc-demo.rs](./code/1.3-shared-ownership-arc-demo.rs)
