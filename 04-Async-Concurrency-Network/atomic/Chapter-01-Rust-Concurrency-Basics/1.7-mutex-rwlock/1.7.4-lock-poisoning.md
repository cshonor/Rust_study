## 1.7.4 锁中毒（Lock Poisoning）

> 书 §1.7 · 索引：[1.7-mutex-rwlock.md](./1.7-mutex-rwlock.md) · 前：[1.7.3 MutexGuard](./1.7.3-mutex-guard.md) · 后：[1.7.5 RwLock](./1.7.5-rwlock.md)

---

持锁线程 **panic** 时，Rust 将 `Mutex` 标记为 **poisoned（中毒）**：防止其他线程在数据可能不一致时继续用。

```rust
let m = Mutex::new(0);
// 若某线程 lock 后 panic，后续 lock() 得 Err(PoisonError)
match m.lock() {
    Ok(guard) => { /* 正常 */ }
    Err(poison) => {
        // poison.into_inner() 仍可强行取 guard（接受风险）
    }
}
```

日常 `.unwrap()` = 「接受中毒、继续用内部 guard」。生产环境可记录日志或重建状态。
