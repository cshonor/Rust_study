## 1.8.2 `thread::park` 与 `Thread::unpark`

> 书 §1.8 · 索引：[1.8-parking-condvar.md](./1.8-parking-condvar.md) · 前：[1.8.1 动机](./1.8.1-waiting-motivation.md) · 后：[1.8.3 Condvar](./1.8.3-condvar-pattern.md)

---

### 基本语义

- **`thread::park()`**：当前线程休眠，直到被 **`unpark`** 或（带超时 API 时）超时。
- **`thread::current().unpark()`** 或保存的 **`Thread`** 句柄上调用 **`unpark()`**。

### 关键：`unpark` 先于 `park` 不丢信号

若 **`unpark` 在 `park` 之前**调用，信号**不会丢失**——下次 `park` **立即返回**（「permits」模型）。

```rust
use std::thread;
use std::time::Duration;

let t = thread::current();
thread::spawn(move || {
    thread::sleep(Duration::from_millis(100));
    t.unpark();
});
thread::park(); // 可能被提前 unpark，也可能在这里睡
```

### 局限

- 适合**简单**暂停 / 恢复；
- **多生产者 / 多消费者**下很难精准唤醒「在等某一个条件」的特定线程；
- 与共享状态配合时，自己处理「检查条件 + 睡眠」易出竞态 → 用 **`Condvar`**。
