## 1.6.5 `Arc<Mutex<T>>` 组合（Send + Sync 落地）

> 书 §1.6 · 索引：[1.6-send-sync.md](./1.6-send-sync.md) · 前：[1.6.4 Sync](./1.6.4-sync-trait.md) · 后：[1.6.6 易错点](./1.6.6-manual-impl-pitfalls.md) · 锁细节见 [1.7](../1.7-mutex-rwlock/1.7-mutex-rwlock.md)

---

```rust
use std::sync::{Arc, Mutex};
use std::thread;

let data = Arc::new(Mutex::new(0));
let mut handles = vec![];

for _ in 0..5 {
    let d = Arc::clone(&data);
    handles.push(thread::spawn(move || {
        *d.lock().unwrap() += 1;
    }));
}
for h in handles {
    h.join().unwrap();
}
println!("{}", *data.lock().unwrap());
```

| 组件 | 贡献 |
|------|------|
| **`Arc`** | 多线程共享**同一把锁**的句柄（`Sync`） |
| **`Mutex`** | 同一时刻只有一个线程改内部 `T` |
| **`move` + `Send`** | 每个线程拿走自己的 `Arc` 克隆 |
