## CONDITION_VARIABLE（Vista+）

| API | 作用 |
|-----|------|
| **`InitializeConditionVariable`** | 初始化 |
| **`SleepConditionVariableCS` / `SleepConditionVariableSRW`** | 与 **Critical Section** 或 **SRWLock** 配合 wait |
| **`WakeConditionVariable`** | 唤醒一个 |
| **`WakeAllConditionVariable`** | 唤醒全部 |

语义与 POSIX **cond_wait + signal/broadcast** 同构：wait 时释放锁，醒后重新获取。

---

## WaitOnAddress 族（Win8+）

更贴近 **futex** 模型：

| API | ≈ futex |
|-----|---------|
| **`WaitOnAddress`** | WAIT（比较地址上值，相等则睡） |
| **`WakeByAddressSingle`** | WAKE one |
| **`WakeByAddressAll`** | WAKE all |

第 9 章 **`atomic_wait`** crate 在 Windows 上封装此族；Linux 上封装 futex。
