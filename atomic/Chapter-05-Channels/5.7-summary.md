## 4. 业务落地场景

| 场景 | 技术点 |
|------|--------|
| **全局唯一 ID / 进度汇总** | `fetch_add` 或 **CAS 循环** 防溢出（`../Chapter-02-Atomics/2.3-compare-and-exchange/2.3-compare-and-exchange-id-allocator-demo.rs`） |
| **极短临界区、多核、低延迟** | 自旋锁 + `spin_loop`（`../Chapter-04-Spin-Locks/4.1-minimal-implementation/4.1-minimal-implementation-demo.rs` / `lock_cas`） |
| **单次结果回传** | One-Shot Channel + park/unpark（本章；思想同 `oneshot::channel`） |
| **临界区长或需休眠** | `Mutex` / 内核 futex，勿 CAS 空转 |

---

## 5. 面试一句话

**第五章用 `MaybeUninit` 存一条消息、`AtomicBool` 的 Release/Acquire 同步就绪、`Sender`/`Receiver` 消耗 self 保证单次收发，接收端 park、发送端 unpark，把原子与线程阻塞合成最小通道。**

---

## 6. 极简背诵卡

| # | 背这句 |
|---|--------|
| 1 | One-Shot：一次 send、一次 receive；类型消耗 self 防二次。 |
| 2 | MaybeUninit 省 Option 判别；就绪后 assume_init_read。 |
| 3 | ready：发送 Release，接收 Acquire；禁 Relaxed 信使。 |
| 4 | 未就绪 receiver park；发送后 unpark 登记线程。 |
| 5 | 短临界自旋；等消息用 park，别自旋烧 CPU。 |
| 6 | 思想同 async oneshot；生产用 std/crate。 |
