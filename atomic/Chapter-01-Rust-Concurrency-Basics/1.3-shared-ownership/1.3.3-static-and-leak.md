## 1.3.3 `static` 与 `Box::leak`

### 1. `static`

编译期或链接期存在的全局数据，类型天然带 **`'static`** 生命周期：

```rust
static GREETING: &str = "Greetings from a 'static string!";

thread::spawn(|| println!("{GREETING}")).join().unwrap();
```

仓库对应：[1.3-shared-ownership-static-demo.rs](./code/1.3-shared-ownership-static-demo.rs)

### 2. `Box::leak`

把堆数据**永久泄漏**成 `'static` 引用，适合「进程级只泄漏一次」的场景：

```rust
let leaked: &'static str = {
    let s = String::from("Hello from a leaked Box!");
    Box::leak(s.into_boxed_str())
};

thread::spawn(move || println!("{leaked}")).join().unwrap();
```

仓库对应：[1.3-shared-ownership-box-leak-demo.rs](./code/1.3-shared-ownership-box-leak-demo.rs)

> **选型**：优先 `Arc`；`leak` 是刻意放弃回收；`static` 适合真正全局常量。
