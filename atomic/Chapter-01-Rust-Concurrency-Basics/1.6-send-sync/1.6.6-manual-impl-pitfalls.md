## 手动控制 Send / Sync

| 手段 | 用途 |
|------|------|
| **自动衍生** | 字段都 `Send`/`Sync` 则 struct 自动满足 |
| **`PhantomData<Cell<T>>`** 等 | 编译期**取消** auto Send/Sync（零运行时开销） |
| **`unsafe impl Send/Sync`** | 封装裸指针等：作者证明跨线程安全 |

> 只有确认封装正确时才 `unsafe impl`；否则 Unsound。

---

## 对照：`Send` vs `Sync`

| | **`Send`** | **`Sync`** |
|---|------------|------------|
| 问的是 | 值 **move** 到子线程？ | **`&T`** 跨线程共享？ |
| 典型 API | `thread::spawn(move \|\| …)` | `Arc::clone` 多线程读 |
| 关系 | `T: Sync` ⇒ `&T: Send` | 反之不成立 |
| 报错方向 | 「cannot be sent between threads」 | 「cannot be shared between threads safely」 |

---

## 新手易错

1. **混淆 Send 与 Sync**：`spawn` 要 **Send**；`Arc` 共享要内部 **`T: Send + Sync`**。
2. **以为 `Arc` 就能改**：只读共享；写要 `Mutex` / 原子。
3. **在多个线程里用 `Rc` / `RefCell`**：应换 `Arc` / `Mutex`。
4. **裸指针自己 `Send`**：指针可 move，但解引用安全要你证明。
