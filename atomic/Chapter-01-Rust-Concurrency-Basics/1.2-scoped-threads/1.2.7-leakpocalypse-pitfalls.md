## 1.2.7 Leakpocalypse 与易错点

> 书 §1.2 · 索引：[1.2](./1.2-scoped-threads.md) · 前：[1.2.6 变量与结果](./1.2.6-variables-results.md) · 后：[1.3 Arc](../1.3-shared-ownership/1.3-shared-ownership.md)

---

Rust **1.0 之前**曾提供基于 **`JoinGuard` + `Drop` 自动 join** 的 `scoped` 线程 API：guard 离开作用域时在 `Drop` 里 join 子线程，看起来能安全借用栈上数据。

**Leakpocalypse**（社区对这次设计失败的称呼）：

1. `mem::forget(guard)`、**循环引用**等可以让 guard **永不 Drop**；
2. 子线程继续跑，却**不再被 join**，仍借用已失效的栈内存 → **Unsound**（安全 API 在 leak 下失效）；
3. 该 API 在 **1.0 前被移除**。

**准则**：**安全 API 不能假设「对象一定会被 Drop」**。

直到 **Rust 1.63**，标准库用 **`thread::scope` 闭包** 重新实现：

- 子线程 handle **不能逃出** scope（类型系统保证）——`mem::forget(handle)` 也无法让子线程在 scope 外继续借栈（与 Leakpocalypse 根本差别）；
- **借用安全**由 **`'scope`** 生命周期编码，**不**假设「用户一定会 drop 某个 guard」；
- **自动 join** 仍由 std 在 scope 收尾时批量调用 `.join()`（常见实现走 `Scope` 析构；见 [1.2.3 §3](./1.2.3-join-auto-join.md#3-分层拆解自动-join-做了什么是不是封装系统调用)），底层与手动 join **同一条 OS 路径**。

---


1. **把 Enjoy 当 Join**：正确是 **`JoinHandle` / `.join()`**（见 [1.1.2 §五](../1.1-threads-in-rust/1.1.2-join-handle.md#五两个易混点)）。
2. **以为 scope = 可以随意并发写同一变量**：没有 `Mutex` / 原子类型时，多个 `&mut` 仍会被编译器拒绝。
3. **在 scope 里 `sleep` 很久却以为「不阻塞主线程」**：`thread::scope` **会阻塞调用它的线程**（通常就是主线程），只是帮你自动 join。
4. **纠结子线程打印顺序**：同一 scope 里多个 `s.spawn` 的执行顺序**不确定**，调试请加线程名或序号。
5. **scope 能借局部变量，就不用 `Arc` 了？** 仅当**所有线程一定在 scope 内结束**且只需**借用**时成立；线程要活得比函数更久 → 仍用 `spawn` + `Arc`（见 [1.3](../1.3-shared-ownership/1.3-shared-ownership.md)）。
6. **把「自动 join」当成 spawn 也有**：只有 **`thread::scope`** 在闭包结束时批量 join；**`thread::spawn` 必须手动**（见 [1.2.3 join / 自动 join](./1.2.3-join-auto-join.md)）。
7. **以为 auto join 是内核自动回收**：OS 不会等子线程；自动 = std 替你调 `.join()` → 仍走 `pthread_join` / `WaitForSingleObject`（见 [1.2.3 §3](./1.2.3-join-auto-join.md#3-分层拆解自动-join-做了什么是不是封装系统调用)）。
8. **把 `s` 当成关键字或全局变量**：`s` 只是 `&Scope` 的参数名，由 `thread::scope` 创建并传入（见 [1.2.5 参数 s](./1.2.5-scope-param-s.md)）。
9. **scope 自动 join 能拿到子线程返回值**：自动 join **丢弃 `Ok(T)`**；要 **`T`** 须在 scope 内手动 `.join()`（见 [1.2.3 §5](./1.2.3-join-auto-join.md#5scope-自动-join-时resultt-_-怎么处理)）。
10. **把 spawn 写成 swap**：创建线程是 **`spawn`**；泛型约束逐条见 [1.1.3](../1.1-threads-in-rust/1.1.3-send-static-spawn.md) 与 [1.2.4 约束对照](./1.2.4-scoped-join-handle.md#两段-spawn-泛型约束对照)。
11. **以为带生命周期就一定是引用**：`'static` / `'scope` 只约束**借用**须活够久；**`move` 转移所有权**无 `&`，同样合法（见 [1.2.4 §5](./1.2.4-scoped-join-handle.md#5-生命周期标注--传入的一定是引用)）。
