## 1.7.3 `MutexGuard<T>` 与作用域陷阱

> 书 §1.7 · 索引：[1.7-mutex-rwlock.md](./1.7-mutex-rwlock.md) · 前：[1.7.2 基本用法](./1.7.2-mutex-basics.md) · 后：[1.7.4 中毒](./1.7.4-lock-poisoning.md)

---

### 1. 是什么

- **`lock()`** → `Result<MutexGuard<T>, PoisonError>`
- **`MutexGuard`**：持锁标记 + 智能指针语义；**不能**安全地越过线程边界发送（guard 非 `Send` 的设计避免锁状态错乱）。

### 2. RAII 与作用域

```rust
let mutex = Mutex::new(0);
{
    let mut g = mutex.lock().unwrap();
    *g += 10;
} // 此处锁已释放
let g2 = mutex.lock().unwrap(); // 可再次获取
```

仓库对应：[1.7-mutex-rwlock-guard-lifetime-demo.rs](./code/1.7-mutex-rwlock-guard-lifetime-demo.rs) — `demo_guard_lifetime`

### 3. 锁作用域陷阱（高频）

```rust
// ❌ guard 活整个 if 块，锁占用过久
if let Some(item) = list.lock().unwrap().pop() {
    slow_work(item); // 仍持锁！
}

// ✅ 先 pop，再处理
let item = {
    let mut guard = list.lock().unwrap();
    guard.pop()
};
if let Some(item) = item {
    slow_work(item);
}
```

`if let`、`match`、`while let` 里直接 `lock()` 都会让 **guard 绑定到整个分支块**。

仓库对应：`demo_guard_lifetime_mistake`、`demo_guard_lifetime_correct`

### 4. 锁内不要 `sleep`

持锁期间 `thread::sleep` 会阻塞其他线程拿锁 → 并行变串行。应**尽快 drop guard** 再做耗时 I/O。
