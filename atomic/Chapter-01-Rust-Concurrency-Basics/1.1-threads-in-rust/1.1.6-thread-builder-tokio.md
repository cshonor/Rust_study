## 一、ThreadId

操作系统配合标准库自动为每一个运行的线程分配唯一的 ID 编号。

- **作用**：只用于日志打印、区分不同线程，支持等值对比；
- **限制**：编号不一定连续生成，无法通过 ID 实现关闭、操控线程。

## 二、三种创建线程方式

### 1. 最简：`thread::spawn`

语法要求：闭包 **`Send + 'static`**（常配合 **`move`**）。

```rust
use std::thread;

fn main() {
    let a = 100;
    let jh = thread::spawn(move || {
        println!("{}", a);
    });
    let _ = jh.join();
}
```

**短板**：无法自定义线程名、栈大小；`spawn` 失败会 panic。

仓库对应：[1.1-threads-in-rust-move-closure-demo.rs](./code/1.1-threads-in-rust-move-closure-demo.rs)

### 2. 自定义：`thread::Builder`

| 方法 | 作用 |
|------|------|
| `.name("…")` | panic / 调试器里可读 |
| `.stack_size(n)` | 自定义栈大小 |

```rust
let builder = thread::Builder::new()
    .name("worker".into())
    .stack_size(1024 * 8);
let jh = builder.spawn(|| { /* … */ })?; // Result，不直接 panic
```

> `thread::spawn` ≈ `Builder::new().spawn(...).unwrap()`。

### 3. 预览：`thread::scope`（§1.2 完整讲）

可**借用**局部变量；scope 结束**自动 join**。见 [1.2-scoped-threads.md](../1.2-scoped-threads/1.2-scoped-threads.md)。

## 三、新手易错：`println!` 打印错乱

`println!` 单次调用内部有锁，**一行内字节不会被打断**；但多线程之间**整行顺序**仍可能交错。

排错：`.name()`、`thread::current().id()`、结构化日志。

仓库对应：[1.1-threads-in-rust-example-demo.rs](./code/1.1-threads-in-rust-example-demo.rs)

## 四、对照：`std::thread::spawn` vs `tokio::spawn`

| | **`std::thread::spawn`** | **`tokio::spawn`** |
|---|--------------------------|---------------------|
| 执行体 | **OS 线程** + 同步闭包 | **async task** |
| 等待 | `.join()` **阻塞** OS 线程 | `.await` 挂起 task |
| 约束 | `Send + 'static` | `Future: Send + 'static` |
| 典型场景 | CPU 密集、阻塞 syscall | I/O 密集 |

```rust
let h = std::thread::spawn(|| 42);
let n: i32 = h.join().unwrap();

let h = tokio::spawn(async { 42 });
let n: i32 = h.await.unwrap();
```
