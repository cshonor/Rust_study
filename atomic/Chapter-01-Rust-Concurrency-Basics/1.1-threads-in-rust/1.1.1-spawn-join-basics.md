## 一、进程与线程通俗理解

1. **进程**：每一个独立运行的软件程序，各自拥有独立内存空间，进程之间天然隔离，没法直接互相读写数据，想要通信要依靠操作系统提供的特殊接口。
2. **线程**：依附在进程内部，**同一个进程里所有线程共用同一份内存空间**，这是多线程之间可以共享数据的根本原因。操作系统依靠高速切换 CPU 执行流，实现多个线程看似同时运行的效果。

Rust 不靠操作系统限制共享内存出错，而是**在编译阶段借助所有权、借用规则**，从语法层面阻止多线程非法争抢内存，规避数据损坏问题。

---

## 二、主线程生命周期关键规则

1. 程序从 `main` 函数所在的主线程启动。
2. **主线程执行完毕、函数 return，整个进程直接结束**，所有还在运行的子线程会被操作系统强制终止，不会等待子线程执行完。
3. `thread::spawn` 创建线程后会返回 `JoinHandle` 类型的值：
   - 调用 `handle.join()`：主线程暂停卡住，阻塞等待对应子线程运行结束之后，主线程才会继续向下执行；
   - 子线程正常运行结束：`join()` 返回 `Ok(子线程返回值)`；
   - 子线程运行中 panic 崩溃：`join()` 返回 `Err`，内部携带 panic 载荷；主线程可用 `downcast_ref` 提取消息 → 详见 [1.1.2 §2.1](./1.1.2-join-handle.md#21-panic-捕获与-downcast_ref)。

示例最简代码：

```rust
use std::thread;

fn main() {
    let jh = thread::spawn(|| {
        println!("子线程正在运行");
    });
    // 主线程阻塞，等待子线程结束
    let _ = jh.join();
    println!("主线程收尾");
}
```

仓库对应：[1.1-threads-in-rust-join-demo.rs](./code/1.1-threads-in-rust-join-demo.rs)

### 逐行导读

```rust
use std::thread;           // 标准库线程模块

fn main() {
    // spawn：闭包 = 子线程入口；返回 JoinHandle<()>
    let jh = thread::spawn(|| {
        println!("子线程正在运行");
    });

    // join：阻塞**当前**线程，直到 jh 对应子线程结束
    let _ = jh.join();       // Result<(), Box<dyn Any + Send + 'static>>

    println!("主线程收尾");
}
```

### 运行结果（有 `join` 时顺序固定）

```text
子线程正在运行
主线程收尾
```

### 去掉 `join()` 的两种现象

**仅 `println!`、无 sleep**——与主线程**竞态**，输出**不保证**：

```rust
thread::spawn(|| println!("子线程正在运行"));
println!("主线程收尾");
// 有时两行都有，有时只剩「主线程收尾」
```

**子线程 `sleep` 很久、`main` 立刻 return**——子线程几乎必被强杀（见 [1.1.2 §4](./1.1.2-join-handle.md)）。

### 补充

- Rust 默认 **`std::thread` = OS 原生线程**，不是 green thread / 协程；轻量并发见 [1.1.6 Tokio 对照](./1.1.6-thread-builder-tokio.md)。
