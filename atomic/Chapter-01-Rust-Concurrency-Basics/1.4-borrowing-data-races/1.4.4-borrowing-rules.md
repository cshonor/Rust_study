## 1.4.4 借用规则如何防数据竞争

> 书 §1.4 · 索引：[1.4-borrowing-data-races.md](./1.4-borrowing-data-races.md) · 前：[1.4.3 UB](./1.4.3-ub-unsound.md) · 后：[1.4.5 路线图](./1.4.5-sync-roadmap-pitfalls.md)

---

Rust 对**单线程内**的引用有两条铁律；**多线程完全沿用**（配合 [1.2 scope](../1.2-scoped-threads/1.2-scoped-threads.md) / **`Sync`**）：

| 借用 | 准确称呼 | 规则 |
|------|----------|------|
| `&T` | **共享借用** | 任意时刻可有**多个**；**只读** |
| `&mut T` | **排他借用** | 任意时刻**至多一个**；写时无人共读 |

**合并一句**：**共享读 OR 独占写，二者不能同时存在** —— 编译期排除「一边写、一边读/写」。

### 示例：scope 内多线程共享读 OK，并发写报错

```rust
use std::thread;

fn main() {
    let mut v = vec![1, 2, 3];
    thread::scope(|s| {
        // 多个 &T 共享读：合法
        s.spawn(|| println!("{:?}", v));
        s.spawn(|| println!("{:?}", v));

        // 编译错误：已有共享读，不能再要 &mut 写
        // s.spawn(|| v.push(4));
    });
}
```

Demo：[1.2-scoped-threads-demo.rs](../1.2-scoped-threads/code/1.2-scoped-threads-demo.rs)

### 新手必记：scope ≠ 放宽借用

| | `thread::scope` | 借用规则 |
|---|-----------------|----------|
| 解决什么 | **生命周期** — 可借局部变量、自动 join | **并发读写** — 禁止数据竞争 |
| 能否并发 `&mut` | ❌ **不能**因 scope 而放宽 | 仍须 Mutex / 单写者等 |

> **scope 管「引用活多久」；借用规则管「能不能同时读写」。**

### 与 `thread::spawn` + `move` 的关系

- **`spawn` + `move`**：每个子线程**独有**数据所有权 → **不共享**同一块可变内存 → **天然无**数据竞争；
- 一旦多线程要**共享同一份**数据 → [1.3 `Arc`](../1.3-shared-ownership/1.3-shared-ownership.md) + 本节规则 +（若要写）[1.7 `Mutex`](../1.7-mutex-rwlock/1.7-mutex-rwlock.md)。
