## 1.4.1 多线程共享内存为什么危险

> 书 §1.4 · 承接 [1.2 scope](../1.2-scoped-threads/1.2-scoped-threads.md) · 索引：[1.4-borrowing-data-races.md](./1.4-borrowing-data-races.md) · 后：[1.4.2 数据竞争](./1.4.2-data-race-definition.md)

---

同一进程内线程**共用地址空间**。若多个线程读写同一块内存却**没有协调（同步）**，就会出现 **数据竞争（Data Race）** → 在 Rust / C/C++ 中属于 **未定义行为（UB）**。

### 现象与致命点

| 表现 | 说明 |
|------|------|
| 值错乱、崩溃、偶现 bug | 难以复现、难调试 |
| 逻辑「诡异」 | 优化器可重排、删分支 —— **UB 之前**的代码也可能变味（见 [1.4.3](./1.4.3-ub-unsound.md)） |

编译器优化（指令重排、死码消除）会**放大**数据竞争 —— 结果**完全不可预测**，不是「偶尔算错数」那么简单。

### Rust 方案（分层）

1. **安全 Rust（默认）**：编译期借用规则，**从语法上禁止**典型数据竞争（本节主体）；
2. 必须绕开规则：**`unsafe`** — 开发者**人为保证**无 UB，责任自负。

### 与 §1.2 的分工

- **[1.2 `thread::scope`](../1.2-scoped-threads/1.2-scoped-threads.md)**：解决**生命周期** — 允许借局部变量、自动 join，**不管**并发读写是否合法；
- **§1.4（本节）**：解决**并发读写** — 不管是不是 scope，都不能「同时写 + 读/写」。
