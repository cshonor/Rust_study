## 虚假唤醒（Spurious Wake-up）

**`park` / `Condvar::wait` 可能在没有明确信号时也苏醒** → 条件检查必须在 **`while` / `loop`** 里，**不能**单独 `if`：

```rust
// ✅
while q.is_empty() {
    q = cv.wait(q).unwrap();
}

// ❌ 唤醒后队列可能仍空
if q.is_empty() {
    q = cv.wait(q).unwrap();
}
```

---

## `notify_one` vs `notify_all`

| 方法 | 行为 | 场景 |
|------|------|------|
| **`notify_one()`** | 唤醒**一个**等待线程 | 单消费者、单条件 |
| **`notify_all()`** | 唤醒**全部**等待者 | 多条件、广播、避免漏唤醒 |

被唤醒的线程仍须**重新检查条件**（可能多个线程同时被唤醒，或条件尚未满足）。

仓库对应：`demo_notify_methods`
