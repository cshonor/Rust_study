## 1.8.3 `Condvar` 基本模式

> 书 §1.8 · 索引：[1.8-parking-condvar.md](./1.8-parking-condvar.md) · 前：[1.8.2 park](./1.8.2-park-unpark.md) · 后：[1.8.4 虚假唤醒](./1.8.4-spurious-notify.md) · 深度：[Condvar与条件变量-贯通笔记.md](../Condvar与条件变量-贯通笔记.md)

---

**必须与 `Mutex` 绑定**。典型结构：

```rust
use std::sync::{Condvar, Mutex};
use std::collections::VecDeque;

struct Queue {
    inner: Mutex<VecDeque<u32>>,
    not_empty: Condvar,
}
```

### 消费者：`while` + `wait`

```rust
let mut q = queue.inner.lock().unwrap();
while q.is_empty() {
    q = queue.not_empty.wait(q).unwrap();
}
let item = q.pop_front().unwrap();
drop(q); // 尽早放锁再处理 item
```

**`wait(guard)` 原子地**：

1. **释放** `Mutex`；
2. 线程休眠；
3. 被唤醒后 **重新加锁**，返回新的 `MutexGuard`。

→ 避免「先 unlock 再 sleep 之间丢通知」的竞态。

### 生产者：`notify`

```rust
{
    let mut q = queue.inner.lock().unwrap();
    q.push_back(item);
}
queue.not_empty.notify_one(); // 或 notify_all()
```
