## 10.2.1 RCU 四步模式

> 书 §10.2 · 索引：[10.2-rcu.md](./10.2-rcu.md) · 后：[10.2.2 回收](./10.2.2-rcu-reclamation.md)

---

1. **Read**：原子 **`load`** 全局指针，读当前版本（只读，不阻塞写者准备）。  
2. **Copy**：在**私有副本**上修改。  
3. **Update**：**`compare_exchange`** — 仅当全局指针仍指向读到的旧版本时替换；失败则重读重试。  
4. **Reclaim**：等所有仍持有旧指针的读者结束后，再 **释放**旧内存（见 [10.2.2](./10.2.2-rcu-reclamation.md)）。

---

## 基石：`AtomicPtr<T>`

复杂数据在堆上，用 **`AtomicPtr<T>`** 原子替换「当前版本」：

```rust
// 读者
let p = global.load(Acquire);
// 只读 *p …

// 写者
let old = global.load(Acquire);
let new = Box::into_raw(new_data);
if global.compare_exchange(old, new, AcqRel, Acquire).is_ok() {
    // 延迟回收 old
} else {
    drop(unsafe { Box::from_raw(new) });
}
```

## 适用

| ✅ | ❌ |
|----|-----|
| 读极多、写很少 | 写频繁、指针 churn 高 |
| 读者可容忍短暂旧快照 | 必须强一致读 |

与 **SeqLock**（[10.6](./10.6-seqlocks.md)）、**RwLock**（[第 9 章](../Chapter-09-Custom-Locks/9.3-reader-writer-lock.md)）同属读优化族。
