## 8.1.2 FUTEX_WAIT（等待）

> 书 §8.1 · Waiting · 索引：[8.1-futex.md](./8.1-futex.md) · 前：[8.1.1 两阶段](./8.1.1-futex-two-phase.md) · 后：[8.1.3 WAKE](./8.1.3-futex-wake.md)

---

## 语义（口语）

**FUTEX_WAIT**（及变体）：

1. 检查用户态原子变量 `*addr` 是否仍等于 **期望值 `val`**；
2. **若相等** → 当前线程加入该 futex 的**内核等待队列**，**睡眠**（阻塞）；
3. **若不等** → **立即返回**（说明已有人改过，应回到用户态重试逻辑）。

→ 避免「没睡成却以为睡了」：只有值仍为期望时才睡。

---

## 与自旋锁对比

| | 自旋（第 4 章） | futex WAIT |
|---|----------------|------------|
| CPU | 一直占用，忙等 | 睡眠，让出 CPU |
| 适用 | 临界区极短、低争用 | 可能长时间等、高争用 |
| syscall | 无 | 有（慢路径） |

---

## 伪代码（Mutex 慢路径示意）

```text
loop:
  if CAS(lock, 0, 1) success → 获得锁，return
  // 锁已被持有
  FUTEX_WAIT(&lock, 1)   // 期望 lock==1，睡直到被 wake
  // 被唤醒后 loop 重试 CAS
```

书中 Mutex 实现细节见第 9 章；本章只建立 **WAIT = 按期望值睡眠** 的概念。

---

## 要点

- WAIT **不会**自己改用户态原子值（唤醒方或锁逻辑负责改）；
- 被唤醒后必须**重新检查**条件（类似 Condvar 的 `while` — [第 1 章 §1.8](../Chapter-01-Rust-Concurrency-Basics/1.8-parking-condvar/1.8.4-spurious-notify.md)）；
- 多个线程可在同一 futex 上 WAIT，形成等待队列。
