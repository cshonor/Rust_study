## 3.3.1 `Ordering::Relaxed`（宽松）

> 书 §3.3 · 前：[3.2 内存模型](./3.2-the-memory-model.md) · 索引：[3.3-memory-order-options.md](./3.3-memory-order-options.md) · 后：[3.3.2 Release/Acquire](./3.3.2-release-acquire.md)

---

| 项 | 内容 |
|----|------|
| **保证** | **无**跨线程 HB；仅同一原子变量的 **总修改顺序（Total Modification Order）** |
| **效果** | 可与其它内存操作重排；他线程可能先看到原子更新，仍看不到此前对普通内存的写 |
| **用于** | **Load**、**Store**、**RMW**（语法允许，但语义上通常不用于「信使」） |

---

## 适用场景

| ✅ 适合 | ❌ 不适合 |
|---------|-----------|
| 独立计数器（统计、进度） | 原子是「**数据已就绪**」标志 |
| 唯一 ID 分配（`fetch_add`） | 保护锁内普通数据的发布 / 订阅 |
| Stop Flag（只控制本线程是否退出循环） | 需要与别变量建立 HB 的任何同步 |

```rust
use std::sync::atomic::{AtomicUsize, Ordering};

static COUNTER: AtomicUsize = AtomicUsize::new(0);

fn bump() {
    COUNTER.fetch_add(1, Ordering::Relaxed);
}
```

第 2 章多数 demo 用 **Relaxed**：只关心**该原子**上的原子性与总序，不同步其它内存。

---

## 与 [3.1 反例](./3.1-reordering-and-optimizations.md) 的关系

```rust
data = 42;
flag.store(true, Ordering::Relaxed); // ❌ 信使场景禁止 Relaxed
```

**Relaxed 只保证 `flag` 本身的读写不可分割**，不保证 `data = 42` 在 store 之前对其它线程可见。

---

## 记忆

> **Relaxed = 最弱；只保单原子总修改顺序；独立统计用，信使禁用。**
