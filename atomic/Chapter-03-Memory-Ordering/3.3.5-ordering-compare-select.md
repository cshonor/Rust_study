## 对照表（必背）

| Ordering | 跨线程 HB | 重排限制（口语） | 典型场景 | 性能（相对） |
|----------|-----------|------------------|----------|--------------|
| **Relaxed** | ❌ 无 | 最弱；只保单原子总序 | 独立计数、进度、Stop Flag（不同步别内存） | 最省 |
| **Release** | ✅ 与配对 Acquire | 之前写不能排到 store 后 | 发布数据后写标志、**解锁** | 中等 |
| **Acquire** | ✅ 读到 Release 后 | 之后读不能排到 load 前 | 读标志后读数据、**加锁** | 中等 |
| **AcqRel** | ✅ RMW 双向 | 读 Acquire + 写 Release | `fetch_*` / CAS / `swap` 当同步点 | 中等 |
| **SeqCst** | ✅ + 全局 SeqCst 序 | 最强；所有 SeqCst 共一条时间线 | 多原子全局先后决策 | 最重（ARM 上尤甚） |

---

## 适用场景速查

| 你的需求 | 选什么 |
|----------|--------|
| 只改一个原子计数，不同步其它变量 | **Relaxed**（[3.3.1](./3.3.1-relaxed.md)） |
| 原子是「数据已就绪」信使 | **Release**（写）+ **Acquire**（读）（[3.3.2](./3.3.2-release-acquire.md)） |
| 自旋锁 / Mutex 语义 | lock：**Acquire**（或 AcqRel 的 swap）；unlock：**Release** |
| RMW 同时发布又订阅 | **AcqRel**（[3.3.3](./3.3.3-acqrel.md)） |
| 必须比较两个原子在**全局**的先后 | 才考虑 **SeqCst**（[3.3.4](./3.3.4-seq-cst.md)） |

---

## 选型口诀（弱 → 强）

```text
  信使原子同步普通内存？ → 禁止 Relaxed，用 Release/Acquire（或 AcqRel 的 RMW）
  仅单变量统计？           → Relaxed
  多原子全局时间线？       → 才考虑 SeqCst
```

与 [3.6 总结](./3.6-summary.md)、[3.5 误区](./3.5-common-misconceptions.md) 对照复习。
