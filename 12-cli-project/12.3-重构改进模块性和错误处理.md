# 12.3 重构改进模块性和错误处理

这一节的目标是把 `main` 里杂糅的职责拆开，并把错误处理变得更“对用户友好”：

- **关注分离**：`main.rs` 只负责启动程序；具体逻辑放到 `lib.rs`
- **配置聚合**：用 `Config` 结构体表达 `query/filename` 的关系
- **错误集中处理**：`Config::new` 与 `run` 返回 `Result`，让 `main` 统一打印错误并用非零状态码退出

对应 demo 项目：`12-cli-project/12.3-minigrep-refactor-demo/`

---

## 二进制项目的关注分离（惯用结构）

- `src/main.rs`
  - 收集参数
  - 构建 `Config`
  - 调用 `run`
  - 处理错误并 `process::exit(1)`
- `src/lib.rs`
  - `Config` 定义与 `Config::new`
  - `run(config)`：真正的业务逻辑

这样 `lib.rs` 里的逻辑更容易测试与复用；`main` 也更小、更清晰。

---

## demo 运行（Windows 记得 `--`）

```bash
cd 12-cli-project/12.3-minigrep-refactor-demo

# 参数不足：友好错误 + 非零退出码
cargo run

# 文件读取成功：打印内容
cargo run -- the poem.txt
```
