# 纯阅读路线（无并发/async 练习仓）

> **前提**：The Book（`00-Book/`）仍建议作**语法与所有权底座**——可与本路线并行或先刷完 1～10 章再按下列三书通读。  
> **刻意跳过**：[rust-concurrency-async-practice](https://github.com/cshonor/rust-concurrency-async-practice) 等外部实战仓；三书自成体系，不靠整套 demo 也能学通。

---

## 顺序（固定）

```text
① RFR 完整通读
      ↓
② Effective Rust 完整通读
      ↓
③ Rust Nomicon（Unsafe / 内存 / 黑魔法）
```

| 顺序 | 资源 | 本地入口 | 读什么 |
|:---:|------|----------|--------|
| **1** | **Rust for Rustaceans** | [`02-RFR/RFR-本书目录.md`](../02-RFR/RFR-本书目录.md) | 内存、类型、trait、异步原理、FFI 等**内功** |
| **2** | **Effective Rust** | [`01-ER/ER-本书目录.md`](../01-ER/ER-本书目录.md) | 35 条**工程习惯**与 API 设计 |
| **3** | **The Rustonomicon** | [`03-Rust_Nomicon/README.md`](../03-Rust_Nomicon/README.md) · [在线 Stable 版](https://doc.rust-lang.org/nomicon/) | **unsafe**、布局、型变、未初始化、FFI |

---

## 为什么是这个顺序？

1. **RFR → ER → Nomicon 递进**：先建立「Rust 怎么在机器上工作」的整体图景（RFR），再收束成可落地的写法清单（ER），最后下潜到 unsafe 边界与编译器/内存契约（Nomicon）。
2. **不依赖练习仓库**：三书 + 本仓库 `00-Book/`、`01-ER/`、`02-RFR/`、`03-Rust_Nomicon/` 笔记即可；并发/async 专题仓**以后**再补。
3. **与「Book 主线 + ER 穿插」不冲突**：若你更习惯先 ER 再 RFR，见 [`02-RFR/学习路径与章节对照.md`](../02-RFR/学习路径与章节对照.md) 里的「Effective Rust 先行」方案；**本文件记录的是你选定的纯阅读顺序**。

---

## 学习节奏建议

- **以看书、整理笔记为主**（本仓库 `topics/` 已扁平在各自 Item 目录下）。
- **只在概念易混处**写几行极简代码验证，例如：
  - `dyn Trait` 与 DST、`&dyn` / `Box<dyn>`
  - `Fn` / `FnMut` / `FnOnce` 向下兼容
  - 静态分发 vs 动态分发 / vtable
- **不必**为每章搭完整 Cargo 练习项目；Item 2 等已有 [`demo/`](../01-ER/Chapter-01-Types/Item-02-express-common-behavior/demo/) 时可 `cargo run` 对照，非必须。
