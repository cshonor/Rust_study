## 一、Cargo.lock 是什么

| 点 | 说明 |
|----|------|
| **自动生成** | `cargo build` / `cargo add` / `cargo update` 后生成/更新，位于**项目根**（Workspace 则在**工作区根**唯一一份） |
| **作用** | 锁定所有依赖的**精确版本** — 所有人、所有环境编译出一致二进制，避免版本漂移 |
| **与 TOML 分工** | `Cargo.toml` 写**兼容范围**（如 `"1.0"` → `^1.0`）；`Cargo.lock` 写**确定版本**（如 `1.0.228`） |

→ Workspace 内多子包共用根 lock：[03 §五](./03-workspaces.md#五cargolock-如何统一第三方版本)

---

## 二、提交规范（git）

| 项目类型 | 是否提交 lock | 原因 |
|----------|:-------------:|------|
| **二进制 / 应用**（`main.rs` / `[[bin]]`，对外交付可执行产物） | **必须提交** | 团队、CI、生产构建版本一致 |
| **纯库**（仅 `[lib]`，发布到 crates.io 供他人依赖） | **不要提交** | 下游**忽略**你的 lock，按各自 TOML 解析 |
| **lib + bin 同包** | **建议提交** | 有可运行入口时按应用对待；库部分仍被下游忽略 lock |

```gitignore
# 纯库项目：可忽略 lock（二进制项目删掉这行）
# Cargo.lock
target/
Cargo.toml.orig
```

## 三、完整示例 `Cargo.lock`

与 [`cargo-lock-demo/Cargo.lock`](./cargo-lock-demo/Cargo.lock) 一致（`serde = { version = "1.0", features = ["derive"] }` 解析出的传递依赖树）：

```toml
# This file is automatically @generated by Cargo.
# It is not intended for manual editing.
version = 4

[[package]]
name = "my-demo"
version = "0.1.0"
dependencies = [
 "serde",
]

[[package]]
name = "proc-macro2"
version = "1.0.106"
source = "registry+https://github.com/rust-lang/crates.io-index"
checksum = "8fd00f0bb2e90d81d1044c2b32617f68fcb9fa3bb7640c23e9c748e53fb30934"
dependencies = [
 "unicode-ident",
]

[[package]]
name = "quote"
version = "1.0.45"
source = "registry+https://github.com/rust-lang/crates.io-index"
checksum = "41f2619966050689382d2b44f664f4bc593e129785a36d6ee376ddf37259b924"
dependencies = [
 "proc-macro2",
]

[[package]]
name = "serde"
version = "1.0.228"
source = "registry+https://github.com/rust-lang/crates.io-index"
checksum = "9a8e94ea7f378bd32cbbd37198a4a91436180c5bb472411e48b5ec2e2124ae9e"
dependencies = [
 "serde_core",
 "serde_derive",
]

# … serde_core · serde_derive · syn · unicode-ident — 见 demo 全文
```

> 直接依赖 1 个（`serde`），lock 里出现 **7 个** `[[package]]` — 含 proc-macro 传递链。

## 四、字段逐行解释

### 1. 头部

```toml
version = 4
```

lock 文件**格式版本**（非 crate 版本）— 当前 Cargo 常见 v3/v4，由工具自动写入。

### 2. 每个 `[[package]]` 块

每一个块 = 一个参与编译的包（**直接 + 传递**依赖）：

| 字段 | 含义 |
|------|------|
| **`name`** | crate 名 |
| **`version`** | **锁定的精确版本**（重点） |
| **`source`** | 来源：`registry+…`（crates.io）· `git+…#commit` · `path=…` |
| **`checksum`** | 校验和，防篡改（registry/git 有；本地根包无） |
| **`dependencies`** | 该包自己的子依赖列表（名字，版本见对应块） |

### 3. 根项目包

`name = "my-demo"` 那块是**你的项目** — **无 `source` / `checksum`**，因为是本地源码。

```text
Cargo.toml: serde = "1.0"        →  范围 ^1.0
Cargo.lock: serde = "1.0.228"    →  本次解析落地版本
              + proc-macro2, quote, syn, …  →  传递依赖
```

## 五、常用操作与 lock 的关系

| 命令 | 对 lock 的影响 |
|------|----------------|
| `cargo add serde@1.0` | 改 TOML + **更新 lock**，锁定符合约束的最新版 |
| `cargo update` | **大量更新** lock — 在 TOML 约束内升到最新 |
| `cargo update -p serde` | **仅升级**指定包及其必要传递依赖 |
| `cargo build --locked` | **严格用 lock**，不自动升级；与 lock 不一致则**报错** — CI/发布推荐 |
| `rm Cargo.lock && cargo build` | 删 lock 重解析 — 按当前 TOML 拉**当下**最新兼容版 |

```bash
cd 02-RFR/Chapter-05-Project-Structure/cargo-lock-demo
cargo build --locked
cargo update -p serde
```

## 六、常见误区

| ❌ 误区 | ✅ 正解 |
|---------|---------|
| 手改 `Cargo.lock` | Cargo 会覆盖 — 用 `cargo add` / `cargo update` |
| 二进制项目不提交 lock | 成员/服务器版本不一致 → 「本地能跑线上崩」 |
| 纯库项目提交 lock | 下游不用；徒增仓库体积；本地 CI 可保留但不 push |
| lock = TOML 的副本 | TOML 是**范围**，lock 是**精确树** |
| 每个 member 各自 lock | **Workspace 只有根目录一份** lock |
| 复制他人 lock 不验证 | checksum 须与 registry 一致 — 用 `cargo build` 生成或 `--locked` 校验 |

## 七、TOML ↔ lock 对照案例（可直接复制）

### `Cargo.toml`（范围）

```toml
[package]
name = "my-demo"
version = "0.1.0"
edition = "2021"

[dependencies]
serde = { version = "1.0", features = ["derive"] }
```

### 对应关系

```text
TOML 直接依赖          lock 落地                         传递依赖（部分）
serde ^1.0         →  serde 1.0.228                 →  serde_core, serde_derive
                   →  serde_derive 1.0.228          →  syn → proc-macro2, quote, unicode-ident
my-demo 0.1.0      →  （根包，无 source）
```

完整可复制文件 → [`cargo-lock-demo/`](./cargo-lock-demo/)：

```bash
cd cargo-lock-demo && cargo build --locked
```

## 八、一句话速记

1. **TOML 提范围，lock 定精确** — 自动生成，别手改。  
2. **应用提交 lock，纯库不提交** — Workspace 根目录唯一一份。  
3. **CI 用 `--locked`** — 防意外升级。  
4. **升级用 `cargo update`** — 全量或 `-p` 单包。
