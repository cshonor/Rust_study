# 第 2 章：类型 (Types) —— 深度解析

《Rust for Rustaceans》第 2 章从**内存中的类型表示**出发，串联 **Trait 的分发机制**、**对象安全**、**相干性与孤儿规则**，并延伸到 **HRTB**、**标记特质** 与 **`impl Trait` 存在类型**。目标是建立对「零成本抽象」在机器层面的直觉，便于写高性能且安全的库代码。

---

## 1. 内存中的类型 (Types in Memory)

类型的主要作用：告诉编译器如何解释某段内存里的比特。

### 1.1 对齐 (Alignment)

- 硬件常以固定宽度（如 64 位平台上的 **字 / word**，常见为 8 字节）为粒度访问内存。
- **自然对齐**：类型的起始地址应满足该类型的 **alignment**（通常为 2 的幂；与类型大小相关但不必处处相等）。若值跨越了对齐边界，可能发生 **非对齐访问 (misaligned access)**：更慢，在部分架构上甚至 **未定义行为 (UB)**。
- **复合类型**：`struct` 等的对齐通常取各字段对齐要求的**最大值**；编译器会插入 **padding** 以满足每个字段的对齐。

### 1.2 布局 (Layout) 与 `repr`

| `repr` | 含义（直觉） |
|--------|----------------|
| **`repr(C)`** | 与 C 兼容：字段顺序固定，按 C 规则插入 padding，便于 FFI 与稳定 ABI。 |
| **`repr(Rust)`（默认）** | 编译器可重排字段等以优化大小/访问；**不保证**跨版本布局稳定。 |
| **`repr(packed)`** | 尽量去掉字段间 padding，体积小，但易产生未对齐字段访问；多用于协议解析或极端空间约束，需谨慎与 UB 风险。 |

### 1.3 动态大小类型 (DST) 与宽指针 (Wide Pointers)

- **`[T]`、`str`、`dyn Trait`** 等在编译期**大小未知**（DST）。
- **宽指针**：在机器层面常视为「两个指针宽」——**数据指针** + **元数据**（如 slice 的 **长度**；`dyn Trait` 的 **vtable 指针**），从而使「指向 DST 的引用」本身在编译期大小固定。

## 2. 特质与特质限定 (Traits and Trait Bounds)

Trait 描述不同类型之间的**可替换行为**与**互操作契约**。

### 2.1 分发 (Dispatch)

- **静态分发（单态化 / monomorphization）**：对每个具体类型生成一份特化代码。优点：**易内联、峰值性能好**；缺点：**编译时间、产物体积**可能上升。
- **动态分发 (`dyn Trait`)**：通过 **vtable** 在运行时解析方法地址。优点：**编译更快、二进制更省**（相对少份代码）；缺点：**间接调用**有固定开销。

### 2.2 对象安全 (Object Safety)

并非所有 Trait 都能写成 `dyn Trait`。若存在无法在 vtable 中统一表达的方法（例如泛型方法、返回 `Self` 且与对象大小相关等），该 Trait 往往 **非对象安全**，不能作为 trait object 的根。

### 2.3 相干性与孤儿规则 (Coherence & Orphan Rule)

为保证「某个类型对某个 Trait」在全局 **至多一个** 实现，Rust 限制 `impl Trait for Type` 的落点：通常要求 **Trait 或类型至少其一由当前 crate 定义**；另有「覆盖 / covered」等规则允许在本地类型出现在合适位置时，为外部类型与外部 Trait 组合实现部分标准模式（如 `impl From<MyType> for Foreign`）。

---

## 3. 进阶 Trait 限定

### 3.1 关联类型 vs 泛型参数

- **关联类型**：同一 `(Type, Trait)` 组合上通常 **只有一种** 关联类型选择（典型：`Iterator::Item`）。
- **泛型 Trait 参数**：同一类型上可对 **不同 RHS** 多次实现（典型：`PartialEq<Rhs>`）。

### 3.2 高阶 Trait 限定 (HRTB)

- 语法：`for<'a> ...`，表示「对**任意**生命周期 `'a`，某约束都成立」。
- 常见场景：闭包/函数要接受**任意生命周期的引用**时，用 HRTB 精确描述签名，避免生命周期写死过窄。

### 3.3 标记特质 (Marker Traits)

如 `Send`、`Sync`、`Copy` 等：**无（或几乎无）可调用方法**，主要向编译器声明**额外语义**（跨线程、共享引用安全、按位复制合法等），参与类型检查与优化假设。

---

## 4. 存在类型 (Existential Types) 与 `impl Trait`

- **`impl Trait`（在返回位置等）**：向调用方承诺「返回值是**某个**满足 `Trait` 的具体类型」，但**隐藏具体类型名**。
- 适用于：闭包组合、长迭代器链、`async` Future 等「类型难以书写或不稳定」的场景，在保持静态分发优势的同时改善 API 可读性（与 `dyn Trait` 的取舍需按场景判断）。

---

## 小结

第 2 章把 **对齐与布局**、**DST / 宽指针** 与 **Trait 的静态/动态分发、对象安全、孤儿规则** 连成一条线：你写的类型与 Trait 不仅是语法，更是**编译器与硬件如何协同解释内存与调用约定**的契约。要写高性能库，**对齐、`repr`、vtable 成本、对象安全边界** 值得刻意内化。
