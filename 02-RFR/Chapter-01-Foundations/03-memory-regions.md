# 1.3 Memory Regions（内存区域）

> 所属：**Talking About Memory** · [← 章索引](./README.md)

Rust 程序主要与三类区域打交道；它们决定**存活时间**与**可访问性**。

| 区域 | 角色 |
|------|------|
| **栈 (Stack)** | 函数调用工作区；每次调用压入**帧 (frame)**，返回时帧销毁。 |
| **静态内存 (Static)** | 程序启动前已布局；字面量、`static` 等。 |
| **堆 (Heap)** | 运行时动态分配；数据可跨调用存活，不受单帧限制。 |

原书还会提到**寄存器**等实现细节——对读 IR / ABI 有用，日常写 safe Rust 以「栈 / 静态 / 堆」三分类为主即可。

## 关于 `'static`

- 表示引用（或类型约束）**可以活得和程序一样久**——不依赖更短的生命周期参数。
- **不必然**等于「数据物理上在静态区」。
- 自包含的拥有型数据（如 `String`、`Vec<u8>`）也可满足 `'static` 约束。
