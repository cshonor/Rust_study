## 一、核心定义

生命周期是**约束引用保持合法有效的代码区间**的命名规则，**不等同于变量词法作用域**。

| | 变量作用域 | 引用生命周期 |
|---|------------|--------------|
| 是什么 | 从 `let` 到 `}` 的**文本范围** | 编译器追踪的**逻辑使用区间** |
| 结束时机 | 离开花括号（drop 所有者） | **最后一次使用**之后可提前结束（NLL） |
| 要求 | — | 使用引用时底层数据必须仍存活 |

```rust
{
    let x = 5;
    let r = &x;
    println!("{r}");  // 使用 r
}   // r 的生命周期在此前已结束；随后 x 被 drop
```

**大白话**：生命周期回答「这条借用能活到哪一行还在用」，不是「变量写在哪个 `{}` 里」。

---

## 二、NLL 非线性生命周期（Non-Lexical Lifetimes）

### 核心规则

只要一条引用在后续代码路径中**不再被读取、写入**，它的生命周期约束可以**提前终止**，不必等到词法作用域末尾。

带来的直观收益：同一块数据，可以在旧引用「逻辑死亡」后立刻创建可变引用，大幅放宽旧版 Rust 的借用限制。

### 关键区分（排坑要点）

读编译报错、分析生命周期时，务必分清两件事：

1. **值的所有权销毁**：变量离开作用域、执行 `Drop`、内存回收。
2. **引用生命周期结束**：仅代表该指针不再使用，**原数据依然存活**，可重新借用。

### NLL 示例

```rust
let mut s = String::from("test");
let r = &s;
println!("{r}");      // r 在此处最后一次使用 → 生命周期提前结束
let r_mut = &mut s;   // ✅ NLL 允许，不与 r 冲突
r_mut.push('!');
```

**若无 NLL**（旧模型）：`r` 要到 `}` 才结束，其间不能 `&mut s` — 许多今天合法的代码会报错。

### 与借用规则的关系

NLL **不**废除独占/共享规则，只让「借用结束点」更精确。

## 三、生命周期方差 Variance（本章难点）

方差控制泛型类型在生命周期/类型参数上的**子类型替换规则**，根本目的是杜绝悬垂引用、保证类型安全（**soundness**）。

→ 深入 + 源码：[Nomicon 03 · 型变](../../03-Rust_Nomicon/03_Lifetime_Variance/04-variance.md)

### 1. 协变 Covariant

子类型关系**同向**传递：若 `A` 是 `B` 的子类型，则 `F<A>` 也是 `F<B>` 的子类型（在型变允许的方向上）。

- **典型**：`&'a T` 对生命周期 `'a` **协变**
- **直观**：长生命周期引用可当作短生命周期使用（在共享引用语境下仍受借用规则约束）

```rust
let static_ref: &'static i32 = &0;
let local: i32 = 1;
{
    let short: &i32 = &local;
    // 'static 比 'short 长 → 协变允许把更长寿命当更短用（在合法借用内）
    let _use: &i32 = static_ref;
}
```

例：`&'static i32` 可赋给需要 `&'a i32` 的位置（`'a` 更短）。

### 2. 不变 Invariant

**完全不允许**子类型替换。

- **典型**：`&mut T` 对类型参数 `T` **不变**
- **安全逻辑**：若 `&mut T` 对 `T` 协变，可把短寿命对象写入被误认为长寿命的可变槽 → **悬垂引用**

**最小反例（为何 `&mut` 对 `T` 必须不变）**：

```rust
// 假设 &mut T 对 T 协变（实际不允许）—— 则可写：
let short = String::from("hi");
let long_slot: &mut &'static str = ...; // 误以为能存 'static
*long_slot = &short;                    // short 离开作用域后 → 悬垂
```

编译器通过**不变**禁止这类替换。详见 Nomicon [04-variance](../../03-Rust_Nomicon/03_Lifetime_Variance/04-variance.md)。

### 3. 逆变 Contravariant

子类型关系**反向**传递；常见于**函数参数**位置。

- **示例**：`fn(T)` 对 `T` **逆变** — 能接收「更宽」参数的函数，不能随意替换为「更窄」参数的函数。

日常写业务代码较少手写逆变，但理解 `Fn` trait、高阶生命周期时会碰到。

### 常见类型方差速查

| 类型构造 | 对 `'a` | 对 `T` |
|----------|---------|--------|
| `&'a T` | 协变 | 协变（多数 `T`） |
| `&'a mut T` | 协变 | **不变** |
| `Box<T>` | — | 协变 |
| `Vec<T>` | — | 协变 |
| `Cell<T>` / `RefCell<T>` | — | **不变** |
| `Mutex<T>` | — | 协变 |

> 完整表可单独立笔记；此处够应付 RFR 第 1 章与面试速答。

## 四、四大核心概念记忆锚点

| 概念 | 一句话核心概括 |
|------|----------------|
| 所有权 Ownership | 谁最终负责销毁、释放内存 |
| 借用 Borrow | 临时获取数据只读/可写访问权，不转移所有权 |
| 生命周期 Lifetime | 借用/引用的访问权必须保持有效的代码区间约束 |
| 方差 Variance | 限制泛型参数替换规则，防止越界借用、悬垂指针 |

---

## 五、常见误区

| 误区 | 纠正 |
|------|------|
| 「生命周期 = 作用域」 | 生命周期常**早于** `}` 结束（NLL） |
| 「标了 `'a` 就一定是某个具体 scope」 | `'a` 是编译器推导的**关系**，不一定是某一行 |
| 「方差是运行时概念」 | **纯编译期**；为 soundness 服务 |
| 「NLL 后可以随便 `&mut`」 | 仍须等**所有**冲突借用逻辑结束 |
| 「`&'static` 万能」 | 只说明引用**可活多久**；仍受借用/别名规则约束 |
