# 3.2 Mutable References（可变引用 `&mut T`）

> 所属：**Borrowing and Lifetimes** · [← 章索引](./README.md)

前置 → [05 共享引用 `&T`](./05-shared-references.md) · [04 所有权](./04-ownership.md)

---

## 一、核心语义

可变引用 `&mut T` 的两条硬性规则，是借用检查器的核心约束：

1. **独占借用**  
   同一块内存，在同一时间点，**不能同时存在任何冲突引用**：
   - 不能同时有第二个 `&mut T`；
   - 不能同时存在任意数量共享引用 `&T`。

2. **唯一可变路径保证**  
   强制任何时刻只有一条合法途径修改目标数据，从语法层面杜绝数据竞争，实现内存安全。

```rust
let mut x = 10;
let r = &mut x;
*r += 1;   // ✅ 通过独占路径修改
```

**大白话**：`&mut T` 是「这段时间只有我能改，别人既不能看（`&T`）也不能改（另一个 `&mut`）」。

---

## 二、对编译器优化的作用（no-aliasing）

`&mut T` 向编译器承诺：当前不存在其他指向同一块数据的有效引用，即 **no-aliasing（无别名）**。

基于该假设，编译器可以做激进优化：

- 指令重排、循环向量化；
- 消除重复读写内存操作（冗余 load / store）。

这是 Rust 兼顾内存安全、同时逼近 C 运行性能的关键特性之一。

```rust
fn double(r: &mut i32) {
    *r = *r * 2;   // 优化器可假定 r 无其它别名，不必保守重读内存
}
```

**为何重要（HFT / 性能）**：热路径里写 `&mut T`，LLVM 能按 **noalias** 做 store/load 优化 — 读 IR 见 [LLVM ch04](../../05_Compilers-and-LLVM-Learning/04_Learn-LLVM-17/part02_src_to_machine/chapter04_ir_basic/README.md)。

**与内部可变性的关系**：`RefCell` / `Mutex` 等会 opt-out 部分假设 — 见 [07 内部可变性](./07-interior-mutability.md)。

## 三、`&T` 共享引用 vs `&mut T` 可变引用

| 特性 | 共享引用 `&T` | 可变引用 `&mut T` |
|------|---------------|-------------------|
| 别名规则 | 允许多个同时共存 | 同一内存仅允许一个独占借用 |
| 修改数据权限 | 默认禁止修改内部数据 | 允许修改引用指向的数据 |
| 编译器优化假设 | 存活期内不会被突变（readonly） | 无其它别名，可激进优化（noalias） |
| 赋值时 | **Copy**（复制指针 + 生命周期） | **Copy**（同上） |

对**同一内存位置**在同一时刻：

| 组合 | 是否允许 |
|------|----------|
| 多个 `&T` | ✅ |
| 一个 `&mut T` | ✅ |
| `&T` + `&mut T` | ❌ |
| 多个 `&mut T` | ❌ |

## 四、典型非法代码（违反独占规则）

```rust
let mut x = 10;
let r1 = &mut x;
let r2 = &mut x; // ❌ 编译报错：同一内存存在两个可变引用

let r3 = &x;     // ❌ 编译报错：可变借用期间不能再有共享引用
```

编译器在**编译期**拒绝，不会留到运行时才发现数据竞争。

---

## 五、合法场景：借用生命周期分离

只要可变引用**作用域结束**（最后一次使用之后），就可以重新创建其它引用：

```rust
let mut x = 10;
{
    let r1 = &mut x;
    *r1 += 1;
} // r1 生命周期结束，独占借用释放

let r2 = &mut x; // ✅ 合法
let s1 = &x;     // ✅ 合法（多个 &T 也可）
```

**记忆**：冲突的是**同一时刻**的重叠借用，不是「曾经借过就永远不能借」。

---

## 六、`no-aliasing` 与 C `restrict`

| | **C `restrict`** | **Rust `&mut T`** |
|---|------------------|-------------------|
| 无别名保证 | 程序员**手动声明** | 借用检查器**强制校验** |
| 违反后果 | UB（编译器仍可能优化） | **编译失败**（常规 Safe 代码） |
| 优化收益 | 可能获得 noalias 类优化 | 同等优化假设，且默认安全 |

Rust 把「这段期间无其它别名」写进类型系统，不必像 C 那样靠约定和审查。

---

## 七、常见误区

| 误区 | 纠正 |
|------|------|
| 「`&mut` 不能 Copy」 | `&mut T` 赋值/传参是 **Copy**（复制指针），复制的是**借用权**，不是数据所有权 |
| 「出了块就能随便 `&mut`」 | 须等**所有**冲突借用结束；嵌套作用域要逐层分析 |
| 「`&mut` 和 C 的 `T*` 一样」 | `&mut` 有**生命周期 + 独占协议**；裸指针无 |
| 「noalias 与性能无关」 | 热循环里 `&mut` 直接影响 LLVM 是否消除冗余内存访问 |
| 「有 `&mut` 就能绕过内部规则」 | `&mut RefCell` 等仍受运行时 borrow 规则约束（见 07） |

## 八、和所有权的关系

| | 所有者 `T` | 可变引用 `&mut T` |
|---|------------|-------------------|
| 拥有数据 | ✅ | ❌（只借改） |
| 触发 Drop | ✅ | ❌（只结束借用） |
| 修改数据 | 通过绑定本身 | 通过解引用 `*r` |
