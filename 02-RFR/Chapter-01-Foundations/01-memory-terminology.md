# 1.1 Memory Terminology（内存术语）

> 所属：**Talking About Memory** · [← 章索引](./README.md)

进阶开发里若混淆 **值 / 变量 / 指针**，复杂借用错误会很难读懂。

## 值 (Value)

**类型**与该类型**定义域内某个元素**的组合。

- 例如：`u8` 的 `6` 是一个值。
- 意义独立于存储位置；可对应字节序列（如 `0x06`），但本质是**逻辑对象**。

## 变量 (Variable)

**存放值的地方**（容器 / 标签 / **具名内存区域**）。

变量**本身**是一块（或一组）有名字的存储；**值放在哪**由类型 + 声明方式决定：

| 写法 | 变量（控制块）在哪 | 值 /  payload 在哪 |
|------|-------------------|---------------------|
| `let x = 5` | 通常 **栈** 槽位 | 数字就在槽里 |
| `let b = Box::new(10)` | **栈** 上的 `Box` 结构体 | **`10` 在堆**；栈上存「指向堆的指针 + 元数据」 |
| `let s = String::from("hi")` | **栈** 上的 `String` 结构体 | **字节在堆**；栈上是 `{ ptr, len, cap }` |
| `static N: i32 = 0` | **Data / BSS**（进程生命周期） | 与变量同一静态区 |
| `&"hello"` 里的 `"hello"` | 无 `let` 变量名 | 字面量在 **只读段**（Text 旁 rodata） |

→ 分区细节见 [03 内存区域](./03-memory-regions.md)（Safe Rust → [03.1](./03-1-rust-memory-model.md)；OS/LLVM → [03.2](./03-2-os-memory-layout.md)）。

**注意**：说「变量在栈上」有时指**这个 `let` 绑定的槽**在栈上；若类型内含指针，**它指向的数据**可能在堆上（`Box`、`String` 都是这种「栈上控制块 + 堆上 payload」）。

## 指针 (Pointer)

其存储的值是**内存地址**的变量。

Rust 里的 **`&T` / `&mut T`** 在机器层面可理解为指针，但附加了语言规定的**协议**：

- 别名规则（共享 vs 独占）
- 对齐与**有效性 (validity)**
- 生命周期约束

### 借用、Box、裸指针 — 是不是「都是指针」？

**机器层面**：很多类型内部都**存地址**或**就是地址**，可以宽泛地说「和指针有关」。

**类型层面**：Rust **不会**把「指针」和「借用」画等号 — 它们是不同概念：

| | 是什么 | 所有权 | 编译器约束 |
|---|--------|--------|------------|
| **裸指针** `*const T` / `*mut T` | 纯地址 | 无 | 几乎无；操作在 `unsafe` |
| **借用** `&T` / `&mut T` | **带协议的指针** | **不拥有**数据；临时查看/修改 | 生命周期 + 别名规则（共享 / 独占） |
| **`Box<T>`** | **拥有堆内存的智能指针** | **拥有**堆上 `T`；drop 时释放 | 独占所有权，可移动 |
| **`String` / `Vec`** | 比 `Box` 更复杂的智能指针 | **拥有**堆上 buffer | 同上 + len/cap 等语义 |

串起来记：

```
裸指针     = 只有地址，自己负责合法
借用       = 地址 + 「我不能抢所有权、不能乱别名」
Box        = 地址 + 「这块堆归我，到期我释放」
String     = 地址 + len/cap + 「这块堆归我，到期我释放」
```

所以：**借用是指针的一种安全用法，但不是「指针 ≡ 借用」**；`Box` 也是指针类抽象，但是**拥有型**，不是借用。

## 赋值在说什么

**要点**：`=` 把右侧表达式的**值**写入左侧命名的**位置 (place)**。

区分「槽里换了什么比特」与「引用仍指向同一槽位」——后文在 [02 变量深入](./02-variables-in-depth.md) 用双模型展开。

## 你平时看到的 vs 底层真正是什么

Rust 里**指针相关**可以分成三层理解（由安全到裸）：

| 层级 | 例子 | 是什么 |
|------|------|--------|
| **安全包装** | `Box<T>`、`String`、`Vec<T>`、`&T` / `&mut T` | 编译器 enforced 的所有权 / 生命周期 / 别名规则 |
| **标准库内部** | `String` 里的 `ptr: *mut u8` + `len` + `cap` | 用 **`unsafe` + 裸指针 + `Drop`** 自己实现「类似 Box + 扩容」 |
| **裸指针** | `*const T`、`*mut T` | 和 C 的 `const T*` / `T*` 同级的**地址**；几乎无安全协议 |

**结论**：`String` **不是** `Box<str>` 类型，也**不是**借用；它是**栈上的结构体**，三个字段里**只有 `ptr` 是指针**，`len`/`cap` 是普通 `usize`。

```rust
// 概念上等价于标准库内部（简化，非逐字源码）
struct String {
    ptr: *mut u8, // 裸指针 → 堆上 UTF-8 字节
    len: usize,
    cap: usize,
}
```

- `String::from("hello")`：堆上分配字节 → 把**地址、len=5、cap=5** 写进**栈上**的 `String`。
- **`let s = String::from("hello")` 创建的是 `String` 结构体实例**（类型就是 `String`），不是「某种神秘指针类型」；只是这个结构体**设计得像**智能指针。
- 离开作用域：`Drop` 按 `ptr/cap` **释放堆** — 和 `Box` drop 释放内存**同一类逻辑**，只是 `String` 多管了 len/cap 与 UTF-8 语义。

`Vec<T>` 同样是 **`{ ptr, len, cap }` + 自定义 Drop**，不是 `Box<[T]>`。

## 裸指针 `*const T` / `*mut T`

| Rust | 近似 C/C++ | 含义 |
|------|------------|------|
| `*const T` | `const T*` | 只读裸指针 |
| `*mut T` | `T*` | 可写裸指针 |

- **所有数据类型的裸指针**都是这两类泛型实例：` *mut i32`、`*const u8`、`*mut String` ……
- **运行时**它们通常就是一个 **地址**（64 位下 often 8 字节），`T` 是编译期注解；和 `void*` 再强转类似，但 Rust 要求显式类型。
- **创建、解引用**裸指针要在 **`unsafe`** 里（或位于 `unsafe fn` 中），因为编译器**不**帮你保证：非 null、对齐、生命周期、无数据竞争。

```rust
let mut x = 42_i32;
let p: *mut i32 = &mut x as *mut i32; // 借用 → 裸指针（coerce 常在此步）
unsafe {
    *p += 1; // 像 C 一样解引用
}
```

和 **`&mut T` 的区别**：借用带**生命周期 + 别名规则**；裸指针**没有**，全靠你自己保证合法。

### 只有这两种吗？

**指向数据的标准裸指针类型就是 `*const T` 和 `*mut T` 两个家族**（泛型参数换 `T` 即可）。

其它相关、但不算「多一种数据裸指针」的：

| 类型 | 说明 |
|------|------|
| **`fn(...) -> R` / `fn(...) -> R` 函数指针** | 指向**代码**（Text 段），不是指向堆/栈数据的 `*mut T` |
| **`NonNull<T>`** | 封装「非 null 的 `*mut T`」，仍基于裸指针，便于 niche 优化 |
| **`usize` 存地址** | 纯整数，不是指针类型；和裸指针可 `as` 互转，更易踩 UB |

日常业务用 **`Box` / `String` / `Vec` / 借用**；HFT / 驱动 / 共享内存 / FFI 才在 **`unsafe` 里用 `*mut T`**，并自己承担 C 级内存责任。
