# 1.1 When to Use Declarative Macros（何时使用声明宏）

> 所属：**Declarative Macros** · [← 章索引](./README.md)

使用 **`macro_rules!`** 定义，调用带 **`!`**（如 `vec!`、`println!`）。

## 削样板

多份**结构相同、仅细节不同**的代码：

- 多组测试骨架
- 多类型相似 `impl`
- 重复的错误处理 / 日志模式

## 泛型 vs 宏（黄金法则）

| 优先 | 当… |
|------|-----|
| **泛型** | 差异主要是**类型参数**能表达的 — 惯例更好、错误信息更清晰 |
| **声明宏** | 差异在于**语法形状** / 重复片段的**结构本身**无法被泛型优雅覆盖 |

```rust
// 泛型够：同一逻辑，不同类型
fn id<T>(x: T) -> T { x }

// 宏更合适：输入「形状」不同（如多种 token 模式）
macro_rules! vec { /* ... */ }
```
