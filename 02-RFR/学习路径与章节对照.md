# 《Rust for Rustaceans》— 学习定位与章节对照

> **核心结论**：RFR 是**原理向深度进阶书**，应放在 **基础语法 + 网络/并发入门之后**，而不是语法刚过完就通读。  
> **对照**：《Effective Rust》= 写好代码的工具书（可早用）；**RFR** = 理解代码的内功（需实战后再吃透）。

---

## 一、与仓库其他板块的关系

| 板块 | 路径 | RFR 扮演的角色 |
|------|------|----------------|
| 原子与锁 | `atomic/` | 第 10 章 **Send/Sync、原子、内存序** ↔ 《Rust Atomics and Locks》 |
| 异步 | `async_tokio/` | 第 8 章 **Future、Pin/Unpin、轮询** ↔ Tokio 报错时的原理查阅 |
| 网络 | `rust_network_programming/` | 第 4 章错误、第 8 章 I/O 模型；写 TCP/UDP 遇坑再翻 |
| Linux | `TLPI/` | 第 11 章 **FFI**、第 9 章内存 ↔ syscall / C API 封装 |
| LLVM | `llvm_insight/` | 第 2 章 **布局/对齐**、第 9 章 unsafe ↔ IR 与机器码直觉 |

**关系一句话**：网络/async/locks 是**主战场**；RFR 是**底层支撑**——解释「为什么会这样」，并为 TLPI/LLVM 搭桥。

---

## 二、推荐四阶段（完整路径）

```
Rust 基础语法
  → 《Effective Rust》（最佳实践，现在就能用）
  → 网络 / async / locks（本仓库 atomic、async_tokio、rust_network_programming）
  → 《Rust for Rustaceans》（先按需查阅 → 再系统通读）
  → LLVM / TLPI（llvm_insight、TLPI/）
```

### 阶段 1：基础语法刚过完

- **用**：《Effective Rust》  
- **别碰**：RFR 通读（类型布局、trait coherence 无场景易懵）

### 阶段 2：学网络 / async / locks 时

- **用法**：RFR 当**辅助参考书**，**问题驱动**，不从头到尾读。  
- **对照**：见下表「何时翻哪一章」。

### 阶段 3：能写基础 TCP、用过 async 和锁之后

- **用法**：**系统通读** RFR 1～13 章，与已写代码对照。  
- **收获**：Send/Sync、Pin/Unpin、unsafe 与实战对齐。

### 阶段 4：LLVM / Linux 系统编程

- **用法**：RFR 作**底层衔接**（布局、FFI、no_std 可选）。

| 你遇到的问题 | 翻 RFR | 同时看仓库 |

## 五、与《Rust Atomics and Locks》的分工

| 主题 | 优先读 |
|------|--------|
| Mutex、Condvar、futex、无锁、RCU | **`atomic/`**（专书更深） |
| Send/Sync、并发模型总览 | **RFR 第 10 章**（建立地图）→ 再进 `atomic/` |
| 内存序、CAS 细节 | **Atomics and Locks**；RFR 第 10 章仅入门 |

避免两本书同一阶段「从头到尾并行通读」——**先实战 + 按需翻 RFR，Atomics 书与 `atomic/` 代码并行深化**。

## 六、一句话总结

- **现在**：语法阶段 → **Effective Rust**；写网络/async/locks → **本仓库主目录 + RFR 按需查表**。  
- **之后**：有 TCP/async/锁实战经验 → **通读 RFR** → **TLPI + llvm_insight**。  
- **本书笔记**：各章 [`*-深度解析.md`](./RFR-本书目录.md) 随读随补；目录总览见 [`RFR-本书目录.md`](./RFR-本书目录.md)。
