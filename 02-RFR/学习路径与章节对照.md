# 《Rust for Rustaceans》— 学习定位与章节对照

> **本仓库主线**：以**看书 + 整理笔记**为主；目录按 **`00` → `01` → `02` → `03` → `04`** 编号，便于顺序浏览。  
> **RFR 定位**：原理向进阶 — 建立「内存、类型、trait、unsafe」在机器上的图景；不是语法第一课。  
> **全书笔记入口**：[`RFR-本书目录.md`](./RFR-本书目录.md) · **纯阅读顺序**：[`docs/纯阅读路线.md`](../docs/纯阅读路线.md)

## 二、推荐学习顺序（与 README 一致）

### 主线：纯阅读三书

```text
（并行或先行）00-Book 1～10 章语法底座
      ↓
① 02-RFR 完整通读          ← 懂原理、内存、trait、async 模型
      ↓
② 01-ER 完整通读           ← 35 条工程习惯
      ↓
③ 03-Rust_Nomicon          ← unsafe 边界与内存契约
      ↓
④ 04_llvm_insight（按需）  ← 第 2 章布局、第 10 章原子 IR 对照
```

详见 [`docs/纯阅读路线.md`](../docs/纯阅读路线.md)。

### 与「先网络/async 再 RFR」的区别

| | **本仓库采用** | **另一常见路径**（未收录专题仓时仅作参考） |
|---|----------------|------------------------------------------|
| 顺序 | **RFR → ER → Nomicon** | 语法 → ER → 网络/async/locks 实战 → RFR 通读 |
| 实战 | Book / ER **demo** + 易混点几行验证 | 独立 `atomic/`、`async_tokio/` 等仓库 |
| 何时通读 RFR | **现在**（笔记驱动） | 有 TCP/async/锁经验后再通读 |

两种都合理；**以 `docs/纯阅读路线.md` 为准**，本表仅说明 RFR 在仓库里的默认位置。

### 阶段速记

| 阶段 | 做什么 | 别做什么 |
|------|--------|----------|
| **底座** | `00-Book` 所有权、trait、错误 | 不必刷完全部 20 章再开 RFR |
| **RFR 通读** | `02-RFR/Chapter-01`～`13` 按章 README 顺序 | 不必为每章搭大项目 |
| **ER 通读** | `01-ER` 35 Item；有 `demo/` 可 `cargo run` | 不必与 RFR 逐章 rigid 对齐 |
| **Nomicon** | `03-Rust_Nomicon/Stable` 为主 | Nightly 分支按需 |
| **LLVM 对照** | 改 `04_llvm_insight/src/lib.rs` 导出 `.ll` | 不必啃 LLVM C++ 源码 |
