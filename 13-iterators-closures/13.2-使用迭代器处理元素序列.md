# 13.2 使用迭代器处理元素序列

对应 demo 项目：`13-iterators-closures/13.2-iterators-demo/`

迭代器模式允许你对一个序列的项进行处理。迭代器负责：
- **遍历每一项**
- **在何时结束（返回 `None`）**

Rust 的迭代器是 **惰性的（lazy）**：在你调用会“消费”迭代器的方法之前，它不会做任何事。

---

## `Iterator` trait 与 `next`

所有迭代器都实现标准库的 `Iterator` trait，其核心是 `next`：
- `next(&mut self) -> Option<Self::Item>`
- 每次调用返回一个元素（`Some(...)`），结束时返回 `None`

因为 `next` 会推进内部状态，所以需要 `&mut self`（因此迭代器变量通常要 `mut`）。

---

## `iter` / `into_iter` / `iter_mut`

以 `Vec<T>` 为例：
- `iter()`：产生 `&T`（不可变引用）
- `iter_mut()`：产生 `&mut T`（可变引用）
- `into_iter()`：产生 `T`（获取所有权，迭代后原集合不能再用）

---

## 消费适配器（consuming adaptors）

这类方法会反复调用 `next`，并 **获取迭代器所有权**（调用后迭代器不能再用）。

例：`sum()` 会遍历并求和：

```rust
let total: i32 = v.iter().sum();
```

---

## 迭代器适配器（iterator adaptors）

这类方法会把一个迭代器“变成”另一个迭代器（例如 `map` / `filter` / `zip` / `skip`），它们 **仍然是惰性的**。

因此通常需要再调用一个消费适配器（如 `collect` / `sum`）才能得到结果。

```rust
let v2: Vec<_> = v1.iter().map(|x| x + 1).collect();
```

---

## `filter` + 捕获环境的闭包

`filter` 接收一个返回 `bool` 的闭包：
- 返回 `true`：保留该元素
- 返回 `false`：丢弃该元素

闭包可以捕获环境中的变量（例如目标鞋码 `shoe_size`）。

---

## 自定义迭代器：实现 `Iterator`

实现 `Iterator` 只需要实现 `next`。一旦有了 `next`，就能免费获得标准库提供的其它方法（`zip`、`map`、`filter`、`sum` 等的默认实现）。

demo 中会实现一个 `Counter`：依次产生 `1..=5`。
