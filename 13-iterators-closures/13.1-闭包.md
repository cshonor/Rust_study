# 13.1 闭包：可以捕获环境的匿名函数

对应 demo 项目：`13-iterators-closures/13.1-closures-demo/`

Rust 的 **闭包（closure）** 是可以保存到变量、也可以作为参数传给函数的匿名函数。与 `fn` 函数不同，闭包可以 **捕获其定义时所在作用域的值**（环境），从而更容易实现“在一处定义行为，在多处按需执行”的抽象。

---

## 使用闭包创建行为的抽象（按需计算 + 避免重复计算）

在“健身计划生成器”示例中，慢计算 `simulated_expensive_calculation` 会花几秒钟：
- 我们希望 **只在需要结果时才计算**（惰性）
- 并且 **相同输入不要重复算**（缓存/memoization）

### 先用变量提取结果的缺点

如果提前计算并保存到变量，确实消除了重复调用，但也会让“某些分支根本不需要结果时”仍然发生计算。

### 用闭包把“计算行为”存起来

闭包语法：
- 用 `|...| { ... }` 定义参数列表和闭包体
- 体只有一行时 `{}` 可省略
- 调用方式与函数类似：`closure(arg)`

```rust
let expensive_closure = |num| {
    println!("calculating slowly...");
    num
};
```

---

## 闭包的类型推断与可选标注

闭包不像 `fn` 那样强制标注参数/返回值类型，编译器通常可从上下文推断类型。

你也可以显式标注以提高可读性：

```rust
let add_one = |x: u32| -> u32 { x + 1 };
```

注意：**同一个闭包一旦根据第一次调用推断出参数类型，就会被“锁定”**，之后不能再用不同类型去调用。

---

## 用泛型 + `Fn` trait 存储闭包：`Cacher`

要把闭包放进结构体字段，必须写出字段类型；但闭包类型是匿名且各不相同的，所以通常用：
- **泛型参数 `T`**
- **trait bound：`T: Fn(...) -> ...`**

最简缓存（只缓存一个输入的结果）：
- `calculation: T` 保存闭包
- `value: Option<u32>` 保存结果（None/Some）

更通用的缓存（支持不同输入 arg）：
- 用 `HashMap<Arg, Ret>` 做缓存
- `Arg` 作为 key 需要 `Eq + Hash`

demo 里两种都会给出。

## 闭包捕获环境：`Fn` / `FnMut` / `FnOnce`

闭包捕获环境的方式，对应三类 trait：
- **`Fn`**：以不可变借用捕获（只读）
- **`FnMut`**：以可变借用捕获（会修改捕获的值）
- **`FnOnce`**：通过移动捕获并“消费”值（只能调用一次）

所有闭包至少实现 `FnOnce`；是否也实现 `FnMut`/`Fn` 取决于闭包体对捕获值的使用方式。

### `move` 闭包

在 `|...|` 前加 `move` 强制把捕获的值移动进闭包：
- 常用于把数据移动到新线程
- 代价是外层作用域不再能使用被 move 的值（如果它不实现 `Copy`）
