# 19.5 宏（Macros）

Rust 的宏是一种“**写生成代码的代码**”（元编程）。本节重点区分：

- **声明宏（declarative macros）**：`macro_rules!`（macros by example）
- **过程宏（procedural macros）**：三类
  - `#[derive(...)]` 自定义派生宏
  - `#[attr]` 类属性宏
  - `foo!(...)` 类函数宏

对应 demo（一个 workspace，包含多个 crate）：`19-advanced-features/19.5-macros-demo/`

---

## 宏 vs 函数：关键区别

- **参数更灵活**：宏可接受可变数量参数/更复杂的 token 结构；函数必须固定参数签名。
- **展开发生在编译期**：宏在编译器处理代码时“展开”成更多 Rust 代码；函数在运行期被调用。
- **能生成实现/定义**：宏可生成 `impl` 等需要编译期存在的代码；函数做不到。
- **代价**：宏更难写、更难读、更难调试；错误信息有时更绕。
- **作用域**：一般需要先定义/引入宏再使用（相对函数更“前置”）。

---

## `macro_rules!` 声明宏（以 `vec!` 思路为例）

`macro_rules!` 的模式匹配的是 **token 结构**（不是运行时值）。最常见的重复写法：

- `$( ... ),*`：零个或多个、逗号分隔的重复
- 捕获片段：`$x:expr`（表达式）、`$i:ident`（标识符）等

demo 里实现了一个精简版 `my_vec!`：`my_vec![1,2,3]` 展开成创建 `Vec` 并 `push` 的代码。

---

## 过程宏（proc-macro）：从 TokenStream 生成代码

过程宏像“编译期函数”：

- 输入：`TokenStream`
- 输出：`TokenStream`

通常配合：

- `syn`：把 token 解析成语法树（AST）
- `quote`：把 AST/模板再生成 Rust token

### 自定义 derive 宏（demo 跑通项）

实现 `#[derive(HelloMacro)]`，为类型生成：

```rust
impl HelloMacro for TypeName {
    fn hello_macro() {
        println!("Hello, Macro! My name is {}", stringify!(TypeName));
    }
}
```

### 类属性宏 / 类函数宏（demo 给出最小骨架）

这两类与 derive 的核心一致：都是 `TokenStream -> TokenStream`，差别主要是“宏挂在哪里、输入 token 是什么”。
