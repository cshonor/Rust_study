# 4.1 什么是所有权？

Rust 通过**所有权系统**管理内存，编译器在编译期检查，运行时零开销。

对应 demo 项目：`04-ownership/4.1-ownership-demo/`

---

## 栈 (Stack) 与堆 (Heap)

Rust 里的栈、堆与 C/C++、主流语言是**同一套概念**——都在**内存 (RAM)** 里，和硬盘无关。

### 栈：后进先出 (LIFO)

**存取规则**：最后放进去的，最先拿出来。函数调用、局部变量默认在栈上。

| 特点 | 说明 |
|------|------|
| 空间固定、连续 | 分配/回收极快 |
| 编译期确定大小 | 只存**固定大小**的数据 |
| 函数结束自动清理 | 栈上变量全部销毁，无需手动管理 |

**Rust 栈上常见类型**：`i32`、`bool`、`char`、引用 `&T`、元素均为固定大小的元组等。

### 堆：无序存取

**存取规则**：没有进出顺序，随时在任意位置分配/释放，靠**指针（地址）**找到数据。  
类比：栈像排队取号，堆像大型仓库——东西随便放，靠标签查找。

| 特点 | 说明 |
|------|------|
| 空间不连续 | 分配/回收比栈慢 |
| 运行时动态申请 | 适合**大小不确定、可变**的数据 |
| 不随函数结束自动清理 | 所有权、内存安全要解决的核心 |

**Rust 堆上常见类型**：`String`、`Vec` 等长度可变的类型。

- **栈上**：存指针、容量 (cap)、长度 (len)——这三项大小固定
- **堆上**：存真实的字符/元素字节——大小随内容变化

```rust
let s = String::from("hello");
// 栈：s 变量本身（ptr + len + cap）
// 堆："hello" 的字节数据
```

### 常见误区：堆 ≠ 硬盘

- 栈、堆都属于**内存**，断电数据丢失
- 硬盘是持久化存储，介质不同

### 栈/堆与所有权的关系

| 位置 | Rust 行为 |
|------|-----------|
| 栈上数据 | 复制成本极低，直接**拷贝**（如 `i32`），无所有权转移 |
| 堆上数据 | 占用大，**同一时间只有一个所有者**；所有者离开作用域自动释放堆内存 |
| 共享使用 | 用**引用 `&`** 借用栈上的指针，不转移所有权 |

**一句话**：栈快、固定大小、自动清理；堆慢、动态大小、靠所有权/引用管理生命周期；所有权本质是为**堆内存**服务的。

---

## 所有权规则

1. 每个值有且仅有一个**所有者**（变量）
2. 当所有者离开**作用域**，值被丢弃（调用 `drop`）

**核心理解**：堆上的值有且仅有一个所有者；所有者出作用域 → 堆内存自动释放。栈上简单值（`i32`/`bool` 等）大多走 **Copy 拷贝**，所有权规则体现得不明显——**所有权主要约束堆数据**。

---

## 三类常见值：所有者与作用域

### 1. 函数内局部变量（最常用）

`main` 或普通函数里 `let` 声明的变量。

- **作用域**：从定义位置开始，到当前代码块 `{}` 结束
- **规则**：变量是绑定数据的**所有者**；出作用域 → 所有者销毁 → 堆内存自动释放

```rust
fn main() {
    let s = String::from("hello"); // s 是 String 的所有者
    println!("{}", s);
} // 代码块结束，s 失效，堆上字符串内存自动释放
```

### 2. 静态变量 `static`（全局长期存在）

**位置规则**：`static` **只能写在模块顶层**（文件最外层、`mod` 内部顶层），**不能**写在函数或代码块内部。

```rust
// ✅ 合法：文件顶层
static NUM: i32 = 100;
static MSG: &str = "全局静态字符串";

fn main() {
    println!("{}", NUM);
    println!("{}", MSG);
    // ❌ 非法：函数内部不能定义 static
    // static INNER: i32 = 200;
}
```

| 要点 | 说明 |
|------|------|
| 声明 | `static 变量名: 类型 = 值;` |
| 作用域 | **整个程序运行期间**存在，不受局部 `{}` 影响 |
| 生命周期 | `'static`（最长） |
| 所有权 | 不属于某个局部变量；程序启动即存在，退出才回收 |
| 可变性 | 默认不可变；要修改需 `mut`，且有额外安全限制 |

**`const` vs `static`**：

| | `const` | `static` |
|---|---------|----------|
| 存储 | 编译期内联到代码，无独立内存地址 | 内存中分配固定空间，程序全程持有 |
| 书写位置 | 语法允许函数内定义，业务上一般也放顶层 | **必须**模块顶层 |
| 用途 | 编译期常量 | 全局常驻数据 |

### 3. 字符串字面量 `"abc"` 与 `&'static str`

所有裸字符串字面量，类型默认就是 **`&'static str`**：

```rust
let s = "hello world"; // 引用只读段里的字面量
```

| 要点 | 说明 |
|------|------|
| 存储位置 | 程序**只读数据段**，不是栈、也不是堆 |
| 生命周期 | `'static`，和程序同寿命 |
| 书写位置 | **任意位置**都能写——顶层、函数内、代码块里均可 |

#### `let s = "xxx"` 不是两个变量

```rust
let s = "xxx";
```

1. **`"xxx"`**：字符串**字面量**，存于只读数据段，生命周期 `'static`。它**不是变量**——编译期就存在，没有所有者、不会被销毁。
2. **`s`**：函数内**局部引用变量**（类型 `&'static str`），在栈上，作用域仅限当前代码块。`s` 只是借用永久存在的 `"xxx"`，不拥有这份数据。

> `"xxx"` = 永久不拆的房子；`s` = 临时门禁卡——卡过期（出作用域）不影响房子本身。

```rust
static GLOBAL_STR: &str = "全局字面量";

fn main() {
    let local_str = "函数内字面量"; // 完全合法
    println!("{}", GLOBAL_STR);
    println!("{}", local_str);
}
```

**和 `static` 的区别（重点）**：

| | `static VAR: &str = "xxx"` | `let s = "xxx"` |
|---|---------------------------|-----------------|
| 变量本身 | 全局静态引用变量，程序全程常驻 | 局部引用变量，出 `{}` 失效 |
| 指向的数据 | 只读段的字面量，`'static` | 同样是只读段字面量，`'static` |

> 引用变量有作用域，但它指向的**字面量本体**在程序运行期间一直存在。

**极易混淆的两种写法**：

| 写法 | 存储 | 所有权 |
|------|------|--------|
| `String::from("xxx")` | 堆上分配 | 变量**拥有**所有权，出作用域释放 |
| `"xxx"` 字面量 | 只读数据段 | 只有引用，**无所有权转移** |

### 4. `const` 常量（无所有权）

**核心结论**：`const` **不存在所有权概念**，也不会因离开作用域被「销毁释放」。

| 特性 | 说明 |
|------|------|
| 编译期求值 | 值直接嵌入代码，运行时**无单独内存实体**，只是字面替换 |
| 书写位置 | 可写全局顶层，也可写函数/代码块内（与 `static` 不同） |
| 出作用域 | 只是**不能再访问**该名字，不存在释放内存——本就没有运行时内存 |
| 与所有权 | 所有权、借用、堆释放针对**运行时栈/堆**；`const` 全程编译期，不参与 |

```rust
fn main() {
    const COUNT: i32 = 100;
    println!("{}", COUNT);

    {
        const NUM: i32 = 200;
        println!("{}", NUM);
    }
    // 出块后不能访问 NUM，但不是「释放所有权」
    // println!("{}", NUM); // ❌ 超出作用域
}
```

### `let` / `const` / `static` 对比

| | `let` 局部变量 | `const` 常量 | `static` 静态变量 |
|---|----------------|--------------|-------------------|
| 运行时内存 | 栈/堆，有实体 | 无（编译期内联） | 全局内存，程序全程 |
| 所有权 | **有**（堆数据） | **无** | **无传统所有权** |
| 出作用域 | 销毁/释放 | 仅不能访问 | 不受影响（顶层） |
| 书写位置 | 任意 `{}` 内 | 顶层或 `{}` 内 | **仅模块顶层** |

### 场景速查

| 场景 | 书写位置 | 变量作用域 | 数据生命周期 | 堆释放 |
|------|----------|------------|--------------|--------|
| 局部 `let`（堆） | 任意 `{}` 内 | 当前代码块 | 堆数据随所有者 | 出块自动释放 |
| `let s = "xxx"` | 任意 `{}` 内 | `s` 随代码块 | 字面量本体 `'static` | 不释放（只读段） |
| `const` | 顶层或 `{}` 内 | 名字随代码块 | 编译期内联 | 无运行时内存 |
| `static` | **仅模块顶层** | 程序全程 | `'static` | 程序退出才回收 |

### 避坑

1. **Copy 类型不谈转移**：`i32`、`bool` 等纯栈数据赋值直接复制，不说「转移所有权」。
2. **作用域决定引用变量何时失效**：局部 `let s = "xxx"` 出 `{}` 后 `s` 不可用，但 `"xxx"` 数据仍在只读段。
3. **`&'static str` 是类型标签**，不限制字面量写在哪；限制位置的是 `static` 关键字。
4. **`const` 出作用域 ≠ 释放内存**，只是名字不可见。

**极简记忆**：`let s = "xxx"` 只有一个局部变量 `s`，`"xxx"` 是永久字面数据；`const` 无所有权，出块只是不能访问；`static` 必须顶层。

## String 类型（堆上可变字符串）

- **`String::from("hello")`**：可变，堆分配，变量拥有所有权，出作用域 `drop` 释放堆内存（RAII）
- **`"hello"` 字面量**：不可变，只读数据段，见上文第三类

```rust
let mut s = String::from("hello");
s.push_str(", world!");
```

---

## 移动 (Move)

```rust
let s1 = String::from("hello");
let s2 = s1;  // s1 被移动，不再有效！
// println!("{}", s1);  // ❌ 错误: borrow of moved value
```

- 只复制栈上的指针、长度、容量，**不**复制堆上数据
- 为使内存安全，Rust 使 **s1 无效**，称为**移动**（不是浅拷贝）
- 避免二次释放 (double free)

---

## 克隆 (Clone)

需要深拷贝堆数据时用 `clone()`：

```rust
let s1 = String::from("hello");
let s2 = s1.clone();
println!("{} {}", s1, s2);  // ✓ 两者都有效
```

---

## Copy 类型（栈上拷贝）

栈上、大小固定的类型实现 `Copy` trait，赋值时**拷贝**而非移动：

```rust
let x = 5;
let y = x;
println!("{} {}", x, y);  // ✓ 两者都有效
```

**实现了 Copy 的类型**：所有整数、bool、f32/f64、char、仅含 Copy 元素的元组（如 `(i32, i32)`）

- `String` 未实现 Copy（需堆分配）
- 实现 `Drop` 的类型不能实现 `Copy`

---

## 所有权与函数

- 传参 = 赋值：会**移动**或**拷贝**
- `String` 传入函数 → 所有权移动，调用后原变量无效
- `i32` 等 Copy 类型 → 拷贝，调用后仍可用

```rust
let s = String::from("hello");
takes_ownership(s);  // s 被移动，s 不再有效

let x = 5;
makes_copy(x);       // x 被拷贝，x 仍有效
```

---

## 返回值与所有权

- 返回值会转移所有权
- 可用元组返回多个值，顺带把所有权“还回去”：

```rust
let (s2, len) = calculate_length(s1);
fn calculate_length(s: String) -> (String, usize) {
    let length = s.len();
    (s, length)  // 返回 s，转移所有权
}
```

## 小结

| 操作 | String (堆) | i32 (Copy) |
|------|-------------|------------|
| 赋值 | 移动，原变量失效 | 拷贝，两者有效 |
| 传参 | 移动 | 拷贝 |
| clone | 深拷贝 | 通常不需要 |
