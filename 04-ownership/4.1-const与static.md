# Rust `const` 常量 vs `static` 静态变量

**核心结论**：`const` 是编译期字面替换，无独立内存；`static` 是运行时全局内存变量，有专属内存地址。二者语法、用途、规则差异很大。

对应 demo：`04-ownership/4.1-ownership-demo/`（§0.6）  
所有权背景见：[4.1-什么是所有权.md](./4.1-什么是所有权.md)

---

## 一、基础语法与位置

### 1. `const` 常量

- 写法：`const 名称: 类型 = 值;`
- 定义位置：**全局顶层、函数内、代码块内都可以**
- 命名惯例：全大写，下划线分隔

```rust
const MAX: i32 = 100;

fn main() {
    const MSG: &str = "hello";
    println!("{}", MAX);
    println!("{}", MSG);
}
```

### 2. `static` 静态变量

- 写法：`static 名称: 类型 = 值;`，可变需 `static mut`
- 定义位置：**只能在模块/文件顶层**，禁止写在函数、代码块内部
- 命名惯例：同样推荐全大写

```rust
static GLOBAL_NUM: i32 = 200;
static mut GLOBAL_MUT: i32 = 0;

fn main() {
    println!("{}", GLOBAL_NUM);

    unsafe {
        GLOBAL_MUT = 10;
        println!("{}", GLOBAL_MUT);
    }

    // ❌ 非法：函数内不能定义 static
    // static INNER: i32 = 99;
}
```

---

## 二、核心本质（最大区别）

### `const`

编译阶段**直接把值嵌入代码**，运行时**没有独立的全局内存地址**。  
相当于代码里的「文本替换」——用到它的地方，编译器直接塞对应值。

### `static`

程序加载时**分配一块全局内存**，运行时存在**唯一内存地址**，整个程序共享，直到程序退出才释放。

> `const` = 试卷上直接写死的标准答案；  
> `static` = 教室角落的公共储物柜，所有人都能读写。

---

## 三、可变性

| | `const` | `static` |
|---|---------|----------|
| 默认 | **永远不可变**，无 `const mut` | 默认不可变 |
| 可变 | 不支持 | `static mut`，访问/修改须 **`unsafe`**（全局可变有线程安全风险） |

---

## 四、取值与引用

| | `const` | `static` |
|---|---------|----------|
| 内存地址 | 无固定的 global 存储，编译期内联 | 有唯一全局地址 |
| 持久引用 | 需要 `&'static T` 时应改用 `static` | `&GLOBAL_NUM` 为 `&'static T` |
| 生命周期 | 无独立生命周期实体 | 数据生命周期固定为 `'static` |

> 语义上：`const` 没有「专属 static 存储」；`static` 的引用始终指向那块全局固定内存。

---

## 五、赋值规则

初始值必须是**编译期可计算的常量表达式**，不能用运行时变量：

```rust
let a = 10;
const B: i32 = a;  // ❌ a 是运行时变量
static C: i32 = a; // ❌ 同样报错
```

---

## 六、作用域与访问

**`const`**：遵循普通作用域——定义在代码块内，出块后不能再访问；无内存释放，只是「看不见了」。

**`static`**：全局可见，整个项目任意位置都能访问，不受局部 `{}` 限制。

---

## 七、使用场景

### 用 `const`

- 固定配置、魔法数字、固定文本
- 只需固定值，不需要全局共享内存、不需要稳定 global 地址
- 想放在函数内部局部使用

### 用 `static`

- 需要**全局共享同一块内存**（全局状态、全局配置）
- 需要获取数据地址、创建 `&'static` 引用
- 极少数全局可变（优先线程安全类型，少用 `static mut`）

---

## 八、速查表

| 对比项 | `const` 常量 | `static` 静态变量 |
|--------|--------------|-------------------|
| 内存 | 无独立内存，编译期内联 | 全局独立内存，有内存地址 |
| 定义位置 | 全局/函数/代码块均可 | 仅文件/模块顶层 |
| 可变性 | 永久不可变 | 可 `static mut`（需 `unsafe`） |
| 持久引用 | 无 global 存储，需 `'static` 时用 `static` | `&VAR` → `&'static T` |
| 生命周期 | 无独立实体 | `'static` |
| 访问范围 | 受局部作用域限制 | 全局任意访问 |

---

## 结合所有权

- **`const`**：无栈/堆运行时内存，**完全不涉及所有权、借用规则**
- **`static`**：全局内存，不属于任何局部变量，**没有常规所有权转移**，仅有借用/访问限制

---

## 九、`const` vs 字符串字面量 `&'static str`

**结论**：二者**不是一回事**。都不占用栈/堆常规内存，但存储与行为不同。

### 统一概念

| | 字符串字面量 `"abc"`（`&'static str`） | `const` 常量 |
|---|----------------------------------------|--------------|
| 存储 | **只读数据段**（程序加载时载入，有固定地址） | **编译期文本替换**，运行时**无专属内存** |
| 运行时 | 真实存在的只读内存 | 用到处直接替换成值，无单独「const 变量」实体 |

### 字符串字面量：只读段有真实内存

```rust
fn main() {
    let s = "hello";
    println!("{:p}", s); // 可以打印地址，指向只读段
}
```

- `"hello"` → 只读数据段（有内存、有地址、运行时常驻）
- `s` 是栈上引用，指向这块只读内存

### 数值 `const`：编译期内联

```rust
const NUM: i32 = 100;

fn main() {
    let a = NUM;
    println!("{}", a);
}
```

编译后等价于：

```rust
fn main() {
    let a = 100;
    println!("{}", a);
}
```

全程没有一块叫 `NUM` 的运行时内存，只是字面替换。

> `&NUM` 在 Rust 里有时能编译，但引用的是编译器生成的**临时副本**，**不是** `NUM` 的专属固定地址——因为 `NUM` 本身就不占独立存储。

### `const` 存字符串：两层拆解

```rust
const TXT: &str = "rust";

fn main() {
    let s = TXT;
    println!("{:p}", s);
}
```

1. 底层 `"rust"` 本体：仍在**只读数据段**（和普通字面量一样）
2. `const TXT` 本身：编译期替换；写 `TXT` 编译后直接变成 `&"rust"`

> **是字符串字面量在只读段，不是 `const` 在只读段。**  
> `const` 只是别名，帮你简写代码。

### 核心区别

| 对比项 | `&'static str` 字面量 | `const` |
|--------|----------------------|---------|
| 存储本质 | 运行时有独立只读内存，有地址 | 编译期替换，无自身内存 |
| 取地址 | 引用可打印地址，指向只读段 | 无专属存储可指 |
| 生命周期 | 自带 `'static` | 无独立生命周期实体 |

**通俗比喻**：
- 只读段的字符串 = 书里**印好的固定文字**
- `const` = 给这段文字起的**外号**——喊外号直接对应原文，外号本身不是新文字

**一句话**：字符串字面量真真切切在只读数据段；`const` 只是编译期名字替换。哪怕 `const` 包裹字符串，也只是引用已存在的只读段数据，`const` 自身依然无独立内存。
