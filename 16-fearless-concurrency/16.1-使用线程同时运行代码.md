# 16.1 使用线程同时运行代码

对应 demo 项目：`16-fearless-concurrency/16.1-threads-demo/`

进程（process）内可有多个**线程（thread）**并发执行。并发会带来调度顺序不确定、**竞态**、**死锁**等问题；Rust 在编译期尽量降低数据竞争风险，但多线程编程仍需谨慎设计。

---

## 1:1 与 M:N

- **1:1**：一个语言线程对应一个 OS 线程（Rust 标准库采用）。
- **M:N**（绿色线程）：M 个语言线程映射到 N 个 OS 线程，需要更大的**语言运行时**。

Rust 追求**小运行时**与易与 C 互操作；若需要 M:N，可选用社区 crate。

---

## `thread::spawn` 与 `JoinHandle`

- `thread::spawn(closure)` 创建新线程；**主线程结束时，子线程可能被提前终止**（未 `join` 时）。
- 返回值 **`JoinHandle<T>`**：调用 **`handle.join()`** 会阻塞当前线程直到子线程结束；`join` 返回 `Result<T, E>`（子线程 panic 时为 `Err`）。

`join` 写在 **`main` 里 `for` 循环之后**与**写在循环之前**，会改变主线程与子线程是否交替运行。

---

## `move` 闭包

`thread::spawn` 的闭包可能活得比当前栈帧久，因此不能默认持有可能失效的**借用**。

- 使用 **`move || { ... }`** 把捕获变量**移入**闭包，保证子线程独占所有权。
- 若主线程在 `move` 之后仍使用被移动的值，会编译失败（如再 `drop(v)`），这正是所有权规则在保护你。
