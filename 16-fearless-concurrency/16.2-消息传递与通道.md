# 16.2 使用消息传递在线程间传送数据

对应 demo 项目：`16-fearless-concurrency/16.2-mpsc-demo/`

**消息传递（message passing）**：线程通过发送**带数据的消息**协作，而不是先共享内存再同步。Go 文档中的说法：**不要通过共享内存来通信；而要通过通信来共享内存。**

Rust 标准库中的 **`mpsc::channel`**：**多生产者，单消费者**（multiple producer, single consumer）。可有多份 `Sender`，通常只有一个 `Receiver` 消费。

---

## 创建与类型

`let (tx, rx) = mpsc::channel();` 返回 `(发送端, 接收端)`。通道会**推断**元素类型（首次 `send` 或显式标注 `channel::<T>()`）。

任一端被 drop 后通道可视为关闭；`send` 在接收端已关闭时会返回 `Err`。

---

## `send` / `recv` / `try_recv`

- **`send(value)`**：取得 `value` 的**所有权**并传给接收端（移动语义，见下）。
- **`recv()`**：阻塞直到收到一条消息，或发送端全部关闭则 `Err`。
- **`try_recv()`**：不阻塞，立刻返回 `Ok` 或“暂无消息”。

也可把 **`rx` 当迭代器**使用：`for msg in rx { ... }`，在通道关闭后结束。

---

## 所有权与 `send`

`send` 会**移动**值进通道，发送后发送者线程不能再使用同一变量（示例 16-9 会编译失败），避免并发下误用已交给别处的数据。

---

## 多生产者：`Sender::clone`

`let tx1 = tx.clone();` 得到第二个发送端，可分别 `move` 进不同 `spawn`，共享同一 `Receiver`（`rx` 只能被一个线程持有，通常主线程迭代接收）。

---

## 运行 demo

```bash
cd 16-fearless-concurrency/16.2-mpsc-demo
cargo run
```

demo 中发送间隔略短于书中的 1 秒，便于重复运行；逻辑与教材一致。
