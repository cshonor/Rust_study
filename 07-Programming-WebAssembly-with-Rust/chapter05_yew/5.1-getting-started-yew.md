# 5.1 Getting Started with Yew

## 一句话

**Yew** = Rust 写的类 React/Elm 前端：**虚拟 DOM + 组件 + 消息驱动状态**；用 **`html!` 宏** 写 UI、**`Component` + `update(Msg)`** 管逻辑 — 首个练手是**计数器**。

---

## 原书要点

### Yew 框架基础

| 概念 | 说明 |
|------|------|
| **设计灵感** | **Elm** + **React** |
| **虚拟 DOM (Virtual DOM)** | Rust 侧 diff，最小化真实 DOM 更新 |
| **组件化 (Components)** | UI 拆成可复用单元 |
| **消息传递** | 事件 → **`Msg` enum** → `update()` 改状态 → 重渲染 |

### 统一技术栈

| Ch4 之前 | Yew |
|----------|-----|
| Rust Wasm + 手写 JS/HTML 胶水 | **纯 Rust** 写 UI + 逻辑（仍编译到 Wasm） |
| JS/HTML/Rust 三角边界痛苦 | 单语言 SPA，保留 Rust **内存安全**与性能 |

↔ **Layer 3**：把「聊天 UI」换成 **回测仪表盘**（K 线、权益曲线、MA 参数）→ [layer03](../../layer03_quant-ma-strategy/README.md)

### 核心 Trait（原书 2019 版 API）

| Trait | 职责 |
|-------|------|
| **`Component`** | 业务逻辑 · 状态 **`Model`** · **`update()`** 处理 **`Msg`** |
| **`Renderable`** | 根据状态生成虚拟 DOM / HTML 视图 |

> **API 演进备注**：新版 Yew（0.19+）合并/重命名了部分 trait（如 `html!` + `fn view` 模式）。读原书代码时对照 [Yew 文档](https://yew.rs/) 当前版本；**概念**（Msg / update / 组件）不变。

### 计数器应用（首个 Yew 应用）

**目标**：熟悉 Yew 工作流 — 点击按钮增减数字。

#### `html!` 宏

在 Rust 里写类 HTML 语法（类似 React JSX）：

```rust
html! {
    <div>
        <p>{ self.count }</p>
        <button onclick={ ctx.link().callback(|_| Msg::Increment) }>
            { "+" }
        </button>
    </div>
}
```

→ 宏展开为虚拟 DOM 节点，由 Yew 渲染到浏览器。

#### 状态与消息

```text
用户点击 → Msg::Increment / Msg::Decrement
         → Component::update() match Msg
         → 修改 self.count
         → 自动触发视图刷新
```

| 元素 | 作用 |
|------|------|
| **`enum Msg`** | 所有 UI 事件的强类型枚举 |
| **`update()`** | `match msg` 安全改状态（无 JS 回调地狱） |
| **`Model` / 字段** | 组件私有状态 |

### 与 Ch4 的衔接

| Ch4 | Ch5 |
|-----|-----|
| wasm-bindgen 底层互操作 | Yew **内置** Wasm + Web 抽象 |
| 手写 DOM / web_sys | **`html!`** 声明式 UI |
| Rogue 引擎在 Rust | 整页 SPA 在 Rust |
