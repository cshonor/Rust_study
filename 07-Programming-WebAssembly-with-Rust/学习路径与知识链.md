## 与高频 / 性能优化的衔接

本书不是 HFT 教材，但下列点与 **低延迟、可预测性、沙箱化策略** 直接相关：

| 主题 | 笔记入口 | 与 HFT 的关联 |
|------|----------|---------------|
| **紧凑内存布局 · 位标志** | [2.2 data-structure-constraints](./chapter02_wasm_checkers/2.2-data-structure-constraints.md) | 棋盘状态用位图 — 同思路可用于订单簿档位、标志位、热路径 cache 友好结构 |
| **栈式 VM · 无 GC 运行时** | [1.2 wasm-architecture](./chapter01_wasm_fundamentals/1.2-wasm-architecture.md) | 可预测执行、无 STW — 对照 native 与 Wasm 插件沙箱 |
| **模块隔离 · 能力安全** | [appendix A2](./appendix/A2-security.md) · [6.1 good-host](./chapter06_nonweb_hosts/6.1-good-host.md) | 策略/指标以 Wasm 插件加载 — 边界清晰、崩溃不拖垮宿主 |
| **非 Web 宿主 · 嵌入式** | [chapter07_iot/](./chapter07_iot/README.md) · [8.3 match-engine](./chapter08_waros/8.3-match-engine.md) | 同一套 Rust 逻辑编译到 ARM / 边缘节点 — 与 **05-network** 低延迟链路可组合 |
| **原子 / 内存序**（前置） | [05-01-atomic](../05-Async-Concurrency-Network/01-atomic/README-学习区.md) | Wasm 线程模型与 native 不同；先掌握 **05** 再在笔记里写「哪些优化只能留在 native」 |

**实践顺序（偏性能向）**：

```text
05-01-atomic（内存序、无锁直觉）
  → 07 ch1～2（线性内存 + Checkers 位布局）
  → 07 ch6（Rust 宿主加载 Wasm — 插件模型）
  → 06 Learn LLVM 17（同一 Rust 函数：.wasm vs .ll vs 机器码）
  → 05-03-network（若做远程 Wasm 服务 / edge）
```
