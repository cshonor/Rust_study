## 一句话

用 **`lazy_static` + 读写锁** 持有全局引擎、**`Into<i32>`** 把 `GamePiece` 压成 Wasm 类型、**`extern "C"` export/import** 对接 JS — Rust 严格可变全局与 Wasm 单例状态之间的桥梁。

---

## 原书要点

### 处理全局可变状态

| 问题 | 书中方案 |
|------|----------|
| Wasm 模块需**全局棋盘** | 单例 `CheckersEngine` |
| Rust 禁止随意 `static mut` | **`lazy_static`** + **`RwLock`**（书中亦提到 **`mut_static`** 等等价思路） |
| 并发 | 宿主单线程调用 export 时 `read()` / `write()` 即可 |

概念代码：

```rust
lazy_static! {
    static ref ENGINE: RwLock<CheckersEngine> = RwLock::new(CheckersEngine::new());
}

#[no_mangle]
pub extern "C" fn get_piece(x: i32, y: i32) -> i32 {
    let engine = ENGINE.read().unwrap();
    engine.get_piece(x, y).into()  // Into<i32>
}
```

> **现代替代**（笔记用）：`std::sync::OnceLock<RwLock<...>>`（Rust 1.70+）— 原书 2019 年版用 `lazy_static` 仍有效。

### 类型转换与导出

| 方向 | 做法 |
|------|------|
| **Rust → Wasm export** | 复杂 `GamePiece` 实现 **`Into<i32>`**，export 只暴露 `i32`（与 Ch2 位标志兼容） |
| **参数** | `get_piece(x,y)`、`move_piece(...)` — 全 **`i32`**，与 JS Number 对接 |
| **隐藏内部** | 不 export `set_piece` 等 — 同 [2.5 防御性 export](../chapter02_wasm_checkers/2.5-testing-wasm-checkers.md) |

### 导入宿主函数

```rust
extern "C" {
    fn notify_piecemoved(x: i32, y: i32);
    fn notify_piececrowned(x: i32, y: i32);
}

// 在 move 逻辑成功后：
unsafe { notify_piecemoved(to_x, to_y); }
```

| 点 | 说明 |
|----|------|
| **`extern "C"`** | 声明 **import**，链接时期望宿主提供 |
| **`unsafe` call** | 外部函数 Rust 无法验证 — 见 [Nomicon · FFI](../../../04-Rust-Nomicon/README.md) |
| **命名空间** | 链接到 JS 的 **`env`** 模块（见 [3.6](./3.6-rusty-checkers-js.md)） |
