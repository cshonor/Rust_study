# 3.6 Playing Rusty Checkers in JavaScript

## 一句话

JS 宿主 **`fetch` 加载 Rust 版 `.wasm`**，在 **`env`** 命名空间挂上 `notify_*` import，实例化模块后即可驱动 Rust 跳棋引擎 — 完成 Ch2 wast 同款 UI 链路。

---

## 原书要点

### 加载 Rust 编译的 Wasm

| 步骤 | 说明 |
|------|------|
| **构建** | `cargo build --target wasm32-unknown-unknown --release` |
| **部署** | 将 `.wasm` 放到静态资源路径 |
| **fetch** | `fetch('rusty_checkers.wasm')` → `arrayBuffer()` |

### WebAssembly 实例化（概念）

```javascript
const imports = {
  env: {
    notify_piecemoved: (x, y) => { /* 更新 UI */ },
    notify_piececrowned: (x, y) => { /* 音效等 */ },
  },
};

WebAssembly.instantiateStreaming(fetch('rusty_checkers.wasm'), imports)
  .then(({ instance }) => {
    instance.exports.init_board();
    instance.exports.move_piece(/* ... */);
  });
```

### 命名空间：`env`

| Ch2 wast | Ch3 Rust |
|----------|----------|
| 自定义 import 模块名 | Rust **`extern "C"`** 默认链到 **`env`** |
| JS 需与 wast 一致 | 前端 import 对象键改为 **`env: { ... }`** |

> 若 instantiate 报 **「 unknown import」**，对照 `wasm-objdump -i` 的 module 名与 JS `imports` 键。

### 与 Layer 2 同构

| 跳棋 Ch3 | Layer 2 订单簿 |
|----------|----------------|
| JS `fetch` **.wasm** | JS `fetch` **Go 订单簿 API** |
| Wasm 算走子 | Wasm **parse + best bid/ask** |
| `env.notify_*` | 可选回调 / console 更新 |
