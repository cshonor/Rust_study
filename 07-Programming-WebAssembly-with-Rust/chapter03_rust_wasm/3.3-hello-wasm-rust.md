## 一句话

**Cargo 库 + `cdylib`**，写 `add_one` 并 **`#[no_mangle]`** 导出 → `cargo build --target wasm32-unknown-unknown` → 用 **`wasm-objdump`** 检视线性内存与 export。

---

## 原书要点

### 使用 Cargo 创建库项目

```bash
cargo new hello_wasm --lib
```

`Cargo.toml` 关键配置：

```toml
[lib]
crate-type = ["cdylib"]
```

| `crate-type` | 作用 |
|--------------|------|
| **`cdylib`** | 生成 C 风格动态库 → Wasm 工具链产出 **`.wasm` 模块**（含 export） |
| 默认 `rlib` | 仅 Rust 静态链接，不适合直接给 JS 加载 |

### 编写与导出函数

```rust
#[no_mangle]
pub extern "C" fn add_one(x: i32) -> i32 {
    x + 1
}
```

| 属性 / 关键字 | 原因 |
|---------------|------|
| **`#[no_mangle]`** | 防止 rustc 改名（如 `_ZN...`），宿主按 **`add_one`** 查找 export |
| **`pub extern "C"`** | C ABI，参数/返回值映射为 Wasm **`i32`** 等简单类型 |
| **`i32` 参数** | 与 [1.2 · 四种 Wasm 类型](../chapter01_wasm_fundamentals/1.2-wasm-architecture.md) 一致 |

### 编译与验证

```bash
cargo build --target wasm32-unknown-unknown --release
wasm-objdump -x target/wasm32-unknown-unknown/release/hello_wasm.wasm
```

检查项：

| 段 / 项 | 期望 |
|---------|------|
| **Export** | `add_one` |
| **Memory** | 通常有线性内存（即使用 `std` 也可能分配） |
| **Code 大小** | 远大于 Ch2 853B — 含 runtime 片段 |

↔ [1.3 · wasm-objdump](../chapter01_wasm_fundamentals/1.3-building-wasm-app.md)
