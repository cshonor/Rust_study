## 一句话

用 **`struct` / `enum` / 迭代器 / 模式匹配** 重写 Ch2 跳棋逻辑，并把**领域代码与 Wasm 边界分离**——引擎可脱离 Wasm 单测，Wasm 层只做薄封装。

---

## 原书要点

### 与第 2 章对比

| Ch2 · wast | Ch3 · Rust |
|------------|------------|
| `offset = (x+y*8)*4` 手动计算 | `Coordinate { x, y }` |
| 位标志 0/1/2/4 | `GamePiece` / `PieceColor` **enum** |
| 手写 `$move` 验证 | **`legal_moves`** 迭代器 + `match` |
| ~853B | 更大 `.wasm`（std + 抽象成本） |

### 利用高级数据结构

书中典型类型（名称随源码略有出入）：

| 类型 | 用途 |
|------|------|
| **`PieceColor`** enum | 黑 / 白 |
| **`Coordinate`** | 棋盘坐标 |
| **`Move`** | 一次走子（from / to，或含跳吃） |
| **`GamePiece`** | 颜色 + 是否加冕等 |

底层仍可映射回 **`i32` 位标志**（与 Ch2 兼容），但源码层不再手写 AND/OR。

### 迭代器与模式匹配

| 能力 | 跳棋中的用法 |
|------|--------------|
| **迭代器 `.filter` / `.map`** | 枚举当前玩家所有 **`legal_moves`** |
| **`match` / `if let`** | 分支：普通走 vs 跳吃 vs 加冕 |
| **组合子风格** | wast 里需数十条 `br_if` 的逻辑，Rust 数行可读 |

### 隔离核心逻辑（重要设计模式）

```text
┌─────────────────────────────────────┐
│  domain/   CheckersEngine           │  ← 纯 Rust，无 wasm 属性
│            单元测试 cargo test       │
└─────────────────┬───────────────────┘
                  │ 薄适配层
┌─────────────────▼───────────────────┐
│  wasm_api/  export get_piece, move  │  ← [3.5](./3.5-rusty-checkers-ffi.md)
└─────────────────────────────────────┘
```

**Layer 2 订单簿** 同样拆分：

- `orderbook/` — 解析快照、best bid/ask（可 `cargo test`）
- `wasm_api/` — `extern "C"` export → [layer02](../../layer02_orderbook-query-wasm/README.md)
