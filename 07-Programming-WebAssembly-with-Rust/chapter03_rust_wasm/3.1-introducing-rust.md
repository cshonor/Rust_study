## 一句话

前两章手写 **wast** 之后，本章引入 **Rust**：用强类型、内存安全与表达力写 Wasm——**Safety · Expressivity · Performance** 三件套，特别适合做 Wasm 模块的源语言。

---

## 原书要点

### 本章转折点

| 之前（Ch1～2） | 之后（Ch3+） |
|----------------|--------------|
| 纯 WAT/wast · 手动 offset / 位运算 | **Rust + Cargo** · struct/enum/迭代器 |
| ~853B 跳棋模块 | 更大体积，但**开发效率与可维护性**大幅提升 |
| 直接理解 VM | 清楚**编译器在背后做了什么**（与 Ch2 对比） |

### Rust 为何适合 WebAssembly

| 特性 | 说明 | Wasm 侧收益 |
|------|------|-------------|
| **Safety 安全性** | 无 null；所有权防数据竞争；无 GC 泄漏典型模式 | 模块边界更可控；FFI 处仍须 `unsafe`（见 [3.5](./3.5-rusty-checkers-ffi.md)） |
| **Expressivity 表现力** | 模式匹配、解构、**迭代器**、代数类型 | 跳棋 `legal_moves` 等规则可读（见 [3.4](./3.4-rusty-checkers.md)） |
| **Performance 性能** | 零 GC 运行时；可接近 C 级性能 | 适合计算密集 Wasm；体积仍大于手写 wast（含 panic/allocator 等） |
