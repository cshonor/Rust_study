## 一句话

用 **Rot.js**（JS）做迷宫/A* 与键盘输入，**Rust Wasm** 做引擎/玩家/战斗 — 通过 **`wasm_bindgen` 双向绑定**：Rust **import** `ROT.Display.draw`，**export** `PlayerCore` / `Engine` 给 JS 实例化。

---

## 原书要点

### 真实项目中的职责分离

| 层 | 语言 | 负责 |
|----|------|------|
| **表现 + 算法库** | **JavaScript** | 键盘事件 · **Rot.js** 随机迷宫 · A* 寻路等 |
| **核心模拟** | **Rust (Wasm)** | 引擎状态 · 玩家 · 战斗 · 胜负逻辑 |

原则：**每种语言做它最擅长的事** — 与 Layer 2「JS fetch 订单簿、Wasm 解析」同构 → [layer02](../../layer02_orderbook-query-wasm/README.md)

### Rot.js 整合

| 点 | 说明 |
|----|------|
| **Rot.js** | 开源 JS **Roguelike / 终端** 库（Display、Map 等） |
| Rust **import JS 类** | `#[wasm_bindgen(module = "...")]` 绑定 **`ROT.Display`** |
| 调用 | Rust 内直接 `.draw(x, y, char, fg, bg)` 等 — bindgen 生成跨边界调用 |

概念（方向示意）：

```rust
#[wasm_bindgen]
extern "C" {
    #[wasm_bindgen(js_class = ROT.Display, js_namespace = ROT)]
    type Display;
    #[wasm_bindgen(method, js_name = draw)]
    fn draw(this: &Display, x: i32, y: i32, /* ... */);
}
```

（实际以书中 / `wasm-bindgen` 导出的 JS 类型为准。）

### Rust 结构体暴露给 JavaScript

| Rust 类型 | JS 侧 |
|-----------|--------|
| **`PlayerCore`** | `new PlayerCore()` · 方法调用 |
| **`Engine`** | 游戏主循环、状态机 |

`#[wasm_bindgen]` 在 struct / impl 上 → 生成 **JS class**，替代 Ch3 纯 `i32` export。

### 趣味设定（原书）

| 元素 | 含义 |
|------|------|
| **宝藏** | 一个 **WebAssembly 模块** |
| **终极 BOSS「Checko」** | **Borrow Checker（借用检查器）** — Rust 开发者梗 |
