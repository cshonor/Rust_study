## 一句话

引入 **`wasm-bindgen`**：用 `#[wasm_bindgen]` 自动生成 JS 包装器，**字符串与复杂类型**可跨边界传递，再配合 **npm + webpack** 走现代前端脚手架——告别 Ch3 手写 `extern "C"` 与纯 `i32`。

---

## 原书要点

### 为何需要 wasm-bindgen

| Ch3 痛点 | bindgen 解决 |
|----------|--------------|
| 手写 `#[no_mangle]` + `extern "C"` | 宏 + CLI **自动生成** JS glue |
| 只能传 **数字** | **String**、结构体、方法可映射到 JS |
| 手动 linear memory 分配/指针 | 模板代码处理 **分配、释放、指针** |
| JS `env` import 易错 | 生成类型安全的 **wrapper** |

### wasm-bindgen 组成

| 部分 | 作用 |
|------|------|
| **`#[wasm_bindgen]` 宏** | 标记 export/import 函数、impl、类型 |
| **`wasm-bindgen` CLI** | 读 rustc 元数据，产出 **`.js` + 辅助 wasm** |
| **`wasm-pack`**（常用） | 封装 build + 发布到 npm 本地包 |

典型 Hello（概念）：

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn greet(name: &str) -> String {
    format!("Hello, {}!", name)
}
```

→ JS 侧可直接 `greet("Wasm")`，无需自己写 UTF-8 进 linear memory。

### 突破数据类型限制

| 方向 | bindgen 行为 |
|------|--------------|
| **`&str` / `String`** | 自动 copy 进/出 Wasm 线性内存 |
| **结构体 + 方法** | 生成 JS **class** 包装（见 [4.2](./4.2-rogue-wasm-game.md)） |
| **复杂类型** | 仍受 Wasm 类型系统约束；更复杂场景用 [4.3 · serde](./4.3-experimenting-further.md) |

### 现代前端工程化

| 工具 | 用途 |
|------|------|
| **Node.js + npm** | 依赖与脚本 |
| **webpack** | 打包 JS + 加载 Wasm |
| **`package.json`** | `npm run start` 开发服务器 |

典型流程：

```bash
wasm-pack build --target web    # 或 bundler / nodejs
# webpack 引入 pkg/rogue_wasm.js
```
