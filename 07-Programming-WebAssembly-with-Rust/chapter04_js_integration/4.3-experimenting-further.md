## 一句话

在 Rogue 基础上继续实验：**`serde` JSON** 把 Rust 状态序列化给 JS；**`js_sys`** 绑定原生 JS 全局 API；**`web_sys`** 绑定浏览器 DOM/Web API — 无需再手写 `extern` 块。

---

## 原书要点

### 引入 `serde` 进行 JSON 序列化

| 场景 | 做法 |
|------|------|
| 向 JS 发送**复杂更新** | 玩家 HP、步数、背包等结构体 |
| **`serde` + `serde_json`** | Rust struct → **JSON 字符串** → JS `JSON.parse` |
| 与 bindgen 关系 | 简单字段用 bindgen 直接暴露；**批量状态快照**用 JSON 更省事 |

概念：

```rust
use serde::Serialize;
use wasm_bindgen::prelude::*;

#[derive(Serialize)]
pub struct Stats { pub hp: u32, pub steps: u32 }

#[wasm_bindgen]
pub fn stats_json(stats: &Stats) -> String {
    serde_json::to_string(stats).unwrap()
}
```

**Layer 2 订单簿**：Go API 返回 JSON → Wasm 内 **`serde` 反序列化** → 固定档位 struct → [layer02](../../layer02_orderbook-query-wasm/README.md)

### `js_sys` — 原生 JavaScript 全局对象

| 能力 | 示例 |
|------|------|
| 已生成的 **JS 内置绑定** | 不必手写 `extern "C"` |
| 常用 | `js_sys::console::log_1(&val)` · `js_sys::eval(...)` · `Array` / `Object` |

```rust
use wasm_bindgen::JsValue;
js_sys::console::log_1(&JsValue::from_str("from Rust Wasm"));
```

↔ 替代 Ch3 [3.5](../chapter03_rust_wasm/3.5-rusty-checkers-ffi.md) 里为 `console.log` 写 import 签名。

### `web_sys` — 浏览器 Web API

| 能力 | 示例 |
|------|------|
| **DOM / Web API** 绑定 | `web_sys::window()` · `document()` · `HtmlElement` |
| 粒度 | crate feature 按需开启（减小 wasm 体积） |

→ Rust 在 Wasm 内可**直接操作 DOM**（仍受同源/安全策略约束），与 [1.1「DOM 须宿主代劳」](../chapter01_wasm_fundamentals/1.1-introducing-wasm.md) 的 1.0 叙述相比，bindgen 生态已大幅简化路径。
