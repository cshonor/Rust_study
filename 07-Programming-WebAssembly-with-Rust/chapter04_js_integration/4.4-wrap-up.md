### 本章三条主线（回顾）

| 线 | 节 | 核心 |
|----|:---:|------|
| **bindgen + 工程化** | 4.1 | `#[wasm_bindgen]` · 字符串/类 · npm/webpack |
| **职责分离实战** | 4.2 | Rot.js (JS) + Engine (Rust) · 双向 class 绑定 |
| **序列化与 Web API** | 4.3 | **serde JSON** · **js_sys** · **web_sys** |

### 与 Ch3 的对比

| | Ch3 手写 FFI | Ch4 wasm-bindgen |
|---|-------------|------------------|
| 字符串 | 手动 memory | 自动生成 |
| JS 类 / Rust struct | 几乎不可行 | **class 互操作** |
| DOM / console | 手写 `extern` | **web_sys / js_sys** |
| 前端构建 | 裸 `fetch` wasm | **webpack + wasm-pack** |

### 读完后应能回答

1. `wasm-bindgen` CLI 相对 `cargo build --target wasm32` **多产出什么**？
2. Rogue 为什么把 **迷宫生成放在 JS**、**战斗放在 Rust**？
3. 何时用 **bindgen 暴露字段**，何时用 **serde JSON 批量同步**？
4. Layer 2 订单簿：Go JSON → Wasm 该用 **serde 反序列化** 还是 bindgen 传字符串？
