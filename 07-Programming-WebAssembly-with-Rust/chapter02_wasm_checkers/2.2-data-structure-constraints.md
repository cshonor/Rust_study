## 一句话

Wasm 没有数组/结构体/枚举——用**线性内存**把 8×8 棋盘**扁平化**为 `offset = (x + y * 8) * 4`，再用 **32 位整数的位标志**打包「空 / 黑 / 白 / 加冕」状态。

---

## 原书要点

### 线性内存与二维数组扁平化

| 事实 | 做法 |
|------|------|
| 无内置数组、结构体、元组 | 在 **linear memory** 里自己布局 |
| 8×8 棋盘，每格一个状态 | 每格 **`i32`（4 字节）** |
| 二维坐标 `(x, y)` | 一维偏移：

```text
offset = (x + y * 8) * 4
```

```text
(x,y) 逻辑棋盘          线性内存 [byte]
┌───┬───┬ … ──┐         [0..3]   (0,0)
│   │   │     │         [4..7]   (1,0)
└───┴───┴ … ──┘           …
                          [252..255] (7,7)
```

- 共 **64 × 4 = 256 字节** → 整模块运行时线性内存约 **256 字节**（[2.5](./2.5-testing-wasm-checkers.md)）。

### 位标志与位掩码 (Bit Flags & Bit Masking)

无法用「对象 / 枚举」表示棋子 → 用一个 **`i32` 的比特位** 打包：

| 值（位） | 含义 |
|:--------:|------|
| **0** | 空位 |
| **1** | 黑棋 |
| **2** | 白棋 |
| **4** | 已加冕 (Crowned) |

**组合示例**：

| 棋子 | 计算 | 值 |
|------|------|:--:|
| 加冕黑棋 | Crowned(4) + Black(1) | **5** |
| 加冕白棋 | 4 + 2 | **6** |

**位运算**（查询 / 修改而不破坏其它位）：

| 操作 | 典型用法 |
|------|----------|
| **AND** | 检测是否含某标志：`piece & 4` → 是否加冕 |
| **OR** | 加标志：`piece | 4` → 加冕 |
| **XOR** | 切换标志（慎用，需配合掩码） |

### 与 Rust / HFT 对照

| Wasm 本章 | Rust / 交易侧 |
|-----------|---------------|
| `i32` 位打包棋子 | 订单簿档位 **occupancy 位图**、flags 字段 |
| 固定 8×8 网格 | 固定 **N 档** 深度，避免 Wasm 内动态扩容 |
| 手动 offset 公式 | `#[repr(C)]` 数组或 `[Price; N]` 线性布局 |
