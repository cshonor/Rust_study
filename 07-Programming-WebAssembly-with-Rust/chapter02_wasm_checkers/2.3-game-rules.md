## 一句话

在扁平内存 + 位标志之上，实现跳棋**规则函数**（如 `$shouldCrown`）和 **回合全局变量 `$currentTurn`**，并配合 `$getPiece` / `$setPiece` / `$inRange` 操纵棋盘。

---

## 原书要点

### 操纵棋盘：存取与边界

| 函数 | 作用 |
|------|------|
| **`$getPiece(x, y)`** | 按 `offset = (x + y*8)*4` 从线性内存 **load** |
| **`$setPiece(x, y, value)`** | 向对应地址 **store** |
| **`$inRange(x, y)`** | **手动边界检查** — Wasm 不会像高级语言那样自动抛越界异常 |

> 越界访问 linear memory 在 Wasm 里是 **trap**（模块 abort），但逻辑上仍须保证坐标在 0..7，避免脏读/写相邻状态。

### 追踪当前回合

| 机制 | 说明 |
|------|------|
| **`(global $currentTurn (mut i32))`** | 可变全局：黑方 / 白方 |
| 查询 / 更新函数 | 模块内封装，供 `$move` 等逻辑调用 |

### 规则示例：`$shouldCrown`

| 条件 | 行为 |
|------|------|
| 棋子到达**对手底线** | 应加冕 → 对棋子值 **OR 4**（Crowned 位） |
| 实现 | 依赖当前玩家方向 + `y` 坐标（黑/白底线不同） |

其它规则（与 [2.4](./2.4-moving-players.md) 重叠）：

- 目标格必须为空
- 移动的棋子须属于 **`$currentTurn`**
- 斜走距离 / 跳吃合法性（书中在 `$move` 路径里验证）

### 宿主通知（import）

规则触发后模块 **import 宿主函数**，让 JS 更新 UI / 音效：

| import | 时机 |
|--------|------|
| **`$notify_piececrowned`** | 加冕完成 |
| **`$notify_piecemoved`** | 移动完成 |

→ Wasm **不能**直接操作 DOM；通知 = [1.1 宿主边界](../chapter01_wasm_fundamentals/1.1-introducing-wasm.md) 的标准模式。
