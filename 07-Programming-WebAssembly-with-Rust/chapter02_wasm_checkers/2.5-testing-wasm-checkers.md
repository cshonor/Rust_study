## 一句话

通过 **`$initBoard` + export/import 边界** 测试跳棋模块；纯手工 WAT 引擎约 **853 字节** 二进制、**256 字节** 线性内存——演示 Wasm 体积极小与**防御性 export** 实践。

---

## 原书要点

### 初始化与导出策略

| 函数 | 可见性 | 说明 |
|------|--------|------|
| **`$initBoard`** | **export** | 宿主启动时摆子 |
| **`$move`** | **export** | 合法走子入口 |
| **`$getCurrentTurn`** 等 | **export**（若书中有） | 宿主读状态 |
| **`$setPiece`** | **私有** | 防止宿主直接改盘 **作弊** |
| 底层内存写 | **私有** | 仅 `$move` / 内部逻辑可改状态 |

→ **防御性编程**：只 export「业务 API」，隐藏可破坏不变量的 primitive。

### 测试方式

| 方式 | 说明 |
|------|------|
| **wabt `wasm-interp`** | 加载模块，call export，检查 memory/global |
| **宿主 JS 测试页** | import 通知函数里 `console.log` / assert |
| **wat2wasm + 浏览器** | 与 Part II 第 3 章 JS 游玩衔接 |

### 体积与内存（书中数据）

| 指标 | 数值 | 含义 |
|------|:----:|------|
| **`.wasm` 文件** | **~853 字节** | 纯手工引擎，无 runtime 膨胀 |
| **线性内存占用** | **256 字节** | 64 格 × 4 字节，无额外堆 |

对比：同逻辑用 Rust + std 编译到 Wasm 往往 **KB～MB 级**——第 3 章会讨论编译器为你加了什么。

### 编译与检视命令

```bash
wat2wasm checkers.wat -o checkers.wasm
wasm-objdump -h checkers.wasm    # 段大小
wasm-objdump -x checkers.wasm    # export/import
ls -l checkers.wasm              # ~853 bytes
```
