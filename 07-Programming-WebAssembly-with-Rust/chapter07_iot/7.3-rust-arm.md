## 一句话

在 PC 上用 **`rustup target add armv7-unknown-linux-gnueabihf`** 交叉编译，产出可在 **树莓派 Linux** 上运行的 **ARM 宿主可执行文件**（加载 indicator Wasm 的那层 Rust 程序）。

---

## 原书要点

### 为何交叉编译

| 事实 | 做法 |
|------|------|
| 开发机在 x86_64 PC | 目标机在 **ARM 树莓派** |
| 宿主是 **native Rust 二进制**（非 Wasm） | 需要 **ARM Linux** triple |
| Wasm 模块仍用 `wasm32-unknown-unknown` | **宿主** 与 **插件** 两个 target |

### 配置步骤（书中）

```bash
rustup target add armv7-unknown-linux-gnueabihf
```

```bash
cargo build --release --target armv7-unknown-linux-gnueabihf
```

| 项 | 说明 |
|----|------|
| **triple** | `armv7-unknown-linux-gnueabihf` — 32 位 ARM · Linux · gnueabihf ABI |
| **链接器** | 交叉链接有时需 `~/.cargo/config.toml` 指定 `linker = arm-linux-gnueabihf-gcc` |
| **部署** | `scp` 二进制 + `.wasm` 到 Pi |

### 双 target 工作流

```text
indicator logic  →  wasm32-unknown-unknown  →  *.wasm
Pi host program  →  armv7-unknown-linux-gnueabihf  →  pi_host (ELF)
```

↔ Layer 3：边缘节点 **native 宿主 + ma_strategy.wasm** → [layer03](../../layer03_quant-ma-strategy/README.md)

### 与 no_std / 嵌入式

本章宿主用 **std**（线程、mpsc、文件监控）；更深层嵌入式见 [ER Item 33 no_std](../../../01-ER/Chapter-06-Beyond-Standard-Rust/Item-33-no-std/README.md) · [RFR Ch12](../../../02-RFR/Chapter-12-Rust-Without-Standard-Library/README.md)
