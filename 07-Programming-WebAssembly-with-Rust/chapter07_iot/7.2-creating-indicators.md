# 7.2 Creating Indicator Modules

> 第 7 章索引：[README](./README.md) · 前：[7.1-indicator-overview](./7.1-indicator-overview.md) · 后：[7.3-rust-arm](./7.3-rust-arm.md)

## 一句话

实现两个 **Wasm 指示器插件**：**电池条**（0–100% → 8 LED 颜色）与 **脉冲动画**（Knight Rider 式关键帧）— 并演示 **无真实硬件** 即可单元测试业务逻辑。

---

## 原书要点

### 模块一：Battery Indicator（电池指示器）

| 输入 | 输出 |
|------|------|
| **`sensor_update`** 传入 0–100 电量 | 8 颗 LED 的 **RGB 数组**（进度条语义） |
| 逻辑在 Wasm | 通过 **`set_led` import** 通知宿主（或写入共享 buffer 再由宿主读） |

映射示例（概念）：

| 电量 % | LED 表现 |
|--------|----------|
| 0–12 | 红 |
| 13–50 | 黄 |
| 51–100 | 绿 |

### 模块二：Pulsing Indicator（脉冲 / 动画指示器）

| 技术 | 说明 |
|------|------|
| **关键帧 (Key frame)** | 预定义多帧 LED 图案 |
| **`apply()`** | 宿主按固定 FPS 调用 · 模块推进帧索引 |
| 效果 | 《霹雳游侠》**Knight Rider** 来回扫描灯带 |

↔ Ch2 [位标志 / 紧凑状态](../chapter02_wasm_checkers/2.2-data-structure-constraints.md) — 帧表也可线性内存布局。

### 免硬件测试（IoT 大优势）

| 做法 | 收益 |
|------|------|
| **`cargo test`** 只测 Wasm 模块逻辑 | 无需示波器 / 真实 LED |
| 宿主 **`set_led` mock** | 断言 RGB 序列是否正确 |
| CI 跑 indicator 单元测试 | 与 [ER Item 30 · 测试](../../../01-ER/Chapter-05-Tooling/Item-30-beyond-unit-tests/README.md) 一致 |

```text
开发机:  test indicator.wasm logic + mock set_led
树莓派:  同一 .wasm + blinkt 真硬件
```

### 编译产物

- Rust → `wasm32-unknown-unknown` → **`battery_indicator.wasm`** / **`pulsing_indicator.wasm`**
- 与 Ch3 [cdylib](../chapter03_rust_wasm/3.3-hello-wasm-rust.md) 相同 target，API 遵守 [7.1 GIMS](./7.1-indicator-overview.md)
