# 7.1 Overview of the Generic Indicator Module

> 第 7 章索引：[README](./README.md) · 后：[7.2-creating-indicators](./7.2-creating-indicators.md)

## 一句话

**GIMS（通用指示器模块系统）**：用 **Wasm 插件**封装「指示器逻辑」，与 **Blinkt LED 等硬件解耦** — 宿主/模块通过 **`sensor_update` / `apply` / `set_led`** 等标准 import/export 契约协作。

---

## 原书要点

### 从 Ch6 控制台到 IoT

| Ch6 | Ch7 |
|-----|-----|
| 控制台宿主 · wasmi | **ARM / 树莓派** 宿主 · 真实 LED |
| 跳棋 `.wasm` 不变 | **指示器 `.wasm` 可热替换** |
| 读 linear memory 画 ASCII | **`set_led` import 驱动硬件** |

↔ [6.1 · 宿主五职责](../chapter06_nonweb_hosts/6.1-good-host.md)

### 硬件解耦问题

| 传统 IoT | GIMS + Wasm |
|----------|-------------|
| 换外设（单色 LED → 彩灯带）→ **大规模重构** | 换 **Wasm 指示器模块** · 宿主 API 稳定 |
| 逻辑与 GPIO 耦合 | **传感器处理** 在 Wasm · **物理输出** 在宿主 import |

### GIMS API 契约（概念）

| 方向 | 函数 | 职责 |
|------|------|------|
| **宿主 → 模块** | **`sensor_update(...)`** | 喂传感器/电量等输入 |
| **宿主 → 模块** | **`apply()`** | 按帧率驱动动画步进（关键帧） |
| **模块 → 宿主 (import)** | **`set_led(index, r, g, b)`** | Wasm 请求宿主点亮某颗 LED |

```text
┌──────────── Pi Host (Rust) ────────────┐
│  mpsc · 10 FPS · blinkt / 终端 mock     │
│  import: set_led                        │
│       ▲                    export:      │
│  ┌────┴──── indicator.wasm ─────────┐  │
│  │  sensor_update · apply · 动画逻辑  │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### 与 Layer 3 / HFT

| GIMS | Layer 3 边缘 |
|------|--------------|
| 热替换 **indicator.wasm** | 热替换 **ma_strategy.wasm**（同类插件模型） |
| 固定 **apply** 帧率 | 固定 **tick 调度**（勿在 Wasm 内 busy-loop） |
| 宿主 **`set_led`** | 宿主 **下单/告警**（策略只 export 信号） |

→ [layer03 · 边缘节点](../../layer03_quant-ma-strategy/README.md)
