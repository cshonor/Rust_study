# 1.1 Introducing WebAssembly

> 第 1 章索引：[README](./README.md) · 后：[1.2-wasm-architecture](./1.2-wasm-architecture.md)

---

## 一句话

Wasm 是**面向栈式虚拟机的便携式二进制指令格式**——高级语言的**编译目标**，不是转译目标、不是 JS 替代品、也不是独立编程语言；必须依附**宿主**沙盒，I/O 全听宿主指挥。

---

## 原书要点

### 它是什么

| 点 | 说明 |
|----|------|
| **本质** | 基于**栈式虚拟机**的**便携式二进制指令格式** |
| **编译目标** | C++、Rust、Go 等高级语言可编译到此格式 |
| **可移植** | 同一模块可在不同 OS / CPU 架构上运行，**无需改源码** |
| **不限于 Web** | 名字带 Web，但只要有**宿主 (Host)**，控制台、嵌入式、云端都能跑 |

### 常见误解（它「不是」什么）

| 误解 | 纠正 |
|------|------|
| **= JS 的转译目标** | **转译**是高级语言→高级语言（如 TS→JS）；Wasm 是高级语言→**低级指令**（类似机器码层级） |
| **会取代 JavaScript** | Wasm **1.0 在浏览器里仍依赖 JS 作宿主**配合加载与胶水代码 |
| **= 一门独立编程语言** | 像**没有主机的游戏卡带**——必须进宿主沙盒；模块自身不能随意 I/O |
| **= 只在浏览器里跑** | 浏览器只是最常见宿主之一；[Part III 第 6 章](../chapter06_nonweb_hosts/README.md) 起会讲非 Web 宿主 |

### 宿主与模块边界

```text
┌────────────── 宿主 (Host) ──────────────┐
│  JS / Rust / 控制台 … 提供 I/O、内存视图   │
│       ▲ import / export                   │
│  ┌────┴──── Wasm 模块 (.wasm) ────┐      │
│  │  纯计算 · 线性内存 · 无直接 DOM  │      │
│  └────────────────────────────────┘      │
└──────────────────────────────────────────┘
```

- 模块**不能**直接访问宿主内存或 DOM；「操作 DOM」= 通过**导入函数**或**共享线性内存 + 宿主代劳**（见 [1.2](./1.2-wasm-architecture.md)）。
