## 一句话

用 **wabt** 工具链手写 **WAT (S-表达式)** → `wat2wasm` 得到 `.wasm` → `wasm2wat` / `wasm-objdump` 做**往返编译与检视**——理解底层，后续 Rust 编译到 Wasm 才不当黑盒。

---

## 原书要点

### WebAssembly 二进制工具包 (wabt)

常用命令：

| 工具 | 作用 |
|------|------|
| **`wat2wasm`** | `.wat` 文本 → `.wasm` 二进制 |
| **`wasm2wat`** | `.wasm` → `.wat`（反汇编为可读文本） |
| **`wasm-objdump`** | 检查模块段、导入/导出、代码结构 |
| **`wasm-interp`** | 解释执行（调试小模块） |

安装（示例）：

```bash
# Windows: choco install wabt  /  scoop install wabt
# 或从 https://github.com/WebAssembly/wabt/releases 下载
wat2wasm --version
```

### WebAssembly 文本格式 (WAT)

- 文件扩展名 **`.wat`**
- 语法：**S-表达式 (S-expressions)**，Lisp 式树状结构
- 现代流程里编译器自动生成 WAT/wasm，但**手写一遍**能看清 [1.2](./1.2-wasm-architecture.md) 里的栈操作与 `export`

最小流程：

```bash
# 1. 编辑 add.wat
# 2. 编译
wat2wasm add.wat -o add.wasm
# 3. 反汇编验证
wasm2wat add.wasm -o add.roundtrip.wat
# 4. 结构 dump
wasm-objdump -x add.wasm
```

### 往返编译 (Round-trip compilation)

```text
add.wat  ──wat2wasm──►  add.wasm  ──wasm2wat──►  add.roundtrip.wat
```

| 启示 | 说明 |
|------|------|
| **便携性** | 二进制可在不同平台间传递、反解为等价 WAT |
| **无秘密** | **任何人都能 wasm2wat 反汇编你的模块** → 商业逻辑不能靠「二进制 obscurity」 |
| **设计后果** | 敏感算法需配合 [附录 A2 · 签名/加密](../../appendix/A2-security.md)；策略逻辑假设可被逆向 |
