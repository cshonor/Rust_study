# 18.3 模式语法

本节汇总常用模式写法：字面量、`|`、`..=`、解构、`_` / `..`、匹配守卫、`@` 等。

对应 demo：`18-patterns/18.3-pattern-syntax-demo/`

---

## 字面量

直接匹配常量值（数字、字符、`&str` 等），适合区分少数离散取值。

---

## `match` 中的变量与遮蔽

`match` 开启新作用域；分支里 `Some(y)` 的 `y` 会**遮蔽**外层的 `y`，与外层 `x`/`y` 比较时需用**匹配守卫**（见下）或改用不同绑定名（如 `Some(n)`）。

---

## 多个模式：`|`

`1 | 2` 表示“其一即可”。

---

## 范围：`..=`

闭区间，仅适用于**数字**与 **`char`**（编译期可判空范围）。如 `1..=5`、`'a'..='j'`。

---

## 解构

- **结构体**：`let Point { x, y } = p;` 或 `Point { x: a, y: b }`；可在模式中写**字面量**约束字段，如 `Point { x, y: 0 }`。
- **枚举**：变体形状须与定义一致（单元 / 类结构体 / 类元组）。
- **嵌套**：`Message::ChangeColor(Color::Hsv(h, s, v))` 等。
- **混合**：`((feet, inches), Point { x, y }) = ...`

---

## 忽略值

- **`_`**：不绑定；用于通配或函数参数 `fn foo(_: i32, y: i32)`。
- **`_x`**：仍**绑定**（可移动），可抑制未使用警告，但会参与所有权规则。
- **`..`**：忽略其余字段或元组中间部分；**每个元组模式只能有一个 `..`**，否则歧义报错。

`if let Some(_s) = s` 仍会把 `s` 内值**移动**进 `_s`；若需保留 `s` 供后续使用，应用 `if let Some(_) = s`（demo 见 **18-22**）。

---

## 匹配守卫（`if`）

写在分支模式之后：`Some(x) if x < 5 =>`。可与 `|` 组合，**守卫作用于整个 `|` 模式组**：`(4 | 5 | 6) if y =>`，而非仅最后一项。

用于比较**外部变量**（避免分支内新 `y` 遮蔽外层 `y`）：`Some(n) if n == y`。

---

## `@` 绑定

`id @ 3..=7`：同时**测试**范围并**绑定**该值到变量。
