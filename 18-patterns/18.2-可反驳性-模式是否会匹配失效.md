# 18.2 可反驳性（Refutability）：模式是否会匹配失效

模式分为两类：

- **不可反驳的（irrefutable）**：对**所有**可能出现的值都能匹配成功（例如 `let x = 5` 里的 `x`，或能兜底的 `_`）。
- **可反驳的（refutable）**：对某些值会**匹配失败**（例如 `Some(x)` 在右侧为 `None` 时失败）。

对应 demo：`18-patterns/18.2-refutability-demo/`

---

## 哪些位置要求哪种模式

| 位置 | 通常要求 |
|------|-----------|
| `let`、函数参数、`for` | **不可反驳**（否则“绑定失败”时程序无法继续） |
| `if let`、`while let` | **可反驳**（本意就是按成功/失败分支） |

`match` 分支：除**最后一个**常用来兜底的“全捕获”分支外，各分支模式多为可反驳；最后一臂常用不可反驳模式（或 `_`）保证穷尽。

---

## 常见错误与修法

- **`let Some(x) = opt;`** 在 `opt` 可能为 `None` 时**不能编译**（`refutable pattern in local binding`），应改为 **`if let Some(x) = opt { ... }`** 或 `match`。
- **`if let x = 5;`** 使用**不可反驳**模式，编译器会警告 **`irrefutable if-let pattern`**（无意义，可直接 `let`）。demo 里用 `#[allow(irrefutable_let_patterns)]` 仅便于把示例放在同一文件；实际应删掉或改成普通 `let`。
