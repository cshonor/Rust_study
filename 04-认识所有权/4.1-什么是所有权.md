# 4.1 什么是所有权？

Rust 通过**所有权系统**管理内存，编译器在编译期检查，运行时零开销。

---

## 栈 (Stack) 与堆 (Heap)

| 栈 | 堆 |
|----|-----|
| 后进先出 (LIFO) | 无序 |
| 数据大小固定且已知 | 大小可变、编译时未知 |
| 入栈/出栈很快 | 分配需查找空位 |
| 存值本身 | 存数据，栈上存指针 |

- **入栈**：数据大小已知，直接压栈
- **堆分配**：请求空间 → 分配器找空位 → 返回指针
- 访问栈比访问堆快（指针跳转少）

---

## 所有权规则

1. 每个值有且仅有一个**所有者**（变量）
2. 当所有者离开**作用域**，值被丢弃（调用 `drop`）

---

## 变量作用域

```rust
{
    let s = "hello";   // s 从此处有效
    // 使用 s
}                      // s 离开作用域，无效
```

---

## String 类型

- **字面量** `"hello"`：不可变，编译时已知，硬编码进可执行文件
- **String**：可变，堆分配，运行时大小可变

```rust
let mut s = String::from("hello");
s.push_str(", world!");
```

- 离开作用域时自动调用 `drop` 释放堆内存（RAII 模式）

---

## 移动 (Move)

```rust
let s1 = String::from("hello");
let s2 = s1;  // s1 被移动，不再有效！
// println!("{}", s1);  // ❌ 错误: borrow of moved value
```

- 只复制栈上的指针、长度、容量，**不**复制堆上数据
- 为使内存安全，Rust 使 **s1 无效**，称为**移动**（不是浅拷贝）
- 避免二次释放 (double free)

---

## 克隆 (Clone)

需要深拷贝堆数据时用 `clone()`：

```rust
let s1 = String::from("hello");
let s2 = s1.clone();
println!("{} {}", s1, s2);  // ✓ 两者都有效
```

---

## Copy 类型（栈上拷贝）

栈上、大小固定的类型实现 `Copy` trait，赋值时**拷贝**而非移动：

```rust
let x = 5;
let y = x;
println!("{} {}", x, y);  // ✓ 两者都有效
```

**实现了 Copy 的类型**：所有整数、bool、f32/f64、char、仅含 Copy 元素的元组（如 `(i32, i32)`）

- `String` 未实现 Copy（需堆分配）
- 实现 `Drop` 的类型不能实现 `Copy`

---

## 所有权与函数

- 传参 = 赋值：会**移动**或**拷贝**
- `String` 传入函数 → 所有权移动，调用后原变量无效
- `i32` 等 Copy 类型 → 拷贝，调用后仍可用

```rust
let s = String::from("hello");
takes_ownership(s);  // s 被移动，s 不再有效

let x = 5;
makes_copy(x);       // x 被拷贝，x 仍有效
```

---

## 返回值与所有权

- 返回值会转移所有权
- 可用元组返回多个值，顺带把所有权“还回去”：

```rust
let (s2, len) = calculate_length(s1);
fn calculate_length(s: String) -> (String, usize) {
    let length = s.len();
    (s, length)  // 返回 s，转移所有权
}
```

更优雅的做法是使用**引用**（下一节）。

---

## 小结

| 操作 | String (堆) | i32 (Copy) |
|------|-------------|------------|
| 赋值 | 移动，原变量失效 | 拷贝，两者有效 |
| 传参 | 移动 | 拷贝 |
| clone | 深拷贝 | 通常不需要 |
