# 4.3 切片 Slice

**Slice** 引用集合中一段连续元素，不拥有数据。解决索引与数据不同步的问题。

---

## 问题：返回索引的缺陷

```rust
fn first_word(s: &String) -> usize {
    // 找到空格返回索引，否则返回 s.len()
}
```

- 返回的 `usize` 与 `String` 分离
- `s.clear()` 后索引仍为 5，但数据已变，产生 bug
- 编译器无法检测这种逻辑错误

---

## 字符串 Slice

类型为 `&str`，引用 String 的一部分：

```rust
let s = String::from("hello world");
let hello = &s[0..5];   // "hello"
let world = &s[6..11];  // "world"
```

### Range 语法

| 写法 | 含义 |
|------|------|
| `[0..5]` | 索引 0 到 4（不含 5） |
| `[..2]` | 同 `[0..2]`，从头开始可省略 0 |
| `[3..]` | 从 3 到末尾 |
| `[..]` | 整个字符串 |

- 长度 = `ending_index - starting_index`
- **注意**：索引必须在 UTF-8 字符边界上，不能从多字节字符中间切

---

## 用 Slice 重写 first_word

```rust
fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]  // 无空格则返回整个字符串
}
```

- 返回值与底层数据绑定
- 借用检查器会阻止在 `word` 存在时调用 `s.clear()`：

```rust
let word = first_word(&s);
s.clear();  // ❌ 错误: cannot borrow `s` as mutable
println!("{}", word);  // word 的不可变引用仍在使用
```

---

## 字符串字面量是 Slice

```rust
let s = "Hello, world!";  // s 的类型是 &str
```

- `&str` 指向二进制中固定位置
- 所以字符串字面量**不可变**

---

## 更好的 API：`&str` 参数

```rust
fn first_word(s: &str) -> &str { ... }
```

- 可接受 `&String`、`&str`、`"字面量"`、`&my_string[..]`
- 利用 deref coercions，更通用

```rust
let word = first_word(&my_string);       // String 引用
let word = first_word(&my_string[..]);   // String 的 slice
let word = first_word("hello world");    // 字面量就是 &str
```

---

## 数组 Slice

```rust
let a = [1, 2, 3, 4, 5];
let slice = &a[1..3];  // [2, 3]，类型 &[i32]
```

- 与字符串 slice 类似：存起始指针 + 长度
- 适用于所有集合类型（第 8 章 Vec）

---

## 小结

| 类型 | 说明 |
|------|------|
| `&str` | 字符串 slice |
| `&[T]` | 数组/T 类型 slice |
| `[start..end]` | Range，不含 end |

- Slice 让返回值与数据绑定，借用规则在编译期避免无效索引
