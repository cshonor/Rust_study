# 4.2 引用与借用

用**引用** (`&`) 使用值但不获取所有权，称为**借用**（borrowing）。

---

## 引用语法

```rust
let s1 = String::from("hello");
let len = calculate_length(&s1);  // &s1 创建引用
println!("'{}' 长度 {}", s1, len);  // s1 仍有效

fn calculate_length(s: &String) -> usize {
    s.len()
}  // s 离开作用域，但不丢弃指向的数据（没有所有权）
```

- `&s1`：指向 s1 的引用，不拥有数据
- `*` 解引用：见第 8、15 章

---

## 不可变引用

默认引用是**不可变**的，不能通过引用修改数据：

```rust
fn change(some_string: &String) {
    some_string.push_str(", world");  // ❌ 错误
}
// cannot borrow `*some_string` as mutable, as it is behind a `&` reference
```

---

## 可变引用 &mut

要修改借用来的值，使用**可变引用**：

```rust
let mut s = String::from("hello");
change(&mut s);

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}
```

- 变量需为 `mut`
- 传参用 `&mut s`
- 函数签名为 `&mut String`

---

## 借用规则

### 规则 1：同一时间只能有一个可变引用

```rust
let r1 = &mut s;
let r2 = &mut s;  // ❌ 错误: cannot borrow `s` as mutable more than once
```

- 避免**数据竞争**（多指针写、无同步）
- 可用大括号限定作用域，错开可变借用：

```rust
{
    let r1 = &mut s;
}  // r1 离开作用域
let r2 = &mut s;  // ✓
```

### 规则 2：不可变与可变引用不能同时存在

```rust
let r1 = &s;
let r2 = &s;
let r3 = &mut s;  // ❌ 错误: cannot borrow as mutable (已有不可变引用)
```

- 多个不可变引用可以共存
- 有可变引用时不能有不可变引用

### 引用的作用域与 NLL

引用的作用域到**最后一次使用**为止（非词法作用域 NLL）：

```rust
let r1 = &s;
let r2 = &s;
println!("{} {}", r1, r2);  // r1、r2 最后一次使用
let r3 = &mut s;  // ✓ r1、r2 已“结束”，不重叠
```

---

## 悬垂引用

Rust 保证引用不会指向已释放的内存（悬垂指针）：

```rust
fn dangle() -> &String {  // ❌ 错误
    let s = String::from("hello");
    &s  // s 离开作用域被释放，返回的引用无效
}
```

- 错误：`missing lifetime specifier`，返回值借用的数据不存在
- 正确做法：返回拥有所有权的值，如 `String`：

```rust
fn no_dangle() -> String {
    let s = String::from("hello");
    s  // 转移所有权
}
```

---

## 引用规则小结

1. **任意时刻**：要么只有一个可变引用，要么只有多个不可变引用
2. **引用必须有效**：不能悬垂

---

## 与所有权的对比

| 方式 | 所有权 | 原变量 |
|------|--------|--------|
| 传值 | 移动 | 失效 |
| 不可变引用 `&T` | 借用 | 仍可用 |
| 可变引用 `&mut T` | 借用并可能修改 | 仍可用 |
