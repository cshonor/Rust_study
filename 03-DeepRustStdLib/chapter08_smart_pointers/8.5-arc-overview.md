## 一句话

**`Arc<T>`**（Atomic Reference Counted）= **`Rc` 的多线程版** — **`ArcInner`** 用 **`AtomicUsize`** 维护 strong/weak；API 与 `Rc` 几乎一致，但 **`Arc` 只保证计数原子安全，不保证 `T` 内部并发安全**。

---

## 原书要点

### 与 Rc 对比

| | **`Rc<T>`** | **`Arc<T>`** |
|---|-------------|--------------|
| 计数 | `usize` | **`AtomicUsize`** |
| 线程 | **!Send 计数逻辑** · 单线程 | **clone/drop 计数线程安全** |
| **`T` 并发** | 单线程 assumed | **仍须 `Mutex`/`RwLock` 等** |

### 原书安全边界（重要）

```text
Arc<T>: Send + Sync  （在 T: Send + Sync 时自动）
  │
  ├── 保证的是：引用计数操作的线程安全
  └── 不保证：多个 &T 同时改 T 的内容
```

**多线程可变** → **`Arc<Mutex<T>>`** / **`Arc<RwLock<T>>`** · [第 11 章](../chapter11_concurrency/README.md)
