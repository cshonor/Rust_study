# 8.2 RawVec<T> 类型分析

> 章索引：[第 8 章](./README.md) · 前：[8.1](./8.1-box.md) · 后：[8.3 · Vec](./8.3-vec-overview.md)

---

## 一句话

**`RawVec<T>`** 是 **`Vec`/`VecDeque` 的底层引擎** — 只管理 **连续堆缓冲的 ptr + cap**，**不跟踪 `len`**、**不保证** 槽位已初始化；扩容 **`grow_amortized`** 封装 realloc + 拷贝。

---

## 原书要点

### 职责边界

| RawVec | Vec |
|--------|-----|
| **`cap`** · **`ptr`**（`NonNull`/`Unique`） | **`len`** + RawVec |
| 申请/扩容/释放 **容量** | 追踪 **有效元素个数** |
| **MaybeUninit 语义** — 未用槽位可能未初始化 | `push` 时才 **write** 初始化 |

```text
RawVec:  「这块堆上有 cap 个 T 大小的空间」
Vec:     「其中前 len 个是合法 T」
```

### 扩容 grow_amortized

| 步骤 | 说明 |
|------|------|
| **新 cap** | 通常 **倍增** · 摊还 O(1) push |
| **alloc 新块** | `Global` / 自定义 `Allocator` |
| **拷贝旧数据** | **`ptr::copy`** / **`copy_nonoverlapping`** |
| **释放旧块** | 旧 RawVec drop |

↔ [6.5 · 切片 mem copy](../chapter06_basic_types_continued/6.5-slice-type.md) · [3.2 · MaybeUninit](../chapter03_memory_model/3.2-maybeuninit.md)

### 设计拆分意义

**容量管理** 与 **长度/初始化** 分离 — `VecDeque` 复用 RawVec 模式 · 减少重复 unsafe 扩容逻辑。
