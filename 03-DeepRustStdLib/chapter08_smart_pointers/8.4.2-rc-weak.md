# 8.4.2 Weak<T> 类型分析

> 章索引：[第 8 章](./README.md) · 前：[8.4.1](./8.4.1-rc-ctor-dtor.md) · 后：[8.4.3 · 其他函数](./8.4.3-rc-other-fns.md)

---

## 一句话

**`Weak<T>`** 只增 **weak 计数**、**不增 strong** — 打破 **`Rc` 循环引用**导致的 **泄漏**；使用前须 **`upgrade() → Option<Rc<T>>`** 判断 data 是否仍在。

---

## 原书要点

### 循环引用问题

```text
A ──Rc──► B
B ──Rc──► A     strong 均 ≥1  →  永不 drop  →  泄漏（非悬垂，是占着堆）
```

| 解决 | 做法 |
|------|------|
| **弱边** | 一侧改为 **`Weak`** — 不阻止 strong 归零 |
| **树** | 父 **`Rc` 子** · 子 **`Weak` 父** |

### Weak 语义

| 方法 | 行为 |
|------|------|
| **`Rc::downgrade(&rc)`** | 从 `Rc` 得 `Weak` · weak+1 |
| **`weak.upgrade()`** | strong>0 → **`Some(Rc)`** · 否则 **`None`** |
| **`Weak::clone`** | weak+1 · **不** 延长 data 寿命 |

```rust
if let Some(rc) = weak_parent.upgrade() {
    // data 仍存活
} else {
    // 已被释放
}
```

### 与悬垂对比

| 问题 | 表现 |
|------|------|
| **悬垂指针** | 非法读 · Rust 编译期/类型系统防 |
| **Rc 环泄漏** | 内存合法但 **永不回收** · **`Weak` 防环** |
