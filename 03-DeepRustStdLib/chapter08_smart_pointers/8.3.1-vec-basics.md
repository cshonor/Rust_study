## 一句话

**`Vec`** 的 **insert / remove / append** 等需在缓冲区内 **挪动元素** — 标准库用 **`unsafe` + `ptr::copy` / `copy_nonoverlapping`** 批量直拷，避免逐个 **move + drop** 的性能灾难。

---

## 原书要点

### 典型操作

| API | 底层直觉 |
|-----|----------|
| **`push`** | `len` 处 `write` · 可能 `RawVec` 扩容 |
| **`insert(i, x)`** | `[i..len]` **右移** 一格 · 写入 `x` |
| **`remove(i)`** | 取出 `i` · `[i+1..]` **左移** |
| **`append`** | 追加另一 Vec 全部元素 · 可能一次 memcpy |

### unsafe 直拷（原书 · 极致性能）

```text
安全逐元素 move:  每步 drop 旧位 + 所有权链
copy_nonoverlapping:  整块内存 bitwise 复制（T 须 valid 且无重叠）
```

| 条件 | std 内证明 |
|------|------------|
| **索引合法** | `i <= len` |
| **不重叠** | 源/目标区间分离或算法保证 |
| **对外** | 用户只见 **`&mut Vec<T>`** safe API |

### HFT

| 实践 | 说明 |
|------|------|
| **`reserve(n)`** | 订单批处理前一次分配 |
| **少 `insert(0, …)`** | 头插 O(n) · 用 **`VecDeque`** 或 ring |
