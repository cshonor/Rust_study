# 8.1 Box<T> 类型分析

> 章索引：[第 8 章](./README.md) · 后：[8.2 · RawVec](./8.2-rawvec.md)

---

## 一句话

**`Box<T>`** 是最基础的 **堆上独占** 智能指针 — **`Box::new`** 经 **`Global` 分配器** 申请固定大小堆块，**`Drop`** 自动 dealloc；**`Box::leak`** 主动放弃管理权，得到 **`'static` 引用**。

---

## 原书要点

### 定位

| 项 | 说明 |
|----|------|
| **用途** | 大对象上堆 · **递归类型**（链表节点）· **trait 对象**（`Box<dyn Trait>`） |
| **所有权** | **唯一** owner — 非 `Rc`/`Arc` |
| **层** | **`alloc`** crate · 依赖 [3.7 · Global](../chapter03_memory_model/3.7-heap-alloc.md) |

### 构造与释放

```text
Box::new(v)
  └── Global::allocate(Layout::new::<T>())
  └── 写入 T
Drop
  └── ptr::drop_in_place + Global::deallocate
```

| 对比 | C/C++ | Rust |
|------|-------|------|
| 释放 | 手动 `free` | **`Drop` 自动** |
| GC | 部分语言 | **无 GC** · 确定性析构 |

↔ [3.5 · Unique](../chapter03_memory_model/3.5-unique.md) — Box 逻辑上的独占堆基座

### Box::leak（资源泄漏 · 有意为之）

```rust
let static_ref: &'static mut [u8] = Box::leak(vec![0; N].into_boxed_slice());
```

| 场景 | 说明 |
|------|------|
| **全局/static 初始化** | 程序全程存活 |
| **FFI 交出所有权** | C 侧负责释放（须契约） |
| **代价** | **永不 drop** — 内存直到进程结束 |

### 图/树

**`Box<Node>`** — 单父单孩或单向边 · **独占** 子树所有权。
