# 5.7.1 成员本身迭代器

> 章索引：[第 5 章](./README.md) · 前：[5.7](./5.7-array-iterators.md) · 后：[5.7.2 · 引用](./5.7.2-array-by-ref-iter.md)

---

## 一句话

**`array::IntoIter<T, N>`** 按 **move** 逐个产出元素 — 内部用 **`MaybeUninit<T>` 数组** 跟踪哪些槽位已移出；**Drop** 时只对 **尚未 move 的已初始化元素** 调用 `drop`，避免 panic 路径 **双重释放 / 泄漏**。

---

## 原书要点

### 风险场景

```rust
for x in arr.into_iter() {
    if bad(x) { panic!(); }  // 部分元素已 move，部分仍在数组中
}
```

| 无防护 | 后果 |
|--------|------|
| 普通 `[T; N]` + panic | 部分 drop · 部分未 drop → **UB / 泄漏** |
| 全部已 move | 数组 **空壳** — 不能再 drop 已移走元素 |

### MaybeUninit 方案

```text
state: [MaybeUninit<T>; N]
index: 下一个待 move 的下标
next():  read_and_drop 当前槽 → 置为未初始化
drop():  对 index..N 中仍 Initialized 的槽 drop
```

↔ [3.2.1 · MaybeUninit 定义](../chapter03_memory_model/3.2.1-maybeuninit-definition.md) · [3.2.2 · ManuallyDrop](../chapter03_memory_model/3.2.2-manuallydrop-definition.md)

### 使用建议

| 场景 | 推荐 |
|------|------|
| 只读遍历 | **`array.iter()`** — 简单安全 |
| 必须消费数组 | `into_iter()` · 知悉底层成本 |
| 定长订单档 | 常 **`&[T; DEPTH]` + iter** 而非 into |
