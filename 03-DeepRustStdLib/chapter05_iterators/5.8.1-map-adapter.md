# 5.8.1 Map 适配器

> 章索引：[第 5 章](./README.md) · 前：[5.8](./5.8-iterator-adapters.md) · 后：[5.8.2 · Chain](./5.8.2-chain-adapter.md)

---

## 一句话

**`Map<I, F>`** 包装内层迭代器 **`I`** 与闭包 **`F`** — 每次 **`next`** 先拉 **`I::Item`**，再 **`f(item)`** 映射为新的 **`Item` 类型**；典型 **1:1 变换** 适配器。

---

## 原书要点

### 结构直觉

```rust
pub struct Map<I, F> {
    iter: I,
    f: F,
}
// next: self.iter.next().map(|x| (self.f)(x))
```

| 点 | 说明 |
|----|------|
| **闭包捕获** | `F: FnMut` — 可变捕获环境 |
| **类型变化** | `Iterator<Item=U>` 即使 `I: Item=T` |
| **惰性** | `f` 仅在 **实际 pull** 时调用 |

### 示例

```rust
(0..10).map(|x| x * 2).filter(|x| x % 3 == 0).collect::<Vec<_>>();
```

### 性能提示

| 建议 | 原因 |
|------|------|
| 热路径优先 **`for` + 内联** | 复杂链可能阻碍优化 |
| 避免 **`map` 内分配** | 每元素 heap 拖垮 cache |
