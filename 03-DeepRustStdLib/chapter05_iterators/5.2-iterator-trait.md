# 5.2 Iterator Trait 分析

> 章索引：[第 5 章](./README.md) · 前：[5.1](./5.1-three-iterators.md) · 后：[5.3 · collect](./5.3-iterator-collections.md)

## 一句话

**`Iterator`** 的统一接口是 **`fn next(&mut self) -> Option<Self::Item>`** — 标准库在其上提供 **`fold`/`count`/`last`/…** 等大量 **默认方法**；**Iterator + 闭包 + 适配器** 构成 Rust 函数式编程底座。

---

## 原书要点

### 核心定义

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
    // fold, map, filter, … 均为 default methods
}
```

| 要点 | 说明 |
|------|------|
| **`Item`** | 每次 `next` 产出的元素类型 |
| **`None`** | 迭代结束 |
| **惰性** | 适配器链在 **`next`/`collect`/消费端** 才真正执行 |

### 常用消费端（default methods）

| 方法 | 作用 |
|------|------|
| **`fold`** | 累积归约 |
| **`count` / `sum` / `product`** | 统计与算术 |
| **`last` / `nth`** | 按序取元素 |
| **`all` / `any` / `find`** | 谓词搜索 |
| **`collect`** | → 见 [5.3](./5.3-iterator-collections.md) |

### 与闭包

**`map(|x| …)`、`filter(|x| …)`** 等接收 **闭包** — 捕获环境、零成本抽象依赖 **单态化 + 内联**。
