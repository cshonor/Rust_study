# 5.3 Iterator 与其他集合类型转换

> 章索引：[第 5 章](./README.md) · 前：[5.2](./5.2-iterator-trait.md) · 后：[5.4 · Range](./5.4-range-iterators.md)

## 一句话

**`collect()`** 将迭代器 **聚合** 为目标容器 — 底层依赖 **`FromIterator`**（及 **`Extend`**）；不同集合间 **「遍历 → 重建」** 因此极为方便（如 `Vec` → `HashSet`、过滤后新 `Vec`）。

---

## 原书要点

### 逆向转换（收集）

```rust
let v: Vec<i32> = (0..10).filter(|x| x % 2 == 0).collect();
let s: HashSet<_> = v.iter().cloned().collect();
```

| Trait | 角色 |
|-------|------|
| **`FromIterator<A>`** | `collect()` 的目标类型须实现 — **从迭代器构造** |
| **`Extend<A>`** | 向已有集合 **追加** 迭代器元素 |
| **`Iterator::collect`** | 语法糖：`FromIterator::from_iter(self)` |

### 类型推断

```rust
.collect::<Vec<_>>()   // turbofish 指定容器
let v: Vec<i32> = …    // 注解指定 Item 与容器
```

### 常见转换

| 源 | 目标 | 示例 |
|----|------|------|
| `Range` | `Vec` | `(0..n).collect()` |
| `Iter` | 新 `Vec` | `v.iter().map(f).collect()` |
| 键值对 iter | `HashMap` | `map.into_iter().collect()` |
