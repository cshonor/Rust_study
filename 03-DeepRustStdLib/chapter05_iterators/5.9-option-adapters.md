# 5.9 Option<T> 适配器

> 章索引：[第 5 章](./README.md) · 前：[5.8.3](./5.8.3-other-adapters.md) · 后：[第 6 章](../chapter06_basic_types_continued/README.md)

## 一句话

**`Option<T>`** 也可视为 **0 或 1 个元素** 的迭代器 — 标准库为其提供 **`iter`/`into_iter`** 及 **`transpose`/`flatten`** 等，使 **Optional 值完美融入** `map`/`and_then`/链式函数式写法。

---

## 原书要点

### 作为迭代器

| 调用 | 等价长度 | `Item` |
|------|----------|--------|
| **`None.iter()`** | 0 | — |
| **`Some(x).iter()`** | 1 | `&T` |
| **`Some(x).into_iter()`** | 1 | `T` |

```rust
let o = Some(42);
let sum: i32 = o.into_iter().map(|x| x + 1).sum(); // 43
// None.into_iter().sum() → 0（空迭代）
```

### 与 Iterator 互操作

| API | 作用 |
|-----|------|
| **`Iterator::flatten`** | `Iter<Item=Option<T>>` → `Iter<Item=T>` 跳过 `None` |
| **`Iterator::collect::<Option<Vec<_>>>`** | 短路与 **`transpose`** 思想相关 |
| **`Option::transpose`** | `Option<Result<T,E>>` ↔ `Result<Option<T>,E>` |
| **`Option::flatten`** | `Option<Option<T>>` → `Option<T>` |

↔ [4.2.3 · Option](../chapter04_primitive_types/4.2.3-option-type.md)

### 设计意图

```text
非集合类型  ──►  统一 Iterator 接口  ──►  同一套 map/chain/collect 思维
```

减少 **`match` 嵌套** — 与 **`?`** 互补（值层 vs 错误层）。
