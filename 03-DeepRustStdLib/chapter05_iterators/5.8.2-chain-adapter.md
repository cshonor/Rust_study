# 5.8.2 Chain 适配器

> 章索引：[第 5 章](./README.md) · 前：[5.8.1](./5.8.1-map-adapter.md) · 后：[5.8.3 · 其他](./5.8.3-other-adapters.md)

---

## 一句话

**`Chain<A, B>`** 顺序 **耗尽 `A`** 再 **接续 `B`** — 将两个同 **`Item` 类型** 的迭代器 **拼接** 为一个逻辑序列，无需先 **`collect` 合并 Vec**。

---

## 原书要点

### 行为

```rust
a.into_iter().chain(b.into_iter())
// 先 yield A 的全部 Item，再 yield B 的全部 Item
```

| 状态 | `next()` |
|------|----------|
| A 未结束 | `A.next()` |
| A 结束 | `B.next()` |
| 皆结束 | `None` |

### 典型用途

| 场景 | 示例 |
|------|------|
| 分段 buffer | 头批 tick **`chain`** 尾批 |
| 常量前缀 | `[0].iter().chain(data.iter())` |
| 扁平只读视图 | 两个 slice 连续扫描 |

### 注意

**`Item` 类型须一致** — 不同型别需先 **`map`** 统一；**`chain` 后类型** 为 **`Chain<A,B>`**，仍可继续适配器。
