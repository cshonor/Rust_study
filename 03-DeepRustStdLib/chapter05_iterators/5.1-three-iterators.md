# 5.1 三种迭代器

> 章索引：[第 5 章](./README.md) · 后：[5.2 · Iterator Trait](./5.2-iterator-trait.md)

---

## 一句话

集合遍历的三种入口 **`into_iter` / `iter` / `iter_mut`** 分别对应 **消费所有权、`&T`、`&mut T`** — 由 **`IntoIterator`** 统一抽象，与 Rust **所有权/借用** 规则一一绑定。

---

## 原书要点

| 方法 | 产出 | 所有权 | 迭代后容器 |
|------|------|--------|------------|
| **`into_iter()`** | `IntoIter`（按值） | **转移** 元素所有权 | **被消费**（部分类型仍可用，如 `Vec` 变空） |
| **`iter()`** | `Iter<'_, T>` | **`&T`** 借用 | 容器 **保留** |
| **`iter_mut()`** | `IterMut<'_, T>` | **`&mut T`** 可变借用 | 容器 **保留** · 可原地修改 |

```rust
let v = vec![1, 2, 3];
for x in v.into_iter() { /* x: i32 */ }   // v 不可用
let v = vec![1, 2, 3];
for x in v.iter()       { /* x: &i32 */ }
for x in v.iter_mut()   { /* x: &mut i32 */ }
```

### `for` 语法糖

```text
for item in collection { body }
  ══►  IntoIterator::into_iter(collection)
       └── Iterator::next() 循环
```

### 选择原则

| 需求 | 选用 |
|------|------|
| 只读扫描 | `iter()` |
| 原地更新元素 | `iter_mut()` |
| 拆容器、移元素到新结构 | `into_iter()` |
