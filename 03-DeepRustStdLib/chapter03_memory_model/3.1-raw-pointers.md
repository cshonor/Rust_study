# 3.1 裸指针——不安全的根源

> 章索引：[第 3 章](./README.md) · 后：[3.1.1 · 具体实现](./3.1.1-raw-pointer-impl.md)

---

## 一句话

**`*const T` / `*mut T`** 只表示「可能存在的内存地址+元数据」，**不保证**对齐、初始化或有效性 — 与 **`&T` / `&mut T`** 的借用检查相对；**~90% 的 `unsafe` 根源** 是把裸指针 **转成引用**。

---

## 原书要点

### 概念与区别

| | 裸指针 | 安全引用 |
|---|--------|----------|
| 语法 | `*const T` · `*mut T` | `&T` · `&mut T` |
| 有效性 | **不保证** 指向合法、已初始化、对齐的内存 | 编译器 **借用检查** 保证 |
| 生命周期 | **无** | 有 · 防悬垂 |
| 解引用 | **`unsafe`** | 安全 |

```text
&T / &mut T  ──编译器证明──►  安全使用
*const / *mut ──程序员证明──►  unsafe 解引用 / 转 &T
```

### Unsafe 的核心（原书）

> **约 90% 的 `unsafe` 代码根源**：**裸指针 → 引用**（`&*ptr` / `ptr.as_ref()` 等）。

| 风险 | 说明 |
|------|------|
| ** invented 生命周期** | 转引用时生命周期由 **返回值/上下文** 决定，须 **手动** 证明仍有效 |
| **未初始化 / 已释放** | 指针仍「存在」但指向 UB |
| **别名违规** | 同时 `&mut` 与读同一地址 |
