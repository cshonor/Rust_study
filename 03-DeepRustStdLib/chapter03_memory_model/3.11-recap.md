## 一句话

第 3 章是 **跨越 Rust 陡峭曲线** 的关键 — 用 **Trait/元数据/生命周期** 把 C/C++ 式危险内存操作 **分类并安全封装**；所有权、借用、生命周期 **最终都可归结为内存操作**。

---

## 原书总结 · 板块速查

| 板块 | 节 | 核心 |
|------|:---:|------|
| **裸指针** | 3.1～3.3 | `*const/*mut` · 瘦/胖指针 · **90% unsafe = ptr→&** |
| **未初始化** | 3.2 | **`MaybeUninit`** · **`ManuallyDrop`** |
| **封装指针** | 3.4～3.5 | **`NonNull`** · **`Unique`** → Box/Vec 基座 |
| **mem** | 3.6 | **take/replace/forget** |
| **堆** | 3.7 | **`Layout`** · **`GlobalAlloc` / Global / System** |
| **静态/drop** | 3.8～3.9 | 全局内存 · **Drop 语义** |
| **理论** | 3.10 | **所有权 · 生命周期 · 型变** |

---

## 读完后应能回答

1. 裸指针与 **`&T`** 的三条核心差异？
2. **`assume_init`** 与 **`write`** 各承担什么安全责任？
3. **`Option<NonNull<T>>`** 为何能 niche 优化？
4. **`Global`** vs **`System`** 各在哪一层？
5. **`&mut T`** 为何对 **`T` 不变（invariant）**？
