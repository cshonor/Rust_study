# 3.2.1 MaybeUninit\<T\> 定义

> 章索引：[第 3 章](./README.md) · 前：[3.2-maybeuninit](./3.2-maybeuninit.md) · 后：[3.2.2-manuallydrop-definition](./3.2.2-manuallydrop-definition.md)

---

## 一句话

**`MaybeUninit<T>`** 是 **`union` 式封装** — 告诉编译器「内部 **可能尚未初始化**」，从而 **禁止** 在未初始化时按 `T` 做安全读取；初始化后用 **`assume_init()`**（unsafe）取出 `T`。

---

## 原书要点

| 能力 | API 方向 |
|------|----------|
| 创建未初始化槽 | `MaybeUninit::<T>::uninit()` |
| 写入初始化 | **`write(self, val)`** · `ptr.write(val)` |
| 取出已初始化 `T` | **`assume_init()`** · **`assume_init_read()`** 等 — **unsafe** |
| 获取指针 | `as_ptr()` / `as_mut_ptr()` — 仍须保证初始化后再解引用 |

### 安全边界

- **`MaybeUninit` 本身 safe** — 不能 **safe** 地读出未初始化的 `T`。
- **`assume_init`** — 程序员证明 **已完整初始化**，否则 UB。

```rust
let mut slot = MaybeUninit::<String>::uninit();
slot.write(String::from("tick"));
let s = unsafe { slot.assume_init() };
```
