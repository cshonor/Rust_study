# 3.10 Rust 所有权、生命周期、借用探讨

> 章索引：[第 3 章](./README.md) · 前：[3.9-drop-summary](./3.9-drop-summary.md) · 后：[3.11-recap](./3.11-recap.md)

## 一句话

底层内存工具之上，Rust 安全理念可归结为：**所有权**（值唯一定位）· **生命周期**（类型 = 结构 + 生命周期）· **变异性**（协变/逆变/不变 — 解释生命周期子类型为何有时通过、有时被拒）。

---

## 原书要点

### 所有权（Ownership）

| 原书观点 | 含义 |
|----------|------|
| **本质** | 值在内存块之间 **转移（浅拷贝指针/位模式）** 时，维持 **「值唯一定位」** 的标志 |
| **move** | 旧位置 **失效** · 新位置 **负责 drop** |
| 与 C++ | Rust **默认 move** + 编译期 **禁止 use-after-move** |

↔ [第 2 章 · 安全封装](../chapter02_rust_features_summary/2.2-memory-safety-overview.md)

### 生命周期（Lifetime）

| 原书观点 | 含义 |
|----------|------|
| **动机** | 防止 **悬垂引用**（借用 outlives 被借数据） |
| **升格** | 生命周期成为 **类型系统一部分** — 概念上变量类型 = **「数据结构类型 + 生命周期」** |
| **标注** | `'a` 约束 **引用有效区间** |

↔ [_supplement/legacy-lifetimes](./_supplement/legacy-lifetimes-and-borrow-check.md) · [Nomicon 03 型变](../../04-Rust-Nomicon/03_Lifetime_Variance/README.md)

### 变异性（Variance）

| 种类 | 直觉（简化） |
|------|--------------|
| **协变 (covariant)** | 生命周期 **更长** 的引用可当作 **更短** 使用（在 safe 子类型方向） |
| **逆变 (contravariant)** | 函数参数等 **反向** 子类型方向 |
| **不变 (invariant)** | **`&mut T`**、含 **`&mut`** 的容器 — **长度必须完全一致** |

> 理解 **为何 `'long: 'short` 有时能赋给短引用、有时 `&mut` 不行** — 泛型生命周期推导的核心。

### 与裸指针章节的闭环

```text
裸指针无生命周期 → 转 &T 时「发明」生命周期
  → 所有权/借用/型变 是 safe Rust 对同一内存问题的上层答案
```
