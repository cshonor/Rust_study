## 一句话

裸指针通过 **`Pointee` trait + Metadata** 区分 **瘦指针**（仅地址，Sized 类型）与 **胖指针**（地址 + 长度或 vtable 等元数据，DST）。

---

## 原书要点

### Pointee 与 Metadata

| 概念 | 说明 |
|------|------|
| **`Pointee`** | 被指向类型 · 决定指针是否带 **元数据** |
| **Metadata** | 附加在指针上的 **第二字**（概念上）— 切片长度、`dyn Trait` 的 vtable 等 |

### 瘦指针 vs 胖指针

| 类型 | 指针形态 | 元数据 |
|------|----------|--------|
| **`i32` · `Struct`** 等 **Sized** | **瘦指针** | 无（仅地址） |
| **`[T]` 切片 DST** | **胖指针** | **长度** |
| **`dyn Trait`** | **胖指针** | **vtable** 指针等 |
| **`str`** | **胖指针** | **字节长度** |

```text
瘦:  [*const i32]     →  地址
胖:  [*const [T]]     →  地址 + len
     [*const dyn Debug] →  地址 + vtable
```

### 与引用

`&[T]`、`&dyn Trait` 在内存表示上也是 **胖指针** — 安全引用携带 **编译器校验过的** 元数据与生命周期。
