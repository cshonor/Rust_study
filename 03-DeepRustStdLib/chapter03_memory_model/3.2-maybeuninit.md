## 一句话

底层数据结构常需 **先占坑、后初始化** — **`MaybeUninit<T>`** 合法表示「可能未初始化」；配合 **`ManuallyDrop<T>`** 避免对未初始化或已转移内存 **误 `drop`**。

---

## 原书要点

### 为何需要

| 编译器默认规则 | 底层现实 |
|----------------|----------|
| 变量使用前必须初始化 | `Vec` 扩容 · **数组槽位** · 内存池 — 先 **`alloc` 槽** 再逐个 **`write`** |
| 对未初始化读 = UB | 需要 **类型系统承认「未初始化」状态** |

### 典型流程

```text
MaybeUninit::uninit()
  → ptr.write(value) 或 write() 方法
  → assume_init()   // unsafe：你已证明已初始化
  → 得到 T
```
