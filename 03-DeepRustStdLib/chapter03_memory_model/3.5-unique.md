# 3.5 智能指针的基座——Unique\<T\>

> 章索引：[第 3 章](./README.md) · 前：[3.4.3-nonnull-other-fns](./3.4.3-nonnull-other-fns.md) · 后：[3.6-mem-module](./3.6-mem-module.md)

## 一句话

**`Unique<T>`**（内部/nightly 概念）在 **`NonNull<T>`** 上叠加 **`PhantomData<T>`** — **逻辑声明独占所有权**，生命周期结束时应对应 **释放** 所指向资源；是 **`Box` / `Vec` 底层** 的常见模式。

---

## 原书要点

### 与 NonNull 对比

| | `NonNull<T>` | `Unique<T>` |
|---|--------------|-------------|
| 非空 | ✅ | ✅ |
| **所有权语义** | 无（仅地址约束） | **`PhantomData<T>`** 表 **拥有** `T` |
| drop 含义 | 不自动释内存 | 类型 **析构** 表示 **独占资源应释放**（由外层容器实现） |

### 在标准库中的体现

现代 `Box`/`Vec` 实现用 **`Unique`/`NonNull` + 分配器** 组合 — 读 `alloc::boxed::Box` 源码可见。

```text
Box<T>  ≈  Unique/NonNull  +  Allocator  +  Drop 释堆
```
