# 14.2.2 Future Trait 分析

> 章索引：[第 14 章](./README.md) · 前：[14.2.1](./14.2.1-coroutine-management.md)

---

## 一句话

**`Future`** — **`fn poll(self: Pin<&mut Self>, cx: &mut Context) -> Poll<Self::Output>`** · **惰性** · **未完成返回 `Pending`** 并通过 **`Context` 中的 `Waker`** 注册唤醒；**`async` 状态机、Pin、epoll** 均服务这一接口。

---

## 原书要点（脉络 · p.421+ 源码待刷书）

### 核心 API

```rust
pub trait Future {
    type Output;
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}

pub enum Poll<T> {
    Ready(T),
    Pending,
}
```

| 设计 | 含义 |
|------|------|
| **`Pin<&mut Self>`** | [7.4 · 自引用状态机](../chapter07_interior_mutability/7.4-pin-unpin.md) 不可被 move |
| **`Pending`** | 「现在没进展」· **非阻塞** |
| **`Waker`** | 就绪后再 poll · 衔接 [14.1.2 · 多路复用](./14.1.2-io-multiplexing.md) |
| **无 `async fn` in trait（旧）** | 动态 async trait 曾受限 · 现语言持续演进 |

### 与阻塞 I/O（Ch13）

```text
阻塞: thread 在 read() 内核 sleep
async:  Future 返回 Pending → epoll 等 fd → wake → 再 poll
```

### HFT

**少 poll、少 wake** — 批处理就绪事件 · 避免 executor 队列抖动；超低延时路径仍可能 **不用 async**。
