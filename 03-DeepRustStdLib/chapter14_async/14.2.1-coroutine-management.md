# 14.2.1 Rust 协程管理

> 章索引：[第 14 章](./README.md) · 前：[14.2](./14.2-coroutine-types.md) · 后：[14.2.2 · Future](./14.2.2-future-trait.md)

## 一句话

**协程管理 = Executor（执行器）** — 维护 **待 poll 的 Future 任务队列** · **收到 Waker 唤醒** 后再 **`poll`**；**`tokio::spawn` / `async_std::task::spawn`** 是生态层，**`std` 无内置全局 executor**（[11.11](../chapter11_concurrency/11.11-runtime.md)）。

---

## 原书要点（脉络 · 待刷书）

| 职责 | 说明 |
|------|------|
| **调度** | 哪个 Future 下一次 `poll` |
| **唤醒** | **`Waker::wake`** → 重新入队 |
| **Pin** | **`Pin<Box<dyn Future>>`** 常见存储自引用状态机 |

```text
spawn(fut) → Task 入队
reactor: epoll 就绪 → Waker::wake(task)
executor: poll(task) until Pending/Ready
```

### 与 Ch11 对比

| | **OS 线程 + Mutex** | **async Executor** |
|---|---------------------|---------------------|
| 切换成本 | 内核调度 | 用户态 poll |
| 阻塞 read | 占线程 | 应改 **async read** + Pending |
