## 同步 vs 异步（Ch11～13 vs Ch14）

| | **同步（Ch9～13）** | **异步（Ch14 + 运行时）** |
|---|---------------------|---------------------------|
| 等待 IO | 线程 **内核阻塞** | Future **`Pending`** · 线程干别的 |
| 并发成本 | 一线程一连接 · 内存大 | 少量线程 + 大量 Future |
| 适用 | **CPU 密集** · 线程池 | **IO 密集** · 网关/爬虫/后端 |
| 锁 | **`std::sync::Mutex`** | 生态 **`tokio::Mutex`**（await 友好） |

二者 **互补** · 非替代。

## 全书递进闭环

```text
Ch3  裸指针/内存        →  unsafe 与布局
Ch4-6 类型/迭代         →  Trait 与零成本
Ch7-8 内部可变/智能指针  →  Pin · Arc
Ch9-13 std+OS          →  SYSCALL · fd · 阻塞 I/O · 锁
Ch14 async             →  Future + 多路复用  →  高并发 I/O 前沿
```

### Rust 设计哲学（章终）

| 原则 | Ch14 体现 |
|------|-----------|
| **零成本抽象** | 不用 async 无运行时开销 |
| **内存安全** | **Pin** 编译期杜绝自引用悬垂 |
| **分层解耦** | core 只给 **Future** · Executor 第三方 |

## 常见误区

| 误区 | 正解 |
|------|------|
| **`async fn` 立刻执行** | **惰性** · 须 **poll/spawn** 才推进 |
| **std 自带 tokio** | **无** · 见 [11.11](../chapter11_concurrency/11.11-runtime.md) |
| **Pin = 堆分配** | **`pin_mut!`** 可栈 Pin · **`Box::pin`** 最常见 |
| **async 一定更快** | IO 密集才收益 · CPU 密集仍用线程 |
| **`.await` = 新线程** | **同线程** 状态机切换 · 多任务靠 executor |
