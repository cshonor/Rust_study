## 一句话

**async I/O 的 OS 底座** — 单线程用 **epoll/kqueue/IOCP** **同时等待多个 fd 就绪** · 就绪后 **Waker 唤醒** 对应 Future；**std 无内置 reactor** · 见 **mio/tokio** · 阻塞模型见 [Ch13](../chapter13_io/13.3-network-io.md)。

---

## 原书要点

### std 与运行时分工

| 层 | 提供方 |
|----|--------|
| **Future · Waker · Pin** | **core/std 抽象** |
| **epoll/kqueue/IOCP 封装** | **mio → tokio/async-std** |
| **阻塞 TcpStream::read** | **std::net（Ch13）** |

### 多路复用机制

```text
register(fd, READ|WRITE) on epoll
loop:
  epoll_wait → 就绪 fd 列表
  → 对应 Task 的 Waker::wake()
  → executor 再 poll 该 Future
```

| API | 平台 |
|-----|------|
| **epoll** | Linux |
| **kqueue** | macOS / BSD |
| **IOCP** | Windows |
| **poll/select** | legacy · 大规模性能差 |

### 阻塞 IO vs 异步 IO（联动 Ch13）

| 阻塞（Ch13） | 异步（Ch14+运行时） |
|--------------|---------------------|
| 一线程 **等一个** socket | **一线程监听上万** socket |
| `read()` **内核 sleep** | Future **`Pending`** · 线程不空等 |
| 上万连接 → **上万线程** · 内存爆炸 | **少量线程 + 大量 Future** |

↔ [9.4 · SYSCALL](../chapter09_userspace_std_basics/9.4-syscall-wrapper.md) · [05-async](../../05-Async-Concurrency-Network/02-async/README.md)

### HFT

**单线程 epoll 网关** · **ET 边缘触发** vs **LT** · 超低延时路径可能 **不用 async**（DPDK 等超出 std）。
