# 14.1.2 Rust 的 I/O 多路复用

## 一句话

**高吞吐 async I/O 的 OS 魔法** — 单线程（或少量线程）用 **`epoll`（Linux）/ `kqueue`（BSD/macOS）/ `IOCP`（Windows）** 同时监听 **大量 socket fd 就绪事件**，避免 [Ch13 · 阻塞 TcpStream](../chapter13_io/13.3-network-io.md) **一线程一连接**。

---

## 原书要点（脉络 · 待刷书）

```text
register(fd, READ|WRITE) on epoll
  loop: epoll_wait → 就绪 fd 列表 → 唤醒对应 Future（Waker）
```

| 机制 | 平台 |
|------|------|
| **epoll** | Linux |
| **kqueue** | macOS / BSD |
| **IOCP** | Windows |
| **poll/select** | legacy · 规模差 |

### 与 std / 生态

| 层 | 说明 |
|----|------|
| **syscall** | [9.4 · SYSCALL](../chapter09_userspace_std_basics/9.4-syscall-wrapper.md) |
| **std** | 阻塞 **`TcpStream::read`** |
| **mio/tokio** | 封装 epoll · 驱动 **Reactor** · [05-async](../../05-Async-Concurrency-Network/02-async/README.md) |

### HFT

**单线程 epoll 行情网关** — 多连接 **边缘触发 ET** · 与 **busy-poll** 权衡；低延时常仍用 **专用线程 + 内核 bypass**（DPDK 等，超出 std）。
