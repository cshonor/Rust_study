# 14.1.1 协程概述

> 章索引：[第 14 章](./README.md) · 前：[14.1](./14.1-coroutine-framework.md) · 后：[14.1.2 · 多路复用](./14.1.2-io-multiplexing.md)

## 一句话

**`async fn` / `async { }`** 在编译期 **desugar 为实现 `Future` 的状态机**（常为 **enum** 各状态 + 局部变量）— **惰性**：不 poll 不前进；与 **Go/Java 内置调度器** 不同，**Rust 无默认全局 async Runtime**。

---

## 原书要点（脉络 · 待刷书）

```rust
async fn fetch() -> Result<Data, Error> {
    let x = read_header().await?;
    parse(x).await
}
// 编译器 ≈ 生成 enum FetchFuture { State0, State1, … }
```

| 点 | 说明 |
|----|------|
| **协程 ≈ Future** | 语言 **coroutine** 通过 **`Future` trait** 落地 |
| **`.await`** | 订阅子 **Future** · 未完成则 **yield Pending** |
| **Send bound** | **`async fn` 跨线程** 须生成的 Future 为 **`Send`** |

### 知识闭环：`Pin`（原书隐含 · [Ch7](../chapter07_interior_mutability/7.4-pin-unpin.md)）

| 问题 | 对策 |
|------|------|
| 状态机 **自引用**（await 点局部变量引用） | **`Pin<&mut impl Future>`** 禁止 unsafe move |
| 编译器生成 **`!Unpin`** 状态机 | **`Box::pin`** / executor 固定堆位置 |
