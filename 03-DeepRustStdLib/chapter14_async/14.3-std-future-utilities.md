## 一句话

**core/std 内置的极简 Future 组合** — **`Ready`/`Pending`** 无需第三方运行时 · **`join!`/`select!`** 基础多任务等待（完整版在 **futures/tokio** 宏）。

---

## 原书要点

### 1. `Ready<T>` / `ready()`

```rust
// core::future::Ready — poll 一次即 Ready
let fut = async { 42 };
// 等价工具：ready(42) 立即完成的 Future
```

| 用途 | 说明 |
|------|------|
| **同步值转 Future** | 适配 async 接口 |
| **测试 / 占位完成** | poll 直接 **Ready** |

### 2. `Pending` / `pending()`

```rust
// 永远 Poll::Pending — 测试 executor · 占位
```

### 3. 组合逻辑（标准库 + 生态）

| 工具 | 所在 | 行为 |
|------|------|------|
| **`Future::poll` 链** | core | 手写组合 |
| **`join!`** | **`std` 有限 / `futures`·`tokio` 宏** | **全部** 完成才 Ready |
| **`select!`** | 生态宏为主 | **任一** 完成即返回 |
| **`poll_fn`** | `core::future::poll_fn` | 闭包实现 Future |

**无需 executor 即可 unit test** 单个 Future 的 poll 行为 · 集成运行仍要 **tokio** 等。
