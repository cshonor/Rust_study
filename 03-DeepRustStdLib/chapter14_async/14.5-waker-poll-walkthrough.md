# 14.5 Waker 与 poll 机制走读

> 章索引：[第 14 章](./README.md) · 相关：[14.2](./14.2-coroutine-types.md) · [14.1.2](./14.1.2-io-multiplexing.md)

## 一句话

**`Waker` = 可克隆的唤醒句柄** — IO 就绪时 **原子标记 + 通知 executor** 再 **poll**；概念骨架对照 **`core::task::Wake`** trait。

---

## A. poll 注册 Waker（概念）

```rust
impl Future for SocketReadFuture {
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<usize> {
        if self.socket.is_readable() {
            return Poll::Ready(self.socket.read_inner());
        }
        // 未就绪：向 reactor 注册 cx.waker()
        self.socket.register_readable(cx.waker());
        Poll::Pending
    }
}
```

**同一 Future 多次 poll** · 每次 Pending 可能 **更新 Waker**（任务迁移线程时）。

---

## B. Wake trait（core 概念）

```rust
pub trait Wake {
    fn wake(self: Arc<Self>);
    fn wake_by_ref(self: &Arc<Self>) { ... }
}

pub struct Waker {
    // type-erased Arc<dyn Wake>
}
impl Waker {
    pub fn wake(&self) { ... }
    pub fn wake_by_ref(&self) { ... }
}
```

**tokio** 为每个 Task 实现 **Wake** · **`wake()` → 入队 executor**。

---

## C. 与 epoll 衔接

```text
1. poll → Pending · register(waker, fd, READ)
2. epoll_wait 返回 fd 可读
3. reactor 查找 fd 对应 Waker
4. waker.wake_by_ref()
5. executor poll 该 Task  again → Ready
```

## D. Waker 原子与内存序（方向）

| 点 | 说明 |
|----|------|
| **wake 可跨线程** | **`Arc<Wake>`** · 原子 **queued 标志** |
| **虚假唤醒** | poll 须 **幂等** · Ready 前可多次 wake |
| **coalesce** | **`core::task::coalesce`** 合并重复 wake（优化） |

细节以 **`library/core/src/task/wake.rs`** · **tokio 源码** 为准。

## E. std::sync::Mutex vs tokio::sync::Mutex（延伸）

| | **`std::sync::Mutex`** | **`tokio::sync::Mutex`** |
|---|------------------------|--------------------------|
| 持锁等 IO | **阻塞 OS 线程** | **`lock().await`** · 释放线程 |
| 底层 | [Ch11 futex](../chapter11_concurrency/11.12-mutex-futex-walkthrough.md) | async 队列 + Waker |
| 场景 | **同步代码 / CPU 临界区** | **async 任务间共享状态** |
