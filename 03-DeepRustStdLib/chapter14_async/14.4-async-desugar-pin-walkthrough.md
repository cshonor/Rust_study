## 一句话

## A. 源码级 async 示例

```rust
async fn demo() -> u32 {
    let a = read_io().await;
    a + 1
}
```

---

## B. 概念脱糖（三状态）

```rust
enum DemoFuture {
    // 初始：开始 poll read_io
    Start,
    // 等待 read_io 完成 · 保存 ReadIoFuture 子状态机
    WaitingRead { read_fut: ReadIoFuture },
    // 已完成（或内嵌结果）
    Done,
}

impl Future for DemoFuture {
    type Output = u32;

    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<u32> {
        let this = self.get_mut(); // 需 unsafe 或 pin-project — 示意
        match this {
            DemoFuture::Start => {
                *this = DemoFuture::WaitingRead { read_fut: read_io() };
                // 递归/循环 poll 子 Future…
                Poll::Pending
            }
            DemoFuture::WaitingRead { read_fut } => {
                match Pin::new(read_fut).poll(cx) {
                    Poll::Ready(a) => Poll::Ready(a + 1),
                    Poll::Pending => Poll::Pending,
                }
            }
            DemoFuture::Done => unreachable!(),
        }
    }
}
```

真实编译器输出更复杂 · 常用 **`pin-project`** / **`async-trait`** 等 crate 辅助手写 Future。

---

## C. 自引用为何发生

```text
状态 WaitingRead 内可能同时存：
  · read_fut 子 Future
  · 指向 read_fut 内部 buffer 的指针（编译器优化布局）
整体 enum 若被 move（栈搬家、放入 Vec  realloc）→ 内部指针悬垂
```

## D. Pin 与 Unpin

| 类型 | Unpin? | 要求 |
|------|--------|------|
| **`i32`、普通 struct** | **是** | 可 **`&mut T`** poll |
| **async 状态机** | **否（默认）** | 必须 **`Pin<&mut Self>`** |
| **堆上固定** | — | **`Box::pin(async { ... })`** |
| **栈上固定** | — | **`pin_mut!(x)`** 宏 |

```rust
let fut = Box::pin(async {
    let x = read_io().await;
    x + 1
});
// spawn 时传入 Pin<Box<impl Future>>
```

↔ [Ch8 · Box](../chapter08_smart_pointers/) · **`Pin<&mut RefCell<...>>`** 等内部可变组合（高级）

---

## E. 常见误区

| 误区 | 正解 |
|------|------|
| **手动 move async 块** | **`!Unpin`** · 用 **Pin** |
| **Pin = 永远堆分配** | **`pin_mut!`** 栈 Pin |
| **Pin 阻止所有移动** | 只约束 **被 pin 的值** · **`Pin<Box<T>>` 仍可 move Box 本身** |
