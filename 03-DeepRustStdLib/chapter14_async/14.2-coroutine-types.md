## 一句话

**前言点名的 async 基石** — **`Poll` · `Context` · `Waker` · `Future`** 在 **`core::future` / `core::task`**；Executor 用它们 **驱动 poll**（[14.2.1](./14.2.1-coroutine-management.md)）· trait 定义见 [14.2.2](./14.2.2-future-trait.md)。

## 原书要点

### 核心三要素

#### 1. `Poll<T>`

```rust
pub enum Poll<T> {
    Ready(T),   // 完成 · 返回 Output
    Pending,    // 未就绪 · 须挂起
}
```

**`poll` 是唯一推进接口** — 运行时反复 **`future.poll(&mut cx)`**。

#### 2. `Waker`

- 保存 **唤醒任务** 的句柄；
- IO 就绪 / 定时器到期 → **`waker.wake()`**（或 `wake_by_ref`）；
- Executor 将 Task **重新入队** · 下次 **poll**。

#### 3. `Context<'_>`

- **仅包装 `&Waker`**；
- 传入 **`poll`** · Future 在 **Pending** 前向 reactor **注册 Waker**。

```text
Executor                         Future (async 状态机)
  loop {                           poll(Pin<&mut Self>, &mut Context)
    ready = task.poll(cx)              ├─ Ready(T)  → 完成
    if Pending → 运行其他任务          └─ Pending   → 已注册 Waker
  }
reactor: epoll 就绪 → Waker::wake(task)
```

| 类型 | 位置 |
|------|------|
| **`Poll`** | `core::task` |
| **`Context` · `Waker`** | `core::task::wake` |
| **`Future`** | `core::future` |
