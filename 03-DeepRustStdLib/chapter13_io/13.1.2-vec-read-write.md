# 13.1.2 向量读/写类型分析

> 章索引：[第 13 章](./README.md) · 前：[13.1.1](./13.1.1-read-trait.md) · 后：[13.1.3 · Stdin API](./13.1.3-stdin-public-api.md)

---

## 一句话

**Vectored I/O** — **`read_vectored` / `write_vectored`** + **`IoSlice` / `IoSliceMut`**，对应 Linux **`readv`/`writev`**：一次 syscall 读写 **多个分散 buffer**，减少 **用户态拷贝** 与 **syscall 次数**。

---

## 原书要点（脉络 · 性能向 · 待刷书）

```text
传统:  read(buf1); read(buf2);     → 2 次 syscall
向量:  read_vectored(&[slice1, slice2])  → 1 次 readv
```

| 类型 | 作用 |
|------|------|
| **`IoSlice<'a>`** | 只读 scatter 源 |
| **`IoSliceMut<'a>`** | 可写 gather 目标 |
| **`Read::read_vectored`** | 默认 fallback 循环 `read` · 平台可 override |
| **`Write::write_vectored`** | 对应 **writev** |

### 零成本与极限性能

| 收益 | 说明 |
|------|------|
| **少 syscall** | 高 QPS 网络/日志 |
| **少拷贝** | 头+体已在不同 buffer · 一次写出 |

### HFT

自定义 **`TcpStream` 封装** 可在热路径用 **writev** 发 **header + payload**；须 profiling 对比 **`write_all` 两次**。

---

## 源码锚点

`library/std/src/io/mod.rs`（`IoSlice`）· `library/std/src/sys/` 平台 readv/writev
