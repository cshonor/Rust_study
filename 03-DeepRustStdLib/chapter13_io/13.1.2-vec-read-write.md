# 13.1.2 向量读/写类型分析

> 章索引：[第 13 章](./README.md) · 前：[13.1.1](./13.1.1-read-trait.md) · 后：[13.1.3 · Stdin API](./13.1.3-stdin-public-api.md)

---

## 一句话

**Vectored I/O** — **`ReadVectored` / `WriteVectored`** + **`IoSlice`/`IoSliceMut`**，对应 Unix **`readv`/`writev`** · Windows **ReadFileScatter/WriteFileGather**：**一次 syscall** 读写 **多块分散 buffer**。

---

## 原书要点

```text
传统:  read(buf1); read(buf2);           → 2 次 syscall + 可能拼接拷贝
向量:  read_vectored(&[slice1, slice2])  → 1 次 readv
```

| 类型 / 方法 | 作用 |
|-------------|------|
| **`IoSlice<'a>`** | 只读 scatter 源 |
| **`IoSliceMut<'a>`** | 可写 gather 目标 |
| **`read_vectored`** | 默认 fallback 循环 `read` · 平台 override → **readv** |
| **`write_vectored`** | 对应 **writev** |

### 性能收益

| 收益 | 场景 |
|------|------|
| **少 syscall** | 高 QPS 网络 · 批量日志 |
| **少用户态拷贝** | header + body 已在不同 buffer · **一次写出** |

### 平台

| Unix | Windows |
|------|---------|
| **`libc::readv` / `writev`** | **ReadFileScatter / WriteFileGather** |

底层在 **`sys`** · trait 在 **`std::io`**。

→ 注释版：[13.4 · 向量 IO 走读](./13.4-vectored-stdio-walkthrough.md)

### HFT

热路径 **writev** 发 **header + payload** · 须 profiling 对比 **两次 write_all**。

`library/std/src/io/mod.rs`（`IoSlice`）· `library/std/src/sys/*/io/` · `File`/`TcpStream` 的 vectored override
