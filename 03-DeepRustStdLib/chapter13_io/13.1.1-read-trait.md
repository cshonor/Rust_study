# 13.1.1 Read Trait

## 一句话

**`Read` / `Write`** 是 **字节流 I/O 的基石** — 文件、**stdin/stdout**、**TcpStream** 均实现它们；上层 **`read`/`read_exact`/`BufReader`** 均建立在此 Trait 之上。

---

## 原书要点（脉络 · 待刷书）

### Read

| 方法 | 作用 |
|------|------|
| **`read(&mut [u8])`** | 读入 **部分或全部** buffer · 返回字节数 |
| **`read_exact`** | 读满或 **Error** |
| **`read_to_end` / `read_to_string`** | 读到 EOF |

### Write（目录未单列 · 与 Read 成对）

| 方法 | 作用 |
|------|------|
| **`write`** | 可能 **部分写** |
| **`write_all`** | 写满 |
| **`flush`** | 刷缓冲 |

```rust
use std::io::{Read, Write};
let n = stream.read(&mut buf)?;
stream.write_all(&packet)?;
```

### 统一抽象（章核心）

```text
磁盘 File · Stdin · TcpStream  ──►  impl Read / Write
  └── 开发者同一套 API · 底层 SYSCALL 不同（read vs recv）
```

### HFT

**协议帧** — **`read_exact(len)`** 定长头 + body · 避免 **`read` 短读** 拼包 bug。
