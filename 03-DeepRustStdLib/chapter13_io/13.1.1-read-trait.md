# 13.1.1 Read Trait

## 一句话

**`Read` / `Write`** 是 **IO 体系根基** — 定义字节流统一行为；**File、stdin/stdout、管道、TcpStream** 全部实现，读写代码 **无需区分** 磁盘、控制台还是网络。

---

## 原书要点

### Trait 核心

```rust
pub trait Read {
    fn read(&mut self, buf: &mut [u8]) -> Result<usize>;
    // read_exact · read_to_end · read_to_string 等默认方法
}
pub trait Write {
    fn write(&mut self, buf: &[u8]) -> Result<usize>;
    fn flush(&mut self) -> Result<()>;
    // write_all · write_fmt 等
}
```

### 常用方法

| Read | 说明 |
|------|------|
| **`read`** | 读 **≤ buf.len()** · 可能短读 |
| **`read_exact`** | 读满或 **Error** |
| **`read_to_end` / `read_to_string`** | 读到 EOF |

| Write | 说明 |
|-------|------|
| **`write`** | 可能 **部分写** |
| **`write_all`** | 写满 |
| **`flush`** | 刷缓冲 |

### IO 多态（章核心）

```text
磁盘 File · Stdin · 管道 · TcpStream  ──►  impl Read / Write
  └── 泛型 fn copy<R: Read, W: Write>(...)  一套逻辑适配所有载体
  └── 底层 syscall 不同：read vs recv · 上层 trait 相同
```
