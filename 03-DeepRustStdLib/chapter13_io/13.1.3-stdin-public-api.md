# 13.1.3 对外接口层

> 章索引：[第 13 章](./README.md) · 前：[13.1.2](./13.1.2-vec-read-write.md) · 后：[13.2 · Stdout](./13.2-stdout.md)

---

## 一句话

**`io::stdin()` / `Stdin` / `StdinLock`** — 将 **fd 0** 以 **`Read`** 暴露；多线程访问 stdin 时 **`lock()` 全局锁**，保证 **并发读终端** 不交错破坏字节流。

---

## 原书要点（脉络 · 待刷书）

```rust
let mut handle = io::stdin().lock(); // StdinLock: Read
handle.read_line(&mut line)?;
```

| 组件 | 说明 |
|------|------|
| **`stdin()`** | 返回 **`Stdin`**（共享句柄） |
| **`Stdin::lock()`** | **`StdinLock<'a>`** · 持 **内部 Reentrant 锁** |
| **底层** | **OwnedFd 0** · [9.5](./chapter09_userspace_std_basics/9.5.1-fd-ownership.md) |

### 为何全局锁

| 问题 | 对策 |
|------|------|
| 两线程同时 **`read`** stdin | 字节 **交错** · 协议崩溃 |
| **`println!` + read** | stdout 亦 **[13.2 锁](./13.2-stdout.md)** |

↔ [11.2 · Mutex 思想](../chapter11_concurrency/11.2-mutex-overview.md)（stdio 用专用全局锁而非用户 Mutex）
