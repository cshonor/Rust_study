## 一句话

**`io::stdin()` / `Stdin` / `StdinLock`** — **fd 0** 以 **`Read`** 暴露；**全局共享** · 多线程须 **`lock()`** 独占，防 **并发读终端字节交错**。

---

## 原书要点

```rust
let mut handle = io::stdin().lock(); // StdinLock: Read
handle.read_line(&mut line)?;
```

| 组件 | 说明 |
|------|------|
| **`stdin()`** | 返回 **`&'static Stdin`** 全局句柄 |
| **`Stdin::lock()`** | **`StdinLock`** · 持 **内部全局 Mutex** |
| **短读** | 部分 API 内部 **短时加锁** |
| **底层** | **fd 0** · 与 [Ch12 File](../chapter12_filesystem/12.2-fs-public-api.md) **同一 Read 逻辑** |

### 为何全局锁

| 问题 | 对策 |
|------|------|
| 两线程同时 **`read` stdin** | 字节 **交错** · 行协议崩溃 |
| **与 stdout 配合** | [13.2](./13.2-stdout.md) 输出亦 **全局锁** |
