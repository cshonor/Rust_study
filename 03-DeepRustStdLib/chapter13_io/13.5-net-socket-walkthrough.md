## 一句话

**Unix Berkeley Socket** 与 **Winsock** 在 **`sys::net`** 收敛 — 对上 **`TcpStream: Read + Write`** · **Drop → close**。

---

## A. TcpStream connect（Unix 概念）

```rust
// sys/unix/net.rs — 骨架
pub fn connect(addr: &SocketAddr) -> io::Result<TcpStream> {
    let socket = Socket::new(addr.domain(), Type::STREAM)?;
    socket.connect(addr)?;
    Ok(TcpStream { inner: socket })
}

impl Read for TcpStream {
    fn read(&mut self, buf: &mut [u8]) -> io::Result<usize> {
        self.inner.read(buf) // recv(2) 或 read on socket fd
    }
}
impl Write for TcpStream {
    fn write(&mut self, buf: &[u8]) -> io::Result<usize> {
        self.inner.write(buf) // send(2)
    }
}
```

---

## B. TcpListener accept

```rust
pub fn accept(&self) -> io::Result<(TcpStream, SocketAddr)> {
    let (stream, addr) = self.inner.accept()?;
    Ok((TcpStream { inner: stream }, addr))
}
```

**`accept` 阻塞** 直到有连接 · async 版在 **tokio/mio + epoll**。

## D. Drop 关闭

```rust
impl Drop for TcpStream {
    fn drop(&mut self) {
        let _ = self.inner.close(); // close(fd) / closesocket
    }
}
```

## E. Windows Winsock 差异

```text
进程启动: WSAStartup（std 运行时/首次 socket 时）
API:      WSASocket · connect · recv/send
句柄:     SOCKET → 内部适配 Read/Write
关闭:     closesocket
```

上层 **`TcpStream::connect("host:port")`** **无 cfg 分支**。
