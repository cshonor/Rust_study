# 13.2 标准输出 Stdout 类型分析

> 章索引：[第 13 章](./README.md) · 前：[13.1.3](./13.1.3-stdin-public-api.md) · 后：[13.3 · 网络](./13.3-network-io.md)

## 一句话

**`Stdout` / `Stderr` + `Write`** — **`println!`** 底层 **stdout 锁 → fd 1** · **`eprintln!` → fd 2**；**行缓冲/块缓冲/无缓冲** 策略 + **全局 Mutex** 防多线程打印交错。

---

## 原书要点

```rust
println!("tick={}", id);  // → stdout().lock().write_fmt(...)
io::stdout().lock().write_all(b"raw\n")?;
eprintln!("err");         // stderr · fd 2
```

| 流 | fd | 缓冲特性 |
|----|:--:|----------|
| **Stdout** | **1** | 终端 **行缓冲**（`\n` 触发 flush）· 重定向到文件 → **块缓冲** |
| **Stderr** | **2** | **无缓冲** · 错误立刻可见 |

| 机制 | 说明 |
|------|------|
| **全局锁** | **`stdout().lock()`** · 批量打印不被打断 |
| **与 File 统一** | 底层 **fd + Read/Write** · [Ch12](../chapter12_filesystem/12.2-fs-public-api.md) |
| **与管道** | [Ch10 · Stdio 重定向](../chapter10_process_management/10.2-redirection.md) **piped()** 改 fd 1/2 目标 |

### 与 Ch10 重定向

**`Command.stdout(Stdio::piped())`** — 子进程 fd 1 非 TTY · 缓冲策略随之变化 · [10.2](../chapter10_process_management/10.2-redirection.md)

### HFT

热路径 **避免 `println!`** — 分配 + 锁 + syscall · 用 **结构化日志 / ring + 异步落盘**。
