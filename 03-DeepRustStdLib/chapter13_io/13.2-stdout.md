## 一句话

**`Stdout` / `StdoutLock`** + **`Write`** — **`println!`/`print!` 宏** 底层 **`write_fmt` → stdout 锁 → fd 1**；与 [13.1.3 · Stdin 锁](./13.1.3-stdin-public-api.md) 对称，保证 **多线程打印不交错**。

---

## 原书要点（脉络 · 待刷书）

```rust
println!("tick={}", id);  // 展开 → stdout().lock().write_fmt(...)
io::stdout().lock().write_all(b"raw\n")?;
```

| 点 | 说明 |
|----|------|
| **fd 1** | stderr 为 **fd 2** · **`eprintln!`** |
| **缓冲** | 终端常 **行缓冲** · **`flush`** 及时落盘/显示 |
| **锁** | **`stdout().lock()`** 避免多行交错 |

### 与 Ch10 重定向

**`Command` `.stdout(Stdio::piped())`** — 子进程 fd 1 非终端 · [10.2](../chapter10_process_management/10.2-redirection.md)

### HFT

**热路径避免 `println!`** — 分配 + 锁 + 可能的 syscall · 用 **结构化日志 / ring + 异步落盘**。
