# 2.6 错误处理的标准库规范

> 章索引：[第 2 章](./README.md) · 前：[2.5 模式匹配](./2.5-pattern-matching-in-stdlib.md) · 后：[2.7 泛型](./2.7-generics-in-stdlib.md)

---

## 一句话

`std` 把 **可恢复失败** 统一成 **`Result<T, E>`** + **`?`** + **`Error::source()` 链** — 与你在 [RFR Ch04](../../02-RFR/Chapter-04-Error-Handling/README.md) 学的分层错误设计同源，I/O 则固定用 **`io::Error`**。

---

## 标准库错误分层

```text
调用方看到的业务 / 库 enum（上层，你的代码）
        │ ? / From
        ▼
std::io::Error / 其他官方 Error（底层）
        │ source()
        ▼
OS 错误码 / 自定义根因
```

| 层次 | 类型 | 典型场景 |
|------|------|----------|
| **泛型结果** | `Result<T, E>` | 所有可恢复 API |
| **I/O** | `io::Error` / `io::Result<T>` | `fs`、`net`、`Read` |
| **格式化** | `fmt::Error` | `write!` 失败（少见） |
| **线程** | `JoinError` | `join` 时 panic |
| **不可恢复** | `panic!` / `unwrap` | 编程错误、不变量破坏 |

---

## 规范要点（读 std / 写库时对齐）

1. **`Result` 表可恢复** — 文件不存在、连接重置 → `Err`；逻辑 bug → `panic` 或 `debug_assert`。
2. **`E: Error + 'static`** — 库类型实现 `Display` + `source()`，便于日志与 `anyhow` 链。
3. **`From` 抬升** — `io::Error` → 你的 `enum` 变体，`?` 自动转换（`thiserror` 生成）。
4. **`Option` vs `Result`** — 「没有值」不一定是错误（`HashMap::get`）；失败语义明确时用 `Result`。

## `std` 源码中的 `?`

`read`、`write`、`flush` 等层层 **`?` 传播 `io::Error`** — 读模块时跟一条调用链即可看懂错误如何冒泡到用户 `main`。
