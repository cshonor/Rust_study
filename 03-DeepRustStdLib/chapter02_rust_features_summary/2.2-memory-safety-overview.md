## 一句话

编译器只提供 **有限安全规则 + 明确 unsafe**；Rust **真正**的内存安全靠程序员用 **所有权、生命周期、Drop、Deref** 构建的 **安全封装类型** — 外部 API 无 `unsafe` 仍保证安全。

---

## 原书要点

### 编译器直接提供的安全（有限）

| 安全侧 | 示例 |
|--------|------|
| 变量必须初始化 | 禁止读未定义值 |
| 严格类型检查 | 禁止隐式错误转换 |
| 借用检查 | 互斥可变 / 共享不可变 |

| 不安全侧（显式） | 示例 |
|------------------|------|
| 裸指针解引用 | `*const T` / `*mut T` |
| **FFI** | `extern "C"` 调用 |

→ 第 3 章展开 · [Nomicon](../../04-Rust-Nomicon/README.md)

### 安全封装：真正的基石

**模式**：标准库内部大量 **`unsafe`** 实现 · 对外 **safe API** 承诺不变式。

| 封装类型 | 解决的问题 |
|----------|------------|
| **`Option<T>`** | 值可能 **不存在** |
| **`Rc<T>` / `Arc<T>`** | 一块内存 **多处引用**（单线程 / 多线程） |
| **`Box<T>`** | **堆** 所有权 |
| **`Mutex<T>`** | 线程间 **临界区**（`std`） |

### 借用 / 可变借用封装

| 类型 | 配合 | 机制 |
|------|------|------|
| **`RefCell<T>`** | **`Ref<T>` / `RefMut<T>`** | 运行时借用计数 · **`Drop` 释放** |
| **`Mutex<T>`** | **`MutexGuard<T>`** | 加锁 guard · **Drop 解锁** |

```text
安全闭环：
  unsafe 块内建立不变式
    → 封装类型对外 safe API
      → Ref/Guard 等 RAII · Drop 自动清理
        → 计数/锁释放 · 无泄漏
```
