## 一句话

实现 **`Deref` / `DerefMut`** 的封装类型（`Box`、`String`、`MutexGuard`…）在 **方法调用** 时由编译器 **自动解引用** — 无需手动 `(*x)` 即可调用内部 `T` 的方法。

---

## 原书要点

### 自动解引用（Deref coercion）

```rust
let s = String::from("hi");
s.len();              // 实际：Deref → str → len()
let b = Box::new(42);
*b + 1;               // 或依赖 Deref 到 i32 的上下文
```

| 机制 | 说明 |
|------|------|
| **`Deref` trait** | `type Target = ...` · `fn deref(&self) -> &Target` |
| **编译器插入** | 找方法时自动 **`&*`** 链直到匹配 |
| **透明用法** | 写起来像直接操作 **`T`** |

### 常见实现 `Deref` 的类型

| 类型 | `Target` |
|------|----------|
| **`Box<T>`** | `T` |
| **`String`** | `str` |
| **`Vec<T>`** | `[T]`（通过 `Deref` 到切片） |
| **`MutexGuard<T>`** | `T` |
| **`Ref<T>`** | `T` |

### 注意

- **自动解引用 ≠ 获得所有权** — 仍是借用内部。
- 过度依赖可能 **隐藏分配/锁** — 读签名知类型。
