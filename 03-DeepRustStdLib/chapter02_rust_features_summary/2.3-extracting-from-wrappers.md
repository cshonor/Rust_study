# 2.3 获取封装类型变量的内部变量

## 一句话

业务代码常要操作 **封装内部的 `T`** — 标准库提供 **五种安全范式**：**`?`** · **Deref 自动解引用** · **闭包/map** · **as_ref/as_ptr** · **take/replace 移所有权**。

---

## 原书要点

### 为何需要「解封装」

| 现实 | 做法 |
|------|------|
| API 返回 `Option<T>`、`MutexGuard<T>` | 要拿到 **`T` 或 `&T`** 才能计算 |
| 直接 `.unwrap()`  everywhere | 粗暴 — 五种模式按场景选 |

### 五种模式（索引）

| # | 模式 | 笔记 |
|:---:|------|------|
| **1** | **`?` 运算符** | [2.3.1](./2.3.1-unwrap-with-try-operator.md) |
| **2** | **函数调用 + Deref/DerefMut** | [2.3.2](./2.3.2-deref-coercion.md) |
| **3** | **闭包 / `map` / `and_then`** | [2.3.3](./2.3.3-closures-for-inner-access.md) |
| **4** | **获取引用 / 裸指针** | [2.3.4](./2.3.4-getting-references.md) |
| **5** | **获取所有权 `take` / `replace`** | [2.3.5](./2.3.5-getting-ownership.md) |

### 选型速查

| 目标 | 优先 |
|------|------|
| 错误传播 | **`?`**（`Try` / `Result` / `Option`） |
| 调用 `T` 的方法 | **Deref**（`Box`、`String`、`MutexGuard`…） |
| 链式变换、无额外绑定 | **`map` / `and_then`** |
| 只要借用、不消费容器 | **`as_ref` / `as_mut` / `as_ptr`** |
| 移出 `T` 且 `T: !Copy` | **`mem::take` / `mem::replace` / `Option::take`** |

**学习重点**（原书）：第 2 章是 **读第 3 章前的过渡** — 解封装是 **日常策略代码** 与 **unsafe 边界** 的交界。
