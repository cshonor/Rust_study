## 一句话

当 **自动解引用不够用** 时，用 **`Option`/`Result` 的 `map`、`filter`、`and_then`** 等 — 在 **仍留在封装内** 的前提下变换内部值，避免手动解包再重包。

---

## 原书要点

### 闭包 / 组合子模式

```rust
let port: Option<u16> = Some("8080".to_string())
    .as_ref()
    .and_then(|s| s.parse().ok());

let doubled = Some(21).map(|x| x * 2);  // Some(42)
```

| API | 作用 |
|-----|------|
| **`map(f)`** | `Some/Ok` 内 **映射** `T → U`，容器不变 |
| **`filter(pred)`** | 条件保留 |
| **`and_then(f)`** | 链式 **flatMap** — `T → Option<U>` / `Result` |
| **`unwrap_or` / `unwrap_or_else`** | 提供默认值（闭包延迟计算） |

### 与 `?` / Deref 的分工

| 场景 | 选用 |
|------|------|
| 函数体内 **早返回错误** | **`?`** |
| 调用 **`T` 的方法** | **Deref** |
| **一行链式** 变换、仍 optional | **`map` / `and_then`** |
| 需要 **`&T` 且不消费** | **`as_ref` + 闭包** |

### 标准库半标准化函数

`Option` / `Result` / `Iterator` 共享 **函数式风格** — 与 [2.4 闭包与 Iterator（补充）](./2.4-closures-iterator-in-stdlib.md) 一脉。
