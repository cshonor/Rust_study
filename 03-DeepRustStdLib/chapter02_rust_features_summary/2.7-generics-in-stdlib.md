# 2.7 泛型编程的标准库规范

> 章索引：[第 2 章](./README.md) · 前：[2.6 错误处理](./2.6-error-handling-in-stdlib.md)

---

## 一句话

标准库是 **泛型风格指南** — `T` 约束、`impl Trait`、关联类型、默认类型参数一起用，在 **不付运行时多态代价** 的前提下复用算法；读完本章可把零散语法 **收成一套读码习惯**。

---

## 常用泛型手法（std 示范）

| 手法 | 例子 | 目的 |
|------|------|------|
| **类型参数 `T`** | `Vec<T>`、`Option<T>` | 同一结构多种元素类型 |
| **约束 `T: Trait`** | `sort_by_key` 要求 `K: Ord` | 编译期验证能力 |
| **`impl Trait` 参数/返回** | `fn collect<B: FromIterator<A>>(self) -> B` | 简洁边界，单态化 |
| **关联类型** | `Iterator::Item` | 每个实现固定一项输出类型 |
| **默认类型参数** | `Result<T, E>` 默认 `E` 各异 | API 简洁 |
| **`where` 子句** | 复杂边界分行写 | 可读性 |

---

## 与零成本抽象的关系

- 泛型函数 **单态化** → 每种 `T` 一份机器码，常可内联。
- **`dyn Trait`** 用于 **运行时多态**（如 `fmt::Display` 擦除、`Box<dyn Error>`）— 付 vtable 成本，std 只在需要类型擦除时用。

→ [RFR Ch02 布局与 DST](../../02-RFR/Chapter-02-Types/04-dst-wide-pointers.md) · [附录 · 设计哲学](./appendix-design-philosophy.md)

## 写库 / 读库规范（对齐 std）

1. **约束尽量紧** — 只要求 `T: Clone`，不要无谓 `T: 'static`。
2. **优先 `impl Trait`** 于返回位置，隐藏长类型（迭代器适配器链）。
3. **用关联类型表达「一个实现一个输出」** — 如 `Iterator::Item`。
4. **`From` / `Into` 做构造弹性** — `String::from("a")`、`Vec::from([1,2])`。
