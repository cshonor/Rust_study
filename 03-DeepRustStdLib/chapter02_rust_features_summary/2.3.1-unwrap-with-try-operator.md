# 2.3.1 使用 `?` 运算符解封装

> 章索引：[第 2 章](./README.md) · 前：[2.3-extracting-from-wrappers](./2.3-extracting-from-wrappers.md) · 后：[2.3.2-deref-coercion](./2.3.2-deref-coercion.md)

## 一句话

对实现 **`Try`**（如 **`Result` / `Option`**）的封装类型，用 **`?`** 在成功时 **解出内部值**，失败时 **早返回** — 替代层层 `match` / `unwrap`。

---

## 原书要点

### `?` 做什么

```rust
fn parse_port(s: &str) -> Result<u16, ParseIntError> {
    let n: u16 = s.parse()?;  // Ok 则解封装为 u16；Err 则 return Err(...)
    Ok(n)
}
```

| 点 | 说明 |
|----|------|
| **解封装** | `?` 从 `Result`/`Option` **取出 Ok/Some 内的 T** |
| **错误传播** | `Err`/`None` 自动 **return**（或 `None` 传播规则依上下文） |
| **`Try` trait** | 统一 `?` 语义的可扩展基础 |

### 适用封装类型

| 类型 | 典型场景 |
|------|----------|
| **`Result<T, E>`** | IO、解析、业务错误 |
| **`Option<T>`** | 可空链（函数返回 `Option` 时） |

### 与暴力解包对比

| | `?` | `unwrap()` |
|---|-----|------------|
| 失败 | 传播错误 | **panic** |
| 策略代码 | ✅ 推荐 | ❌ 生产慎用 |
