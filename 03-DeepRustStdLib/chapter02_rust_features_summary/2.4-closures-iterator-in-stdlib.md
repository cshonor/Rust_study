## 一句话

**`Iterator` trait + 闭包** 撑起了标准库里绝大部分「遍历与变换」— `for`、`map`、`filter`、`collect` 是同一套零成本抽象的不同面孔。

---

## `Iterator` 如何支撑迭代器设计

```text
核心契约（简化）
  type Item;
  fn next(&mut self) -> Option<Self::Item>;

一切适配器（map / filter / zip …）都包一层 Iterator
  → 编译器内联后 ≈ 手写 while 循环
```

| API | 本质 |
|-----|------|
| `for x in v` | 语法糖：`IntoIterator` + 反复 `next()` |
| `iter.map(|x| …)` | `Map<I, F>` 持有子迭代器 + 闭包 `F: FnMut` |
| `collect::<Vec<_>>()` | `FromIterator` 消费迭代器 |
| `Iterator::sum` | 默认方法，基于 `next` 累加 |

---

## 闭包三 trait 与 std

| Trait | 捕获 | std 典型场景 |
|-------|------|----------------|
| **`Fn`** | 共享不可变捕获 | `Iterator::find` 谓词 |
| **`FnMut`** | 可变捕获 | `Iterator::for_each`、排序 `sort_by` |
| **`FnOnce`** | 消耗捕获 | `thread::spawn`、按值 `map` 后进 `collect` 消耗迭代器 |

## 读源码入口

| 位置 | 看什么 |
|------|--------|
| `core::iter::traits::Iterator` | `next` + 默认适配器实现 |
| `alloc::vec::IntoIter` | `Vec` 消费式迭代，move 元素 |
| `std::collections::hash_map::Iter` | 借 `HashMap` 的 `&K`/`&V` |

**量化语境**：行情数组过滤、`map` 成信号、`collect` 进 `Vec` — 热路径优先 **迭代器链** 或 **显式索引**；读 std 是为了确认 **不会隐藏多余分配**（如 `collect` 会分配新容器）。
