# 2.3.5 获取所有权

> 章索引：[第 2 章](./README.md) · 前：[2.3.4-getting-references](./2.3.4-getting-references.md) · 后：[2.4-recap](./2.4-recap.md)

---

## 一句话

当 **`T: !Copy`** 且不能把内部 **`T` 用 `=` 移出** 封装时，用 **`mem::take` / `mem::replace`** 或 **`Option::take`** — 通过 **浅拷贝占位 + 移所有权** 安全掏空容器。

---

## 原书要点

### 问题场景

```rust
// 不行：不能从 &mut Option<String> 里「复制出」String 又保留 Option
// let s = opt.unwrap(); // 消费整个 Option
```

需要：**外层封装仍存在**（或变为空状态），**内层 `T` 归调用者所有**。

### `take` / `replace`

```rust
use std::mem;

let mut slot = Some(String::from("order"));
let owned = slot.take();           // slot → None, owned → Some(String)
let old = mem::replace(&mut slot, Some(String::from("new"))); // 返回旧值
```

| API | 行为 |
|-----|------|
| **`Option::take(&mut self)`** | `self → None`，返回原 `Option<T>` |
| **`mem::take(&mut T)`** | `T: Default` — 用默认值 **替换** 并返回旧值 |
| **`mem::replace(&mut T, new)`** | 放入 `new`，返回被替换掉的旧 **`T`** |

### 「浅拷贝」含义（原书）

`replace`/`take` **不 deep clone `T`** — 是 **移动语义** 交换槽位内容；对 `String`/`Vec` 是 **指针所有权转移**。

### 其它「移出」API（补充）

| API | 类型 |
|-----|------|
| **`Box::into_inner`**（nightly/部分类型） | `Box<T>` → `T` |
| **`Mutex::into_inner`** | 取得锁内 `T`（无 poison 时） |
