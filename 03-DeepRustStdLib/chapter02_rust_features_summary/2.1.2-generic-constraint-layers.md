# 2.1.2 泛型约束的层次

> 章索引：[第 2 章](./README.md) · 前：[2.1.1-generic-functions-and-traits](./2.1.1-generic-functions-and-traits.md) · 后：[2.2-memory-safety-overview](./2.2-memory-safety-overview.md)

---

## 一句话

Rust 泛型约束形成 **从一般到特殊** 的层次：**基层 `?Sized`** → **一级**（固定大小、指针、切片、引用、`T: Trait`、元组）→ **二级**（具体类型或嵌套如 `*const [T]`）。

---

## 原书要点

### 层次结构（自顶向下）

```text
基层：T: ?Sized          ← 最大集合（含 DST：str、[T]、trait object…）
  │
一级子层
  ├─ T                     固定大小类型（默认 T: Sized）
  ├─ *const T / *mut T     裸指针
  ├─ [T]                   切片 DST
  ├─ &T / &mut T           引用
  ├─ T: SomeTrait          额外 trait 约束
  └─ (T, U) / 复合类型     元组等
  │
二级子层
  ├─ *const u8             一级 + 具体类型
  └─ *const [T]            一级 + 嵌套（T 仍可为泛型）
```

### 基层：`T: ?Sized`

| 点 | 说明 |
|----|------|
| **`Sized`** | 编译期已知大小 — 默认泛型 **`T` 隐含 `T: Sized`** |
| **`?Sized`** | **opt-out** — 允许 `str`、`[T]`、`dyn Trait` 等 **DST** |
| **意义** | 写 **`fn foo<T: ?Sized>(x: &T)`** 才能统一接受 sized 与 unsized 借用 |

### 一级子层（常见）

| 形式 | 约束含义 |
|------|----------|
| **`T`** | 固定大小 · 可栈分配 · 可 `memcpy` 式移动 |
| **`*const T` / `*mut T`** | 裸指针 — **unsafe 解引用**（第 3 章） |
| **`[T]`** | 切片 — 长度运行时已知 |
| **`&T` / `&mut T`** | 借用 — 生命周期 + 别名规则 |
| **`T: Trait`** | 能力约束（`Clone`、`Send`…） |

### 二级子层（实例）

| 写法 | 读法 |
|------|------|
| `*const u8` | 字节裸指针 |
| `*const [T]` | 指向 **切片头** 的裸指针（仍含 `T` 泛型） |
| `&'a [u8]` | 一级引用 + 具体元素类型 |

### 与读 `std` 的关系

读 **`Vec<T>`、`Box<T>`、`Iterator<Item=T>`** 时，先问：**`T` 默认 Sized 吗？有没有 `?Sized`？** — 避免对 `str` / trait object 误用。
