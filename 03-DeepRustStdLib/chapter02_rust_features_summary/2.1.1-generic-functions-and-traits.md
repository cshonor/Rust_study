## 一句话

**泛型函数** 与 **trait 上的泛型方法** 是标准库与业务代码的主干 — 用 **`T: Trait`** / **`where`** 表达约束，而不是复制粘贴具体类型版本。

---

## 原书要点

### 泛型函数

```rust
fn max<T: Ord>(a: T, b: T) -> T {
    if a > b { a } else { b }
}
```

| 要素 | 作用 |
|------|------|
| **`<T>`** | 类型参数 |
| **`T: Ord`** | trait bound — 保证可比较 |
| **单态化** | `max::<i32>`、`max::<f64>` 各生成一份机器码 |

### 基于泛型的 Trait

| 形式 | 示例方向 |
|------|----------|
| **泛型 trait** | `Iterator<Item = T>` |
| **trait 上泛型方法** | `impl<T> Vec<T> { pub fn push(&mut self, value: T) }` |
| **关联类型** | 一种类型参数固定为 trait 内别名（第 4～5 章深入） |

### 为何「离开泛型无法写程序」

标准库 API 几乎全是 **`Option<T>`、`Result<T,E>`、`Vec<T>`、`impl Iterator<Item=T>`** — 不理解泛型约束，读 `std` 签名会处处卡住。
