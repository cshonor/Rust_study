# 2.1 Rust 核心特性与 `std` 的结合

> 章索引：[第 2 章](./README.md) · 前：[1.4 读源码](../chapter01_std_overview/1.4-reading-stdlib-source.md) · 后：[2.2 Trait](./2.2-traits-in-stdlib.md)

---

## 一句话

[RFR](../../02-RFR/RFR-本书目录.md) 里学的 **所有权、借用、移动** 不是语法习题 — 它们直接决定 **`Vec::push`、`String::clone`、`HashMap::insert`** 在标准库里的 API 形状与实现方式。

---

## 所有权 → 标准库 API

| 语言规则 | `std` 中的体现 |
|----------|----------------|
| **移动默认** | `Vec<T>` 进函数消耗所有权；`append` / `extend` 拿走对方 `Vec` |
| **Copy 例外** | 小标量、`IpAddr` 等按位复制；大对象（`String`）必须显式 `clone` |
| **单一所有者** | `Box<T>` 独占堆数据；`Vec` 独占缓冲区 |
| **共享读** | `Arc<T>` + 不可变引用；写路径另加 `Mutex` 等 |

```rust
// std 设计：push 取得 T 的所有权，不借 &T 长期悬挂在 vec 里
let mut v = vec![1, 2];
v.push(3); // T moved into vec's buffer
```

读源码：`alloc::vec::Vec::push` — 扩容时 **旧缓冲区整体 drop 或 relocate**，依赖 **唯一所有权** 不变量。

---

## 借用 → 标准库 API

| 借用规则 | `std` 中的体现 |
|----------|----------------|
| **&T 共享读** | `slice::iter`、`HashMap::get` 返回 `Option<&V>` |
| **&mut T 独占写** | `Vec::as_mut_slice`、`get_mut`；`Mutex::lock` → `MutexGuard` |
| **不可同时存在冲突借用** | `RefCell` 运行时检查；违反则 panic |

**与 RFR 衔接**：你在 RFR Ch01 学的「共享不可变、可变独占」，在 `std::collections` 与 `std::sync` 里变成 **具体类型分工** — `&` 遍历 vs `&mut` 修改 vs `Arc<Mutex<_>>` 跨线程。

---

## 移动 + RAII → 资源类型

标准库大量类型 **析构 = 释放资源**：

| 类型 | Drop 时做什么 |
|------|----------------|
| `File` | 关闭 fd |
| `MutexGuard` | 解锁 |
| `Vec` | 释放堆缓冲区 |
| `TcpStream` | 关闭 socket |

## 读源码时的检查清单

1. 这个方法是 **拿走 `self`**、**`&self`** 还是 **`&mut self`**？
2. 返回值是 **owned** 还是 **引用**？引用生命周期绑在谁上？
3. 内部有没有 **`unsafe`** 在「拉长」借用 — 通常伴随 `unsafe` 块与 invariant 注释。
