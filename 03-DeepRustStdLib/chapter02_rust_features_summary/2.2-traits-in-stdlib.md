# 2.2 Trait 系统在标准库中的落地

> 章索引：[第 2 章](./README.md) · 前：[2.1 核心特性](./2.1-core-features-and-std.md) · 后：[2.3 生命周期](./2.3-lifetimes-in-stdlib.md)

## 一句话

标准库是 **trait 的教科书** — `Read`/`Write`、`Iterator`、`Clone`、`Drop`、`Fn` 族把「行为」从具体类型里抽出来，支撑泛型算法与零成本抽象。

---

## 分层：标准 trait 地图

| 类别 | 代表 trait | 作用 |
|------|------------|------|
| **标记** | `Copy`、`Send`、`Sync`、`Sized` | 编译期约束，常无方法 |
| **转换** | `From`/`Into`、`AsRef`、`Borrow` | API 弹性（`String` vs `&str`） |
| **行为** | `Iterator`、`Read`、`Write` | 统一遍历与 I/O |
| **析构** | `Drop` | RAII |
| **比较** | `PartialEq`、`Ord`、`Hash` | 集合与排序 |
| **格式化** | `Display`、`Debug` | 用户输出 vs 调试 |

---

## 在 `std` 源码里的典型模式

### 1. 泛型边界收紧能力

```rust
// 示意：Read 只要求「能读字节流」
pub fn read_to_end<R: Read>(r: &mut R, buf: &mut Vec<u8>) -> io::Result<usize>
```

`R` 可以是 `File`、`TcpStream`、`&[u8]` — **同一算法，多类型复用**。

### 2. 扩展 trait（sealed 或文档约定）

- `std::io::Read` 的 **默认方法** `read_to_end`、`take` 在 trait 定义里给出。
- 部分 trait 用 **sealed** 防止库外 impl 破坏不变量（实现细节因版本而异）。

### 3. `Deref` / `DerefMut` 智能指针转发

`String` → `str`、`Vec<T>` → `[T]`、`MutexGuard` → 内部 `T` — 见 [第 4 章 智能指针（规划）](../README.md#目录)。
