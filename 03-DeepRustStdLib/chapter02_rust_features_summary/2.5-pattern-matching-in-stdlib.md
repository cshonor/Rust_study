# 2.5 模式匹配在 `std` 源码中的常用场景

> 章索引：[第 2 章](./README.md) · 前：[2.4 闭包与 Iterator](./2.4-closures-iterator-in-stdlib.md) · 后：[2.6 错误处理](./2.6-error-handling-in-stdlib.md)

---

## 一句话

标准库源码里 **`match` / `if let` / `while let`** 无处不在 — 用来处理 **`Option`/`Result`、枚举状态机、I/O 返回值**，是读 `libstd` 时第二常见的控制流（第一是 `?`）。

---

## 高频场景

### 1. `Option` / `Result` 分支

```rust
// 典型：next()、read()、get()
match self.inner.next() {
    Some(item) => { ... }
    None => { ... }
}

// 等价简写
while let Some(x) = iter.next() { ... }
if let Ok(n) = parse::<i32>(s) { ... }
```

`Vec`、`HashMap`、`io::Read` 内部大量 **「有值继续 / 无值结束」** 结构。

### 2. 枚举状态机

| 类型 | 匹配什么 |
|------|----------|
| `Poll<T>` | `Ready` vs `Pending`（async 底层） |
| `ControlFlow` | 提前 break 遍历 |
| `Ordering` | `Less` / `Equal` / `Greater` 比较 |
| `IpAddr` | `V4` vs `V6` 分支 |

### 3. `match` 解构 + 绑定

```rust
match entry {
    Occupied(mut o) => { ... o.get_mut() ... }
    Vacant(v) => { v.insert(value); }
}
```

`HashMap::entry` API 把 **「存在 / 不存在」** 变成 **穷尽枚举**，避免 TOCTOU。

### 4. 守卫与 `@` 绑定

源码里偶见 `match x { n if n > 0 => ..., _ => ... }` — 复杂条件仍保持穷尽。

---

## 与 Book / RFR 衔接

- 穷尽匹配保证 **漏分支 = 编译错误** — std 用枚举表达 **协议状态**，减少非法组合。
- **`if let` 单分支** 适合「只关心一种情况」— 如 `if let Some(e) = err.source()` 挖错误链 → [RFR Ch04 错误链](../../02-RFR/Chapter-04-Error-Handling/01-error-source-chain.md)

→ [Book 06 枚举与模式匹配](../../00-Book/06-enums-pattern-matching/)

## 读源码习惯

见到 **`match` on `Result`** → 区分 **可恢复错误**（返回 `Err`）vs **不变量破坏**（`panic` / `unwrap` 仅 invariant 路径）。
