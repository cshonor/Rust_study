# 6.4 字符串类型

> 章索引：[第 6 章](./README.md) · 前：[6.3](./6.3-char-type.md) · 后：[6.5 · 切片](./6.5-slice-type.md)

## 一句话

**`str`** = 变长 **UTF-8 `[u8]`** — 须 **`is_char_boundary`** 防切片 panic；**`find`** 不写死一种算法，而经 **`Pattern` / `Searcher` Trait** 统一多种 needle；子串对子串则 **TwoWay** 双向匹配 + **周期性跳跃**，极低额外内存。

---

## 原书要点

### UTF-8 与边界安全

| 陷阱 | 标准库对策 |
|------|------------|
| **`&s[i..j]` 按字节切** | 切在码元中间 → **panic** |
| **`is_char_boundary(n)`** | 判断索引 `n` 是否为 **合法字符起始字节** |
| **安全 API** | `split_at` · `floor_char_boundary` · **`chars()`** 按字符迭代 |

```rust
assert!(s.is_char_boundary(0));
// s[k..] 前须保证 k 为 char boundary
```

↔ [6.3 · char 与 UTF-8](./6.3-char-type.md)

### Pattern 与 Searcher Trait（高度抽象）

**`str.find(needle)`** 的 `needle` 不必是 `&str`：

| 实现 `Pattern` 的 needle | 示例 |
|--------------------------|------|
| **`&str` / `&[char]`** | 子串 · 字符集 |
| **闭包** | `|c: char| c.is_numeric()` |
| **外部 crate** | 可扩展（原书提及正则等） |

```text
find / match_indices / split
  └── Pattern::into_searcher(haystack) → Searcher
        └── next_match() 驱动查找
```

| Trait | 职责 |
|-------|------|
| **`Pattern`** | 将「模式」转为 **`Searcher`** |
| **`Searcher`** | 在 haystack 上 **逐步** 报告匹配区间 |
| **`DoubleEndedSearcher`** | 反向搜索支持 |

### TwoWay 算法（字符串找字符串 · 极致性能）

最复杂场景：**`&str` needle in `&str` haystack** → **`TwoWaySearcher`**。

| 要点 | 说明 |
|------|------|
| **周期性（Period）** | 模式自重复时 **大步跳过** · 非逐字节 +1 |
| **双向** | 前后缀信息压缩比较 |
| **空间** | **O(1) 额外堆** — 常数级预处理 |

```text
非编译器魔术 ──► Pattern/Searcher 抽象 + TwoWay 实现
```

### HFT

**symbol 前缀/后缀查找** — `&str` 零拷贝 + `find`；避免未校验字节索引切片。
