## 一句话

**`[T]`** 胖指针（ptr + len）承载 **连续元素视图** — 排序、`split_at`、`get` 等是容器章的前奏；插入排序等热路径在 **安全边界内** 用 **`ptr::copy_nonoverlapping`** **直拷内存**，绕过逐元素 move/drop 的开销。

### 深度补充：排序中的内存直拷（原书 · 底层黑魔法）

**小数组/插入排序** 等需 **将元素向前挪动** 时，若用安全 Rust **逐元素 swap/move**：

```text
每次 move → 可能 drop 旧位 + 所有权链  →  指令膨胀、cache 不友好
```

标准库在 **已证明不越界、不重叠** 的前提下调用：

```rust
// library/core/src/ptr/mod.rs
pub unsafe fn copy_nonoverlapping<T>(src: *const T, dst: *mut T, count: usize)
```

| 点 | 说明 |
|----|------|
| **语义** | 等同 **C `memcpy`** — 按位复制 `count` 个 `T` |
| **前提** | **`unsafe`** · src/dst **不重叠** · 目标已分配 |
| **为何快** | **跳过** 逐元素赋值时的 **drop/move 检查** |
| **仍安全对外** | 公开 API（如 `slice::sort`）内封装 unsafe · 对外 **`&mut [T]`** |

```text
安全抽象（sort） ──内部──► copy_nonoverlapping ──► 极致吞吐
         └── 边界证明由 std 作者承担 ──► 用户仍写 safe 代码
```

↔ [3.1 · 裸指针](../chapter03_memory_model/3.1-raw-pointers.md) · [Nomicon](../../04-Rust-Nomicon/README.md)

### 常用 API

| 方法 | 作用 |
|------|------|
| **`sort` / `sort_by`** | 排序（Tim sort + 插入排序混合 · 内部 mem copy） |
| **`binary_search`** | 有序切片 |
| **`split_at` / `chunks`** | 分段视图 |
