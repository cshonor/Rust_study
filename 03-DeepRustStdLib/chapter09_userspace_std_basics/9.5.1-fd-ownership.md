# 9.5.1 文件描述符所有权设计

> 章索引：[第 9 章](./README.md) · 前：[9.5](./9.5-fd-and-handles.md) · 后：[9.5.2 · 逻辑层](./9.5.2-fd-logic-layer.md)

## 一句话

**`OwnedFd` / `File`** 等 **独占** 一个 FD — **`Drop` 恰好 `close` 一次**；**`try_clone` → 新 FD** · **`into_raw_fd` 转移** 后 Rust **不再** 负责关闭 — 用类型系统复刻 **唯一 owner** 语义。

---

## 原书要点

### C/C++ 痛点

| 错误 | 后果 |
|------|------|
| 忘记 **`close(fd)`** | **FD 泄漏** |
| **重复 close** | UB / 误关他人 FD |
| **use-after-close** | 悬垂 fd 号 |

### Rust 设计

```rust
{
    let f = File::open("log.txt")?;  // open → OwnedFd
    // … read/write …
}  // Drop → close 一次
```

| 机制 | 说明 |
|------|------|
| **`OwnedFd`** | 构造即 **取得** FD 所有权 |
| **`Drop`** | **`close`  syscall** |
| **`FromRawFd`（unsafe）** | 外部 FD **接管** — 程序员保证有效 |
| **`IntoRawFd`** | **泄漏**关闭责任给 C |

### 与内存类比

```text
Box<T>  Drop → free          OwnedFd  Drop → close
Rc  不适用于 FD（无 dup 计数在 Rust 层统一）→ 多 owner 用 try_clone 得新 FD
```

### HFT

**socket fd** 生命周期绑定 **`TcpStream` Drop** — 避免热路径异常路径漏关。
