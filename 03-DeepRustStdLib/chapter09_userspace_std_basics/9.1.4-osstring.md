## 一句话

**`OsString` / `OsStr`** 存储 **操作系统原生路径/环境字节** — Windows **宽字符 UTF-16** · Unix **任意字节（不必 UTF-8）** — 避免把 **非 UTF-8 路径** 硬塞进 `String` 造成 **编码灾难**。

---

## 原书要点

| 对比 | `String`/`str` | `OsString`/`OsStr` |
|------|----------------|---------------------|
| 语义 | 应用层 **UTF-8 文本** | **OS 原生** 路径/argv/env |
| Linux | UTF-8 常见但 **非强制** | 保留 **原始字节** |
| Windows | 不适用 | **UTF-16** 宽串 |

```rust
use std::ffi::OsStr;
let path = OsStr::new("/tmp/log");  // 平台编码
std::fs::read(path);                 // open  syscall 前转换
```

### 与 C / SYSCALL 的桥

```text
Rust Path/OsStr  ──►  platform encode  ──►  *const c_char / wchar  ──►  open()
```

| API | 作用 |
|-----|------|
| **`Path` / `PathBuf`** | 基于 **`OsStr`** 的路径语义 |
| **`into_vec` / `from_*`** | 与字节/宽字符缓冲互转 |

### HFT

日志路径、**U 盘/遗留挂载** 上非 UTF-8 文件名 — **`Path`** 而非 `str` 切片。
