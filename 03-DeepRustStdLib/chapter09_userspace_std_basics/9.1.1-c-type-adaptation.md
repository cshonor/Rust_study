# 9.1.1 C 语言的类型适配

> 章索引：[第 9 章](./README.md) · 前：[9.1](./9.1-rust-c-interop.md) · 后：[9.1.2 · va_list](./9.1.2-c-va-list.md)

## 一句话

**`core::ffi`** 提供与 C ABI 对齐的 **`c_int` / `c_long` / `c_void` / `c_char`** 等 — 宽度 **随平台变化**，禁止假设等于 Rust 的 `i32`/`u64`。

---

## 原书要点

| Rust（FFI） | 用途 |
|-------------|------|
| **`c_int` / `c_uint`** | 与 C `int` / `unsigned` 对应 |
| **`c_long` / `c_ulong`** | LP64 vs LLP64 差异 |
| **`c_char`** | 可能是 **`i8` 或 `u8`**（平台） |
| **`*const c_void` / `*mut c_void`** | 不透明指针 |
| **`c_size_t`** | `sizeof` / `malloc` 大小 |

```rust
extern "C" {
    fn open(path: *const c_char, flags: c_int) -> c_int;
}
```

| 规则 | 说明 |
|------|------|
| **`extern "C"`** | 固定 C 调用约定 |
| **不跨 FFI 传 `String`** | 用 **`*const c_char`** + 生命周期契约 |
