## 一句话

**第 9 章起进入 `std` 用户态** — 此前 **core/alloc** 可无 OS；此后一切扎根 **SYSCALL**。**FFI** 是第一站：OS API 多为 **C 定义**，Rust 须先完成 **类型 · 可变参数 · 字符串 · 路径** 的 C 侧适配。

### 为何 FFI 优先

| 事实 | 后果 |
|------|------|
| Linux/Windows **libc / Win32** 为 C ABI | `std` 底层大量 **`extern "C"`** |
| Rust **UTF-8 `str`** ≠ C **NUL 字节串** | 须 **`CString` / `OsString`** 桥接 |
