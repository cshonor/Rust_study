## 一句话

**`Box`/`Vec`** 用的 **`Global` / `GlobalAlloc`** 在 **用户态 `std`** 中落地为 **`System` 分配器** — 封装 **`malloc`/`free`**（或 **Windows Heap** / **mmap 类 OS 接口**），把 [3.7 · 堆分配](../chapter03_memory_model/3.7-heap-alloc.md) 接到真实 OS。

---

## 原书要点

### 从 alloc 到 std

| 层 | 分配器 |
|----|--------|
| **`core` / `alloc`** | **`Global`** trait · 可 **自定义** `#![global_allocator]` |
| **`std` 默认** | **`System`** implements **`GlobalAlloc`** |

```text
Vec::push → RawVec grow → Global::allocate
  └── std 中 Global 默认 = System
        └── libc malloc 或 platform heap
```

### 与内核对比

| 环境 | 堆来源 |
|------|--------|
| **内核 / `no_std`** | 自写 **`GlobalAlloc`** · **无 malloc** |
| **用户态 std** | **OS/C 运行时** 提供堆 |

↔ [8.2 · RawVec](../chapter08_smart_pointers/8.2-rawvec.md) · [1.2 · alloc](../chapter01_std_overview/1.2-alloc-crate.md)

### 自定义

```rust
#[global_allocator]
static A: MyAllocator = MyAllocator;
// HFT：jemalloc/mimalloc 替换 System
```
