# 9.2 代码工程中的一个技巧

## 一句话

**`std` 的工程技巧**：用 **`cfg(unix)` / `cfg(windows)`** 与 **`std::sys` 私有模块** 把 **OS 相关实现** 压到最底层 — 对外 **`std::fs` / `net`** 等 API **稳定**，对内 **按平台换 libc / 直接 syscall**。

---

## 原书要点

### 分层（惯用结构）

```text
pub API（fs · net · process）
  └── sys::unix / sys::windows / sys::hermit …
        └── libc 或 inline asm syscall
```

| 技巧 | 收益 |
|------|------|
| **`#[cfg(...)]` 分文件** | 编译期只编当前 OS |
| **`sys` 不 stable** | 自由改底层 · 用户依赖 **stable `std`** |
| **统一 error 映射** | `io::Error` 包装 errno / Win32 码 |
