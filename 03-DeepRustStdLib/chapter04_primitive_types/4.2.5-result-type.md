# 4.2.5 Result<T, E> 类型

> 章索引：[第 4 章](./README.md) · 前：[4.2.4](./4.2.4-ref-match-syntax.md) · 后：[4.3 · 基本 Trait](./4.3-basic-traits.md)

---

## 一句话

**`Result<T, E>`** 同样是 **`core` 中的 enum**（`Ok(T)` | `Err(E)`）— Rust **错误处理的标准方案**，用 **`?` / `map` / `and_then`** 替代 try/catch，把失败路径 **类型化** 而非异常控制流。

---

## 原书要点

### 定义

```rust
pub enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

| 对比 | C++/Java | Rust |
|------|----------|------|
| 机制 | 异常栈展开 | **返回值** `Result` |
| 遗漏处理 | 可 silent 忽略 | 类型强制考虑 `Err`（或 `unwrap` 显式 panic） |
| 传播 | throw | **`?` → `Try` trait**（[4.3.3](./4.3.3-try-trait.md)） |

### combinator（与 Option 平行）

| 方法 | 作用 |
|------|------|
| `map` / `map_err` | 映射 Ok / Err 侧 |
| `and_then` | 链式 flatMap |
| `unwrap_or_else` | Err 时恢复 |
| `?` | 早退传播 `Err` |

↔ [2.6 · std 中的错误处理](../chapter02_rust_features_summary/2.6-error-handling-in-stdlib.md)
