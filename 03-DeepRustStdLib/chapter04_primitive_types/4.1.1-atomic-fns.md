## 一句话

**`atomic_*` intrinsic** 是多核并发下 **load/store/CAS/fence** 的硬件基石 — `AtomicUsize` 等类型与 `Mutex`/`RwLock` 临界区保护 **最终都依赖** 这些原子原语。

---

## 原书要点

| 能力 | 说明 |
|------|------|
| **原子读写** | 单变量上的不可分割读/写，配合 **Ordering**（Relaxed/Acquire/Release/SeqCst…） |
| **CAS** | compare_exchange — 无锁队列、自旋锁、版本计数 |
| **fence** | 跨核内存序屏障 |
| **与并发原语** | `std::sync::Mutex` 等 **用户态 API** 内部仍可能落到 OS + 原子；无锁路径 **直接** 用 `Atomic*` |

```rust
// 用户态典型用法（非直接 intrinsic）
use std::sync::atomic::{AtomicU64, Ordering};
let seq = AtomicU64::new(0);
seq.fetch_add(1, Ordering::Relaxed);
```

### HFT 关联

| 场景 | 用法 |
|------|------|
| 订单序号 | `AtomicU64` Relaxed 递增 |
| 行情版本 | CAS 更新快照指针 |
| 无锁队列 | `compare_exchange` 入队/出队 |
