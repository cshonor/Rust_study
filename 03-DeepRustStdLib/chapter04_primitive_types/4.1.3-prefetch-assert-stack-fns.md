## 一句话

**分支提示**（`likely`/`unlikely`）、**内存预取**（`prefetch`）、**编译期/运行时断言**（`assert_*`）与 **栈指针查询** 等 intrinsic — 指导编译器优化与 debug 诊断，**热路径** 与 **内核/嵌入式** 场景常见。

---

## 原书要点

### 分支优化

| 函数 | 作用 |
|------|------|
| **`likely(b)`** | 提示 `b` **大概率** 为真 — 影响 **基本块布局**（减少分支误判惩罚） |
| **`unlikely(b)`** | 提示 **小概率** 分支（错误路径、罕见状态） |

```text
if unlikely(err) { cold_path } else { hot_path }  // 编译器倾向把 hot_path 顺序排放
```

### 指令预取

| 项 | 说明 |
|----|------|
| **`prefetch_read/write`** | 提前把地址对应 cache line **拉入 CPU 缓存** |
| 适用 | 顺序扫描大数组、订单簿深度遍历 — **须 profiling**，误用可降性能 |

### 断言类

| 类型 | 说明 |
|------|------|
| **`debug_assert!`** | 仅 debug 构建检查 |
| **`assert_unsafe_precondition!`** 等 | 库内部 unsafe 前置条件（rustc 内置） |

### 栈获取

| 用途 | 说明 |
|------|------|
| 栈指针 intrinsic | 诊断 **栈溢出**、协程/绿色线程栈切换、嵌入式栈水位 |
