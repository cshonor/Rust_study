# 4.2.4 引用类型 match 语法研究

> 章索引：[第 4 章](./README.md) · 前：[4.2.3](./4.2.3-option-type.md) · 后：[4.2.5 · Result](./4.2.5-result-type.md)

## 一句话

对 **`&T` / `&mut T`** 做 **模式匹配** 时，编译器在 **自动解引用（Deref）**、**绑定模式** 与 **生命周期推断** 之间做协调 — 表面语法是「match 引用」，底层仍是 **按值/按引用解构 + 借用规则**。

---

## 原书要点

### 常见形态

| 写法 | 效果 |
|------|------|
| `match &opt { Some(x) => … }` | **`x: &T`** — 不移动内部值 |
| `match &mut v { … }` | 可变借用绑定 |
| `match *r { … }` | 显式解引用（须 `Copy` 或移动） |
| `ref` / `ref mut` 绑定 | 模式内 **强制** 以引用绑定 |

### 底层逻辑

```text
match expr
  ├─ expr 若为引用 ──► 模式默认匹配「被指向的值」（Deref 模式匹配）
  ├─ 绑定变量 ──► 推断 &T 或 T（是否 move）
  └─ 生命周期 ──► 绑定引用不得长于被 match 的借用
```

### 易错点

| 陷阱 | 说明 |
|------|------|
| **move vs borrow** | `Some(x)` 在 `match &opt` 中 **`x` 是引用** |
| **ref 关键字** | `Some(ref x)` 在 match **值** 时仍绑定为引用 |
| **mut 模式** | `&mut` 匹配要求表达式为可变借用 |
