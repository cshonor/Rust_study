## 一句话

**`f32` / `f64`** 遵循 IEEE 754 — 标准库封装数学函数与分类方法；**`NaN` 破坏全序** 导致只能实现 **`PartialOrd`** 而非 **`Ord`**（见 [4.3.2](./4.3.2-arithmetic-traits.md)）。

---

## 原书要点

| 主题 | 说明 |
|------|------|
| **特殊值** | `INFINITY` · `-INFINITY` · `NAN` · `NEG_INFINITY` |
| **分类** | `is_nan()` · `is_finite()` · `is_infinite()` · `is_sign_positive()` |
| **比较陷阱** | `NaN == NaN` 为 **false**；**不可全序** → 排序需 `total_cmp`（Rust 1.62+）或自定义 key |
| **数学** | `sin`/`sqrt`/… — 常经 libm 或 CPU FPU intrinsic |

### 与 HFT

| 建议 | 原因 |
|------|------|
| 价格/数量用 **整数 tick** | 避免浮点累积误差与 NaN 污染 |
| 若必须用 `f64` | 比较用 **epsilon** 或转定点 |
