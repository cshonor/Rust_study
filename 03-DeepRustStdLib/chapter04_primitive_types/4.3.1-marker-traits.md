## 一句话

**Marker Trait**（`Send`、`Sync`、`Copy`、`Sized` 等）**无方法体** — 仅标记类型的 **特殊属性**；**Auto Trait** 由编译器按成员 **自动推导**，是 **跨线程安全** 与 **泛型约束** 的隐形基石。

---

## 原书要点

### 核心 Marker

| Trait | 含义 |
|-------|------|
| **`Send`** | 所有权可 **转移到** 其他线程 |
| **`Sync`** | **`&T` 可安全共享** 于多线程（`T: Sync` ⇔ `&T: Send`） |
| **`Copy`** | 按位复制，**不 move**（与 `Drop` 互斥于常见类型） |
| **`Sized`** | 编译期已知大小；**默认泛型 `T: Sized`** |
| **`Unpin`** | 移动后不 invalidate 自引用（async 相关） |

### Auto Trait 规则（直觉）

```text
若 T 的所有字段都 Send  ──►  T 自动 Send（除非手动 unsafe impl 或 !Send 字段）
若 T 的所有字段都 Sync  ──►  T 自动 Sync
```

| 反例 | 说明 |
|------|------|
| `Rc<T>` | **!Send** — 非原子引用计数 |
| `Cell<T>` | **!Sync** — 内部可变性非线程安全 |
| `*const T` | **!Send / !Sync**（裸指针） |

### HFT

**行情快照 struct** 能否 `Arc` + 跨线程广播 → 首先看字段是否 **`Send + Sync`**。
