# 4.3.3 `?` 运算符 Trait

> 章索引：[第 4 章](./README.md) · 前：[4.3.2](./4.3.2-arithmetic-traits.md) · 后：[4.3.4 · Range](./4.3.4-range-traits.md)

---

## 一句话

**`?` 不是魔法** — 它是 **`Try` Trait 的语法糖**：把 **`Result`/`Option`/`ControlFlow`** 等的 **`Break` 变体** 早退为函数返回值，**`Continue`** 则解包继续执行。

---

## 原书要点

### 展开直觉

```rust
// expr?  大致等价于：
match Try::branch(expr) {
    ControlFlow::Continue(v) => v,
    ControlFlow::Break(e)    => return From::from(e),
}
```

| 组件 | 作用 |
|------|------|
| **`Try`** | 定义 **`from_output`** / **`branch`** — 哪些类型可用 `?` |
| **`ControlFlow<B, C>`** | **`Continue(C)`** 继续 · **`Break(B)`** 早退 |
| **`From`** | 错误类型转换（`?` 在 `Err` 侧） |

### 适用类型

| 类型 | `?` 行为 |
|------|----------|
| **`Result<T,E>`** | `Err` → return `Err`（可 `From` 转换） |
| **`Option<T>`** | `None` → return `None` |
| **`ControlFlow<E,T>`** | 自定义控制流 |

↔ [4.2.5 · Result](./4.2.5-result-type.md) · [4.2.3 · Option](./4.2.3-option-type.md)

### 与 try/catch 对比

```text
异常：隐式栈展开，类型系统不参与
?  ：Err 必须是函数返回类型的一部分 — 失败路径「类型可见」
```
