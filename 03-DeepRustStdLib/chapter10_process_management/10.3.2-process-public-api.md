# 10.3.2 对外接口层

> 章索引：[第 10 章](./README.md) · 前：[10.3.1](./10.3.1-process-os-layer.md) · 后：[第 11 章 · 并发](../chapter11_concurrency/README.md)

---

## 一句话

**`std::process::Command` / `Child` / `ExitStatus`** 是 **stable 安全 API** — 链式配置后 **`spawn`**，管道与 wait 全封装；相对 C **`system()`**，默认 **argv 数组、无 shell**，且 **FD 随 `Child` Drop 回收**。

---

## 原书要点

### 核心类型

| 类型 | 作用 |
|------|------|
| **`Command`** | Builder · **`arg`/`env`/`current_dir`/`stdin/stdout/stderr`** |
| **`Child`** | 运行中子进程 · **`stdin()`/`stdout()`/`stderr()`** 管道端 · **`wait`/`kill`** |
| **`ExitStatus`** | 退出码 · 平台扩展（Unix 信号 / Windows code） |
| **`Output`** | **`output()`** 一次性捕获 stdout+stderr |

### 教材典型示例（管道 + EOF + wait）

```rust
use std::process::{Command, Stdio};
use std::io::{Read, Write};

fn main() {
    let mut child = Command::new("cat")
        .stdin(Stdio::piped())
        .stdout(Stdio::piped())
        .spawn()
        .unwrap();

    let mut stdin = child.stdin.take().unwrap();
    stdin.write_all(b"hello pipe").unwrap();
    drop(stdin); // 关闭写端 → 子进程 stdin EOF

    let mut output = String::new();
    child.stdout.take().unwrap().read_to_string(&mut output).unwrap();
    println!("{}", output);

    let status = child.wait().unwrap();
    println!("exit code: {:?}", status);
}
```

| 步骤 | 对应机制 |
|------|----------|
| **`Stdio::piped()`** | [10.1](./10.1-anonymous-pipe.md) + [10.2](./10.2-redirection.md) |
| **`drop(stdin)`** | 关父侧写端 · 子进程 **read 得 EOF** |
| **`wait()`** | [10.3.1](./10.3.1-process-os-layer.md) **`waitpid` / Wait** |

### 安全设计（相对 C）

| 特性 | 说明 |
|------|------|
| **无 shell 除非显式 `sh -c`** | 少 **命令注入** |
| **FD 所有权** | **`ChildStdout` Drop** · 不泄漏 pipe |
| **`io::Result`** | spawn/wait 失败可传播 |

### 与 10.1～10.2

| 配置 | 底层 |
|------|------|
| **`.stdin(Stdio::piped())`** | `imp::pipe` + 子 **fd 0** |
| **`.stdout(Stdio::from(file))`** | 文件 fd 作 **fd 1** |

### HFT

**spawn 辅助进程**（日志归档、合规导出）— **`current_dir`/`env`** 隔离配置 · **勿** 在 tick 热路径频繁 spawn。
