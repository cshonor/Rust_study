---

## 原书要点

### STD 两大分层（Ch9～13 通用）

| 层 | 模块示例 | 特征 |
|----|----------|------|
| **底层 sys 适配** | `std::sys::process` · `unix/` · `windows/` | **`unsafe`** · 裸 fd/Handle · SYSCALL |
| **上层对外 API** | `std::process` · `Command` · `Child` · `Stdio` | **safe** · RAII · **`io::Result`** |

```text
开发者  Command::spawn()          ← 仅见这一层
  └── std::process（stable）
        └── sys::unix::process / sys::windows::process
              └── fork+exec / CreateProcess + pipe + wait
```

### 关键安全设计（std 相对裸 syscall 的价值）

| 设计 | 效果 |
|------|------|
| **RAII** | `PipeReader`/`Child`/`Stdio` Drop → 自动 close |
| **所有权隔离** | 管道端绑定 **`Child`** · drop 时关对应 fd |
| **unsafe 边界** | 裸 syscall **收敛到 sys** · 上层 API **全 safe** |
| **跨平台** | fork+exec vs CreateProcess **编译期 cfg 分发** · 业务零分支 |

### 与前后章

| 方向 | 关联 |
|------|------|
| **前置 Ch9** | `File` · Read/Write · FD 所有权 — 管道是 **特殊 IO 流** |
| **后置 Ch11** | 线程 · 同进程并发（对比 **独立地址空间**） |
| **后置 Ch12～13** | 文件系统 · 网络/stdio I/O — 同一 **sys + 公开 API** 套路 |
