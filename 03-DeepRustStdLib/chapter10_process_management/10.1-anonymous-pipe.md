# 10.1 匿名管道

> 章索引：[第 10 章](./README.md) · 后：[10.2 · 重定向](./10.2-redirection.md)

---

## 一句话

**匿名管道** = 父子/协作进程 IPC 的 **单向字节流** — OS 提供一对 **读/写 fd（或 Handle）**；`std` 用 **`PipeReader`/`PipeWriter`**（或 `AnonPipe` → `Stdio`）做 **RAII**，是 [10.2](./10.2-redirection.md) 重定向与 [10.3](./10.3-process-mgmt.md) spawn 的 **通信载体**。

### 顶层安全入口 `std::io::pipe()`

```rust
pub fn pipe() -> Result<(PipeReader, PipeWriter)> {
    let (read, write) = imp::pipe()?; // imp = sys 平台适配
    Ok((PipeReader(read), PipeWriter(write)))
}
```

| 要点 | 说明 |
|------|------|
| **RAII** | 两端离开作用域自动 **close**，杜绝 fd/Handle 泄漏 |
| **半双工** | 读端阻塞等数据 · 写端满则阻塞 |
| **EOF** | 全部写端销毁后，读端 **`read` 返回 0** |
| **与 `Command`** | `Stdio::piped()` 内部同样走 **`imp::pipe()`** |

### Unix 底层（`sys/unix/pipe.rs`，`unsafe` 集中）

```rust
pub fn pipe() -> io::Result<(AnonPipe, AnonPipe)> {
    let mut fds = [0; 2];
    unsafe {
        cvt(libc::pipe(fds.as_mut_ptr()))?; // 直接 libc，无 Rust 安全校验
    }
    let read = AnonPipe::from_raw_fd(fds[0]);
    let write = AnonPipe::from_raw_fd(fds[1]);
    Ok((read, write))
}
```

**`AnonPipe` 关键方法**：

| 方法 | 用途 |
|------|------|
| **`read_fd()` / `write_fd()`** | 取出裸 fd，供 spawn 前 **dup2 / 继承** |
| **`into_stdio()`** | 转为 **`Stdio`**，挂到 **`Command.stdin/stdout/stderr`** |

### Windows 差异

| Unix | Windows |
|------|---------|
| **`libc::pipe` → 两个 fd** | **`CreatePipe` → 两个 `RawHandle`** |
| fd 号 + **`dup2`** | **`bInheritHandle`** 控制子进程继承 |

适配层统一为 **`AnonPipe`**，上层 **`pipe()` / `Stdio::piped()`** **零 `cfg` 分支**。
