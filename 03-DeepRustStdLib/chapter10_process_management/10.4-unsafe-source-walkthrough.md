## Unix 分块

### A. 创建匿名管道

```rust
// sys/unix/pipe.rs — 逻辑骨架
pub fn pipe() -> io::Result<(AnonPipe, AnonPipe)> {
    let mut fds = [0i32; 2];
    unsafe {
        // SAFETY: fds 指向合法栈数组；pipe(2) 写入两个有效 fd
        cvt(libc::pipe(fds.as_mut_ptr()))?;
    }
    // from_raw_fd: 接管内核返回的 fd 所有权
    Ok((
        AnonPipe::from_raw_fd(fds[0]),
        AnonPipe::from_raw_fd(fds[1]),
    ))
}
```

### B. spawn 前 stdio → dup2（子进程视角）

```rust
// sys/unix/process — exec 前 fd 表 setup（概念代码）
fn setup_stdio(stdin: &imp::Stdio, stdout: &imp::Stdio, ...) -> io::Result<()> {
    // 若 stdout 为 piped：write_end 已通过 pipe() 创建
    if let Some(fd) = stdout.pipe_write_fd() {
        unsafe {
            // SAFETY: fd 为有效 open fd；1 为 stdout 标准号
            cvt(libc::dup2(fd, libc::STDOUT_FILENO))?;
        }
        // 关闭 dup2 后多余的原始 write fd（保留 fd 1）
    }
    // stdin / stderr 同理 → fd 0 / 2
    Ok(())
}
```

### C. fork + pre_exec + exec

```rust
// 概念：父进程调 fork，子进程分支
match unsafe { libc::fork() } {
    -1 => return Err(io::Error::last_os_error()),
    0 => {
        // ── 子进程：从 fork 返回后仅单线程 ──
        setup_stdio(...)?;           // dup2、关多余 pipe 端
        pre_exec_closures.run()?;    // 用户 pre_exec：禁止锁/分配
        unsafe {
            libc::execvp(prog, argv); // 失败则 _exit
        }
        unreachable!();
    }
    child_pid => {
        // ── 父进程：关闭子侧 pipe 端，保留读端给 ChildStdout ──
        Ok(child_pid)
    }
}
```

| 约束 | 原因 |
|------|------|
| **`pre_exec` 无锁/无堆** | fork 后 **仅 async-signal-safe** 函数安全 |
| **关「错端」pipe** | 否则 **`read` 永不 EOF**（经典 bug） |

### D. wait 回收

```rust
unsafe {
    // SAFETY: child_pid 为本进程 spawn 的有效 pid
    cvt(libc::waitpid(pid, &mut status, 0))?;
}
// status → ExitStatus（退出码 + 是否 signal）
```

## Windows 分块

### A. CreatePipe（无 Unix pipe syscall）

```rust
// sys/windows/pipe.rs — 逻辑骨架
pub fn pipe() -> io::Result<(AnonPipe, AnonPipe)> {
    let mut read = null_mut();
    let mut write = null_mut();
    unsafe {
        // SAFETY: 输出指针有效；SECURITY_ATTRIBUTES 设 bInheritHandle
        cvt(CreatePipe(&mut read, &mut write, &sa, 0))?;
    }
    Ok((
        AnonPipe::from_raw_handle(read),
        AnonPipe::from_raw_handle(write),
    ))
}
```

### B. CreateProcess + 句柄继承

```rust
// 概念：STARTUPINFO 指定 hStdInput/hStdOutput/hStdError
let mut si: STARTUPINFOW = ...;
si.dwFlags |= STARTF_USESTDHANDLES;
si.hStdOutput = stdout_write_handle; // piped 时绑定管道写端
// bInheritHandles = TRUE 时子进程继承已标记可继承的 Handle

unsafe {
    CreateProcessW(
        app, cmd_line, ..., /* bInheritHandles */ TRUE, ..., &mut si, &mut pi,
    )?;
}
// 父进程 CloseHandle 子侧已复制的端，保留读端
WaitForSingleObject(pi.hProcess, INFINITE);
```

| 与 Unix 对照 | |
|--------------|--|
| **无 fork** | 一次 **CreateProcess** 即新进程 |
| **Handle ≠ fd 号** | **`dup2` 等价物** 由 STARTUPINFO 完成 |
| **继承标志** | **`bInheritHandle`** · 适配层内部设置 |
