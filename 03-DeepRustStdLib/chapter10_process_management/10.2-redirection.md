## 一句话

**重定向** = 在 **spawn 前** 安全地 **替换子进程 fd 0/1/2**（stdin/stdout/stderr）— `Command` 通过 **`Stdio::piped` / `inherit` / `null` / 文件 `From<Stdio>`** 配置，底层 **`dup2` 类 syscall** 在 **OS 适配层** 完成，避免 C 式 **泄漏错误 fd**。

---

## 原书要点（目录脉络 · 正文待刷书）

### 标准流重定向

| `Stdio` 变体 | 子进程效果 |
|--------------|------------|
| **`inherit()`** | 沿用父进程 **同 fd 号** |
| **`piped()`** | 新建管道 · 父持一端 · 子接 **0 或 1** |
| **`null()`** | 指向 **`/dev/null`** / `NUL` |
| **文件 / 自定义 fd** | 指定路径或已有 **`OwnedFd`** |

```rust
Command::new("grep")
    .arg("ERROR")
    .stdin(Stdio::piped())
    .stdout(Stdio::inherit())
    .spawn()?;
```

### 原书强调：安全操作 FD

| C 痛点 | Rust |
|--------|------|
| spawn 后忘记关 **多余 pipe 端** | 父进程 **`ChildStdin`/`ChildStdout`** 持有 · **Drop 关闭** |
| **错误 dup** 导致 fd 泄漏 | **`sys` 层集中** · **`io::Result`** |
| 子进程继承 **不该继承的 fd** | 平台策略 + **`pre_exec`（unix）** 等（细节待刷书） |

```text

```
