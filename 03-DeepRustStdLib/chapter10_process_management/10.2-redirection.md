# 10.2 重定向实现分析

> 章索引：[第 10 章](./README.md) · 前：[10.1](./10.1-anonymous-pipe.md) · 后：[10.3 · 进程管理](./10.3-process-mgmt.md)

## 一句话

**重定向** = spawn 前把子进程 **fd 0/1/2** 换成 **继承 / 管道 / 空设备 / 文件** — 上层 **`Stdio`** 枚举配置 **`Command`**，底层 **sys 层 dup2（Unix）或句柄继承（Windows）** 完成绑定；[10.1](./10.1-anonymous-pipe.md) 的管道经此接到 stdout/stdin。

---

## 原书要点

### `Stdio` 上层接口

```rust
pub struct Stdio {
    inner: imp::Stdio, // 平台私有实现
}
impl Stdio {
    pub fn inherit() -> Self;           // 沿用父进程同 fd/句柄
    pub fn null() -> Self;              // /dev/null · NUL
    pub fn piped() -> Self;             // 自动 pipe()，父子通信
    pub fn from_file(file: File) -> Self;
}
```

| 变体 | 子进程效果 |
|------|------------|
| **`inherit()`** | 与父进程 **同 fd 号 / 句柄** |
| **`piped()`** | 新建管道 · 父持一端 · 子接 **0 或 1/2** |
| **`null()`** | 丢弃 IO |
| **文件 / 自定义 fd** | 指定路径或 **`OwnedFd`** |

### `Command.spawn()` 重定向链路（以 `stdout(Stdio::piped())` 为例）

```text
1. 配置阶段：底层 pipe() → 得 read_fd + write_fd
2. spawn 前：适配层 fd/句柄继承
     · 父进程保留 read 端 → 稍后 ChildStdout 读取
     · 子进程仅 write 端 → dup2 到 stdout (fd 1)
3. 子进程 exec/CreateProcess：输出写入管道
4. 父进程：child.stdout.take() → Read trait 读数据
```

```rust
Command::new("grep")
    .arg("ERROR")
    .stdin(Stdio::piped())
    .stdout(Stdio::inherit())
    .spawn()?;
```

### shell 管道 vs Rust

```text
shell:  cmd1 | cmd2
Rust:   cmd1.stdout(piped) + cmd2.stdin(piped from cmd1.stdout)
```

### unsafe 风险点（原书底层重点）

| 机制 | 风险 / 标准库对策 |
|------|-------------------|
| **`FromRawFd` / `FromRawHandle`** | 裸句柄 **不受所有权管控** → trait 本身 **`unsafe`** · 调用者保证有效、不 double-free |
| **Unix `pre_exec` 闭包** | **fork 后、exec 前** · **仅单线程** · 闭包 **禁止持锁 / 堆分配** · std 限制捕获 |
| **Windows 句柄继承** | 须 **`bInheritHandle`** · 适配层内部设置 · 上层不可见 |
| **多余 pipe 端未关** | 父 **`ChildStdin`/`ChildStdout` Drop** · 子进程侧 spawn 逻辑 **关闭未用端** |
