# 7.2.2 Cell<T> 分析

> 章索引：[第 7 章](./README.md) · 前：[7.2.1](./7.2.1-unsafecell.md) · 后：[7.3 · RefCell](./7.3-refcell-overview.md)

---

## 一句话

**`Cell<T>`** 包装 **`UnsafeCell<T>`**，提供 **`get`/`set`/`replace`/`take`** — **全量值替换**，**永不** 返回指向内部的 **`&T`**；最适合 **`Copy`** 类型或 **整包替换** 场景。

---

## 原书要点

| 方法 | 行为 |
|------|------|
| **`get()`** | **`T: Copy`** — 拷贝出当前值 |
| **`set(val)`** | 覆盖内部（通过 unsafe 写 `UnsafeCell`） |
| **`replace`** | 旧值返回 + 新值写入 |
| **`take()`** | 替换为 **`Default`** · 取出旧值 |

### 限制（原书）

| 限制 | 原因 |
|------|------|
| **无 `&T` / `&mut T`** | 避免与外层 **`&Cell`** 形成 **双重别名** |
| **推荐 `T: Copy`** | `get` 依赖拷贝；非 Copy 用 **`replace`/`take`** |
| **!Sync** | 不可 **`Arc<Cell<_>>` 跨线程** |

```rust
let c = Cell::new(0);
fn inc(c: &Cell<i32>) { c.set(c.get() + 1); }  // 仅需 &Cell
```

### HFT

单线程热路径 **flags / 小计数** — 无原子、无锁；**禁止** 多线程共享 `Cell`。
