# 7.2 Cell<T> 类型分析

> 章索引：[第 7 章](./README.md) · 前：[7.1](./7.1-borrow-borrowmut.md) · 后：[7.2.1 · UnsafeCell](./7.2.1-unsafecell.md)

---

## 一句话

**`Cell<T>`** 族建立在 **`UnsafeCell<T>`** 之上 — 在持有 **`&Cell<T>`**（不可变引用）时仍可用 **`set`/`get`** 替换/读取内部值；**无** `&T`/`&mut T` 指向内部，故 **!Sync**、**单线程** 内部可变。

---

## 原书要点

```text
7.2.1 UnsafeCell  ──►  7.2.2 Cell  ──►  7.3 RefCell（要 &T 引用时用）
```

| 对比 | `Cell<T>` | `RefCell<T>` |
|------|-----------|--------------|
| 访问方式 | **值拷贝** `get`/`set` | **`borrow()` → `&T`** |
| 运行期计数 | **无** | **`BorrowFlag`** |
| **`T: Copy`** | 常见 | 非必须 |
| 跨线程 | **禁止**（!Sync） | **禁止** |

### 设计定位（原书）

内部可变性 **不是** 绕过编译器的「补丁」— 是 Rust **有意** 提供的架构空间：在 **编译期证明太难** 的场景（图、回调计数）把检查 **下推** 到 API 设计或运行期。
