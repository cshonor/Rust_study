# 7.2.1 UnsafeCell<T> 分析

> 章索引：[第 7 章](./README.md) · 前：[7.2](./7.2-cell-overview.md) · 后：[7.2.2 · Cell](./7.2.2-cell.md)

---

## 一句话

**`UnsafeCell<T>`** 是 **所有内部可变性的唯一语言内建基石** — 标准库中 **唯一能** 从 **`&self`** 合法导出 **`*mut T`**、**脱离「&T 不可变别名」编译期假设** 的类型；**unsafe · 不暴露给业务**。

---

## 原书要点

### 原理

| 正常 Rust | `UnsafeCell` |
|-----------|--------------|
| `&T` 存在 ⇒ 不可变别名 | **`get()` → `*mut T`** 允许后续 **unsafe 写** |
| 编译器优化假设别名 | **`#[lang = "unsafe_cell"]`** 标记 opt-out |

```rust
// 概念（实际 API 为 unsafe get）
let c = UnsafeCell::new(0);
let ptr = c.get();  // *mut i32 — 即使 c 仅 &UnsafeCell
```

### 上层封装链

```text
UnsafeCell<T>
  ├── Cell<T>      set/get · 无引用泄露
  ├── RefCell<T>   borrow + BorrowFlag
  ├── Mutex<T>     跨线程 + 锁
  └── Atomic*      原子 + 内部 mut
```

↔ [3.1 · unsafe 根源](../chapter03_memory_model/3.1-raw-pointers.md)

### 安全责任

**`UnsafeCell` 本身不保证安全** — `Cell`/`RefCell` 用 **API 约束**（无 `&T`、运行期计数）把 invariant 封回去。
