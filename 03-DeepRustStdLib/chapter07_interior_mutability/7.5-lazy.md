# 7.5 Lazy<T> 类型分析

> 章索引：[第 7 章](./README.md) · 前：[7.4](./7.4-pin-unpin.md) · 后：[第 8 章 · 智能指针](../chapter08_smart_pointers/README.md)

---

## 一句话

**`OnceCell<T>`** 保证 **全局仅初始化一次**；**`Lazy<T>`** 在其上封装 **闭包延迟初始化** — **首次 `Deref`** 时才执行 init，优化 **启动时间** 与 **按需占内存**（符号表、配置、regex 编译等）。

---

## 原书要点

### OnceCell<T>

| 方法 | 行为 |
|------|------|
| **`get()`** | `Option<&T>` — 未 init 则 `None` |
| **`set(val)`** | **仅首次** `Ok` · 再次 `Err` |
| **`get_or_init(f)`** | 原子/线程安全变体见 **`OnceLock`**（std） |

```text
程序生命周期内  ──►  最多一个 T 实例被写入
```

### Lazy<T>

```rust
static CONFIG: Lazy<HashMap<…>> = Lazy::new(|| {
    load_heavy_config()  // 第一次访问才跑
});
fn use_cfg() { CONFIG.get("key"); }
```

| 点 | 说明 |
|----|------|
| **延迟** | 闭包 **第一次解引用** 时运行 |
| **Once** | 底层 **`OnceCell`** — 只 init 一次 |
| **线程** | `std::sync::LazyLock` / `once_cell` crate 历史 |

### HFT

| 场景 | 用法 |
|------|------|
| **合约元数据表** | 首次订阅时加载 |
| **regex / 解析器** | 避免冷启动全量编译 |

↔ 与 **`static mut`** / 手写 init 相比 — **安全且单次**
