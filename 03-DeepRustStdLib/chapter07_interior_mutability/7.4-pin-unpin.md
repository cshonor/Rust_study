# 7.4 Pin<T> / Unpin<T> 类型分析

> 章索引：[第 7 章](./README.md) · 前：[7.3.3](./7.3.3-refcell-other-fns.md) · 后：[7.5 · Lazy](./7.5-lazy.md)

---

## 一句话

**`Pin<P>`** **钉住** 堆/栈上值 **禁止安全移动** — 解决 **自引用结构**（成员指针指向同结构体内存）在 **搬家后变悬垂** 的问题；**`Unpin`** 是 **Auto Marker**：**无自引用** 的普通类型 **可自由 move**，不受 Pin 约束。

---

## 原书要点

### 痛点：自引用

```text
struct State {
  buf: [u8; N],
  ptr: *const u8,   // 指向 buf 内部
}
// mem::swap / move 后 ptr 仍指旧地址 → 悬垂
```

| 场景 | 需要 Pin |
|------|----------|
| **async 状态机** | 编译器生成的 **自引用 Future** |
| ** intrusive 链表节点** | 指针嵌在结构体内 |
| **C 互操作** | 固定地址回调 user_data |

### Pin 语义

| 点 | 说明 |
|----|------|
| **`Pin<Ptr<T>>`** | 对 **`T`** 施加 **「不可 unpinned move」**  invariant |
| **堆 Pin** | `Box::pin(t)` — 常见 |
| **`Unpin`** | **默认** 所有类型 **除含 !Unpin 字段** — 可 **`Pin::get_unchecked_mut`** 等 |
| **!Unpin** | 显式 **`PhantomPinned`** 或编译器生成 |

```text
Pin  ≠  不可变
Pin  =  地址稳定（仍可通过 Pin + unsafe 或 Pin<&mut T> 改内容）
```

↔ [4.3.1 · Marker · Unpin](../chapter04_primitive_types/4.3.1-marker-traits.md)

### 与内部可变性

Pin 解决 **位置**；Cell/RefCell 解决 **&self 下改值** — **async** 常组合 **`Pin<&mut Self>`**。
