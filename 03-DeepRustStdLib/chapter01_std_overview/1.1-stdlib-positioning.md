# 1.1 Rust 标准库定位：`core` · `alloc` · `std`

> 章索引：[第 1 章](./README.md) · 后：[1.2 核心模块概览](./1.2-core-modules-overview.md)

---

## 一句话

**`std` 不是从零造轮子** — 它在 **`core`（语言内核）** 与 **`alloc`（堆分配）** 之上，再封装 **与操作系统相关的抽象**（文件、网络、线程、进程环境等），形成应用开发默认入口。

---

## 三层关系

```text
                    ┌─────────────────────────────────────┐
                    │  std（标准库 · 应用默认 use std）      │
                    │  fs / net / thread / env / process … │
                    └──────────────────┬──────────────────┘
                                       │ 复用 + OS 封装
                    ┌──────────────────▼──────────────────┐
                    │  alloc（需全局分配器 · 堆类型）         │
                    │  Box / Vec / String / Arc …          │
                    └──────────────────┬──────────────────┘
                                       │ 无堆也可编译
                    ┌──────────────────▼──────────────────┐
                    │  core（#![no_std] 最小子集）            │
                    │  Option / Result / slice / mem / …   │
                    └─────────────────────────────────────┘
```

| 层 | crate | 典型能力 | 能否单独 `#![no_std]` |
|----|-------|----------|----------------------|
| **core** | `core` | 语言原语、切片、迭代器、部分 `fmt` | ✅ 嵌入式 / 内核常用 |
| **alloc** | `alloc` | `Vec`、`String`、`Box` — **要堆** | ✅ 需自备 `GlobalAlloc` |
| **std** | `std` | 在 alloc 上 + **OS**：I/O、线程、时间、随机… | ❌ 隐含 OS 与默认分配器 |

---

## `std` 在 `core` 上多做了什么

1. **重导出 `core` + `alloc`** — 日常 `use std::…` 即可，不必分清 crate 边界。
2. **操作系统抽象** — `std::fs`、`std::net`、`std::thread`、`std::process` 等把 POSIX/Windows API 包成统一 Rust 接口。
3. **运行时 glue** — 默认 panic 钩子、栈展开、部分启动代码（`rt`）与平台 libc 协作。
4. **为后面拆源码铺垫** — 读 `libstd` 时常见模式：**对外 Safe API** 在 `std`，**可 `no_std` 复用的算法** 在 `core`，**堆容器实现** 在 `alloc`。

---

## 量化 / 实盘语境

| 场景 | 常用层 |
|------|--------|
| 普通策略进程、行情服务 | **`std`** 即可 |
| 嵌入式网关、自定义分配器 | **`no_std` + `alloc`** 或纯 **`core`** |
| 读 `Vec`/`String` 源码理解性能 | 从 **`alloc`** 与 **`core::slice`** 对照看 |
