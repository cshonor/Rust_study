# 1.3 `std` 库

> 章索引：[第 1 章](./README.md) · 前：[1.2 `alloc` 库](./1.2-alloc-crate.md) · 后：[第 2 章 Rust 特性小结](../chapter02_rust_features_summary/README.md)

---

## 一句话

**`std`** 是应用开发的 **默认入口** — 在 **`core` + `alloc`** 之上封装 **操作系统相关能力**（文件、网络、线程、进程、环境），并提供运行时 glue（panic、启动代码等）。

---

## 在三层架构中的位置

```text
                    ┌─────────────────────────────────────┐
                    │  std ← 本节                           │
                    │  fs / net / thread / env / process … │
                    └──────────────────┬──────────────────┘
                                       │ 复用 alloc + core
                    ┌──────────────────▼──────────────────┐
                    │  alloc                                │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  core                                 │
                    └─────────────────────────────────────┘
```

| 属性 | 说明 |
|------|------|
| **能否单独 `no_std`** | ❌ 隐含 OS 与默认全局分配器 |
| **日常用法** | `use std::…` — 自动获得三层能力 |
| **源码路径** | `library/std/src/`（`lib.rs` 为入口） |

---

## 相对 `alloc` 多做了什么

1. **重导出 `core` + `alloc`** — 不必分清 crate 边界即可写业务代码。
2. **操作系统抽象** — `fs`、`net`、`thread`、`process`、`env` 等统一跨平台接口。
3. **运行时 glue** — 默认 panic 钩子、栈展开、`rt` 启动与 libc / 平台 API 协作。
4. **平台扩展** — `std::os::unix` / `windows` 等在稳定 API 之外的系统能力。

## 模块地图（`std` 层职责）

```text
std
├── 数据与集合 ── collections / vec / string（多来自 alloc 重导出）
├── 内存与指针 ── boxed / rc / sync::Arc / cell / pin
├── 并发 ──────── thread / sync (Mutex, RwLock, mpsc) / atomic
├── I/O ───────── io / fs / net / process / env
├── 时间与随机 ── time / thread_rng (feature)
├── FFI ────────── ffi / os::raw
├── 错误 ──────── error（Result 在 core）
└── 宏与底层 ──── format / dbg / …
```

| 领域 | 主要模块 / 类型 | 边界 |
|------|-----------------|------|
| **并发** | `thread`、`sync`、`atomic` | OS 线程 + 同步原语；**≠** async 运行时（见 `tokio`） |
| **I/O** | `Read`/`Write`、`fs`、`net` | 阻塞式 OS I/O；async 在生态 crate |
| **FFI** | `std::ffi`、`CString`、`OsString` | 与 C / OS 编码交互；**unsafe** 在调用方 |
| **错误** | `io::Error`、`Error` trait | 可恢复失败传播 → [RFR Ch04](../../02-RFR/Chapter-04-Error-Handling/README.md) |

## 三层对照（读完第 1 章）

| 层 | 问自己 |
|----|--------|
| **core** | 这段逻辑能否 **无堆、无 OS** 复用？ |
| **alloc** | 是不是 **堆所有权 / 容器** 问题？ |
| **std** | 是不是 **线程、文件、网络、进程**？ |

## 量化 / 实盘语境

| 场景 | 常用层 |
|------|--------|
| 普通策略进程、行情服务 | **`std`** 即可 |
| 嵌入式网关、自定义分配器 | **`no_std` + `alloc`** 或纯 **`core`** |
| 读 `Vec`/`String` 性能 | 从 **`alloc`** 与 **`core::slice`** 对照，经 `std` 入口跳转 |
