# 1.3 `std` 库（用户态标准库）

> 章索引：[第 1 章](./README.md) · 前：[1.2 `alloc` 库](./1.2-alloc-crate.md) · 后：[1.4 回顾](./1.4-recap.md)

---

## 一句话

**`std`** 建立在 **操作系统 SYSCALL** 之上 — **仅适用于用户态**；在 **`core` + `alloc`** 之上整合重导出，并针对 **OS 资源** 提供线程、文件、标准 I/O、网络等 API。

---

## 原书定位

| 层 | 运行环境 |
|----|----------|
| **`core` / `alloc`** | 内核态 ✅ · 用户态 ✅ · **可脱离 OS 独立运行** |
| **`std`** | **用户态 only** — 必须依赖 OS 系统调用 |

---

## 在三层架构中的位置

```text
                    ┌─────────────────────────────────────┐
                    │  std ← 本节                           │
                    │  SYSCALL · 线程 · 文件 · 网络 · stdio  │
                    └──────────────────┬──────────────────┘
                                       │ 映射整合 core + alloc
                    ┌──────────────────▼──────────────────┐
                    │  alloc                                │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  core                                 │
                    └─────────────────────────────────────┘
```

| 属性 | 说明 |
|------|------|
| **默认入口** | 普通 `cargo` 二进制隐式链接 **`std`** |
| **`no_std`** | ❌ 不能同时使用完整 `std` |
| **源码** | `library/std/src/` |

---

## 核心内容（原书 §1.3）

### 1. 对 `core` 与 `alloc` 的映射整合

- 业务代码通常 **`use std::…`** — 不必区分 crate。
- **`std::vec::Vec`** 等即 **`alloc`** 类型的再导出 + 平台 glue。
- 读性能/实现时仍要 **下钻 `alloc` / `core`**。

### 2. 针对 OS 资源的类型、Trait 与函数

| 领域 | 原书要点 | 主要模块 |
|------|----------|----------|
| **线程管理** | 临界区 / **互斥锁**、**消息通信** | `thread`、`sync`（`Mutex`、`RwLock`、**`mpsc`**） |
| **文件与目录** | 路径、读写、metadata | `fs`、`path` |
| **标准输入输出** | stdin / stdout / stderr | `io`（`Read` / `Write`） |
| **网络通信** | TCP/UDP 等 | `net` |

→ 本仓库延伸：[第 9～13 章](../chapter09_userspace_std_basics/README.md) · [05-atomic](../../05-Async-Concurrency-Network/01-atomic/README-学习区.md) · [05-network stage03/07](../../05-Async-Concurrency-Network/03-rust_network_programming/README.md)

### 3. 相对 `alloc` 额外能力（归纳）

1. **操作系统抽象** — 统一跨平台 SYSCALL 封装（内部 `sys` 模块）。
2. **运行时 glue** — panic、栈展开、`rt` 启动、默认 **`GlobalAlloc`**。
3. **平台扩展** — `std::os::unix` / `windows` 等。

---

## 模块地图

```text
std
├── 重导出 ──────── core / alloc 类型（Vec / String / Arc …）
├── 并发 ─────────── thread / sync / atomic
├── I/O ──────────── io / fs / net / process / env
├── 错误 ─────────── io::Error · Error trait（Result 在 core）
└── FFI ──────────── ffi · os::raw
```

## 量化 / 实盘语境

| 场景 | 层 |
|------|-----|
| 行情服务、策略进程、日志、落盘 | **`std`** |
| 内核模块、裸机、自定义 RTOS | **`core` ± `alloc`** |
| 读容器性能 | 从 **`alloc`** 跳，不要只停 `std::vec` 文档 |
