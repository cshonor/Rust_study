# 1.2 `alloc` 库

> 章索引：[第 1 章](./README.md) · 前：[1.1 `core` 库](./1.1-core-crate.md) · 后：[1.3 `std` 库](./1.3-std-crate.md)

---

## 一句话

**`alloc`** 在 **`core`** 之上增加 **堆分配能力** — 提供 `Box`、`Vec`、`String`、引用计数指针与基于堆的集合，但 **仍不绑定具体操作系统**。

---

## 在三层架构中的位置

```text
                    ┌─────────────────────────────────────┐
                    │  std（OS 抽象 + 重导出）                │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  alloc ← 本节                         │
                    │  Box / Vec / String / Arc / BTree …  │
                    └──────────────────┬──────────────────┘
                                       │ 依赖 core
                    ┌──────────────────▼──────────────────┐
                    │  core                                 │
                    └─────────────────────────────────────┘
```

| 属性 | 说明 |
|------|------|
| **能否单独用** | ✅ `#![no_std]` + **自备 `GlobalAlloc` 实现** |
| **堆** | ✅ 所有「拥有堆缓冲」的标准容器 |
| **OS** | ❌ 仍无 `fs` / `net` / `thread` |
| **源码路径** | `library/alloc/src/` |

---

## 核心模块（按职责）

| 领域 | 主要模块 / 类型 | 边界 |
|------|-----------------|------|
| **独占堆所有权** | `boxed::Box` | 单所有者堆分配；常与 trait 对象、`Pin` 配合 |
| **动态数组与字符串** | `vec::Vec`、`string::String` | 连续堆缓冲；增长策略在 `alloc::raw_vec` |
| **共享所有权** | `rc::Rc`、`sync::Arc` | 单线程 / 多线程引用计数（`Arc` 的 OS 无关部分在 alloc） |
| **集合** | `collections`（`BTreeMap`、`BinaryHeap`、`LinkedList`…） | 堆上结构；**不**替代数据库索引 |
| **分配器接口** | `alloc::GlobalAlloc`、`Layout` | 自定义分配器、jemalloc 等对接点 |

```text
alloc（节选）
├── boxed ──────────── Box<T>
├── vec / raw_vec ──── Vec 与容量管理
├── string ─────────── String
├── rc / sync ──────── Rc / Arc（计数与弱引用）
└── collections ────── BTree / LinkedList / …
```

---

## 与 `core` 的分工

| 问题 | 在哪一层 |
|------|----------|
| `slice::iter`、`Iterator` trait | **`core`** |
| `Vec` 的 `push`、扩容、`drop` 遍历 | **`alloc`**（算法常调用 `core::ptr` / `slice`） |
| `HashMap` 默认实现 | 主要在 **`std`** 重导出路径，底层与 `hashbrown` / 分配器协作 |
| 读 `Vec` 源码 | **`alloc::vec`** + **`core::slice`** 对照看 |

---

## `no_std` + `alloc` 典型场景

嵌入式或内核态已有 **静态池 / 伙伴分配器** 时，实现 `GlobalAlloc` 后即可使用 `Vec` / `String`，无需完整 `std`。

```rust
// 概念示意：no_std crate 需声明全局分配器
// #![no_std]
// extern crate alloc;
// use alloc::vec::Vec;
```

## 量化 / 实盘语境

| 场景 | 与 `alloc` 的关系 |
|------|-------------------|
| 行情缓冲、`Vec<Quote>` | 性能热点常在 **扩容与 drop** — 读 `alloc` 而非只看 `std::vec` |
| 自定义 arena | 可绕过默认 `GlobalAlloc`，但类型仍常是 `alloc` 里的 `Vec` / `Box` |
| 跨线程共享行情快照 | `Arc` 主体在 **`alloc::sync`**，`std` 再导出并接 poison / 平台细节 |
