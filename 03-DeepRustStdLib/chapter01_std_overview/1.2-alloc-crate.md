# 1.2 `alloc` 库（智能指针与集合库）

> 章索引：[第 1 章](./README.md) · 前：[1.1 `core` 库](./1.1-core-crate.md) · 后：[1.3 `std` 库](./1.3-std-crate.md)

---

## 一句话

**`alloc`** 中 **所有类型都基于堆内存**，**仅依赖 `core`** — 同样可用于 **OS 内核** 与 **用户态**；提供 **`Allocator` / `Global`**、智能指针与 `Vec` / 集合等动态结构。

---

## 原书定位

在已有 **`core`** 的前提下，只需实现 **全局分配器**（`GlobalAlloc`），即可在内核态或嵌入式中使用 **`Box` / `Vec` / `String`**，而仍不必链接完整 **`std`**（无 SYSCALL）。

---

## 在三层架构中的位置

```text
                    ┌─────────────────────────────────────┐
                    │  std（用户态 · 重导出 + OS）            │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  alloc ← 本节（堆 · 仅依赖 core）       │
                    │  内核态 ✅（自备分配器）  用户态 ✅       │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  core                                 │
                    └─────────────────────────────────────┘
```

| 属性 | 说明 |
|------|------|
| **依赖** | **仅 `core`** |
| **堆** | ✅ 全部类型基于堆 |
| **OS** | ❌ 仍无 `fs` / `net` / `thread` |
| **源码** | `library/alloc/src/` |

## 核心内容（原书 §1.2）

### 1. 内存申请与释放

| 项 | 说明 |
|----|------|
| **`Allocator` trait** | 分配器抽象（Rust 较新版本分配器 API 体系） |
| **`Global` 单元类型** | 默认全局分配入口的实现者之一 |
| **对接** | 用户态 `std` 默认 **`GlobalAlloc`**；`no_std` 需 **`#[global_allocator]`** 自备 |

→ [Nomicon 08 实现 Vec / 自定义分配器](../../04-Rust-Nomicon/08_Impl_Vec_Arc/README.md)

### 2. 智能指针体系

| 类型 | 用途 |
|------|------|
| **`Box<T>`** | 独占堆所有权 |
| **`Rc<T>`** | 单线程共享所有权 |
| **`Arc<T>`** | **并发安全** 引用计数（原子计数；与 `std::sync::Arc` 关系见源码） |
| **`String`** | 堆上 UTF-8 字符串 |

→ 第 8 章 [智能指针](../chapter08_smart_pointers/README.md)

### 3. 动态数据结构

| 类型 | 说明 |
|------|------|
| **`RawVec<T>`** | `Vec` 底层容量管理（未稳定公开 API，读源码关键） |
| **`Vec<T>`** | 动态数组 |
| **`LinkedList<T>`** | 链表 |
| **`VecDeque<T>`** | 双端队列（原书「VecQueue」即此类） |
| **`BTreeSet<T>` / `BTreeMap<K, V>`** | 有序集合 / 映射 |

```text
alloc（原书地图）
├── alloc / GlobalAlloc / Allocator ── 分配接口
├── boxed ── Box
├── rc / sync ── Rc / Arc
├── string ── String
├── vec / raw_vec ── Vec / RawVec
└── collections ── LinkedList / VecDeque / BTree*
```

## 与 `core` 的分工

| 问题 | 层 |
|------|-----|
| `Iterator` trait、`slice` 算法 | **`core`** |
| `Vec::push`、扩容、`RawVec` | **`alloc`** |
| `HashMap` 默认路径 | 多经 **`std`** 重导出 + `hashbrown` |
| **`unsafe` 集中** | 分配与指针写 **`alloc` + `core::ptr`** |
