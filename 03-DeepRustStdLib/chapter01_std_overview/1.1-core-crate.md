# 1.1 `core` 库

> 章索引：[第 1 章](./README.md) · 后：[1.2 `alloc` 库](./1.2-alloc-crate.md)

---

## 一句话

**`core`** 是 Rust 标准库的 **最底层 crate** — 不依赖堆分配器、不依赖操作系统，承载语言原语与可在 **`#![no_std]`** 环境复用的算法与类型。

---

## 在三层架构中的位置

```text
                    ┌─────────────────────────────────────┐
                    │  std（应用默认入口）                    │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  alloc（堆类型 · 需 GlobalAlloc）       │
                    └──────────────────┬──────────────────┘
                                       │ 无堆也可编译
                    ┌──────────────────▼──────────────────┐
                    │  core ← 本节                          │
                    │  Option / Result / slice / iter …   │
                    └─────────────────────────────────────┘
```

| 属性 | 说明 |
|------|------|
| **能否单独用** | ✅ `#![no_std]` + `extern crate core`（通常自动链接） |
| **堆** | ❌ 不提供 `Vec` / `String` / `Box` |
| **OS** | ❌ 无文件、网络、线程 API |
| **源码路径** | `library/core/src/`（`rust-lang/rust` 仓库） |

---

## 核心模块（按职责）

| 领域 | 主要模块 / 类型 | 边界 |
|------|-----------------|------|
| **代数类型** | `option`、`result` | `Option<T>` / `Result<T, E>` — 错误与可选值的根基 |
| **切片与数组** | `slice`、`array`、`str`（`str` 为 DST） | 视图与定长数组；**不**拥有堆缓冲 |
| **迭代** | `iter`（`Iterator`、`IntoIterator`） | 零成本迭代抽象；具体容器在 `alloc` |
| **内存原语** | `mem`、`ptr`、`alloc::Layout`（布局在 core） | `size_of`、`align_of`、裸指针运算的类型约束 |
| **格式化** | `fmt`（`Display`、`Debug`、`Write` trait） | 无堆 `fmt` 也可工作（栈上缓冲） |
| **并发标记** | `marker`（`Send` / `Sync` / `PhantomData`） | 纯类型层契约，不提供线程 API |
| **固定与内部可变性** | `pin`、`cell`（`Cell` / `RefCell` 部分在 core） | 自引用与单线程内部可变 |
| **比较与运算** | `cmp`、`ops`、`hash`（trait 定义） | trait 在 core；`HashMap` 容器在 `alloc` |

```text
core（节选）
├── option / result ── 和与积类型
├── slice / str ────── 借用视图
├── iter ───────────── Iterator 生态
├── mem / ptr / pin ── 内存与自引用
├── fmt ────────────── 格式化 trait
└── marker ─────────── Send / Sync / PhantomData
```

---

## `core` 不做什么

- **不分配堆** — 动态 `Vec`、引用计数指针在 **`alloc`**。
- **不碰 OS** — 文件、套接字、进程在 **`std`**（内部再调 libc / 系统 API）。
- **不等于「只能嵌入式」** — 普通 `std` 程序每天都在用 `core` 里的类型，只是通过 `std::` 重导出。

## 量化 / 实盘语境

| 场景 | 与 `core` 的关系 |
|------|------------------|
| 固件、网关 MCU | 往往 **仅 `core`** 或 `core` + 自备 `alloc` |
| 普通策略进程 | 经 `std` 重导出间接使用；读 `Vec` 算法时常要 **下钻到 `core::slice`** |
| 热路径优化 | 关注 `core::iter` 适配器是否被内联、是否无额外分配 |
