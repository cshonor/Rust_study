# 1.1 `core` 库（语言核心库）

> 章索引：[第 1 章](./README.md) · 后：[1.2 `alloc` 库](./1.2-alloc-crate.md)

---

## 一句话

**`core`** 是与 **CPU 架构无关的可移植库** — 同时服务于 **OS 内核** 与 **用户态**；承载 intrinsics、基本 trait/类型、**`alloc`/`mem`/`ptr` 内存模块** 及 `Option`/`Result` 等语言根基，**不依赖堆与操作系统**。

---

## 原书定位

Rust 被设计为可编写 **OS 内核** 的系统级语言；标准库最底层 **`core`** 必须能在 **无 OS、无堆** 环境独立运行。日常 `std` 程序通过重导出使用同一套类型，但读源码、写 `no_std`、做 HFT 热路径优化时，要会 **下钻到 `core`**。

---

## 在三层架构中的位置

```text
                    ┌─────────────────────────────────────┐
                    │  std（用户态 · SYSCALL）               │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  alloc（堆 · 仅依赖 core）              │
                    └──────────────────┬──────────────────┘
                                       │
                    ┌──────────────────▼──────────────────┐
                    │  core ← 本节                          │
                    │  内核态 ✅  用户态 ✅  无 OS ✅         │
                    └─────────────────────────────────────┘
```

| 属性 | 说明 |
|------|------|
| **硬件** | 与具体 CPU 架构解耦的可移植抽象（intrinsic 由编译器按目标 codegen） |
| **堆** | ❌ 无 `Vec` / `String` / `Box` |
| **OS** | ❌ 无文件、网络、线程 |
| **源码** | `library/core/src/` |

## 核心内容（原书 §1.1）

### 1. 编译器内置（intrinsic）函数

| 类别 | 示例方向 |
|------|----------|
| **内存操作** | `copy_nonoverlapping`、`write_bytes` 等 — 常 **`unsafe`** |
| **数学** | 平台相关优化、浮点/整数原语 |
| **原子** | `atomic_*` 系列 — 与 CPU 内存序硬件相关 |

→ 由 **编译器实现**，非普通 Rust 函数；读 `core` 时会频繁见到 `intrinsics` 模块。

### 2. 基本特征（Trait）

| 类别 | 代表 |
|------|------|
| **运算符 `ops`** | `Add`、`Sub`、`BitAnd` … |
| **Marker** | `Send`、`Sync`、`Sized`、`Copy` … |
| **迭代器 `iter`** | `Iterator`、`IntoIterator`、`ExactSizeIterator` … |
| **类型转换** | `From` / `Into`、`AsRef` / `AsMut`、`TryFrom` … |

Trait **定义在 `core`**；具体容器实现多在 **`alloc` / `std`**。

### 3. 基本数据类型与内嵌类型

| 类型 | 说明 |
|------|------|
| 整数 / 浮点 / `bool` / `char` | 语言内置 |
| **`Option<T>` / `Result<T, E>`** | 视为 Rust 语法不可或缺的 **代数类型** — 在 `core` 定义 |

### 4. 内存操作模块（原书重点）

| 模块 | 职责 |
|------|------|
| **`core::alloc`** | **`Layout`** 等 — 布局与分配 **契约**（非堆分配器本身） |
| **`core::mem`** | `size_of`、`align_of`、`replace`、`forget` … |
| **`core::ptr`** | 裸指针 `*const T` / `*mut T` 的安全抽象边界 |

> 原书：**Rust 中约 90% 的 `unsafe` 语法都可归结到 `alloc`、`mem`、`ptr` 三个模块**（此处 `alloc` 指 **`core::alloc` 布局模块**，与 crate **`alloc`** 不同名同质不同层 — 读源码勿混）。

→ 第 3 章 [内存操作](../chapter03_memory_model/README.md) 会展开。

### 5. 其他基础结构

| 领域 | 内容 |
|------|------|
| **数组 / 切片 / Range** | `[T; N]`、`[T]`、`Range`/`RangeInclusive` |
| **字符串与格式化** | `str`（DST）、`fmt`（`Display`、`Debug`、`Write`） |
| **内部可变性** | `UnsafeCell<T>`、`Cell<T>`、`RefCell<T>`（单线程） — 第 7 章专章 |

```text
core（原书地图）
├── intrinsics ─────── 编译器内置
├── ops / marker / iter / convert ── Trait 根基
├── option / result ── 和与积类型
├── mem / ptr / alloc::Layout ── unsafe 主战场
├── slice / array / str / range
├── fmt ────────────── 格式化
└── cell / pin ─────── 内部可变与自引用
```

## `core` 不做什么

- **不分配堆** — 动态容器在 **`alloc` crate**。
- **不碰 SYSCALL** — 线程/文件/网络在 **`std`**。
- **不等于「只能嵌入式」** — 普通程序每天都在用 `core` 类型，只是经 `std::` 重导出。

## 量化 / 实盘语境

| 场景 | 与 `core` 的关系 |
|------|------------------|
| 固件 / 网关 MCU | 常 **仅 `core`** 或 + 自备 `alloc` |
