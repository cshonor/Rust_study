## 一句话

快速建立 **标准库全局地图** — 弄清集合、并发、I/O、FFI 等子系统的 **职责边界**，后面按模块拆源码时不迷路。

---

## 模块地图（按职责）

```text
std
├── 数据与集合 ── collections / vec / string / btree / hash
├── 内存与指针 ── boxed / rc / sync::Arc / cell / pin
├── 并发 ──────── thread / sync (Mutex, RwLock, mpsc) / atomic (re-export)
├── I/O ───────── io / fs / net / process / env
├── 时间与随机 ── time / thread_rng (feature)
├── FFI ────────── ffi / os::raw
├── 错误 ──────── error / result (core) / panic
└── 宏与底层 ──── format / dbg / concat / …
```

---

## 核心模块边界速查

| 领域 | 主要模块 / 类型 | 边界（干什么 / 不干什么） |
|------|-----------------|---------------------------|
| **集合** | `Vec`、`String`、`HashMap`、`BTreeMap` | 堆上动态结构；**不**替代数据库 / 磁盘索引 |
| **智能指针** | `Box`、`Rc`、`Arc`、`RefCell`、`Mutex` | 所有权与共享模型；**不**解决业务层缓存策略 |
| **并发** | `thread`、`sync`、`atomic` | OS 线程 + 同步原语；**不**等于 async 运行时（见 `tokio`） |
| **I/O** | `Read`/`Write`、`fs`、`net` | 阻塞式 OS I/O 抽象；async 在生态 crate |
| **FFI** | `std::ffi`、`CString`、`OsString` | 与 C 字符串 / OS 编码交互；**unsafe** 边界在调用方 |
| **错误** | `Result`、`Error`、`io::Error` | 可恢复失败传播；与 [RFR Ch04](../../02-RFR/Chapter-04-Error-Handling/README.md) 衔接 |
