# 11.6 Once 类型分析

> 章索引：[第 11 章](./README.md) · 前：[11.5](./11.5-barrier.md) · 后：[11.7 · OnceLock](./11.7-oncelock.md)

---

## 一句话

**`std::sync::Once`** 保证 **`call_once` 闭包** 多线程下 **只执行一次** — **无内嵌存储 `T`**（存储见 OnceLock）；底层 **原子状态 + futex 等待** 初始化完成。

---

## 原书要点

| 对比 | `Once` | `OnceLock<T>` |
|------|--------|---------------|
| 存储 | 无 · 闭包内自行处理 | **内嵌 `T`** |
| API | **`call_once(|| { … })`** | **`get_or_init`** |

### 执行逻辑

```text
线程 A: 原子 CAS 抢到 init 权 → 执行闭包 → 标记完成 → wake
线程 B/C: CAS 失败 → futex wait → 被唤醒 → 直接返回
```
