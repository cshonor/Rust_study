## 一句话

**MPSC**（Multi-Producer, Single-Consumer）— **`std::sync::mpsc`** 体现 **「通过通信共享内存」** · **`Sender` 多克隆** + **唯一 `Receiver`** · 底层 **队列 + futex 阻塞/唤醒 + oneshot 变体**。

---

## 原书要点

```text
Go: 不要通过共享内存来通信，而要通过通信来共享内存
Rust std: send/recv ──► 所有权 move T，减少 Mutex 争用
```

### 核心结构

| 类型 | 说明 |
|------|------|
| **`Sender<T>`** | **`Clone`** · 多线程并发 **send** |
| **`Receiver<T>`** | **唯一** · **recv** 阻塞读 |

### 底层四要素

1. **无界/有界** 队列存消息（`channel` / `sync_channel(n)`）
2. **原子计数** 跟踪 Sender 数量
3. **recv 空** → futex/park 阻塞 · **send** → wake
4. **关闭**：全部 Sender drop → **recv 返回 `Err`（通道关闭）**
