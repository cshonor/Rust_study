# 11.3 Condvar 类型分析

> 章索引：[第 11 章](./README.md) · 前：[11.2.3](./11.2.3-mutex-public-api.md) · 后：[11.3.1 · OS 层](./11.3.1-condvar-os-layer.md)

---

## 一句话

**`Condvar`** 实现 **「等条件成立再往下走」** — **必须** 与 **`Mutex`** 配对：**`wait(guard)`** 原子 **释放锁 + 阻塞** · **`notify_one/all`** 唤醒后 **重新抢锁**；底层 **futex 等待队列**。

---

## 原书要点

| API | 行为 |
|-----|------|
| **`wait(&mut MutexGuard)`** | 释放 Mutex · 睡眠 · 唤醒后 **再 lock** |
| **`notify_one()`** | 唤醒 **一个** 等待线程 |
| **`notify_all()`** | 唤醒 **全部** 等待线程 |

```text
线程 A: lock → 条件不满足 → wait（释放锁 + 睡）
线程 B: lock → 改状态 → notify → A 被唤醒再 lock
```

**三层** 同 [11.2](./11.2-mutex-overview.md)：[11.3.1](./11.3.1-condvar-os-layer.md) · [11.3.2](./11.3.2-condvar-os-agnostic.md) · [11.3.3](./11.3.3-condvar-public-api.md)

底层仍依赖 [11.1 · Futex](./11.1-futex.md) · MPSC **recv 阻塞** 亦用类似 **park/condvar**（[11.10.2](./11.10.2-mpsc-block-wake.md)）。
