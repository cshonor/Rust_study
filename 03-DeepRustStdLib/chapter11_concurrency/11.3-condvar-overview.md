# 11.3 Condvar 类型分析

## 一句话

**`Condvar`** 实现 **「等条件成立再往下走」** — 必须与 **`Mutex`** 配对：**`wait(guard)`** 原子释放锁并睡眠 · 被 **`notify`** 唤醒后重新抢锁；同样 **OS / 无关 / 对外** 三层。

---

## 原书要点

| 模式 | API |
|------|-----|
| **等待** | `condvar.wait(&mut mutex_guard)` |
| **唤醒** | `notify_one` / `notify_all` |

```text
线程 A: lock → 条件不满足 → wait（释放锁 + 睡）
线程 B: lock → 改状态 → notify → A 被唤醒再 lock
```
