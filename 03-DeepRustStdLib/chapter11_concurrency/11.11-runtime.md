## 一句话

**Rust 无 Go/Java 式 GC + 全局协程调度器** — **`std` 极简 Runtime** ≈ **1:1 OS 线程 + futex 同步 + park + 全局分配器 + panic**；**async 调度在 tokio/async-std**，不在本章 std 范围。

---

## 原书要点

### std 里有什么 / 没有什么

| 组件 | 是否在 `std` Runtime |
|------|----------------------|
| **GC** | ❌ · **所有权 + Drop** |
| **绿色线程 / M:P:G 调度** | ❌ · **1:1 内核线程** |
| **Mutex / MPSC / thread** | ✅ |
| **堆 `System`** | ✅ |
| **async executor** | ❌ · 第三方 · 见 [Ch14](../chapter14_async/README.md) |

### 与 Go / Java 对比

| | **Rust std** | **Go** | **Java** |
|---|-------------|--------|----------|
| 线程模型 | **OS 线程** | **M/P/G 协程** | JVM 线程池 |
| 调度 | **操作系统** | **Go runtime** | **JVM + GC** |
| 并发开销 | **仅 futex/同步原语** | **runtime + GC** | **runtime + GC** |
| 适用 | **系统底层 · 零额外负担** | 高并发服务 | 企业级 JVM 生态 |
