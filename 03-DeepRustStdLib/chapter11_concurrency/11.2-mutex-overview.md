# 11.2 Mutex<T> 类型分析

## 一句话

**`Mutex<T>`** 保证 **`T` 多线程独占访问** — 与 [Ch10](../chapter10_process_management/10.3-process-mgmt.md) 同构 **三层**：**futex OS 层 → poison/状态机 → `lock()`/`MutexGuard`**。

---

## 原书要点

### 结构组成

```rust
pub struct Mutex<T> {
    inner: sys::Mutex,       // 底层 futex 锁
    poison: AtomicBool,      // panic 持锁 → 中毒
    data: UnsafeCell<T>,     // 锁内内部可变性（Ch7）
}
```

### 关键机制

| 机制 | 说明 |
|------|------|
| **中毒 poison** | 持锁线程 panic → **`PoisonError`** · 防脏数据 |
| **`lock()`** | 阻塞至获锁 |
| **`try_lock()`** | 非阻塞 |
| **`MutexGuard`** | RAII · Drop **自动 unlock** |

```text
Mutex::lock()
  └── 11.2.3 对外 API（poison · MutexGuard · Sync）
        └── 11.2.2 OS 无关（锁 word · 排队）
              └── 11.2.1 OS 层（futex wait/wake）
```

|:--:|------|
| [11.2.1](./11.2.1-mutex-os-layer.md) | **futex / WaitOnAddress** |
| [11.2.2](./11.2.2-mutex-os-agnostic.md) | **状态 0/1/≥2 · parking** |
