# 11.2.3 Mutex — 对外接口层

## 一句话

**`std::sync::Mutex`** — **`lock()` → `MutexGuard`** · **`Deref`/`DerefMut` 访问 `T`** · **`Sync` 当 `T: Send`**；开发者只见 **safe API**，不见 futex。

---

## 原书要点

| API | 说明 |
|-----|------|
| **`lock()`** | 阻塞至获得锁 · poison 时 **`PoisonError`** |
| **`try_lock()`** | 非阻塞 |
| **`MutexGuard`** | 作用域结束 **自动 unlock** · 禁止手动释放 |
| **`into_inner`** | 拆 Mutex 取 `T`（需处理 poison） |

```rust
let m = Arc::new(Mutex::new(0));
let mut book = m.lock().unwrap();
*book += 1;
// drop guard → unlock
```

### 安全边界

- **`unsafe` 仅在 sys** · 上层 **`lock` 全 safe**
- **`T: Send`** 才能 **`Mutex<T>: Sync`**

### HFT

**订单簿全局 Mutex** — 简单正确 · 竞争大时延迟高 · 常改 **RwLock 读多写少** 或 **分片**。
