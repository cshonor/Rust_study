# 12.1 OS 相关适配层

> 章索引：[第 12 章](./README.md) · 前：[12.0 · 总览](./12.0-chapter-overview.md) · 后：[12.1.1 · 路径](./12.1.1-path-types.md)

## 一句话

**`std::sys::fs` + `sys::path`** — 文件系统 **OS 差异最大** 的领域；所有 **裸 SYSCALL、fd/HANDLE、平台路径字节** 收敛于此（**全 `unsafe`**），向上只暴露 **`io::Result`** 与内部 **`imp::File`**。

---

## 原书要点

### 分层（延续 Ch9～11）

```text
12.2  std::fs / Path     （对外 safe · File + Drop）
  └── 12.1  sys::fs      （SYSCALL · 平台路径）
        └── 9.5 OwnedFd / Handle
  └── std::os::unix/windows::fs  （扩展 Trait · 按需）
```

| 子节 | sys 职责 |
|:--:|------|
| [12.1.1](./12.1.1-path-types.md) | **`sys::path`** · OsStr 存储 · `components()` |
| [12.1.2](./12.1.2-file-ops.md) | **`sys::fs::File`** · open/read/write |
| [12.1.3](./12.1.3-dir-ops.md) | **`ReadDir`** · mkdir/stat/link/symlink |

### 其他底层封装

| 操作 | Unix | Windows |
|------|------|---------|
| **metadata** | `stat`/`lstat` | `GetFileAttributesW` 等 |
| **remove** | `unlink`/`rmdir` | `DeleteFileW` |
| **hard_link** | `link` | `CreateHardLinkW` |
| **symlink** | `symlink` | 文件/目录两种 API |

统一映射为上层 **`Metadata`/`DirEntry`** 标准字段。
