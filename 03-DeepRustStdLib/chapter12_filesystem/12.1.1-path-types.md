# 12.1.1 路径名类型分析

> 章索引：[第 12 章](./README.md) · 前：[12.1](./12.1-fs-os-layer.md) · 后：[12.1.2 · 文件操作](./12.1.2-file-ops.md)

---

## 一句话

**路径** 在底层是 **OS 原生字节序列** — [9.1.4 · `OsString`/`OsStr`](../chapter09_userspace_std_basics/9.1.4-osstring.md) 之上，**`Path` / `PathBuf`** 增加 **语义化路径操作**（join、parent、extension），**不保证 UTF-8**。

---

## 原书要点（脉络 · 待刷书）

| 类型 | 所有权 | 说明 |
|------|--------|------|
| **`&Path`** | 借用 | 类似 **`&str`** 之于 String |
| **`PathBuf`** | 拥有 | 可 **`push`** 段 |
| **`OsStr`** | 底层 | Windows **UTF-16** · Unix **任意字节** |

### 跨平台差异

| 项 | Unix | Windows |
|----|------|---------|
| 分隔符 | `/` | `\` 或 `/`（部分 API） |
| 根 | `/` | 盘符 `C:\` |
| 编码 | 常 UTF-8 非强制 | **宽字符** |

```rust
let p = Path::new("/var/log/app.log");
p.file_name(); p.extension();
```

### 与 SYSCALL

**`open(path)`** 前 **`Path` → OsStr → 平台字节** — [12.1.2](./12.1.2-file-ops.md)

### HFT

策略 **Linux 部署** · 日志路径用 **`PathBuf::from`** · 避免硬编码 `\`。
