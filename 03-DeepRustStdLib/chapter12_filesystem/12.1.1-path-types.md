# 12.1.1 路径名类型分析

> 章索引：[第 12 章](./README.md) · 前：[12.1](./12.1-fs-os-layer.md) · 后：[12.1.2 · 文件操作](./12.1.2-file-ops.md)

---

## 一句话

**路径底层 = OS 原生字节/宽字符** — **`String/str` → `OsString/OsStr` → `PathBuf/Path`** 四层包装；**禁止** 把非 UTF-8 路径当 **`&str`** 用，从类型层面杜绝跨平台乱码。

---

## 原书要点

### 平台编码根本差异（本章难点）

| 平台 | 原生表示 | SYSCALL 入参 |
|------|----------|--------------|
| **Unix** | **字节流** `&[u8]` · 主流 UTF-8 非强制 | `open(path_bytes, …)` |
| **Windows** | **UTF-16** `WCHAR` | **`CreateFileW`** 等仅宽字符 API |

### 四层关系

| 层 | Unix 内部 | Windows 内部 | 角色 |
|----|-----------|--------------|------|
| **`OsString`** | `Vec<u8>` | `Vec<u16>` | 平台原生 **拥有** |
| **`OsStr`** | `&[u8]` | `&[u16]` | 借用 · 等同 **`str`** |
| **`PathBuf`** | 包 `OsString` | 同左 | **可改** · join/push |
| **`Path`** | 包 `OsStr` | 同左 | **只读** · 轻量 |

```rust
let p = Path::new("/var/log/app.log");
p.file_name();
p.extension();
p.components(); // Root / CurDir / ParentDir / Normal
```

### 底层核心逻辑

| 机制 | 说明 |
|------|------|
| **分隔符抹平** | **`join`/`push`** 自动 `/` 或 `\` · 禁止硬编码 |
| **`components()`** | 根、`..`、普通段 → **`Component`** 枚举 · 逻辑在 **`sys::path`** |
| **无损转换** | **`OsStr::to_str()` → `Option<&str>`** · 非 UTF-8 返回 **`None`** |
| **fs 入口** | **`impl AsRef<Path>`** → **`OsStr`** → 平台字节/宽字符 → syscall |

### HFT

策略 **Linux 部署** · 日志路径 **`PathBuf::from`** · 避免硬编码 `\`。

→ 编码转换极简注释：[12.5 · Path 走读](./12.5-path-encoding-walkthrough.md)
