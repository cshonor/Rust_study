# 12.1.3 目录操作分析

## 一句话

**目录 sys 层** — Unix **`opendir/readdir/closedir` + mkdir/stat/unlink/link/symlink`** · Windows **`FindFirstFileW/FindNextFileW + CreateDirectoryW`** — 统一封装为 **`ReadDir` 迭代器** 与 **`Metadata`** 标准字段。

---

## 原书要点

### read_dir 底层

| 平台 | 遍历 API |
|------|----------|
| **Unix** | `opendir` → `readdir` → `closedir` · dirent 文件名/inode |
| **Windows** | `FindFirstFileW` → `FindNextFileW` · **WIN32_FIND_DATA** |

上层 **`fs::read_dir(path)?`** → **`ReadDir` → `DirEntry`** · 隐藏两套 API。

### 常用操作对照

| 对外（12.2） | Unix | Windows |
|--------------|------|---------|
| **`create_dir` / `create_dir_all`** | `mkdir` | `CreateDirectoryW` |
| **`remove_file` / `remove_dir_all`** | `unlink` / `rmdir` | `DeleteFileW` |
| **`metadata` / `symlink_metadata`** | `stat` / `lstat` | 属性 API |
| **`hard_link` / `symlink`** | `link` / `symlink` | `CreateHardLinkW` / 符号链接 API |
| **`canonicalize`** | `realpath` 类 | `GetFullPathNameW` 类 |

```rust
for entry in fs::read_dir(path)? {
    let entry = entry?;
    let p: PathBuf = entry.path();
    let meta = entry.metadata()?;
}
```

### 权限与符号链接

| 点 | 说明 |
|----|------|
| **权限** | Unix **rwx mode** · Windows **ACL/只读/隐藏** → **`Metadata`** 通用字段 + **`os` 扩展** |
| **symlink** | **`metadata`** 跟随链接 · **`symlink_metadata`** 不跟随 |
