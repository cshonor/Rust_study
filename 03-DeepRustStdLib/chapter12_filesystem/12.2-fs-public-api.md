## 一句话

**`std::fs` + `std::path`** — 开发者 **stable safe API**；**`File` Drop 自动 close fd/HANDLE** · 实现 **Read/Write/Seek** · 与 [Ch10 管道](../chapter10_process_management/10.1-anonymous-pipe.md) **同一 RAII fd 模型**。

---

## 原书要点

### File 安全设计（本章核心）

```rust
pub struct File {
    inner: imp::File, // sys 层 fd / HANDLE
}
// Drop → inner 关闭 · 作用域结束必释放
```

| 特性 | 说明 |
|------|------|
| **RAII** | 离开作用域 **自动 close** · 无 C 式泄漏 |
| **Drop 忽略 close 错误** | 需可靠落盘 → **`sync_all()`** 后再 drop |
| **IO trait** | **Read/Write/Seek** · 复用 [Ch9/Ch13 IO 体系](../chapter13_io/README.md) |
| **不可 Copy** | **`try_clone()`** 显式 dup fd · 返回 **`Result`** |

### DirEntry / Metadata

| 类型 | 作用 |
|------|------|
| **`DirEntry`** | **`path()`/`file_type()`/`metadata()`** · 屏蔽 dirent vs WIN32_FIND_DATA |
| **`Metadata`** | **len/mtime/is_dir/is_symlink** · Unix rwx · Windows 只读/隐藏 **统一映射** |

### 高频工具函数

```rust
fs::read(path)           // → Vec<u8>
fs::write(path, data)    // 创建/覆盖
fs::create_dir / create_dir_all
fs::remove_file / remove_dir_all
fs::canonicalize         // 规范化 · 解析 ..
fs::hard_link / symlink
```

```rust
let mut f = File::create(path)?;
f.write_all(b"log line\n")?;
// drop(f) → close
```

### TOCTOU 竞态（文档重点）

```text
错误：try_exists() 为 false → 再 File::create()
      间隙其他进程/线程已创建同名文件 → 逻辑错误

推荐：OpenOptions::new().create_new(true).open(path)
      单次 syscall 原子「不存在则创建」
```

### 平台扩展层

**`std::os::unix::fs::OpenOptionsExt`** · **`std::os::windows::fs`** — 按需 **Unix O_* / Windows 共享模式**，不污染跨平台代码。
