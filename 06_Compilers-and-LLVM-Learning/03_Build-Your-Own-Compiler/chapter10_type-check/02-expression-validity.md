## 主要目的

**语法合法、语义不可求值** 的表达式 — 在 **类型相容** 之前先拦。

| 非法例 | 原因 |
|--------|------|
| **`1 = 2 + 3`** | 左值必须是 **可赋值** 对象 |
| **`*1`** | `*` 要求 **指针** 操作数 |
| **`&1`** | `&` 要求 **左值**（lvalue） |

检查 **运算前提**（是否 lvalue、是否指针）— 与 ch10 §3 **类型是否匹配** 互补。

---

## DereferenceChecker 与避免连锁错误

复杂式 **`***(1++)`** — 内层 `1++` 非法时，外层可能 **连续报多条** 衍生错。

**cbc 策略**：

```text
发现错误 → 立刻 throw SemanticError
try { 检查子表达式 }
catch (SemanticError) { 跳到 **语句末尾** }
→ 本语句停止深入，**继续检查下一条语句**
```

| 目标 | 说明 |
|------|------|
| **不泛滥** | 一个根因一条（或少量）诊断 |
| **不中断全文件** | catch 后 **继续** 遍历 AST |

→ 编译器 UX：类似 **rustc** 有时 **abort 本子表达式** 继续后面。

## 隐式指针生成（decay）

C/C♭：**数组类型**、**函数类型** 在表达式中 ** decay 为指针**。

| 上下文 | 转换 |
|--------|------|
| 数组名 | **`T[]` → `T*`**（指向首元素） |
| 函数名 | **函数 → 函数指针** |

**本阶段**：对数组/函数类型 **特别处理**，在 AST 上 **视为地址/指针类型** — 供后续 `*`、下标、调用 使用。

→ RFR/CI 对比：Rust **无** 数组隐式 decay — `&arr` 显式。
