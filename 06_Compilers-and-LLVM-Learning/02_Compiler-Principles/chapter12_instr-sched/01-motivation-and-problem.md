## 硬件延迟与停机（Stalls）

| 现象 | 说明 |
|------|------|
| **多发射 / 超标量** | CPU 可在长延迟操作进行时启动**其他独立**操作 |
| **真依赖（RAW）** | 后续指令需要前序**结果** → 必须等待 |
| **Stall / 气泡** | 流水线空转 — **load**、**FP mul**、**cache miss** 等尤甚 |

```text
cycle:  1    2    3    4    5
        load r1, [a]     ← 3 cycle 延迟
        add  r2, r1, r3  ← stall：r1 未就绪
```

**调度目标**：在依赖允许范围内，把**无关指令提前**，填满 stall 槽 — **隐藏延迟**。

---

## 调度的目标与约束

| 必须 | 禁止 |
|------|------|
| **缩短关键路径 / 总周期（近似）** | 改变程序**可观测语义** |
| 尊重 **数据 / 控制 / 内存** 依赖 | 违反内存序、volatile 等 |

与 ch1 [正确性](../chapter01_overview/02-two-fundamental-principles.md) 一致 — 调度是**性能变换**，非语义变换。

---

## NP 完全性

**最优**指令调度（给定精确延迟模型）是 **NP-complete**。

| 推论 | 工程应对 |
|------|----------|
| 无多项式最优算法 | **启发式** — 列表调度等 |
| 全局最优难 | 常按**基本块**或**区域**局部调度 |
| 模型不精确 | 用**典型延迟表** + 迭代调优 |

→ 编译器追求 **「足够好」**，非绝对最优。

---

## 依赖图（概念）

调度前构建 **DAG**：

```text
  load r1 ──→ add r2,r1,r3 ──→ store …
                ↑
  load r4 ──────┘
```

- **边** = 必须满足的先后顺序
- **调度** = DAG 的**拓扑序** + 填隙优化

与 ch11 表达式树 DAG、ch8 数据流分析不同层 — 此处是**机器指令级**依赖。

---

## HFT 联想

热点循环内一条 **load 后立刻 use** 未调度 → 每迭代多几个 cycle；对低延迟路径累积明显。
