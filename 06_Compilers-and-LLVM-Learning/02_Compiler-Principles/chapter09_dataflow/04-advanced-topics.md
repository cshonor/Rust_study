## 结构数据流算法与可约性（Reducibility）

| 概念 | 说明 |
|------|------|
| **可约 CFG** | 可通过特定变换归约为单节点（结构化控制流） |
| **不可约 CFG** | `goto` 乱 jump 等 — 更难分析 |
| **影响** | 某些数据流算法在可约图上**更快**或**更简单** |

**现代语言**：结构化控制为主 → 多数函数 CFG **可约**；`unsafe goto` / 异常边增加复杂度。

---

## 过程间数据流分析（Interprocedural）

将分析从**单函数**扩展到**跨调用**：

| 问题 | 说明 |
|------|------|
| **调用点摘要** | 被调函数对调用者的效应（mod/ref） |
| **上下文敏感** | 不同调用点区分分析 |
| **代价** | 可指数爆炸 — 工程用 **上下文不敏感** 或 **摘要** 近似 |

与 [ch8 §5 内联](../chapter08_intro-opt/05-interprocedural-opt.md) 互补：

| 手段 | 效果 |
|------|------|
| **内联** | 把过程间变过程内 — 分析变简单 |
| **IPO 分析** | 不内联也跨过程收集事实 |

**LTO**：链接期全程序分析 — Whole-program 数据流。
