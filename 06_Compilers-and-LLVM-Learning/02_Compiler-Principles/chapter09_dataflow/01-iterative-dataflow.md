## 核心目标

在**编译期**推导程序**运行时**的值流向与状态 — 为优化提供**可证明安全**的事实。

| 典型问题 | 数据流分析实例 |
|----------|----------------|
| 某点哪些表达式已算过？ | **可用表达式** → [ch8 §4](../chapter08_intro-opt/04-global-redundancy.md) |
| 某点哪些变量还会被用？ | **活跃变量**（本章重点） |
| 某定值能到达哪？ | **到达定值** → ch10 |

---

## 迭代框架（通用）

在 **CFG** 上对每个基本块维护 **IN / OUT** 集合：

```text
OUT[B] = gen[B] ∪ (IN[B] − kill[B])
IN[B]  = ⋃ OUT[P]   （P 为 B 的前驱）
```

- **正向分析**：沿执行方向传播（可用表达式）
- **反向分析**：沿控制流**逆**向传播（**活跃变量**）

反复迭代至 **meet-over-paths** 不动点。

---

## 活跃变量（Live Variables）

| 定义 | 变量 **v** 在程序点 **p** **活跃** ⇔ 从 **p** 到出口某路径上 **v 会被读** 且此前未被重新定义 |
|------|--------------------------------------------------------------------------------------------------|

**用途**：

| 优化 | 如何用 LIVE |
|------|-------------|
| **寄存器分配** | 不活跃 → 寄存器可复用 |
| **死代码消除（DCE）** | 对不活跃变量的赋值可删 |

### LIVEOUT 示例

对每个基本块 **B** 计算 **LIVEOUT(B)** — 离开 B 时活跃的变量集。

```text
// 反向方程（概念）
LIVEIN(B)  = use(B) ∪ (LIVEOUT(B) − def(B))
LIVEOUT(B) = ⋃ LIVEIN(S)   （S 为 B 的后继）
```

→ 全书用此例教**迭代算法**的写方程、初始化、收敛。

## 局限与工程问题

| 局限 | 说明 |
|------|------|
| **单调框架假设** | 需有限格、单调传递函数 |
| **复杂度** | 迭代次数、格高度 |
| **指针/别名** | 精确分析难 — 需保守近似或指针分析 |
| **过程间** | 单函数分析不够 → [§4](./04-advanced-topics.md) |

**Rust/LLVM**：函数内 live analysis 是 regalloc、DCE 标配；别名由 `AliasAnalysis` 等补充。
