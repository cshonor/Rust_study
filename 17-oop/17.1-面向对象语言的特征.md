# 17.1 面向对象语言的特征

“算不算面向对象”在社区里没有唯一标准。Rust 受多种范式影响；从常见 OOP 视角看，可以对照三件事：**对象（数据 + 行为）**、**封装**、**继承/多态**。

对应 demo：`17-oop/17.1-averaged-collection-demo/`（`AveragedCollection` 封装示例）

---

## 对象：数据 + 过程（方法）

《设计模式》中的经典表述：程序由**对象**构成；对象**打包数据**以及**操作数据的过程**（常称方法）。

在 Rust 里，**结构体 / 枚举**承载数据，**`impl` 块**提供方法。虽通常不口头称“对象”，但能力与上述定义一致。

---

## 封装：隐藏实现、只暴露 API

**封装**：外部不能随意碰到内部表示，只能通过类型提供的**公有 API** 交互，从而在不影响调用方的前提下重构内部。

Rust 通过 **`pub`** 控制可见性（模块、字段、函数、方法等），默认私有；例如 `pub struct` 对外可见，但字段仍可保持私有，由 `add` / `remove` / `average` 等方法维护不变式（如示例中的平均值缓存）。

---

## 继承与多态

**继承**：子类型复用父类型的字段与方法定义。Rust **没有**“结构体继承另一个结构体字段与方法”的语法。

常见动机与 Rust 中的常见替代：

1. **复用实现**  
   - 可用 **trait 的默认方法**（类似“父类有默认实现，实现者可选择覆盖”），见第 10 章 `Summary` 与 `summarize` 一类例子。

2. **子类型可替代父类型（多态）**  
   - Rust 更常用 **泛型 + trait bounds**（有界参数多态），以及 **trait 对象**（`dyn Trait`）实现运行时多态；后文会展开 trait 对象。

多态（polymorphism）泛指**同一段接口可作用于多种类型**；不限定于继承树。
