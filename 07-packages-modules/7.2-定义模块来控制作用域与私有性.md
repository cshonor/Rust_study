# 7.2 定义模块来控制作用域与私有性

模块（module）用来把一个 crate 内的代码分组，提升可读性与复用性；同时还能控制“哪些项对外可见”。

对应 demo 项目：`07-packages-modules/7.2-modules-privacy-demo/`

---

## 模块树（module tree）与 crate root

`src/main.rs` 或 `src/lib.rs` 会作为 **crate root**，其内容构成模块树的根（隐式模块 `crate`）。

下面这段代码（餐厅例子）会形成一棵模块树：

```rust
mod front_of_house {
    mod hosting {
        fn add_to_waitlist() {}
        fn seat_at_table() {}
    }

    mod serving {
        fn take_order() {}
        fn serve_order() {}
        fn take_payment() {}
    }
}
```

对应的模块树可以理解为：

```text
crate
 └── front_of_house
     ├── hosting
     │   ├── add_to_waitlist
     │   └── seat_at_table
     └── serving
         ├── take_order
         ├── serve_order
         └── take_payment
```

这种结构很像“文件系统目录树”：模块负责组织，接下来就需要用 **路径（path）** 去定位到模块/函数等项。

---

## 私有性（privacy）：默认私有，`pub` 才公开

Rust 的可见性规则（先记住最核心的两条）：

- **模块内的项默认是私有的**（同一模块内部可访问；模块外不可直接访问）
- 用 `pub` 把模块或项标记为公有后，模块外才可能通过路径访问到

demo 会用“对外暴露一个函数，内部调用私有实现”的方式，避免把内部细节直接暴露出去。
