# Effective Rust — 目录结构

> **约定**：一个 Item = 一个目录；笔记与相关代码放在同一目录下，便于日后扩展。

## 布局

```
ER/
├── ER-本书目录.md          # 全书索引
├── ER-拓展索引.md          # §6 拓展汇总
├── 目录结构.md             # 本文件
├── Chapter-01-Types/
│   ├── Item-03-option-result-transforms/
│   │   ├── README.md       # 导航 hub（状态、对照、专题索引）
│   │   ├── topics/         # 专项小笔记（按需阅读）
│   │   └── demo/
│   └── Item-01-express-data-structures/
│       ├── README.md
│       ├── topics/         # 01-fundamental-types.md … cheat-sheet.md
│       └── demo/           # 可选
├── …
├── Cargo.toml              # workspace 根（members 指向各 Item/*/demo）
├── rust-toolchain.toml
├── ER-demos/               # MSRV / clippy 文档与配置
│   ├── WORKSPACE.md
│   └── README.md
└── scripts/
```

## 命名

| 路径 | 含义 |
|------|------|
| `Item-NN-slug/README.md` | **导航 hub**：状态、Book 对照、一句话、专题索引 |
| `Item-NN-slug/topics/*.md` | **专项笔记**：每个文件只讲一块，便于分段阅读 |
| `Item-NN-slug/topics/cheat-sheet.md` | 可选：背诵提纲 / 考前速览 |
| `Item-NN-slug/demo/` | 主 demo crate（多数 Item） |
| `Item-NN-slug/demo-*/` | 额外 demo（如 Item 4 的 `demo-core-error`、Item 35 的 `demo-bindgen` / `demo-sys-workspace`） |

## 笔记拆分约定

长 Item 按 **topics/** 拆成多个小文件，`README.md` 只做目录：

```
Item-02-express-common-behavior/
├── README.md              # hub（~50 行）
└── topics/
    ├── 01-core-concepts.md
    ├── 02-logic-flow.md
    ├── 03-key-takeaways.md
    ├── 04-examples.md
    ├── 05-pitfalls.md
    └── cheat-sheet.md
```

Item 01 按知识点拆得更细（`01-fundamental-types` … `07-pitfalls`）；其余 Item 统一用上面 6 个文件名。批量拆分脚本：`scripts/split-item-topics.py`。

## 新增 Item 内容时

1. 在对应 `Item-NN-slug/README.md` 维护 hub；正文放进 `topics/` 专项文件。
2. 若有代码，在同级建 `demo/`（或 `demo-foo/`），并在 [`ER/Cargo.toml`](./Cargo.toml) 的 `members` 里加入路径。
3. 在 [ER-拓展索引.md](./ER-拓展索引.md) 与 Item 的 §6 中更新链接。

## 运行 demo

```bash
cd ER
cargo test --workspace
```

独立 workspace（Item 24、Item 35 sys）在各自 `demo/` 或 `demo-sys-workspace/` 目录内运行，见各 README。
