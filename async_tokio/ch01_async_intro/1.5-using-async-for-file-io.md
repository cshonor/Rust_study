# 1.5 Using Async for File I/O（异步文件 I/O）

## 核心梳理

- **书中实战**：监控文件变更——`tokio::fs` 读元数据 + 后台循环 + **`watch::channel`** 通知主循环无阻塞刷新内容。
- **误区**：async **不会**让磁盘物理读写更快；收益是**不堵住**执行器/其它请求。
- **平台现实**：`tokio::fs` 常在线程池模拟；Linux 更彻底路线含 **`io_uring`** / `tokio-uring`。
- **铁律**：轮询循环里用 **`tokio::time::sleep`**，禁止 **`std::thread::sleep`**（见 §1.7）。
