# 1.4 Where Can We Utilize Async?（在哪里利用异步？）

## 核心梳理

- **最佳场景**：存在**潜在延迟**的操作——**文件 I/O**、**网络**、用户输入等。
- **数量级**：内存 ~**ns** vs 网络 RTT ~**ms**（差 10⁴～10⁶）→ I/O 常是瓶颈；async 把等待期 CPU 让给其它任务。
- **不适用**：**CPU-bound**（加密、转码、大规模数值）——靠更多核、更好算法、SIMD；套 async **不会魔法加速**。

## 决策简表

| 类型 | 是否优先 async |
|------|----------------|
| 等网络/磁盘 | 是 |
| 纯计算 | 否（用线程池/并行库） |
