# 1.2 Introduction to Processes（进程简介）

## 核心梳理

- **定义**：进程是 OS **分配资源**的边界，独立虚拟地址空间，有 **PID**。
- **硬隔离**：一进程崩溃通常不直接破坏另一进程内存（如 PostgreSQL 每连接一进程的模式）。
- **局限**：
  - 创建/销毁**昂贵**（页表、句柄、内核结构）；
  - **扩展性差**（大量连接时进程数爆炸）；
  - **IPC** 需序列化 + 内核中转，延迟与复杂度高。

## 与异步的关系

标准 **async 服务端**多为**单进程 + 多线程/async 运行时**，在同地址空间用 `Arc`/通道共享状态；多进程用于隔离部署，而非每个请求一进程。
