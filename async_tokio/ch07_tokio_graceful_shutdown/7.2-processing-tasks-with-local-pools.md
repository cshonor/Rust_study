# 7.2 Processing Tasks with Local Pools（使用本地池处理任务）

## `LocalPoolHandle` 做什么

在 **Tokio 生态内**（相对第 3 章「从零写执行器」）用 **`tokio_util::task::LocalPoolHandle`**：

- 将任务绑定到**指定本地线程池**，甚至 **`spawn_pinned_by_idx`** 固定到某索引 worker。
- 运行 **未实现 `Send` 的 Future**（线程局部、`Rc` 等）。
- 利用**线程亲和性**：数据留在同一 worker 缓存，减少多线程争用**全局 Mutex**。

## 与默认工作窃取对比

| 特性 | 默认 Tokio 工作窃取 | 本地池 + pinned |
|------|---------------------|-----------------|
| 负载均衡 | 自动窃取 | 无窃取，可能单线程过载 |
| `!Send` | 不可跨线程 spawn | 可在固定线程跑 |
| 锁竞争 | 可能争用全局结构 | 可本地化状态 |

## 示例代码

```rust
use tokio_util::task::LocalPoolHandle;

#[tokio::main]
async fn main() {
    let pool = LocalPoolHandle::new(4);
    let handle = pool.spawn_pinned_by_idx(
        || async { 42 },
        0, // worker 索引 < num_threads
    );
    let result = handle.await.unwrap();
}
```
