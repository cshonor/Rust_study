# 3.2 Increasing Workers and Queues（增加工作线程与队列）

## 核心梳理

- **从单消费者扩展**：克隆 `flume::Receiver`，为每个 worker 线程各持一份接收端。
- **线程池**：多个线程同时 `recv` / 执行 Runnable，提高 CPU 利用率。
- **注意**：`flume` 每条消息通常只被一个 receiver 消费；多 receiver 竞争同一流上的任务，符合「多 worker 抢队列」模型。

## 实现要点

```text
Sender (spawn 侧) ──► flume ──► Receiver clone × N ──► worker 循环 run task
```

## 陷阱

worker 内勿 `std::thread::sleep` 阻塞整条执行路径；演示用可，生产应协作式等待或 park。
