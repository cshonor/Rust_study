# 11.7 Fine-Grained Future Testing（细粒度 Future 测试）

> Demo：[11.7-fine-grained-future-testing/11.7-fine-grained-future-testing-demo.rs](./11.7-fine-grained-future-testing/11.7-fine-grained-future-testing-demo.rs)

## `tokio-test` 能做什么

对底层状态机 **逐步 `poll`**，而不跑完整运行时调度：

- `tokio_test::task::spawn(fut)` 得到可手动推进的 handle；
- **`assert_pending!()`** / **`assert_ready!()`** 检查当前 poll 结果；
- 可**调换**多个 Future 的 poll 顺序，复现「自然调度很难碰到」的锁竞争序。

## 与第 10 章关系

第 10 章用 **std + RawWaker** 手写 `block_on`；`tokio-test` 在 Tokio 生态里提供同类能力，且与 `tokio::sync::Mutex` 等集成更好。

## 典型场景

| 场景 | 断言 |
|------|------|
| A 持锁未释放 | B 的 Future 下一次 poll 为 `Pending` |
| 唤醒后 | B 变为 `Ready` |
| 取消 | drop 后不再 poll |

## 依赖

```toml
[dev-dependencies]
tokio-test = "0.4"
```

## Demo

本目录 demo 用 **纯 std** 实现 `CounterFuture` 两次 `poll`（Pending → Ready），不引 `tokio-test` 依赖；对照 §10.2 noop waker demo。
