# 11.5 Testing Channel Capacity（测试通道容量）

> Demo：[11.5-testing-channel-capacity/11.5-testing-channel-capacity-demo.rs](./11.5-testing-channel-capacity/11.5-testing-channel-capacity-demo.rs)

## 有界 `mpsc`

`tokio::sync::mpsc::channel(cap)`：`cap == 0` 为 rendezvous；`cap > 0` 为缓冲。发送在满时：

- `send().await` → **阻塞**直到有空间；
- `try_send()` → 立即返回 `Err(TrySendError::Full)`。

## 要测的行为

1. **背压**：生产者不能无限超前；
2. **恢复**：消费者 `recv` 后，阻塞的发送应完成；
3. **超时**：若发送在 `timeout` 内未完成，说明背压符合预期（或死锁）。

## 测试骨架

```rust
let (tx, mut rx) = tokio::sync::mpsc::channel(2);
tx.send(1).await.unwrap();
tx.send(2).await.unwrap();
let send_more = tokio::spawn(async move { tx.send(3).await });
let blocked = timeout(Duration::from_millis(100), send_more).await.is_err();
assert!(blocked);
let _ = rx.recv().await;
// 此时 send(3) 应能完成
```

## 与生产

通道过大 → 掩盖背压问题；过小 → 吞吐下降。测试应锁定**业务承诺**（满时阻塞还是丢弃）。
