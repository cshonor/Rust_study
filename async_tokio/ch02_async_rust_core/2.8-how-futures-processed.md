# 2.8 How Are Futures Processed?（Future 如何被处理）

## 全景生命周期

```text
创建 Future（async fn 调用 / 手动 new）
    → await 或 spawn 包装为 Task
    → 执行器入队
    → poll：Ready → 完成并产出结果；Pending → 登记 Waker / 让出
    → wake → 重新入队 → 再次 poll
    → 完成后释放 Task / 状态机
```

## 角色分工

| 组件 | 职责 |
|------|------|
| **Future** | 状态机；`poll` 推进 |
| **Task** | 持有 Future；供执行器调度 |
| **Executor** | 就绪队列；调用 `poll` |
| **Waker** | 就绪通知 → 重新调度 |
| **Pin** | 固定自引用状态机 |
