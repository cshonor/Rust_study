# 2.4 Context in Futures（Future 中的 Context）

## 核心梳理

- **`Context` 的核心作用**：在 `poll` 时提供 **`Waker`** 句柄。
- **流程**：Future 发现未就绪 → `Pending` → 把 `Waker` 交给 I/O/通道/定时器 → 就绪时 `wake` → 执行器再次 `poll`。
- **避免忙等**：不应在 `Pending` 上自旋；应依赖事件 + `Waker`（教学里 `wake_by_ref()` 立即再 poll 仅用于理解控制流）。

## 超时与取消安全（Cancel Safety）

- **`select!` / 超时**：被取消的分支可能**未完成**；须保证无半初始化资源、无持锁跨取消。
- **持 `std::sync::MutexGuard` 再 `.await`**：危险——让出时锁未释放，易死锁；用 `tokio::sync::Mutex` 或缩小临界区。

## 与第 1 章衔接

不要在 async 任务里 `std::thread::sleep`；用 `tokio::time::sleep` 等返回 `Pending` 的 API。
