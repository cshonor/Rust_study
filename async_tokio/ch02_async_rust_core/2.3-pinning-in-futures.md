# 2.3 Pinning in Futures（Future 中的 Pin）

## 核心梳理

- **`async fn` 展开**为状态机；跨 `await` 的局部变量可能形成**自引用**（字段指针指向同结构体另一字段）。
- **移动即 UB**：若该状态机被移动（传参、`Vec` 重分配），内部指针仍指向旧地址 → 段错误 / 未定义行为。
- **`Pin<&mut T>`**：约定在 `poll` 期间**不能安全移动 `T`**，从而固定自引用目标的地址。
- **`Unpin`**：无自引用需求的类型（如本仓库 `CounterFuture` 仅 `u32`）可 `Unpin`，在 `Pin` 内仍可用 `get_mut()` 安全改字段。

## 新手要点

| 现象 | 含义 |
|------|------|
| 到处见 `Pin` | 保护编译器生成的 async 状态机 |
| `CounterFuture` 不用复杂 Pin API | 无自引用，教学简化 |
