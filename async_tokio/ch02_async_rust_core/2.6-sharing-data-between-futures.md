# 2.6 Sharing Data Between Futures（Future 之间共享数据）

> Demo：[2.6-sharing-data-between-futures/2.6-sharing-data-try-lock-demo.rs](./2.6-sharing-data-between-futures/2.6-sharing-data-try-lock-demo.rs)

## 核心梳理

- **需求**：多个 Future 读写同一份状态 → 需要 **`Arc`** 共享所有权 + **`Mutex`** 互斥。
- **手写 `poll` 的铁律**：必须用 **`try_lock()`**（或非阻塞路径）；**禁止**在 `poll` 里阻塞式等锁，否则会卡住整个运行时线程上的所有任务。
- **争用表现**：锁忙时返回 `Pending`，待下次 `poll` 再试；两路 `join!` 的完成顺序可能不确定。

## 与 `std::sync::Mutex` + `.await` 的区别

在 `async fn` 里持 `MutexGuard` 再 `.await` 同样危险（让出时仍持锁）。底层 `poll` 路径更要 **try_lock** 或专用异步原语。
