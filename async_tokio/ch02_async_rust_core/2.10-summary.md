# 2.10 Summary（总结）

## 核心梳理

- **任务驱动 + 协作式调度**：在少量 OS 线程上 multiplex 大量 I/O 等待，提升吞吐与响应。
- **Future / poll / Pin / Waker / Context**：构成 Rust 异步的底层词汇；`async/await` 是语法糖，底层仍是状态机与 `poll`。
- **共享状态**：`Arc` + 互斥；手写 `poll` 用 `try_lock`，高级路径用 `tokio::sync::Mutex`。
- **复杂性代价**：取消安全、顺序非确定性、禁止阻塞执行器——需在架构层用队列/Actor 等补齐。
