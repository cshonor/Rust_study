# 2.2 Futures（Future 特型）

> Demo：[2.2-futures/2.2-futures-counter-demo.rs](./2.2-futures/2.2-futures-counter-demo.rs)

## 核心梳理

- **Future**：尚未完成的异步操作的**占位符**；完成后产出 `Output`。
- **惰性**：只创建 Future 而不 `await` / 不 `poll`，体内逻辑往往**不会执行**。
- **`poll`**：同步调用；返回 **`Poll::Ready(T)`** 或 **`Poll::Pending`**。执行器在 `Pending` 时去跑别的任务，而不是空转占满 CPU。

```rust
fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
```

## 计数器 Future（书中主线）

`CounterFuture` 每次 `poll` 将计数 +1；未满 5 次则 `wake_by_ref()` 并返回 `Pending`（教学用「立刻再调度」）；满 5 返回 `Ready`。与书中「理解 Pending/Ready」一致。

## 为什么是轮询而不是回调？

Rust 将 `async` 编译为**状态机** + `poll`，减少回调链上的分配与生命周期负担，贴近零成本抽象（具体仍受 `Box`、动态分发影响）。
