# 6.4 Getting User Input via Callbacks

## 无阻塞键盘输入

书中使用 **`device_query`** 库：在**事件循环**里注册**回调 (Callbacks)**，动态获取键盘输入，而不阻塞异步运行时。

## 渲染重构

将此前「仅温度变化才刷新」的逻辑改为：**收到输入即刷新界面**，交互更跟手。

## 回调地狱 (Callback Hell)

嵌套过多回调会导致：

- 控制流难以追踪；
- 错误处理分散；
- 难以测试与复用。

**对策**：优先 `async/await`、事件总线（§6.5+），或有限深度的回调 + 明确状态机。

## 本仓库 Demo 范围

`6.4-user-input-via-callbacks/` 仅保留**同步回调最小例**（与书中「操作完成再调 callback」同一模式）。完整 `device_query` 需自行 `cargo add device_query` 并按书接入。

```rust
fn perform_operation_with_callback<F>(callback: F)
where
    F: Fn(i32),
{
    let result = 42;
    callback(result);
}
```
