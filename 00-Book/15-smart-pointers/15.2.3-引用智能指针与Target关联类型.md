## 第一部分：借用 & 解引用（都能用 `*`、`.`，底层不一样）

### 两个基础概念

| | 普通引用 `&T` / `&mut T` | 智能指针 `Box` / `Rc` / `Arc` / … |
|---|---------------------------|-------------------------------------|
| **是什么** | 语言原生的**借用标记** — 栈上地址，不拥有数据 | **拥有**堆数据所有权的**结构体**（堆指针 + 元数据/计数） |
| **所有权** | 无，仅临时借 | 有（独占或共享） |

---

### 1. 解引用 `*` 分别干了啥

#### ① 普通引用 — 原生剥一层

```rust
let s = String::from("hello");
let r: &String = &s;
// *r 若 T 非 Copy 会 move — 日常多用 . 或 &*r 借看
let x = 42;
let rx = &x;
assert_eq!(*rx, 42); // 原生：* 直接穿透引用层
```

- `&T` 是语法层「地址包裹」，**只有一层**。
- **不经过** `Deref` trait。

#### ② 智能指针 — 走 `Deref` trait

```rust
let b = Box::new(String::from("hello"));
let inner: String = *b; // *b ⇔ *(b.deref()) → 先 &String 再 move 出 String
```

```text
Box 结构体存堆地址
  → *b 调用 Deref::deref(&self)
  → &Box<String> 变成 &String（Target 类型）
  → 再 * 得到 String（若 move）
```

| 对比 | 普通 `&T` | `Box<T>` 等 |
|------|-----------|-------------|
| `*` 来源 | **语言原生** | **`impl Deref`** 语法糖 |
| 本质 | 借用标记 | 自定义结构体 + trait |

→ 速记 · unsafe：[15.2.2](./15.2.2-Deref解引用与unsafe速记.md)

---

### 2. 点号 `.` 自动解引用 — **两条路线**（最易混）

方法调用 / 传参时，编译器**循环**解包。包装是 **`&T`** → 语言原生剥 `&`（**无 `Deref`**）；包装是 **Box / Rc / …** → 必须 **`impl Deref`**。

```rust
let s = String::from("hi");
let r3 = &&&s;
r3.len(); // 原生：&&&String → … → String → len

let b = Box::new(String::from("hi"));
b.len();  // Deref：Box → &String → 剥 & → len
```

```rust
// 没 impl Deref 的 struct — mb.len() 编译失败
struct RawWrap<T>(T);
```

混搭多层链 → [15.2.2 §一.3 `&&Box<Rc<String>>`](./15.2.2-Deref解引用与unsafe速记.md#3-多层混搭boxrcstringlen-完整链) · demo：`cargo run -- dot-paths`

### 3. 对照表

| 类型 | 所有权 | `*` / `.` 来源 | 用途 |
|------|--------|----------------|------|
| `&T` | 无，只读借 | 原生引用 | 临时只读，生命周期约束 |
| `&mut T` | 无，排他借 | 原生引用 | 临时可变借 |
| `Box<T>` | 独占堆上 | `Deref` / `DerefMut` | 堆分配 |
| `Rc<T>` / `Arc<T>` | 共享 | `Deref` | 多所有者 |
| `RefCell<T>` | 独占 + 内部可变性 | `Deref`（通过 `.borrow()` 守卫时） | 运行时借用检查 |

### 第一部分一句话

1. 普通引用：`*` 与 `.` 剥 `&` 是**语言原生** — **不靠 `Deref`**。  
2. 智能指针：**必须 `impl Deref`** 才参与 `*` 语法糖与点调用解包。  
3. 标准库已全实现 — 手感一致，**机制不同** → [15.2.2 §一](./15.2.2-Deref解引用与unsafe速记.md)

## 第二部分：`trait` / `impl` 里的 `type` — 关联类型

### 典型写法

```rust
trait MyTrait {
    type Target; // 类型占位符 — 定义时不指定具体类型
    fn get(&self) -> Self::Target;
}

impl MyTrait for i32 {
    type Target = String; // impl 时把占位填成具体类型
    fn get(&self) -> Self::Target {
        self.to_string()
    }
}
```

| 位置 | 写法 | 含义 |
|------|------|------|
| **trait 内** | `type Target;` | 声明：此 trait 绑定一个叫 `Target` 的类型，**以后再定** |
| **impl 内** | `type Target = SomeType;` | **落地**：此 `(Self, Trait)` 的 Target 就是 `SomeType` |
| **使用** | `Self::Target` / `<i32 as MyTrait>::Target` | 当前 impl 填好的那个类型 |

→ 系统讲解：[RFR Ch02 §06.1 关联类型 vs 泛型](../../02-RFR/Chapter-02-Types/06-1-associated-vs-generic.md)

---

### 1. 关联类型 vs 泛型 trait

#### 方案 A：泛型 `<T>` — 可多次 impl

```rust
trait MyTrait<T> {
    fn get(&self) -> T;
}
impl MyTrait<String> for i32 {}
impl MyTrait<i32> for i32 {}  // 同一类型，不同 T，多份 impl
```

#### 方案 B：关联 `type` — 通常只 impl 一次

```rust
trait MyTrait {
    type Target;
    fn get(&self) -> Self::Target;
}
impl MyTrait for i32 {
    type Target = String; // i32 对 MyTrait 只能有一份，Target 唯一
}
```

| | **关联类型** | **泛型参数** |
|---|-------------|-------------|
| 同一 `Self` + trait | **一种**配套类型 | **多种** `T` / `Rhs` |
| 典型 | `Iterator::Item`、`Deref::Target` | `PartialEq<Rhs>`、`From<T>` |

---

### 2. 标准库联动：`Deref::Target`（串起第一部分）

```rust
pub trait Deref {
    type Target: ?Sized;           // 关联类型：解引用后是什么
    fn deref(&self) -> &Self::Target;
}

impl<T> Deref for Box<T> {
    type Target = T;              // Box 剥完包装 → 里面的 T
    fn deref(&self) -> &T {
        &*self.as_ref()            // 标准库实现略
    }
}
```

**串联记忆**：

```text
智能指针能 * 和 . 自动解引用
  ← 因为实现了 Deref
  ← Deref 用 type Target 规定「剥完是什么类型」
  ← Box<T> 写 type Target = T
  ← 所以 *Box<T> 最终面向 T
```

普通 `&T` **不走**这套 trait + `Target` — 只是语言原生借用。

---

### 3. 何时用关联类型 vs 泛型

| 场景 | 选用 |
|------|------|
| 一种 impl **固定唯一**配套类型（Box 解引用永远是内部 `T`） | **关联类型** |
| 同一类型要对**多种**外部类型 impl（`PartialEq<u32>`、`PartialEq<u64>`） | **泛型** |

### 第二部分极简总结

1. trait 里 `type Target;` = 开类型空位。  
2. impl 里 `type Target = X;` = 填空。  
3. `Self::Target` = 当前 impl 的 X。  
4. **`Deref::Target`** = 智能指针剥完露出的类型。

---

## 整体复盘（两个问题合在一起）

```text
普通 &T     →  原生借用，* 剥一层，无 Deref trait
智能指针     →  结构体 + impl Deref { type Target = …; fn deref … }
              →  * / . 的语法糖根源在这里
关联类型     →  trait 里 type 占位，impl 里落地
              →  Deref::Target 决定 *Box<T> 面向 T
```
