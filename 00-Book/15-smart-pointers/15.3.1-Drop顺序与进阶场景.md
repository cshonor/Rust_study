## 一、同一作用域：LIFO（后进先出）

### 1. 示例：`Tag` 与 `self.0` 是什么？

```rust
struct Tag(&'static str);  // 元组结构体：一个无名字段

impl Drop for Tag {
    fn drop(&mut self) {
        println!("drop {}", self.0);
    }
}
```

#### `self.0` 到底是什么？

| 概念 | 说明 |
|------|------|
| **`struct Tag(&'static str)`** | **元组结构体** — 只有**一个**字段，类型 `&'static str`，**没有字段名** |
| **`.0`** | 访问**第 0 个**（唯一）字段的语法；多字段则 `.1`、`.2`… |
| **`drop(&mut self)` 里的 `self`** | 正在被销毁的 `Tag` 实例，类型 **`&mut Tag`** |
| **`self.0`** | 该实例里包裹的 **`&'static str`**（如 `"a"`、`"b"`） |

若改成**命名字段**的普通 struct，则写 `self.value`：

```rust
struct Tag { value: &'static str }

impl Drop for Tag {
    fn drop(&mut self) {
        println!("drop {}", self.value);
    }
}
```

> 元组 struct 在 Drop / Deref 演示里很常见（`MyBox(T)` → `self.0`）→ [15.2 §二](./15.2-通过Deref将智能指针当作引用.md#二最简示例mybox) · [15.1.1 `&**self`](./15.1.1-Box的Deref与双层星号.md)

### 2. LIFO 销毁顺序

```rust
{
    let a = Tag("a"); // 先创建
    let b = Tag("b"); // 后创建
}
// 输出：drop b → drop a
```

**原因**：局部变量在栈上按创建顺序排列，销毁时**逆序** —— 与 C++ 局部对象析构一致。

→ demo：`cargo run -- lifo` · `cargo test drop_order_lifo`

## 二、嵌套类型与容器

### 1. 带 `Drop` 的嵌套 struct

```rust
struct Inner;
struct Outer(Inner);

impl Drop for Inner { fn drop(&mut self) { println!("Inner"); } }
impl Drop for Outer { fn drop(&mut self) { println!("Outer"); } }

let _o = Outer(Inner);
// 输出：Outer → Inner
```

**规则**：外层类型的 `drop` **先**执行（此时字段仍有效，可访问 `self.xxx`）；`drop` 返回后，字段按**声明逆序**依次 drop。

→ 与 [15.3 §二](./15.3-使用Drop运行清理代码.md#二默认-drop-vs-手写-drop替换不是叠加) 一致：**先你的外部资源逻辑，再字段内存**。

### 2. 无自定义 `Drop` 的 struct

字段按**声明逆序** drop：

```rust
struct Pair { first: Tag, second: Tag }
// drop second → drop first
```

### 3. `Vec<T>` / `Box<[T]>`

- **元素**：从**第一个**到**最后一个**依次 `drop`（与数组一致）
- **容器自身**：元素全部 drop 后，释放堆缓冲区

```rust
struct E(u32);
impl Drop for E { fn drop(&mut self) { println!("E({})", self.0); } }

let v = vec![E(1), E(2), E(3)];
// drop E(1) → E(2) → E(3) → 释放 Vec 堆内存
```

### 4. `Box<T>`

- `drop` 顺序：先 `T::drop`，再释放 `Box` 堆内存
- `*box` move 出 `T` 后，空壳 `Box` 仍会在出作用域时 drop（不再重复 drop 已移走的 `T`）

→ demo：`cargo run -- nested` · `cargo test vec_drop_reverse`

## 三、`Copy` 与自定义 `Drop` 互斥

```rust
struct Handle(i32);

impl Drop for Handle {
    fn drop(&mut self) { /* 关 fd */ }
}

// impl Copy for Handle {}  // ❌ 不能：Copy 类型不能有自定义 Drop
```

| | `Copy` | 自定义 `Drop` |
|---|--------|---------------|
| 旧值 after 赋值 | 仍有效 | 各自独立 drop |
| 适用 | 纯数据、无资源 | 管 fd/锁/堆所有权 |

**设计口诀**：要管资源 → 别 `Copy`；要 `Copy` → 别手写 `Drop`。

## 四、`ManuallyDrop`（底层 / 高性能）

`std::mem::ManuallyDrop<T>` 包裹后，**不会**在出作用域时自动调用 `T::drop`。

```rust
use std::mem::ManuallyDrop;

struct Resource;
impl Drop for Resource {
    fn drop(&mut self) { println!("Resource freed"); }
}

let mut slot = ManuallyDrop::new(Resource);
// 出作用域前不会打印

let taken = unsafe { ManuallyDrop::take(&mut slot) }; // take 为 unsafe：须保证不 double-drop // 取出 Resource
drop(taken); // 此时才手动触发 Drop
// slot 已是空壳，不再 drop Resource
```

**典型场景**：

| 场景 | 为何需要 |
|------|----------|
| FFI / 自管内存 | 自己决定何时 `free` |
| 内存池 / arena | 批量回收，单块不单独 drop |
| 无锁数据结构 | 避免并发路径上意外 drop |
| `Pin` + 自引用 | 控制销毁时机 |

⚠️ **泄漏风险**：忘记 `take` + `drop` → 资源永不释放。只在明确需要时用。

→ demo：`cargo run -- manual`

## 五、panic 与 `Drop`

| panic 策略 | `Drop` 行为 |
|------------|-------------|
| **`unwind`（默认）** | 栈展开，逐层 drop 局部变量 → [9.1](../09-error-handling/9.1-panic-与不可恢复的错误.md) |
| **`abort`** | 进程直接终止，**不**跑 `Drop` |

### `Drop` 里再 panic

若 `drop` 执行中又 `panic!` → **double panic → abort**（无法安全展开）。  
`Drop` 实现应保持简单，避免 panic。

## 六、常见坑

| 现象 | 原因 | 修法 |
|------|------|------|
| `explicit destructor calls not allowed` | 写了 `x.drop()` | 用 `mem::drop(x)` |
| 锁持有过久 | guard 作用域太大 | 缩小 `{}` |
| 释放失败无处报告 | `drop` 无返回值 | 提供 `release() -> Result` |
| 异步关闭 | `Drop` 不能 `.await` | 专门 shutdown 模式 |
| 在 `drop` 里手动 `drop(field)` | 编译器已会 drop 字段 | **删掉**；只写外部资源清理 |

### 不能主动调 vs 可以 `mem::drop`

```rust
// x.drop();        // ❌ trait 方法，对象仍在作用域
mem::drop(x);       // ✅ move 走 x，生命周期结束，再 drop
```

---

## 七、一句话总结

1. **LIFO**：同作用域后建先毁；嵌套 struct 外层 `drop` 先于字段。
2. **容器**：`Vec` 元素**正序** drop，再释堆。
3. **`Copy` ⊥ 自定义 `Drop`** —— 类型设计二选一。
4. **`ManuallyDrop`**：禁用自动 drop，自管销毁（FFI/池化/HFT 进阶）。
5. **panic unwind** 仍 drop；`abort` 不 drop；`Drop` 里别 panic。
