# 15.5 `RefCell<T>` 与内部可变性 · `Rc` 组合

← [15.4 Rc](./15.4-Rc引用计数智能指针.md) · [15.6 循环引用/Weak](./15.6-引用循环与Weak.md) · demo：`15.5-refcell-demo/`

> **由浅入深**：`Rc` 只读 → 内部可变性 → `Rc<Cell>` / `Rc<RefCell>` → 循环泄漏 → [15.6 Weak](./15.6-引用循环与Weak.md)

```bash
cargo run              # Rc<RefCell> 共享尾列表修改
cargo run -- cell      # Rc<Cell<i32>>
cargo run -- refcell   # Rc<RefCell<String>>
cargo test
```

---

## 核心结论（先背）

1. **`Rc<T>` 共享但只读** —— 要**共享 + 改**，必须加 **内部可变性**（`Cell` / `RefCell`）。
2. **简单 `Copy` 值** → `Rc<Cell<T>>`；**复杂类型 / 要引用** → **`Rc<RefCell<T>>`**（单线程共享可变标准写法）。
3. **运行时检查** —— `RefCell` 违反借用规则 → **panic**（编译期不报错）。
4. **`Rc` 互指** → 强计数不归零 → **泄漏** → 用 **`Weak`** → [15.6](./15.6-引用循环与Weak.md)

---

## 一、前置：为什么需要内部可变性？

| 类型 | 所有权 | 能否在 `&T` 下修改？ |
|------|--------|----------------------|
| `Box<T>` | 独占 | ✅（唯一 owner 时 `mut`） |
| `Rc<T>` | 共享 | ❌ 多个 `&T`，不能同时 `&mut` |
| `RefCell<T>` | 单一 | ✅ **运行时** borrow |
| `Rc<RefCell<T>>` | 共享 + 内部可变 | ✅ 多 owner 轮流 `borrow_mut` |

**内部可变性**：在持有**不可变引用**时仍能改内部数据 —— 靠类型内部 `unsafe` + **运行时**借用检查。

---

## 二、两个工具：`Cell` vs `RefCell`

| 特性 | `Cell<T>` | `RefCell<T>` |
|------|-----------|--------------|
| 适用类型 | **必须 `Copy`**（`i32`、`bool`…） | 任意类型 |
| 取值 | `.get()` **复制**出值 | `.borrow()` → `Ref<T>` |
| 改值 | `.set(v)` | `.borrow_mut()` → `RefMut<T>` |
| 借用规则 | **无借用**，不会 panic | 运行时检查，冲突 → **panic** |
| 拿内部引用 | ❌ | ✅ |

**口诀**：简单数字 → `Cell`；字符串/结构体/要引用 → `RefCell`。

---

## 三、组合 1：`Rc<Cell<T>>`

共享 + 修改简单 `Copy` 值，**无 borrow**：

```rust
use std::rc::Rc;
use std::cell::Cell;

let num = Rc::new(Cell::new(10));
let n1 = Rc::clone(&num);
let n2 = Rc::clone(&num);

n1.set(20);
println!("{}", n2.get()); // 20
```

→ demo：`cargo run -- cell`

---

## 四、组合 2：`Rc<RefCell<T>>`（最常用）

```rust
use std::rc::Rc;
use std::cell::RefCell;

let s = Rc::new(RefCell::new(String::from("hello")));
let s1 = Rc::clone(&s);
let s2 = Rc::clone(&s);

s1.borrow_mut().push_str(" world");
println!("{}", s2.borrow()); // hello world
```

| 方法 | 含义 |
|------|------|
| `.borrow()` | 不可变借用（可多个） |
| `.borrow_mut()` | 可变借用（**同一时刻仅一个**） |
| 冲突 | 运行时 **panic** |

### 共享尾链表 + 改共享结点（Book 例）

```rust
let value = Rc::new(RefCell::new(5));
let a = Rc::new(Cons(Rc::clone(&value), Rc::new(Nil)));
// b、c 共享 a …
*value.borrow_mut() += 10; // 所有指向 value 的结点看到 15
```

→ demo：`cargo run`

## 五、`Box` / `Rc` / `RefCell` 选型（单线程）

| 类型 | 所有者 | 检查时机 | 可变？ |
|------|--------|----------|--------|
| `Box<T>` | 1 | 编译期 | ✅ |
| `Rc<T>` | 多 | 编译期 | ❌ 只读 |
| `RefCell<T>` | 1 | **运行期** | ✅ |
| `Rc<Cell<T>>` | 多 | 无 borrow | ✅ Copy 值 |
| `Rc<RefCell<T>>` | 多 | **运行期** | ✅ |

多线程 → `Arc` + `Mutex`/`RwLock` → [16.3](../16-fearless-concurrency/16.3-共享状态并发.md)

`Messenger::send(&self, …)` 不能改成 `&mut self`，测试需记录消息 → `RefCell<Vec<String>>` + `borrow_mut().push(...)`。

## 七、一句话总结

1. **要共享** → `Rc`；**要不可变引用下修改** → `Cell`/`RefCell`。
2. **`Rc<RefCell<T>>`** = 单线程共享可变标准组合。
3. **循环 `Rc`** → 泄漏 → **`Weak`** → [15.6 §二–§三](./15.6-引用循环与Weak.md)
