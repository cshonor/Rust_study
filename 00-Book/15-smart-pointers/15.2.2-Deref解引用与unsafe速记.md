## 一、自动解引用（`Deref` / `DerefMut`）核心规则

### 1. 触发场景

| 场景 | 行为 |
|------|------|
| **函数传参** | 需要 `&T` / `&mut T`，传入 `&Box` / `&Rc` / `&Arc` 等 — 编译器自动补 `deref()` 适配类型 |
| **`.` 点语法** | 调用方法、改结构体字段 — 编译器**多层**自动解包，日常**不用手写 `*`** |

```rust
let mut b = Box::new(10);
*b = 20;        // 整体赋值：见 §二，须手动 *
b.abs();        // 方法调用：自动 deref → i32 的方法

let mut guard = mutex.lock().unwrap();
*guard += 1;    // 整体改值
guard.len();    // 若 Target 有方法，点语法自动 deref
```

### 2. 本质

编译器自动调用 `deref()` / `deref_mut()` — **安全语法糖**，**完全不需要 `unsafe`**。

```text
*y  ⇔  *(y.deref())     // 不可变
&mut *y  ⇔  y.deref_mut()  // 可变链
```

### 3. 限制

| 有自动解引用 | 无自动解引用 |
|--------------|--------------|
| `&T` / `&mut T` | |
| 实现 `Deref` / `DerefMut` 的智能指针、锁守卫 | |
| `String` → `str` 等标准库 coercion | |
| | **裸指针** `*const T` / `*mut T` — 无 `Deref` coercion |

## 二、`&mut T` 修改值：何时要手动 `*`？

### ✅ 不用手动解引用（绝大多数日常）

**`.` 改字段、调方法** — 点语法自带自动解引用：

```rust
struct Point { x: i32, y: i32 }

let mut p = Box::new(Point { x: 0, y: 0 });
p.x = 10;           // ✅ 自动 deref_mut → 改字段
p.y += 1;           // ✅

let mut v = Box::new(vec![1, 2]);
v.push(3);          // ✅ 方法调用自动 &mut Vec
(*v)[0] = 99;       // 索引也可写，但 push 等用 . 更常见
```

### ⚠️ 必须手动 `*`（整体赋值）

给可变引用**整体覆盖赋值**时，编译器分不清你是改「引用本身」还是改「引用指向的数据」：

```rust
let mut b = Box::new(10);

// ❌ b = 20;           // 类型不对：不能把 i32 赋给 Box<i32>
// ✅ 整体替换堆上值：
*b = 20;               // 必须 * — 改的是 i32，不是 Box 变量本身

let mut r = &mut 42;
*r = 100;              // 普通 &mut 同理
```

| 写法 | 含义 |
|------|------|
| `ref.field = v` | 改**内部字段** — 点语法自动 deref |
| `ref.method()` | 调**内部方法** — 自动 deref |
| `*ref = new_value` | **整体替换**指向的值 — 必须 `*` |

## 三、`unsafe` 与解引用的关系（纠正误区）

### 1. 安全路径 — 不需要 `unsafe`

| 操作 | 是否需要 `unsafe` |
|------|-------------------|
| `&T` / `&mut T` 解引用（含手动 `*ref`） | ❌ 不需要 |
| `Box` / `Rc` / `Arc` / 锁守卫 的自动或手动 `*` | ❌ 不需要 |
| `Deref` / `DerefMut` coercion | ❌ 不需要 |

### 2. 裸指针 — 解引用必须 `unsafe`

```rust
let x = 42;
let p: *const i32 = &x;   // ✅ 创建裸指针：Safe

unsafe {
    let v = *p;           // ✅ 裸指针手动 *：必须 unsafe
}
```

只有 **`*const T` / `*mut T` 的手动 `*` 解引用** 属于 unsafe 五类能力之一。

→ [RFR Ch09 §01 unsafe 五类](../../02-RFR/Chapter-09-Unsafe-Code/01-unsafe-keyword.md) · [§02 裸指针](../../02-RFR/Chapter-09-Unsafe-Code/02-raw-pointers.md)

### 3. 常见误区

| 误区 | 正解 |
|------|------|
| 「解引用都要 unsafe」 | **只有裸指针**手动 `*` 要 unsafe |
| 「只有裸指针才需要 unsafe」 | ❌ 错 — 还有：调用 `unsafe fn`、实现 `unsafe trait`、读写 `static mut`、访问 `union` 等 |
| 「智能指针 `*` 和裸指针一样危险」 | 智能指针 `Deref` 在借用检查保护下 — **Safe** |

---

## 四、一句话速记

1. **点号**改字段 / 调方法 → 自动 deref，**不用 `*`**。  
2. **整体赋值**给可变借用 → **必须 `*ref = 新值`**。  
3. **安全引用 + 智能指针**解引用 → 与 **`unsafe` 无关**。  
4. **裸指针**手动 `*` → **必须 `unsafe`**；且 unsafe 不限于裸指针。
