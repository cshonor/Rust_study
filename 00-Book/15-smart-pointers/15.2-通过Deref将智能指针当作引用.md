## 一、基础定义

`Deref` / `DerefMut` 是 Rust **解引用重载** trait：让自定义类型支持 `*` 解引用，并参与**自动解引用强制转换**（Deref coercion）。

```rust
// 不可变解引用
pub trait Deref {
    type Target: ?Sized;
    fn deref(&self) -> &Self::Target;
}

// 可变解引用（必须先实现 Deref）
pub trait DerefMut: Deref {
    fn deref_mut(&mut self) -> &mut Self::Target;
}
```

**关键点：**

1. `DerefMut` **必须**先实现 `Deref`
2. `deref` / `deref_mut` **永远返回引用**，不会转移所有权
3. `Target` **唯一**（与 `AsRef` 可多目标不同）→ coercion 无歧义 → [ER Item 8 §2](../../01-ER/Chapter-01-Types/Item-08-references-pointers/README.md#deref-强制转换)

## 二、最简示例：`MyBox`

元组结构体默认**不能**用 `*`（`cannot be dereferenced`）。实现 `Deref` 后：

```rust
use std::ops::Deref;

struct MyBox(String);

impl Deref for MyBox {
    type Target = String;
    fn deref(&self) -> &Self::Target {
        &self.0
    }
}

fn main() {
    let b = MyBox("hello".to_string());

    // 手动解引用：*b ⇔ *(b.deref())
    let s: &String = &*b;
    println!("{s}");

    // 自动解引用强制转换（Deref coercion）
    fn take_str(s: &str) {}
    take_str(&b); // &MyBox → &String → &str，编译期链式转换
}
```

**书写 `*y` 时 Rust 展开为：**

```text
*y  ⇔  *(y.deref())
```

`deref` 返回**引用**而非按值 `T`，避免把内部值从 `self` 移出；最外层 `*` 再得到目标类型的值/引用。

→ demo：`cargo run` §MyBox · 无 coercion 时的冗长写法 `hello(&(*m)[..])`

## 三、两大核心能力

### 1. 手动解引用 `*`

对实现 `Deref` 的类型，`*T` 等价于 `*(T.deref())`。

```rust
let b = MyBox(String::from("test"));
let inner: String = (*b).clone(); // deref 得 &String，再 clone 得 String
// Box<String> 上 *box 则直接 move 出内部 String（见 15.2.1 §一）
```

### 2. 自动解引用强制转换（高频考点）

**场景**：传引用、**方法调用**时，编译器自动逐层 `deref`，直到类型匹配。

| 规则 | 条件 | 转换 |
|------|------|------|
| 不可变 | `T: Deref<Target = U>` | `&T` → `&U` |
| 可变 | `T: DerefMut<Target = U>` | `&mut T` → `&mut U` |
| 降级 | `T: Deref<Target = U>` | `&mut T` → `&U`（可变→不可变 ✅；反向 ❌） |

可**连续多层**：`&MyBox<String>` → `&String` → `&str`。

#### 方法调用自动补 `*`

```rust
let b = MyBox(String::from("abc"));
b.len();
// 编译器展开 ≈ (*b).len() 或 String::len(&*b)
```

因此 `Box<T>`、`Rc<T>`、`Arc<T>`、**锁守卫**可直接调用内部类型方法。

## 四、标准库典型实现（天天碰）

| 类型 | `Target` | 用途 |
|------|----------|------|
| `Box<T>` | `T` | 堆指针，最基础智能指针 |
| `Rc<T>` / `Arc<T>` | `T` | 共享所有权，透明读内部 |
| `String` | `str` | **`&String` → `&str`**，最高频 coercion |
| `Vec<T>` | `[T]` | `&Vec` → `&[T]` 切片视图 |
| `MutexGuard<T>` | `T` | 锁 RAII，**Deref/DerefMut** 暴露数据 |
| `RwLockReadGuard<T>` / `WriteGuard<T>` | `T` | 读写锁守卫 |

### 锁守卫（并发路线前置）

```rust
use std::sync::Mutex;

let m = Mutex::new(10);
let guard = m.lock().unwrap(); // MutexGuard<i32>
println!("{}", *guard);        // Deref → &i32
// guard 离开作用域 Drop → 自动解锁
```

→ 详解：[16.3 共享状态并发](../16-fearless-concurrency/16.3-共享状态并发.md) · **Atomics and Locks** 路线：守卫 + RAII + `Deref` 是标准模式

→ demo：`cargo run -- guards`

## 五、重要规则 & 坑（Effective Rust）

### 1. 解引用与所有权

- `deref()` 返回**引用** → `*&T` 得到 `T`（复制或移动取决于 `T`）
- `Box<T>`：`*box` **move** 出堆上 `T`，盒子随后通常不可再用
- coercion **只借看**，不转移所有权

### 2. 自动解引用**只发生在引用上下文**

| 会触发 | 不会触发 |
|--------|----------|
| 函数/方法传参 `&T` | 赋值 `let x = smart_ptr` |
| 方法调用 `smart_ptr.method()` | 多数模式匹配 |
| 部分索引场景 | 期望值类型却给智能指针 |

### 3. 不要滥用 Deref（Effective Rust 核心）

> **只有智能指针 / 守卫 / 透明包装器才应实现 `Deref`。**  
> 普通业务结构体、领域模型**不要**随便 `Deref` 到内部类型。

| ✅ 适合 | ❌ 不适合 |
|---------|----------|
| `Box` / `Rc` / 锁守卫 | 用户实体、配置 struct |
| 透明缓冲区包装 | 带校验的 DB 封装（会绕过 API） |

→ 反例 demo：[ER `LeakyDb`](../../01-ER/Chapter-01-Types/Item-05-type-conversions/demo/src/over_deref.rs) · [Item 6 newtype](../../01-ER/Chapter-01-Types/Item-06-newtype-pattern/README.md)

**更稳妥**：显式方法（`get` / `insert`）、`AsRef`、或 `Deref` 到**更窄**视图而非整个内部容器。

### 4. `Deref` 与生命周期

```rust
fn deref(&self) -> &Self::Target;
```

返回引用生命周期与 `&self` 绑定，**自动继承**，绝大多数场景无需手写。

### 5. `Deref` vs `AsRef`

| | `Deref` | `AsRef` |
|---|---------|---------|
| 触发 | 编译器**自动** coercion | 调用方**显式** `.as_ref()` |
| 目标 | **唯一** `Target` | 可实现多个 `AsRef<U>` |
| 场景 | 智能指针透明访问 | API 多视图设计 |

→ [Item 5 §5](../../01-ER/Chapter-01-Types/Item-05-type-conversions/README.md) · [item-05-06 `as_ref_vs_into`](../../01-ER/Chapter-01-Types/Item-05-type-conversions/demo/src/as_ref_vs_into.rs)

## 七、一句话总结（背诵版）

1. `Deref` / `DerefMut` = **解引用重载**，支持 `*` 与自动 coercion。
2. 核心价值：智能指针、锁守卫、`String`→`str` **透明访问**内部数据。
3. 编译器自动解引用：**传参、方法调用**最常用；编译期完成，零运行时开销。
4. 原则：**只给智能指针类实现**；业务 struct 别乱用（Effective Rust 重点）。
5. 并发、锁、自定义内存包装（HFT/底层方向）重度依赖这套机制。
