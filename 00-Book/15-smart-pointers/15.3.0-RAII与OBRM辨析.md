## 核心关系（先背）

| 术语 | 含义 |
|------|------|
| **RAII** | **R**esource **A**cquisition **I**s **I**nitialization — 经典范式：构造拿资源，析构放资源 |
| **OBRM** | **O**wnership-**B**ased **R**esource **M**anagement — Rust/Nomicon：**带严格所有权规则的 RAII** |

```text
RAII = 思想（C++ 起源）
OBRM = Rust 用所有权 + 借用 + 编译期检查 锁死的 RAII 落地形态
```

日常说「Rust 的 RAII」，本质指的就是 **OBRM**。

---

## 一、RAII：经典基础范式

### 全称

**Resource Acquisition Is Initialization** — 资源获取 = 初始化。

### 核心逻辑（C++）

1. **构造**时申请资源（堆、文件、锁、Socket…）  
2. **析构**（出作用域）自动释放 — 不用手动 `free` / `close` / `unlock`  
3. 资源生命周期 **绑定对象生命周期**

### C++ 极简例子

```cpp
struct FileGuard {
    FILE* fp;
    FileGuard(const char* path) { fp = fopen(path, "r"); }
    ~FileGuard() { if (fp) fclose(fp); }
};

void demo() {
    FileGuard f("test.txt");
} // 出作用域 → 析构 → 关文件
```

| 优点 | 缺点（无所有权体系时） |
|------|------------------------|
| 确定性释放、无 GC、性能高 | 浅拷贝 → **double free**；悬垂 → **use-after-free** |

---

## 二、OBRM：Rust 升级版

### 全称

**Ownership-Based Resource Management** — Nomicon 明确：**规范化、加了所有权约束的 RAII**。

### 在 RAII 上追加的三条（= 所有权三规则）

| # | OBRM 规则 | 对应 Rust |
|---|-----------|-----------|
| 1 | 资源绑定**唯一所有者** | 同一时刻一个变量负责 drop |
| 2 | 所有权**可转移**、默认**不可浅拷贝** | move 语义；非 `Copy` 禁止隐式复制 |
| 3 | 所有者出作用域 → **`Drop` 释放** | `impl Drop { fn drop(&mut self) }` 自动调用 |

### Rust 文件示例（对比 C++）

```rust
use std::fs::File;
use std::io;

fn main() -> io::Result<()> {
    let f = File::open("test.txt")?; // 获取资源，f 是唯一所有者
    Ok(())
} // f 出作用域 → Drop → 关闭 fd
```

### 智能指针全是 OBRM 实现

| 类型 | OBRM 语义 |
|------|-----------|
| `Box<T>` | 独占；出作用域 Drop 释堆 |
| `Rc<T>` / `Arc<T>` | 共享；计数归零才 Drop |
| `MutexGuard` | 锁 RAII；Guard Drop 解锁 |
| `Vec` / `String` | 堆缓冲 OBRM |

## 三、RAII vs OBRM 对照

| 维度 | RAII（C++ 原版） | OBRM（Rust） |
|------|------------------|--------------|
| 核心 | 构造拿、析构放 | 继承 RAII + **唯一所有权** |
| 拷贝 | 可浅拷贝，多对象管一份资源 | 编译期禁止；只能 move |
| 安全 | 运行时才暴雷 | **编译期**拦违规 |
| 释放 | 析构函数 | `Drop` trait |
| 借用 | 无规范 | `&T` / `&mut T` 规则 |
| 与 GC | 无关（确定性释放） | 无关 |

**一句话**：RAII 是思想；OBRM 是 Rust 把 RAII **锁死、规范化、编译期强制安全**的形态。

## 四、与借用、Deref、智能指针串联

```rust
let b = Box::new(100); // b：所有者，OBRM 管控堆内存
let r = &*b;           // 借用访问；r 消失 **不会** Drop
// 只有 b 出作用域才 Drop 释放
```

| 机制 | 作用 | 是否改变所有权 / 释放时机 |
|------|------|---------------------------|
| **所有权 + Drop** | 谁负责、何时释放 | ✅ 核心 |
| **`&T` 借用** | 临时只读/可变访问 | ❌ 不转移、不触发所有者 Drop |
| **`Deref` / `.`** | 像引用一样访问内部 | ❌ 只访问数据，**不**改变归属 |

→ 点号两条路线：[15.2.2 §一](./15.2.2-Deref解引用与unsafe速记.md)

## 五、常见误区

| 误区 | 正解 |
|------|------|
| RAII = GC | ❌ — RAII/OBRM 是**作用域结束立刻**确定性释放 |
| Rust 只有 OBRM 没有 RAII | OBRM **就是** Rust 语境下的 RAII 全称 |
| 借用多了会提前 Drop | ❌ — **只有所有者**出作用域才 Drop；`&` 再多也不释放 |
| `Deref` 会把所有权转走 | ❌ — 解引用只是访问；move 才转移所有权 |

## 七、一句话速记

1. **RAII**：构造拿资源，析构放资源 — 通用模式。  
2. **OBRM**：RAII + **唯一所有权 + move + 编译检查** — Rust 内存/文件/锁/智能指针全靠它。  
3. **借用、`Deref`**：访问语法 — **不改变** OBRM 的归属与释放时机。
