## 核心结论（先背）

1. **`Box<T>`**：**独占所有权**，move 后原变量失效；**不能**多变量共享同一块堆数据。
2. **`Rc<T>`**：**共享所有权**（引用计数），`Rc::clone` 只 **+1 计数、不深拷贝**；计数归 **0** 才 `Drop` 释堆。
3. **`Rc` 单线程**；多线程用 **`Arc`** → [16 章](../16-fearless-concurrency/)。默认**只读共享**，要改数据 → **`Rc<RefCell<T>>`** → [15.5](./15.5-RefCell与内部可变性.md)。

## 一、`Box<T>`：独占，不能共享

```rust
let a = Box::new(10);
let b = a;        // 所有权 move 给 b
// println!("{a}"); // ❌ a 已失效
```

| 要点 | 说明 |
|------|------|
| **所有权唯一** | 同一时间只有一个所有者 |
| **非 `Copy`** | 不能按值复制，只能 move |
| **典型场景** | 递归 Cons 节点、大数据上堆、独占资源 → [15.1 Box](./15.1-使用Box指向堆上的数据.md) |

**为何共享尾链表不行？** `Cons(3, Box::new(a))` 会把 `a` **move** 进 `b`，无法再让 `c` 共享同一个 `a`。

---

## 二、`Rc<T>`：共享所有权

```rust
use std::rc::Rc;

let a = Rc::new(10);
println!("count: {}", Rc::strong_count(&a)); // 1

let b = Rc::clone(&a);  // 计数 +1，不深拷贝数据
println!("count: {}", Rc::strong_count(&a)); // 2

let c = Rc::clone(&a);  // 3
println!("a={a}, b={b}, c={c}"); // 三者同时可用

{
    let d = Rc::clone(&a);
    println!("scope内 count: {}", Rc::strong_count(&a)); // 4
}
println!("d drop 后 count: {}", Rc::strong_count(&a)); // 3
// a、b、c 都出作用域 → 计数 0 → 释放堆
```

### 关键语义

- **`Rc::clone(&a)`**：复制**指针** + **强引用计数 +1**（习惯写法，区别于深拷贝 `.clone()`）
- **多个变量**同时持有，无所有权冲突
- **计数为 0**：最后一个 `Rc` 被 drop → 释放堆上 `T`

→ demo：`cargo run -- count`

## 三、共享尾链表（Book 经典例）

```rust
enum List {
    Cons(i32, Rc<List>),
    Nil,
}

let a = Rc::new(Cons(5, Rc::new(Cons(10, Rc::new(Nil)))));
let b = Cons(3, Rc::clone(&a)); // a 仍可用
{
    let c = Cons(4, Rc::clone(&a));
    // ...
} // c 消失，计数 -1
// b 仍持有对 a 尾部的共享引用
```

→ demo：`cargo run`

---

## 四、`Rc::strong_count`

```rust
Rc::strong_count(&rc)  // 当前强引用个数
```

- clone → **+1**；`Rc` 出作用域 → **Drop 自动 -1**
- 另有 `weak_count`（与 `Weak<T>` 相关）→ [15.6](./15.6-引用循环与Weak.md)
- **无需手动**减计数

## 五、只读共享与 `Deref`

多个 `Rc` 指向同一 `T` 时，通过 **`Deref`** 得到 **`&T`**（共享不可变），不破坏别名规则。

需要**可变**共享 → `Rc<RefCell<T>>` → [15.5 §四](./15.5-RefCell与内部可变性.md#四组合-2rcrefcellt最常用)

## 六、对照表（速查）

| 特性 | `Box<T>` | `Rc<T>` |
|------|----------|---------|
| **所有权** | 独占 | 共享（多所有者） |
| **共享访问** | ❌ move 后原变量失效 | ✅ `Rc::clone` 增计数 |
| **线程** | ✅ 可跨线程 move | ❌ **单线程**（多线程用 `Arc`） |
| **默认可变？** | ✅ 有唯一所有者时可 `mut` | ❌ 只读（要 `RefCell`） |
| **引用计数** | 无 | `strong_count` |
| **典型场景** | Cons/Box、独占堆数据 | 单线程多所有者（图/树共享子树） |

→ 限制详解 · 循环引用：[15.4.1](./15.4.1-Rc限制对比与循环引用.md)

---

## 七、一句话总结

1. **`Box`** = 独占指针；**`Rc`** = 引用计数共享指针。
2. **`Rc::clone`** = 计数 +1，**不**复制堆上 `T`。
3. **计数 0** 才释放；**单线程 + 默认不可变** 是两大前提。
