## 核心结论（先背）

1. **`Box<T>` 的 `deref` 返回 `&T`**，标准库写法核心是 **`&**self`**（从 `&Box<T>` 穿透到堆上 `T` 的引用）。
2. **递归 `List` 必须 `Box<List>`** —— 栈上 `Sized` 指针 + 堆上节点。
3. **`Box = Deref（怎么用）+ Drop（怎么收）`** —— 智能指针链的起点。

---

## 一、`Box<T>` 的 Deref 实现（逐行）

```rust
impl<T> Deref for Box<T> {
    type Target = T;

    fn deref(&self) -> &Self::Target {
        &**self
    }
}
```

### 1. `type Target = T`

关联类型：声明**解引用后的目标类型**。  
`Box<i32>` → `Target = i32`；`Box<String>` → `Target = str`（若写成 `Box<str>` 则 unsized，此处不展开）。

### 2. `fn deref(&self) -> &Self::Target`

| 部分 | 含义 |
|------|------|
| `&self` | 接收 **`Box` 的不可变引用**（`self` 的类型是 `&Box<T>`） |
| 返回值 | 堆上 **`T` 的不可变引用** `&T` |

### 3. 核心：`&**self`（从右往左）

在 `deref` 方法体内，`self` 的类型是 **`&Box<T>`**（不是 `Box<T>` 本体）。

| 步骤 | 表达式 | 类型 / 含义 |
|------|--------|-------------|
| 起点 | `self` | `&Box<T>` |
| 第一个 `*` | `*self` | 对 `&Box<T>` 解引用 → 经 **`Deref` 强制** → **`&T`**（指向堆上数据） |
| 第二个 `*` | `**self` | 对 `&T` 再解引用 → **`T`**（`i32` 等 `Copy` 类型可复制；非 `Copy` 则 move） |
| 最前 `&` | `&**self` | 对 `T` 取借用 → 最终返回 **`&T`** ✅ |

```text
&Box<T>  --*-->  &T  --*-->  T  --&-->  &T   （返回）
```

> **精度补充**：第一个 `*` 不是「拿出 `Box<T>` 本体」，而是 **`Deref` 语义的解引用**——`&Box<T>` 上写 `*` 会直接得到 `&T`。第二个 `*` 才是从引用读到值。教学口诀：**穿透盒子，借堆上数据**。

### 4. 等价写法（理解用）

```rust
// 与 deref() 返回值同一思路（b 是 Box<T> 变量）
let b = Box::new(10);
let r: &i32 = &*b;           // 日常最常用
let r2: &i32 = b.deref();    // 显式调 trait 方法

// 标准库 impl 方法体内 self: &Box<T>，故写 &**self：
fn peel(bx: &Box<i32>) -> &i32 {
    &**bx  // 与 Box::deref 体内同形；外部变量 b 请用 &*b
}
```

> **易混点**：`&**self` 出现在 **`fn deref(&self)` 内部**（`self` 是 `&Box<T>`）。若 `b` 已是 `Box<T>`，外部写 **`&*b`** 即可，不要写 `&**b`（第二个 `*` 会对 `i32` 解引用而报错）。

→ demo：`cargo run -- deref`

### 5. 隐式解引用（日常写法）

```rust
let b = Box::new(10);
println!("{}", b);
// 编译器自动走 Deref：&b → &i32，再交给 println!
// 手动写：println!("{}", *b);
```

口诀：**`Deref` 让智能指针像普通引用一样用，少写 `*`**。→ [15.2 §四](./15.2-通过Deref将智能指针当作引用.md#四derefsomex--iternext手动消费)

## 二、递归 `List`：为何直接嵌套报错

```rust
enum List {
    Cons(i32, List), // ❌
    Nil,
}
```

Rust 要求类型 **`Sized`**（编译期可知大小）：

1. `Cons` 大小 = `i32` + `sizeof(List)`
2. `List` 又可含 `Cons(i32, List)` → **无限套娃**
3. 编译器算不出字节数 → **拒绝编译**

形象理解：**盒子 = 数字 + 同款盒子，没有尽头，无法规定盒子多大。**

## 三、`Box<List>` 如何修复

```rust
enum List {
    Cons(i32, Box<List>),
    Nil,
}

use List::{Cons, Nil};
let list = Cons(1, Box::new(Cons(2, Box::new(Nil))));
```

| 要点 | 说明 |
|------|------|
| 指针宽固定 | 64 位下 `Box<List>` 占 8 字节 |
| `Cons` 可 `Sized` | `i32` + 8 字节指针 |
| 下一节点在堆 | 栈上只留指针，打破无限递归 |

```text
栈：Cons { head: i32, tail: Box → ... }
堆：Box 指向的下一个 List 节点
```

> 工程用 **`Vec<T>`**；Cons 学 **递归 + Box + 内存模型**。→ [15.1 §二](./15.1-使用Box指向堆上的数据.md#二递归类型报错--cons-链表完整) · `cargo run`

## 四、Deref 与 Drop：两大基石

### 1. `Deref` — 怎么用

| | |
|---|---|
| **作用** | 智能指针 **隐式 → `&T`** |
| **范围** | `Box` / `Rc` / `Arc` / 锁守卫… |
| **可变** | 另有 `DerefMut` → `&mut T` → [15.2.1](./15.2.1-Deref嵌套可变与编译坑.md) |

### 2. `Drop` — 怎么收

| | |
|---|---|
| **作用** | 出作用域 **自动** 释资源 |
| **`Box`** | 先 drop 堆上 `T`，再释放分配器块 |
| **规则** | 被动调用；禁止 `b.drop()`；提前用 `mem::drop(b)` → [15.3](./15.3-使用Drop运行清理代码.md) |

```rust
{
    let b = Box::new(String::from("hello"));
    println!("{b}");
} // b 失效 → Drop → 堆内存回收
```

→ demo：`cargo run -- drop`

## 五、整条知识链（背诵版）

```text
Cons(i32, List)           →  无限大小，编译失败
Cons(i32, Box<List>)      →  栈指针 + 堆节点，Sized ✅

Box::new(x)               →  堆分配
println!("{}", b)         →  Deref → &T
*b / b.deref()            →  手动穿透
出作用域                   →  Drop 释堆

Box<T> = Deref + Drop      →  Rc/Arc/RefCell 的底座
```

| # | 知识点 | 一句 |
|---|--------|------|
| 1 | 递归 enum | 嵌自身 ❌；嵌 `Box<自身>` ✅ |
| 2 | `Box<T>` | 栈指针 + 堆 `T` |
| 3 | `Deref` | `&**self` 穿透到 `&T`；日常自动解引用 |
| 4 | `Drop` | 生命周期结束自动释堆 |
