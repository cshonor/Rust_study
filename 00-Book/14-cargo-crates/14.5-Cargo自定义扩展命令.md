# 14.5 Cargo 自定义扩展命令

Cargo 支持在不改 Cargo 源码的前提下**扩展子命令**：若在 **`PATH`** 中存在名为 **`cargo-<子命令>`** 的可执行文件，则可在终端里像内置子命令一样执行：

```bash
cargo <子命令>
```

例如磁盘上有 `cargo-watch`（具体名称以实际 crate 为准），则可直接运行：

```bash
cargo watch
```

实现方式：该可执行文件需以 `cargo-` 为前缀，Cargo 会把后面的部分当作子命令名转发执行。

---

## 与 `cargo --list` 的关系

安装或放置好自定义命令后，部分扩展会出现在 **`cargo --list`** 列出的一览中（具体以当前 Cargo 版本行为为准），便于发现本机已可用的扩展命令。

---

## 安装扩展

与安装其它二进制 crate 一样，通常可用 **`cargo install <crate名>`** 将提供 `cargo-xxx` 的工具装到 **`~/.cargo/bin`**（Windows 多为 **`%USERPROFILE%\.cargo\bin`**），并确保该目录在 **`PATH`** 中。

## 本章小结

通过 **Cargo** 与 **crates.io** 分享代码，是 Rust 生态能覆盖多种任务的重要一环：标准库刻意保持小而稳定，而 crate 可以独立演进、版本节奏灵活。若在 crates.io 上发布对你有用的库或工具，也很可能同样惠及他人。
