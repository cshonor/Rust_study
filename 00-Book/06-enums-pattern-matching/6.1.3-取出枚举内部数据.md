## 先敲定：标签 vs 内部数据

| # | 规则 |
|---|------|
| 1 | 变量**类型 = 外层枚举**：`Option<i32>` / `IpAddr` |
| 2 | `Some`、`None`、`V4`、`V6` = **分类标签**，本身不是业务数据 |
| 3 | `Some(内部值)`：标签 + 一份数据；`None`：只有标签、无数据 |
| 4 | 业务最终用的是**括号/大括号里的内部数据**；标签只用来区分「有没有 / 是哪种」 |

```rust
let num: Option<i32> = Some(99);
// Some 是标签，99 才是要用的数字
```

## 四种取出方式（Option · IpAddr 通用）

对照例子：

- 例 1：`Option<i32> = Some(10)`
- 例 2：`IpAddr = IpAddr::V4(192, 168, 1, 2)`

### 方式 1：`match` — 最标准、最安全（首选）

靠标签分支，解构掏出内部数据：

```rust
let opt = Some(10);
match opt {
    Some(x) => println!("取出：{}", x),
    None => println!("没有数据"),
}

let ip = IpAddr::V4(192, 168, 1, 2);
match ip {
    IpAddr::V4(a, b, c, d) => println!("{}.{}.{}.{}", a, b, c, d),
    IpAddr::V6(s) => println!("ipv6: {}", s),
    IpAddr::Unknown => println!("无 ip"),
}
```

> 匹配到标签时，自动把变体绑定的数据绑定到新变量。

→ 详解 [6.2 match](./6.2-match.md)

### 方式 2：`if let` — 只关心一个变体

```rust
if let Some(val) = Some(10) {
    println!("{}", val);
}

if let IpAddr::V4(w, x, y, z) = ip {
    println!("{}", w);
}
```

→ 详解 [6.3 if let](./6.3-if-let.md)

### 方式 3：Option 专属 — `unwrap` / `unwrap_or`

**仅标准库 `Option`** 内置；自定义 enum 需自己 `impl`（见 demo `MyOption`）。

```rust
Some(5).unwrap();              // 取出 5；None 则 panic
None::<i32>.unwrap_or(0);      // None → 0，安全
```

⚠️ `unwrap` 底层本质就是 `match`。

### 方式 4：`let else`（Rust 1.65+）

```rust
let Some(data) = Some(88) else { return; };
println!("{}", data);
```

等价于：

```rust
let data = match Some(88) {
    Some(v) => v,
    None => return,
};
```

## 关键总结

1. **标签作用**：分类判断 — `Some`/`V4` 有内嵌数据可提取；`None`/`Unknown` 无数据可取  
2. **取值本质**：**解构** — 匹配标签，剥离枚举外壳，拿到内部原始数据  
3. **`Option`** 只是标准库预装 enum，与手写 `IpAddr` **机制完全一致**

---

## 自定义 enum 仿 `unwrap`（IpAddr 示例）

标准库不给 `IpAddr` 配 `unwrap`，只能自己封装：

```rust
fn get_v4(ip: IpAddr) -> (u8, u8, u8, u8) {
    match ip {
        IpAddr::V4(a, b, c, d) => (a, b, c, d),
        _ => panic!("不是 ipv4"),
    }
}
```

与 `Option::unwrap()` **思路一模一样**。
