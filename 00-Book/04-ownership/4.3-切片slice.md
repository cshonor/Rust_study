## 一、核心本质

## 二、字符串切片 `&str`（语法）

### Range（左闭右开）

| 写法 | 等价 |
|------|------|
| `[0..5]` | 0～4 |
| `[..2]` | `[0..2]` |
| `[3..]` | `[3..len]` |
| `[..]` | 全切片 |

```rust
let s = String::from("hello world");
let hello = &s[0..5];
let world = &s[6..11];
```

⚠️ 边界须在 **UTF-8 字符边界**，否则 panic。

### `first_word` 返回 `&str`

```rust
fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}
```

`as_bytes()` 用来找下标 `i`；返回 `&s[0..i]` 而不是 `i`。  
为何 `s.clear()` 会报错 → 只看 **[4.3.2](./4.3.2-切片与借用规则.md)**，不在此重复。

## 三、数组切片 `&[T]`

```rust
let arr = [1, 2, 3, 4, 5];
let sl: &[i32] = &arr[1..3]; // [2, 3]
```

`Vec<T>` 同理 `&vec[..]`（第 8 章）。

| 类型 | 含义 |

## 五、HFT（零拷贝 `&[u8]`）

1. 解析 UDP/共享内存：**不拷整包**，字段用 `&pkt[a..b]` 子切片。
2. **避免裸 `usize` 分段**；用切片绑缓冲区生命周期 → [4.3.1](./4.3.1-usize下标与悬空索引.md)。
3. 入参统一 `fn parse(pkt: &[u8])`。

```rust
let mut buf: Vec<u8> = receive_udp_packet();
let hdr = &buf[0..14]; // hdr 存活时 buf 不能 clear/resize（见 4.3.2）
```

实战：`4.3-slices-demo` 中 `parse_tick()`、`vec_packet_buffer_demo()`。

---

## 六、易错点

| 陷阱 | 后果 |
|------|------|
| 非 UTF-8 边界切片 | 运行时 panic |
| 返回 `usize` 索引 | 生命周期脱钩 → 4.3.1 |
| 切片存活时 `clear` | 编译错误（预期）→ 4.3.2 |
| 混淆静态/堆 `&str` | → 4.3.3 |

---

## 七、记忆卡片

| 要点 | 一句 |
|------|------|
| `usize` | 只是数；访问已清空数据 → 悬空访问（4.3.1） |
| `&str` + `clear` | 不可变借 vs `&mut`（4.3.2） |
| 两类 `&str` | 堆切片 vs `'static`（4.3.3） |
| Slice | 指针 + 长度，不拥有数据 |
| HFT | `&[u8]` 零拷贝，少 `clone` |
