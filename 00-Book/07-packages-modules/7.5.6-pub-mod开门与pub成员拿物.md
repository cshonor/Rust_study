# 7.5.6 `pub mod` 只开门：`pub` 成员 / `pub use` 才拿物

← [7.5.3 `lib.rs` mod vs pub mod](./7.5.3-lib.rs里mod与pub-mod.md) · [7.5.5 两位置何时 pub mod](./7.5.5-顶层lib与子mod里pub-mod何时写.md) · [7.4.2 `use` vs `pub use`](./7.4.2-use与pub-use.md)

对应 compile-fail：`7.1-packages-crates-demo/bin_plus_lib/compile_fail/private_mod/`（模块私有 vs 项 `pub`）

---

## 结论（先给你）

1. **`lib.rs` 里 `pub mod xxx;` 只是「开放模块入口」** — 允许别人**进入**这个模块路径，不等于里面任何东西都能调。
2. 想让外部调用**函数 / 结构体 / 枚举**，必须在子模块里用 **`pub` 修饰该项**（或 **`pub use` 重导出**）；只开门、内部全私有 → **编译不报错**，但外部**调用时报错**（私有项）。
3. 模块内部完全不对外暴露任何东西时，`pub mod` **失去实际意义**（合法空壳）。

---

## 一、层级权限：开门 vs 拿物

| 层级 | 写法 | 比喻 |
|------|------|------|
| **crate 根 `lib.rs`** | `pub mod scanner;` | **开门** — 外部能走到 `scanner` 这一层 |
| **子模块内** | `pub fn scan()` / `pub struct Token` | **把东西拿出来** — 外部能调用具体 API |
| **子模块内** | `pub use token::Token;` | **把子文件里的东西提到门口** — 同上，路径更短 |

> 只开门、不放东西：门能打开，里面啥也用不了 — **无「多余」编译错误，仅无实用价值**。

---

## 二、分场景演示（`scanner`）

目录：

```
src/
├── lib.rs
└── scanner/
   ├── mod.rs
   └── token.rs
```

### 场景 1：`pub mod scanner;`，内部全私有 — 不报错，但用不了

`lib.rs`：

```rust
pub mod scanner;   // 开放 scanner 模块入口
```

`scanner/mod.rs`：

```rust
mod token;

fn internal_work() {
    println!("内部逻辑");
}
```

外部尝试调用：

```rust
use crate::scanner;

fn main() {
    scanner::internal_work(); // ❌ 编译错误：private function
}
```

| 现象 | 说明 |
|------|------|
| **`pub mod scanner;` 本身** | ✅ 不报错；编译器认可 `scanner` 模块存在 |
| **`scanner::internal_work()`** | ❌ 报错 — 函数无 `pub`，对模块外私有 |
| **结果** | 公开的模块像**空壳**：能 `use crate::scanner`，却调不到任何功能 |

→ 私有项规则 [7.2](./7.2-定义模块来控制作用域与私有性.md)

### 场景 2：子模块直接 `pub` 成员 — 最基础暴露

`scanner/mod.rs`：

```rust
mod token;

pub fn scan() {
    // ...
}
```

外部：

```rust
use crate::scanner;

fn main() {
    scanner::scan(); // ✅
}
```

- **不需要 `pub use`** — 在 `mod.rs` 里直接 `pub fn` 即可。
- 适合逻辑少、不拆多个 `.rs` 的小模块。

### 场景 3：拆文件 + `pub use` 重导出 — clox 主流

`scanner/token.rs`：

```rust
pub struct Token {
    pub lexeme: String,
}
```

`scanner/mod.rs`：

```rust
mod token;                    // 私有登记 token.rs

pub use token::Token;         // 把 Token 提到 scanner 顶层

pub fn scan() {}
```

外部：

```rust
use crate::scanner::{Token, scan}; // ✅
```

→ [7.4.2](./7.4.2-use与pub-use.md) · [7.5.5 子 mod 规范](./7.5.5-顶层lib与子mod里pub-mod何时写.md)

## 三、`pub mod` 与内部 `pub` / `pub use` 的关系

### 1. 两层权限要都满足

外部要 `use crate::scanner::Token`，需要：

```
lib.rs:     pub mod scanner;     ← 子模块对 crate 外可见（若从别的 crate 用，还要这一层 pub mod）
scanner:    pub struct Token      ← 或 pub use token::Token
token.rs:   pub struct Token      ← 源头也要 pub（pub use 只能重导出已是 pub 的项）
```

任一层缺 `pub`，路径在那一层就断掉。

### 2. 两种暴露内容的方式（可单独用、可组合）

| 方式 | 写法 | 何时用 |
|------|------|--------|
| **A. 直接 `pub` 成员** | `pub fn scan()` / `pub struct X` 写在 `mod.rs` | 简单模块、逻辑集中在一个文件 |
| **B. `pub use` 重导出** | `mod token;` + `pub use token::Token;` | 拆多个 `.rs`，隐藏文件结构 |

**不是**「写了 `pub mod` 就必须 `pub use`」— `pub use` 只是**拆文件后**的便捷收拢方案。

### 3. 误区纠正

| | |
|---|---|
| ❌ | `pub mod scanner` 一开，里面自动全公开 |
| ❌ | 必须用 `pub use`，不能直接 `pub fn` |
| ✅ | 只 `pub mod`、内部全私有：**编译通过**，外部调私有项才报错 |
| ✅ | 想让模块**有实际用途**，必须在模块内 **`pub` 项或 `pub use`** |

### 4. 反向：`mod` 私有模块 + 内部 `pub` 项

`lib.rs`：`mod math;`（私有模块）

`math/mod.rs`：`pub fn add() { … }`

- **同 crate 内**：`crate::math::add()` ✅
- **别的 crate**：`use mylib::math::add` ❌ — 模块本身未从 `lib.rs` `pub mod` 放出

→ [7.5.3 §五 compile-fail](./7.5.3-lib.rs里mod与pub-mod.md)

## 四、clox 实操规范

### 标准搭配（有实际意义）

**`lib.rs`** — 开放一级目录：

```rust
pub mod scanner;
pub mod parser;
pub mod ast;
```

**各子 `mod.rs`**：

```rust
mod token;
mod lex;

pub use token::{Token, TokenType};
pub use lex::scan_source;

pub fn scan_all(source: &str) -> Vec<Token> {
    scan_source(source)
}
```

### 反面写法（合法但多余）

```rust
// lib.rs
pub mod helper;

// helper/mod.rs
mod inner;

fn do_something() {}   // 全私有
```

- ✅ 整个 crate **能编译**
- ❌ 外部永远调不到 `helper` 里任何东西 — `pub mod helper` **纯属多余**

若 helper 仅 crate 内用 → `lib.rs` 写 **`mod helper;`** 更合适。

---

## 五、背诵表

| 写法 | 作用 | 单独用够吗 |
|------|------|------------|
| **`pub mod scanner;`** | 开放模块入口（开门） | ❌ 不够 — 还要内部 `pub` 项 |
| **`pub fn` / `pub struct`** | 暴露具体 API（拿物） | ✅ 在已可达的模块内够用 |
| **`pub use token::Token;`** | 从重导出项拿物 + 路径上浮 | ✅ 拆文件时推荐 |
| **`mod token;` + 全私有** | 仅内部实现 | 配合上面二者之一才有对外 API |

---

## 一句话速记

| | |
|---|---|
| **`pub mod`** | 开门 |
| **`pub` 成员 / `pub use`** | 拿物 |
| **只开门不拿物** | 不报错，没用 |
| **要真能调用** | 必须对外暴露成员 |
