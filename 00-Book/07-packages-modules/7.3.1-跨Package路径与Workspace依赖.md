# 7.3.1 跨 Package 路径与 Workspace 依赖

← 同 crate 内路径：[7.3 路径](./7.3-路径用于引用模块树中的项.md) · `path` 依赖：[7.1.1](./7.1.1-二进制与库crate.md) · Workspace：[14.3](../14-cargo-crates/14.3-Cargo工作空间.md) · 字段 `pub`：[7.2.2](./7.2.2-结构体与字段私有性.md)

对应 demo：`00-Book/07-packages-modules/7.3-cross-package-paths-demo/`

---

## 一句话结论

> **跨 Package 不能用 `super::`、不能用相对路径！`super`、`self` 只限制在【当前同一个 Crate 内部】；跨包靠「Cargo 依赖 + 对方包 `pub` 导出 + `包名::` 绝对路径」调用。**

| 范围 | 可用路径 |
|------|----------|
| **同 crate 内** | `crate::` · `super::` · `self::` · 相对模块名 |
| **跨 Package / 跨 crate** | **`包名::`**（如 `std::` · `pkg_b::`）— 无 `super` |

---

## 场景：Workspace 里 A 调 B

```
workspace/
├── Cargo.toml          ← members = ["pkg_a", "pkg_b"]
├── pkg_b/              ← Package B（被依赖的库）
│   ├── Cargo.toml
│   └── src/
│       ├── lib.rs      ← lib crate 出口
│       └── b_mod.rs
└── pkg_a/              ← Package A（调用方）
    ├── Cargo.toml
    └── src/main.rs
```

> `pkg_a` 与 `pkg_b` 是**两个独立 Crate、两棵互不连接的模块树** — 不在同一棵树下，`super` 摸不到对方。

---

## 步骤 1：pkg_b 向外暴露（lib.rs 做 `pub` 导出）

`pkg_b/src/b_mod.rs`：

```rust
pub fn hello_b() {}

pub struct Data {
    pub val: i32,
}
```

`pkg_b/src/lib.rs`（**只有在这里 `pub`，外面才能看见**）：

```rust
pub mod b_mod;
```

| 写法 | 结果 |
|------|------|
| `pub mod b_mod;` | pkg_a 可 `pkg_b::b_mod::…` |
| `mod b_mod;`（无 pub） | pkg_a **永远找不到** b_mod → 编译报错 |

零散 `.rs` 文件不会自动对外 — 必须在 **B 的 `lib.rs`** 用 `pub mod` 放行。

---

## 步骤 2：pkg_a 的 `Cargo.toml` 声明依赖

```toml
[dependencies]
pkg_b = { path = "../pkg_b" }
```

没有这行，Rust **不会链接** pkg_b，代码里写 `pkg_b::…` 也无效。

Workspace 根 `Cargo.toml` 还需：

```toml
[workspace]
members = ["pkg_a", "pkg_b"]
resolver = "2"
```

---

## 步骤 3：pkg_a 里调用 — 只用 `包名::` 绝对路径

`pkg_a/src/main.rs`：

```rust
fn main() {
    pkg_b::b_mod::hello_b();
    let d = pkg_b::b_mod::Data { val: 1 };
    println!("{}", d.val);
}
```

或先 `use` 再短名：

```rust
use pkg_b::b_mod::{hello_b, Data};

fn main() {
    hello_b();
    let d = Data { val: 1 };
}
```

---

## 关键纠正

### ❌ 跨 Package 不能这样写

```rust
super::pkg_b::b_mod::hello_b();  // 编译错误
```

`super` = 找**当前 Crate 的父模块**。pkg_a、pkg_b 是两棵独立模块树，**不是父子关系**。

同理：`front_of_house::…` 这类**相对模块名**也只在本 crate 内有效。

### ✅ 跨包统一写法

| 来源 | 路径前缀 |
|------|----------|
| 标准库 | `std::collections::HashMap` |
| Workspace 成员 | `pkg_b::b_mod::hello_b()` |
| crates.io | `serde::Serialize` |

---

## 完整规则三条

1. **B 包内部**：零散 `*.rs` → 在 B 的 `lib.rs` `pub mod xxx;` 对外放行  
2. **A 包配置**：A 的 `Cargo.toml` 加 `path` / 版本依赖 B  
3. **A 包调用**：`use pkg_b::b_mod;` 或直接 `pkg_b::b_mod::func()`

同 Package 内 bin 调 lib（`use 包名::…`）→ [7.1.7](./7.1.7-同Package的lib与main.md)  
非 Workspace 的跨项目 `path` 依赖 → [7.1.1 §六](./7.1.1-二进制与库crate.md)（`user_demo/`）

---

## 结构体跨包：沿用 7.2.2 的 `pub` 链

```rust
// pkg_b/b_mod.rs
pub struct Data {
    pub val: i32,   // 字段也要 pub，A 才能 Data { val: 1 }
    name: String,   // 私有 → A 不能直读，需 getter
}
```

必须同时满足：

- `struct` 加 `pub`
- 需要外部读写的**字段**加 `pub`
- 模块经 `lib.rs` 的 `pub mod b_mod;` 导出

---

## 一句话总结

| | |
|---|---|
| **同 crate** | `super` / `self` / 相对路径 / `crate::` |
| **跨 Package** | Cargo 依赖 + **`包名::`** 绝对路径 |
