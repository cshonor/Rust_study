## 一、核心类比（最容易记）

把 **模块树 = 电脑文件夹目录**，**`::` = `/` 斜杠**。

| 前缀 | 等价目录符号 | 含义 | 路径类型 |
|------|--------------|------|----------|
| `crate::` | `/`（磁盘根） | 从**当前 crate 根**出发，全局绝对路径 | 绝对 |
| `super::` | `../` | 跳到**父模块** | 相对 |
| `self::` | `./` | 留在**本模块内部** | 相对 |
| 直接写模块名（`front_of_house::`） | 同级文件夹 | 与当前模块**平级**的子模块 | 相对 |

> **规则：绝对永远从 crate 根起步；相对永远从当前所在模块起步。**

外部 crate 的绝对路径：直接写 **库名::**，例如 `std::collections::HashMap`（`std` 是标准库 crate 名）。

## 二、模块树（官方 front_of_house 示例）

```
crate（根）
├── front_of_house
│   └── hosting → add_to_waitlist()
├── back_of_house
│   └── fix_incorrect_order() → cook_order() + super::serve_order()
└── serve_order()              ← crate 根上的函数
```

## 三、两种路径详细拆分

### 1. 绝对路径：`crate::xxx::xxx`

**固定写法：永远 `crate::` 开头，不受当前代码所在位置影响。**

```rust
// 任意文件、任意模块里都能这么写，换目录层级后代码不用改
crate::front_of_house::hosting::add_to_waitlist();
```

### 2. 相对路径（随当前代码位置变化）

#### ① 同级直接写模块名

当前代码在 **crate 根同级**（`lib.rs` / `main.rs` 顶层）时，`front_of_house` 与当前模块同级：

```rust
front_of_house::hosting::add_to_waitlist();
```

> 换到 `back_of_house` 模块**内部**就不能直接写 `front_of_house::…`，层级变了，需 `super::front_of_house::…` 或 `crate::…`。

#### ② `super::` 向上找父模块（`../`）

`back_of_house` 里的 `fix_incorrect_order` 要调用**根上的** `serve_order()`：

```rust
fn fix_incorrect_order() {
    cook_order();
    super::serve_order(); // super = back_of_house 的父 = crate 根
    // 等价绝对路径：crate::serve_order();
}
```

多层父：`super::super::xxx` 往上跳两层祖先模块。

#### ③ `self::` 当前模块内找（`./`）

```rust
// 在 back_of_house 内部
self::cook_order(); // 等价直接写 cook_order()
```

日常几乎可省略 `self::`，只有**同名冲突**时才显式加。

## 四、工程目录 + `mod.rs`（文件夹写法）

```
src/
├── lib.rs              // crate 根
├── front_of_house/
│   ├── mod.rs          // pub mod hosting;
│   └── hosting.rs      // pub fn add_to_waitlist() {}
└── back_of_house/
    └── mod.rs          // fix_incorrect_order + 路径示例
```

1. `lib.rs`：`pub mod front_of_house;` · `mod back_of_house;`
2. `front_of_house/mod.rs`：`pub mod hosting;`

### 在 `back_of_house/mod.rs` 里调用 `hosting`

```rust
// 1. 绝对（万能，换位置不变）
crate::front_of_house::hosting::add_to_waitlist();

// 2. 相对：先 super 到 crate 根，再进同级 front_of_house
super::front_of_house::hosting::add_to_waitlist();
```

## 五、`pub` 隐私与路径联动（衔接 7.2 / 7.2.2）

**路径能定位 ≠ 能访问**，沿途每一项的可见性都要满足：

| 写法 | 效果 |
|------|------|
| `mod front_of_house;` 无 `pub` | 外部 crate **路径找不到**这个模块 |
| `pub mod hosting;` | `hosting` 对外暴露，路径能索引到 |
| `fn add_to_waitlist()` 无 `pub` | 路径能定位也**调不了**（私有） |
| `pub fn add_to_waitlist()` | 跨模块可调用 |

### 结构体字段沿用同一套

```rust
pub struct User {
    pub id: u32,   // pub → 跨模块可 u.id 读写
    name: String,  // 私有 → 同模块随便改，跨模块不可见
}
```

→ 详解 [7.2.2 结构体与字段私有性](./7.2.2-结构体与字段私有性.md)

### `pub struct` vs `pub enum`

- **`pub struct`**：类型公有，**字段默认仍私有**（字段须单独 `pub`）
- **`pub enum`**：类型公有时，**变体默认全公有**

## 六、实战怎么选

| 场景 | 建议 |
|------|------|
| **库项目 `lib.rs`** | 优先 **`crate::` 绝对路径** — 后续调整目录层级不易崩 |
| **内部子模块互调** | 优先 **`super::` / 相对路径** — 更简洁 |
| **main 二进制** | 两种混用均可 |

---

## 七、跨 Package：无 `super` / 无相对路径

**`super`、`self`、相对模块名只在本 crate 内有效。** 跨 Package（含 Workspace 成员之间）必须用：

1. A 的 `Cargo.toml` 声明 `path` / 版本依赖  
2. B 的 `lib.rs` `pub mod …` 导出  
3. A 里 **`包名::模块::函数`**（如 `pkg_b::b_mod::hello_b()`）

→ 详解 [7.3.1 跨 Package 路径与 Workspace 依赖](./7.3.1-跨Package路径与Workspace依赖.md) · demo `7.3-cross-package-paths-demo/`
