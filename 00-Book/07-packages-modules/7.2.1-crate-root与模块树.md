## 一、核心定义

### 1. Crate Root（箱子根）

> **`src/lib.rs`（库 crate）或 `src/main.rs`（bin crate）= 当前 Crate 的根文件 = 模块树顶层隐形根模块 `crate`**

| 入口 | Crate 类型 |
|------|------------|
| `src/lib.rs` | Library crate root |
| `src/main.rs` | 默认 Binary crate root |
| `src/bin/xxx.rs` | 独立 Bin crate，**各自**是自己的 crate root |

- 编译器 **从 crate root 扫描整棵模块树**
- root 里代码默认挂在顶层虚拟模块 **`crate`** 下
- **一个 Crate 只有 1 个 crate root**

### 2. Module Tree（模块树）

用 `mod 名字 { … }` 嵌套声明；所有模块从 **`crate`** 向下分叉，像目录树；用 **`::`** 寻址函数 / 结构体 / 子模块。

### 餐厅代码（写在 crate root，如 `src/lib.rs`）

```rust
mod front_of_house {
    mod hosting {
        fn add_to_waitlist() {}
        fn seat_at_table() {}
    }
    mod serving {
        fn take_order() {}
        fn serve_order() {}
        fn take_payment() {}
    }
}
```

#### 生成的模块树

```
crate                          ← 顶层根（来自 crate root 文件）
 └── front_of_house            ← 一级子模块
     ├── hosting
     │   ├── add_to_waitlist
     │   └── seat_at_table
     └── serving
         ├── take_order
         ├── serve_order
         └── take_payment
```

## 二、三条规则（Package / Crate / Module）

| # | 规则 |
|---|------|
| 1 | **Package**：`Cargo.toml` 项目夹；最多 1 lib + 多 bin；**每个 Crate 各自 1 个 crate root、各自 1 棵模块树** |
| 2 | **Crate** = 一棵完整模块树；树根 = `crate`（由 crate root 生成） |
| 3 | **Module** = 树上的节点；不能脱离 Crate；**写了 `.rs` 但不 `mod xxx;` → 不会进树，编译器忽略** |

### 同 Package：lib + main = 两棵树

```
Package
├ lib crate  → root: lib.rs  → 模块树 A
└ bin crate  → root: main.rs → 模块树 B（互不干扰）
```

→ [7.1.7 lib + main 双 crate](../07-packages-modules/7.1.7-同Package的lib与main.md)

**bin 要用 lib 的模块**：不能直接在 main 树里 `mod front_of_house`，而 **`use 包名::…`** 引入 **lib 那棵树** 里 `pub` 出来的 API。

## 三、路径寻址

### 绝对路径（从 `crate::`）

```rust
crate::front_of_house::hosting::add_to_waitlist();
```

### 相对路径（从当前模块）

```rust
// 在 front_of_house 模块内部
hosting::add_to_waitlist();
```

### 默认私有

模块内函数默认 **私有**；**加 `pub`** 才能跨模块调用 → [7.2 私有性](./7.2-定义模块来控制作用域与私有性.md)

demo 中 `eat_at_restaurant()` 通过 `front_of_house::hosting::add_to_waitlist()` 调用 **已 `pub` 的** 函数。

---

## 四、拆分到单独文件（逻辑树不变）

1. `src/lib.rs`（crate root）：`mod front_of_house;`
2. `src/front_of_house.rs`：`mod hosting; mod serving;`
3. 或 `src/front_of_house/mod.rs` + `hosting.rs` / `serving.rs`

> **物理文件怎么拆，逻辑上的模块树结构不变。** 内嵌 `{ }` vs `mod xxx;` 分文件 → [7.5 将模块分割进不同文件](./7.5-将模块分割进不同文件.md)

---

## 五、层级总结

```
Package
├─ Lib-Crate  → root: lib.rs  → crate → mod …
└─ Bin-Crate  → root: main.rs → crate → mod …（另一棵）
```
