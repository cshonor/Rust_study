# 7.5.5 两个位置分清：`lib.rs` 顶层 vs 子目录 `mod.rs` 里何时写 `pub mod`

← [7.5.3 `lib.rs` 里 mod vs pub mod](./7.5.3-lib.rs里mod与pub-mod.md) · [7.5.4 逐个声明与模板](./7.5.4-mod.rs逐个声明与批量宏.md) · [7.4.2 `use` vs `pub use`](./7.4.2-use与pub-use.md)

---

## 结论（先给你）

> **`pub mod` 语法到处都能写，合法。** 但**写在哪儿、推不推荐**，取决于位置：
>
> - **顶层 `lib.rs`**：`pub mod 文件夹;` → **高频、必备**
> - **子目录 `mod.rs` 里对同级 `.rs`**：`mod 名字;` + **`pub use` 收拢** → **标准**；`pub mod token;` → **极少用、破坏封装**

全程用 clox 风格目录举例。

---

## 基础目录

```
src/
├── lib.rs              # crate 根入口
├── scanner/
│   ├── mod.rs          # scanner 子模块入口
│   ├── token.rs
│   └── lex.rs
└── parser/
    └── mod.rs
```

---

## 一、核心问题：`pub mod` 能不能在子 `mod.rs` 里写？

**能写，完全合法。** 只是和顶层 `lib.rs` **场景不同、推荐度不同**。

---

## 二、位置 1：顶层 `lib.rs` — `pub mod 目录名;` **非常常用，必写**

### 作用

把**一整个文件夹模块**挂进 crate 树，并决定是否对 **crate 外** 开放。

### clox 标准写法

```rust
// src/lib.rs
pub mod scanner;   // 公开 scanner 整棵子树
pub mod parser;
// pub mod ast;
// pub mod chunk;
// pub mod vm;
```

### 解读

| 写法 | 效果 |
|------|------|
| **`pub mod scanner;`** | 本 crate 内外都能走 `crate::scanner::…` / `包名::scanner::…` |
| **`mod scanner;`（无 pub）** | 仅 **crate 内** 可访问；外部 crate、依赖你的 bin **进不来** |

→ 业务一级模块（scanner / parser / vm…）在 `lib.rs` **大量 `pub mod` 是正常标配** → 详见 [7.5.3](./7.5.3-lib.rs里mod与pub-mod.md)

内部只想 crate 自用、不对外暴露的辅助模块（如 `helper`）→ `mod helper;` 即可。

---

## 三、位置 2：子目录 `mod.rs` — 对同级 `.rs` **尽量不用 `pub mod`**

### 语法合法，但不推荐

```rust
// src/scanner/mod.rs
pub mod token;   // 合法，但日常少这么写
```

### 效果

外部可以直接摸到**内部文件名**：

```rust
use crate::scanner::token::Token;   // 多一层 token，且绑死文件结构
```

### 为什么不推荐？

| 问题 | 说明 |
|------|------|
| **破坏封装** | `token.rs`、`lex.rs` 是**内部实现细节**；`pub mod token` 等于把目录结构暴露出去 |
| **重构成本高** | 改文件名、合并/删除 `token.rs`，所有 `scanner::token::…` 调用方全炸 |
| **路径啰嗦** | 调用方多写一层 `.token::` |

---

## 四、子 `mod.rs` 的正确做法（业界规范）

### 规则

1. 同级 `.rs` → **`mod 名字;`（不带 pub）**，隐藏内部文件
2. 要对外的类型/函数 → **`pub use` 重导出到当前模块顶层**

### 标准示例（照着写）

```rust
// src/scanner/mod.rs

// 第一步：登记同级文件 — 私有，外部看不到 token、lex 这两个子模块路径
mod token;
mod lex;

// 第二步：只导出 API，不导出文件结构
pub use token::{Token, TokenType};
pub use lex::scan_source;
```

### 外部使用

```rust
use crate::scanner::Token;        // 简洁
use crate::scanner::scan_source;
```

**看不到**内部有 `token.rs` 这件事 — 以后随便改内部拆分，只要 `pub use` 不变，外部不用动。

→ `pub use` 详解 [7.4.2](./7.4.2-use与pub-use.md) · 模板 [7.5.4](./7.5.4-mod.rs逐个声明与批量宏.md)

## 五、子 `mod.rs` 里**什么时候才真的写 `pub mod`？**

**少数场景**：当前目录下还有**嵌套子文件夹**，且这个**子文件夹本身**要对外当作独立模块。

```
scanner/
├── mod.rs
├── token.rs
└── utils/           # 嵌套子文件夹
    ├── mod.rs
    └── format.rs
```

若希望外部能直接 `crate::scanner::utils::…`：

```rust
// scanner/mod.rs
mod token;
mod lex;

pub mod utils;       // 嵌套文件夹要对 crate 外当独立模块 → 才用 pub mod
pub use token::{Token, TokenType};
pub use lex::scan_source;
```

| 情况 | 写法 |
|------|------|
| 同级单个 `.rs`（token.rs、lex.rs） | **`mod` 即可，不要 `pub mod`** |
| 嵌套子文件夹且要对 crate 外开放子树 | **`pub mod 子文件夹;`** |
| 嵌套子文件夹仅 scanner 内部用 | **`mod utils;`** |

普通**单层级**拆文件，**不要在子 `mod.rs` 里 `pub mod 单个rs文件名`**。

## 六、背诵表

| 书写位置 | 写法 | 频率 | 适用场景 |
|----------|------|------|----------|
| **顶层 `lib.rs`** | `pub mod 文件夹;` | 🔴 高频必写 | 开放一级业务模块（scanner / parser / vm…） |
| **顶层 `lib.rs`** | `mod 文件夹;` | 🟡 偶尔 | 仅 crate 内用的 helper / internal |
| **子 `mod.rs`（同级 `.rs`）** | `mod 文件名;` | 🟢 标准 | 管理内部拆分，隐藏实现 |
| **子 `mod.rs`（同级 `.rs`）** | `pub mod 文件名;` | ⚫ 极少、不推荐 | 暴露内部文件结构 |
| **子 `mod.rs`（嵌套子文件夹）** | `pub mod 子文件夹;` | 🟡 偶尔 | 多层目录，子文件夹对外也是独立模块 |
| **子 `mod.rs`（任意）** | `pub use …;` | 🟢 标准 | 对外暴露**内容**，不暴露**文件路径** |

## 七、写 clox 时直接套用

### 1. `src/lib.rs`

所有一级业务文件夹：

```rust
pub mod scanner;
pub mod parser;
// pub mod ast;
// pub mod chunk;
// pub mod vm;
```

→ 这里 **`pub mod` 放心写**，是项目标配 — 但还要在子模块里 **`pub` 项 / `pub use`** 才有可调 API → [7.5.6](./7.5.6-pub-mod开门与pub成员拿物.md)

仅内部工具、不想给外部 crate 用的：`mod helper;`。

### 2. 每个子文件夹的 `mod.rs`（`scanner/mod.rs`、`parser/mod.rs`…）

| 做什么 | 怎么写 |
|--------|--------|
| 登记 `token.rs`、`lex.rs` | **`mod token;` `mod lex;`** — 不加 pub |
| 对外暴露 `Token`、`scan` | **`pub use token::…;` `pub use lex::…;`** |
| 同级零散 `.rs` | **坚决不要** `pub mod token;` |

### 3. 一句话区分

| 关键字 | 记法 |
|--------|------|
| **`pub mod 文件夹`** | 对外开放**一整棵子模块目录**（多在 `lib.rs`） |
| **`pub use`** | 对外开放**里面的类型/函数**，路径上浮、隐藏文件 |
| **子模块内零散 `.rs`** | 默认 **`mod` 私有**；用 **`pub use` 当对外 API** |
