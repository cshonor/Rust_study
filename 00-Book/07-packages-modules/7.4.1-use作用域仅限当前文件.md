# 7.4.1 `use` 作用域：仅限当前文件，没有全局 `use`

← 主笔记：[7.4 use / pub use](./7.4-使用use关键字将名称引入作用域.md) · **`use` vs `pub use` 场景**：[7.4.2](./7.4.2-use与pub-use.md) · `mod.rs` 场景：[7.5.1](./7.5.1-mod.rs里的mod与use.md)

对应 demo：`7.5-split-modules-demo/`（`lib.rs` 与 `domain/` 注释对照）

---

## 一句话

> **所有 `use` 只作用在「当前写 `use` 的这个文件」的作用域；没有全局生效的 `use`；`lib.rs` 里的 `use` 也不能自动全项目通用。**

---

## 1. 在某个文件夹的 `mod.rs` 写 `use`

只在 **这个 `mod.rs` 文件内部** 生效；**同目录别的 `.rs` 用不了这个别名**。

```
src/domain/
├── mod.rs
└── user.rs
```

`domain/mod.rs`：

```rust
pub mod user;
use self::user::UserEntity; // 仅限 mod.rs 本文件能简写 UserEntity
```

`user.rs` 里面 **不能直接写 `UserEntity` 当别名**（若指的是 mod.rs 导入的那个简写），**必须在本文件自己 `use` 或写全路径**。

---

## 2. 根目录 `lib.rs` 写 `use`

也 **只在 `lib.rs` 自身生效，子模块全部不继承**。

`lib.rs`：

```rust
pub mod domain;
use std::collections::HashMap; // 只有 lib.rs 本文件能用 HashMap 这个短名
```

`domain/mod.rs`、`user.rs` 想用 `HashMap`，**必须各自再 `use std::collections::HashMap;`**。

---

## 3. 唯独 `pub use` 不一样（企业项目重点）

| | 作用 |
|---|------|
| 普通 **`use`** | 仅限**本文件**内部简写 |
| **`pub use`** | 把名字**提升到当前模块的对外 API**；别的文件/别的包可通过**模块路径**引入 |

```rust
// domain/mod.rs
pub mod user;
pub use user::UserEntity;
```

外层 `lib.rs` 或别的模块：

```rust
use crate::domain::UserEntity; // 得益于 pub use，不用写 domain::user::UserEntity
```

> **`pub use` 不是全局** — 是把类型挂到**当前模块的出口**；外部仍须自己 `use crate::domain::UserEntity`。

---

## 4. 不存在「一处 `use` 全工程受用」

Rust **没有** C 语言 `#include` 式全局导入。**没有任何写法**在一处 `use` 后让整个 Package 自动可用。

---

## 极简总结

| # | 规则 |
|---|------|
| 1 | **`use xxx`**：仅限**当前文件**简写，出了文件失效 |
| 2 | **`pub use xxx`**：暴露到**本模块出口**，别人经模块路径 `use` |
| 3 | **没有** Package 级全局 `use` |

---

## 小例子（固化）

```rust
// domain/mod.rs
pub mod user;
use self::user::UserEntity; // ✅ 只有 mod.rs 本文件

// domain/user.rs
// ❌ 不能指望 mod.rs 的 use；须本文件自己写：
use crate::domain::order::OrderEntity;
// 或 use crate::domain::user::UserEntity;（若在本文件引用自己模块的类型用全路径）
```

## demo

```bash
cd 00-Book/07-packages-modules/7.5-split-modules-demo

```

- `lib.rs`：`use std::collections::HashMap` 仅在本文件 `demo_use_scope_lib_only()` 使用
