# 7.5.1 `mod.rs` 里的 `mod` 与 `use`

← 分文件语法：[7.5](./7.5-将模块分割进不同文件.md) · `use` 详解：[7.4](./7.4-使用use关键字将名称引入作用域.md) · 工程范例：[7.1.3 Actix 目录](./7.1.3-Actix-web项目目录范例.md)

对应 demo：`7.5-split-modules-demo/src/domain/`（与 `front_of_house/` 并列）

---

## 一句话

> **`mod.rs` 里写 `use` = 在当前模块内部偷懒简写路径，不用每次写一长串全路径。**

| 关键字 | 管什么 |
|--------|--------|
| **`mod`** | 挂靠子文件、**搭建模块树**（没有它，`.rs` 不会被纳入编译） |
| **`use`** | 在本文件**省事调用代码**（只优化写法，不改变模块树） |

两件事互不冲突，所以 `mod.rs` 里经常同时出现 `pub mod xxx;` + `use xxx::yyy;`。

---

## 工程真实例子：`domain/mod.rs`

目录：

```
src/domain/
├── mod.rs
├── user.rs
└── order.rs
```

### `domain/mod.rs`

```rust
// 第一步：mod 把同目录文件挂进模块树（必写）
pub mod user;
pub mod order;

// 第二步：use 在当前 domain 模块内简写（方便本 mod.rs 里的代码）
// 若同时要对外导出，一条 pub use 即可（内部 + 外部都能用短名）：
pub use user::UserEntity;
pub use order::OrderEntity;

pub fn check() {
    let u = UserEntity::new();
    let o = OrderEntity::new();
    let _ = (u, o);
}
```

| 行 | 作用 |
|----|------|
| `pub mod user;` | **加载 `user.rs`，注册子模块** — 没有这行 `user.rs` 等于作废 |
| `pub use user::UserEntity;` | **本 `mod.rs` 内简写 + 对外重导出** — 不必再写 `use self::user::UserEntity`（两者同名会冲突） |

仅内部用、不对外导出时，写普通 `use self::user::UserEntity;` 即可。

---

## 两个关键字本质

| | `mod` | `use` |
|---|-------|-------|
| **职责** | 文件 ↔ 模块绑定，建模块树 | 当前文件的名字简写 |
| **影响编译单元** | ✅ 决定哪些 `.rs` 进树 | ❌ 不引文件、不改树结构 |
| **可见性** | 配合 `pub mod` 控制对外暴露 | 路径须全 `pub` 才能 `use` |

---

## 三种常见在 `mod.rs` 写 `use` 的场景

### 场景 1：子模块互相调用（最常用）

`domain/user.rs` 要引用 `order` 里的类型：

```rust
use crate::domain::order::OrderEntity;

pub struct UserEntity;

impl UserEntity {
    pub fn new() -> Self {
        let _ = OrderEntity::new();
        Self
    }
}
```

### 场景 2：`pub use` 对外收拢 API（lib / mod.rs 高频）

```rust
// domain/mod.rs
pub mod user;
pub mod order;

pub use user::UserEntity;
pub use order::OrderEntity;
```

`pub use` **同时**在本 `mod.rs` 内提供短名、对外重导出 — 不必再写 `use self::user::UserEntity`（同名会 E0252 冲突）。

外部：`use your_crate::domain::{UserEntity, OrderEntity};` — **不必知道内部 `user.rs` / `order.rs` 文件夹结构**。

→ 重导出详解 [7.4 §四](./7.4-使用use关键字将名称引入作用域.md)

### 场景 3：引入第三方 Package

```rust
// domain/mod.rs
use serde::Serialize;
```

crates.io 依赖的 Package，靠 `use` 拉进**当前文件**作用域。

---

## 破除误区

| | |
|---|---|
| ❌ | 写了 `pub mod a;`，当前文件就能直接用 `a` 里的类型名 |
| ✅ | `pub mod a;` 只是**注册模块**；想要简写，必须额外 **`use`**（或写全路径 `a::Type`） |

---

## 极简总结

- **`mod`**：建档、挂文件  
- **`use`**：拿名字、省代码  

所有 `.rs`（包括各个 `mod.rs`）里 **既有 `mod` 又有 `use` 是常态**。

**`use` 不跨文件** → [7.4.1 use 作用域](./7.4.1-use作用域仅限当前文件.md) · **`pub mod` ≠ 自动导入** → [7.5.2](./7.5.2-pub-mod登记与pub-use收拢.md)
