# 7.4.2 `use` vs `pub use`：内部简写 vs 重导出共享

← [7.4 use / pub use](./7.4-使用use关键字将名称引入作用域.md) · [7.4.1 use 仅限当前文件](./7.4.1-use作用域仅限当前文件.md) · [7.5.3 `mod` vs `pub mod`](./7.5.3-lib.rs里mod与pub-mod.md)

对应 demo：`7.4-use-demo/`（`lib.rs` 的 `pub use hosting`）· `7.5-split-modules-demo/src/domain/`（`pub use UserEntity`）

---

## 结论（先给你）

> **完全正确。** `use` / `pub use` 的公私逻辑，和 `mod` / `pub mod` **同一套规则**：不加 `pub` = 仅当前作用域；加 `pub` = 对外开放。

---

## 一、核心区分

| 写法 | 当前模块内 | 同 crate 其他 mod | crate 外 / 别的 Package |
|------|------------|-------------------|---------------------------|
| **`use …;`** | ✅ 简写可用 | ❌ **看不到**这个别名 | ❌ |
| **`pub use …;`** | ✅ 简写可用 | ✅ 经**模块路径** `use`（如 `crate::scanner::Token`） | ✅ 经 `包名::…` 路径 `use` |

### 1. `use`（不带 `pub`）

只在**当前文件 / 当前模块**内简化路径、导入名字；**外部模块、其他 crate 看不到**这个简写。

### 2. `pub use`（带 `pub`）

不仅当前模块能用，还把名字**提升到当前模块的出口**（重导出 re-export）—— 外部可通过模块路径访问，同时起到「路径上浮、少写一层」的作用。

> 普通 `use` = **拿过来自己用**；`pub use` = **拿过来，再转手分享给别人用**。

---

## 二、clox 风格举例：`scanner` + `Token`

目录：

```
src/
├── scanner/
│   ├── mod.rs
│   └── token.rs      // pub struct Token { … }
└── lib.rs
```

`token.rs`：

```rust
pub struct Token {
    pub lexeme: String,
    pub line: usize,
}
```

### 场景 1：普通 `use`（仅 scanner 内部）

`scanner/mod.rs`：

```rust
pub mod token;

// 不带 pub — 只在 scanner/mod.rs 本文件内简写
use self::token::Token;

fn do_something(t: Token) {}
```

| | |
|---|---|
| ✅ | `scanner/mod.rs` 本文件：可直接写 `Token` |
| ❌ | 同 crate 的 `parser`、`vm`：**看不到**这个别名，须 `crate::scanner::token::Token` 或自己 `use` |
| ❌ | 外部 crate：同上 |

→ `use` **不跨文件** → [7.4.1](./7.4.1-use作用域仅限当前文件.md)

### 场景 2：`pub use`（对外公开 + 路径上浮）

`scanner/mod.rs`：

```rust
pub mod token;

// 带 pub：把 Token 重导出到 scanner 模块顶层
pub use self::token::Token;
```

| | 路径 |
|---|------|
| ✅ scanner 内部 | 本文件可简写 `Token`；`token.rs` 仍须自己 `use` 或全路径 |
| ✅ 同 crate 其他 mod | `use crate::scanner::Token`（少一层 `token`） |
| ✅ 外部依赖本 crate | `use your_clox::scanner::Token` |

---

## 三、顶层 `lib.rs`：`pub use` 做 crate 级收拢

```rust
// lib.rs
pub mod scanner;

// 再往上导出一层 → 全 crate 可 use crate::Token
pub use scanner::Token;
```

之后 crate 内任意文件：

```rust
use crate::Token;
```

企业 lib 常见模式：**内部层级 `scanner/token.rs` 藏起来，对外只暴露 `your_crate::Token`** → 与 [7.1.5 `lib.rs` 出口](./7.1.5-lib.rs是Crate出口.md) 一致。

---

## 四、补充细节

### 1. 可见性规则和 `mod` 完全统一

| 不加 `pub` | 加 `pub` |
|------------|----------|
| `mod` / `use` → **私有**，仅限当前作用域 | `pub mod` / `pub use` → **公开**，外部可经路径访问 |

一套规则记到底 → [7.5.3](./7.5.3-lib.rs里mod与pub-mod.md) · [7.2 私有性](./7.2-定义模块来控制作用域与私有性.md)

### 2. 私有父模块里的 `pub use` 出不去

`lib.rs`：

```rust
mod scanner;   // 私有 — 无 pub
```

哪怕 `scanner/mod.rs` 里写了 `pub use token::Token`，**crate 外依旧访问不到** `scanner` 这棵子树。

> **父模块私有，内部再怎么 `pub` / `pub use` 也无法越过 crate 边界。**

compile-fail 对照 → [7.5.3 §五](./7.5.3-lib.rs里mod与pub-mod.md)

### 3. `use` 导入 vs `pub use` 重导出

| | 普通 `use` | `pub use` |
|---|------------|-----------|
| 本质 | 路径别名，**仅本文件** | 把项挂到**当前模块 API** |
| 是否改模块树 | ❌ | ❌（树仍靠 `mod` 建） |
| 兄弟 `.rs` 能继承吗 | ❌ 各自写 `use` | ❌ 外部须 `use 模块路径::名字` |

`pub mod` 登记文件 vs `pub use` 上浮名字 → [7.5.2](./7.5.2-pub-mod登记与pub-use收拢.md)

---

## 五、与 `mod` / `pub mod` 对照（一张表记全）

| 关键字 | 不带 `pub` | 带 `pub` |
|--------|------------|----------|
| **`mod`** | 挂进模块树，**crate 内**可用 | 子模块对 **crate 外** 可见 |
| **`use`** | **本文件**内简写 | **`pub use`**：重导出到当前模块出口，别人经路径 `use` |

---

## 一句话总结

| 目标 | 写法 |
|------|------|
| **自己内部用**，简化路径，对外隐藏 | `use …;` |
| **自己用 + 对外共享**，路径上浮、开放给别人 `use` | `pub use …;` |
