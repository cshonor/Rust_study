# 13.1.2 参数 vs 捕获（两套独立能力）

← [13.1.1 四段式语法](./13.1.1-闭包四段式语法.md) · [13.1 hub](./13.1-闭包.md) · 下一条 **[13.1.3 类型推导](./13.1.3-闭包类型推导易错点.md)** · demo：`13.1-closures-demo/`（`syntax_examples` §参数/捕获）

> **无参闭包照样能捕获外部变量** — **有没有形参**与**抓不抓外部变量**是两回事，互不干扰。

---

## 一、分清两个概念

| | **闭包形参 `\|n\|`** | **外部捕获** |
|--|----------------------|--------------|
| 写在哪 | `\|\|` **里面** | `\|\| { … }` **体内**直接用外层已有变量 |
| 谁提供 | 调用时**外部传入** | 从**当前作用域抓进来** |
| 与无参关系 | 无参 = 形参列表为空 | **与有没有形参无关**，无参也能捕获 |

---

## 二、无参闭包 + 外部捕获（三种常见）

### ① 不可变捕获（只读 → `Fn`）

```rust
let num = 100;
let cb = || println!("{}", num);

cb();
cb(); // 可多次调用
```

### ② 可变捕获（修改外部 → `FnMut`，`&mut` 借用）

> **无 `move` 永远拿不到所有权** — 改的是外面的变量 → [13.1.5 §一](./13.1.5-move闭包详解.md#一不加-move只有借用无所有权)

```rust
let mut num = 0;
let mut cb = || num += 1; // &mut 借用，所有权仍在外面

cb();
println!("{}", num); // 1
```

### ③ `move` 捕获（所有权进闭包，无参也生效）

> 想要所有权**必须写 `move`** → [13.1.5 §二–§三](./13.1.5-move闭包详解.md)

```rust
let s = String::from("hello");
let cb = move || println!("{}", s);

cb();
// println!("{}", s); // ❌ 外层 s 已 move（非 Copy）
```

→ 捕获规则 · Fn trait：[13.1.4 Fn 系列](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md) · `move` 详解：[13.1.5](./13.1.5-move闭包详解.md)

---

## 三、有参 + 捕获（最常见）

```rust
let base = 10;
let cb = |n: i32| base + n;

println!("{}", cb(5)); // 15
```

### 只有参数、无捕获

```rust
let add = |n: i32| n + 1;
println!("{}", add(2)); // 3
println!("{}", add(5)); // 6
```

### 只有捕获、无参数

```rust
let base = 10;
let add = || base + 1;
println!("{}", add()); // 11
```

---

## 四、一句话

- **有没有参数，不影响抓取外部变量** — 无参闭包就是 `|| { 直接用外面的变量 }`，编译器按借用 / `move` 规则自动处理捕获。
- **有参数**：动态入参、逻辑更复用；**无参数**：依赖固定环境。

> 闭包**形参** ≠ **捕获** — Fn trait 只看捕获怎么用 → [13.1.4 §三.4](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#4-其他误区与-capture-无关)
