## 一、迭代器基础定义

> 迭代器 = **对序列遍历逻辑的封装**，自动维护游标与边界终止，不用手写 `i`、不用 `i < len`。

### 惰性 Lazy（第一核心特征）

**只写链式逻辑（map/filter）= 只保存规则，不运算；只有消费（next/for/collect/sum）才真正逐个执行。**

```rust
let arr = [1, 2, 3];
let iter = arr.iter().map(|&x| x * 2).filter(|&v| v > 2);
println!("走到这里，map、filter 完全没执行");
let res: Vec<i32> = iter.collect();
// res = [4, 6]
```

通俗记：**定义不动，消费才跑** → 分步打印直观看：[13.2.4 惰性演示](./13.2.4-惰性演示-适配器与消费器.md)（`cargo run -- lazy`）

## 二、Iterator trait 拆解

```rust
pub trait Iterator {
    type Item; // 关联类型 → [10.2.4 type Item](../10-generics-traits-lifetimes/10.2.4-type关键字与关联类型.md)
    fn next(&mut self) -> Option<Self::Item>;
}
```

### ① `type Item`

**一个迭代器从头到尾只产出同一种 `Item`。**

| 生成方式 | `Item` |
|----------|--------|
| `arr.iter()` | `&i32`（不可变借用） |
| `arr.into_iter()` | `i32`（所有权） |

### ② `fn next(&mut self) -> Option<Self::Item>`

1. **`&mut self`**：修改内部游标 → 迭代器变量必须 **`mut`**
2. 返回规则：

| 返回值 | 含义 |
|--------|------|
| `Some(val)` | 取出元素，游标 +1 |
| `None` | 遍历结束；之后再调永远 `None` |

```rust
let arr = [1, 2, 3];
let mut iter = arr.iter();
assert_eq!(iter.next(), Some(&1));
assert_eq!(iter.next(), Some(&2));
assert_eq!(iter.next(), Some(&3));
assert_eq!(iter.next(), None);
assert_eq!(iter.next(), None);
```

> **只实现 `next`**，自动获得 `map` / `filter` / `zip` / `sum` / `enumerate` 等默认方法。

## 三、适配器（惰性）+ 消费器（触发执行）

→ **比喻 · 定义 · 对照表**：[13.2.2 适配器与消费器详解](./13.2.2-适配器与消费器详解.md) · **三者关系 · 调用链路**：[13.2.3](./13.2.3-三者关系与调用链路.md) · **分步打印 demo**：[13.2.4](./13.2.4-惰性演示-适配器与消费器.md)（`cargo run -- lazy`）

### 1. 适配器 Iterator Adaptors

**惰性、返回新迭代器、不消耗原迭代器** — 只包装规则。

`Filter` 伪实现原理：

```rust
struct Filter<I, P> { inner: I, pred: P }

impl<I: Iterator, P> Iterator for Filter<I, P> {
    type Item = I::Item;
    fn next(&mut self) -> Option<Self::Item> {
        loop {
            match self.inner.next() {
                Some(v) if (self.pred)(&v) => return Some(v),
                Some(_) => continue,
                None => return None,
            }
        }
    }
}
```

```rust
let chain = vec![1, 2, 3, 4, 5].iter()
    .map(|&x| x + 1)
    .filter(|x| *x % 2 == 0);
// 此时仍无运算
```

### 2. 消费器 Consuming Adaptors

**消耗迭代器、触发 next、用完作废：**

```rust
let s: i32 = chain.sum();
// chain 已消费，不能再次 collect
```

| 消费器 | 说明 |
|--------|------|
| `collect` / `sum` / `count` / `fold` | 显式消费 |
| `for x in iter` | 编译器自动循环 `next()` 直到 `None` |

## 四、iter / iter_mut / into_iter（高频易混）

→ **完整版（签名 · 踩坑 · `for` 语法糖 · zip/collect）**：[13.2.5 三种迭代生成方式](./13.2.5-三种迭代生成方式.md)

| 方法 | `Item` | 权限 | 原集合之后 |
|------|--------|------|------------|
| **`iter()`** | `&T` | 不可变借用 | ✅ 仍可用 |
| **`iter_mut()`** | `&mut T` | 可变借用 | ✅ 仍可用（遍历中勿别名） |
| **`into_iter()`** | `T` | move 所有权 | ❌ 集合失效 |

```rust
let mut arr = vec![10, 20, 30];
for v in arr.iter() { println!("{}", v); }
for v in arr.iter_mut() { *v += 1; } // [11, 21, 31]
let owned: Vec<i32> = arr.into_iter().collect();
// dbg!(&arr); // ❌ 所有权已转移
```

> **`for x in arr`** 等价 **`for x in arr.into_iter()`**（默认拿走所有权）。

→ 与 [8.3.1 zip/collect](../08-collections/8.3.1-zip-collect与HashMap.md) 同一套所有权逻辑。

## 五、impl Iterator vs Box<dyn Iterator>

### 1. 适配器链返回值 → `impl Iterator`

编译器生成匿名结构体，**无法手写类型名**：

```rust
fn get_iter() -> impl Iterator<Item = i32> {
    vec![1, 2, 3].into_iter().map(|x| x * 2)
}
// ❌ 一般不用 dyn：迭代器 Sized，且静态分发更快
```

### 2. 运行期多种迭代器 → `Box<dyn Iterator>`

```rust
fn get_dyn(flag: bool) -> Box<dyn Iterator<Item = i32>> {
    if flag {
        Box::new(vec![1, 2].into_iter())
    } else {
        Box::new((0..5).into_iter())
    }
}
```

→ [10.2.5 impl vs dyn](../10-generics-traits-lifetimes/10.2.5-impl与dyn-Trait对比.md)

## 六、自定义迭代器：CounterRange

```rust
struct CounterRange { curr: usize, max: usize }

impl Iterator for CounterRange {
    type Item = usize;
    fn next(&mut self) -> Option<Self::Item> {
        if self.curr < self.max {
            let res = self.curr;
            self.curr += 1;
            Some(res)
        } else {
            None
        }
    }
}

// CounterRange { curr: 0, max: 3 } → 0, 1, 2, None
// CounterRange { curr: 1, max: 5 }.map(|x| x * 2).sum() → 20
```

demo 里还有书版 **`Counter`（1～5）** + `zip`/`skip` 链式示例。

## 七、一句话总结

1. **惰性** = 适配器存逻辑，消费器触发 `next` 才执行
2. **Iterator** 只强制写 `next`，`Item` 定类型，其余方法白送
3. **iter 借引用、iter_mut 借可变、into_iter 拿所有权**
4. 普通返回 **`impl Iterator`**；异构分支 → **`Box<dyn Iterator>`**

4. **改错**：`fn f() -> dyn Iterator<Item = i32> { (0..3).into_iter() }` 错在哪？
5. **填空**：集成测试站在外部视角，只能调用库的 ______ API（对比单元测试可测私有函数 → 见 [11.3](../11-testing/11.3-测试的组织结构.md)）。

3. `into_iter`  
4. 返回位不能裸 `dyn`，须 `Box<dyn Iterator<Item = i32>>` 等指针包裹；且函数体类型须一致
