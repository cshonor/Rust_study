# 13.1.3 闭包类型推导易错点

← [13.1.2 参数 vs 捕获](./13.1.2-参数与捕获.md) · [13.1 hub](./13.1-闭包.md) · 下一条 **[13.1.4 Fn 系列](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)** · demo：`13.1-closures-demo/`（`type_inference_examples`）

> Rust 闭包默认**不写类型**，靠**第一次调用 / 使用上下文**推导参数与返回类型。

---

## 一、错误：只定义，不调用、不标注

```rust
let add = |x| x + 1;
// ❌ E0282: type annotations needed — 无法推断 x 的类型
```

**原因**

闭包 `|x| x + 1` 里 `x` 可以是 `i32`、`i64`、`f64`……编译器**没有任何上下文**，不知道选哪种，直接拒绝编译。

---

## 二、正确方式一：至少调用一次

```rust
let add = |x| x + 1;
add(5i32); // ✅ 推导出 x: i32，返回 i32
```

一旦用 `i32` 调用过，`add` **就被固定为接受 `i32` 的闭包**，再传别的类型会报错：

```rust
// add(3.14); // ❌ expected i32, found floating-point
```

→ 与 [13.1 §一](./13.1-闭包.md#类型锁定易错) · [13.1.1 §二](./13.1.1-闭包四段式语法.md) 同一机制：**首次使用锁定类型**。

---

## 三、正确方式二：直接写全类型标注（最稳妥）

```rust
let add = |x: i32| -> i32 { x + 1 }; // ✅ 明确，无需调用也能编译
add(5);
```

参数类型、返回类型、多语句体 — 四段式演变见 [13.1.1](./13.1.1-闭包四段式语法.md)。

---

## 四、类比（好记）

| 场景 | 空上下文 | 解决 |
|------|----------|------|
| `let v = Vec::new();` | ❌ 不知元素类型 | `Vec<i32>` 或立刻 `push(5)` |
| `let add = \|x\| x + 1;` | ❌ 不知 `x` 类型 | `\|x: i32\|` 或立刻 `add(5i32)` |

**空上下文 → 无法推导 → 必须标注或调用一次。**

---

## 五、速查表

| 情况 | 能否编译 | 做法 |
|------|----------|------|
| 只写 `\|x\| x + 1`，从不调用 | ❌ | 加类型或调用一次 |
| 调用 `add(5i32)` 后再 `add(3.14)` | ❌ | 类型已锁 i32 |
| `\|x: i32\| -> i32 { x + 1 }` | ✅ | 最稳妥 |
| 传给 `Fn(i32) -> i32` 参数 | ✅ | 签名提供上下文，可不写闭包类型 |
| 存入 `Cacher<T: Fn(u32) -> u32>` | ✅ | 泛型 bound 提供上下文 |
