## 一、map + collect（最核心）

闭包里加 `println!`，观察**执行时机**：

```rust
let v = vec![1, 2, 3];

let iter = v.iter().map(|x| {
    println!("正在处理元素: {}", x); // 日志
    x * 2
});

println!("===== 构建完迭代器，还没消费 =====");

let result: Vec<_> = iter.collect(); // 消费器：从这里才开始跑
println!("最终结果: {:?}", result);
```

**典型输出：**

```
===== 构建完迭代器，还没消费 =====
正在处理元素: 1
正在处理元素: 2
正在处理元素: 3
最终结果: [2, 4, 6]
```

→ 先打印分割线 → **`map` 写完时内部 `println` 没跑**；`collect()` 才逐个执行。

## 二、链式适配器：map + filter

多个适配器连续调用 = **层层包装规则**，依然惰性：

```rust
let iter = v.iter()
    .map(|x| {
        println!("map 处理: {}", x);
        x * 2
    })
    .filter(|&x| {
        println!("filter 判断: {}", x);
        x > 5
    });

println!("===== 规则组装完成，未执行 =====");
let res: Vec<_> = iter.collect();
```

**典型输出（节选）：**

```
===== 规则组装完成，未执行 =====
map 处理: 1
filter 判断: 2
map 处理: 2
filter 判断: 4
...
筛选结果: [6, 8, 10]
```

→ 消费时按**流水线顺序**逐个元素执行（filter 可能提前跳过，见 Filter 伪实现 [13.2.1 §三](./13.2.1-迭代器精讲.md)）。

## 三、skip / take

| 适配器 | 作用 |
|--------|------|
| `skip(n)` | 跳过前 `n` 个 |
| `take(n)` | 只取前 `n` 个 |

```rust
let v = vec![10, 20, 30, 40];

let iter = v.iter().skip(2);
println!("组装完成，未执行");
let res: Vec<_> = iter.collect(); // [30, 40]

let iter = v.iter().take(2);
let res: Vec<_> = iter.collect(); // [10, 20]
```

## 四、不同消费器（都触发执行）

| 消费器 | 作用 |
|--------|------|
| `collect()` | 收集到集合 |
| `sum()` | 求和 |
| `for x in iter` | 循环（反复 `next()`） |
| `while let Some(x) = iter.next()` | 手动逐个 `next()`（最底层） |
| `count()` | 计数 |

```rust
// sum
let iter = v.iter().map(|x| x * 2);
let total: i32 = iter.sum(); // 2+4+6 = 12

// for
for num in iter { println!("元素: {}", num); }

// count
let cnt = v.iter().filter(|&&x| x % 2 == 0).count(); // 偶数个数
```

## 五、三种消费器同链对比（while let / for / collect）

**同一条** `map + filter` 流水线，分别用三种消费器收尾——观察**谁触发执行、输出长什么样**。

```rust
let v = vec![1, 2, 3, 4, 5];

// 闭包里 println!，看清每次 next() 驱动 pipeline 走一步
fn traced(v: &Vec<i32>) -> impl Iterator<Item = i32> + '_ {
    v.iter()
        .map(|x| { println!("  map: {}", x); x * 2 })
        .filter(|&x| { println!("  filter: {}", x); x > 5 })
}

// A) while let — 每轮循环显式 next()，中间可 println
let mut iter = traced(&v);
while let Some(x) = iter.next() {
    println!("  while let 拿到: {}", x);
}

// B) for — 语法糖，内部等价于反复 next()
for x in traced(&v) {
    println!("  for 拿到: {}", x);
}

// C) collect — 内部循环 next() 直到 None，只出最终 Vec
let res: Vec<_> = traced(&v).collect();
println!("  collect 成品: {:?}", res);
```

**结论（背下来）：**

| 消费器 | 本质 | 特点 |
|--------|------|------|
| `while let … next()` | 手动驱动 | 最灵活，可控制每步时机 |
| `for x in iter` | `while let` 语法糖 | 写法最简，日常首选 |
| `collect()` | 内部 `while` 收齐全部 | 一次出集合，中间不插手 |

三种方式**都会**逐元素跑完适配器链；差别只在**你怎么写循环、要不要中途插手**。

→ 概念拆解：[13.2.2 §四](./13.2.2-适配器与消费器详解.md#四while-let-somex--iternext手动消费)

## 六、消费后不能复用

迭代器被消费（`collect`/`sum`/`for`/`while let`）后**耗尽**，不能二次使用：

```rust
let mut iter = v.iter().map(|x| x * 2);

let res1: Vec<_> = (&mut iter).collect(); // 耗尽迭代器，但变量仍在
println!("第一次: {:?}", res1);

let res2: Vec<_> = iter.collect();
println!("第二次: {:?}", res2); // [] — 已空
```

---

## 七、极简总结

1. **适配器**（`map`/`filter`/`skip`/`take`）：只写逻辑、不运行 — **搭流水线**
2. **消费器**（`collect`/`sum`/`for`/`while let … next()`/`count`）：**按开关** — 启动流水线
3. **核心规则**：流水线搭好不动，**按下开关（消费）才开工**
