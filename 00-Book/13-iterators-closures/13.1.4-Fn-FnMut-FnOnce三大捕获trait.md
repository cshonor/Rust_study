# 13.1.4 三大捕获 Trait：Fn / FnMut / FnOnce

← [13.1.3 类型推导](./13.1.3-闭包类型推导易错点.md) · [13.1 hub](./13.1-闭包.md) · 下一条 **[13.1.5 move 闭包](./13.1.5-move闭包详解.md)** · demo：`13.1-closures-demo/`（`closure_capture_examples` · `fn_trait_examples`）

> **先拆开两大概念（互不绑定）**  
> - **捕获规则**：默认**借用**（`&T` / `&mut T`）；`move` = **强制所有权进闭包** → [13.1.5](./13.1.5-move闭包详解.md)  
> - **Fn / FnMut / FnOnce**：编译器看**怎么用**捕获变量（只读 / 修改 / 调用时消耗），自动 impl  
>  
> **核心结论**  
> 1. **`Fn` / `FnMut` 绝大多数只用借用，不拿所有权**；只有调用时**消耗**捕获才往往只剩纯 `FnOnce`。  
> 2. **`move` ≠ `FnOnce`** — 只转移所有权进闭包，不决定 trait。  
> 3. **不加 `move` = 默认借用**，通常得到 `Fn` 或 `FnMut`。

---

## 一、三个 trait 区别（按捕获规则）

| Trait | 语义 | 触发条件 | 闭包实际 impl |
|-------|------|----------|---------------|
| **`FnOnce`** | 至少能调用一次；消耗捕获时**只能一次** | 拿走所有权 / 消耗不可复制变量；**默认所有闭包都有** | 必有 |
| **`FnMut`** | 可多次调用，**能改**捕获变量 | 可变借用 `&mut T` 捕获 | `FnMut` + `FnOnce`，**不含 Fn** |
| **`Fn`** | 可多次调用，**只读**捕获 | 不可变借用 / 无捕获 / 只捕获 `Copy` | `Fn` + `FnMut` + `FnOnce` |

### Trait 层级 → 详见 [§二 Trait 继承](#二trait-继承约束由严到宽)

---

## 二、Trait 继承：约束由严到宽

> **不是「权限高低」**，而是 **trait 继承 + 调用能力契约**：参数 bound **由严到宽**；`FnOnce` 是**最底层父 trait**（要求最少、适用范围最广）。

### 标准库定义（源码）

```rust
trait FnOnce() {}
trait FnMut(): FnOnce() {}
trait Fn(): FnMut() {}
```

人话：

| 继承 | 含义 |
|------|------|
| **`Fn` : `FnMut`** | impl `Fn` → 自动满足 `FnMut` |
| **`FnMut` : `FnOnce`** | impl `FnMut` → 自动满足 `FnOnce` |

反过来：

- 是 **`Fn`** → 一定也是 **`FnMut`**、**`FnOnce`**
- 是 **`FnMut`** → 一定也是 **`FnOnce`**
- 是 **纯 `FnOnce`** → **未必**是另外两个

```
Fn  ⊂  FnMut  ⊂  FnOnce     （子 trait 约束更严 · 父 trait bound 更宽）
```

---

### 三个 trait 的「调用能力契约」

与**有没有所有权**是两回事 — 描述**允许多少次、怎样调用**：

| Trait | 契约（允许多少） |
|-------|------------------|
| **`FnOnce`** | **至少能调用一次**（门槛最低） |
| **`FnMut`** | **可反复调用**，且调用时可**修改**捕获 |
| **`Fn`** | **可反复调用**，且调用时**不修改**捕获 |

**核心逻辑**：**可多次调用 ⊇ 可调用一次**

---

### 为什么 `FnMut` 也必然 impl `FnOnce`？

`FnMut` 靠**可变借用**，变量不被消耗，可以 `f(); f(); …`  
→ 能调无数次 → **当然也能调至少一次** → 满足 `FnOnce` 契约。

```rust
let mut x = 0;
let mut f = || x += 1; // FnMut：可变借用，无所有权转移

f(); // 第 1 次 → 符合 FnOnce
f(); // 第 2 次 → 符合 FnMut
// 自动 impl：FnMut + FnOnce（不含 Fn）
```

> **Q：FnMut 没拿所有权，为啥还要 FnOnce？**  
> **A：捕获方式**决定闭包**属于哪一类**；**trait 继承**描述**调用能力包含关系** — 能多次 ⇒ 必能一次。

---

### 为什么 `Fn` 也 impl `FnMut` + `FnOnce`？

- 能**无限次只读调用** → 满足「可多次调用」→ **`FnMut`**
- 能多次 → 也满足「至少一次」→ **`FnOnce`**

类比：

| Trait | 类比 |
|-------|------|
| **`Fn`** | 公共**阅览室**：反复来、不能乱动书 |
| **`FnMut`** | **自习室**：反复来、能写写画画 |
| **`FnOnce`** | **临时落脚点**：能进来坐**一次**就行 |

能当阅览室 → 当然也能当自习室、临时落脚点。

```rust
let a = 10;
let f = || println!("{}", a); // Fn

f(); // 一次 → FnOnce ✓
f(); // 多次只读 → Fn / FnMut ✓
```

---

### 捕获方式 ≠ Trait 继承（再强调）

| 维度 | 决定什么 |
|------|----------|
| **捕获**（借用 / move / 是否消耗） | 闭包**属于** Fn / FnMut / **纯** FnOnce |
| **继承**（语法父子关系） | impl 子 trait 时**连带** impl 父 trait |

- **纯 `FnOnce`**：`move || drop(s)` — 调一次消耗，**不能**当 FnMut/Fn
- **`Fn` / `FnMut`**：借用、可多次 → **必然**连带 impl `FnOnce`

**一句话**：「只能用一次」是最低门槛；「能用很多次」一定包含「能用一次」。

---

### 反向验证

| 闭包类型 | 能当 `FnOnce`？ | 能当 `FnMut`？ | 能当 `Fn`？ |
|----------|-----------------|----------------|-------------|
| 纯 **`FnOnce`** | ✅ | ❌ | ❌ |
| **`FnMut`** | ✅ | ✅ | ❌ |
| **`Fn`** | ✅ | ✅ | ✅ |

## 三、捕获规则 vs Fn trait（分开看）

### 日常对照表（够用）

| 写法 | 捕获方式 | 最终 trait | 多次调用 |
|------|----------|------------|----------|
| 无 `move`，只读外部 | 不可变借用 `&T` | **`Fn`** 全套 | ✅ |
| 无 `move`，修改外部 | 可变借用 `&mut T` | **`FnMut` + `FnOnce`** | ✅ |
| `move`，内部只读 | 所有权在闭包内 | **`Fn`** 全套 | ✅ |
| `move`，调用时消耗变量 | 所有权 + 消耗 | **仅 `FnOnce`** | ❌ 一次 |

---

### 1. 不加 `move`（默认：借用，不拿所有权）

```rust
let s = String::from("hi");
let c = || println!("{}", s); // &T 只读借用
c(); c();
// Fn 全套 — 全程没拿 s 的所有权，外面 s 仍可用
```

```rust
let mut x = 0;
let mut c = || x += 1; // &mut T 可变借用
c(); c();
// FnMut + FnOnce — 也没拿所有权
```

👉 **借用 → `Fn` / `FnMut`**（主流路径）

---

### 2. 加 `move`（所有权进闭包 — 仍不直接等于 FnOnce）

**情况 A：move + 只读，调用不消耗** → 仍是 **`Fn`**，可多次调

```rust
let s = String::from("hi");
let c = move || println!("{}", s);
c(); c(); // ✅ 有 move、所有权在闭包里，但不是纯 FnOnce
// 外面 s 不可用；闭包内 s 每次调用都还在
```

**情况 B：move + 调用时消耗**（`drop` / 返回 / 转移）→ **仅 `FnOnce`**

```rust
let s = String::from("hi");
let c = move || drop(s);
c();
// c(); // ❌ 仅 FnOnce
```

👉 **`move + 调用就消耗` → 纯 `FnOnce`**（日常最常见的纯 FnOnce 组合）

---

### 3. 两个常见问题

**Q1：三个 trait 都必须拿所有权吗？**

**不是。** `Fn` / `FnMut` 主流是**借用**；只有调用时销毁/转移捕获，才往往变成**纯 `FnOnce`**（此时才一定涉及所有权消耗）。

**Q2：纯 `FnOnce` 一定绑 `move` 吗？**

**语法上不等价**；但日常几乎总是 **`move` + 消耗** 才出现纯 `FnOnce` — 因为默认借用不会被「调一次就失效」。不加 `move` 也能 FnOnce（极少），了解即可。

---

### 4. 其他误区（与 capture 无关）

| 误区 | 正解 |
|------|------|
| 无参闭包 = `Fn` | 看**捕获**，不看形参 — 详见 [13.1.2](./13.1.2-参数与捕获.md) |
| 在循环里 = `Fn` | 能循环调 → 至少 `Fn`/`FnMut`；纯 `FnOnce` 进不了 |

```rust
let mut x = 0;
let mut c = |v: &mut i32| { x += *v; }; // 改捕获 x → FnMut
```

---

### 5. 一句话理顺

1. **不加 `move` → 永远是借用**（`&T` / `&mut T`），**永远拿不到所有权**。  
2. **无 `move` + 修改** → 改**原作用域变量**，靠 `&mut` → 通常 **`FnMut`**。  
3. **`move` + 修改** → 改**闭包内副本/自有变量**（`Copy` 外面仍可用但值不变；非 `Copy` 外面失效）。  
4. **`move` = 所有权进闭包**，与 Fn/FnMut/FnOnce **无强制绑定**。  
5. **`move` + 调用时销毁/转移** → 纯 **`FnOnce`**。

## 四、通用判断流程（编译器同逻辑）

```
① 调用时会消耗捕获的所有权？  → 仅 FnOnce
        ↓ 否
② 会修改捕获的外部变量？      → FnMut + FnOnce（无 Fn）
        ↓ 否
③ 无捕获 / 只读借用 / Copy / move 后只读  → Fn + FnMut + FnOnce
```

| 步骤 | 条件 | impl 结果 |
|------|------|-----------|
| **①** | `drop(捕获)`、返回捕获值、传给要所有权的函数等 | **仅 `FnOnce`** |
| **②** | 对捕获变量赋值、`+=` 等修改 | **`FnMut` + `FnOnce`** |
| **③** | 既不消耗也不修改 | **`Fn` 全套** |

---

## 五、代码速查（与 §三 对照）

### 1. 无捕获 / 只读 → 实现 `Fn`（连带 `FnMut` + `FnOnce`）

```rust
let x = 10;
let c = || println!("{}", x); // 不可变借用
c();
c();
// c：Fn + FnMut + FnOnce
```

### 2. 可变捕获 → 实现 `FnMut` + `FnOnce`，**不实现 `Fn`**

```rust
let mut x = 10;
let mut c = || x += 1; // 可变借用
c();
c();
// c：FnMut + FnOnce，**没有 Fn**
```

### 3. move + 消耗 → 仅 `FnOnce`（§三.2B）

```rust
let s = String::from("hi");
let c = move || drop(s);
c();
```

### 口诀

> **借用 → Fn / FnMut** · **move+消耗 → 纯 FnOnce** · **`move` 单独 ≠ FnOnce**

## 六、编译器自动 impl（底层逻辑）

编译器分析闭包的**变量捕获方式**和**内部行为**，为每个闭包的**匿名类型**自动生成对应 `impl`，规则：

| 编译器 impl | 连带 impl | 不 impl |
|-------------|-----------|---------|
| **`Fn`** | `FnMut` + `FnOnce` | — |
| **`FnMut`** | `FnOnce` | **`Fn`** |
| **仅 `FnOnce`** | — | **`FnMut` · `Fn`** |

### 判断依据

**1. 是否 impl `Fn`**

- 不可变借用外部变量、**无捕获**、或只捕获 **`Copy`** 类型
- 闭包体内**不修改**任何捕获值  
→ 编译器自动 impl **`Fn`**（连带 `FnMut` + `FnOnce`）

**2. 是否 impl `FnMut`（且无 `Fn`）**

- 通过 **`&mut T`** 可变借用捕获，体内会**修改**捕获值  
→ 自动 impl **`FnMut` + `FnOnce`**，**不 impl `Fn`**

**3. 是否仅 impl `FnOnce`**

- 调用时会**消耗**捕获的所有权（`drop`、返回捕获值、转移给要 `T` 的函数等）  
→ **只 impl `FnOnce`**

> **`move` 本身不够** — 必须再结合「调用时是否消耗」；move 后只读打印 → 仍可能是 `Fn`（§三.2A）。

→ trait **继承**为何 FnMut 连带 FnOnce：§二

### 补充三点

**① 闭包是匿名类型 — 无法手写 impl**

```rust
let c = || println!("hi");
// ❌ 不存在：impl Fn() for ??? { ... }
// 闭包类型由编译器生成，trait impl 全自动
```

**② 普通函数也实现这三个 trait**

独立 `fn`、方法默认 impl **`Fn`**（连带 `FnMut` / `FnOnce`），与**无捕获闭包**表现一致：

```rust
fn greet() { println!("hi"); }
// greet 可当 Fn() 传给 take_fn(greet)
```

**③ 开发者能「主动控制」的只有泛型 bound**

不用写 impl，但在接收闭包时**限定**要哪一类：

```rust
fn take_fn<F: Fn()>(f: F) {
    f();
} // 只读、可多次

fn take_fnmut<F: FnMut()>(mut f: F) {
    f();
} // 可改捕获值 — 注意 mut f

fn take_fnonce<F: FnOnce()>(f: F) {
    f();
} // 最宽，Fn / FnMut / FnOnce 都能传
```

→ 传参升级/降级规则见 §七（与 §二 继承一致）。

**一句话**：只写闭包逻辑，**捕获规则决定 impl 哪几个 trait，编译器代劳**；你在 API 上用 `Fn` / `FnMut` / `FnOnce` 做约束即可。

## 七、传参约束（易错：Trait 不能降级）

函数泛型 bound 决定**能传哪种闭包** — 与 §二 **继承链**一致：**只能从子 trait 升级到父 trait（bound 变宽）**，不能降级。

| 参数要求 | 可传 | 不可传 |
|----------|------|--------|
| **`Fn`** | 只读闭包（`Fn`） | `FnMut` · 仅 `FnOnce` |
| **`FnMut`** | `FnMut` · `Fn` | 仅 `FnOnce` |
| **`FnOnce`** | **`Fn` · `FnMut` · `FnOnce`**（最宽） | — |

### 错误：降级失败

```rust
fn need_fn<F: Fn()>(f: F) {
    f();
}

let mut x = 0;
let f = || {
    x += 1;
}; // FnMut
// need_fn(f); // ❌ expected Fn, found FnMut
```

### 正确：升级允许

```rust
fn need_fnonce<F: FnOnce()>(f: F) {
    f();
}

let s = String::from("a");
let f = || println!("{}", s); // Fn（只读）
need_fnonce(f); // ✅ Fn 自动满足 FnOnce
```

> **铁律**：`Fn → FnMut → FnOnce` 方向可传；**反向不行**。

## 八、`move` 与 trait（补充）

→ move 语法：[13.1.5](./13.1.5-move闭包详解.md)

**关键**：`move` 解决**谁拥有捕获变量**；**Fn 系列**看**怎么用**捕获变量。

```rust
let x = 10i32; // Copy
let f = move || println!("{x}");
f(); f(); // Copy 复制 → Fn
println!("{x}");

let s = String::from("hi");
let g = move || println!("{s}"); // move 进闭包，但只读 → 仍可 Fn，可多次调
g(); g();
// println!("{s}"); // ❌ 外层 s 已 move 走
```

## 九、速查表（一页背）

| 要点 | 内容 |
|------|------|
| **继承链** | `Fn : FnMut : FnOnce` — **约束由严到宽**，非权限高低 |
| **核心逻辑** | **可多次 ⊇ 可一次** → FnMut/Fn 必连带 FnOnce |
| **两大概念** | **捕获规则**（借用/move）≠ **Fn trait** |
| **Fn/FnMut 要所有权吗？** | **否** — 主流**借用** |
| **无 move** | 借用 → **`Fn` / `FnMut`** |
| **move + 只读** | 所有权在闭包内 → 仍 **`Fn`** |
| **move + 消耗** | **仅 `FnOnce`** |
| **move = FnOnce？** | **否** |
| **判定三步** | 消耗 → 修改 → 只读（§四） |
| **bound** | `Fn ⊂ FnMut ⊂ FnOnce` |

---

## 十、极简总结

1. **继承链** `Fn → FnMut → FnOnce`：越往右 **bound 越宽**；`FnOnce` 是**父 trait**（要求最少）。  
2. **可多次调用 ⊇ 可调用一次** — 借用型 Fn/FnMut **必然** impl `FnOnce`。  
3. **捕获规则**与**继承**分开看（§三）；纯 FnOnce ≈ 调用时消耗所有权（§三.2B）。  
4. API bound：**只能升级**（§七）；编译器自动 impl（§六）。
