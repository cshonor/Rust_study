# 13.1.5 `move` 闭包：强制捕获所有权

← [13.1.4 Fn 系列](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md) · [13.1 hub](./13.1-闭包.md) · [13.2 迭代器](./13.2-使用迭代器处理元素序列.md) · demo：`13.1-closures-demo/`

> **先给结论：不加 `move`，永远拿不到所有权**；修改外部变量靠的是**可变借用 `&mut T`**。  
> **想要所有权 → 必须显式写 `move`**。  
> `move` 管**捕获规则**；`Fn` 系列管**怎么用**捕获 — 二者互不绑定 → [13.1.4 §三](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#三捕获规则-vs-fn-trait分开看)

---

## 一、不加 `move`：只有借用，无所有权

Rust 闭包默认**借用捕获**：

| 闭包内用法 | 捕获方式 | 典型 trait |
|------------|----------|------------|
| 只读 | `&T` 不可变借用 | `Fn` |
| 修改 | `&mut T` 可变借用 | `FnMut` |

全程变量**所有权还在原作用域**，闭包只是「借过来用」。

```rust
let mut a = 1;
let mut f = || a += 1; // 无 move：&mut 借用，改的是外面的 a
f();
println!("{}", a); // 2 — 所有权始终在外面
```

```rust
let s = String::from("hello");
let f = || println!("{}", s);
f();
println!("{}", s); // ✅ 只读借用，s 仍可用
```

> 「消耗捕获 → FnOnce」看**调用时是否销毁/转移**，不是默认借用路径 — 详见 [13.1.4](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)。

---

## 二、想拿所有权，**必须写 `move`**

`move` **强制**把变量所有权**转移进闭包**，与改不修改无关：

```rust
let s = String::from("hello");
let f = move || println!("{}", s);
f();
// println!("{}", s); // ❌ E0382：非 Copy，外面已不可用
```

| 类型 | `move` 后原变量 |
|------|-----------------|
| **`Copy`**（`i32`） | 仍可用 — 拷贝进闭包，原值还在 |
| **非 `Copy`**（`String`） | **失效**，不能再读 |

→ **move 不直接等于 FnOnce** — 只读 move 仍可能是 `Fn`：[13.1.4 §三.2A](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#2-加-move所有权进闭包--仍不直接等于-fnonce)

---

## 三、关键区分：改的是「外面」还是「闭包内」

**修改行为**分两种场景：

| 场景 | 实际改谁 | 机制 |
|------|----------|------|
| **无 `move`** | **原作用域的变量** | `&mut` 借用，无所有权转移 |
| **有 `move` + `Copy`** | **闭包内部的副本** | 拷贝进闭包；外面仍能用，但**值不变** |
| **有 `move` + 非 `Copy`** | **闭包内部拥有的那份** | 所有权已归闭包；外面不可用 |

```rust
// 无 move：改外面的 a
let mut a = 1;
let mut f = || a += 1;
f();
assert_eq!(a, 2);

// move + Copy：改闭包内副本，外面 a 仍是 1
let mut a = 1;
let mut f = move || a += 1;
f();
assert_eq!(a, 1); // 外面没被改

// move + 非 Copy：外面彻底失效
let mut s = String::from("hi");
let f = move || s.len();
f();
// s 已不可用
```

**铁律**：**不写 `move` → 一定是借用**；**想要所有权 → 必须显式 `move`**。

### 联动 Fn trait

| 写法 | 典型 trait |
|------|------------|
| 无 `move` + 修改外部变量 | **`FnMut`**（`&mut` 借用） |
| `move` + 修改内部变量、不消耗 | 仍是 **`FnMut`** |
| `move` + 调用时销毁/转移变量 | **`FnOnce`** |

→ 完整对照表：[13.1.4 §三](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#三捕获规则-vs-fn-trait分开看)

## 四、什么时候必须用 `move`？

| 场景 | 原因 |
|------|------|
| **跨线程** `thread::spawn` | 线程可能比栈上变量活得更久，必须拥有数据 |
| **返回闭包** | 闭包要带走的变量不能借外层栈 |
| **`FnOnce` / 消耗捕获** | 如 `move \|\| v` 把 `Vec` 整个拿走 |

### 经典：线程必须 `move`

```rust
use std::thread;

fn main() {
    let v = vec![1, 2, 3];

    // ❌ 不 move：闭包只借 &v，主线程结束 v 可能已销毁 — 编译器通常直接报错
    // let h = thread::spawn(|| println!("{:?}", v));

    // ✅ move：v 所有权进线程，生命周期独立
    let h = thread::spawn(move || println!("{:?}", v));
    h.join().unwrap();
}
```

## 五、`move` 与 Fn trait（消耗场景）

- `move || { ... }` **常见**为 **`FnOnce`**（所有权被拿走，语义上可能只跑一次）
- 若捕获的是 **`Copy`** 且闭包内只读，仍可能同时满足 **`Fn`**

```rust
let v = vec![1, 2, 3];
let consume = move || v; // FnOnce：返回 v，消耗捕获
assert_eq!(consume(), vec![1, 2, 3]);
// consume(); // ❌ 第二次调用可能报错（FnOnce）
```

→ Fn 系列详解：[13.1.4 Fn / FnMut / FnOnce](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)

---

## 六、小结（直接背）

| 要点 | 一句话 |
|------|--------|
| 默认捕获 | **不写 `move` = 借用**（`&T` / `&mut T`），永远拿不到所有权 |
| 修改外部 | 无 `move` 靠 **`&mut`** 改原变量 → 通常 **`FnMut`** |
| `move` | **强制所有权**进闭包；`Copy` 是拷贝副本，非 `Copy` 外面失效 |
| 跨线程 | 几乎总是 `thread::spawn(move \|\| ...)` |
