# 10.2.8 多 Trait → 多虚表（`dyn T1+T2` 不生成新表）

← [10.2.7 胖指针与虚表](./10.2.7-dyn胖指针与虚表.md) · 阅读 **⑧** · [10.2 hub](./10.2-trait.md) · demo：`cargo run -- multi_vtable`

---

## 核心答案

**一个类型每实现一个不同的 `Trait`，就对应一张独立的虚表。**

类型 `A` 同时实现 **T1、T2、T3** → **3 张独立虚表**（互不干扰）。

---

## 一、演示代码

```rust
trait T1 { fn f1(&self); }
trait T2 { fn f2(&self); }
trait T3 { fn f3(&self); }

struct A;

impl T1 for A { fn f1(&self) {} }
impl T2 for A { fn f2(&self) {} }
impl T3 for A { fn f3(&self) {} }
```

---

## 二、虚表对应关系

Rust 为 **`A` 这个类型** 生成 **3 张虚表**（按 **Trait** 分，不是按实例）：

| 虚表 | 内容 |
|------|------|
| **A 的 T1 虚表** | 只存 `f1` 的函数指针 |
| **A 的 T2 虚表** | 只存 `f2` 的函数指针 |
| **A 的 T3 虚表** | 只存 `f3` 的函数指针 |

> **虚表与 Trait 一一绑定**（更准确：与 **(类型, Trait)** 对绑定），不是「一个类型只有一张表」。

同一类型的**所有 `A` 实例**仍共用各自的那 3 张表（类型级，见 [10.2.7 §三](./10.2.7-dyn胖指针与虚表.md)）。

---

## 三、运行时：`dyn T1` vs `dyn T2`

### 当成 `dyn T1`

```rust
fn use_t1(x: &dyn T1) {
    x.f1();
}
```

胖指针 vtable 字段 → **A 的 T1 虚表**，只能查 `f1`。

### 当成 `dyn T2`

```rust
fn use_t2(x: &dyn T2) {
    x.f2();
}
```

vtable 字段 → **A 的 T2 虚表**，只能查 `f2`。

**一句话：当前用哪个 `dyn Trait`，就挂载哪一张虚表。**

数据指针始终指向同一个 `A` 实例；换 `dyn Trait` = **换 vtable 字段**。

---

## 四、`dyn T1 + T2`（多 trait 叠加 · stable 注意）

**概念**：不会生成「T1+T2 合并虚表」；`f1` / `f2` 仍各查各 Trait 的虚表。

**Stable Rust 限制**：trait object **只能有一个主 non-auto trait**（另可加 `Send`/`Sync` 等 auto trait）。  
因此 **`&dyn T1 + T2`（T1、T2 均为独立 trait）通常编译不过**（E0225）。

常见写法：

```rust
// 写法 A：同一对象，分别转成 dyn T1 / dyn T2（两张虚表各用一次）
let a = A;
use_t1(&a as &dyn T1);
use_t2(&a as &dyn T2);

// 写法 B：supertrait — T2 继承 T1，dyn T2 可调 f1、f2
trait T2: T1 { fn f2(&self); }
fn use_as_t2(x: &dyn T2) {
    x.f1();
    x.f2();
}
```

> 学习阶段记：**多 Trait → 多虚表**；**不会**为 `T1+T2` 凭空合成一张新表。语法上 `dyn T1+T2` 受 object-safe / 单主 trait 规则约束。

---

## 五、易混总结

| 说法 | 对错 |
|------|------|
| 实现 N 个不同 Trait → N 张虚表 | ✅ |
| 虚表和 **Trait**（+类型）绑定，不是和「单个对象」绑定 | ✅ |
| 胖指针**同一时刻** vtable 字段指向**其中一张**（单 trait `dyn T`） | ✅ |
| `dyn T1+T2` 会合成一张新虚表 | ❌（且 stable 下独立双 trait 的 `dyn T1+T2` 常 E0225） |

---

## 六、超简记忆

Trait 是啥能力，虚表就是哪本地址簿：

- 会唱歌 (T1) → 唱歌簿  
- 会跳舞 (T2) → 跳舞簿  
- 会跑步 (T3) → 跑步簿  

有几项能力，就有几本簿。**用哪项，翻哪本。**
