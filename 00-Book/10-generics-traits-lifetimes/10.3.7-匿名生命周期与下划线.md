## 一句话说透

| | 含义 |
|--|------|
| **生命周期 `'a`** | 给引用的有效期**起个名字**，编译器和人都能看懂「谁活得比谁久」 |
| **匿名生命周期 `'_`** | **知道这里有生命周期，但不想起名** — 编译器自己推 |

---

## 1. 为什么会有生命周期？（极简）

Rust 保证：**引用永远不能比被引用的东西活得更久**。

```rust
let x = 10;
let y = &x; // y 借用 x
```

`y` 不能在 `x` 销毁后还存在。编译器用**生命周期标签**标记这段借用的有效期：

- 起名版：`'a`、`'b`
- 匿名版：`'_`

---

## 2. `'_` 到底是什么？

**不是新种类，是「省略命名」的语法糖。**

| 写法 | 含义 |
|------|------|
| `&'a str` | 明确：这个引用叫 `'a` |
| `&'_ str` | 这里有生命周期，**你编译器推，我不命名** |

等价：

```rust
fn f(s: &'_ str) { /* ... */ }
// 等价于
fn f<'a>(s: &'a str) { /* ... */ }
```

---

## 3. 什么时候写 `'_`？

**语法上要求写生命周期参数，但不想/不需要起名时。**

### 场景 1：结构体含引用，返回它

```rust
struct Wrapper<'a>(&'a str);

fn make_wrapper(s: &str) -> Wrapper<'_> {
    Wrapper(s)
}
```

`Wrapper<'_>`：返回类型**带生命周期**，由编译器从参数 `s` 推导，不必手写 `'a`。

### 场景 2：参数显式标「这里有借用」

```rust
fn print(s: &'_ str) { println!("{s}"); }
```

与 `&str` 编译行为相同，可读性上**强调存在借用**。

### 场景 3：嵌套泛型 / 复杂类型

```rust
fn take(state: State<'_, MyData>) { /* ... */ }
```

`State` 内部有生命周期 → `'_` = **存在，名字省略**。

### 场景 4：迭代器（实战高频）

```rust
fn iter(&self) -> Iter<'_, T>; // v.iter() 的返回类型
```

→ [13.2.6 Iter / IterMut / IntoIter](../13-iterators-closures/13.2.6-迭代器结构体与生命周期.md)

---

## 4. `'_` vs 直接写 `&str`（省略）

**函数参数上：几乎没区别。**

```rust
fn f1(s: &str) {}
fn f2(s: &'_ str) {}
```

| | `&str` | `&'_ str` |
|--|--------|-----------|
| 编译 | 相同（都会 elision） | 相同 |
| 可读性 | 简洁 | **明示「有借用/生命周期」** |

在**结构体、返回值、嵌套泛型**上，有时**必须**写 `'_`（或 `'a`），不能完全省略。

---

## 5. 通俗比喻

| | 比喻 |
|--|------|
| **`'a`** | 发学号 `A01` — 大家能指名道姓 |
| **`'_`** | 「这位同学」— 不喊名，但都知道指谁；编译器自动对应正确生命周期 |

---

## 6. 什么时候**不能**用 `'_`？

需要表达**多个生命周期之间的关系**时，必须命名 `'a`、`'b`：

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

这里必须 `'a`：**返回值寿命 = x 与 y 的共同上限**。  
`'_` 只代表**单个**、未命名的生命周期，无法表达「这两个入参绑在一起」。

| 需求 | 用 |
|------|-----|
| 单引用、编译器能推 | `'_` 或省略 |
| 多引用同组 / 返回与入参关系 | **命名 `'a`** |

→ 详解：[10.3.2 同 `'a` 与红线](./10.3.2-同a约束与红线.md) · [10.3.4 longest](./10.3.4-longest与get_first.md)

---

## 7. 对比小结（背两句）

- **`'a`**：有名字，标注和控制引用有效期；多参数关系靠它表达。
- **`'_`**：有生命周期、无名字；编译器推导；简化写法、明确借用。

## 自测（2 题）

1. `fn g() -> Wrapper` 缺什么？→ **`Wrapper<'_>` 或 `Wrapper<'a>`**（结构体含引用，返回值须标生命周期）
2. `fn h(x: &'_ str, y: &'_ str) -> &'_ str` 能替代 `longest<'a>` 吗？→ **不能** — 两个 `'_` **互不关联**，无法保证返回值与哪个入参同寿
