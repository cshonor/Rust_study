## 一、两大模式：编译期确定 vs 运行期确定

```rust
trait Hello {
    fn say(&self);
}

struct A;
impl Hello for A { fn say(&self) { println!("A"); } }

struct B;
impl Hello for B { fn say(&self) { println!("B"); } }
```

### 1. `impl Trait` / 泛型 `<T: Trait>` → 静态分发（编译期搞定）

编译器**编译时**就知道传入的具体是哪个类型：

- **不需要虚表**
- 直接把 `t.say()` **替换成对应类型的函数地址**（单态化）
- 运行时直接调用，**无查表开销**

```rust
// 泛型：静态分发
fn static_call<T: Hello>(t: T) {
    t.say();
}

// impl 语法糖：同上，每个调用点一种具体类型
fn static_impl(t: impl Hello) {
    t.say();
}

fn main() {
    static_call(A); // 编译期确定 → 调用 A::say
    static_call(B); // 编译期确定 → 调用 B::say
}
```

**流程：编译期绑定类型 + 方法 → 运行直接执行。**

### 2. `&dyn Trait` → 动态分发（运行期查表）

```rust
fn dynamic_call(t: &dyn Hello) {
    t.say(); // 运行时查虚表
}

fn main() {
    let a = A;
    let b = B;
    dynamic_call(&a);
    dynamic_call(&b);
}
```

**整体流程（四步）：**

| 步 | 发生什么 |
|----|----------|
| 1 | 传 **`&实例`**（`dyn` 不定长，必须 `&` / `Box` 包装） |
| 2 | 每个 `impl Hello` 的类型，**编译期自动生成一张虚表**（存方法函数指针） |
| 3 | `&dyn Hello` = **胖指针**：数据 ptr + vtable ptr（创建时两个地址同时写好） |
| 4 | **运行时** `t.say()`：拿 vtable ptr → 查 `say` 地址 → 跳转执行，数据 ptr 当 `&self` |

> 虚表**不是运行时临时检查有没有** — 类型 `impl` 了 trait，**编译阶段就建好表**；运行期只做**查表调用**。

→ 胖指针两字段分工：[10.2.7](./10.2.7-dyn胖指针与虚表.md)

## 二、内存视角（`&dyn Hello` 胖指针）

```
&dyn Hello
├─ 数据指针 → 指向 A / B 的实例
└─ 虚表指针 → 指向 A 的 vtable / B 的 vtable

A 的虚表: [ A::say 函数地址 ]
B 的虚表: [ B::say 函数地址 ]
```

运行逻辑：

1. 传 `&a` → 胖指针绑定「A 实例 + A 的虚表」
2. `t.say()` → 查虚表 → 跑 `A::say`
3. 传 `&b` → 胖指针绑定「B 实例 + B 的虚表」
4. `t.say()` → 查虚表 → 跑 `B::say`

## 三、打比方（Animal 版，与 demo 一致）

`trait Animal` = **技能清单**（必须会 `cry()`）。

- `Dog`、`Cat` = 不同选手，各自实现叫唤
- **虚表 = 技能地址簿** — 存每个类型的 `cry` 函数地址

用 `dyn Animal` 调 `cry()`：不知道当前是狗还是猫 → **先翻地址簿** → 找到 `cry` → 执行。

```rust
trait Animal { fn cry(&self); }
struct Dog;
struct Cat;
impl Animal for Dog { fn cry(&self) { println!("汪汪"); } }
impl Animal for Cat { fn cry(&self) { println!("喵喵"); } }

fn call_impl(animal: impl Animal) {
    animal.cry(); // 编译期写死 Dog::cry / Cat::cry
}

fn call_dyn(a: &dyn Animal) {
    a.cry(); // 运行时查 vtable
}
```

### 虚表是类型级的

每个实现 `Animal` 的**类型**一张表（所有 `Dog` 实例共用 Dog 那张）：

| 类型 | vtable 里 |
|------|-----------|
| Dog | `cry` → `Dog::cry` |
| Cat | `cry` → `Cat::cry` |

一个类型实现多个 trait → 多张表 → [10.2.8](./10.2.8-多Trait多虚表.md)

## 四、完整流程：`call_dyn(&dog)` 逐步

```rust
let dog = Dog;
call_dyn(&dog);
```

| 步骤 | 发生什么 |
|------|----------|
| 1 | `&Dog` → 胖指针：数据 ptr = `dog`；vtable ptr = **Dog 的 Animal 虚表** |
| 2 | 进入 `call_dyn`，执行 `a.cry()` |
| 3 | 用 **虚表指针** 找到 Dog 的 vtable |
| 4 | 在表里查 `cry` 的函数地址 |
| 5 | 跳转 `Dog::cry`，数据 ptr 当 `&self` |

传 `&Cat` → 换 **Cat 的 vtable** → 执行 `Cat::cry`。

👉 **`dyn Trait` 调方法 = 运行时查虚表**。

## 五、关键对比（背下来）

| | **`impl` / 泛型 `<T: Trait>`** | **`&dyn Trait`** |
|--|--------------------------------|------------------|
| 何时定类型 | **编译期**锁定 | **运行期**通过胖指针识别 |
| 虚表 | ❌ 不需要 | ✅ 编译期生成，运行期查 |
| 方法调用 | 直接替换为 `Type::method` | vtable 取地址再跳转 |
| 异构 `Vec` | ❌ 单一类型 | ✅ 多类型共存 |
| 必须 `&`/`Box` | ❌（Sized 类型可直接用） | ✅（`dyn` 是 `!Sized`） |

1. **普通 impl / 泛型**：编译期就锁定「哪个类型 → 哪个方法」，**无虚表、无查表**。
2. **`dyn Trait`**：虚表**编译期提前建好**；调用**运行期查表**；必须搭配 `&` / `&mut` / `Box`。

## 六、为什么 `dyn` 能异构 `Vec`？

```rust
let list: Vec<Box<dyn Animal>> = vec![
    Box::new(Dog),
    Box::new(Cat),
];
```

每个元素都是胖指针 — 各自数据 ptr + 各自 vtable ptr；遍历 `cry()` 各查各表。  
`impl Trait` 编译期固定单一类型 → **不能**这样混放。

## 七、串起 Fn 闭包（动态分发同款流程）

```rust
fn run(f: &dyn Fn()) {
    f();
}

fn main() {
    let c1 = || println!("闭包1");
    let c2 = || println!("闭包2");
    run(&c1);
    run(&c2);
}
```

- 每个闭包是**不同匿名类型**，编译期各生成 **`Fn` 对应的虚表**
- `&dyn Fn()` = 胖指针：闭包实例 + 虚表
- `f()` → 查虚表 → 执行对应闭包

→ 闭包 Fn 系列：[13.1.4 Fn / FnMut / FnOnce](../../13-iterators-closures/13.1.4-Fn-FnMut-FnOnce三大捕获trait.md) · demo [10.2.5 §0](./10.2.5-impl-dyn-demo/)

## 八、背诵版

| | **虚表 / dyn** | **impl / 泛型（无虚表）** |
|--|----------------|---------------------------|
| 分发 | 运行时查表 · **动态** | 编译期绑定 · **静态** |
| 结构 | 胖指针（数据 + vtable） | 普通引用/值 |
| 异构 Vec | ✅ | ❌ |
| 开销 | 多一次查表 | 无 vtable |

**一句话：**

- **`impl Trait` / 泛型**：编译前就知道是谁 → **直接喊人**，不查本子。
- **`dyn Trait`**：统一接口、运行时才定 → **先翻地址簿（虚表）**，再干活。

**你的理解 checklist（全对）：**

1. `dyn Trait` 靠**虚表**实现动态调用
2. 虚表**编译期生成**，方法**运行期查表**执行
3. `dyn Trait` 不定长 → 前面**必须加 `&` / `Box`** 才能传递
4. 与编译期绑定的 `impl`、泛型是**两套机制**
