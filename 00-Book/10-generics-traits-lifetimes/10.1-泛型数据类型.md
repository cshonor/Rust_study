# 10.1 泛型数据类型

对应 demo 项目：`00-Book/10-generics-traits-lifetimes/10.1-generics-demo/`

泛型让你用“类型参数”写可复用的定义（函数 / 结构体 / 枚举 / 方法），减少重复代码。

---

## 在函数中使用泛型

如果要写一个“找最大值”的函数，`i32` 与 `char` 版本代码几乎一样：

```rust
fn largest_i32(list: &[i32]) -> i32 { /* ... */ }
fn largest_char(list: &[char]) -> char { /* ... */ }
```

用泛型统一：

```rust
fn largest<T>(list: &[T]) -> T { /* ... */ }
```

但它**不能直接编译**，因为函数体里用了 `>` 比较：并非所有 `T` 都能比较。

解决：给 `T` 加 trait bound（后面 10.2 trait 会展开）。常用写法：

```rust
fn largest<T: PartialOrd + Copy>(list: &[T]) -> T { /* ... */ }
```

---

## 在结构体中使用泛型

```rust
struct Point<T> {
    x: T,
    y: T,
}
```

如果希望 `x/y` 类型可不同：

```rust
struct Point<T, U> {
    x: T,
    y: U,
}
```

---

## 在枚举中使用泛型

标准库例子：

```rust
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

---

## 方法中的泛型

在 `impl` 里声明泛型：

```rust
impl<T> Point<T> {
    fn x(&self) -> &T { &self.x }
}
```

也可以只给具体类型实现方法（例如只给 `Point<f32>` 添加距离方法）。

方法本身也能引入新的泛型参数（与结构体定义的泛型不同），例如 `mixup`：

```rust
fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> { /* ... */ }
```

---

## 泛型代码的性能：单态化（monomorphization）

Rust 在**编译期**把泛型“展开”为具体类型版本（为每个用到的具体类型生成专门代码），因此：

- **运行时没有额外开销**
- 性能接近手写每个具体类型版本
