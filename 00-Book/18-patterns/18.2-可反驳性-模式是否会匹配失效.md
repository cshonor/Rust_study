# 18.2 可反驳性（Refutability）：模式是否会匹配失效

## 核心结论（先背）

1. **不可反驳（Irrefutable）**：对**所有**可能值都能匹配 → 只能用于 `let`、函数/闭包参数、**`for`**。
2. **可反驳（Refutable）**：对某些值会**匹配失败** → 用于 `match` 臂、`if let`、`while let`。
3. **`let Some(x) = opt`** 编译失败；改用 **`if let`** / **`match`**。
4. **`if let` 绑定变量仅块内有效**；**不检查穷尽**（与 `match` 不同）。

---

## 一、定义

| 类型 | 含义 | 例子 |
|------|------|------|
| **不可反驳** | 一定匹配成功 | `x`、`(a,b)`、`_`、`..` |
| **可反驳** | 可能失败 | `Some(x)`（遇 `None`）、`Ok(x)`（遇 `Err`）、字面量 `1` |

---

## 二、强制规则（编译器硬性要求）

| 使用位置 | 允许的模式 | 原因 |
|----------|------------|------|
| **`let`**、**函数/闭包参数**、**`for`** | **仅不可反驳** | 必须保证每轮/每次一定能绑定 |
| **`match` 臂** | **可反驳**（整体须穷尽） | 为「匹配/不匹配」设计 |
| **`if let` / `while let`** | **可反驳** | 失败走 else / 退出循环 |

> **易错**：`for` **不是**可反驳上下文 —— `for Some(x) in iter` 若 iter 含 `None` → `refutable pattern in for loop binding`。

---

## 三、分场景演示 + 易错点

### 1. `let`（仅不可反驳）

✅ 合法：

```rust
let x = 10;
let (a, b) = (1, 2);
let (_, y) = (3, 4);
```

❌ 编译失败：

```rust
// let Some(x) = Some(5);  // E0005: refutable pattern in local binding
```

✅ 替代：

```rust
let opt = Some(5);
if let Some(x) = opt {
    println!("{x}");
}
```

### 2. 函数参数（仅不可反驳）

✅ 合法：

```rust
fn print_pos(&(x, y): &(i32, i32)) {
    println!("x={x}, y={y}");
}
```

❌ 非法：

```rust
// fn bad(Some(val): Option<i32>) {}
```

### 3. `match`（可反驳 + 穷尽）

```rust
match num {
    Some(n) => println!("值：{n}"),
    None => println!("空值"),
}
```

必须写全分支或用 `_` 兜底。

### 4. `if let` / `else if let`

```rust
let v: Option<i32> = None;
if let Some(1) = v {
    println!("1");
} else if let Some(2) = v {
    println!("2");
} else {
    println!("其他");
}
```

**作用域陷阱**：

```rust
let res: Result<i32, &str> = Ok(20);
// if let Ok(age) = res && age > 18 { }  // ❌ age 不能这样用

if let Ok(age) = res {
    if age > 18 {
        println!("成年");
    }
}
```

**无意义警告**：`if let x = 5` → `irrefutable_let_patterns`（应直接用 `let`）。

### 5. `while let`（可反驳）

```rust
let mut stack = vec![1, 2, 3];
while let Some(val) = stack.pop() {
    println!("出栈：{val}");
}
// 3 → 2 → 1，None 时退出
```

### 6. `for`（不可反驳，但可解构）

```rust
for (idx, item) in v.iter().enumerate() {
    println!("{idx}: {item}");
}
```

解构 `(usize, &T)` 总是成功；**不能**写 `for Some(x) in options`（除非类型保证永无 `None`）。

## 四、忽略语法（`_` / `..`）

| 语法 | 作用 |
|------|------|
| `_` | 忽略单个值，不绑定 |
| `..` | 忽略连续多个字段/元组元素 |

```rust
let (x, _) = (10, 20);
let (a, ..) = (1, 2, 3, 4);
let (.., d) = (1, 2, 3, 4);
```

→ 更多语法：[18.3](./18.3-模式语法.md)

---

## 五、速记（背诵版）

1. **六处都是模式**：`match` / `if let` / `while let` / `for` / `let` / 参数。
2. **不可反驳** → `let`、参数、**`for`**。
3. **可反驳** → `match` 臂、`if let`、`while let`。
4. **`if let` 变量块内有效**；不与 `&&` 混写外部条件。
5. **`match` 穷尽**；**`if let` 链不穷尽**。

---

## 六、整体脉络

```text
绑定一定成功？  →  let / fn 参数 / for  →  不可反驳
允许匹配失败？  →  if let / while let / match  →  可反驳（match 还须穷尽）
枚举部分分支？  →  禁止 let Some(x) = opt  →  if let / match
```

## 常见编译信息

| 报错 / 警告 | 含义 |
|-------------|------|
| `refutable pattern in local binding` | `let` 用了 `Some(x)` 等 |
| `refutable pattern in for loop binding` | `for Some(x)` 等 |
| `irrefutable_let_patterns` | `if let x = 5` 无意义 |
| `non-exhaustive patterns` | `match` 漏分支 |
