## 核心结论（先背）

Rust 里**凡能写变量、解构的地方都是模式**，共 **6 类场景**：

| # | 场景 | 典型写法 |
|---|------|----------|
| 1 | `match` 分支 | `match v { P => … }` |
| 2 | `if let` / `else if let` | `if let Some(x) = opt { … }` |
| 3 | `while let` | `while let Some(x) = iter.next()` |
| 4 | `for` 迭代头 | `for (i, x) in v.iter().enumerate()` |
| 5 | `let` 声明 | `let (a, b) = (1, 2)` |
| 6 | 函数/闭包参数 | `fn f(&(x, y): &(i32, i32))` |

→ **可反驳 vs 不可反驳**：[18.2](./18.2-可反驳性-模式是否会匹配失效.md)

---

## 一、`match` 分支

```rust
match VALUE {
    PATTERN => EXPRESSION,
    ...
}
```

- 必须**穷尽**所有可能值；`_` 匹配剩余且不绑定
- 各分支可用**可反驳**模式；整体须覆盖全部情况

→ [6.2 match](../06-enums-pattern-matching/6.2-match.md)

---

## 二、`if let` / `else if let`

链式条件，不必写完整 `match`：

```rust
if let Some(color) = favorite_color {
    // ...
} else if let Ok(age) = age {
    if age > 30 { /* ... */ }
} else {
    // ...
}
```

| 要点 | 说明 |
|------|------|
| **可反驳** | 匹配失败走 `else` / 跳过 |
| **不穷尽** | 漏分支编译器**不**强制报错 |
| **作用域** | `if let` 绑定变量**仅块内**有效 |

❌ `if let Ok(age) = res && age > 18` —— 不能合并外部条件  
✅ 拆成 `if let Ok(age) = res { if age > 18 { … } }`

---

## 三、`while let` 条件循环

```rust
while let Some(top) = stack.pop() {
    println!("{top}");
}
```

匹配成功 → 执行循环体；失败 → **退出循环**。→ [13.2.2 §四](../13-iterators-closures/13.2.2-适配器与消费器详解.md#四while-letsomex--iternext手动消费)

---

## 四、`for` 循环

```rust
for (index, value) in v.iter().enumerate() {
    println!("{index}: {value}");
}
```

**注意**：`for` 的模式必须是**不可反驳**的 —— 每一轮迭代都必须能绑定（`for Some(x) in …` 若迭代器可能产出 `None` 会编译失败）。  
`(i, x)` 来自 `enumerate` 时总是 `(usize, &T)`，故不可反驳。

---

## 五、`let` 语句

```rust
let PATTERN = EXPRESSION;
let (x, y, z) = (1, 2, 3);
let (_, y) = (3, 4);
```

**仅允许不可反驳模式**（见 [18.2 §三.1](./18.2-可反驳性-模式是否会匹配失效.md#1-let-语句仅允许不可反驳模式)）。

---

## 六、函数参数 / 闭包参数

```rust
fn print_coordinates(&(x, y): &(i32, i32)) {
    println!("({x}, {y})");
}
```

参数位置 = 模式，规则同 `let`（**不可反驳**）。
