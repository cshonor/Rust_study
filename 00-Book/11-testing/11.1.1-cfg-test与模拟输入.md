## 一、先分清两个标记

### 1. `#[cfg(xxx)]` — 条件编译（编不编进程序）

`cfg` = configure。**写在模块/函数上方**，控制**要不要编译**。

| 写法 | 何时编译 |
|------|----------|
| **`#[cfg(test)]`** | 仅 **`cargo test`** |
| **`#[cfg(not(test))]`** | 仅 **`cargo build` / `cargo run`**（正式打包） |

```rust
fn sum(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod test_code {
    fn demo() {
        println!("我只存在于测试编译");
    }
}
```

- **`cargo build`** → `test_code` **整模块删掉**，不进 exe  
- **`cargo test`** → 编译并加载

> **作用**：测试代码不污染正式程序体积。

```rust
#[cfg(not(test))]
fn init_log() {
    // 上线才初始化日志；测试编译时删掉
}
```

### 2. `#[test]` — 测试用例标记（跑不跑这条测试）

标记函数为测试用例；**`cargo test` 自动逐个运行**。

**应放在 `#[cfg(test)]` 模块内** — 否则 `cargo build` 也会把 `#[test]` 函数打进正式程序。

```rust
fn sum(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_sum_1() {
        assert_eq!(sum(2, 3), 5);
    }

    #[test]
    fn test_sum_2() {
        assert_eq!(sum(-1, 1), 0);
    }
}
```

| 命令 | 行为 |
|------|------|
| **`cargo build`** | `tests` 模块被删，无测试代码 |
| **`cargo test`** | 编译 tests → 找所有 `#[test]` → 运行；`assert_eq!` 失败则报错 |

### 一句话

- **`#[cfg(test)]`**：一扇门 — 只有测试编译才保留这个模块  
- **`#[test]`**：模块里一个个小任务 — `cargo test` 挨个执行  

→ 断言宏详解：[11.1 §三](./11.1-如何编写测试.md)

## 二、模拟输入（两种场景）

### 场景 A：模拟控制台输入（`stdin`）→ `Cursor`

**问题**：单元测试没法手动敲键盘。

**原则**：业务函数**不要写死 `stdin()`**，入参用 **`impl Read`**。

```rust
use std::io::Read;

fn read_data<R: Read>(mut source: R) -> String {
    let mut buf = String::new();
    source.read_to_string(&mut buf).unwrap();
    buf.trim().to_string()
}

// 正式：read_data(std::io::stdin());
```

测试：`Cursor` 把字符串变成 `Read` 流：

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use std::io::Cursor;

    #[test]
    fn mock_input_test() {
        let mock_input = Cursor::new("abc123");
        let result = read_data(mock_input);
        assert_eq!(result, "abc123");
    }
}
```

`cargo test` **全程无需手动输入**。

### 场景 B：模拟依赖对象 → `mockall`

业务依赖 trait，测试**伪造返回值**，不读真实文件/网络。

```toml
# Cargo.toml
[dev-dependencies]
mockall = "0.13"
```

```rust
trait ConfigSource {
    fn get_config(&self) -> String;
}

fn load_app_config(source: impl ConfigSource) -> String {
    source.get_config()
}

#[cfg(test)]
mod tests {
    use super::*;
    use mockall::mock;

    mock! {
        FakeConfig {}
        impl ConfigSource for FakeConfig {
            fn get_config(&self) -> String;
        }
    }

    #[test]
    fn test_config_mock() {
        let mut mock = MockFakeConfig::new();
        mock.expect_get_config()
            .return_const("mock_config_data".to_string());
        assert_eq!(load_app_config(mock), "mock_config_data");
    }
}
```

> **`mockall`** = Rust 常用 mock 库 — 替换依赖、规定调用期望与返回值。

## 三、附加标签

| 注解 | 作用 |
|------|------|
| **`#[ignore]`** | 默认跳过；`cargo test -- --ignored` 才跑 |
| **`#[cfg(feature = "xxx")]`** | 功能开关，须在 `Cargo.toml` 定义 `[features]` |

```rust
#[test]
#[ignore]
fn slow_test() {}
```

## 四、复盘

1. **`#[cfg(test)]`** — 圈代码，仅测试编译保留  
2. **`#[test]`** — 标记单条用例  
3. **模拟键盘** → `Read` + **`Cursor`**  
4. **模拟依赖** → trait + **`mockall`**
