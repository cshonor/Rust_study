## 核心结论（先背）

1. **unsafe ≠ 关闭安全检查** — 程序员主动承担不变式责任；`&T`/`&mut T` 借用规则仍有效
2. **仅 5 种操作需要 unsafe**：裸指针解引用 · `unsafe fn` · `static mut` · `unsafe impl` · `union` 字段
3. **原则**：缩小 `unsafe` 块，封装为**安全 API**（如 `split_at_mut`）
4. **裸指针无生命周期/借用检查** — 空指针、野指针 → **UB（未定义行为）**

## 一、核心概念

### unsafe 的真实含义

常规 Rust 在编译期检查内存安全。**unsafe** 允许编译器**无法代为证明**安全的五类操作，由开发者向编译器承诺：你会自己维护安全不变式。

静态分析偏保守：宁可拒绝部分合法程序，也不误放行 UB。

### 五大 unsafe 超能力

| # | 操作 | 说明 |
|---|------|------|
| 1 | 解引用裸指针 `*const T` / `*mut T` | 安全代码可**创建**，**解引用**须 unsafe |
| 2 | 调用 `unsafe fn` / unsafe 方法 | 如 `std::ptr::write` |
| 3 | 读/写 `static mut` | 多线程极易数据竞争 |
| 4 | `unsafe impl` 不安全 trait | 如手动 `Send`/`Sync` |
| 5 | 访问 `union` 字段 | C 互操作常见 |

### 其他要点

- **`unsafe` 块范围有限**：只影响块内直接操作，不会自动扩展到内部安全函数
- **`Send`/`Sync`**：标记 trait，手动 `unsafe impl` 须确保线程安全
- **典型场景**：FFI、底层数据结构、安全 API 内部实现

## 二、代码示例

### 裸指针 + unsafe 函数

```rust
let mut y = 20;
let r1: *const i32 = &y;      // 安全代码可创建裸指针
let r2: *mut i32 = &mut y;

unsafe {
    println!("裸指针解引用：{}", *r1);
    std::ptr::write(r2, 30);  // unsafe 函数
}
println!("修改后 y: {}", y);  // 30
```

### 安全封装：`split_at_mut` 思路

借用检查器无法证明两段可变 slice 不重叠；在 `mid <= len` 已证明时，用裸指针构造（见 demo `split_at_mut_i32`）。

### FFI · 静态变量 · union · unsafe trait

demo 还演示：`extern "C" { fn abs(...) }`、`static mut COUNTER`、`union IntOrFloat`、`unsafe trait UnsafeMarker`。

## 三、分节详解

### 裸指针

- 不保证有效、可为 null、可同时存在多个可变指针
- 解引用空指针/野指针 → **UB**

### `unsafe fn`

调用者须满足额外契约；须在 `unsafe { }` 或 `unsafe fn` 体内调用。

### FFI：`extern "C"`

外部函数在 Rust 侧**一律 unsafe 调用**。导出给 C：`pub extern "C" fn` + `#[no_mangle]`。

### `static mut`

读/写均须 unsafe；多线程优先用第 16 章并发原语（`Mutex` 等）。

## 四、易错点

| # | 易错 | 正确理解 |
|---|------|----------|
| 1 | unsafe = 关闭所有检查 | 借用检查等对 `&T`/`&mut T` **仍有效** |
| 2 | 裸指针像引用一样安全 | 无生命周期、无借用检查；空/野指针 → UB |
| 3 | unsafe 块“传染”子函数 | 只影响**直接包含**的 unsafe 操作 |
| 4 | 随意 `unsafe impl Send` | 须**证明**线程安全，否则运行时数据竞争 |
| 5 | `static mut` 随便用 | 多线程 UB；优先 `Mutex`/`Atomic*` |
| 6 | 在生产照搬任意地址示例 | demo 仅示意，勿对非法地址解引用 |

---

## 五、速记

1. unsafe = **承诺不变式**，不是关检查
2. 五超能力：裸指针 · unsafe fn · static mut · unsafe impl · union
3. 裸指针：**创建安全，解引用 unsafe**
4. 封装原则：**最小 unsafe 块 + 安全 API 对外**
