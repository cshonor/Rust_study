## 核心结论（先背）

1. **newtype**：元组结构体包装 → **新类型**，强类型安全 + 封装
2. **`type` 别名**：同义词，**不**创建新类型，完全互通
3. **`!`**：永不返回，可 coerce 为任意类型
4. **DST**：`str` / `[T]` / `dyn Trait` — **必须**加指针
5. **`Sized` / `?Sized`**：泛型默认 `T: Sized`；接受 DST 写 `?Sized` + `&T`

## 二、newtype 模式：类型安全与封装

### 1. 核心定义

**单元素元组结构体**包装已有类型 → **完全独立的新类型**，与原类型不互通。

### 2. 两大用途

#### （1）类型安全 — 区分语义相同、底层一致的值

```rust
struct Millimeters(u32);
struct Meters(u32);

let mm = Millimeters(1000);
let m = Meters(1);
// let sum = mm + m;  // E0308：类型不匹配
```

#### （2）封装抽象 — 隐藏内部实现

```rust
struct People(HashMap<i32, String>);  // 字段默认私有

impl People {
    fn new() -> Self { Self(HashMap::new()) }
    fn add(&mut self, id: i32, name: String) { self.0.insert(id, name); }
    fn get_name(&self, id: i32) -> Option<&str> {
        self.0.get(&id).map(|s| s.as_str())
    }
}
// people.0;  // E0616：私有字段
```

#### （3）结合孤儿规则

为**外部类型** impl **外部 trait** → newtype 包装（见 [19.2 §五](./19.2-高级trait.md)）。

## 三、类型别名 `type`

### 1. 基础语法

`type 别名 = 原类型;` — **不创建新类型**，编译器完全互通。

```rust
type Kilometers = i32;

let a: Kilometers = 10;
let b: i32 = 20;
let c = a + b;  // OK
```

### 2. 常用场景

**简化超长类型**：

```rust
type Thunk = Box<dyn Fn() + Send + 'static>;
type ClosureBox = Box<dyn Fn(u32) -> u32>;
```

**统一 Result 错误类型**（标准库经典）：

```rust
// std::io::Result<T> 等价于：
type Result<T> = std::result::Result<T, std::io::Error>;
```

### 3. newtype vs 类型别名

| 特性 | newtype | `type` 别名 |
|------|---------|-------------|
| 创建新类型 | ✅ 是 | ❌ 否 |
| 类型互通 | 完全不互通 | 完全互通 |
| 主要用途 | 类型安全、封装、孤儿规则 | 简化书写、统一接口 |

## 四、永不类型 `!`（Never Type）

### 1. 定义

空类型，**无合法值** — 表达式**永不正常返回**（发散）。

### 2. 典型场景

- `panic!()`
- 无限 `loop {}`
- `continue` / `break`（部分分支）

```rust
fn diverge() -> ! {
    panic!("程序终止，永不返回");
}
```

### 3. 核心特性

`!` 可**隐式转换**为任意类型 → `match`/`if` 分支类型统一：

```rust
let num = Some(5);
let res = match num {
    Some(n) => n,
    None => panic!("无值"),  // ! → i32
};

let val = if choice { 100 } else { crash() };  // else 分支 ! → i32
```

## 五、DST & `Sized` / `?Sized`

### 1. 什么是 DST

编译期**无法确定大小**的类型：

- `[T]`（非 `&[T]`）
- `str`（非 `&str`）
- `dyn Trait`（trait 对象底层）

### 2. 黄金规则

DST **不能单独使用**，必须包裹在指针/智能指针中：

| 合法 | 非法 |
|------|------|
| `&str`、`Box<str>` | `let s: str = ...` |
| `&[T]`、`Box<[T]>` | `let a: [i32] = ...` |
| `&dyn Trait`、`Box<dyn Trait>` | `let v: Vec<str>` |

### 3. `Sized` 与 `?Sized`

```rust
fn normal<T>(t: T) {}           // 等价 fn normal<T: Sized>(t: T)
fn accept_dst<T: ?Sized>(t: &T) {}  // 可接收 str、dyn Trait 等
```

- **`Sized`**：编译期已知固定大小（默认约束）
- **`?Sized`**：放宽约束，允许 DST — 参数**必须**用 `&T` / `Box<T>` 等

### 4. 典型应用

通用工具函数同时兼容定长类型与 DST：

```rust
fn print_content<T: ?Sized>(val: &T)
where
    T: std::fmt::Display,
{
    println!("内容：{val}");
}
// print_content("test");  // &str
// print_content(&99);     // &i32
```

## 七、高频易错点

| # | 易错 | 正确理解 |
|---|------|----------|
| 1 | newtype 与 `type` 混淆 | 隔离/单位校验 → newtype；简化书写 → `type` |
| 2 | 裸 DST `let s: str` | 必须 `&str` / `Box<str>` 等指针 |
| 3 | `?Sized` 但参数是 `T` | DST 须 `&T` / `Box<T>` → **E0277** |
| 4 | `let x: ! = ...` | `!` 仅作返回值/发散分支 |
| 5 | 默认泛型接收 DST | `fn f<T>(t: T)` 等价 `T: Sized` |
| 6 | `Vec<str>` | DST 不能作 Vec 元素 → `Vec<String>` / `Vec<Box<str>>` |
| 7 | `Kilometers` 防单位混用 | **别名不防混用** — 须 newtype |

---

## 八、速记

- **newtype**：造新类型，强安全 + 封装
- **`type`**：同义词，完全互通
- **`!`**：发散，可转任意类型
- **DST**：大小未知，**必须加指针**
- **`Sized`**：默认；**`?Sized`**：放宽 + `&T`
