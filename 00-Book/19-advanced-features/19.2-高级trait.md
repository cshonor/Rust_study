## 核心结论（先背）

1. **关联类型**：trait 内 `type`，实现方指定 — **一个类型仅 impl 一次**（`Iterator::Item`）
2. **泛型 trait**：同一类型可**多次 impl**（不同泛型参数）— 调用易歧义
3. **默认泛型参数**：`trait Add<Rhs = Self>` — 运算符重载常用
4. **完全限定语法**：`<Type as Trait>::func()` — 消歧义
5. **supertrait**：`trait A: B` — 实现 A 必先实现 B
6. **newtype**：包装外部类型 — **绕过孤儿规则**，零运行时开销

## 一、关联类型（Associated Types）

### 核心回顾

Trait 内部用 `type` 定义**关联类型**，由**实现方**指定具体类型；同一类型**只能对该 trait 实现一次**（对比泛型 trait）。

### 标准参考：简化版 Iterator

```rust
trait Iterator {
    type Item;  // 迭代产出的元素类型
    fn next(&mut self) -> Option<Self::Item>;
}
```

### 完整 Demo：`Container`

```rust
trait Container {
    type Elem;
    fn push(&mut self, val: Self::Elem);
    fn pop(&mut self) -> Option<Self::Elem>;
}

struct IntBag(Vec<i32>);

impl Container for IntBag {
    type Elem = i32;
    fn push(&mut self, val: Self::Elem) {
        self.0.push(val);  // 委托内部 Vec，避免与 trait 方法同名递归
    }
    fn pop(&mut self) -> Option<Self::Elem> {
        self.0.pop()
    }
}
```

> **易错**：直接 `impl Container for Vec<i32>` 且方法名也叫 `push`，方法体内 `self.push(val)` 会**无限递归**。应用 newtype 包装或换方法名。

### 关联类型 vs 泛型 Trait

#### 写法 1：泛型 trait `Trait<T>` — 可多次 impl

```rust
trait Iterable<T> {
    fn next(&mut self) -> Option<T>;
}

struct MyIter;
impl Iterable<i32> for MyIter { /* ... */ }
impl Iterable<String> for MyIter { /* ... */ }  // 同一类型，不同 T
```

调用处可能需要类型标注，容易歧义。

#### 写法 2：关联类型 — 只能 impl 一次

同一类型对 `trait Container` **只能有一种** `Elem`，无歧义 — 这也是标准库 `Iterator` 选关联类型的原因。

| | 关联类型 | 泛型 trait |
|--|----------|------------|
| impl 次数 | **一次** | **可多次** |
| 调用 | 无需再选 `Item` | 可能需类型标注 |
| 典型 | `Iterator::Item` | 少见，特殊多态需求 |

## 二、默认泛型类型参数 + 运算符重载

### 语法

`Trait<Param = DefaultType>` — 给泛型参数默认值，简化常规使用。

```rust
trait Add<RHS = Self> {
    type Output;
    fn add(self, rhs: RHS) -> Self::Output;
}
```

### 场景 1：同类型相加（默认 `RHS = Self`）

```rust
#[derive(Debug, PartialEq)]
struct Point { x: i32, y: i32 }

impl Add for Point {  // 等价 impl Add<Point> for Point，使用默认 RHS
    type Output = Point;
    fn add(self, rhs: Point) -> Self::Output {
        Point { x: self.x + rhs.x, y: self.y + rhs.y }
    }
}

// p1 + p2 → Point { x: 4, y: 6 }
```

### 场景 2：不同类型相加（显式 `RHS`）

```rust
struct Millimeters(u32);
struct Meters(u32);

impl Add<Meters> for Millimeters {
    type Output = Millimeters;
    fn add(self, rhs: Meters) -> Self::Output {
        Millimeters(self.0 + rhs.0 * 1000)  // 1m = 1000mm
    }
}

// Millimeters(500) + Meters(2) → Millimeters(2500)
```

### 作用总结

- 大部分场景用**默认类型**，代码简洁
- 特殊场景**显式指定泛型参数**，向下兼容旧代码

## 三、完全限定语法（消歧义）

多个同名方法/关联函数时，明确指定调用哪一个。

### 1. 同名方法（带 `self`）

```rust
trait Pilot { fn fly(&self); }
trait Wizard { fn fly(&self); }
struct Human;
impl Human { fn fly(&self) { /* 人类 */ } }
impl Pilot for Human { fn fly(&self) { /* 飞行员 */ } }
impl Wizard for Human { fn fly(&self) { /* 法师 */ } }

let person = Human;
person.fly();           // 类型自身方法（优先）
Pilot::fly(&person);    // Pilot 的 fly
Wizard::fly(&person);   // Wizard 的 fly
```

### 2. 同名关联函数（无 `self`）— 必须完全限定

编译器无法从「接收者」推断，须 `<Type as Trait>::func()`：

```rust
trait Animal { fn baby_name() -> String; }
struct Dog;
impl Dog { fn baby_name() -> String { "小狗崽".into() } }
impl Animal for Dog { fn baby_name() -> String { "幼犬".into() } }

Dog::baby_name();                  // Dog 自身
<Dog as Animal>::baby_name();      // Animal trait 实现
```

## 四、父 Trait（Supertrait）

语法：`trait A: B` — **要实现 A，必须先实现 B**；A 的方法可使用 B 的能力。

```rust
trait OutlinePrint: Display {
    fn outline_print(&self) {
        let s = self.to_string();  // 依赖 Display
        println!("{}", "*".repeat(s.len() + 2));
        println!("*{}*", s);
        println!("{}", "*".repeat(s.len() + 2));
    }
}

impl OutlinePrint for i32 {}  // i32 已实现 Display
```

## 五、Newtype 模式（绕过孤儿规则）

### 孤儿规则

不能为**外部类型**实现**外部 trait**（两者都不在本 crate）。

```rust
// impl Display for Vec<String> {}  // E0117 违反孤儿规则
```

### 解决方案

元组结构体包装 → 创造**本地新类型**：

```rust
struct Wrapper(Vec<String>);

impl fmt::Display for Wrapper {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "[{}]", self.0.join(", "))
    }
}
```

### 补充

1. **无运行时开销** — 编译期包装，优化后与直接使用内部类型等价
2. 需手动转发方法，或 `Deref` 部分暴露内部 API
3. `#[repr(transparent)]` 用于 FFI/ABI，**不是**性能优化手段

## 六、易错点 & 编译报错

| # | 易错 | 说明 |
|---|------|------|
| 1 | `impl Container for Vec` 方法名 `push` 递归 | 须 newtype 或 `Vec::push(self, v)` 全路径 |
| 2 | 关联类型与泛型 trait 混用场景 | 关联类型 = 一种；泛型 = 多种 impl |
| 3 | FQS 只写 `Animal::baby_name()` | 无 `self` 时须 `<Dog as Animal>::baby_name()` |
| 4 | `impl OutlinePrint for T` 未 impl Display | supertrait 不满足 → 编译失败 |
| 5 | 直接 `impl Display for Vec<String>` | E0117 孤儿规则 |
| 6 | newtype「有性能开销」 | **零运行时开销** |

---

## 七、速记

1. **关联类型**：`type Item`，**一次 impl**，Iterator 经典
2. **默认泛型**：`Add<Rhs = Self>`，运算符重载
3. **FQS**：`<Type as Trait>::func()`
4. **supertrait**：`A: B`，先 B 后 A
5. **newtype**：绕过孤儿规则，零开销
