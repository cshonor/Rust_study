## 核心结论（先背）

1. **`fn(...)`**：函数**指针类型**（具体类型，非 trait）；**不捕获环境**
2. **`Fn`/`FnMut`/`FnOnce`**：**trait**；闭包与函数指针均可实现
3. **日常 API**：优先 `F: Fn(...)` — 兼容闭包 + 函数指针
4. **返回闭包**：匿名类型不能直接写 → `impl Fn`（单一类型）或 `Box<dyn Fn>`（多分支）
5. **`map`**：可传闭包、方法项、构造器

---

## 一、函数指针 `fn`

### 核心概念

1. 语法：`fn(参数) -> 返回值` — **具体类型**，不是 trait
2. 普通函数、关联函数可转为函数指针，作为参数传递
3. 函数指针自动实现 `Fn`/`FnMut`/`FnOnce` → 接收 `F: Fn` 的 API **也能传函数指针**
4. **选型**：
   - 日常：`F: Fn(...)` 泛型 — 灵活性最高
   - 特殊：FFI / C 回调 — 强制 `fn` 类型

### 1. 基础用法

```rust
fn calc(num: i32, func: fn(i32) -> i32) -> i32 {
    func(num)
}

fn double(x: i32) -> i32 { x * 2 }

fn main() {
    let res = calc(10, double);  // 函数名 → 函数指针
    // calc(10, |x| x * 2)  // 闭包不是 fn 类型，此处不行
}
```

### 2. 函数指针 vs 泛型 `F: Fn`

```rust
fn run_func<F: Fn(i32) -> i32>(x: i32, f: F) -> i32 {
    f(x)
}

fn add_one(x: i32) -> i32 { x + 1 }

run_func(5, add_one);    // 函数指针 OK
run_func(5, |x| x - 1);  // 闭包 OK
```

| | `fn(i32) -> i32` | `F: Fn(i32) -> i32` |
|--|------------------|---------------------|
| 接受闭包 | ❌ | ✅ |
| 接受函数指针 | ✅ | ✅ |
| 典型场景 | FFI | 标准库迭代器 |

## 二、迭代器 `map`：传函数 / 方法项

`Iterator::map` 参数是 `Fn`，除闭包外还可传**方法项、构造器**。

### 1. 方法项 `ToString::to_string`

```rust
let nums = vec![1, 2, 3];
let s1: Vec<_> = nums.iter().map(|n| n.to_string()).collect();
let s2: Vec<_> = nums.iter().map(ToString::to_string).collect();
// s1 == s2
```

### 2. 结构体 / 枚举构造器

```rust
struct Wrapper(u32);
enum Status { Value(u32), Stop }

(0u32..3).map(Wrapper).collect::<Vec<_>>();
(0u32..5).map(Status::Value).collect::<Vec<_>>();
// Wrapper / Status::Value 符合 fn(T) -> U 签名
```

## 三、返回闭包

### 问题根源

- 闭包是编译器生成的**匿名独有类型**
- 每个闭包类型不同
- **不能**直接写 `fn bad() -> (i32) -> i32 { |x| x + 1 }`

### 解决方案：`Box<dyn Fn>`

1. 闭包 impl `Fn`/`FnMut`/`FnOnce`
2. 装箱为 `Box<dyn Fn(...) -> ...>`（DST + 胖指针）
3. 支持运行时多态 — **多分支返回不同闭包**

### 1. 基础返回

```rust
fn return_closure() -> Box<dyn Fn(i32) -> i32> {
    Box::new(|x| x + 1)
}
```

### 2. 多分支返回不同闭包（`dyn` 核心优势）

```rust
fn select_closure(flag: bool) -> Box<dyn Fn(i32) -> i32> {
    if flag {
        Box::new(|x| x * 2)
    } else {
        Box::new(|x| x / 2)
    }
}
// impl Fn 无法做到 — 两个分支是不同匿名类型
```

### 3. 区分 `Fn` / `FnMut` / `FnOnce` 返回

| 行为 | 返回类型 |
|------|----------|
| 只读，多次调用 | `Box<dyn Fn()>` |
| 修改捕获变量 | `Box<dyn FnMut()>` |
| 消耗捕获 | `Box<dyn FnOnce()>` |

```rust
fn get_fnmut() -> Box<dyn FnMut()> {
    let mut cnt = 0;
    Box::new(move || {
        cnt += 1;
        println!("计数: {cnt}");
    })
}
```

### `impl Fn` vs `Box<dyn Fn>`

| | `impl Fn(...)` | `Box<dyn Fn(...)>` |
|--|----------------|---------------------|
| 分发 | 静态 | 动态（vtable） |
| 多分支不同闭包 | ❌ | ✅ |
| 性能 | 更优 | 微小开销 |

## 四、闭包 trait 速查

| Trait | 调用 | 典型捕获 |
|-------|------|----------|
| `Fn` | `&self` | 共享借用 / Copy |
| `FnMut` | `&mut self` | 可变借用 |
| `FnOnce` | `self` | 移动 |

函数指针**不捕获** → 同时 impl 三个 trait。

---

## 五、易混点 & 易错点

| # | 易混/易错 | 正确理解 |
|---|-----------|----------|
| 1 | `fn` vs `Fn` | `fn` = 指针**类型**；`Fn` = **trait** |
| 2 | `calc(10, \|x\| x*2)` 传给 `fn` 参数 | 闭包不是 `fn` 类型 → 用 `F: Fn` 或装箱 |
| 3 | `impl Fn` 多分支返回 | 只能**单一**类型 → `Box<dyn Fn>` |
| 4 | `Box<dyn Fn>` 零开销 | 有 vtable 动态分发 |
| 5 | 返回 `FnMut` 闭包却用 `Box<dyn Fn>` | 会修改捕获 → `Box<dyn FnMut>` |
| 6 | `thread::spawn` 闭包 | 常需 `Send + 'static`，注意捕获生命周期 |

---

## 六、速记（背诵版）

1. **`fn`**：函数指针，具体类型，FFI 场景；日常用 **`F: Fn`**
2. **map**：闭包 / `ToString::to_string` / 构造器
3. **返回闭包**：不能直接写 → **`Box<dyn Fn>`** 或 **`impl Fn`**
4. **多分支闭包**：只有 **`Box<dyn Fn>`** 可行
5. **`fn(...)`** 存函数 · **`Fn(...)`** 是 trait
