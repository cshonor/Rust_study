# 12.6 将错误信息输出到标准错误而不是标准输出

命令行程序通常把**正常输出**写到 stdout，把**错误信息**写到 stderr。这样用户把 stdout 重定向到文件时，仍然能在屏幕上看到错误信息。

对应 demo 项目：`00-Book/12-cli-project/12.6-minigrep-stderr-demo/`

---

## `println!` vs `eprintln!`

- `println!`：写到 **stdout**
- `eprintln!`：写到 **stderr**

我们把 `main.rs` 里打印错误的地方从 `println!` 改为 `eprintln!`。

---

## PowerShell 重定向验证

在 PowerShell 里：

- `>` 只重定向 stdout（成功输出流）
- `2>` 重定向 stderr

在 demo 目录里运行：

```powershell
cd 00-Book/12-cli-project/12.6-minigrep-stderr-demo

# 触发“缺少参数”的错误：stdout 写入 output.txt，stderr 仍显示在屏幕
cargo run --quiet > output.txt

# 正常运行：stdout 写入 output.txt
cargo run --quiet -- to poem.txt > output.txt
```
