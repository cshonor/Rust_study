## 核心结论（先背）

传统 OOP 三大基石：**对象、封装、继承 & 多态**。

| 特征 | Rust 支持情况 | 落地方式 |
|------|---------------|----------|
| **对象** | ✅ 完整 | 结构体/枚举存数据，`impl` 写方法 |
| **封装** | ✅ 完整 | 默认私有，`pub` 暴露 API，保护不变式 |
| **继承** | ❌ 无语法 | Trait 默认方法复用逻辑；**组合**复用字段 |
| **多态** | ✅ 两种 | 泛型 + Trait 边界（静态）；`dyn Trait`（动态） |

Rust 是**多范式语言**，吸收 OOP 优点，用 **Trait 体系**替代继承树，偏向**组合优于继承**。

---

## 一、对象：数据 + 行为（方法）

经典定义（《设计模式》）：对象 = **内部数据** + **操作数据的方法**。

| 角色 | Rust 载体 |
|------|-----------|
| 数据（状态） | `struct` / `enum` |
| 行为（方法） | `impl` 块 |

```rust
struct Book {
    title: String,
    pages: u32,
}

impl Book {
    // 关联函数（类似构造器）
    fn new(title: &str, pages: u32) -> Self {
        Self { title: title.to_string(), pages }
    }
    // 实例方法
    fn info(&self) -> String {
        format!("《{}》共{}页", self.title, self.pages)
    }
}
```

日常不口头称「对象」，但语义与经典定义一致。**枚举 + `impl`** 同样满足（状态机、模式匹配场景常见）。

## 二、封装：隐藏实现，暴露公有 API

### 核心规则

- **所有成员默认私有**；用 `pub` 手动提升可见性。
- 目的：保护**内部数据不变式**；外部只能通过公开方法读写；内部重构不影响调用方。
- 封装范围：结构体、字段、方法、函数、模块、枚举变体；**私有方法**也是封装的一部分。

### 标准案例：`AveragedCollection`

需求：维护数字集合并**缓存平均值**，保证 `list` 与 `average` **永远同步**。

设计：

1. `list`、`average` → **私有字段**，外部不能直接改。
2. 对外 `add` / `remove` / `average` → **公有方法**。
3. 每次增删 → 内部 `update_average()` 自动更新。

完整代码见 demo：`17.1-averaged-collection-demo/src/lib.rs`。

```rust
pub struct AveragedCollection {
    list: Vec<i32>,    // 私有
    average: f64,      // 私有
}

impl AveragedCollection {
    pub fn add(&mut self, value: i32) {
        self.list.push(value);
        self.update_average();  // 私有方法，仅内部调用
    }
    // ...
    fn update_average(&mut self) { /* ... */ }
}
```

❌ 外部访问私有字段编译失败：

```rust
// let coll = AveragedCollection::new();
// coll.list;  // E0616: field `list` is private
```

## 三、继承 & 多态：与传统 OOP 的差异

### （1）继承：Rust 没有结构体继承

不支持 `struct B : A` 这类语法。传统继承两大用途与 Rust 替代：

| 传统动机 | Rust 替代 |
|----------|-----------|
| **复用方法实现** | **Trait 默认方法** — 实现者可复用或覆盖 |
| **复用字段** | **组合** — 嵌套结构体，「组合优于继承」 |

Trait 默认方法示例：

```rust
trait Summary {
    fn summarize(&self) -> String {
        String::from("默认摘要")  // 默认实现
    }
}

struct News;
impl Summary for News {}  // 不重写，直接复用默认
```

Rust 拒绝原生继承的原因：规避**菱形继承**、层级臃肿、父类改动波及子类等经典问题。

### （2）多态：同一接口，多种类型

| 方式 | 机制 | 特点 |
|------|------|------|
| **泛型 + Trait 边界** | 编译期单态化（静态分发） | 零运行时开销，性能高 |
| **`dyn Trait` 特征对象** | 虚表（动态分发） | 异构类型进同一容器，经典 OOP 式多态 |

```rust
// 静态分发
fn print_summary<T: Summary>(item: T) {
    println!("{}", item.summarize());
}

// 动态分发
fn print_all(items: &[&dyn Summary]) {
    for item in items {
        println!("{}", item.summarize());
    }
}
```

## 四、易混点 & 考点

| 考点 | 要点 |
|------|------|
| Rust 算不算 OOP？ | 具备对象与封装；无继承语法；多态靠 Trait |
| 封装 vs 隐私 | `pub struct` 可见，字段仍可私有 |
| 默认方法 vs 继承 | 默认方法只复用**行为**，不复用**字段** |
| 何时用 `dyn`？ | 需要运行时异构集合；否则优先泛型 |
| 枚举也是「对象」 | `impl` 绑定方法 + 变体存不同状态 |

---

## 五、速记（背诵版）

1. **对象**：结构体/枚举 + `impl`，语义等价传统对象。
2. **封装**：默认私有 + 公有 API + 不变式（`AveragedCollection` 是标准案例）。
3. **继承**：无语法 → Trait 默认方法（逻辑）+ 组合（字段）。
4. **多态**：`T: Trait`（静态）/ `dyn Trait`（动态）。
5. Rust 多范式，Trait 体系重构了继承与多态。
