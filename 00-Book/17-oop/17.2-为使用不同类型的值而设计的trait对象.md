## 核心结论（先背）

1. **类型可穷举** → 枚举（编译期固定变体，**无法扩展**）。
2. **类型需外部扩展、异质混存** → `Vec<Box<dyn Trait>>`（trait 对象）。
3. **trait 对象本质**：**DST（不定大小）** + 胖指针 + 类型擦除 + vtable + **动态分发**。
4. **`dyn Trait` 不能单独用** — 必须配合 `&` / `&mut` / `Box` 等指针。
5. **对象安全**：不返回 `Self`、无独立泛型、一般无 `async fn` → 才能 `dyn Trait`。

---

## 一、场景引入：枚举 vs Trait 对象

### 1. 枚举：类型集合**固定**

所有可能类型在编码阶段能全部枚举时，优先用枚举 — 编译期确定变体，类型安全、无 vtable 开销。

```rust
enum UiComponent {
    Button(Button),
    TextField(TextField),
}
```

**缺陷**：新增 `SelectBox` / `Image` 必须**修改原枚举并重新编译**；第三方无法自由扩展 — 不适合框架、插件、GUI。

→ 同类思路：`SpreadsheetCell`（第 8 章）

### 2. Trait 对象：类型**动态扩展**

核心诉求：**不修改原有代码**，任意实现指定 trait 的类型都能存入同一容器、调用统一接口。

```rust
struct Screen {
    components: Vec<Box<dyn Draw>>,
}
```

遵循**开闭原则**：对扩展开放、对修改关闭。

## 二、Trait 对象核心概念与原理

### 1. 基础语法与所有权形态

| 写法 | 含义 | 适用场景 |
|------|------|----------|
| `&dyn Trait` | 不可变借用，无所有权 | 临时只读引用 |
| `&mut dyn Trait` | 可变借用 | 需修改内部状态（见 §七-3） |
| `Box<dyn Trait>` | 堆分配，持有所有权 | `Vec` 异质集合（**最常用**） |
| `Rc<dyn Trait>` / `Arc<dyn Trait>` | 共享所有权 | 多处共享同一 trait 对象 |

### 2. 核心机制

1. **类型擦除**：编译期抹除 `Button` / `TextField`，统一抽象为 `dyn Draw`。
2. **胖指针**（双字长）：数据指针 + vtable 指针。
3. **动态分发**：运行时查 vtable 调用方法；静态分发则编译期直接绑定、可内联。

> `dyn Trait` 本身是 **DST（Dynamically Sized Type，不定大小类型）**，编译器不知道其大小，**不能直接放栈、不能直接 `Vec<dyn Trait>`**。

## 三、标准 Demo：GUI 绘图组件

库提供 `Draw`、`Screen`、`Button`、`TextField`；**使用者在 `main` 实现 `SelectBox`**。

```rust
pub trait Draw { fn draw(&self); }

pub struct Screen {
    pub components: Vec<Box<dyn Draw>>,
}

impl Screen {
    pub fn add_component(&mut self, comp: Box<dyn Draw>) {
        self.components.push(comp);
    }
    pub fn run(&self) {
        for comp in &self.components { comp.draw(); }
    }
}
```

运行 `cargo run` 输出初始渲染 + 扩展 `SelectBox` 后再次渲染。

## 四、泛型 + Trait Bound vs Trait 对象

### 1. 泛型（静态分发 · 同质）

```rust
struct HomogeneousScreen<T: Draw> {
    components: Vec<T>,
}
```

| 维度 | 说明 |
|------|------|
| 容器 | **同质** — 全是 `Button` 或全是 `TextField` |
| 分发 | **静态** — 单态化、零运行时开销、可内联 |
| 扩展 | 不能在同一 `Vec` 混放多种类型 |

### 2. Trait 对象（动态分发 · 异质）

```rust
struct Screen { components: Vec<Box<dyn Draw>> }
```

| 维度 | 说明 |
|------|------|
| 容器 | **异质** — 任意 `impl Draw` 混存 |
| 分发 | **动态** — vtable 间接调用，微小开销 |
| 扩展 | 框架 / GUI / 插件首选 |

### 选型口诀

- 类型**固定、已知、求性能** → **泛型 + Trait Bound**
- 类型**需外部扩展、混存多种** → **`Box<dyn Trait>`**

→ 六维对比表：[10.2.5 §五](../10-generics-traits-lifetimes/10.2.5-impl与dyn-Trait对比.md)

## 六、对象安全（Object Safe）— 重难点

只有**对象安全**的 trait 才能 `dyn Trait`，否则 **E0038**。

### 两条核心规则（必考）

#### 规则 1：方法不能返回 `Self`

trait 对象已擦除具体类型，编译器无法构造 `Self`。

```rust
trait Clone {
    fn clone(&self) -> Self;  // 返回 Self
}
// let x: Box<dyn Clone>;  // E0038: trait `Clone` cannot be made into an object
```

`Clone`、`Default` 等含 `Self` 返回值的 trait **不能**作 trait 对象。

#### 规则 2：方法不能有**独立泛型参数**

泛型会实例化多套形状，与类型擦除冲突。

```rust
trait Demo {
    fn func<T>(&self, val: T);  // 独立泛型
}
// let x: Box<dyn Demo>;  // E0038
```

### 补充规则（实战易错）

| 情况 | 对象安全？ | 说明 |
|------|-----------|------|
| `async fn` | ❌ 一般否 | 底层带隐式 lifetime / 泛型 |
| 无 `&self` 的关联函数 | ❌ 不能通过 trait 对象调用 | vtable 只绑实例方法 |
| 关联常量 | ✅ 不影响 | 仅方法受约束 |
| `where Self: Sized` 排除方法 | ✅ 技巧 | 标准库让部分 trait 可对象化 |

### 快速判断口诀

> 有返回 `Self`、有独立泛型、有 `async fn` → **不能** `dyn Trait`。

## 七、高频易错点 & 编译报错实战

### 易错 1：`dyn Trait` 不能单独使用（DST）

`dyn Draw` 大小未知，不能直接存栈或裸放 `Vec`。

```rust
// let v: Vec<dyn Draw> = Vec::new();
// E0277: the size for values of type `(dyn Draw + 'static)` cannot be known at compilation time

// let d: dyn Draw = Button { label: "x".into() };
// E0277: doesn't have a size known at compile-time
```

✅ **修正**：必须包指针

```rust
let v: Vec<Box<dyn Draw>> = Vec::new();
let d: Box<dyn Draw> = Box::new(Button { label: "x".into() });
```

### 易错 2：泛型容器不能混存多种类型

```rust
// let screen = HomogeneousScreen {
//     components: vec![
//         Button { label: "a".into() },
//         TextField { content: "b".into() },  // E0308: mismatched types
//     ],
// };
```

`Vec<T>` 中 `T` 只能是**一种**具体类型；混存需 `Vec<Box<dyn Draw>>`。

### 易错 3：未实现 trait 不能装箱

```rust
struct Image { path: String }
// screen.add_component(Box::new(Image { path: "a.png".into() }));
// E0277: the trait `Draw` is not implemented for `Image`
```

Rust「鸭子类型」= 编译期 trait 约束，不是运行时探测。

### 易错 4：非对象安全 trait 不能 `dyn`

```rust
// let x: Box<dyn Clone> = ...;  // E0038
// let x: Box<dyn NotObjectSafe> = ...;  // E0038
```

### 易错 5：可变 trait 对象

需修改内部状态时，用 `&mut dyn Trait` 或 `Box<dyn Trait>`（通过 `Box` 可变借用内部）：

```rust
trait Count {
    fn inc(&mut self);
    fn get(&self) -> u32;
}

let mut c = Counter { num: 0 };
let obj: &mut dyn Count = &mut c;
obj.inc();
```

trait 方法签名须为 `&mut self` 才能通过 trait 对象修改状态。

### 易错 6：trait 对象 ≠ 继承

Rust **无类继承**；trait 对象是基于**接口的多态**（vtable），与传统 OOP 继承多态原理不同 → [17.1](./17.1-面向对象语言的特征.md)。

### 边界：`&mut dyn Trait` 与多个可变借用

同一时刻对同一 trait 对象只能有一个 `&mut`；混存于 `Vec<Box<dyn Trait>>` 时，遍历时用 `for comp in &mut screen.components` 逐个获取可变借用。

## 八、全局速查（背诵版）

1. **场景**：固定 → 枚举；扩展 + 异质 → `Box<dyn Trait>`。
2. **本质**：DST + 胖指针 + 擦除 + vtable + 动态分发。
3. **对比**：泛型同质静态高性能；trait 对象异质动态可扩展。
4. **对象安全**：不返回 `Self`、无独立泛型、一般无 `async fn`。
5. **语法硬约束**：`dyn Trait` 必须配合指针；`Vec<dyn T>` 非法，`Vec<Box<dyn T>>` 合法。
