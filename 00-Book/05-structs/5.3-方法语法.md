## 一、从函数到方法

```rust
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}
```

| 概念 | 说明 |
|------|------|
| **`impl Type { ... }`** | 该类型的实现块 |
| **方法** | `impl` 内、**第一个参数是 `self`** 的函数 |
| **`&self`** | `self: &Self` 简写；`Self` = 当前类型 |
| **调用** | `rect.area()`，不是 `area(&rect)` |

## 二、三种 `self`

| 签名 | 含义 | 调用方 | 调用后实例 |
|------|------|--------|------------|
| **`&self`** | 不可变借用，只读 | 不必 `mut` | 仍可用 |
| **`&mut self`** | 可变借用，可改字段 | 须 **`mut`** | 仍拥有 |
| **`self`** | 拿走所有权 | 任意 | **move 失效** |

```rust
impl Rectangle {
    fn area(&self) -> u32 { self.width * self.height }
    fn set_size(&mut self, w: u32, h: u32) { self.width = w; self.height = h }
    fn destroy_to_tuple(self) -> (u32, u32) { (self.width, self.height) }
    fn new(w: u32, h: u32) -> Self { Self { width: w, height: h } }
}

let mut rect = Rectangle::new(10, 20);
let _ = rect.area();                    // rect 仍可用
rect.set_size(50, 60);
let (w, h) = rect.destroy_to_tuple();   // rect 此后不可用
```

完整可运行示例 → demo 第一节。

## 三、自动引用与调用

### 为什么没有 `->`

C：`ptr->f()` = `(*ptr).f()`。Rust **统一用 `.`**，编译器按方法签名给 **self 接收者** 自动补 `&` / `&mut`：

| 方法签名 | 你写 | 编译器等价 |
|----------|------|------------|
| `fn area(&self)` | `r.area()` | `(&r).area()` |
| `fn set(&mut self, ...)` | `r.set(30, 40)` | `(&mut r).set(...)` |
| `fn into_data(self)` | `r.into_data()` | 直接 move，不加引用 |

```rust
r1.can_hold(&r2);
(&r1).can_hold(&r2); // 等价
```

**口诀**：**点号左边（self）自动加引用；括号里其它参数不自动加 `&`。**

### 带其它参数：`can_hold`

```rust
impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}

rect1.can_hold(&rect2); // other 须手写 &
```

### getter：方法名与字段同名

```rust
impl Rectangle {
    fn width(&self) -> bool { self.width > 0 }
}
// rect.width()  → 方法
// rect.width    → 字段
```

Rust 不自动生成 getter；字段私有时常用手写 getter 作只读 API。

## 四、关联函数与多 `impl`

### 关联函数（无 `self`）

**不是方法**；用 **`类型名::函数名`** 调用（如 `String::from`）。

```rust
impl Rectangle {
    fn square(size: u32) -> Self {
        Self { width: size, height: size }
    }
}

let sq = Rectangle::square(3);
```

### 多个 `impl` 块

同一类型可拆成多块，**编译等价于一个大 `impl`**，便于按功能分组：

```rust
impl Rectangle { /* 只读：area, can_hold */ }
impl Rectangle { /* 修改：set_size */ }
impl Rectangle { /* 关联：new, square */ }
```

demo 的 `main.rs` 即三块 `impl`。

## 五、对照 Python

| Rust | 调用 | Python 近似 |
|------|------|-------------|
| `fn f(&self)` | `u.greet()` | `def f(self)` |
| `fn f(&mut self)` | `mut u.set(...)` | 改实例属性 |
| `fn f(self)` | `u.into_...()` | 无（消费对象） |
| 无 `self` | `User::new()` | `@staticmethod` |

```rust
impl User {
    fn greet(&self) { /* ... */ }
    fn new(name: String) -> Self { User { name } }
}
```

| | Python | Rust |
|--|--------|------|
| `self` | 隐式传入 | **显式**第一个参数 |
| 区分静态/实例 | `@classmethod` / `@staticmethod` | **有无 `self` 参数** |
| 内存 | 无借用规则 | `&` / `&mut` / `self` 绑定所有权 |

## 六、记忆卡片

1. **方法** = `impl` 里第一个参数是 `self`；**关联函数** = 无 `self` → `Type::f()`。
2. **三种 self**：只读 `&self` · 改字段 `&mut self`（须 `mut`）· 消费 `self`（move）。
3. **调用**：统一 `.`；self 自动 `&` / `&mut`；**其它参数手动 `&`**。
4. **多 `impl`**：同类型可拆分，仅组织代码。
