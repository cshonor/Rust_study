# 5.1.3 字段所有权：`String` vs `&str`

← 主笔记：[5.1](./5.1-定义并实例化结构体.md) · `&str` / DST：[4.3.3](../04-ownership/4.3.3-str与str类型.md)、[4.3.4](../04-ownership/4.3.4-DST与Sized.md) · demo：[5.1-structs-demo](./5.1-structs-demo/)

---

## 默认：字段用 `String`

```rust
struct User {
    username: String,
    email: String,
}
```

结构体**拥有**数据，不绑外部生命周期。存引用需显式生命周期（第 10 章），否则 `missing lifetime specifier`。

---

## `mut` + 字段是 `&str`：换指向，不改只读内存

> **`mut` 管字段里存的引用，不管指向内存能不能写。**

```rust
struct User {
    name: &'static str,
    num: u32,
}

let mut u = User { name: "zhangsan", num: 10 };
u.name = "lisi"; // ✅ 换指针；不改 .rodata 里 "zhangsan" 字节
```

| | `&'static str` 字段 | `String` 字段 |
|--|---------------------|---------------|
| `u.name = 新值` | 换引用指向 | 换堆所有权 |
| 改内存内容 | ❌ | ✅ `push` 等 |

**借外部 `String` 的 `&str`**

```rust
struct User { name: &'str }

let mut outer = String::from("123");
let mut u = User { name: &outer };
u.name = &outer[1..]; // 换指针，不改 outer
// outer drop 后 u.name 失效 → 第 10 章
```

---

## 小结

| 要点 | 一句 |
|------|------|
| 持久化 / 拥有 | 字段用 `String` |
| 协议常量 | 可用 `&'static str` |
| `mut` + `&str` 字段 | 换指向 OK，不改只读区 |
