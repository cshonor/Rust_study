# 9.1.4 整数溢出：wrapping · panic · saturating

← [9.1.2 dev/release 配置](./9.1.2-dev-release与HFT编译配置.md) · demo：`9.1.4-overflow-demo/`

---

## 结论先说

**默认 release：整数溢出 ≠ panic，不会崩溃，走环绕回卷；只有手动开 `overflow-checks = true`，release 溢出才会 panic。**

| 场景 | `overflow-checks` | 裸运算溢出 |
|------|-------------------|------------|
| **dev 默认** | `true`（默认，常省略不写） | **panic** |
| **release 默认** | `false`（默认，常省略不写） | **环绕 wrapping，程序继续跑** |
| **release 手动开 true** | `true` | **panic**（与 dev 一致） |

---

## 一、release 出厂默认（`overflow-checks = false`）

```toml
[profile.release]
# overflow-checks = false  ← 默认隐藏，不写也是 false
```

```rust
let mut a: u8 = 0;
a -= 1; // release → a = 255，不崩溃、不 panic
```

无符号 / 有符号整数加减：**默默绕回**，等价于 mod 2ⁿ。

### 示例：`u8`（0～255）

```rust
let mut y: u8 = 255;
y += 1; // release → 0
```

### 有符号 `i32`（补码）

| 运算 | release 环绕结果 |
|------|------------------|
| `-2147483648 - 1` | `2147483647` |
| `2147483647 + 1` | `-2147483648` |

---

## 二、手动开启溢出检查

```toml
[profile.release]
overflow-checks = true
```

此时 **release 与 dev 一致**：裸运算溢出 → **直接 panic**。

---

## 三、三种溢出行为 + 显式 API

| # | 配置 | 裸运算溢出 |
|---|------|------------|
| 1 | dev 默认 | panic |
| 2 | release 默认 | 环绕，无 panic |
| 3 | release + `overflow-checks = true` | panic |

**不改全局 Cargo** 时，单点控制：

| 模式 | API | 行为 |
|------|-----|------|
| wrapping | `.wrapping_add/sub` | 强制环绕 |
| saturating | `.saturating_add/sub` | 触界停在最值 |
| checked | `.checked_add/sub` | 溢出 → `None` |

```rust
0u8.wrapping_sub(1)    // 255
0u8.saturating_sub(1)  // 0
255u8.saturating_add(1) // 255
```

---

## 四、HFT 重点

1. **生产不要开 `overflow-checks = true`**（性能损耗）。
2. **订单量、Tick 计数** 用 `.checked_add()` / `.checked_sub()`，溢出得 `None`，走风控 — **不靠 panic，也不靠默认绕回**。

```rust
let x = 1u32;
match x.checked_add(u32::MAX) {
    Some(v) => { /* 正常 */ }
    None => { /* 溢出 → 告警 / 拒单 */ }
}
```
