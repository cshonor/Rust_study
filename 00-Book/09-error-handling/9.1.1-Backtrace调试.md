# 9.1.1 Backtrace 完整调试用法

← 主笔记：[9.1 panic!](./9.1-panic-与不可恢复的错误.md) · demo：`9.1.1-backtrace-demo/`（含 `.cargo/config.toml` 示例）

> 三种场景：**panic 自动堆栈** · **手动打印栈** · **catch 后自定义打印** · **配置文件永久开栈**（§1.2）。

---

## 结论先说

**`RUST_BACKTRACE=1/full` 不能写在 `Cargo.toml [profile]` 里** — profile 只管**编译期**参数；backtrace 是**运行时**环境变量。

| 配置位置 | 管什么 |
|----------|--------|
| **`Cargo.toml [profile.*]`** | `debug` / `panic` / `overflow-checks`（编译期） |
| **`.cargo/config.toml [env]`** | 注入 `RUST_BACKTRACE` 等（运行时，**不用 .env、不用每次 export**） |
| **Shell `$env:` / `export`** | 临时覆盖 |

---

## 一、环境变量：panic 自动堆栈

### 1.1 临时：Shell 导出

```bash
export RUST_BACKTRACE=full   # Linux / macOS
cargo run -- oob
```

```powershell
$env:RUST_BACKTRACE = "full"
cargo run -- oob
```

### `1` vs `full`

| 值 | 效果 |
|----|------|
| **`0`** | 关闭 |
| **`1`** | **精简**堆栈（常用，输出较短） |
| **`full`** | **完整**堆栈（含更多帧，调试首选） |

### 1.2 永久：`.cargo/config.toml`（推荐）

**不是 `Cargo.toml`**，是项目根 `.cargo/config.toml` — cargo 运行/测试时**自动注入**：

```toml
# .cargo/config.toml
[env]
RUST_BACKTRACE = "full"
RUST_LIB_BACKTRACE = "1"   # 手动 Backtrace::capture 解析用
```

- `cargo run` / `cargo test` **自动带 backtrace**，无需 shell export
- demo 已含此文件：`9.1.1-backtrace-demo/.cargo/config.toml`

> **注意**：Cargo **没有** `[env.dev]` / `[env.release]` 官方语法 — `[env]` 对**所有** cargo 子命令生效。若只想 dev 开栈：release 前注释掉该行，或 dev 专用分支 config。

### 1.3 release 要打栈：编译 + 运行缺一不可

光开 `RUST_BACKTRACE` 不够 — **release 默认 `debug = false` 无符号，栈只有地址**：

```toml
[profile.release]
debug = "line-tables-only"   # 保留行号，体积远小于 debug=true
panic = "unwind"             # abort 直接杀进程，无栈回溯
```

| 编译期（profile） | 运行时（env） | 结果 |
|-------------------|---------------|------|
| `debug = false` | `RUST_BACKTRACE=1` | 多半只有地址 |
| `debug = line-tables-only` | `RUST_BACKTRACE=full` | 可有行号 |
| `panic = "abort"` | 任意 | **无栈展开**，catch/backtrace 受限 |

### 1.4 代码内强制开启（脱离 shell / config）

```rust
fn main() {
    std::env::set_var("RUST_BACKTRACE", "full");
    // …
}
```

适合临时调试；**HFT 生产勿硬编码**。

### 示例：越界自动打栈

```rust
let arr = vec![1, 2, 3];
arr[10]; // 配合 RUST_BACKTRACE + debug 符号
```

## 二、手动打印当前调用栈（不 panic）

### 方案 1：`std::backtrace`（Rust 1.65+，无需第三方）

```rust
use std::backtrace::Backtrace;

fn func_b() {
    let bt = Backtrace::capture();
    println!("当前调用堆栈:\n{bt}");
}

fn func_a() {
    func_b();
}

fn main() {
    func_a();
}
```

| API | 作用 |
|-----|------|
| **`Backtrace::capture()`** | 捕获当前栈（按需解析） |
| **`Backtrace::force_capture()`** | 强制全量捕获 |

手动栈解析常配合：

```bash
RUST_LIB_BACKTRACE=1 cargo run -- manual
```

release 要行号：`debug = true` 或 **`debug = "line-tables-only"`** → [9.1.2](./9.1.2-dev-release与HFT编译配置.md)。

### 方案 2：第三方 `backtrace` crate（老版本 Rust）

```toml
backtrace = "0.3"
```

```rust
use backtrace::Backtrace;

fn test() {
    let bt = Backtrace::new();
    println!("{bt:?}");
}
```

## 三、`catch_unwind` + 自定义打印堆栈

> **`panic = "abort"` 时无效** — 进程直接退出，无法捕获。

```rust
use std::backtrace::Backtrace;
use std::panic::catch_unwind;

fn risky_code() {
    let v = vec![1];
    v[5];
}

fn main() {
    let res = catch_unwind(risky_code);
    if res.is_err() {
        println!("异常，手动堆栈:\n{}", Backtrace::force_capture());
    }
}
```

→ panic 策略见 [9.1 §一](./9.1-panic-与不可恢复的错误.md)。

## 四、HFT 成套配置示例

### `Cargo.toml`

```toml
[profile.dev]
opt-level = 0
debug = true
overflow-checks = true
panic = "unwind"

[profile.release]
opt-level = 3
debug = "line-tables-only"   # 留行号便于回溯（上线可改 false）
panic = "unwind"             # 要栈用 unwind；实盘常改 abort
overflow-checks = false
```

### `.cargo/config.toml`（开发自动全栈）

```toml
[env]
RUST_BACKTRACE = "full"
RUST_LIB_BACKTRACE = "1"
```

**上线实盘**：去掉或注释 `[env]` 中 backtrace；`panic = "abort"` + `debug = false` → 用 `perf record -g` 排错。

→ profile 总表：[9.1.2](./9.1.2-dev-release与HFT编译配置.md)

## 五、常见问题

| 现象 | 处理 |
|------|------|
| 堆栈只有地址、无行号 | `debug = true` 或 `line-tables-only` + `RUST_BACKTRACE=full` |
| 写进 `[profile]` 不生效 | **`RUST_BACKTRACE` 是运行时 env**，用 `.cargo/config.toml [env]` |
| `abort` 下无栈 / catch 无效 | 设计如此 → [9.1.3](./9.1.3-catch_unwind与panic模式.md) |
| 手动栈 `<disabled>` | `RUST_LIB_BACKTRACE=1` 或 `force_capture()` |
