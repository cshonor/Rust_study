# 3.1 变量和可变性

Rust 中变量**默认不可变**（immutable），可充分利用安全性和简单并发。需要时可声明为可变（mutable）。

对应 demo 项目：`00-Book/03-common-concepts/3.1-variables-demo/`

---

## 不可变变量

一旦值绑定到变量名，就**不能更改**。

```rust
fn main() {
    let x = 5;
    println!("The value of x is: {}", x);
    x = 6;  // ❌ 编译错误！
    println!("The value of x is: {}", x);
}
```

**错误**：`cannot assign twice to immutable variable 'x'`

- 编译器提示：`consider making this binding mutable: mut x`
- 不可变性有助于避免 bug：某处假设值不变，另一处却改了它
- Rust 编译器在编译期就保证「声明不变的值真的不变」

---

## 可变变量 mut

在变量名前加 `mut` 使其可变：

```rust
fn main() {
    let mut x = 5;
    println!("The value of x is: {}", x);
    x = 6;  // ✓ 允许
    println!("The value of x is: {}", x);
}
```

- 输出：`5` 然后 `6`
- `mut` 也提示读者：该变量的值会在别处被修改
- 大型数据结构时，原地修改可能比复制更快；小结构时函数式风格通常更清晰

---

## 常量 const

常量与不可变变量的区别：

| 特性 | 常量 const | 不可变变量 let |
|------|-----------|---------------|
| 可变性 | 永远不可变，不能用 mut | 默认不可变 |
| 声明 | `const` 关键字 | `let` 关键字 |
| 类型 | **必须**显式标注 | 可推导 |
| 赋值 | 仅常量表达式（编译期可算） | 可为运行时计算结果 |
| 作用域 | 可在全局等任意作用域 | 一般局部 |
| 命名 | 全大写 + 下划线 | snake_case |

```rust
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3;
```

- 常量名：全大写，如 `THREE_HOURS_IN_SECONDS`
- 不能是函数调用的结果或运行时才能算出的值
- 适合硬编码值（最高分、光速等），便于维护和修改

## 遮蔽 shadowing

用 `let` 再次声明同名变量，新变量会**遮蔽**旧变量：

```rust
fn main() {
    let x = 5;
    let x = x + 1;        // x = 6
    {
        let x = x * 2;    // 内部 x = 12
        println!("inner scope: {}", x);  // 12
    }
    println!("outer: {}", x);  // 6，内部遮蔽结束
}
```

- 遮蔽是**新变量**，不是修改原变量
- 除非再用 `let`，否则不能重新赋值（仍遵守不可变规则）
- 转换完成后，变量保持不可变
- 
- 本质：创建了一个全新的变量，只是复用了 spaces 这个名字，旧变量被隐藏（无法再访问）。
- 特性：新变量可以拥有完全不同的类型，内存位置也可能不同。
- 
### 遮蔽可改变类型

```rust
let spaces = "   ";      // &str
let spaces = spaces.len(); // usize，遮蔽后类型改变
```

用 `mut` 则**不能**改变类型：

```rust
let mut spaces = "   ";
spaces = spaces.len();   // ❌ 错误：expected `&str`, found `usize`
```
❌ 编译报错：mut 仅允许修改同一个变量的**值**，类型必须严格一致。

- 本质：在同一个内存地址上修改值，变量的类型在声明时就已固定，无法更改。

- 特性：只能修改同类型的值，不能改变变量的类型。


|特性|	变量遮蔽 (Shadowing)|	mut 可变变量|
|------|-----------|---------------|
|本质|	创建新变量，复用名称，旧变量被隐藏|	修改同一个变量的值，内存地址不变|
|类型变更|	✅ 允许（可从 &str 变为 usize）|	❌ 禁止（类型在声明时固定）|
|可变性	|新变量默认不可变（需 let mut 才可变）|	声明时需 mut 才允许修改|
|作用域|	遮蔽仅在当前作用域生效，作用域结束后旧变量恢复|	作用域内始终是同一个变量|
|内存操作|	可能分配新内存，旧变量在作用域结束后销毁|	直接修改原内存，无新分配|

## 未初始化 vs `Default` 默认值

### 核心结论

函数内 **`let x: T;` 声明后不赋值，没有隐式默认值，不能直接使用**（除非编译器能证明**所有代码路径**上都已赋值）：

```rust
// fn main() {
//     let num: i32;
//     let flag: bool;
//     let ch: char;
//     let pi: f64;
//     println!("{}", num);  // ❌ 未初始化
//     println!("{}", flag); // ❌
// }
```

**所有基础类型的局部变量**（`i32`、`bool`、`char`、`f64` 等）规则相同：在函数（含 **`main`**）里用 `let` 声明后，**不赋值就读取，编译必报错**。Rust **不会**自动补 0 / `false` 等。

### `main` 与普通函数规则相同

`main` 只是程序**入口函数**，局部变量规则与其它函数**完全一致**。

**先声明、后赋值**可以，但必须：

1. 使用 **`let mut`**（不可变 `let` 不能写 `num = 66`）；
2. 在**第一次读取之前**，每条代码路径都已赋值。

```rust
fn main() {
    let mut num: i32;
    num = 66;
    println!("{}", num); // 66
}
```

想拿类型默认值须 **`Default::default()`** 或 **`Type::default()`**，与是否在 `main` 无关。

### 何时能拿到「默认值」

需要**显式**构造，常见方式：

| 方式 | 示例 |
|------|------|
| **`Default::default()`** | `let n: i32 = Default::default();` → `0` |
| **类型简写** | `let v: Vec<i32> = Vec::default();` → 空 Vec |
| **数组** | `[0; 5]` 或 `[T; N]::default()`（需 `T: Default`） |
| **`#[derive(Default)]` 结构体** | 各字段按自身 `Default` 填充 |

### 常用类型的 `Default` 值

| 类型 | 默认值 |
|------|--------|
| 整型 `i8`…`i128` / `u8`…`u128` | `0` |
| `f32` / `f64` | `0.0` |
| `bool` | `false` |
| `char` | `'\0'`（U+0000） |
| `String` | `""`（空串） |
| `(T1, T2, …)` | 各元素各自的 `Default` |
| `Option<T>` | `None` |
| `&T` / `&mut T` | **无** `Default` |

```rust
let num: i32 = Default::default();
let flag: bool = Default::default();
let s: String = String::default(); // 与 Default::default() 等价
let mut v: Vec<i32> = Vec::default();
```

**记忆**：局部变量**无自动默认值**；要默认值，**手动** `Default` / `Type::default()`。

> **区分**：`static` 等与局部 `let` 的初始化规则不同；见 Rust 参考手册。

## 小结

| 概念 | 关键字 | 要点 |
|------|--------|------|
| 不可变变量 | `let` | 默认，不能二次赋值 |
| 可变变量 | `let mut` | 可修改值 |
| 常量 | `const` | 必须标类型，仅常量表达式 |
| 遮蔽 | `let` 同名 | 新建变量，可换类型 |
| 默认值 | `Default::default()` | 局部变量无隐式默认，须显式初始化 |
| 占位不用 | `_` / `let _ =` | 见 [3.1-下划线占位符](./3.1-下划线占位符.md) |

**延伸**：[下划线占位符 `_`](./3.1-下划线占位符.md)（`let _ =`、`for _ in`）
