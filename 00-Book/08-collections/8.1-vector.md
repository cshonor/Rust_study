# 8.1 Vector：用 `Vec<T>` 储存一系列的值

## 一句话

> **`Vec<T>` = 堆上动态连续数组** — 同类型元素、自动扩容。对标 Java `ArrayList`、C++ `vector`。

---

## 一、三种创建方式

### 1. 空 Vec：`Vec::new()`

```rust
let mut v: Vec<i32> = Vec::new();
```

无元素时编译器无法推断 `T`；刚创建时**通常不分配堆内存**，第一次 `push` 才向堆申请空间。

#### `Vec::new()` 类型推断详解

**核心：看「上下文」，不是看单行**

`let mut v = Vec::new();` **本身没错** —— Rust 会结合**同作用域里前后代码**一起推断 `v` 的类型。  
只要某处能确定 `T`（常见是后面 `push(5i32)`），就不必写 `Vec<i32>`；**整段上下文都推不出 `T`** 才报错。

**① 没有任何推断线索 → 必须标注**

```rust
// let mut v = Vec::new();              // ❌ 后面不用、前面也没有线索 → E0282
let mut v: Vec<i32> = Vec::new();       // ✅ 手动指定泛型
```

- `Vec::new()` 创建**空动态数组**，创建时通常还没有元素；
- **这一行单独**看不出要存 `i32` 还是 `String` —— 要靠标注，或靠上下文推断。

**② 后面有明确类型操作 → 可省略标注**

```rust
let mut v = Vec::new();
v.push(5i32); // 编译器从 push 的值反推 Vec<i32>
v.push(20);
```

**③ 前面/别处已有类型线索 → 也可省略标注**

```rust
fn takes_vec(v: Vec<i32>) {}

let mut v = Vec::new();
takes_vec(v); // 从函数参数反推 v 必须是 Vec<i32>
```

（同理：赋给已标注类型的变量、匹配带类型的模式等，都能提供线索。）

**④ 已手动标注后 `push` → 不用改标注**

```rust
let mut v: Vec<i32> = Vec::new();
v.push(10); // 验证标注正确；只能继续 push i32
```

**⑤ `vec![]` 自带元素 → 永远不用标注**

```rust
let mut v = vec![1, 2, 3]; // 元素是 i32 → 自动推断 Vec<i32>
```

| 场景 | 要不要写 `Vec<i32>` |
|------|---------------------|
| `Vec::new()` 且**前后都没有**类型线索 | **要**（否则编译报错） |
| `Vec::new()` 后 **`push(5i32)`** 等能定类型的操作 | **不用** |
| `Vec::new()` 后 **传给 `fn(Vec<i32>)`** 等 | **不用** |
| `vec![1, 2, 3]` | **不用** |

### 2. 初始化赋值：`vec![]` 宏（最常用）

```rust
let v = vec![1, 2, 3];   // 自动推导 Vec<i32>
let v2 = vec![0; 5];     // [0, 0, 0, 0, 0] — 仅 Copy 类型可用
```

`vec!` 编译期展开为 `Vec::new()` + 多次 `push`，是简写语法糖。

### 3. 预分配容量：`Vec::with_capacity(N)`（性能优化）

```rust
let mut v = Vec::with_capacity(10);
// 预先预留 10 个元素的空间，前 10 次 push 通常不触发扩容
```

---

### 关键概念：`len` vs `capacity`

| | 含义 |
|---|------|
| **`len()`** | 当前真实存了多少个元素 |
| **`capacity()`** | 堆上已申请、能容纳多少元素（不扩容的前提下） |

当 **`len == capacity`** 时再 `push` → 触发 **`realloc` 扩容**（申请更大块 → 拷贝 → 释放旧块）。

## 二、修改 Vec：`push` 追加尾部

修改必须 **`mut`** — Rust 不可变变量不能改内容。

```rust
let mut v = Vec::new();
v.push(5);
v.push(6);
```

## 四、两种读取：`[]` 下标 vs `.get()`（必考）

### 代码原样

```rust
let v = vec![1, 2, 3, 4, 5];

// ① [] 下标取值
let third = &v[2];
println!("{}", third); // 输出 3

// ② get() 取值，返回 Option<&T>
match v.get(2) {
    Some(val) => println!("{}", val),
    None => println!("无该元素"),
}
```

### 核心区别

| 写法 | 越界（如 `v[10]`） | 返回类型 | 使用场景 |
|------|-------------------|----------|----------|
| **`&v[下标]`** | **panic 崩溃、程序终止** | `&T`（元素引用） | 100% 确定下标合法；追求简洁；出错直接崩 |
| **`v.get(下标)`** | **不崩溃，返回 `None`** | `Option<&T>` | 下标来自用户输入、不确定越界；需容错 |

### 可变版本：`get_mut()`

```rust
let mut v = vec![1, 2, 3];
if let Some(val) = v.get_mut(1) {
    *val = 99; // Option<&mut T>
}
// v == [1, 99, 3]
```

### 越界一眼看懂

```rust
let v = vec![1, 2, 3];
// &v[10];  // ❌ 运行 panic

assert!(v.get(10).is_none()); // ✅ 返回 None，程序继续
```

> **确定下标没问题 → `[]`** · **下标不确定、可能越界 → 必用 `get()`，靠 `Some`/`None` 处理**

### 选型口诀（必背）

| 场景 | 读 | 改 |
|------|-----|-----|
| **下标确定合法** | `&v[i]` | `v[i] = x`（`v` 须 `mut`） |
| **下标不确定** | `v.get(i)` | `v.get_mut(i)` |

**确定下标用 `[]`，不确定用 `get` / `get_mut`。**

## 五、经典报错：有不可变引用时不能 `push`（核心难点）

```rust
let mut v = vec![1, 2, 3, 4, 5];
let first = &v[0];
v.push(6); // ❌ 编译报错 E0502
println!("{}", first);
```

### 底层原理（扩容）

1. `first` 保存堆上**某个地址**的不可变借用；
2. 若 `len == capacity`，`push` 可能 **realloc**：
   - 堆上开更大新块
   - 拷贝所有元素到新地址
   - **释放旧堆内存**
3. 原 `first` 指向的旧内存已无效 → **悬垂引用**；
4. 借用检查器**编译期拦截**，杜绝野指针。

### ✅ 规避：引用用完再 `push`

```rust
let mut v = vec![1, 2, 3];
{
    let first = &v[0];
    println!("{}", first);
} // first 生命周期结束
v.push(6); // ✅
```

## 六、常用拓展方法

```rust
let mut v = vec![1, 2, 3];
v.pop();            // 弹出末尾 Option<T>
v.remove(0);        // 删除指定下标（后面元素前移，O(n)）
v.clear();          // 清空元素，capacity 通常保留
v.shrink_to_fit();  // 释放多余空闲内存，让 capacity ≈ len
```

## 七、遍历

### 不可变遍历

```rust
let v = vec![100, 32, 57];
for i in &v {
    println!("{}", i);
}
```

### 可变遍历并修改

```rust
let mut v = vec![100, 32, 57];
for i in &mut v {
    *i += 50;
}
```

## 八、用枚举在 Vec 中存多种类型

Vec **只能存同一类型**；混合类型用 **enum** 包装：

```rust
enum SpreadsheetCell {
    Int(i32),
    Float(f64),
    Text(String),
}

let row = vec![
    SpreadsheetCell::Int(3),
    SpreadsheetCell::Text(String::from("blue")),
    SpreadsheetCell::Float(10.12),
];
```

编译期无法确定类型集合时 → trait 对象（第 17 章）。
