## 一、`str` / `&str`

| | |
|--|--|
| 本质 | UTF-8 **字符序列**（本体） |
| 大小 | **不定长**（DST） |
| 栈变量 | ❌ `let x: str`；✅ `let x: &str` |
| 所有权 | **借用** |

```rust
let s: &str = "hello world";
```

---

## 二、`String`

| | |
|--|--|
| 本质 | **定长** struct（ptr / len / cap） |
| 指向 | 通常**堆**上 UTF-8（可扩容） |
| 栈变量 | ✅ `let s: String` |
| 所有权 | **有**；出作用域 drop 堆 |

```rust
let mut s = String::from("hello");
s.push_str(" world");
```

### 对照

| | `&str` | `String` |
|--|--------|----------|
| 栈上 | 定长**引用** | 定长**管理头** |
| 数据 | 只读段 / 某堆块 | **自己的堆** |
| 扩容 | ❌ | ✅ |

---

## 三、常见疑问

### `str` 是本体，`String` 是包装？

✅ **`str` = 纯数据；`String` = 定长容器，常见用法是堆上扩容。**

### `str` 只读吗？

| 形态 | 改内容 | 改长度 |
|------|--------|--------|
| 字面量 `&'static str` | ❌ 只读段 | ❌ |
| 从 `String` 借的 `&str` | ❌ 共享借用 | ❌ |
| `&mut str` | ✅ 原地改 | ❌ 不能加长 |
| `String` | ✅ | ✅ |

要**变长、追加** → **`String`**。

### 口语「字符串变量」

- 业务常说「一个 `String`」；
- `&str` 多用于只读参数、`fn foo(s: &str)`。

## 四、三条链路 + 互转

```rust
let s1: &str = "hello";              // 静态段 + &str
let s2: String = String::from("hello"); // 拷到堆
let s3: &str = &s2;                  // 借 String 的视图
```

```rust
let s = String::from("rust");
let r: &str = &s;                    // 零拷贝借

let r: &str = "rust";
let s = String::from(r);             // 或 r.to_string()
```

| 写法 | 说明 |
|------|------|
| `&s` | 零拷贝借用 |
| `String::from` / `to_string` | 新堆 + 拷贝 |
| `format!(...)` | 得到 `String` |

---

## 背诵

- **`&str`**：定长书签在栈；指向静态/栈/堆上的 `str`。
- **`String`**：定长头在栈；动态内容多在堆。
