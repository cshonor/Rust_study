# 8.3 哈希 map 储存键值对（HashMap<K, V>）

← [8.2 字符串](./8.2-string.md) · demo：`00-Book/08-collections/8.3-hashmap-demo/`

---

## 结论（先看这三句）

1. **键要 `Hash + Eq`**；`String`/`&str`/`i32` 可直接做键，**`f32`/`f64` 不行**。
2. **`insert` 对 `String` 是 move、对 `i32` 是拷贝**；想保留原变量就 `insert(&key, val)` 存借用。
3. **更新分两条路**：`insert` 无脑覆盖；`entry(k).or_insert(v)` 缺则插、有则不动（词频统计标配）。

## 基础前置

```rust
use std::collections::HashMap; // 不在 prelude，须手动导入
```

| 要点 | 说明 |
|------|------|
| **作用** | 键 `K` → 值 `V`（词频、配置、分数） |
| **存储** | 键值**全在堆**；同一 Map 内 K 同类型、V 同类型 |
| **键约束** | `K: Hash + Eq`；自定义 struct 需 `#[derive(Hash, Eq, PartialEq)]` |

---

## 快速对照表

| 操作 | 写法 | 特点 |
|------|------|------|
| 全覆盖 | `insert(k, v)` | 覆盖旧值，返回 `Option<旧值>` |
| 缺则插 | `entry(k).or_insert(v)` | 存在不动 |
| 基于原值改 | `*entry(k).or_insert(0) += 1` | 词频 |
| 安全查 | `get(&k)` | `Option<&V>`，不 panic |
| 必存在 | `map["k"]` | 无 key **panic** |
