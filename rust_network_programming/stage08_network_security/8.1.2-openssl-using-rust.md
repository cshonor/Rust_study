## 一、本节在学什么

**`rust-openssl`** 绑定系统 OpenSSL，适合：

- 代码内生成 **RSA 密钥**、**X.509 证书**（集成测试、内网）
- PKCS#12 转换、与遗留 OpenSSL 生态互操作

**生产公网**仍优先 **Let's Encrypt + rustls**；自签名仅 **dev / 内网**。

---

## 二、X.509 证书主要字段

| 字段 | 含义 |
|------|------|
| Version | 证书版本 |
| Serial Number | 序列号 |
| Signature Algorithm | 如 SHA256withRSA |
| Issuer | 颁发者 DN |
| Validity | 生效 / 过期时间 |
| Subject | 主体（CN、O 等） |
| Subject Public Key Info | 公钥及算法 |
| Extensions | SAN（域名列表）、Key Usage 等 |

书中示例流程：

1. 生成 **RSA 3072** 私钥  
2. 构造 X.509，**自签名**（Issuer = Subject）  
3. 签名 **SHA512**，有效期 **365 天**

---

## 三、PEM vs PKCS#12

| 格式 | 内容 | 常见扩展名 |
|------|------|------------|
| **PEM** | Base64 文本，`-----BEGIN CERTIFICATE-----` | `.pem`、`.crt`、`.key` |
| **PKCS#12** | 二进制容器：证书 + 私钥 + 链 | `.pfx`、`.p12` |

命令行自签名（概念）：

```bash
openssl req -x509 -newkey rsa:3072 -nodes -days 365 \
  -keyout key.pem -out cert.pem
```

转 PKCS#12（书中 **tokio-tls + native-tls** 服务端常需要）：

```bash
openssl pkcs12 -export -in cert.pem -inkey key.pem -out server.pfx
```

**rustls 路径**：通常直接配置 **PEM 或 DER**，不必转 PKCS12 — 选型不同，步骤不同。

## 四、`rust-openssl` 适用场景

```text
rust-openssl  →  与系统 OpenSSL 库绑定
适合：OpenSSL 特有 API、PKCS#12、遗留互操作、测试证书生成
不适合：追求零 C 依赖、许可证简单 → 选 rustls
```

本仓库 demo **不链接 OpenSSL**，打印字段与命令流程；完整生成需本机安装 OpenSSL 并启用 `openssl` crate。

仓库对应：[8.1.2-openssl-pem-overview-demo.rs](./8.1.2-openssl-using-rust/8.1.2-openssl-pem-overview-demo.rs)

## 五、新手易错

1. **自签名当生产方案**：浏览器/客户端默认不信任。
2. **CN vs SAN**：现代校验看 **SAN**；仅 CN 可能不够。
3. **私钥与证书不匹配**：启动 TLS 报 handshake 错误。
4. **Windows 上 OpenSSL 路径**：`rust-openssl` 需正确 `OPENSSL_DIR` 或 vcpkg。
5. **混淆 rustls 与 openssl 加载方式**：两套 API，勿混用同一 acceptor 配置。
