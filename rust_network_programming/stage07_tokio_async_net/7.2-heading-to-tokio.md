## 一、本节在学什么

把 Ch3 的多线程阻塞 echo 换成 **Tokio 异步 echo**：

- **`#[tokio::main]`**：启动运行时（Reactor + 线程池）
- **`tokio::net::TcpListener` / `TcpStream`**：与 `std::net` 同名，API 加 `.await`
- **`tokio::spawn`**：每连接一个 **task**（不是 OS 线程）

理解 **`accept().await` 不阻塞其它连接** 是本章核心。

---

## 二、Tokio 网络栈架构

**一句话**：`mio` 注册 fd → `poll` 就绪 → wake task → 再 poll Future。

| 层（书中 `tokio-proto` 概念） | 职责 | 现代近似 |
|-------------------------------|------|----------|
| **Codec** | 字节流 ↔ **帧** | `tokio_util::codec::Framed`、`bytes::BytesMut` |
| **Protocol** | 分帧 + 状态机、多路复用 | 自定义 state machine、`hyper` |
| **Service** | 业务：请求 → 异步响应 | `async fn handler(req) -> Response` |

```text
Socket 字节
   ↓ Codec（分帧）
   ↓ Protocol（状态机、多路复用）
   ↓ Service（业务逻辑）
   ↓ 响应编码回去
```

书中 **`tokio-core` Reactor** → 现为 **`tokio::runtime::Runtime`**。

## 三、最小异步 echo 服务器

与 Ch3 结构一一对应：`bind` → 循环 `accept` → spawn 处理。

```rust
use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::net::TcpListener;

const ADDR: &str = "127.0.0.1:37082";

#[tokio::main]
async fn main() -> std::io::Result<()> {
    let listener = TcpListener::bind(ADDR).await?;
    loop {
        let (mut socket, addr) = listener.accept().await?;
        tokio::spawn(async move {
            let mut buf = [0u8; 1024];
            match socket.read(&mut buf).await {
                Ok(0) => return,
                Ok(n) => {
                    let _ = socket.write_all(&buf[..n]).await;
                    println!("echoed {n} bytes for {addr}");
                }
                Err(e) => eprintln!("read error: {e}"),
            }
        });
    }
}
```

| 步骤 | Ch3 阻塞 | 本章 Tokio |
|------|----------|------------|
| 监听 | `TcpListener::bind` | `bind(...).await` |
| 接受 | `incoming()` / `accept()` | `accept().await` |
| 并发 | `thread::spawn` | `tokio::spawn` |
| 读写 | `Read` / `Write` | `AsyncReadExt` / `AsyncWriteExt` |

仓库 server：[7.2-heading-to-tokio-echo-server-demo.rs](./7.2-heading-to-tokio/7.2-heading-to-tokio-echo-server-demo.rs) · 监听 **`127.0.0.1:37082`**

## 四、异步客户端

```rust
use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::net::TcpStream;

#[tokio::main]
async fn main() -> std::io::Result<()> {
    let mut stream = TcpStream::connect("127.0.0.1:37082").await?;
    stream.write_all(b"tokio 7.2\n").await?;
    let mut buf = [0u8; 1024];
    let n = stream.read(&mut buf).await?;
    println!("echo: {}", String::from_utf8_lossy(&buf[..n]));
    Ok(())
}
```

仓库 client：[7.2-heading-to-tokio-echo-client-demo.rs](./7.2-heading-to-tokio/7.2-heading-to-tokio-echo-client-demo.rs)

## 五、新手易错

1. **先起 server 再起 client**；端口 `37082` 被占用时 `bind` 失败。
2. **`spawn` 的 future 须 `'static`**：不能把栈上借用塞进 long-lived task（常见：先 `move` 拿走 `socket`）。
3. **只读一次就关连接**：demo 读一次即结束；长连接需循环 read 或协议分帧。
4. **在 async 里用 `std::thread::sleep`**：会阻塞 worker；用 `tokio::time::sleep`。
5. **默认 multi-thread runtime**：CPU 密集仍应 `spawn_blocking`，别占满 async worker。
