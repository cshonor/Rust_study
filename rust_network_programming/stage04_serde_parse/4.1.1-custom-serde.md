## 4.1.1 Custom serialization and deserialization（自定义序列化与反序列化）

> 书 §4.1.1 · 贯通稿：[notes/Ch04-…](./notes/Ch04-数据序列化反序列化与解析-学习笔记.md) · 上一节：[4.1 Serde 基础](./4.1-serde-serialization.md) · 下一节：[4.2 文本解析](./4.2-parsing-textual.md)

---

## 一、本节在学什么

`#[derive(Serialize, Deserialize)]` 覆盖绝大多数结构体；以下情况需要**自定义**：

-  wire 字段名与 Rust 字段名不一致（也可用属性宏解决）
- 枚举的 JSON 形态（externally / internally / adjacently tagged）
- 与遗留二进制布局**逐字段对齐**
- 跳过字段、默认值、扁平化嵌套等特殊规则

本节路径：**先属性宏** → **再手写 trait** → **用 `serde_test` 验证**。

---

## 二、首选：`#[serde(...)]` 属性

多数「改名 / 忽略 / 默认值」不必手写 trait。

| 属性 | 作用 |
|------|------|
| `rename = "wire_name"` | JSON 键与 Rust 字段不同 |
| `rename_all = "camelCase"` | 整结构体字段统一风格 |
| `skip_serializing` / `skip_deserializing` | 某字段不参与某一方向 |
| `default` | 缺失字段用 `Default::default()` |
| `tag` / `content` | 枚举序列化形态（adjacent 等） |

### 示例：`rename`（本仓库 demo）

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Serialize, Deserialize)]
struct Point {
    #[serde(rename = "custom_x")]
    x: f64,
    #[serde(rename = "custom_y")]
    y: f64,
}

fn main() -> serde_json::Result<()> {
    let p = Point { x: 3.0, y: 4.0 };
    let json = serde_json::to_string(&p)?;
    // 输出形如 {"custom_x":3.0,"custom_y":4.0}
    let back: Point = serde_json::from_str(&json)?;
    Ok(())
}
```

仓库对应：[4.1.1-custom-serde-rename-demo.rs](./4.1.1-custom-serde/4.1.1-custom-serde-rename-demo.rs)

## 三、手写 `Serialize`

宏无法表达时，实现 `serde::Serialize`，调用序列化器的**结构化 API**：

```rust
use serde::ser::{Serialize, SerializeStruct, Serializer};

struct MyType {
    x: f64,
    y: f64,
}

impl Serialize for MyType {
    fn serialize<S: Serializer>(&self, serializer: S) -> Result<S::Ok, S::Error> {
        let mut st = serializer.serialize_struct("MyType", 2)?;
        st.serialize_field("custom_x", &self.x)?;
        st.serialize_field("custom_y", &self.y)?;
        st.end()
    }
}
```

要点：

1. **`serialize_struct(name, len)`** 的 `len` 是字段个数，错填可能导致格式错误。
2. 也可用 `serialize_tuple`、`serialize_map` 等，取决于目标格式。
3. 与 `#[serde(rename)]` 等价场景优先用属性，减少维护成本。

## 四、手写 `Deserialize` 与 Visitor

反序列化更复杂：输入可能是 **map、seq、字符串、数字** 等，需实现 **`Visitor`**，在回调里**逐步构造**目标类型。

```text
JSON/二进制输入 → Deserializer → Visitor::visit_* → 你的 Struct
```

常见回调：

| 方法 | 何时调用 |
|------|----------|
| `visit_map` | JSON 对象 `{ "k": v }` |
| `visit_seq` | JSON 数组 `[ ... ]` |
| `visit_str` | 字符串字段 |
| `visit_f64` / `visit_i64` | 数值字段 |

```rust
// 概念骨架（完整实现需处理 map 的 key/value 状态机）
use serde::de::{self, Deserialize, Deserializer, Visitor};

impl<'de> Deserialize<'de> for MyType {
    fn deserialize<D: Deserializer<'de>>(deserializer: D) -> Result<Self, D::Error> {
        deserializer.deserialize_struct("MyType", &["custom_x", "custom_y"], MyVisitor)
    }
}

struct MyVisitor;

impl<'de> Visitor<'de> for MyVisitor {
    type Value = MyType;
    fn expecting(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        write!(f, "struct MyType")
    }
    // visit_map / visit_seq ... 组装 MyType
}
```

书中通过 `visit_map` 读键值对、`visit_str` 读字符串，是理解 Serde **插件式格式**的关键。

## 五、单元测试：`serde_test`

手写实现应测**令牌流（token stream）**，而不必每次 round-trip 真 JSON：

```rust
// 概念示例（需 dev-dependency serde_test）
use serde_test::{assert_tokens, Token};

assert_tokens(
    &MyType { x: 1.0, y: 2.0 },
    &[
        Token::Struct { name: "MyType", len: 2 },
        Token::Str("custom_x"),
        Token::F64(1.0),
        Token::Str("custom_y"),
        Token::F64(2.0),
        Token::StructEnd,
    ],
);
```

适合验证输出是否与**协议规范**逐字段一致。

## 六、何时用哪种方式

| 需求 | 推荐 |
|------|------|
| 字段改名、大小写 | `#[serde(rename)]` / `rename_all` |
| 可选字段、缺省 | `Option<T>` + `#[serde(default)]` |
| 枚举 JSON 形态 | `#[serde(tag = "type", content = "data")]` 等 |
| 非 JSON 二进制布局 | 手写 `Serialize` / `Visitor`，或换 `bincode` + derive |
| 协议合规测试 | `serde_test::assert_tokens` |

---

## 七、新手易错

1. **一上来就手写 Visitor**：先查 [Serde 属性列表](https://serde.rs/attributes.html)，多数需求属性即可。
2. **`serialize_struct` 字段数不符**：少声明或多 `serialize_field` 会导致运行时错误。
3. **`Visitor` 只实现一个 `visit_*`**：反序列化 map 时要处理未知键、缺键、类型错（返回 `invalid_type`）。
4. **rename 只改一边**：序列化与反序列化用同一套属性；手写时两边都要对齐 wire 名。
5. **与 [4.1](./4.1-serde-serialization.md) 分帧混淆**：自定义 Serde 解决「字段形态」；TCP 粘包仍在 I/O 层用长度前缀等（[4.3](./4.3-parsing-binary.md)）。
