## 2. 四大迭代维度（§1.10～1.13）

### 2.1 协议层（§1.2 / 1.3 · 第 11 章 `getaddrinfo`）

```
硬编码 IPv4 (AF_INET, sockaddr_in)
    → 硬编码 IPv6 (AF_INET6, sockaddr_in6)
    → AF_UNSPEC + getaddrinfo（协议无关）
```

| 阶段 | 特点 | HFT |
|------|------|-----|
| 写死 IPv4/IPv6 | 一套代码只跑单栈 | 早期 demo / 内网固定环境 |
| **协议无关** | 运行期解析，双栈兼容 | **网关工程标配**（机房 IPv4/IPv6 混部） |

---

### 2.2 IO 模型（**HFT 最高权重** · 第 5 / 6 / 14 章）

```
阻塞 IO（read/accept 卡死等待）
    → select
    → poll
    → epoll + 非阻塞 IO
```

| 模型 | 行为 | HFT |
|------|------|-----|
| **阻塞** | 一处等待，全进程卡住 | 仅适合单连接 demo |
| **select / poll** | 多路复用，可轮询多 fd | 过渡理解 |
| **epoll** | 海量连接、单线程事件循环 | **行情收发底层核心** |

> **看书优先级**：凡涉及 **select / poll / epoll** 的改造章节，作为 UNP 精读**第一优先级**。

---

### 2.3 并发模型（第 5 章进程 · 第 26 章 pthread）

```
迭代服务器（串行：处理完 A 再 accept B）
    → fork 每连接一子进程
    → pthread 线程池
```

| 模型 | 行为 | HFT |
|------|------|-----|
| **迭代** | 无并发 | 教学起点 only |
| **fork** | 每连接一进程，开销大 | 传统 C 服务，HFT 少见 |
| **线程池** | 固定工作线程处理连接/事件 | **网关主流**（配合 epoll 分发） |

---

### 2.4 传输协议（第 8 章 UDP · 第 21 章 SCTP）

```
TCP 面向连接（Daytime）
    → UDP 无连接（Echo / 组播行情）
    → SCTP（多流、部分场景）
```

| 协议 | 典型用途 | HFT |
|------|----------|-----|
| **TCP** | 可靠、流式、握手 | **交易指令、回报、管理通道** |
| **UDP** | 无握手、低延迟、可组播 | **A 股等证券行情组播（重点）** |
| SCTP | 多宿主 / 多流 | 特定交易所或电信场景，按需 |

> **UDP 演进至关重要**：国内行情大量为 **UDP 组播**；Echo 回射案例是理解无连接收发的模板。

## 4. HFT 定向学习路线

推荐优化顺序（与自研行情 / 交易网关一致）：

```text
1. 跑通 TCP 阻塞 Daytime / Echo demo（理解 API）
2. Echo → epoll 非阻塞（单线程事件循环）
3. 线程池分发连接或报文（pthread / 线程池章节）
4. getaddrinfo / 双栈（部署与容灾）
5. UDP 组播接入行情（第 8 章 + 实际交易所规范）
```

| 优先级 | 内容 |
|--------|------|
| **P0** | epoll、非阻塞、`EAGAIN`、边缘/水平触发 |
| **P0** | UDP、组播、`SO_REUSEPORT`（视场景） |
| **P1** | 线程池、连接与 session 生命周期 |
| **P1** | 协议无关地址、双栈 |
| **P2** | SCTP、fork 模型（了解即可） |
