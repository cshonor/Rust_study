# 14.1 采用发布配置自定义构建

在 Rust 里，**发布配置（release profiles）** 是预定义、可定制的一组编译选项；不同配置彼此独立，用来在「开发时快速迭代」和「发布时优化运行」之间做权衡。

对应 demo 项目：`14-cargo-crates/14.1-profiles-demo/`

---

## Cargo 的两个主要配置

| 命令 | 使用的配置 | 典型用途 |
|------|------------|----------|
| `cargo build` / `cargo run` | **dev** | 开发：默认不追求最高优化，编译更快 |
| `cargo build --release` / `cargo run --release` | **release** | 发布：更高优化，运行更快，编译更慢 |

构建日志里的 `dev` / `release` 就是在标明当前用的是哪套配置。

---

## 默认与覆盖：`[profile.*]`

若 `Cargo.toml` 里**没有** `[profile.*]`，Cargo 对每种 profile 使用内置默认值。

常见默认值（概念上）：

- **`[profile.dev]`**：`opt-level = 0`（几乎不优化，编译快）
- **`[profile.release]`**：`opt-level = 3`（较高优化，运行快）

`opt-level` 取值一般为 **0～3**：数值越大，优化越多，**编译时间越长**，通常 **运行越快**。

在 `Cargo.toml` 里增加对应小节即可**只覆盖**你想改的那几项，其余仍用默认。例如希望在 **dev** 下也用一点优化（比 0 强、比 release 弱）：

```toml
[profile.dev]
opt-level = 1
```

之后 `cargo build` 会沿用 dev 的其它默认，只把 `opt-level` 从默认的 0 改成 1。

完整选项与默认值以 [Cargo 手册：Profiles](https://doc.rust-lang.org/cargo/reference/profiles.html) 为准。

---

## demo 说明

`14.1-profiles-demo/Cargo.toml` 里演示了在 dev 中设置 `opt-level = 1`。若你希望恢复「完全默认的 dev」，删掉 `[profile.dev]` 整段即可。

验证构建使用的 profile：

```bash
cd 14-cargo-crates/14.1-profiles-demo
cargo build
cargo build --release
```
