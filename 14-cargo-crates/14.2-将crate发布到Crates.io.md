# 14.2 将 crate 发布到 Crates.io

[crates.io](https://crates.io) 用来分发 **crate 的源代码**，通常面向开源。发布前需要：写好文档与对外 API、补全 `Cargo.toml` 元数据、登录账号，再用 `cargo publish` 上传。

对应 demo 项目：`14-cargo-crates/14.2-crates-io-publish-demo/`（本地学习用，**请勿随意 `cargo publish`**）

---

## 文档注释：`///` 与 `//!`

- **`///`**：写在**紧挨着的下一个项**之前，为该函数/类型等生成文档；支持 Markdown。
- **`//!`**：为**包含这段注释的容器**写文档（常见于 `src/lib.rs` 或模块文件开头），用于描述 **crate 或整个模块** 的用途。

运行 `cargo doc` 生成 HTML 到 `target/doc`；`cargo doc --open` 会在浏览器中打开（含依赖文档）。

---

## 文档里常用小节

在 `///` 中可用 Markdown 标题组织内容，例如：

- **Examples**：用法示例
- **Panics**：可能 `panic!` 的条件
- **Errors**：若返回 `Result`，可能出现的错误
- **Safety**：`unsafe` 相关约定（本书后文）

---

## 文档示例即测试：`cargo test`

文档注释里用 ** fenced code block（\`\`\`）** 写的示例会被当作 **doc test** 运行。这样文档与代码不易脱节。

---

## `pub use` 与对外 API

内部可以按模块拆分（`kinds`、`utils`），对外却可以用 `pub use` **重导出**，让用户写成 `use crate_name::Type` 而不是深层路径。内部组织与对外展示可以解耦。

---

## 发布前：账号与 `cargo login`

1. 在 [crates.io](https://crates.io) 用 GitHub 登录  
2. 在账户页获取 **API token**  
3. 本地执行：`cargo login <token>`（token 会写入凭据文件，勿泄露）

---

## `Cargo.toml` 元数据

发布时至少需要 **`description`** 与 **`license`**（或 `license-file`）。`license` 常用 [SPDX](https://spdx.org/licenses/) 标识符，例如 `MIT`、`Apache-2.0`，或组合 `MIT OR Apache-2.0`。

还需全局唯一的 **`name`**（先到先得）。更多字段见 [Cargo 手册：The package section](https://doc.rust-lang.org/cargo/reference/manifest.html#the-package-section)。

---

## `cargo publish`

- 会打包并上传**某一版本**；**已发布版本不可删除、不可覆盖**（保证依赖长期可解析）。
- 新版本：修改 `version`（建议遵循[语义化版本](https://semver.org/lang/zh-CN/)）后再 `cargo publish`。

---

## `cargo yank`

- **撤回（yank）**：阻止**新**项目把该版本写进依赖；已有 `Cargo.lock` 的项目仍可解析到该版本。  
- 示例：`cargo yank --vers 1.0.1`，撤销：`cargo yank --vers 1.0.1 --undo`  
- yank **不会**删除已上传源码；误传密钥等需轮换密钥，不能指望“删掉版本”。

---

## demo 命令

```bash
cd 14-cargo-crates/14.2-crates-io-publish-demo
cargo test          # 含单元测试与 doc test
cargo doc --open    # 查看渲染后的文档（可选）
```
