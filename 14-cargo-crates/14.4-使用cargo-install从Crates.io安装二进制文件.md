# 14.4 使用 `cargo install` 从 Crates.io 安装二进制文件

`cargo install` 用于在**本机**安装并运行已在 [crates.io](https://crates.io) 上发布的**二进制 crate**（带可执行目标的包）。它**不是**系统包管理器（如 apt、Homebrew）的替代品，而是方便 Rust 开发者快速安装别人发布的命令行工具。

本小节为**工具与路径说明**，无需单独 demo 项目；可在本机联网后自行尝试。

---

## 能装什么：二进制目标

只有 crate 里存在**二进制目标**时才能安装，例如：

- 默认的 `src/main.rs`，或
- `[[bin]]` 等配置出的可执行文件

**纯库**（只有 `src/lib.rs`、供别的程序依赖）不能通过 `cargo install` 变成“可运行程序”。具体是库、二进制或两者兼有，通常见 crate 的 README。

---

## 安装到哪里

用 **rustup** 默认安装时，二进制一般会装到 Rust 工具链根目录下的 **`bin`**：

| 平台 | 典型路径 |
|------|-----------|
| Linux / macOS | `$HOME/.cargo/bin` |
| Windows | `%USERPROFILE%\.cargo\bin` |

安装过程末尾会打印实际路径与可执行文件名（例如 `ripgrep` 包提供的命令往往是 **`rg`**）。

---

## 如何直接运行：加入 `PATH`

把上述 **`bin` 目录**加入系统的 **`PATH`**，才能在任意终端里直接输入命令名（如 `rg --help`），而不用写全路径。

---

## 示例：安装 ripgrep

```bash
cargo install ripgrep
```

成功后若 `PATH` 已包含 `~/.cargo/bin`（或 Windows 下对应目录），即可：

```bash
rg --help
```

（包名是 `ripgrep`，可执行文件常见名为 `rg`，以安装输出为准。）

---

## 说明

- 会从 registry 下载并**编译**指定版本，首次可能较久。
- 与项目内 `[dependencies]` 不同：`cargo install` 面向**全局/用户级**工具安装；项目依赖仍写在各 crate 的 `Cargo.toml` 中。
