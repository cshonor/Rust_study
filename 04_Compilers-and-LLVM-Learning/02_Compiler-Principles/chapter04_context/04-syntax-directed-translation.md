鉴于严格**属性文法**的性能与工程瓶颈，工业编译器普遍采用 **Ad Hoc Syntax-Directed Translation（特定语法制导翻译）** — 在语法分析过程中**嵌入代码**传递语义信息。

---

## 动作嵌入（Actions）

| 做法 | 说明 |
|------|------|
| **时机** | 识别**句柄**、**归约**或构造子树时 |
| **内容** | 执行 C/代码片段 — 建符号表、查类型、生成 IR 片段 |
| **优势** | 无需保留整棵分析树；按需计算 |

**自顶向下**：在递归下降函数里直接写语义逻辑（**CI jlox**）。  
**自底向上**：在 YACC/Bison 产生式旁写 `{ … }` 动作。

---

## YACC / Bison 栈上值传递

```yacc
expr : expr '+' term  { $$ = makeAdd($1, $3); }
     | term           { $$ = $1; }
     ;
```

| 伪变量 | 含义 |
|--------|------|
| **`$1`, `$2`, …** | 产生式右部各符号的**语义值**（栈上） |
| **`$$`** | 左部非终结符的**结果值**（归约后推回栈顶） |

极高效率 — 与 LR **移入-归约栈**一体。

→ [ch3 §4 LR 表](../chapter03_parsers/04-lr-automation.md)

## 其他 SDT 用途

| 用途 | 例子 |
|------|------|
| **变量声明** | 登记符号表、分配存储类 |
| **作用域** | 进入/退出块时 push/pop 环境 |
| **类型检查** | 归约 `expr + expr` 时检查 `$1.type` 与 `$3.type` |

**jlox Resolver**（ch11）≈ 单独一遍 SDT，在解释前解析绑定 — 与「parser 内嵌 action」分工不同，目的一致。
