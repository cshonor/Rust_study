视线**局限在单个基本块**内的简单策略 — 教学基线，全局方案的基础构件。

---

## 共同设定

块内临时维护 **k 个**物理寄存器「占用表」；遇到新 def/use 时若无空闲 → **淘汰**某一占用者（可能 spill）。

---

## 自顶向下（Top-Down）

| 思路 | 说明 |
|------|------|
| 扫描块内指令 | 遇 **use** 确保操作数已在寄存器（否则 load） |
| 遇 **def** | 分配空闲 preg；无则淘汰 |
| 淘汰启发式 | **下次使用最远**、**使用次数少** |

```text
// 概念：块内仅看「距离下次 use 的指令数」
def v1 … ; … 很多指令 … ; use v1  → v1 可晚分配
def v2 … ; use v2              → v2 急需寄存器
```

**优点**：实现直观。**缺点**：不看**块外 use** — 可能把块外仍需要的值 spill 错。

---

## 自底向上（Bottom-Up）

| 思路 | 说明 |
|------|------|
| 预计算块内各点 **next-use 距离** | 自后向前扫描 |
| 分配 / 淘汰 | 优先保留 **next-use 近** 的值 |

与 **Belady 式**「最远下次使用先踢」类似 — 块内近似最优。

---

## 局部方法的局限

| 局限 | 后果 |
|------|------|
| **忽略跨块 LIVE** | 块入口/出口与邻块不一致 |
| **无全局 spill 代价** | 可能反复 load/store 同一值 |
| **循环** | 块内频率 ≠ 全局热度 |

→ 需要 [§3 存活范围](../chapter13_regalloc/03-live-ranges.md) + [§4 图着色](./04-graph-coloring.md)。
