# 第 11 章 · 指令筛选 · §1 简单树遍历方案

← [本章目录](./README.md) · 上一节：[00-overview.md](./00-overview.md) · 下一节：[02-tree-pattern-matching.md](./02-tree-pattern-matching.md)

---

最直观的基础方法：把 IR 表达式看成**树**，遍历节点逐一生成机器指令。

---

## 基本思路

| 方向 | 做法 |
|------|------|
| **自顶向下** | 从根操作符出发，为每个运算符 emit 对应指令 |
| **自底向上** | 先处理子树，结果进临时/寄存器，再组合 |

```text
        +
       / \
      *   c
     / \
    a   b

// 概念序列（朴素）
load a → load b → mul → load c → add
```

**寄存器/临时**：子树结果通常先放**虚拟寄存器**或栈槽 — 具体分配留给 ch13。

---

## 优点

| 优点 | 说明 |
|------|------|
| **实现简单** | 教学、原型后端首选 |
| **局部正确** | 每个节点有明确一对一映射 |
| **易调试** | 生成顺序与树结构一致 |

---

## 局限

| 局限 | 说明 |
|------|------|
| **忽略融合指令** | 现代 ISA 常有 `load-add`、`lea`、MAC 等**一条顶多条** |
| **寻址模式** | `base+index*scale` 等模式难在「一节点一指令」里体现 |
| **全局代价** | 局部贪心 ≠ 整树最小指令数/周期 |

→ 需要 **§2 树模式匹配** 在整树上做**最优/近优铺盖**。

---

## 与 LLVM / Rust

- 早期或极简后端可能类似树 walk；**LLVM Legalize + SelectionDAG** 已升级为模式匹配。
- `rustc` 不自己做 x86 选择 — 交给 LLVM；理解树 walk 有助于读 **DAG 可视化**。
