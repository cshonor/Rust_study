**Peephole Optimization** — 极具实效的**局部**代码改进，常与指令筛选**串联**或作为后处理 Pass。

---

## 基本原理

在**很小、滑动**的指令窗口（**窥孔**）内检查**相邻**刚生成的目标码：

```text
[ mov r1, mem(a) | add r1, r2 | mov mem(b), r1 ]
         ↑____________ 窥孔窗口 ____________↑
```

若窗口内匹配**已知低效模式** → 替换为更短/更快的**等效序列**。

| 特点 | 说明 |
|------|------|
| **局部性** | 不看全函数 — 实现简单、速度快 |
| **后处理** | 可在树匹配之后「捡漏」 |
| **机器相关** | 规则随 ISA 而变 |

---

## 窥孔转换器（Peephole Transformers）

典型替换模式：

| 低效模式 | 改进 |
|----------|------|
| `store` 后立即 `load` 同一地址 | 删冗余 load，或合并 |
| `mov r, x` · `mov y, r` | 若 r 仅中转 → `mov y, x` |
| 零寄存器写法 | `add r, 0` → 删或改 nop |
| 未用的 `push`/`pop` 对 | 删除 |

**寻址模式**：把 `load` + `add` + `store` 合成一条带 **base+index** 的内存操作（若 ISA 支持）。

---

## 与树模式匹配的分工

```text
IR 树  →  [树模式匹配]  →  初始汇编
                ↓
         [窥孔 Pass 多轮]
                ↓
         更紧凑的目标码
```

| 手段 | 视野 | 擅长 |
|------|------|------|
| **树匹配** | 表达式树全局铺盖 | 融合算术、选正确指令形态 |
| **窥孔** | 2～5 条指令窗口 | 删冗余 move/load、微调序列 |

二者**互补** — LLVM 在指令选择后仍有 **MachineInstr peephole**。
