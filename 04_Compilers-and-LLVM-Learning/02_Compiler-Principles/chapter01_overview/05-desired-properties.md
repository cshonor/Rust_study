除**正确翻译**外，现代编译器在工程上还期望：

---

## 1. 速度与空间

| 对象 | 期望 |
|------|------|
| **生成程序** | 运行快、体积合理 |
| **编译器自身** | **编译速度**够快（开发迭代） |
| **编译过程** | 占用内存（空间）合理 |

**Trade-off**：`-O0` 编译快、运行慢；`-O3` 编译慢、运行快 — 同一编译器在不同维度做实用性权衡。

---

## 2. 反馈与调试

| 能力 | 说明 |
|------|------|
| **错误反馈** | 源程序不合法时，给出**简明有用**的诊断（行号、原因、建议） |
| **调试透明** | 保留 **源码 ↔ 目标码** 映射（行号、变量、内联回溯），供 **Debugger** 使用 |

**Rust**：`rustc` 错误信息、`#[track_caller]`、debuginfo（`-C debuginfo=2`）均属此列。

---

## 本章收束

第一章搭建的是**全书地图**：

- **契约**：正确性 + 实用性
- **结构**：前端 → 优化器 → 后端
- **过程**：理解 → 环境 → 优化 → 代码生成（选指令 / 寄存器 / 调度）
- **工程**：速度、空间、诊断、可调试性
- **本质**：在各种约束间的 **Trade-offs**

后续章节逐站深入各 Pass 的算法与实现。
