# 第 1 章 · 编译总览 · §4 翻译过程关键步骤

← [本章目录](./README.md) · 上一节：[03-three-phase-structure.md](./03-three-phase-structure.md) · 下一节：[05-desired-properties.md](./05-desired-properties.md)

---

本章用赋值语句 **`w <- w * 2 * x * y * z`** 拆解编译器全流程（书中以 **ILOC** 汇编作目标示例）。

---

## 1. 理解输入（前端）

| 层次 | 做什么 |
|------|--------|
| **词法** | 识别 `w`、`*`、`2`、`x`、`y`、`z` 等 Token |
| **语法** | 确认句子结构合法（赋值语句、表达式树） |
| **语义** | 类型是否匹配、变量是否已声明、运算是否有意义 |

→ 确认程序具有**明确意义**，才进入 IR。

---

## 2. 创建与维护运行环境

- 为变量 **分配存储**（寄存器或内存地址）。
- 管理程序执行时的 **地址空间映射** 与 **过程间交互**（调用约定、栈帧等）。

**Rust/HFT**：栈槽 vs 堆、调用约定、`repr(C)` 布局 — 都在「运行环境」层落地。

---

## 3. 改进代码（优化）

通过 **数据流分析** 等手段推断运行时**值的流向**，在**不改变语义**前提下做安全变换。

| 典型优化 | 例子 |
|----------|------|
| **循环不变式外提** | 把循环内不变计算移到循环外 |
| **常量传播 / 折叠** | 编译期算掉已知常量 |
| **死代码消除** | 删除不可达或无效果的 IR |

→ [04 ir_samples/optimize_compare](../../../04_Learn-LLVM-17/ir_samples/optimize_compare/) · CI ch30

---

## 4. 代码生成（后端三难点）

将 IR 落到真实机器，书中强调三个主要难点：

### 4.1 指令筛选（Instruction Selection）

- 为每个 IR 操作选择合适的**机器指令**。
- 书中示例目标：**ILOC** 汇编（教学用假想指令集）。

### 4.2 寄存器分配（Register Allocation）

- 物理寄存器**数量有限**。
- 决定在程序各点哪些值驻留寄存器、哪些 spill 到内存。
- 目标：**最小化内存访问**（HFT 热点路径上极敏感）。

→ RFR 第 2 章 · 寄存器 vs 栈

### 4.3 指令调度（Instruction Scheduling）

- **重排**指令顺序，减少 CPU 等待（如内存读延迟、流水线气泡）。
- 直接影响**运行性能** — 与 HFT **延迟**直接相关。

---

## 流水线对照（一句话）

```text
理解输入 → 分配环境 → 优化 IR → 选指令 + 分寄存器 + 调度 → 可执行目标码
```

与 Rust：`rustc` 前端 → MIR 优化 → LLVM IR → LLVM 后端（含 regalloc + scheduling）。
