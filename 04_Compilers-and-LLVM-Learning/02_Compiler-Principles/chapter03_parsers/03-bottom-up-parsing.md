从输入**叶子（Token）**开始，**自下而上**组合符号，直至归约到**开始符号**。

---

## 移入-归约（Shift-Reduce）

| 动作 | 含义 |
|------|------|
| **移入（Shift）** | 将当前输入符号**压栈** |
| **归约（Reduce）** | 栈顶一串符号匹配某产生式**右部**（**句柄 handle**）→ 替换为左部非终结符 |

```text
栈:  …  id  +  id     输入:  ;
     归约 expr → expr + term
     …
     最终栈:  program  →  接受
```

分析器在每一步决定 **shift** 还是 **reduce** — LR 族算法的核心。

---

## LR(1) 分析

| 项目 | 说明 |
|------|------|
| **L** | 从左到右扫描 |
| **R** | **最右推导**的逆过程（自底向上） |
| **(1)** | 向前看 1 个 Token |

- **文法覆盖范围**比 LL(1) **更广** — 多数编程语言结构可表达。
- **YACC / Bison** 的默认底层技术。

---

## 自顶向下 vs 自底向上

| | **自顶向下（LL / 递归下降）** | **自底向上（LR / 移入-归约）** |
|---|------------------------------|--------------------------------|
| 方向 | 根 → 叶 | 叶 → 根 |
| 典型实现 | 手写函数 | **表驱动**（Action/Goto） |
| 工具 | 少见生成器 | **yacc / bison** |
| 本仓库 | **CI jlox / clox Pratt** | 橡书理论 · 大型语言 parser |

---

## 句柄（Handle）

自底向上正确性的关键：在每一步，**唯一**应归约的右部片段。LR 自动机正是为了**识别句柄**而构造。
