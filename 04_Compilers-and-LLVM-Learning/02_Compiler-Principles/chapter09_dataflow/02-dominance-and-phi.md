## 支配（Dominance）

| 定义 | 节点 **A 支配 B** ⇔ 从入口到 **B** 的**每条**路径都经过 **A** |
|------|-------------------------------------------------------------------|

| 衍生 | 说明 |
|------|------|
| **立即支配（idom）** | 严格支配 B 的**最小**支配者 — 形成**支配树** |
| **支配边界（DF）** | 放置 φ 的关键位置 |

```text
       entry
        │
        A  ← A 支配 B、C
       / \
      B   C
       \ /
        D  ← D 的汇合：可能需要 φ
```

**用途**：高效决定 **φ 放哪** — [ch8 支配者 VN](../chapter08_intro-opt/03-beyond-basic-blocks.md)

---

## 放置 φ 函数

**目标**：在控制流**汇合点**合并不同路径的值，且 **φ 个数最少**（Cytron 等经典算法）。

| 步骤（概念） | |
|--------------|---|
| 1 | 对每个变量，找**定值块** |
| 2 | 在定值块的 **支配边界** 上插入 φ |
| 3 | 迭代直到不动点 |

```text
if (c)  x1 = …;  else  x2 = …;
        x3 = φ(x1, x2)   ← 汇合块入口
```

**LLVM**：IR **已是 SSA** — `phi` 指令；`rustc` 前端产出 MIR 后再 lowering。
