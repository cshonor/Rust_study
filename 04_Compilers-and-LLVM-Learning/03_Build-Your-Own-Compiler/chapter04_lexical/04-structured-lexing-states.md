## 最长匹配的局限

**块注释** `/* … */`、**字符串** `"…"`、**字符** `'…'` 有 **起始/结束 delimiter** 与 **中间任意内容**。

纯 **最长匹配 + 单条正则** 可能 **过度匹配**：

```text
/* comment */ int x;
     ↑ 若正则贪婪错误，可能把 */ 后代码也吞进「注释」
```

需要 **分阶段识别** — **词法状态** + **MORE**。

---

## 状态迁移（Lexical States）

JavaCC 支持多种 **扫描状态**；某状态下 **仅该状态的规则生效**。

| 状态 | 作用 |
|------|------|
| **`DEFAULT`** | 初始；普通 TOKEN / SKIP |
| **`IN_BLOCK_COMMENT`** | 只在块注释 **内部** 匹配直到 `*/` |
| **（字符串/字符专用态）** | 类似，直到闭合 quote |

```text
DEFAULT:  见 "/*"  →  切 IN_BLOCK_COMMENT
IN_BLOCK_COMMENT:  见 "*/"  →  回 DEFAULT，结束注释
```

**效果**：注释内字符 **不会** 被 DEFAULT 里的标识符/关键字规则误读。

---

## MORE 命令

**`MORE`**：当前匹配 **不立即结束 token**，继续 **累积同一 token** 的图像 — 表示「还在扫同一个结构化单词」。

| 对比 | 行为 |
|------|------|
| **仅 SKIP 分块** | 文件在注释/字符串 **未闭合** 时结束 → 可能 **静默错误** |
| **MORE + 状态** | 未闭合即 EOF → **正确报错** |

```text
进入 "/*  → MORE 累积
… 注释体 …
见 */    → 完成 SKIP 或 SPECIAL，回到 DEFAULT
EOF 在未闭合 →  lex error
```

---

## 实际应用（本书实现）

| 结构 | 手段 |
|------|------|
| **块注释** `/* … */` | DEFAULT → IN_BLOCK_COMMENT + MORE → 见 `*/` 结束 |
| **字符串** `"Hello"` | 专用态 + MORE 直到 `"`（处理转义见书中规则） |
| **字符** `'A'` | 更短闭合态 + MORE |

→ 完成后 token 流与 C 习惯一致；非法未闭合串在 **词法阶段** 失败。
