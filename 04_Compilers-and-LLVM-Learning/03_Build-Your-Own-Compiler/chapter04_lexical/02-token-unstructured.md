## TOKEN 命令

**`TOKEN`**：定义 **会生成 token** 的正则规则 — 匹配成功 → 输出对应种类的 token。

```text
TOKEN : {
  <INTEGER: ["0"-"9"]+>
  <IDENTIFIER: ["a"-"z","A"-"Z","_"] (["a"-"z",…,"0"-"9","_"])*>
  …
}
```

## 标识符与保留字 · 最长匹配

| 原则 | 说明 |
|------|------|
| **最长匹配** | 所有规则 **同时尝试**，取 **匹配长度最长** 者 |
| **同长优先** | 长度相同 → **文件中先定义** 的规则优先 |

```text
规则顺序示例:
  "if"      → 保留字 IF
  IDENT     → 标识符

输入 if     → 关键字 if（非标识符）
```

**工程要点**：**保留字规则应放在标识符规则之前**（或与最长匹配配合），避免 `int` 被当成 ident 前缀。

## 数值字面量

用正则 **组合** 覆盖 C♭ 支持的进制：

| 进制 | 概念模式 |
|------|----------|
| **十进制** | 连续数字 |
| **十六进制** | `0` + `x`/`X` + 十六进制 digit |
| **八进制** | `0` + 八进制 digit 序列 |

匹配成功后 token 携带 **整型语义值**（JavaCC 可在动作里解析数值）。

## 「没有结构」的含义

单词 **无嵌套/无跨行配对** — 单段正则即可完整描述：

- 关键字、标识符、数字、单字符运算符
- **不含** `/* … */`、`" … "` 这类 **起止 delimiter 跨内容**

→ 有结构的留给 [§4](./04-structured-lexing-states.md)。
