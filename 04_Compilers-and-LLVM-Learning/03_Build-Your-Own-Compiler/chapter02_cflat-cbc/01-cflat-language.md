## C 的简化版

C♭ = 为 **教学编译器** 设计的 **C 子集** — **保留核心**，**删掉实现负担大或易混淆** 的特性。

| 保留 | 说明 |
|------|------|
| **指针** | C 内存模型核心 — 全书 codegen/链接仍按 C 语义 |
| 基本类型、控制流、函数 | 足够写系统级小程序 |

| 删减（相对 C） | 原因（直觉） |
|----------------|--------------|
| **预处理器** | `#include` / 宏增加前端复杂度 |
| **浮点数** | 简化 codegen 与类型系统 |
| **`enum`** | 可后用 `int` 常量替代 |
| **`const` / `volatile`** | 语义与优化边界更复杂 |

→ 编译器实现 **条理更清晰** — 非语言设计「更好」，而是 **更适合从零写 cbc**。

---

## `import` 替代 `#include`

无预处理器 → 不能 `#include <stdio.h>`。

```c
// C♭ 概念
import stdio;

int main() {
    printf("Hello\n");
    return 0;
}
```

| 对比 | C | C♭ |
|------|---|-----|
| 引入外部定义 | `#include` + 预处理器 | **`import`** 语句 |
| 解析时机 | 预处理期文本展开 | **编译器**解析符号/类型 |

**语义**：从 **外部程序库** 导入函数、变量、类型定义 — 链接阶段仍需要对应 **库/目标文件**（第4部分 ch19）。

---

## 与 Rust 读者

| C♭ | Rust |
|----|------|
| `import stdio;` | 更接近 `use` + 链接库，而非文本 include |
| 无宏 | 无 C 预处理器 — 类似 Rust 拒绝宏污染（但 Rust 有声明宏/过程宏另一路） |
