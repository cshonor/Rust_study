## 代码分析的三阶段

为降低复杂度，编译器把「理解源码」拆成 **三个独立阶段**：

```text
  源代码文本
      ↓
  ① 词法分析（扫描）   Scanner
      ↓  token 流
  ② 语法分析           Parser  →  语法树 / AST
      ↓
  ③ 语义分析           类型 · 作用域 · …
```

| 阶段 | 问的问题 |
|------|----------|
| **词法** | 下一串字符是哪个 **单词**？ |
| **语法** | 这些单词按什么 **结构** 组合？ |
| **语义** | 结构 **含义** 是否合法（类型等）？ |

→ [ch1 四阶段](../chapter01_start/02-four-compiler-stages.md) 中，①② 属 **第1部分**；③ 主要在 **第2部分** ch9～10。

## 词法分析（扫描）

**Scanner** 负责：

| 任务 | 说明 |
|------|------|
| **切分** | 源码 → 单词序列 |
| **剔除** | 空白、注释等 **无语法意义** 部分 |
| **分类** | 每个单词的 **种类**（整数、标识符、`if` …） |
| **语义值** | 与种类绑定的值（如 `"54"` → 种类 **整数**，值 **54**） |

### Token

**token** = **字面表示 + 种类 + 语义值** 的组合。

```text
源码:  54
token:  kind=INTEGER,  image="54",  value=54
```

→ [CI ch4 扫描](../../../01_Crafting-Interpreters/part02_jlox/chapter04_scanning/) 手写 token；本书 ch4 用 **JavaCC** 描述扫描规则。

## 语法分析 · AST 与节点

**Parser** 消费 **token 序列**，构建 **树状结构**。

| 概念 | 说明 |
|------|------|
| **抽象语法树 AST** | 表达程序 **结构** 的树 |
| **节点 node** | 树上一个语法单元（表达式、语句、声明…） |
| **省略** | 仅作格式的 **`;`、括号** 等常 **不** 进 AST — 只留核心结构 |

```text
// 概念：if (x) { y; }  →  IfNode(cond=x, body=…)
// 分号可能不出现在树里
```

**语法树 vs AST（本书 ch1 用语）**：实践中 AST 已含 **语义导向** 的节点；严格「纯语法树」在语义分析后会 **精炼** 为 AST — ch7～8 在 cbc 里落地。
