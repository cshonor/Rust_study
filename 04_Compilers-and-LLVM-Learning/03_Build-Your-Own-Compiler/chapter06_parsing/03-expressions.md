## 运算符优先级

表达式内部 **不对等** — **优先级低** 的运算符在语法树上 **更靠近根**。

```text
a + b * c
     +
    / \
   a   *
      / \
     b   c
```

**Parser 任务**：按优先级 **正确归约** — 不能 flat 成「expr 碰 token 列表」。

→ [CI ch7 表达式解析](../../../01_Crafting-Interpreters/part02_jlox/chapter07_evaluating-expressions/) 递归下降同类问题。

## 分层解析（优先级塔）

JavaCC 用 **多层非终端** — **上层调用下层**，嵌套顺序 = 优先级从 **低到高**：

```text
expr      ← 最低：赋值 =
  ↓
expr10    ← 条件 ?: （若 C♭ 支持）
  ↓
expr9 … expr1   ← 逻辑 → 关系 → 算术 → …
  ↓
term 层   ← ch6 §4 unary/postfix/primary
```

| 层 | 典型运算符（概念） |
|----|-------------------|
| **`expr`** | `=` 赋值 |
| **`expr10`** | `?:` 三元 |
| **`expr9`～`expr1`** | `\|\|`、`&&`、比较、`+`、`*`、移位… |
| 更深层 | 见 cbc 文法 — 原则：**越外层优先级越低** |

**技巧**：**不必** 显式写优先级表 — **产生式嵌套** 即 **优先级分割**（LL 经典写法）。

```text
expr():  expr10() ( "=" expr() )?     // 赋值最松，在顶层
expr1(): … 调用 term 层 …           // 紧运算符靠近叶子
```

## 与语句的区别

| | 语句 `stmt` | 表达式 `expr` |
|---|-------------|---------------|
| 结构 | 多分支 `\|`（13 种） | **优先级分层** + 递归 |
| 冲突 | dangling else | 左递归需改写为 `*`/`+` 循环（LL） |

C♭ 赋值、条件表达式若在子集内 — 规则在对应 `expr` 层展开。
