# Crafting Interpreters — 目录结构

> **约定**：与 [`01-ER/目录结构.md`](../../01-ER/目录结构.md) 相同——**一章 = 一个目录**；**一小节一个 `.md`**；`README.md` 只做导航 hub。

## 布局

```text
01_Crafting-Interpreters/
├── 本书目录.md
├── 目录结构.md
├── README.md
├── part02_jlox/
│   └── chapter04_scanning/
│       ├── README.md           # hub · 专项笔记索引表
│       ├── 00-overview.md      # 本章定位
│       ├── 01-the-interpreter-framework.md   # §4.1
│       ├── 02-lexemes-and-tokens.md        # §4.2
│       ├── …
└── scripts/
    ├── split-chapter-sections.py
    ├── refresh-chapter-readmes.py
    └── …
```

## 命名

| 路径 | 含义 |
|------|------|
| `README.md` | **hub**：状态、一句话、**小节索引表** |
| `00-overview.md` | **本章定位**、小节路线图 |
| `01-<slug>.md` … | **每个 § 或主题块** 独立一篇 |

## 与 01-ER 对照

| ER | Crafting Interpreters |
|----|------------------------|
| `Item-NN-slug/README.md` | `chapterNN_<slug>/README.md` |
| `01-core-concepts.md` 等 | `01-<slug>.md`（按 § 顺序编号） |

## 新增 / 修改笔记时

1. 一小节一文件：`chapterNN/02-<slug>.md`。
2. 更新 `README.md` 索引表（或运行 `scripts/refresh-chapter-readmes.py`）。
3. 节与节之间用文首 `← 本章目录 · 上一节 · 下一节` 串联。

## 脚本

| 脚本 | 作用 |
|------|------|
| `split-chapter-sections.py` | 整章 md → 按 `##` 拆成多文件 |
| `refresh-chapter-readmes.py` | 按磁盘文件重建 README 索引表 |
| `fix-section-slugs.py` | 修正 `-section.md` 等占位文件名 |
| `flatten-chapter-notes.py` | （已完成）去掉 `notes/` 子目录 |
