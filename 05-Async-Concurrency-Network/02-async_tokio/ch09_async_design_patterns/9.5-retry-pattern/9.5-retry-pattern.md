# 9.5 The Retry Pattern（重试模式）

## 核心

网络失败、503、数据库过载时，按策略重试；配合 **指数退避（Exponential backoff）** + 抖动，避免对故障方「请求轰炸」。

## 示例逻辑

```rust
for i in 0..max_retries {
    match perform_async_call().await {
        Ok(v) => return Ok(v),
        Err(e) => {
            sleep(backoff(i)).await; // 如 1 << i，生产应加上限与 jitter
        }
    }
}
```
