# 6.8 Summary（总结）

## 两大核心动作

1. **轮询主题**：原子变量 + 观察者 Future（§6.1–6.3）。
2. **异步事件分发**：事件总线 + 句柄 + Tokio 任务 + GC（§6.5–6.7）。

## 工程忠告

- 响应式 / 事件驱动会增加**移动部件**与运维复杂度。
- **切勿过度工程 (overengineer)**：先简单同步或原子方案；仅在出现**明确性能或解耦瓶颈**时再引入总线与多订阅者。
