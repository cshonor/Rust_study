# 7.1 Building a Runtime（构建运行时）

## 为何手动构建

跳出 Tokio 默认配置，用 **`runtime::Builder`** 精细控制任务如何在多线程上调度，满足低延迟、分优先级队列或资源上限等需求。

## 关键 Builder 参数

| 参数 | 作用 |
|------|------|
| `worker_threads(n)` | 工作线程数量，影响并行度 |
| `max_blocking_threads(n)` | 阻塞线程池上限，防止 `spawn_blocking` 拖垮进程 |
| `global_queue_interval` | 全局队列轮询间隔，影响窃取与公平性 |
| `thread_stack_size` | 线程栈大小，深递归或大 Future 时需调大 |
| `thread_name` / `on_thread_start` / `on_thread_stop` | 可观测性与生命周期钩子 |

## 双运行时模拟高/低优先级

书中用 **两个独立 `Runtime`**（或不同 Builder 配置）模拟前文自定义队列中的**高/低优先级**：高优任务跑在专用较小池，避免被大批量低优任务饿死。
