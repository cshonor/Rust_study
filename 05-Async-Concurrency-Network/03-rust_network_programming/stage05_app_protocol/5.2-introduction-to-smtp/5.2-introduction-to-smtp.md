## 一、本节在学什么

**SMTP（Simple Mail Transfer Protocol）** 是互联网**发信**的标准**文本协议**，跑在 **TCP** 上。本节要点：

- 理解 SMTP 命令/响应形态（`HELO`、`MAIL FROM`、`RCPT TO`、`DATA` 等）
- 用 Rust 生态 **`lettre`** 组装 MIME 邮件，配置 **TLS** 与凭据后 `send`
- 不必手写 SMTP 状态机；生产场景还需 SPF/DKIM、限流与反垃圾

本章 demo **只构建 `Message` 对象**，**不真实连接 SMTP**（避免硬编码密码、依赖外网）。

---

## 二、SMTP 协议要点

**一句话**：客户端与邮件服务器用**一行行文本命令**交互，在 `DATA` 之后提交邮件正文（含 MIME 头与 body）。

| 项目 | 说明 |
|------|------|
| 用途 | 互联网**投递 / 转发**邮件（本章侧重**发**） |
| 传输 | 通常 **TCP** |
| 常见端口 | **25**（MTA 间转发）、**587**（客户端提交 + STARTTLS）、**465**（SMTPS，TLS 从头建立） |
| 形态 | 文本命令 / 三位数响应码（`250`、`354` 等） |
| 收信 | 用户收信常用 **POP3 / IMAP**，不在本章 SMTP 发信主线内 |

典型会话（概念）：

```text
客户端 → 服务器: HELO client.example.com
服务器 → 客户端: 250 OK
客户端 → 服务器: MAIL FROM:<sender@example.com>
客户端 → 服务器: RCPT TO:<ops@example.com>
客户端 → 服务器: DATA
服务器 → 客户端: 354 End data with <CR><LF>.<CR><LF>
客户端 → 服务器: Subject: ...\r\n\r\n正文...\r\n.
服务器 → 客户端: 250 Message accepted
```

适合场景：**告警邮件**、定时报表、注册验证（生产常配合监控与模板系统）。

## 三、Rust：`lettre` 发信流程

不必手写上述状态机，用 **`lettre`**：

| 步骤 | 做法 |
|------|------|
| 1. 业务结构 | 定义如 **`CrashReport`**（崩溃栈、主机名等） |
| 2. 组装邮件 | `Message::builder()`：from / to / subject + `singlepart` 或 multipart |
| 3. 传输配置 | **`SmtpTransport::relay("smtp.example.com")?`** + 端口 + **TLS** |
| 4. 认证 | **`Credentials::new(user, pass)`**（从环境变量读取，勿写进仓库） |
| 5. 发送 | `.build().send(&email)?` |

```rust
// 概念流程（API 随 lettre 版本略有差异）
// SmtpTransport::relay("smtp.example.com")?
//     .port(587)
//     .credentials(Credentials::new(
//         std::env::var("SMTP_USER")?,
//         std::env::var("SMTP_PASS")?,
//     ))
//     .build()
//     .send(&email)?;
```

### 端口与 TLS 注意

| 配置 | 说明 |
|------|------|
| **587 + STARTTLS** | 云厂商与邮箱服务最常见 |
| **465 SMTPS** | 连接即 TLS；部分老服务仍用 |
| **25** | 常被 ISP/云屏蔽；服务器间转发用，桌面客户端少用 |

凭据、API Key 一律 **`std::env::var`** 或密钥管理，**不要 commit 进 Git**。

**与 Ch4**：邮件正文可以是纯文本 / HTML；附件涉及 **MIME** 编码（`multipart`），属应用层格式，不等同于 Serde JSON。

## 四、本章 Demo：组装 `Message`（dry-run）

仓库 demo 模拟 **CrashReport 告警邮件**：用 `lettre` builder 拼出一封纯文本信，打印 `Debug` 后退出，**不调用 `SmtpTransport::send`**。

```rust
use lettre::message::SinglePart;
use lettre::Message;

let email = Message::builder()
    .from("crash-report@example.com".parse()?)
    .to("ops@example.com".parse()?)
    .subject("5.2 SMTP demo — CrashReport")
    .singlepart(SinglePart::plain(String::from(
        "Server panic at line 42 (dry-run, not sent)",
    )))?;

println!("Built lettre Message (dry-run, not sent):");
println!("{email:?}");
println!("Next: SmtpTransport::relay(...) + credentials + send.");
```

### 要点

1. **`parse()?`**：`from` / `to` 需解析为 `Mailbox`；非法地址会报错。
2. **`singlepart(SinglePart::plain(...))`**：最简单单段纯文本；HTML 用 `SinglePart::html`，附件用 `MultiPart`。
3. **dry-run  intentional**：学完 builder API 后，在本地或 CI 外网环境再配真实 SMTP 做联调。
4. **`Cargo.toml`** 已为 `lettre` 启用 `builder` + `smtp-transport` feature，真实发送时无需改依赖名。

仓库对应：[5.2-smtp-lettre-build-demo.rs](./code/5.2-smtp-lettre-build-demo.rs)

## 五、从 demo 到真实发信

```text
demo（本章）          生产
─────────────────────────────────────
Message::builder      + SmtpTransport::relay
打印 Debug            + TLS + Credentials
不联网                + SPF/DKIM/DMARC（域配置）
                      + 速率限制、退信处理
```

联调 checklist：

- [ ] SMTP 主机、端口、TLS 模式与服务商文档一致
- [ ] 凭据来自环境变量
- [ ] 发件人域名已配 SPF/DKIM（否则易进垃圾箱）
- [ ] 失败时记录 `lettre` 返回的错误与响应码

## 六、新手易错

1. **混淆「发信 SMTP」与「收信 IMAP/POP3」**：本章只讲**往外发**。
2. **25 端口在本地跑不通**：换 **587** 或服务商指定端口。
3. **忘记 TLS**：明文登录会被拒或泄露密码；云邮箱几乎都要求 TLS。
4. **把密码写进源码或提交到 Git**：用 `.env`（且 `.gitignore`）或 CI secret。
5. **正文里的换行**：SMTP `DATA` 阶段对 `\r\n`、`.` 行结束符有规则；`lettre` 会处理，手写协议时才需背细节。
