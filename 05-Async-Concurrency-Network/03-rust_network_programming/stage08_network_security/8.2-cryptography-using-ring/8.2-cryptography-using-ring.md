## 8.2 Cryptography using ring（使用 ring 进行底层密码学操作）

## 一、本节在学什么

TLS 握手内部已含 **ECDH 密钥协商**；本章用 **`ring`** 单独演示 **X25519 Diffie-Hellman**，理解「共享密钥从哪来」。

**`ring`**：底层原语（`agreement`、哈希、AEAD）— **不是**完整 TLS 栈。生产 Web/API 用 **rustls** 即可。

---

## 二、DH / ECDH 要解决的问题

在不安全信道上，如何让双方得到**相同的对称密钥**，且窃听者无法从公开信息推导？

| 步骤 | 客户端 | 服务器 |
|------|--------|--------|
| 1 | 生成 ephemeral 私钥 `c_priv` | 生成 `s_priv` |
| 2 | 计算公钥 `c_pub` | 计算 `s_pub` |
| 3 | 交换公钥（明文可传） | 同左 |
| 4 | `shared = DH(c_priv, s_pub)` | `shared = DH(s_priv, c_pub)` |
| 5 | 两端 **shared 相同** | 用于 AEAD 加密 |

**前向保密**：每次握手 ephemeral 密钥，长期私钥泄露不危及历史会话（TLS 1.3 默认 ECDHE）。

---

## 三、`ring` X25519 示例

```rust
use ring::agreement::{self, EphemeralPrivateKey, UnparsedPublicKey, X25519};
use ring::rand::SystemRandom;

let rng = SystemRandom::new();
let client_priv = EphemeralPrivateKey::generate(&X25519, &rng)?;
let server_priv = EphemeralPrivateKey::generate(&X25519, &rng)?;
let client_pub = client_priv.compute_public_key()?;
let server_pub = server_priv.compute_public_key()?;

let client_shared = agreement::agree_ephemeral(
    client_priv,
    &UnparsedPublicKey::new(&X25519, server_pub.as_ref()),
    |key| key.to_vec(),
)?;

let server_shared = agreement::agree_ephemeral(
    server_priv,
    &UnparsedPublicKey::new(&X25519, client_pub.as_ref()),
    |key| key.to_vec(),
)?;

assert_eq!(client_shared, server_shared);
```

| API | 作用 |
|-----|------|
| `EphemeralPrivateKey::generate` | 临时密钥对 |
| `compute_public_key` | 公开部分 |
| `agree_ephemeral` | 双方得到相同 shared secret |

仓库对应：[8.2-cryptography-ring-x25519-demo.rs](./code/8.2-cryptography-ring-x25519-demo.rs)

## 四、ring vs TLS

```text
应用层自定义协议（少见）：
    ring X25519 + ChaCha20-Poly1305 — 需密码学专家审查

生产 Web/API：
    TLS（rustls）— 内部已含 ECDHE、证书校验、AEAD
```

**与 Ch5 gRPC**：常默认 TLS；不必手写 X25519。

---

## 五、新手易错

1. **重复发明 TLS**：自定义「加密协议」极易出错；优先 rustls。
2. **shared secret 当最终密钥**：通常还要 KDF 派生 AEAD 密钥与 nonce 策略。
3. **静态 DH**：无 ephemeral 则无前向保密；TLS 1.3 禁用静态 RSA 密钥传输。
4. **公钥不验证**：MITM 可替换公钥；TLS 用**证书**绑定身份，裸 DH 不行。
5. **打印 shared secret**：调试后勿泄露到日志。
