---

## 一、本节在学什么

公网 HTTPS 需要**受信任证书**。**Let's Encrypt** 提供免费短期证书；**ACME** 协议自动化签发与续期。

| 项目 | 说明 |
|------|------|
| **Let's Encrypt** | 非营利 CA，免费证书（常 **90 天**） |
| **ACME** | 自动化协议：HTTP-01 / DNS-01 等质询证明域名控制权 |
| **续期** | 到期前 re-issue（certbot、acme 客户端、cron） |

---

## 二、ACME 流程

```text
1. 申请证书（CSR / 账户密钥）
2. CA 下发质询（challenge）
   - HTTP-01：在 http://domain/.well-known/acme-challenge/TOKEN 放指定内容
   - DNS-01：在 DNS 添加指定 TXT 记录
3. CA 验证通过后签发
4. 得到 fullchain.pem + privkey.pem
5. 到期前自动 renew
```

| 质询 | 优点 | 注意 |
|------|------|------|
| **HTTP-01** | 简单，适合单机 Web | 须 80 端口可达 |
| **DNS-01** | 支持通配符 `*.example.com` | 需 DNS API 权限 |

---

## 三、Rust 与部署侧

| 方式 | 说明 |
|------|------|
| **`acme-client`** 等 crate | Rust 内嵌 ACME 客户端 |
| **certbot** | 命令行，最常用 |
| **云 LB / CDN** | 托管证书，应用无感 |
| **Rocket `tls` feature** | 配置 PEM 路径启动 HTTPS（书中示例） |

典型文件：

| 文件 | 内容 |
|------|------|
| `fullchain.pem` | 站点证书 + 中间链 |
| `privkey.pem` | 私钥（权限 600） |

```rust
// demo 打印流程要点，非真实 ACME 调用
println!("ACME (Let's Encrypt) flow:");
println!("  1. prove domain control (HTTP-01 or DNS-01)");
println!("  2. receive cert + chain (fullchain.pem)");
println!("  3. private key (privkey.pem)");
println!("  4. auto renew before expiry (~90 days)");
```

仓库对应：[8.1.1-letsencrypt-acme-overview-demo.rs](./code/8.1.1-letsencrypt-acme-overview-demo.rs)

## 四、与 rustls 客户端的关系

| 角色 | 证书来源 |
|------|----------|
| **服务端** | ACME 签发的 PEM，加载到 Rocket / axum + tokio-rustls |
| **客户端** | 系统或 `webpki-roots` 信任 LE 根；`reqwest` 默认校验 |

客户端 **不需要** 自己跑 ACME；只要信任链正确即可访问 `https://yourdomain`.

---

## 五、新手易错

1. **忘记续期**：90 天过期后浏览器报 ERR_CERT_DATE_INVALID。
2. **HTTP-01 被防火墙挡**：80 端口必须对 LE 验证服务器开放。
3. **staging vs production ACME**：测试用 staging 避免 rate limit；上线换 production endpoint。
4. **私钥泄露**：`privkey.pem` 等同服务器身份；勿提交 git。
5. **仅证书无完整链**：部分客户端需 `fullchain`，不要只配 leaf cert。
