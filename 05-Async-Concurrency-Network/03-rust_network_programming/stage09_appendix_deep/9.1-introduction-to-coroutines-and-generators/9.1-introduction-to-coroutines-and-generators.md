## 一、本节在学什么

[Ch7](../stage07_tokio_async_net/7.1-looking-into-the-future/7.1-looking-into-the-future.md) 的 **Future + 组合子** 是 Rust 异步主线；附录补充 **协程 / 生成器** 视角：

- 控制流可 **`yield`** 挂起，恢复时从断点继续
- 写法更接近同步代码，便于理解「谁在驱动谁」

不要求全部用于生产；与 **无栈 `async/await`** 对照学习。

## 二、无栈 vs 有栈

| 类型 | 状态保存 | 特点 |
|------|----------|------|
| **无栈（Stackless）** | 编译为状态机（`async fn`） | 轻量；Rust **`async/await` 即此类** |
| **有栈（Stackful）** | 独立协程栈 | 任意深度 `yield`；资源占用更高 |

```text
async/await（无栈）  ←  Ch7 主线、async_tokio/
may / 用户态线程（有栈）  ←  9.1.1、Go 风格
```

## 三、Nightly：`Generator` + `yield`

标准库实验特性（需 **nightly** + `#![feature(generators)]` 等）：

- **`Generator`** trait  
- **`yield`** 产出下一项  

书中用生成器实现 **Collatz 序列** — 与 `Iterator` 对比：

| | **Iterator** | **Generator** |
|---|--------------|---------------|
| 驱动 | 调用方 `next()` | 调用方 `resume` / `yield` 交出 |
| 状态 | 在迭代器结构体 | 在生成器帧 |
| 稳定 Rust | ✅ | ❌ nightly 实验 |

## 四、async task vs OS thread（概念）

```rust
async fn async_work() {
    println!("async: state machine on Tokio executor");
}

fn main() {
    thread::spawn(|| {
        thread::sleep(Duration::from_millis(50));
        println!("thread: OS-scheduled stack");
    });
    let rt = tokio::runtime::Runtime::new().unwrap();
    rt.block_on(async_work());
}
```

| | **OS thread** | **async task** |
|---|---------------|----------------|
| 调度 | 内核 | 用户态 executor |
| 栈 | ~MB 级 | 状态机，小 |
| 阻塞 I/O | 占线程 | `.await` 让出 |

仓库对应：[9.1-coroutines-async-vs-thread-demo.rs](./code/9.1-coroutines-async-vs-thread-demo.rs)

## 五、新手易错

1. **把附录当主线**：生产 Web 仍以 **Tokio + hyper/axum** 为主。
2. **混淆 Generator 与 async**：二者实现机制不同；`async` 是语言稳定特性。
3. **有栈 = 一定更快**：视场景；生态与维护度更重要。
4. **nightly Generator 上生产**：实验特性 API 会变。
