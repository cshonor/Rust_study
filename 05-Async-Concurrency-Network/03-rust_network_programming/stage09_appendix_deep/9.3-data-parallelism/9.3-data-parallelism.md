## 9.3 Data parallelism（数据并行）

> 书 §9.3 · I/O 异步：[Ch07](../stage07_tokio_async_net/) · 对照 [`atomic/rayon/`](../../01-atomic/rayon/) · 贯通稿：[notes/附录-…](../notes/附录-扩展与替代方案-学习笔记.md)

---

## 一、本节在学什么

**任务并行**（多连接、多 `async` task）≠ **数据并行**（同一数据集分块独立处理）。

| | 任务并行 | 数据并行 |
|---|----------|----------|
| 划分 | 不同**任务** | 同一数据**分块** |
| 典型 | Tokio、OS 线程 | **`rayon`** |
| 等待 | I/O | CPU 计算 |

网络服务：**I/O 用 Tokio**；解析大文件、批量校验、压缩等 **CPU 密集** 可用 **rayon** 或 `spawn_blocking`。

---

## 二、`rayon` 核心 API

| API | 用途 |
|-----|------|
| **`rayon::scope`** | 分叉子任务，作用域结束自动 join |
| **`par_iter()`** | 并行迭代器，替代 `iter()` |
| **`par_sort`** 等 | 并行排序、划分 |

```rust
use rayon::prelude::*;

let data: Vec<i32> = (0..10_000).collect();
let seq: i32 = data.iter().map(|x| x * x).sum();
let par: i32 = data.par_iter().map(|x| x * x).sum();
assert_eq!(seq, par);
```

书中示例还包括：并行 **二分查找**（`scope` 分治）、**criterion** 基准对比顺序 vs 并行。

仓库对应：[9.3-data-parallelism-pariter-demo.rs](./code/9.3-data-parallelism-pariter-demo.rs)

---

## 三、与 async 组合

```text
Tokio task 收到大 payload
    ↓
tokio::task::spawn_blocking(|| {
    data.par_iter().map(expensive).collect()
})
    ↓
.await 拿回结果，继续 async 写 socket
```

| 做法 | 适用 |
|------|------|
| 纯 `par_iter` 在 sync main | CLI、批处理 |
| **`spawn_blocking` + rayon** | async 服务内 CPU 密集 |
| 纯 async 里 `par_iter` | ❌ 阻塞 worker，应用 blocking 池 |

---

## 四、新手易错

1. **小数据用 rayon**：线程开销可能大于收益。
2. **在 Tokio worker 上直接 `par_iter`**：占满线程池，拖慢 I/O。
3. **数据竞争**：`par_iter` 要求闭包无共享可变；用 map/reduce 模式。
4. **与 Ch7 混淆**：rayon 不替代 `async` 网络 I/O。
