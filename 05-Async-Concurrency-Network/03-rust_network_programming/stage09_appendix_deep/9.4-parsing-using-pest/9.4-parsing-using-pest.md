## 一、本节在学什么

[Ch04](../stage04_serde_parse/) 用 **`nom`**（组合子 + 宏）；附录给出 **PEG 路线 `pest`**：

- 规则写在独立 **`.pest` 文法文件**
- **`pest_derive`** 生成解析器
- 书中称 **失败提示更友好**、文法集中易读

用 Pest **重写 HTTP 请求行**，与 Ch4 nom 示例对照风格差异。

---

## 二、nom vs pest

| | **nom** | **pest** |
|---|---------|----------|
| 规则写法 | Rust 宏 / 组合子函数 | 独立 **`.pest` 文件** |
| 生成 | 编译期展开宏 | **`pest_derive`** |
| 错误信息 | 依赖调试技巧 | 往往更直观 |
| 维护 | 复杂协议拆子解析器 | 文法集中 |
| 二进制 | `bits!` 等 | 更偏文本 PEG |

---

## 三、工作流

**1. 文法** `http_line.pest`：

```pest
method = { "GET" | "POST" }
path = @{ (ASCII_ALPHANUMERIC | "/" | "_" | "-")+ }
version = { "HTTP/" ~ ASCII_DIGIT+ ~ "." ~ ASCII_DIGIT+ }
request = { method ~ " " ~ path ~ " " ~ version }
```

**2. 派生解析器**：

```rust
use pest::Parser;
use pest_derive::Parser;

#[derive(Parser)]
#[grammar = "9.4-parsing-using-pest/http_line.pest"]
struct HttpLineParser;
```

**3. 遍历 `pair` 树取语义**：

```rust
let input = "GET /api/users HTTP/1.1";
let mut pairs = HttpLineParser::parse(Rule::request, input)?;
let request = pairs.next().unwrap();
let mut inner = request.into_inner();
let method = inner.next().unwrap().as_str();
let path = inner.next().unwrap().as_str();
let version = inner.next().unwrap().as_str();
println!("pest: {method} {path} {version}");
```

## 四、选型建议

| 场景 | 倾向 |
|------|------|
| 团队熟 PEG 文法 | **pest** |
| 纯 Rust 组合、嵌入式、二进制 | **nom** |
| JSON/MessagePack | **serde**（Ch4） |
| 完整 HTTP | **httparse** / hyper（Ch6） |

---

## 五、新手易错

1. **`grammar` 路径**：`#[grammar = "..."]` 相对 crate 根或模块，改目录要同步。
2. **静默规则 `@` vs `_`**：影响 AST 是否包含中间节点。
3. **PEG 贪婪与优先级**：文法顺序影响匹配，需读 pest 文档。
4. **只解析不验证**：请求行合法 ≠ 整个 HTTP 消息合法。
