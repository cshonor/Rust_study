## 一、标准库够用什么、缺什么

**`std::net`** 适合常见 **TCP/UDP 应用**（bind、connect、阻塞读写）。以下能力标准库**不覆盖**或很薄：

| 需求 | 缺口 |
|------|------|
| CIDR / 网段计算 | 无 `192.168.0.0/24` 一类类型 |
| 高并发非阻塞 I/O | 无事件循环 |
| 链路层 / 原始帧 | 无抓包、自构以太网帧 |
| 完整 DNS（MX、TXT…） | `connect` 只做名字→地址 |

## 二、选型速查表

| Crate | 弥补的能力 | 要点 |
|-------|------------|------|
| **`ipnetwork`** | **CIDR** | 网段大小、判断 IP 是否属于某前缀 |
| **`mio`** | **非阻塞 I/O + 事件循环** | `Poll` 监控多 fd；**Token** 区分事件源；**Tokio 的底层基石** |
| **`libpnet`** | **数据链路层 / 原始抓包** | 从网卡读以太网帧，自解析 IP/TCP 头 |
| **`trust-dns` / `hickory-dns`** | **完整 DNS** | 任意记录类型；同步 + 异步解析器 |

仓库对应：[3.4-related-crates-demo.rs](./code/3.4-related-crates-demo.rs)（打印选型表，无额外依赖）

## 三、`mio` 精读（通向 Tokio）

### 心智对照

```text
多线程阻塞：每连接一线程 → 简单，连接数大时线程开销高
mio / epoll：单线程（或少量线程）事件循环 → 多连接复用，为 async 铺路
```

| | **本章 `std::net`** | **`mio`** | **Tokio（Ch7）** |
|---|---------------------|-----------|------------------|
| 阻塞 | 默认阻塞 | 非阻塞 + `Poll` | 非阻塞 + executor |
| 并发模型 | `thread::spawn` | 事件驱动 | `async` task |
| 学习成本 | 低 | 中 | 中高 |

**学习路径**：**阻塞 `std::net`（本章）→ 理解 mio 在等什么 → [stage07](../stage07_tokio_async_net/) Tokio**。

## 四、`ipnetwork`（CIDR）

运维与防火墙规则里常见 **`10.0.0.0/8`**。`ipnetwork` 提供：

- 前缀长度、网络地址、广播地址
- **`contains(ip)`** 判断是否落在网段内

标准库只有单个 `IpAddr`，没有「网段」一等公民。

---

## 五、`libpnet`（链路层）

- 绕过常规 socket API，直接读**以太网帧**。
- 需管理员权限 / Linux **`CAP_NET_RAW`** 等。
- 用途：协议学习、安全分析、自定义协议栈实验。

## 六、DNS crate（hickory）

| 场景 | 用法 |
|------|------|
| 客户端连服务器 | `TcpStream::connect("host:port")` 够用 |
| 邮件 MX、服务发现 TXT | hickory / trust-dns |

## 七、为何不用散落裸 `libc`？

可直接调 POSIX **`socket` / `bind` / `listen`**，但：

- 易破坏 Rust **内存与类型安全**；
- 更推荐 **`nix`** 等 **纯 Rust 封装**，把 `unsafe` 集中在边界审查。

---

## 八、新手易错

1. **一上来就用 Tokio**：先写通阻塞 echo，再迁异步。
2. **用 `libpnet` 做普通 HTTP 客户端**：杀鸡用牛刀；`std` 或 `reqwest` 即可。
3. **忽略 mio 与 Tokio 关系**：Tokio 不是替代 TCP 协议，而是替代「一线程一连接」的阻塞模型。
