## 一、一句话回顾

用 Rust 生态三件套处理 HTTP：**Hyper**（底层异步服务底座）、**Rocket**（宏路由全栈 Web）、**reqwest**（高层客户端）；数据格式靠 **Serde**（Ch4），运行时从 Ch3 阻塞 **演进到 Tokio 异步**（Ch6/7）。

---

## 二、核心术语

| 术语 | 含义 |
|------|------|
| **HTTP/1.1** | 文本头 + 长度/chunked 体；keep-alive |
| **HTTP/2** | 多路复用；gRPC 默认传输（Ch5） |
| **Hyper** | HTTP 解析/生成 + Tokio 驱动连接 |
| **`SO_REUSEPORT`** | 多监听同端口，内核分摊新连接 |
| **Rocket 路由宏** | `#[get]` / `#[post]` 绑定 handler |
| **Data Guard** | 请求体/参数提取与校验（常配合 Serde） |
| **diesel + r2d2** | ORM + 连接池（书中博客 CRUD） |
| **reqwest** | 阻塞/异步 HTTP 客户端 |
| **`blocking::Client`** | 同步 reqwest，适合 CLI |

## 四、三库分工

| Crate | 角色 | 典型用途 |
|-------|------|----------|
| **Hyper** | HTTP 协议 + 异步 I/O 底座 | 自建服务器、网关、底层代理 |
| **Rocket** | 全栈 Web 框架 | 博客 API、模板页、快速 REST |
| **reqwest** | HTTP 客户端 | 调第三方 API、爬虫、微服务间 HTTP |

```text
写服务：Hyper（底层） 或 Rocket（高层）
写客户端：reqwest
数据格式：Serde JSON（Ch4）
运行时：Tokio（Ch3 阻塞 → Ch6/7 异步）
```

### 与 Ch3 阻塞模型对照

| | **Ch3 `std::net`** | **Ch6 Hyper/Rocket** |
|---|-------------------|----------------------|
| I/O | 阻塞 | 非阻塞 + async |
| 并发 | 每连接一线程 | reactor / 异步任务 |
| 协议 | 自己组帧或裸字节 | HTTP 库解析 |
| 学习价值 | Socket 与 TCP 语义 | 应用层 Web 编程 |

## 八、新手易错（全章）

1. **手写 HTTP 报文漏 `\r\n`** → 用库或对照 §6.1 demo 格式。  
2. **混淆「服务」与「客户端」crate** → 本章服务看 6.1/6.2，调 API 看 6.3。  
3. **Rocket 与章根 Cargo 混编** → 6.2 单独 manifest，命令带 `--manifest-path`。  
4. **阻塞 server 线程模型照搬 Ch3** → Hyper 是异步多连接，不是一线程一连接。  
5. **外网 reqwest demo 失败** → 先查网络，再查 TLS feature。
