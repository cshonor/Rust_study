## 5. `Send` / `Sync`

```rust
unsafe impl<T: Send + Sync> Send for CustomArc<T> {}
unsafe impl<T: Send + Sync> Sync for CustomArc<T> {}
```

- **`T: Send`**：值可在线程间移动（最后一个 drop 可能在任意线程）。  
- **`T: Sync`**：多线程可同时 `&T`（多个 `CustomArc` 并发 `Deref`）。

与标准库 `Arc` 的 auto trait 规则一致（教学实现仍显式写出便于对照）。

---

## 6. 与标准库差异（本书未全实现部分）

| 特性 | 标准 `Arc` | 本章 `CustomArc` |
|------|------------|------------------|
| **Weak** | 有，防循环引用 | 未实现 |
| **布局** | 可能 `Thin` / 计数与分配器优化 | 单块 `ArcInner` |
| **计数宽度** | 平台相关、饱和策略 | `AtomicUsize` |
| **生产使用** | 是 | **否**，仅学习 |

---

## 8. 小结

- **Arc = 原子计数 + 控制块 + 正确的释放序**  
- **clone → Relaxed 加一**；**最后一次 drop → Release 减一 + Acquire fence + dealloc**  
- 与 Mutex/自旋锁不同：Arc 解决**所有权共享**，不解决**数据竞争**（`T` 仍须 `Sync` 才能并发读）
