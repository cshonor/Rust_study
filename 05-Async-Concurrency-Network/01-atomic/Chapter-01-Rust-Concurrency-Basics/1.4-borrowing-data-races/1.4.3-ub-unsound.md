## 1.4.3 UB 与 Unsound

> 书 §1.4 · 索引：[1.4-borrowing-data-races.md](./1.4-borrowing-data-races.md) · 前：[1.4.2 数据竞争](./1.4.2-data-race-definition.md) · 后：[1.4.4 借用规则](./1.4.4-borrowing-rules.md)

---

### 1. UB（未定义行为）是什么

代码违反语言**内存模型 / 别名规则**时，编译器可假设「规则永不被打破」，从而：

- **指令重排**、删除「看似无效」的分支；
- 让 UB **之前**看起来正确的代码也产生荒谬结果（书中「**时光倒流**」式例子）。

**数据竞争 = UB 的一种**；后果不是普通逻辑错误，而是**整个程序语义不再被语言保证**。

### 2. Unsound（不健全）是什么

特指：对外宣称是 **`safe` 的 API**，但内部写法（或可被用户触发的路径）会 **UB**。

| 情况 | 性质 |
|------|------|
| `unsafe` 块里自己写错 → UB | 写法错误，责任在作者 |
| 包装成 **safe 接口**，调用者在 **safe 代码**里也能触发 UB | **Unsound** — 破坏 Rust 安全承诺 |

**联动前文**：[1.2.7 Leakpocalypse](../1.2-scoped-threads/1.2.7-leakpocalypse-pitfalls.md) — 旧 scoped 线程 + `mem::forget(guard)` 导致子线程借悬垂栈 → **安全 API 在 leak 下 Unsound**。

### 3. `unsafe` 不等于「随便写」

`unsafe` 只是**关闭编译器检查**；开发者仍须手动遵守内存 / 借用规则，否则 UB 或 Unsound。
