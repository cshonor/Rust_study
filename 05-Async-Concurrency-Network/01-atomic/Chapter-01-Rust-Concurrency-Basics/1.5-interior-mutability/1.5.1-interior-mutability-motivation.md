## 1.5.1 为什么需要内部可变性

> 书 §1.5 · 承接 [1.4.4 借用规则](../1.4-borrowing-data-races/1.4.4-borrowing-rules.md) · 索引：[1.5-interior-mutability.md](./1.5-interior-mutability.md) · 后：[1.5.2 类型一览](./1.5.2-interior-mutability-types.md)

---

普通规则：有 **`&T`（共享引用）** 就不能再有 **`&mut T`**。但有些 API 必须用 **`&self`**（不可变引用）却还要改内部状态——例如 `Counter::increment(&self)`。

**内部可变性（Interior Mutability）**：在类型**内部**提供「绕过表面不可变」的合法修改通道，由类型作者保证不 UB。

| 层面 | 手段 |
|------|------|
| 编译期（[1.4](../1.4-borrowing-data-races/1.4-borrowing-data-races.md)） | `&T` / `&mut T` 互斥 |
| 单线程运行时（本节） | `Cell` / `RefCell` |
| 多线程运行时（[1.7](../1.7-mutex-rwlock/1.7-mutex-rwlock.md)） | `Mutex` / `RwLock` / 原子 |
