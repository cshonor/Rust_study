#### 3.4 `T: 'static` 到底是什么？

**规则**：类型 **`T`** 满足 **`T: 'static`**，当且仅当：

> **`T` 内部不存在任何非 `'static` 生命周期的引用（借用）。**

这与「是不是引用类型」无关——**任意类型**都可以（或不可以）满足 **`T: 'static`**：

| 概念 | 含义 |
|------|------|
| **`&'static T`** | **静态引用**：带生命周期标注的**引用类型**（如字面量 `"hello"` → `&'static str`） |
| **`T: 'static`** | **类型约束**：该类型里**所有**借用的生命周期都须 ≥ `'static`；**纯自有、无内部 `&` 的类型自动满足** |

**一句话**：

- 带 **`&`** 的是「引用自带生命周期」；
- **`T: 'static`** 是对**任意类型**的筛选——**有没有引用都可以套这条规则**。

| | **`&'a T` 里的 `'a`** | **`T: 'a` / `T: 'static`** |
|--|----------------------|---------------------------|
| 修饰对象 | **引用**能活多久 | **类型**里所有借用至少要活多久 |
| 特例 | `'a = 'static` → 静态引用 | **`T: 'static`** → 内部借用须活到程序结束；**无借用的自有类型也成立** |

#### 3.5 三类典型：`String` / 借引用 / `Arc`

#### 情况 1：自有所有权、无内部引用 → 自动 `T: 'static`

`String`、`Vec<i32>`、`i32`、`Box<T>` 等**完全持有数据、内部不带 `&`** 的类型，均 **`T: 'static`**。

```rust
let s = String::from("abc");
thread::spawn(move || println!("{s}")); // move 后闭包只拥有 s，F: 'static ✅
```

#### 情况 2：捕获局部引用 `&'a T` → 违反 `F: 'static`

```rust
let s = String::from("abc");
let r = &s; // &'局部 String
thread::spawn(|| println!("{r}")); // 闭包内含非 'static 引用 → 编译失败 ❌
```

#### 情况 3：`Arc<T>` 共享所有权 → `Arc<T>: 'static`

`Arc` **不借**外层栈，只在堆上共享计数；`Arc<String>: 'static`：

```rust
use std::sync::Arc;
let s = Arc::new(String::from("abc"));
let s_clone = Arc::clone(&s);
thread::spawn(move || println!("{s_clone}")); // ✅
```

#### 3.6 极易混淆：`&'static T` vs `T: 'static`

**完全不是一回事**：

```rust
// 1. &'static — 引用类型，指向只读静态段
let static_str: &'static str = "hello";

// 2. String: 'static — 类型约束：String 内部无短命引用；与「静态字符串」无关
let owned_str: String = String::from("hello");
```

| 写法 | 是什么 |
|------|--------|
| **`&'static str`** | **引用**，生命周期 = 整个程序 |
| **`String: 'static`** | **约束**：`String` 这种类型可以不含短命借用；普通 `String::from(...)` 变量仍满足 |
