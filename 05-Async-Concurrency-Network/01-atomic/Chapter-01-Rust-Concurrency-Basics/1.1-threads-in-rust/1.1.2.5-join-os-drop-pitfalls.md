## 五、两个易混点

1. **单词是 Join，不是 Enjoy**：口语 / 听写常把 **`Join`**（等待线程「汇合」）误听成 Enjoy。Rust 里对应 **`JoinHandle`**（线程句柄）与 **`.join()`**（阻塞等待子线程）；scope 内则是 **`ScopedJoinHandle`**，同样有 **`.join()`**，另见 [1.2.3 scope 自动 join](../1.2-scoped-threads/1.2.3-join-auto-join.md)。
2. **`JoinHandle` 不只有 `join()`**：还有 **`.thread()`**；**同步等待的核心 API 仍是 `join`**。
3. **句柄 ≠ 线程本体**：用来 **等待 + 取返回值 + 回收 OS 资源**。
4. **把 `JoinHandle<T>` 的 `T` 当成 `Result` 的业务泛型**：`T` 是闭包正常返回值；**`.join()` 外层 `Result`** 只区分「正常结束 / panic」（见 [1.1.2.2 §T vs Result](./1.1.2.2-spawn-join-types.md#joinhandlet-的-t-vs-join-的-result两个独立层次)）。

## 六、底层原理：OS → 标准库 → `join()`

#### （1）OS：`pthread_join`

| 概念 | 说明 |
|------|------|
| 未 join 的已结束线程（Unix） | 可能暂存为 **僵尸线程**，直到 `join` 回收 |
| **`pthread_join` / `WaitForSingleObject`** | 调用线程**内核阻塞**，直到目标线程结束 |

**OS 一句话**：`join` = 内核「等另一线程结束」原语；Rust 在其上封装 panic 传播与 `T` 回传。scope 自动 join 最终也走此路径（[1.2.3 §3](../1.2-scoped-threads/1.2.3-join-auto-join.md#3-分层拆解自动-join-做了什么是不是封装系统调用)）。

#### （2）教学心智模型 vs 标准库实装

`Arc` + `Condvar` 伪代码仅帮助理解阻塞/唤醒；**Unix 上 `join()` 通常直接 `pthread_join`**，阻塞在**内核**。读源码：`library/std/src/thread/`。

#### （3）`join(self)` 语义

1. 阻塞直到子线程结束；2. 取 `Result<T, _>`；3. **消耗 handle**，只能一次。

---

## 七、为什么对外几乎只暴露 `join`？

| 原因 | 说明 |
|------|------|
| **只能 join 一次** | `join(self)` 消耗 handle；重复 join 在 OS 上是 UB |
| **职责单一** | `spawn` 启动；`JoinHandle` 等待 + 回收 |
| **不提供 kill** | 强杀破坏内存安全 / 锁 |

---

## 八、drop 而不 `join`

> **Drop 机制与 detach**：[1.1.7 §主动放掉 handle 与 Drop](./1.1.7-thread-lifecycle-four-states.md#主动放掉-handle与-drop谁主动谁被动) · [15.3 Drop](../../../../00-Book/15-smart-pointers/15.3-使用Drop运行清理代码.md)

| 行为 | 后果 |
|------|------|
| **`drop(handle)` / handle 出作用域、且未 `join`** | 触发 **`impl Drop for JoinHandle`** → 线程 **detach**（分离态） |
| **`.join(self)`** | **消耗** handle，走 join 路径，**不**走 detach |
| **进程退出** | OS 强杀未完成线程 |

```rust
let h = thread::spawn(|| { /* … */ });
drop(h); // 等价于让 h 立刻出作用域 → Drop → detach

{
    let h = thread::spawn(|| { /* … */ });
} // 块结束同样 Drop(h)
```

**不能**写 `h.drop()`；分离逻辑在标准库 `JoinHandle` 的 **`Drop` 实现**里，你只控制 handle **何时**离开作用域。

## 九、极简总结

1. **`JoinHandle<T>` 的 `T`** = 子线程闭包正常返回的类型；**`.join()`** → `Result<T, panic载荷>`；panic 捕获见 [1.1.2.4 §2.1](./1.1.2.4-panic-capture-downcast.md#21-panic-捕获与-downcast_ref)，**`&&str` 层级**见 [1.1.2.4 §2.2](./1.1.2.4-panic-capture-downcast.md#22-downcast_refstrturbofish-与类型层级精读)。
2. 内核 `pthread_join`；句柄非线程本体；`join()` 阻塞 + 消耗 handle。
3. Condvar 模型仅助理解；scope 自动 join 同路径见 [1.2.3 §3](../1.2-scoped-threads/1.2.3-join-auto-join.md#3-分层拆解自动-join-做了什么是不是封装系统调用)。
