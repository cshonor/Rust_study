## 1.2.5 参数 `s` 与 `Scope`

> 书 §1.2 · 索引：[1.2](./1.2-scoped-threads.md) · 前：[1.2.4 句柄与约束](./1.2.4-scoped-join-handle.md) · 后：[1.2.6 变量与结果](./1.2.6-variables-results.md)

---

闭包里的 **`s`** 不是关键字，也不是全局变量——只是你给参数起的名字；本质是标准库传入的 **`&Scope<'scope, 'env>`** 引用。

### 1. `thread::scope` 签名（简化）

```rust
pub fn scope<'env, F, T>(f: F) -> T
where
    F: for<'scope> FnOnce(&'scope Scope<'scope, 'env>) -> T,
```

| 部分 | 含义 |
|------|------|
| **`thread::scope`** | 标准库函数；你传入一个闭包 `f` |
| **`for<'scope> …`** | 高阶生命周期：闭包接收的 `Scope` 引用带 **`'scope`** |
| **`&'scope Scope<…>`** | 就是闭包参数 **`s`** 的类型（写成 `\|s\|` 只是习惯命名） |
| **返回值 `T`** | scope 闭包最后一行的表达式，可带回 scope 外 |

### 2. 执行流程（通俗版）

```rust
thread::scope(|s| {
    // s = &Scope，由 std 在调用闭包前创建并传入
    s.spawn(|| { /* … */ });
});
```

1. 调用 `thread::scope(你的闭包)`；
2. std **内部新建** `Scope` 结构体，维护尚未 join 的 `ScopedJoinHandle` 列表；
3. std 把 **`&Scope`** 传给你的闭包，绑定到变量 `s`（名字可改）；
4. 每次 **`s.spawn(...)`** 创建 OS 线程，登记 handle，返回 **`ScopedJoinHandle<'scope, T>`**；
5. 闭包执行完毕 → **`Scope` 析构** → 对未 join 的 handle **批量 `.join()`**（见 [1.2.3 §3](./1.2.3-join-auto-join.md#3-分层拆解自动-join-做了什么是不是封装系统调用)）；
6. `thread::scope` 返回闭包的结果 `T`。

### 3. `Scope` 与 `s.spawn`（伪代码）

> **`impl<'env> Scope<'env>` 从哪来？** 见 [1.2.4 §2.0 符号拆解](./1.2.4-scoped-join-handle.md#20-符号先拆清implenv-scopeenv)（教材简化 vs 标准库双生命周期）。

```rust
struct Scope<'scope, 'env: 'scope> {
    // 登记 spawn 出的 handle，供收尾 batch join
    handles: /* 内部容器 */,
}

impl<'scope, 'env: 'scope> Scope<'scope, 'env> {
    pub fn spawn<F, T>(&self, f: F) -> ScopedJoinHandle<'scope, T>
    where
        F: FnOnce() -> T + Send + 'scope,
        T: Send + 'scope,
    {
        // 创建 OS 线程，包装成 ScopedJoinHandle，登记到 handles
    }
}
```

- **`s` 的类型**：`&Scope<'scope, 'env>`（闭包里常写成 `&Scope<'_>`）；
- **改名合法**：`thread::scope(|scope_obj| { scope_obj.spawn(|| {}); })` 与 `|s|` 完全等价；
- **外部拿不到 `Scope`**：实例只在 `thread::scope` 内部临时存在，生命周期 = 闭包执行期间，无法 return 到 scope 外。

### 4. 极简对照

| | `thread::spawn()` | `s.spawn()` |
|---|-------------------|-------------|
| 返回 | `JoinHandle<T>` | `ScopedJoinHandle<'scope, T>` |
| 捕获 | 强制 `move` / `'static` | 可借 scope 外栈变量 |
| 等待 | 手动 join | scope 收尾自动 join + 可选手动 join |
| **`s` 从哪来** | — | `thread::scope` 内部构造 `Scope` 并把 **`&Scope`** 传入闭包 |
