## 1.2.6 变量规则、结果带出与总对照

> 书 §1.2 · 索引：[1.2](./1.2-scoped-threads.md) · 前：[1.2.5 参数 s](./1.2.5-scope-param-s.md) · 后：[1.2.7 Leakpocalypse](./1.2.7-leakpocalypse-pitfalls.md)

---

| 变量位置 | 在 `s.spawn` 里怎么用 | scope 外还能用吗 |
|----------|----------------------|------------------|
| scope **上面**（闭包外、调用前定义） | 直接借用，**不必 move** | ✅ scope 结束后仍可用（若没被 move 进某线程） |
| scope **内部**（闭包 `{ }` 里定义） | 通常 **`move`** 进子线程 | ❌ 不能把 `inside_scope` 当普通变量拿到外面 |
| scope **返回值** | 闭包最后一行表达式 | ✅ `let x = thread::scope(\|s\| { … })` |

```rust
use std::thread;

fn main() {
    let before = 10;

    let result = thread::scope(|s| {
        let inside = 20;

        s.spawn(|| println!("借 before: {before}"));

        s.spawn(move || println!("move inside: {inside}"));

        before + inside // scope 闭包返回值
    });

    println!("{result:?}");
    // println!("{inside}"); // 编译错误：inside 作用域仅在 scope 闭包内
}
```

子线程若在 scope **内部**创建，其栈上局部变量**不能**直接在外面访问。常用三种出路：

### 1. scope 闭包返回值（最简单）

```rust
let sum = thread::scope(|s| {
    let nums = vec![1, 2, 3, 4, 5];
    let total: i32 = nums.iter().sum();
    s.spawn(move || println!("子线程: sum = {total}"));
    total // 返回给 scope 外
});
```

### 2. 在 scope 外创建 `Arc<Mutex<T>>`（多线程汇总）

```rust
use std::sync::{Arc, Mutex};
use std::thread;

let acc = Arc::new(Mutex::new(0));

thread::scope(|s| {
    for i in 1..=5 {
        let acc = Arc::clone(&acc);
        s.spawn(move || {
            *acc.lock().unwrap() += i;
        });
    }
});

println!("{}", *acc.lock().unwrap());
```

> `Arc` 本身 `'static` 友好；scope 只解决「**借用栈上数据**」的问题，**长期跨线程共享可变状态**仍靠 [1.3](../1.3-shared-ownership/1.3-shared-ownership.md) / [1.7 Mutex](../1.7-mutex-rwlock/1.7-mutex-rwlock.md)。

### 3. 数据本来就在 scope 外，子线程只负责写

把要修改的结构定义在 `thread::scope` **之前**，子线程里通过 `Mutex` / 原子类型更新；scope 结束后读最终值（同 Demo 示例 4）。

> 两个 spawn 带类型标注见 [1.2.2 §0](./1.2.2-two-spawns.md#0-两个-spawn-完整区分带类型标注)；`s` 来源见 [1.2.5 参数 s](./1.2.5-scope-param-s.md)。

| | **`thread::spawn`** | **`thread::scope`** |
|---|---------------------|---------------------|
| 线程寿命 | 与进程同级，可超过创建者栈帧 | **限于 scope 块**，结束必 join |
| 借用局部变量 | ❌ 须 `'static` / `move` | ✅ 可借 `'scope` 内数据 |
| 等待方式 | 手动 `.join()` 每个 handle | scope 结束**自动** join 全部 |
| handle 逃逸 | 可以（也易漏 join） | **不可**把 `ScopedJoinHandle` 带出 scope |
| 典型场景 | 后台任务、与 caller 生命周期解耦 | 短生命周期并行、map-reduce 式批处理 |
| 稳定版本 | 1.0 起 | **1.63** 起 |

```rust
use std::thread;

let vec = vec![1, 2, 3];

// spawn：必须 move，vec 所有权进子线程
let h = thread::spawn(move || println!("{:?}", vec));
h.join().unwrap();
// vec 此处已不可用

let vec2 = vec![4, 5, 6];
thread::scope(|s| {
    s.spawn(|| println!("{:?}", vec2)); // 借用即可
});
// vec2 仍可用
```
