**两条主线**：

1. **9.1 → 9.2**：Mutex / Condvar — **三态减 syscall**、混合自旋、notify 规范；
2. **9.3**：RwLock — 避免写者忙等 + **写优先** 状态机（奇偶 `state` 编码）。

**一句话**：**有锁手写 = 状态机 + 只在「真有人等」时才 syscall；RwLock 用奇数 state 拦新读者防写者饥饿。**

## 依赖说明

书中示例依赖 **`atomic_wait`** crate（封装 futex / `WaitOnAddress`）。本仓库 **未默认加入 `Cargo.toml`**，笔记以**阅读与状态机**为主；对照 [第 8 章](../Chapter-08-OS-Primitives/8.3-condition-variables/8.3.3-rust-os-mapping.md) 理解底层。

日常开发：**优先 `std::sync::Mutex` / `Condvar` / `RwLock`**。
