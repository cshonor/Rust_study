## 一句话

青木峰郎带读者 **从零实现 cbc** — 编译 **C♭**（含指针的 C 子集）为 **x86 ELF**；**Java + JavaCC** 写前端，手写 **x86 汇编生成**，并讲清 **汇编 · 链接 · 加载** 全链路 — **动手吃透** C 与计算机底层。

---

## 本书解决什么问题

| 痛点 | 本书做法 |
|------|----------|
| 编译书只讲前端、做不出二进制 | 一路做到 **可执行 ELF** |
| 不懂 C 程序如何跑起来 | 链接器、加载器、栈帧、PIC |
| 优化/后端像黑盒 | 手写 codegen + 基础优化（ch17） |
| 理论太多、代码太少 | **跟着写 cbc** |

---

## 技术栈

| 组件 | 选型 |
|------|------|
| **源语言** | C♭（C 子集） |
| **实现语言** | Java |
| **Parser 生成** | JavaCC |
| **目标** | x86（IA-32）· GNU as |
| **输出** | ELF 目标文件 → 链接 → 加载 |

## 适合读者（与作者意图）

- 想搞清 **C 语言运行机制** 与 **x86/Linux 执行环境**
- 有志 **设计新语言** 并需要可落地 pipeline
- 已完成 **CI 在线版**，要「系统做一遍能跑的二进制」
- 接受 **Java** 作实现载体（概念可迁 Rust）

## 阅读建议

1. **先** [01 CI Part II](../01_Crafting-Interpreters/part02_jlox/) 扫描/解析（降低 ch3～6 坡度）。
2. **精读** 第2～4部时对照 `gcc -S` / `readelf` / `objdump`。
3. **并行** RFR 第 1 章内存、第 2 章 layout（ch14～16 栈帧）。
4. **不必** 复刻全部 Java 代码即可收获得 — 关键路径：IR → 汇编 → ELF → 链接。
