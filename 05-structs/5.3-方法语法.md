# 5.3 方法语法

**方法（method）** 与函数类似，但它们在某个类型（结构体/枚举/trait 对象）的上下文中定义，并且**第一个参数总是 `self`**（代表调用该方法的实例）。

---

## 在 `impl` 块中定义方法

把针对 `Rectangle` 的 `area` 函数改为方法：

```rust
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}
```

- `impl Rectangle { ... }`：与 `Rectangle` 类型相关联的实现块
- `&self` 是 `self: &Self` 的简写（在 `impl Rectangle` 中，`Self` = `Rectangle`）
- 方法可以选择：
  - `&self`：不可变借用（只读）
  - `&mut self`：可变借用（可修改实例）
  - `self`：获取所有权（较少见，常用于“消费自身并转换”）

---

## 方法名与字段同名：getter

```rust
impl Rectangle {
    fn width(&self) -> bool {
        self.width > 0
    }
}

// rect.width() 调方法
// rect.width  访问字段
```

Rust 不会自动为字段生成 getter。手写 getter 常用于：字段私有、方法公开，作为公共 API 的只读入口。

---

## 自动引用与解引用（为什么没有 `->`）

Rust 没有 C/C++ 的 `->`。调用方法时，Rust 会自动为接收者插入 `&` / `&mut` / `*`，以匹配方法签名。

下面等价：

```rust
p1.distance(&p2);
(&p1).distance(&p2);
```

---

## 带参数的方法：`can_hold`

```rust
impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}
```

调用：

```rust
rect1.can_hold(&rect2)
```

---

## 关联函数（associated function）

在 `impl` 中定义但**不以 `self` 为第一参数**的函数，称为关联函数（不是方法）。

典型用途：构造函数风格的创建器，比如正方形：

```rust
impl Rectangle {
    fn square(size: u32) -> Rectangle {
        Rectangle { width: size, height: size }
    }
}

let sq = Rectangle::square(3);
```

通过 `类型名::函数名` 调用（例如 `String::from`）。

---

## 多个 `impl` 块

同一个类型可以拆成多个 `impl` 块（语法允许；是否拆分取决于组织习惯与场景）：

```rust
impl Rectangle {
    fn area(&self) -> u32 { self.width * self.height }
}

impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool { ... }
}
```
