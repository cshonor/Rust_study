# 1.3 Hello, Cargo!

**Cargo** 是 Rust 的**构建系统**和**包管理器**，负责：
- 构建代码
- 下载依赖库（dependency）
- 编译依赖

> 官方安装包自带 Cargo。检查安装：`$ cargo --version`

---

## 使用 Cargo 创建项目

```bash
$ cargo new hello_cargo
$ cd hello_cargo
```

### 生成的项目结构

```
hello_cargo/
├── Cargo.toml    # 配置文件
├── src/
│   └── main.rs   # 源码
└── .gitignore    # Git 忽略文件（自动初始化 Git 仓库）
```

> 在已有 Git 仓库中运行 `cargo new` 不会生成 Git 相关文件。可加 `--vcs=git` 强制生成。

---

## Cargo.toml

```toml
[package]
name = "hello_cargo"
version = "0.1.0"
edition = "2021"

[dependencies]
```

| 部分 | 说明 |
|------|------|
| `[package]` | 包配置 |
| `name` | 项目名称 |
| `version` | 版本号 |
| `edition` | Rust 大版本（2015/2018/2021） |
| `[dependencies]` | 依赖的 crate 列表 |

- 使用 **TOML** 格式
- 代码包在 Rust 中称为 **crate**

---

## 项目结构约定

| 位置 | 用途 |
|------|------|
| `src/` | 源代码 |
| 根目录 | README、配置、许可协议等 |

Cargo 要求源码放在 `src/` 中。

## Cargo 命令

### cargo build — 构建

```bash
$ cargo build
```

- 生成可执行文件：`target/debug/hello_cargo`（Windows: `hello_cargo.exe`）
- 首次运行会生成 `Cargo.lock` 记录依赖版本

### cargo run — 构建并运行

```bash
$ cargo run
```

- 一步完成编译和运行
- 若源码未改动，直接运行已编译的可执行文件

### cargo check — 快速检查

```bash
$ cargo check
```

- 只检查能否编译，**不生成**可执行文件
- 比 `cargo build` 更快
- 适合开发时频繁检查

---

## 命令对比

| 命令 | 作用 |
|------|------|
| `cargo build` | 构建项目 |
| `cargo run` | 构建并运行 |
| `cargo check` | 只检查编译，不生成二进制 |

---

## 发布构建

```bash
$ cargo build --release
```

- 输出目录：`target/release/`（而非 `target/debug/`）
- 启用优化，运行更快
- 编译时间更长
- **基准测试**时请使用 release 版本

| 配置 | 用途 |
|------|------|
| `target/debug/` | 开发，快速编译 |
| `target/release/` | 发布，优化性能 |

---

## 现有项目转 Cargo

1. 将源码移到 `src/` 目录
2. 在根目录创建 `Cargo.toml`

---

## 克隆并构建已有项目

```bash
$ git clone example.org/someproject
$ cd someproject
$ cargo build
```

---

## 小结

- `rustup` 安装 Rust
- `rustc` 直接编译简单程序
- **Cargo** 管理项目、依赖、构建
- 跨平台命令统一，无需区分系统
