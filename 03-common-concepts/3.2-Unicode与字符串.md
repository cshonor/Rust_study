# Rust 与 Unicode 核心要点

先分清 **「编号体系」** 和 **「存盘格式」**，再看 Rust 里的 `char` / `String`。

对应 demo：`03-common-concepts/3.2-unicode-demo/`  
更深入字符串 API 见：`08-collections/8.2-string.md`

---

## 零、Unicode 与编码（概念层）

### 1. Unicode：只定编号，不管二进制

给全球字符分配**唯一数字编号（码点 / code point）**，规定「这个字是几号」，**不规定**在磁盘里用几个字节、字节长什么样。

例：`中` → **U+4E2D**（十进制 20013）。

### 2. UTF-8、GB2312、GBK：编码规则

把「编号」翻译成**一串二进制字节**写进文件/内存的规则，都叫**编码**：

| 名称 | 说明 |
|------|------|
| **UTF-8** | Unicode 码点的**主流**编码；变长 1~4 字节/标量 |
| **GB2312 / GBK** | 国内旧编码；**另一套独立编号/映射**，不是 Unicode 全集 |

**同一个字，编码不同 → 字节不同**：

- `中` 在 **UTF-8** 里常见为 **3 字节**（如 `E4 B8 AD`）
- 在 **GB2312** 里往往是 **另外 2 字节**（与 UTF-8 无对应关系）

### 3. Rust 里怎么对应

| Rust 类型 | 存的是什么 |
|-----------|------------|
| **`char`** | 一个 **Unicode 标量值**（码点），内存上约 **4 字节 `u32`** |
| **`String` / `&str`** | 强制按 **UTF-8** 把文本编成**字节序列** |

`char` ↔ UTF-8 字符串之间的转换由标准库完成（如 `chars()`、`encode_utf8`）；在**类型正确、来源确实是 UTF-8** 的前提下，不会出现「Rust 自己把编号和字节搞混」的乱码。

### 4. 乱码从哪来

乱码 = **字节的真实编码** 与 **程序按什么规则解码** 不一致。

例：文件实际是 **GB2312** 字节，程序却当 **UTF-8** 去 `String::from_utf8` / 读入 `str` → 规则对不上，显示异常（或 `from_utf8` 直接报错）。

读外部文件、网络、旧系统接口时，要先确认或转换编码；Rust 默认世界假设是 **UTF-8**。

## 一、`char`（Unicode 标量值）

1. **本质**：单个 **Unicode 标量值**，内存上为 **4 字节（`u32`）**
2. **语法**：单引号 `'…'`，且只能一个标量
   ```rust
   let ch1 = 'a';
   let ch2 = '中';
   let ch3 = '😀';
   ```
3. **范围**：合法标量大致为 `U+0000..=U+D7FF` 与 `U+E000..=U+10FFFF`（由 `char::from_u32` 等 API 校验）
4. **常用**
   ```rust
   println!("{}", '中' as u32);              // 码点 20013
   if let Some(c) = char::from_u32(20013) {
       println!("{}", c);                    // 中
   }
   let c = '\u{4E2D}';                       // 转义写法
   println!("{}", c.is_ascii_digit());       // false
   println!("{}", c.is_alphabetic());        // true（Unicode 字母类）
   ```

> **注意**：人眼的一个「字」有时是多个标量（组合 emoji、部分印地语等）；`chars()` 按**标量**切，不是按字形簇 (grapheme)。

## 二、`String` / `&str`（UTF-8）

- 字符串在 Rust 里是 **UTF-8 字节序列**，不是 `char` 数组。
- 每个标量在 UTF-8 中占 **1~4 字节**（ASCII 1、中文通常 3、部分 emoji 4）。
- **`s[i]` 取的是第 i 个字节**，不是第 i 个字符；从多字节字符中间切会 **panic**。

### 字节 vs 字符

```rust
let s = "中文";
println!("字节数: {}", s.len());           // 6，不是 2
for b in s.bytes() { /* 每个 u8 */ }
for c in s.chars() { /* 每个 char */ }     // 正确按标量遍历
```

### 按「第 n 个字符」访问

不能 `s[n]`，要先收集或迭代：

```rust
let s = "Rust编程";
let third = s.chars().nth(2).unwrap();     // 第 3 个标量
let chars: Vec<char> = s.chars().collect();
```

## 三、对比表

| 类型 | 含义 | 单位 | 常用访问 |
|------|------|------|----------|
| `char` | Unicode 标量 | 固定 4 字节 | `'中'`，`as u32` |
| `&str` / `String` | UTF-8 文本 | 1~4 字节/标量 | `.chars()`、`.len()` 为**字节**数 |
| `bytes()` | 原始 UTF-8 字节 | `u8` | `.bytes()` |

## 四、高频操作

**字符个数（标量个数）**

```rust
let s = "Hello 世界 😂";
let count = s.chars().count();   // 不是 s.len()
```

**Unicode 转义**

```rust
let c = '\u{4E2D}'; // 中
```

## 五、实用模板（切割 / 截取 / 翻转）

见 demo 中函数：`char_at`、`take_chars`、`skip_chars`、`reverse_chars`。  
按**字节**切安全子串（必须在字符边界上）用 `get(start..end)` 或 `&s[start..end]`，边界可用：

```rust
fn is_char_boundary(s: &str, index: usize) -> bool {
    if index == 0 || index == s.len() {
        return true;
    }
    s.is_char_boundary(index)
}
```

更完整的「按字形簇」处理可用 crate **`unicode-segmentation`**（本 demo 不引入依赖，仅用 `chars()`）。

---

## 六、常见踩坑

1. **`str[i]` 当字符** → 实际是字节；多字节字符会 panic 或逻辑错误。
2. **`len()` 当字符数** → `len()` 是 UTF-8 **字节**长度。
3. **`"a"` 当 `char`** → 字符串用双引号，标量用单引号。
4. **切片范围未对齐字符边界** → `&s[0..2]` 对 `"中"` 会 panic（`"中"` 占 3 字节）。
