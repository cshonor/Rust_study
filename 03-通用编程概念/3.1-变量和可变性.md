# 3.1 变量和可变性

Rust 中变量**默认不可变**（immutable），可充分利用安全性和简单并发。需要时可声明为可变（mutable）。

---

## 不可变变量

一旦值绑定到变量名，就**不能更改**。

```rust
fn main() {
    let x = 5;
    println!("The value of x is: {}", x);
    x = 6;  // ❌ 编译错误！
    println!("The value of x is: {}", x);
}
```

**错误**：`cannot assign twice to immutable variable 'x'`

- 编译器提示：`consider making this binding mutable: mut x`
- 不可变性有助于避免 bug：某处假设值不变，另一处却改了它
- Rust 编译器在编译期就保证「声明不变的值真的不变」

---

## 可变变量 mut

在变量名前加 `mut` 使其可变：

```rust
fn main() {
    let mut x = 5;
    println!("The value of x is: {}", x);
    x = 6;  // ✓ 允许
    println!("The value of x is: {}", x);
}
```

- 输出：`5` 然后 `6`
- `mut` 也提示读者：该变量的值会在别处被修改
- 大型数据结构时，原地修改可能比复制更快；小结构时函数式风格通常更清晰

---

## 常量 const

常量与不可变变量的区别：

| 特性 | 常量 const | 不可变变量 let |
|------|-----------|---------------|
| 可变性 | 永远不可变，不能用 mut | 默认不可变 |
| 声明 | `const` 关键字 | `let` 关键字 |
| 类型 | **必须**显式标注 | 可推导 |
| 赋值 | 仅常量表达式（编译期可算） | 可为运行时计算结果 |
| 作用域 | 可在全局等任意作用域 | 一般局部 |
| 命名 | 全大写 + 下划线 | snake_case |

```rust
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3;
```

- 常量名：全大写，如 `THREE_HOURS_IN_SECONDS`
- 不能是函数调用的结果或运行时才能算出的值
- 适合硬编码值（最高分、光速等），便于维护和修改

---

## 遮蔽 shadowing

用 `let` 再次声明同名变量，新变量会**遮蔽**旧变量：

```rust
fn main() {
    let x = 5;
    let x = x + 1;        // x = 6
    {
        let x = x * 2;    // 内部 x = 12
        println!("inner scope: {}", x);  // 12
    }
    println!("outer: {}", x);  // 6，内部遮蔽结束
}
```

- 遮蔽是**新变量**，不是修改原变量
- 除非再用 `let`，否则不能重新赋值（仍遵守不可变规则）
- 转换完成后，变量保持不可变

### 遮蔽可改变类型

```rust
let spaces = "   ";      // &str
let spaces = spaces.len(); // usize，遮蔽后类型改变
```

用 `mut` 则**不能**改变类型：

```rust
let mut spaces = "   ";
spaces = spaces.len();   // ❌ 错误：expected `&str`, found `usize`
```

---

## 小结

| 概念 | 关键字 | 要点 |
|------|--------|------|
| 不可变变量 | `let` | 默认，不能二次赋值 |
| 可变变量 | `let mut` | 可修改值 |
| 常量 | `const` | 必须标类型，仅常量表达式 |
| 遮蔽 | `let` 同名 | 新建变量，可换类型 |
