# 11.2 控制测试如何运行

对应 demo 项目：`11-testing/11.2-control-tests-demo/`

`cargo test` 会在测试模式下编译并运行测试二进制。你可以通过命令行参数控制：并发线程数、是否显示输出、运行哪些测试、忽略哪些测试等。

> 参数分两类：  
> - 传给 `cargo test` 本身的参数  
> - 传给测试二进制的参数  
> 用 `--` 分隔：`cargo test [cargo参数] -- [测试二进制参数]`

---

## 并行或连续运行测试

默认测试并行运行（多线程）。若测试共享状态（文件、环境变量、当前工作目录等），可能互相干扰。

指定线程数（例如 1：串行执行）：

```bash
cargo test -- --test-threads=1
```

---

## 显示通过测试的输出

默认：通过的测试会截获 stdout（例如 `println!` 输出不显示）；失败的测试会显示输出。

显示通过测试的输出：

```bash
cargo test -- --show-output
```

---

## 通过名字运行部分测试

运行单个测试（按名字过滤）：

```bash
cargo test one_hundred
```

运行包含某子串的一组测试：

```bash
cargo test add
```

> 测试的“全名”包含模块路径，例如 `tests::add_two_and_two`。

---

## 忽略某些测试：`#[ignore]`

标记耗时测试：

```rust
#[test]
#[ignore]
fn expensive_test() { /* ... */ }
```

默认 `cargo test` 不运行 ignored 测试：

```bash
cargo test
```

只运行 ignored 测试：

```bash
cargo test -- --ignored
```
