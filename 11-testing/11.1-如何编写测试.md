# 11.1 如何编写测试

对应 demo 项目：`11-testing/11.1-tests-demo/`

测试函数用于验证非测试代码是否按预期运行，通常包含三步：

1. **准备数据/状态**
2. **运行被测代码**
3. **断言结果**

---

## 测试函数剖析：`#[test]`

最简单的测试：在函数前加 `#[test]`。

```rust
#[cfg(test)]
mod tests {
    #[test]
    fn it_works() {
        assert_eq!(2 + 2, 4);
    }
}
```

运行：

```bash
cargo test
```

- `#[cfg(test)]`：只在运行测试时编译该模块
- 每个测试在独立线程中运行，panic 会导致该测试失败

---

## 断言宏

### `assert!`

断言条件为 `true`：

```rust
assert!(larger.can_hold(&smaller));
assert!(!smaller.can_hold(&larger));
```

### `assert_eq!` / `assert_ne!`

用于“相等/不等”断言，失败时会打印两边的值，便于定位问题：

```rust
assert_eq!(4, add_two(2));
assert_ne!(add_two(2), 5);
```

> 参与比较的值需实现 `PartialEq` 和 `Debug`。自定义类型一般用 `#[derive(PartialEq, Debug)]`。

---

## 自定义失败信息

可给断言宏追加 format! 风格的消息：

```rust
assert!(
    result.contains(\"Carol\"),
    \"Greeting did not contain name, value was `{}`\",
    result
);
```

---

## `#[should_panic]`

当你期望代码必须 panic 才算通过：

```rust
#[test]
#[should_panic]
fn greater_than_100() {
    Guess::new(200);
}
```

更精确的写法：`expected = \"...\"`，要求 panic 信息包含指定子串：

```rust
#[should_panic(expected = \"less than or equal to 100\")]
```

---

## 用 `Result` 写测试

测试函数也可以返回 `Result`，用 `Err` 表示失败；这样可在测试中使用 `?`：

```rust
#[test]
fn it_works() -> Result<(), String> {
    if 2 + 2 == 4 { Ok(()) } else { Err(\"bad\".into()) }
}
```

注意：返回 `Result` 的测试不能配合 `#[should_panic]`。
