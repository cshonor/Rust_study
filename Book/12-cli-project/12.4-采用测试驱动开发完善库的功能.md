# 12.4 采用测试驱动开发完善库的功能

这一节用 TDD（测试驱动开发）方式把 `minigrep` 的核心“搜索”能力补齐，并让它可以被单元测试直接覆盖。

对应 demo 项目：`Book/12-cli-project/12.4-minigrep-search-demo/`

---

## TDD 基本循环

- 写一个**会失败**的测试（失败原因要符合预期）
- 写/改最少的代码让测试通过
- 重构代码，确保测试仍通过
- 重复

---

## `search(query, contents)` 的期望行为

- 输入：查询字符串 `query` + 待搜索文本 `contents`
- 输出：所有包含 `query` 的**行**（借用自 `contents`）

签名采用书里的形式（显式生命周期）：

```rust
pub fn search<'a>(query: &str, contents: &'a str) -> Vec<&'a str>
```

---

## demo 运行

```bash
cd Book/12-cli-project/12.4-minigrep-search-demo

# 运行测试
cargo test

# 运行搜索
cargo run -- frog poem.txt
cargo run -- body poem.txt
```
