# 8.2.1 `str`、DST 与定长规则

← 主笔记：[8.2 字符串](./8.2-string.md) · 下一篇：[8.2.2 不定长放哪](./8.2.2-不定长数据放哪.md)

---

## 结论（两句）

1. **`str` 是 DST（不定长）**：不能 `let s: str`，要用 **`&str`** 等定长外壳。
2. **「不定长」≠「一定在堆」** → [8.2.2](./8.2.2-不定长数据放哪.md)

---

## 一、定长 vs 不定长

### 栈的规则

栈上**不能直接放**不定长类型（`str`、`[T]`）当独立变量——编译期不知道占多少字节。

### 定长（可直接放栈）

| 类型 | 说明 |
|------|------|
| `i32`、`bool` | 大小固定 |
| `[u8; 4]` | 长度写死在类型里 |

```rust
let a: i32 = 10;
let buf: [u8; 4] = [0; 4];
```

### 不定长（`str`、`[T]`）

`str` = 连续 UTF-8 字节；不同文本长度不同 → **DST**，与 `[u8]` 同类。

---

## 二、为什么不能用 `let s: str`？

```rust
// let s: str = "abc"; // ❌
```

1. 编译器不知道栈上留多少字节；
2. Rust **禁止**编译期大小未知的类型当栈变量。

**规定：** 必须搭配带长度的引用 → **`&str`**（胖指针：ptr + len，本身定长）。

```rust
let s: &str = "abc"; // ✅ 栈上是「书签」，不是整本书
```

```text
栈:  &str { ptr, len }  ──►  [ a b c ]
```

---

## 三、极简背诵

1. **`str`**：不定长；不能 `let s: str`。
2. **铁律**：栈上不能直接存不定长变量 → 用 `&str`、`String` 等定长外壳。
3. **`&str`**：栈上定长胖指针，指向别处的 `str` 数据。
