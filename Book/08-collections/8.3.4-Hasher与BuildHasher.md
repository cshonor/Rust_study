# 8.3.4 Hasher 与 BuildHasher

← 主笔记：[8.3 HashMap](./8.3-hashmap.md) · demo：`8.3.4-hasher-demo/`

> 对应原文：**默认 hasher 偏安全抗 DoS，不一定最快；瓶颈时可换 `BuildHasher`（crates 现成库）。**

---

## 一、两个核心 Trait

### 1. `Hasher` — 单次哈希计算器

`std::hash::Hasher`：对数据**分段写入**，最后算出 **`u64` 哈希值**。一个实例只算一次。

| 方法 | 作用 |
|------|------|
| **`write(&mut self, bytes: &[u8])`** | 灌入字节 |
| **`finish(&self) -> u64`** | 收尾，返回哈希值 |

还有 `write_u32` / `write_usize` 等便利方法。

```rust
use std::hash::{Hash, Hasher};
use std::collections::hash_map::DefaultHasher;

let mut h = DefaultHasher::new();
123u32.hash(&mut h);
let hash_val = h.finish();
```

### 2. `BuildHasher` — Hasher 工厂（HashMap 第三个泛型 `S`）

`std::hash::BuildHasher`：**生产 `Hasher` 实例的工厂**。

```rust
pub struct HashMap<K, V, S = RandomState> { /* … */ }
//                          ↑ S: BuildHasher
```

| 方法 | 作用 |
|------|------|
| **`build_hasher(&self) -> impl Hasher`** | 每次 insert / get 时生成**全新** Hasher |

默认 **`S = RandomState`**（标准库默认哈希工厂）。

---

## 二、标准库默认：`RandomState` + SipHash 1-3

1. **算法：SipHash-1-3**
   - **安全**：进程启动时**随机 seed**，同 key 每次运行哈希值不同 → 攻击者难批量构造冲突 key，防 **HashDoS**（恶意冲突使表退化成链表、O(n) 卡死）
   - **性能**：中等长度字符串较优；**小 key**（`u32`/`i32`）、**超长字符串**偏慢
2. **`DefaultHasher`** = `RandomState` 产出的 `Hasher` 实现

```rust
// 默认 HashMap：S = RandomState，底层 SipHash
let mut map: HashMap<u32, u32> = HashMap::new();
```

> **原文要点**：默认 hasher **安全优先**，不一定最快； profiling 显示哈希是瓶颈时，换 `BuildHasher` 实现。

---

## 三、主流高性能替换（crates.io）

牺牲 DoS 抗性换速度 — **仅适合可信输入**（见 §五）。

| crate | 算法 | 适用场景 | 优缺点 |
|-------|------|----------|--------|
| **`fnv`** | FNV-1a | 小 key：数字、短字符串 | 整数极快；超长字符串一般；**无防 DoS** |
| **`rustc_hash`**（FxHash） | FxHash | 编译器、内存缓存、内部索引 | rustc 原生；小键性能顶尖 |
| **`ahash`** | AHash | 大数据、高频 HashMap | CPU 有 AES 指令时极快 |
| **`hashbrown`** | FoldHash 等 | drop-in 替换 std 表 | 瑞士表 + 快哈希 |

## 四、替换 HashMap 哈希器（两种写法）

### 写法 1：`BuildHasherDefault` + type 别名（单 Map 替换）

以 **`fnv`** 为例：

```toml
# Cargo.toml
fnv = "1.0"
```

```rust
use std::collections::HashMap;
use std::hash::BuildHasherDefault;
use fnv::FnvHasher;

type FnvHashMap<K, V> = HashMap<K, V, BuildHasherDefault<FnvHasher>>;

let mut map: FnvHashMap<u64, &str> = FnvHashMap::default();
map.insert(42, "ok");
```

### 写法 2：`HashMap::with_hasher` 运行时指定工厂

```rust
use std::collections::HashMap;
use rustc_hash::FxBuildHasher;

let mut map: HashMap<u64, i32> =
    HashMap::with_hasher(FxBuildHasher::default());
```

`rustc_hash` 也提供现成别名：

```rust
use rustc_hash::FxHashMap;
let mut map: FxHashMap<u64, i32> = FxHashMap::default();
```

## 五、选型准则

| 场景 | 建议 |
|------|------|
| **外网不可控 key**（HTTP 参数、用户自定义字符串） | **必须默认 SipHash**（`HashMap::new()`） |
| **程序内部数据**（枚举、自增 id、固定配置） | 可换 FxHash / AHash / FNV |
| **百万级高频读写**，profiling 显示哈希占 CPU | 优先考虑 FxHash / AHash |
| **离线计算、本地缓存、量化内存索引** | FxHash / AHash 常见选择 |

## 六、`Hash` trait 与 `Hasher` 的关联

键要能进 `HashMap`，必须 **`Hash + Eq`**：

```rust
pub trait Hash {
    fn hash<H: Hasher>(&self, state: &mut H);
}
```

流程：`key.hash(&mut hasher)` → 把自身字节写入 `Hasher` → `finish()` 得 u64 → 寻址桶位。

自定义 struct：

```rust
#[derive(Hash, Eq, PartialEq)]
struct MyKey { id: u64 }
```

## 附录：可直接复制的 type 别名模板

```rust
// ── FNV ──────────────────────────────────────
use std::collections::HashMap;
use std::hash::BuildHasherDefault;
use fnv::FnvHasher;
type FnvHashMap<K, V> = HashMap<K, V, BuildHasherDefault<FnvHasher>>;

// ── FxHash（rustc_hash）──────────────────────
use rustc_hash::FxHashMap; // 或 FxBuildHasher + with_hasher

// ── AHash ────────────────────────────────────
use ahash::AHashMap; // crate 提供的别名

// ── 默认（抗 DoS）────────────────────────────
type SafeHashMap<K, V> = HashMap<K, V>; // S = RandomState
```
