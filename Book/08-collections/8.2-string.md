## 一、定长类型 vs 不定长类型

### 两个前提

1. **栈规则**：栈上**不能直接放**不定长类型（`str`、`[T]`）当独立变量——编译期不知道占多少字节。
2. **存放位置**：不定长**数据本体**可在**静态只读段、栈、堆**，取决于谁承载；不是只有堆。

> **栈上的变量**必须编译期定大小；**不定长数据**可以住在别处，用 `&str` / `String` 等去指。

### 定长类型（可直接放栈）

| 类型 | 说明 |
|------|------|
| `i32`、`bool` | 大小固定，如 `i32` 永远 4 字节 |
| `[u8; 4]` | 长度写死在类型里 |

```rust
let a: i32 = 10;        // ✅ 编译期知大小
let buf: [u8; 4] = [0; 4];
```

### 不定长类型（`str`、`[T]`）

`str` 本质是**一串连续 UTF-8 字节**：

- `"abc"` → 3 字节
- `"abcdef"` → 6 字节
- **不同文本长度不同**，编译期无法为「这一个 `str`」定死栈上占多少字节

👉 **`str` 是不定长类型（DST）**，和 **`[u8]`**（无长度后缀的切片元素类型）同一类：**底层不定长的原生序列**。

## 二、为什么不能用 `let s: str`？

```rust
// let s: str = "abc"; // ❌ 编译报错
```

**原因：**

1. `str` 长度不固定，编译器不知道在栈上为它留多少字节；
2. Rust **禁止**把「编译期大小未知」的类型，直接当作栈变量。

**规定：** `str` 不能单独用，必须搭配**带长度的引用** → **`&str`**。

`&str` 是什么？**胖指针**（指针 + 长度）：

| 字段 | 作用 |
|------|------|
| 指针 | 指向 UTF-8 数据（**静态只读段 / 栈 / 堆**，见 §三） |
| 长度 | 字节数（不是「几个字」） |

**引用本身大小固定**（64 位系统上通常 16 字节：指针 8 + 长度 8），所以可以安心放在栈上。

```rust
let s: &str = "abc"; // ✅ 栈上放的是「书签」，不是整本书塞进格子
```

```text
栈:  &str { ptr, len }  ──►  [ a b c ]   （数据可在静态段 / 栈 / 堆）
```

## 三、不定长本体放哪？

**DST 本身不强制在堆**；常见三种载体：

| 场景 | 不定长本体在哪 | 栈上有什么 |
|------|----------------|------------|
| 字面量 `"hello"` | **静态只读段**（非栈、非堆） | 定长 `&str` 指向它 |
| `String` / `Vec` | **堆**（可扩容） | 定长结构体 + 指针 |
| 数组切片 `&arr[..]` | 往往在**栈**（跟着定长数组） | 定长 `&[T]` / `&str` |

### 场景 1：字面量 → 静态只读段

```rust
let s: &str = "hello";
```

- `"hello"` 本体是 `str`（不定长），编译进程序 **只读数据段**，全程常驻。
- 栈上只有 **`&str`**（ptr + len，定长），指向静态段。
- ✅ **不定长 `str` 可以不在堆里。**

### 场景 2：`String` → 堆

```rust
let mut s = String::from("hello");
s.push_str(" world");
```

- 栈：`String { ptr, len, cap }`（**定长**结构体）。
- 堆：指针指向的不定长 UTF-8 数据；适合**动态扩容**。

### 场景 3：数组切片 → 本体常在栈

```rust
let arr = [1, 2, 3];
let slice: &[i32] = &arr;
```

- `[i32; 3]` **定长**，整块在栈。
- `&[i32]` 指向的底层 `[i32]` 是**不定长类型**，数据仍跟数组一起待在**栈**上。

👉 与 `str` 同一逻辑：**不定长类型不一定在堆。**

### 规律小结

1. **本体**（`str` / `[T]`）可放：静态只读段、栈、堆。
2. **为何常以为「不定长 = 堆」？** 业务里要**追加、扩容**的，几乎都用 `String` / `Vec`，它们把数据放堆；**固定、只读**的（字面量、栈数组切片）不必上堆。
3. **铁律**：栈上不能 `let x: str` / `let x: [T]`；必须用 **`&str`、`String`、`Vec`** 等定长外壳访问。

```text
        静态只读段          栈                    堆
           │                │                     │
  "hello" str 数据    [1,2,3] + &[i32]     String → str 数据
           ▲                ▲                     ▲
           └── &str ─────────┴──── &str ───────────┘
                 （胖指针本身总在栈或寄存器里，大小固定）
```

## 四、类比理解

### 类比 1：和数组切片对齐

| | 定长 | 不定长（只能借） |
|--|------|------------------|
| 数组 | `[u8; 3]` 可直接 `let` | `[u8]` 不能单独变量 |
| 切片引用 | — | `&[u8]` |
| 字符串 | — | `str` → 只用 `&str` |

### 类比 2：书本与书签

- **`str`** = 书**正文**：页数不固定，没法把「页数未知的整本书」塞进固定大小的栈格子；
- **`&str`** = **书签**（ptr + 页数）：书签大小固定，拿着它就能找到正文。

### 类比 3：和 `Vec` / `String`

- `str` ≈ 连续字节**内容**（不定长，可在静态/栈/堆）
- `String` ≈ `Vec<u8>`：**定长结构体**在栈上；**需要动态改长时**才把数据放堆

## 五、`str` 与 `String` 核心区分

### `str` / `&str`

| | |
|--|--|
| 本质 | UTF-8 **字符序列**（字符串本体） |
| 大小 | **不定长**（DST） |
| 栈变量 | ❌ 不能 `let x: str`；✅ `let x: &str` |
| 所有权 | 无；**借用** |
| 字面量 `"xxx"` | 底层是 `str` 数据，在**程序只读段**；类型常为 `&'static str` |

```rust
let s: &str = "hello world";
```

### `String`

| | |
|--|--|
| 本质 | **定长** `struct`（栈上 ptr / len / cap） |
| 指向 | 通常**堆**上不定长数据（要扩容时）；也可用 `&str` 借静态/栈上的 `str` |
| 栈变量 | ✅ `let s: String` —— 容器大小编译期固定 |
| 所有权 | **有**；可扩容、修改；出作用域 **drop** 堆 |

```rust
struct String {  // 简化
    ptr: *mut u8,
    len: usize,
    cap: usize,
}

let mut s = String::from("hello");
s.push_str(" world");
```

### 对照一眼

| | `&str` | `String` |
|--|--------|----------|
| 栈上是什么 | 固定大小的**引用** | 固定大小的**管理头** |
| 数据在哪 | 只读段 或 某堆块 | **自己的堆** |
| 能扩容？ | ❌ | ✅ |

## 六、常见疑问

### `str` 是本体，`String` 是包装？

✅ **`str` = 纯数据（可在静态/栈/堆）；`String` = 定长容器，常见用法是把数据放在堆并负责扩容。**

### `str` 是不是只读？

| 形态 | 能否改内容 | 能否改长度 |
|------|------------|------------|
| 字面量 `&'static str` | ❌ 只读段，硬件/链接层面只读 | ❌ |
| 从 `String` 借的 `&str` | ❌ 共享借用，语法只读 | ❌ |
| `&mut str` | ✅ 原地改已有字节 | ❌ `str` 长度仍固定 |
| `String` | ✅ | ✅ |

```rust
let mut s = String::from("abc");
let s_mut: &mut str = &mut s;
s_mut.make_ascii_uppercase(); // ABC，不扩容
```

**记住：** 要**变长、追加** → 用 **`String`**；`&mut str` 只能在小范围内原地改。

### 口语「字符串变量」指谁？

- 业务里常说「一个 `String` 变量」；
- `&str` 多用于**只读参数**、字面量、`fn foo(s: &str)`。

## 七、打通：链路 + 互相转换

### 三条代码链（串起 DST + 所有权）

```rust
// 1. 静态只读段里的 str，用 &str 指向
let s1: &str = "hello";

// 2. 把内容拷到堆，交给定长容器 String
let s2: String = String::from("hello");

// 3. 从 String 借出底层 str 的视图
let s3: &str = &s2;
```

```text
"hello"  ──&str──►  [ 只读静态段 ]
String   ──堆──►   [ h e l l o ... ]  ◄── &str 借视图
         栈: { ptr, len, cap }
```

### 互转（日常）

```rust
let s = String::from("rust");
let r: &str = &s;              // 借，不拷贝

let r: &str = "rust";
let s = String::from(r);       // 或 r.to_string()，拷到堆
```

| 写法 | 说明 |
|------|------|
| `&s` | 零拷贝借用 |
| `String::from` / `to_string` | 新堆 + 拷贝 |
| `format!(...)` | 得到 `String` |

## 八、极简背诵版

1. **`str`**：不定长（DST）；不能 `let s: str`。
2. **铁律**：栈上**不能直接存**不定长类型变量 → 用 `&str`、`String` 等定长外壳。
3. **「不定长」≠「一定在堆」**：字面量→静态段；数组切片→常在栈；`String`/`Vec`→堆。
4. **`&str`**：栈上定长书签；**`String`**：栈上定长头，动态内容多在堆。

## 九、UTF-8 实操

`String` / `&str` 内容都是 **UTF-8**；`.len()`、切片按**字节**。

### 新建与更新

```rust
let mut s = String::new();
let s = String::from("initial contents");
s.push_str("bar");
s.push('l');

let s1 = String::from("Hello, ");
let s2 = String::from("world!");
let s3 = s1 + &s2; // s1 被 move
```

### 为什么不能用 `s[i]`？

```rust
// let h = s[0]; // ❌
```

内部类似 `Vec<u8>`；「第 i 个」是字节还是 `char` 语义不清，且不能保证 O(1) 随机访问。

### 字符串 slice（字节 range）

```rust
let hello = "Здравствуйте";
let s = &hello[0..4]; // "Зд"（须在 UTF-8 字符边界上）
```

### 遍历

```rust
for c in "नमस्ते".chars() { /* Unicode 标量 */ }
for b in "नमस्ते".bytes() { /* 字节 */ }
```
