# 8.2.2 不定长数据放哪（静态 / 栈 / 堆）

← [8.2.1 str 与 DST](./8.2.1-str与DST定长规则.md) · 主笔记：[8.2](./8.2-string.md) · 下一篇：[8.2.3 str 与 String](./8.2.3-str与String互转.md)

---

## 核心三句

1. **字面量 `"abc"`**：字节在 **`.rodata` 只读常量段**，不在栈、不在堆  
2. **`String::from(...)`**：**堆**上存 UTF-8 字节（本体类型仍是 `str`）  
3. **`&str`**：永远只是**栈上胖指针**，指向 `.rodata` / 堆 / 栈数组中的某段 `str`

---

## 场景 1：字面量 → `.rodata` 只读常量段

```rust
let s: &str = "abc";
```

1. 字节 `[97, 98, 99]` **编译期**打进二进制 **`.rodata`**；运行时 OS 映射进进程，**只读、全程存活**  
2. **不在栈**：栈是函数临时区，函数结束即回收；字面量全局存活  
3. **不在堆**：堆是运行时 `alloc` 动态申请；字面量内容编译期已定，无需堆  

变量 **`s` 在栈**：类型常为 **`&'static str`**，只存 ptr + len（3）：

| 字段 | 指向 |
|------|------|
| ptr | `.rodata` 里 `'a'` 的地址 |
| len | 3 |

---

## 场景 2：`String` → 堆；`as_str()` → 借堆

```rust
let owned = String::from("abc"); // 栈：String{ptr,len,cap}；堆：[97,98,99]
let borrow: &str = owned.as_str(); // 栈：&str 胖指针 → 指向 owned 的堆字节
```

- 堆上字节**本体**类型本质是 **`str`（DST）**  
- 不能直接持有「堆上裸 `str`」，只能通过 **`&str`** 访问

---

## 场景 3：数组切片 → 本体常在栈

```rust
let arr = [1, 2, 3];
let slice: &[i32] = &arr;
```

定长数组在栈；`&[i32]` 指向的数据常跟数组一起在**栈**上。

---

## 总结表

| 写法 | 本体 `str` 数据在哪 | 栈上有什么 |
|------|---------------------|------------|
| `"hello"` | **`.rodata` 只读段** | 无（仅字面量）；`let x: &str = "hello"` → **x（胖指针）在栈** |
| `String::from("hello")` | **堆** | `String { ptr, len, cap }` |
| `owned.as_str()` | 同 `String`（**堆**） | **`&str` 胖指针在栈** |

---

## 五、`'static` 与 `.rodata`

### 字面量的 `'static` 从哪来？

**对。** 字符串字面量 `"abc"` 的类型是 **`&'static str`**，因为：

- 数据在 **`.rodata`**，程序整个生命周期有效  
- 引用 `'static` = **「活得和整个程序一样长」**

### `'static` 不只来自 `.rodata`

| 来源 | 例子 |
|------|------|
| **字面量** | `"hello"` → `&'static str`（数据在 `.rodata`） |
| **泄漏到堆** | `Box::leak(s.into_boxed_str())` → `&'static str`（数据在堆，但永不释放） |
| **全局 static** | `static S: &str = "x";` |

> **口诀**：字面量 `'static` **因为** 数据在 `.rodata`；但 `'static` 含义是**生命周期**，不限于 `.rodata` 一种来源。

---

## DST 通用规律（不止 `str`）

| 本体（DST） | 胖指针（栈上定长） |
|-------------|-------------------|
| `str` | `&str` |
| `[T]` | `&[T]` |

裸 `[T]` / `str` **不能**单独当栈变量；靠胖指针借。

---

## 规律小结

1. **`str` 本体**：不单独上栈（长度不确定）  
2. **字面量**：数据在 **`.rodata`**，不走堆  
3. **动态字符串**：数据在 **`String` 的堆**，靠 **`&str`** 借用
