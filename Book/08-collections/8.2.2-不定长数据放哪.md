# 8.2.2 不定长数据放哪（静态 / 栈 / 堆）

← [8.2.1 str 与 DST](./8.2.1-str与DST定长规则.md) · 主笔记：[8.2](./8.2-string.md) · 下一篇：[8.2.3 str 与 String](./8.2.3-str与String互转.md)

---

## 核心

**DST 本体不强制在堆**；常见三种载体：

| 场景 | 不定长本体在哪 | 栈上有什么 |
|------|----------------|------------|
| 字面量 `"hello"` | **静态只读段** | 定长 `&str` |
| `String` / `Vec` | **堆**（可扩容） | 定长结构体 + 指针 |
| 数组切片 `&arr[..]` | 往往在**栈** | 定长 `&[T]` / `&str` |

---

## 场景 1：字面量 → 静态只读段

```rust
let s: &str = "hello";
```

- `"hello"` 编译进 **只读数据段**，非常驻栈/堆；
- 栈上只有 **`&str`**（ptr + len）。

---

## 场景 2：`String` → 堆

```rust
let mut s = String::from("hello");
s.push_str(" world");
```

- 栈：`String { ptr, len, cap }`（定长）；
- 堆：UTF-8 数据，可扩容。

---

## 场景 3：数组切片 → 本体常在栈

```rust
let arr = [1, 2, 3];
let slice: &[i32] = &arr;
```

定长数组在栈；`&[i32]` 指向的不定长切片数据跟着数组在**栈**上。

---

## 规律小结

1. **本体**（`str` / `[T]`）可放：静态只读段、栈、堆。
2. **为何常以为「不定长 = 堆」？** 业务里要追加、扩容的多用 `String`/`Vec`；字面量、栈数组切片不必上堆。
3. **铁律**：栈上不能 `let x: str`；用 `&str`、`String`、`Vec` 等定长外壳。

```text
        静态只读段          栈                    堆
           │                │                     │
  "hello" str 数据    [1,2,3] + &[i32]     String → str 数据
           ▲                ▲                     ▲
           └── &str ─────────┴──── &str ───────────┘
```

---

## 类比

| | 定长 | 不定长（只能借） |
|--|------|------------------|
| 数组 | `[u8; 3]` | `[u8]` → `&[u8]` |
| 字符串 | — | `str` → `&str` |

**书签类比**：`str` = 正文（页数不固定）；`&str` = 书签（ptr + 长度，定长）。
