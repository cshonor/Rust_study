# 8.1 Vector：用 `Vec<T>` 储存一系列的值

← 集合章入口 · 下一节：[8.2 String](./8.2-string.md)

## 一句话

> **`Vec<T>` = 堆上动态连续数组** — 同类型元素、自动扩容。对标 Java `ArrayList`、C++ `vector`。

---

## 一、三种创建方式

### 1. 空 Vec：`Vec::new()`

```rust
let mut v: Vec<i32> = Vec::new();
```

无元素时编译器无法推断 `T`；刚创建时**通常不分配堆内存**，第一次 `push` 才向堆申请空间。

#### `Vec::new()` 类型推断详解

**① 只写 `Vec::new()`、后面暂时不用 → 必须标注**

```rust
// let mut v = Vec::new();              // ❌ 空容器无元素，推导不出 T
let mut v: Vec<i32> = Vec::new();       // ✅ 手动指定泛型
```

- `Vec::new()` 创建**空动态数组**，堆里通常还没有元素；
- 没有数据参考，编译器无法判断未来存 `i32` / `String` / `u8` → **类型推导失败**。

**② 已标注后 `push`，不用改标注**

```rust
let mut v: Vec<i32> = Vec::new();
v.push(10);
v.push(20);
// push(10) 验证标注正确；后续只能 push i32，push String 会报错
```

**③ `vec![]` 自带元素 → 永远不用标注**

```rust
let mut v = vec![1, 2, 3]; // 1、2、3 是 i32 → 自动推断 Vec<i32>
```

**④ 空 Vec + 紧接着 `push` → 可省略标注（后置推导）**

```rust
let mut v = Vec::new();
v.push(5i32); // 第一次 push 带入 i32，编译器反向锁定 Vec<i32>
```

> **规则**：**定义这一行**若无法推导 → 报错；**定义后立刻 `push` 带类型数据** → 编译器可后置推导，不必写 `Vec<i32>`。

| 场景 | 要不要写 `Vec<i32>` |
|------|---------------------|
| 单独 `Vec::new()`，暂不使用 | **要** |
| `Vec::new()` 后马上 `push(5i32)` | **不用** |
| `vec![1, 2, 3]` | **不用** |

### 2. 初始化赋值：`vec![]` 宏（最常用）

```rust
let v = vec![1, 2, 3];   // 自动推导 Vec<i32>
let v2 = vec![0; 5];     // [0, 0, 0, 0, 0] — 仅 Copy 类型可用
```

`vec!` 编译期展开为 `Vec::new()` + 多次 `push`，是简写语法糖。

### 3. 预分配容量：`Vec::with_capacity(N)`（性能优化）

```rust
let mut v = Vec::with_capacity(10);
// 预先预留 10 个元素的空间，前 10 次 push 通常不触发扩容
```

---

### 关键概念：`len` vs `capacity`

| | 含义 |
|---|------|
| **`len()`** | 当前真实存了多少个元素 |
| **`capacity()`** | 堆上已申请、能容纳多少元素（不扩容的前提下） |

当 **`len == capacity`** 时再 `push` → 触发 **`realloc` 扩容**（申请更大块 → 拷贝 → 释放旧块）。

## 二、修改 Vec：`push` 追加尾部

修改必须 **`mut`** — Rust 不可变变量不能改内容。

```rust
let mut v = Vec::new();
v.push(5);
v.push(6);
```

## 三、生命周期：离开作用域自动销毁

```rust
{
    let v = vec![1, 2, 3];
} // v 调用 Drop：堆内存 + 所有元素释放，无泄漏
```

## 四、两种读取：`[]` 索引 vs `get()`（必考）

```rust
let v = vec![1, 2, 3, 4, 5];

let third = &v[2];           // 方式 1：下标索引

match v.get(2) {              // 方式 2：Option
    Some(val) => println!("{}", val),
    None => println!("无该元素"),
}
```

| 写法 | 越界行为 | 适用场景 |
|------|----------|----------|
| **`&v[x]`** | **panic** 崩溃 | 下标确定合法、快速失败 |
| **`v.get(x)`** | 返回 **`None`** | 用户输入下标、业务容错 |

可变读取：**`v.get_mut(idx) → Option<&mut T>`**

## 五、经典报错：有不可变引用时不能 `push`（核心难点）

```rust
let mut v = vec![1, 2, 3, 4, 5];
let first = &v[0];
v.push(6); // ❌ 编译报错 E0502
println!("{}", first);
```

### 底层原理（扩容）

1. `first` 保存堆上**某个地址**的不可变借用；
2. 若 `len == capacity`，`push` 可能 **realloc**：
   - 堆上开更大新块
   - 拷贝所有元素到新地址
   - **释放旧堆内存**
3. 原 `first` 指向的旧内存已无效 → **悬垂引用**；
4. 借用检查器**编译期拦截**，杜绝野指针。

### ✅ 规避：引用用完再 `push`

```rust
let mut v = vec![1, 2, 3];
{
    let first = &v[0];
    println!("{}", first);
} // first 生命周期结束
v.push(6); // ✅
```

## 六、常用拓展方法

```rust
let mut v = vec![1, 2, 3];
v.pop();            // 弹出末尾 Option<T>
v.remove(0);        // 删除指定下标（后面元素前移，O(n)）
v.clear();          // 清空元素，capacity 通常保留
v.shrink_to_fit();  // 释放多余空闲内存，让 capacity ≈ len
```

## 七、遍历

### 不可变遍历

```rust
let v = vec![100, 32, 57];
for i in &v {
    println!("{}", i);
}
```

### 可变遍历并修改

```rust
let mut v = vec![100, 32, 57];
for i in &mut v {
    *i += 50;
}
```

## 八、用枚举在 Vec 中存多种类型

Vec **只能存同一类型**；混合类型用 **enum** 包装：

```rust
enum SpreadsheetCell {
    Int(i32),
    Float(f64),
    Text(String),
}

let row = vec![
    SpreadsheetCell::Int(3),
    SpreadsheetCell::Text(String::from("blue")),
    SpreadsheetCell::Float(10.12),
];
```

编译期无法确定类型集合时 → trait 对象（第 17 章）。
