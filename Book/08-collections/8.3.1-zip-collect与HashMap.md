# 8.3.1 `zip` → `collect` → `HashMap`

← 主笔记：[8.3 HashMap](./8.3-hashmap.md) · demo：`8.3-hashmap-demo/` §2

---

## 一句话

> **两个 `Vec` 用 `zip` 拼成 `(key, val)` 元组迭代器 → `collect` 按类型标注装进 `HashMap` 集合。**

| 角色 | 是什么 |
|------|--------|
| **`zip`** | 迭代器方法，**配对**两个迭代器（不属于集合本身） |
| **`collect`** | 迭代器**终结者**，把数据打包成 `Vec` / `HashMap` / … |
| **`HashMap`** | `std::collections` 里的**最终容器** |

---

## 1. `zip`：两个迭代器一对一配对

```rust
let k_list = vec!["A", "B"];
let v_list = vec![10, 20];

// k_list.iter().zip(v_list.iter())
// 产出：(&"A", &10), (&"B", &20)  — 元组迭代器
```

- 左 + 右捆成 `(左, 右)` 元组  
- **长短不一以短的为准**，多余丢弃  

---

## 2. `collect`：按类型标注决定容器

`.collect()` 能把迭代器收进**任意集合** —— 靠**左侧类型标注**：

```rust
// 收进 HashMap
let map: HashMap<_, _> = k.iter().zip(v.iter()).collect();

// 不收进 HashMap，默认思路是 Vec<(K,V)>
let pairs: Vec<_> = k.iter().zip(v.iter()).collect();
```

> 没写 `HashMap<_, _>`，编译器**不知道**你要字典，必须标注（或靠上下文推断）。

### `HashMap<_, _>` 里的 `_`

`_` = 让编译器**自动推导** K、V 类型，不必手写 `HashMap<String, i32>`。

---

## 3. 与 Collection（集合）的关系

`std::collections` = Rust 标准库集合包：

`HashMap` · `Vec` · `HashSet` · `BTreeMap` … **都是 Collection**

```
zip（迭代器配对）
    ↓
collect（按标注选容器）
    ↓
HashMap / Vec / …（集合载体）
```

- **`zip`**：只负责配对（迭代器层）  
- **`collect`**：负责**存入**集合  
- **`HashMap`**：键值字典这种**具体容器**

---

## 完整代码

```rust
use std::collections::HashMap;

let keys = vec![String::from("蓝"), String::from("黄")];
let vals = vec![10, 50];

// iter → (&String, &i32)；into_iter → (String, i32) 所有权
let map: HashMap<_, _> = keys.iter().zip(vals.iter()).collect();
```

### `iter` vs `into_iter`（常混）

| 链 | Map 里存的 K/V |
|----|----------------|
| `keys.iter().zip(vals.iter())` | `&String`, `&i32` |
| `keys.into_iter().zip(vals)` | `String`, `i32`（move 进 Map） |

→ demo 用 `into_iter().zip()` 便于 `map["A"]` 用 `&str` 查。

---

## 流程图

```text
Vec keys ──iter/into_iter──┐
                          ├── zip() ──► (K,V) 迭代器 ──collect()──► HashMap
Vec vals ──iter/into_iter──┘              ▲
                                          │
                              类型标注 HashMap<_,_>
```
