# 7.1.6 一个 Package 只能有一个 `lib.rs`

← 主笔记：[7.1](./7.1-包和crate.md) · [7.1.5 lib.rs 是 Crate 出口](./7.1.5-lib.rs是Crate出口.md) · 多库 → [14.3 Workspace](../14-cargo-crates/14.3-Cargo工作空间.md)

---

## 核心死规则（Cargo 硬性语法）

> **1 个 Package：最多 1 份 `src/lib.rs` → 最多 1 个 Library Crate。**

| | |
|---|---|
| ✅ | `src/lib.rs` — 固定路径、固定文件名，**唯一** lib crate 入口 |
| ❌ | 同一 Package **不能**有第二个 `lib.rs`；Cargo **不识别**第二个 lib crate |
| ❌ | `src/lib2.rs`、`src/other_lib.rs` **不会**自动变成第二个 Library Crate |

---

## 一、想要三个库怎么办？

**不能塞在同一个 Package 里。** 拆成 **3 个独立 Package**，放进 **Workspace**：

```
workspace/
├ Cargo.toml          # [workspace] members = [...]
├ lib_order/          # Package1 → 唯一 src/lib.rs → 库 1
├ lib_market/         # Package2 → 唯一 src/lib.rs → 库 2
└ lib_common/         # Package3 → 唯一 src/lib.rs → 库 3
```

三个文件夹 = **三个 Package**，各自 **一份** `lib.rs` = **三个独立 Library Crate**。

```toml
# gateway/Cargo.toml 示例
[dependencies]
lib_order = { path = "../lib_order" }
lib_common = { path = "../lib_common" }
```

demo 思路 → [14.3-hft-workspace-demo](../14-cargo-crates/14.3-hft-workspace-demo/)（`engine` + `market_data` 各为独立 lib Package）

## 二、最大误区：「库模块」≠「库 Crate」

❌ **错**：项目里建 `order/`、`market/`、`common/` 三个文件夹 = 三个库 crate  

✅ **对**：三个文件夹只是 **同一个 lib crate 内部的子模块**，全由 **唯一** `src/lib.rs` 管控 → 编译只出 **1 个** `.rlib`。

```
mypackage/              ← 1 个 Package
├ Cargo.toml
└ src/
   ├ lib.rs             ← 全局唯一 lib 入口 = 只生成 1 个 Library Crate
   ├ order/mod.rs       ← 模块（格子），不是 crate
   ├ market/mod.rs
   └ common/mod.rs
```

`lib.rs`：

```rust
pub mod order;
pub mod market;
pub mod common;
```

| | 是什么 | 编译产物 |
|---|--------|----------|
| `order` / `market` / `common` | **module**，代码分组 | 合在 **同一个** lib crate 里 |
| 整个 `mypackage` 的 lib | **1 个 Library Crate** | **1 个** `.rlib` |

外部 `use mypackage::order::…` — 用的是 **一个库** 里的 **一个模块**，不是「第二个库 crate」。

→ 模块 vs crate 层级：[7.1.3 §二](../07-packages-modules/7.1.3-Actix-web项目目录范例.md)

## 三、两种 Crate 再复盘

| | Lib Crate | Bin Crate |
|---|-----------|-----------|
| **入口** | `src/lib.rs`（**至多 1 份 / Package**） | `src/main.rs` + `src/bin/*.rs`（**可多个**） |
| **产物** | 库（`.rlib`），供链接 / `use` | exe |
| **数量 / Package** | **0 或 1** | **不限** |

---

## 四、若「强行」加 `src/lib2.rs` 会怎样？

Cargo **只认** `src/lib.rs` 为 lib crate root。  
`src/lib2.rs` 若不在模块树里被 `mod` 引用 → **死代码，不参与编译**；  
若在 `lib.rs` 写 `mod lib2;` → 只是 **crate 内又一个模块文件**，**仍是 1 个 lib crate**。

```rust
// lib.rs — 仍是同一个 Library Crate
mod lib2;   // 对应 src/lib2.rs，是模块名，不是第二个 lib crate
```

- `src/lib2.rs` 演示 → `bin_plus_lib/src/lib2.rs`（`mod lib2;` 仍是同一 lib crate）

---

## 五、一句话总结

| 需求 | 做法 |
|------|------|
| 同 Package 多文件夹 | **多个 module**，共用 **1 个** `lib.rs` / **1 个** lib crate |
| 多个独立库 | **多个 Package**，各 **1 个** `lib.rs`，丢 **Workspace** |

### 口诀

**同 Package：文件夹 = 子模块，一盒多分格；要多盒 → 多 Package + Workspace。**
