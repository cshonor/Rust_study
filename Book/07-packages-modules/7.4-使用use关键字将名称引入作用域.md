# 7.4 使用 use 关键字将名称引入作用域

很多时候完整路径会又长又重复。`use` 可以把路径一次性“引入当前作用域”，后续像本地名称一样使用。

对应 demo 项目：`Book/07-packages-modules/7.4-use-demo/`

---

## `use`：把路径引入作用域

把模块引入作用域后，就能用更短的路径调用：

```rust
use crate::front_of_house::hosting;

pub fn eat_at_restaurant() {
    hosting::add_to_waitlist();
}
```

`use` 仍然会检查私有性：**路径上任何一段不可见，都不能引入**。

---

## 惯用的 `use` 写法

- **函数**：通常引入其父模块，调用时保留 `module::func()`，让读者一眼看出“不是本地函数”
- **结构体/枚举/类型**：通常直接引入类型名本身（例如 `use std::collections::HashMap;`）

---

## 同名冲突与 `as`

当要引入两个同名项（例如两个 `Result`）时：

- 更常见：引入父模块来区分（`fmt::Result` vs `io::Result<()>`）
- 或者：用 `as` 起别名（`use std::io::Result as IoResult;`）

---

## `pub use`：重导出（re-export）

`use` 引入的名称默认只在当前模块可见；如果希望**外部代码也能用这个新路径**，用 `pub use`：

```rust
pub use crate::front_of_house::hosting;
```

这样外部就可以直接用 `your_crate::hosting::...`，而不必知道你的内部模块树长什么样。

---

## 嵌套路径（nested paths）

把有相同前缀的导入合并到一行：

```rust
use std::{cmp::Ordering, io};
use std::io::{self, Write};
```

---

## glob `*`：一次性导入所有公有项（谨慎）

```rust
use std::collections::*;
```

会把 `std::collections` 下所有公有项引入作用域。测试代码里较常见；业务代码里用它要谨慎，避免作用域里突然出现一堆不清楚来源的名字。
