## 一、一个 Package 必备三样东西

| # | 组成 | 说明 |
|---|------|------|
| 1 | **项目根文件夹** | 整个项目目录 |
| 2 | **`Cargo.toml`** | 清单配置，Package 身份证，**必填** |
| 3 | **`src/`** | 源码目录，放各类 `.rs` |

**附带自动生成**：`Cargo.lock` — Cargo 写入，**不用手动建**，锁定依赖精确版本；也算 Package 内文件。

> **Package = 项目根文件夹 + `Cargo.toml` + `src/` 源码 + `Cargo.lock`**

## 二、Package 和 Crate 别混（层级）

| | Package | Crate |
|---|---------|-------|
| **谁管** | Cargo | `rustc` |
| **是什么** | **项目包**（构建管理单位） | **编译单元**（最小编译单位） |
| **数量关系** | 1 个项目 = **1 个 Package** | 1 个 Package 里 **≥ 1 个 Crate** |

### Package 里能装几个 Crate

- **0 或 1 个**库 crate（`src/lib.rs`）
- **任意多个**二进制 crate（`src/main.rs`、`src/bin/*.rs`）

### Crate 二选一（每个 crate 本身）

| 入口 | 类型 | 产物 |
|------|------|------|
| `src/main.rs` | binary crate | exe |
| `src/lib.rs` | library crate | 库文件，供 `use` |

代码举例 → **[7.1.1](./7.1.1-二进制与库crate.md)**

### 标准举例：一个 Package，两个 Crate

```
my_project/          ← 这整个文件夹 = 1 个 Package = 1 个 Project
├ Cargo.toml         ← Package 配置
├ Cargo.lock         ← 自动生成
└ src/
   ├ lib.rs          ← 库 crate
   └ main.rs         ← 二进制 crate
```

**文件夹 = Package = Project；里面装了 2 个 Crate。**

demo → `bin_plus_lib/`

---

## 三、Package 三条硬性规则

1. **最多 1 个** library crate（`src/lib.rs`）
2. **任意多个** binary crate
3. **至少 1 个** crate（库或二进制）

---

## 四、两种最简 Package + 一种扩展

### ① 只有 exe（无 `lib.rs`）

```
demo/
├ Cargo.toml
├ Cargo.lock
└ src/main.rs
```

→ 1 个 Package，**1 个二进制 crate**。demo → `only_bin/`

### ② 只有库（无 `main.rs`）

```
mylib/
├ Cargo.toml
├ Cargo.lock
└ src/lib.rs
```

→ 1 个 Package，**1 个库 crate**。demo → `only_lib/`

### ③ 扩展：多二进制（`src/bin/`）

```
demo/
├ Cargo.toml
└ src/
   ├ lib.rs
   └ bin/cli1.rs …
```

→ 1 lib + 多 bin。`cargo run --bin cli1`。demo → `multi_bin/`

---

## 五、Crate Root

> **crate root** = 编译入口，自动成为该 crate 根模块 `crate::`。

| 文件 | 角色 |
|------|------|
| `src/main.rs` | 主 binary crate root；名 = package 名 |
| `src/lib.rs` | library crate root；名 = package 名 |

---

## 六、层级一句话

**Package（Project 文件夹）≥ 1 个 Crate → 每个 Crate 从 Crate Root 编译整棵模块树**

```
my_project/  （Package = Project）
 ├── Cargo.toml / Cargo.lock
 └── src/
      ├── main.rs  → binary crate  → exe
      └── lib.rs   → library crate → 库（至多一个）
```

---

## 七、为什么要区分 crate

crate 把能力放在独立命名空间，避免冲突：`rand::Rng` 与自建 `struct Rng` 互不混淆。

---

## 八、精简笔记

1. **Package = 完整项目文件夹（Project）**，靠 `Cargo.toml` 定义
2. `.rs` 源码、`Cargo.toml`、`Cargo.lock` 都属于 Package 内部
3. **一个 Package ≥ 1 个 Crate**（编译产物：exe / 库）
