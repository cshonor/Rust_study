## 一、一个 Package 必备三样东西

| # | 组成 | 说明 |
|---|------|------|
| 1 | **项目根文件夹** | 整个项目目录 |
| 2 | **`Cargo.toml`** | 清单配置，Package 身份证，**必填** |
| 3 | **`src/`** | 源码目录，放各类 `.rs` |

**附带自动生成**：`Cargo.lock` — Cargo 写入，**不用手动建**，锁定依赖精确版本；也算 Package 内文件。

> **Package = 项目根文件夹 + `Cargo.toml` + `src/` 源码 + `Cargo.lock`**

## 二、Package 和 Crate 别混（层级）

| | Package | Crate |
|---|---------|-------|
| **谁管** | Cargo | `rustc` |
| **是什么** | **项目包**（构建管理单位） | **编译单元**（最小编译单位） |
| **数量关系** | 1 个项目 = **1 个 Package** | 1 个 Package 里 **≥ 1 个 Crate** |

### Package 里能装几个 Crate

- **0 或 1 个**库 crate（`src/lib.rs`）
- **任意多个**二进制 crate（`src/main.rs`、`src/bin/*.rs`）

### Crate 二选一（每个 crate 本身）

| 入口 | 类型 | 产物 |
|------|------|------|
| `src/main.rs` | binary crate | exe |
| `src/lib.rs` | library crate | 库文件，供 `use` |

代码举例 → **[7.1.1](./7.1.1-二进制与库crate.md)**

### 标准举例：一个 Package，两个 Crate

```
my_project/          ← 这整个文件夹 = 1 个 Package = 1 个 Project
├ Cargo.toml         ← Package 配置
├ Cargo.lock         ← 自动生成
└ src/
   ├ lib.rs          ← 库 crate
   └ main.rs         ← 二进制 crate
```

**文件夹 = Package = Project；里面装了 2 个 Crate。**

demo → `bin_plus_lib/`

## 三、Cargo 硬性规定（官方，必记）

| 种类 | 数量限制 | 来源 |
|------|----------|------|
| **library crate** | **0 或 1 个** | 仅 `src/lib.rs`（全局唯一，不能多个） |
| **binary crate** | **不限** | `src/main.rs`（默认主 bin，至多 1 个）+ `src/bin/*.rs`（每个文件 = 1 个 exe） |

另：**至少 1 个 crate**（全是 bin、全是 lib、或 lib+bin 均可）。

### 1. 库 crate：只能 0 个或 1 个

- 有 `src/lib.rs` → 本 Package **唯一 1 个** lib crate
- **不能**建第二个 `lib.rs`，Cargo **不支持**一个 Package 多个库 crate

| | |
|---|---|
| ❌ | 一个 Package 里多个 lib crate |
| ✅ | lib **要么没有、要么仅此一个** |

> 想要多个库 → 拆成**多个独立 Package**，在各自 `Cargo.toml` 里 `path` 依赖（如 `only_lib/` + `user_demo/`）。**不能**在同一 Package 放多个 `lib.rs` → [7.1.6](./7.1.6-一个Package一个lib.rs.md)

### 2. 二进制 crate：数量不限

三种来源：

| 来源 | 说明 |
|------|------|
| `src/main.rs` | 默认**主**二进制（至多 1 个） |
| `src/bin/xxx.rs` | **每个 `.rs` = 独立 exe**；10 个文件 = 10 个 bin crate |
| （无 lib 时）仅上述 bin | 纯 exe 项目也合法 |

```
proj/
├ Cargo.toml
└ src/
   ├ lib.rs          // 唯一库 crate（0/1 二选一）
   ├ main.rs         // 主二进制 1 号
   └ bin/
       ├ cli1.rs     // 二进制 2 号
       ├ cli2.rs     // 二进制 3 号
       └ tool.rs     // 二进制 4 号
```

```bash
cargo run              # 多个 bin 时需 default-run 或 --bin（见 multi_bin/Cargo.toml）
cargo run --bin cli1   # 单独跑 cli1，互不干扰
```

demo → `multi_bin/`（1 lib + `main.rs` + `bin/cli1` + `bin/cli2`）

`src/bin/` 语义、多工具 vs 微服务、HFT workspace → **[7.1.4](./7.1.4-src-bin与多exe.md)**

### 一句话

- **Lib：0 或最多 1 个**（全局唯一 `lib.rs`）
- **Bin：≥ 0 个，随便加**；`main.rs` 只是默认的那一个

## 四、两种最简 Package + 一种扩展

### ① 只有 exe（无 `lib.rs`）

```
demo/
├ Cargo.toml
├ Cargo.lock
└ src/main.rs
```

→ 1 个 Package，**1 个二进制 crate**。demo → `only_bin/`

### ② 只有库（无 `main.rs`）

```
mylib/
├ Cargo.toml
├ Cargo.lock
└ src/lib.rs
```

→ 1 个 Package，**1 个库 crate**。demo → `only_lib/`

### ③ 扩展：多二进制（`src/bin/`）

```
demo/
├ Cargo.toml
└ src/
   ├ lib.rs
   └ bin/cli1.rs …
```

→ 1 lib + 多 bin。`cargo run --bin cli1`。demo → `multi_bin/`

## 五、Crate Root

> **crate root** = 编译入口，自动成为该 crate 根模块 `crate::`。

| 文件 | 角色 |
|------|------|
| `src/main.rs` | 主 binary crate root；名 = package 名 |
| `src/lib.rs` | library crate root；名 = package 名 |

---

## 六、层级一句话

**Package（Project 文件夹）≥ 1 个 Crate → 每个 Crate 从 Crate Root 编译整棵模块树**

```
my_project/  （Package = Project）
 ├── Cargo.toml / Cargo.lock
 └── src/
      ├── main.rs  → binary crate  → exe
      └── lib.rs   → library crate → 库（至多一个）
```

---

## 七、为什么要区分 crate

crate 把能力放在独立命名空间，避免冲突：`rand::Rng` 与自建 `struct Rng` 互不混淆。

---

## 八、精简笔记

1. **Package = Project 文件夹** + `Cargo.toml` + `src/` + `Cargo.lock`
2. **Lib crate：0 或 1**（只有一份 `lib.rs`；要多库 → 多 Package）
3. **Bin crate：不限**（`main.rs` 至多 1 个 + `src/bin/*.rs` 每个文件 1 个 exe）
4. **一个 Package ≥ 1 个 Crate**（编译产物总和）
