# 7.1 包和 crate

crate 是 Rust 的**编译单元**：要么是一个二进制（binary），要么是一个库（library）。

package（包）是 Cargo 的**构建单元**：一个目录 + 一个 `Cargo.toml`，描述如何构建这个包里包含的一个或多个 crate。

对应 demo 项目：`Book/07-packages-modules/7.1-packages-crates-demo/`

---

## crate、crate root

- **crate**：一个二进制项或库。
- **crate root**：Rust 编译器从某个源文件开始构建 crate 的“入口文件”，它会成为该 crate 的根模块（root module）。

Cargo 的默认约定：

- `src/main.rs` 是与 package 同名的**二进制 crate** 的 crate root
- `src/lib.rs` 是与 package 同名的**库 crate** 的 crate root

---

## package（包）的规则

一个 package 中 crate 的数量遵循这些规则：

- **至多一个** library crate（最多只有一个 `src/lib.rs`）
- **可以有任意多个** binary crate
- **至少有一个** crate（要么是 binary，要么是 library）

常见布局：

- 只有 `src/main.rs`：只有 1 个 binary crate
- 同时有 `src/main.rs` + `src/lib.rs`：1 个 binary crate + 1 个 library crate（名字都与 package 相同）
- `src/bin/*.rs`：每个文件都会编译成一个**独立的 binary crate**

---

## 为什么要区分 crate 的作用域

crate 会把一组相关能力放在自己的命名空间里，方便复用，也避免冲突。

比如依赖 `rand` 后：

- `rand::Rng` 指的是 `rand` crate 里的 `Rng` trait
- 你也可以在自己的 crate 里定义 `struct Rng`，两者不会混淆

---

## demo 看点（你运行后会看到什么）

这个 demo 同时包含：

- `src/main.rs`（binary crate，入口 `main`）
- `src/lib.rs`（library crate，对外导出函数）
- `src/bin/alt.rs`（另一个独立 binary crate）

运行方式：

```bash
cd Book/07-packages-modules/7.1-packages-crates-demo

# 跑默认二进制（src/main.rs）
cargo run

# 跑另一个二进制（src/bin/alt.rs）
cargo run --bin alt

#（可选）显式指定 main.rs 对应的 bin
cargo run --bin packages_crates_demo
```
