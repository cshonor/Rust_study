# 7.1 包（Package）与 Crate

**Package**（Cargo 包）= 文件夹 + `Cargo.toml`，管理构建与依赖。  
**Crate**（箱子）= Rust **编译最小单元**，产物是 exe 或库。

## 一、两个视角（先分清层级）

| 概念 | 谁管 | 是什么 |
|------|------|--------|
| **Crate** | `rustc` | 编译单元：binary **或** library，二选一 |
| **Package** | Cargo | 构建单元：1 个目录 + 1 个 `Cargo.toml`，可含多个 crate |

> 单独 `rustc foo.rs` → 该文件就是 1 个 crate，不依赖 Cargo。

---

## 二、Crate 二选一

| 类型 | 入口 | 产物 | 特点 |
|------|------|------|------|
| **binary crate** | `fn main()` | exe | 可 `cargo run` |
| **library crate** | 无 `main` | `.rlib` 等 | 供 `use`；不能 run |

代码举例与 `pub` 规则 → **[7.1.1](./7.1.1-二进制与库crate.md)**

---

## 三、Package 三条硬性规则

1. **最多 1 个** library crate（`src/lib.rs`）
2. **任意多个** binary crate
3. **至少 1 个** crate（库或二进制）

---

## 四、Crate Root 与三种目录树

> **crate root** = 编译入口；自动成为该 crate 的根模块 `crate::`。

| 文件 | 角色 |
|------|------|
| `src/main.rs` | 主 **binary** crate root；名 = package 名 |
| `src/lib.rs` | **library** crate root；名 = package 名 |

### 三种常见布局

| # | 结构 | Package 内含 | demo |
|---|------|--------------|------|
| ① | 仅 `main.rs` | 1 bin | `only_bin/` |
| ② | `main.rs` + `lib.rs` | 1 bin + 1 lib（最常用） | `bin_plus_lib/` |
| ③ | `lib.rs` + `src/bin/*.rs` | 1 lib + 多 bin | `multi_bin/` |

```
① only_bin          ② bin_plus_lib           ③ multi_bin
demo/               demo/                    demo/
├ Cargo.toml        ├ Cargo.toml             ├ Cargo.toml
└ src/main.rs       └ src/                   └ src/
                       ├ main.rs                 ├ lib.rs
                       └ lib.rs                  └ bin/cli1.rs …
```

② 中 bin **自动依赖**同包 lib → `main.rs` 可 `use 包名::…`（见 [7.1.1](./7.1.1-二进制与库crate.md)）。  
③ 中 `cargo run --bin cli1` 指定二进制。

---

## 五、层级一句话

**Package ≥ 1 个 Crate → 每个 Crate 从 Crate Root 编译整棵模块树**

```
Cargo Package（Cargo.toml）
 ├── binary crate（main.rs / src/bin/*.rs）→ exe
 └── library crate（lib.rs，至多一个）    → 库，pub 对外 API
```

---

## 六、为什么要区分 crate

crate 把能力放在独立命名空间，避免冲突：

- `rand::Rng` → 依赖 crate 里的 trait
- 本 crate 也可 `struct Rng`，互不混淆
