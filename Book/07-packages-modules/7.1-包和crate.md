## 一、两个核心定义（层级区分）

### 1. Crate（**Rust 编译最小单元**，`rustc` 视角）

一个 crate **二选一**：

| 类型 | 特征 | 产物 |
|------|------|------|
| **binary crate（二进制）** | 必须有 `fn main()` | 可执行文件（exe） |
| **library crate（库）** | 无 `main` | `.rlib` 等库文件，供别的项目 `use`（如 `rand`、`std`） |

> 单独 `rustc xxx.rs`，这个 `.rs` 文件本身就是 **1 个 crate**，不依赖 Cargo 也成立。

### 2. Package（包，**Cargo 构建管理单元**，Cargo 视角）

**一个项目文件夹 + `Cargo.toml` = 1 个 Package**。

作用：统一管理**同一个项目里多个 crate** 的依赖、版本、编译配置。

### 3. Package 硬性规则（必记）

1. **最多 1 个库 crate**（`src/lib.rs`）
2. **可以有任意多个二进制 crate**
3. **至少要有 1 个 crate**（库或二进制二选一）

---

## 二、Crate Root（Crate 根文件）

> **crate root** = 编译器**编译该 crate 的入口源码**，自动成为这个 crate 的**根模块** `crate::`。

Cargo 默认识别：

| 文件 | 含义 |
|------|------|
| `src/main.rs` | **主二进制 crate** 的 root；crate 名 = package 名 |
| `src/lib.rs` | **库 crate** 的 root；库名 = package 名 |

### 三种常见目录结构

#### ① 只有二进制（`cargo new xxx` 默认）

```
demo/
 ├ Cargo.toml
 └ src/main.rs
```

→ 1 个 Package、**1 个二进制 crate、0 个库 crate**。

#### ② 二进制 + 库共存（最常用）

```
demo/
 ├ Cargo.toml
 └ src/
    ├ main.rs   // bin crate root
    └ lib.rs    // lib crate root
```

→ 1 个 Package、**1 库 + 1 二进制**（两个独立 crate）；  
二进制**默认自动依赖同包的库**，`main.rs` 可直接 `use demo::xxx;`。

#### ③ 多个二进制（`src/bin/`）

```
demo/
├ Cargo.toml
└ src/
   ├ lib.rs
   └ bin/
       ├ cli1.rs
       └ cli2.rs
```

`src/bin/` 下**每个 `.rs` 都是独立二进制 crate**，文件名 = 程序名；  
`cargo run --bin cli1` 单独运行对应程序。

## 三、极简代码示例（② 库 + 二进制）

```rust
// src/lib.rs — 库 crate root
pub fn greet(name: &str) -> String {
    format!("hello from lib: {name}")
}
```

```rust
// src/main.rs — 二进制 crate root（假设 package 名 demo）
fn main() {
    // 同包库 crate 自动可用，crate 名 = Cargo.toml 的 name
    println!("{}", demo::greet("main.rs"));
}
```

---

## 四、为什么要区分 crate 的作用域

crate 把一组能力放在自己的命名空间里，方便复用、避免冲突。

依赖 `rand` 后：

- `rand::Rng` → `rand` crate 里的 trait
- 本 crate 里也可 `struct Rng`，两者不混淆

---

## 五、一句话层级总结

**Package（Cargo 项目包） ≥ 1 个或多个 Crate（编译产物：exe / 库） → 每个 Crate 从 Crate Root 入口编译整棵模块树**
