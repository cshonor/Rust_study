# 7.1.1 二进制 crate 与库 crate（main.rs / lib.rs）

← 主笔记：[7.1 包与 Crate](./7.1-包和crate.md) · demo：[7.1-packages-crates-demo](./7.1-packages-crates-demo/) · 模块 `pub` 细则 → [7.2](./7.2-定义模块来控制作用域与私有性.md)

---

## 核心一句话

| 文件 | Crate 类型 | 产物 | 特点 |
|------|------------|------|------|
| **`src/main.rs`** | **Binary crate** | exe 可执行程序 | 必须有 `fn main()`，可 `cargo run` |
| **`src/lib.rs`** | **Library crate** | 库文件（`.rlib` 等） | 无 `main`，靠 **`pub`** 暴露接口，供 `use` |

> 一个 Cargo 项目（Package）可以**同时拥有 `main.rs` + `lib.rs`**：既有可执行程序，又有可被本包 / 外部引用的库。

---

## 场景 1：只有 `main.rs` → 纯二进制 crate

```
demo/
├ Cargo.toml
└ src/main.rs
```

```rust
fn main() {
    println!("我编译后是 exe，直接运行");
}
```

| 命令 | 结果 |
|------|------|
| `cargo build` | `target/debug/demo.exe` |
| `cargo run` | 直接运行 |
| 外部 `use` | ❌ **不能被别的项目当库导入** |

demo → `only_bin/`

---

## 场景 2：只有 `lib.rs` → 纯库 crate

```
mylib/
├ Cargo.toml
└ src/lib.rs
```

```rust
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn sub(a: i32, b: i32) -> i32 {
    a - b // 无 pub → 私有，外部 use 报错
}
```

- `cargo build` → 只生成库，**无 exe，不能 `cargo run`**
- demo → `only_lib/`（包名 `only_lib_demo`）

### 新项目如何 `use` 这个库？

```
user_demo/
├ Cargo.toml
└ src/main.rs
```

`user_demo/Cargo.toml`：

```toml
[dependencies]
mylib = { path = "../mylib" }   # 本地路径依赖
```

`user_demo/src/main.rs`：

```rust
use mylib::add;

fn main() {
    println!("{}", add(10, 20));
    // use mylib::sub;  // ❌ sub 无 pub，私有
}
```

demo → `user_demo/`（`path = "../only_lib"`，库名 `only_lib_demo`）

```bash
cd user_demo && cargo run
```

## 场景 3：`main.rs` + `lib.rs` + 子模块（最常用）

```
app/
├ Cargo.toml
└ src/
   ├ lib.rs      // 库 crate：封装业务、统一导出
   ├ main.rs     // 二进制 crate：程序入口
   └ calc/
       └ mod.rs
```

### `src/calc/mod.rs`

```rust
pub fn mul(a: i32, b: i32) -> i32 {
    a * b
}

fn div(a: i32, b: i32) -> i32 {
    a / b // 私有
}
```

### `src/lib.rs`（库的对外总关口）

```rust
pub mod calc;

pub use calc::mul; // 可选：再导出，外部可 use app::mul
```

### `src/main.rs`

```rust
use app::calc::mul; // 包名 = Cargo.toml 的 name

fn main() {
    println!("{}", mul(5, 6));
}
```

- `cargo run` → 编译 bin crate 生成 exe 并运行
- 同包的 **`app` 库 crate** 也可被外部项目 `path` 依赖引入

demo → `bin_plus_lib/`（包名 `bin_plus_lib_demo`，含 `calc/` 子模块）

## 对外暴露规则（重点）

| 规则 | 含义 |
|------|------|
| **不加 `pub`** | 模块 / 函数 / 结构体仅当前 crate 内部可见，外部 `use` 报错 |
| **`pub mod xxx;`** | 整个模块对外开放 |
| **`pub fn xxx()`** | 单个函数对外开放 |
| **`lib.rs`** | 库 crate 的**导出总关口**——想对外暴露什么，在这里 `pub` 收拢 |

### 三种 `pub` 速查

| 写法 | 谁能用 |
|------|--------|
| `mod calc;` | 仅 lib crate 内部 |
| `pub mod calc;` | 同包 bin crate、外部依赖均可 `use 包名::calc` |
| 模块内 `pub fn mul()` | 函数公开；无 `pub` 仅模块内部 |

> bin crate（`main.rs`）与 lib crate（`lib.rs`）是**两个独立 crate**；`main` 只能访问 lib 里 **`pub` 出来的项**。

### 反面例子：`mod calc;` 无 pub

```bash
cd bin_plus_lib/compile_fail/private_mod
cargo check   # 预期 E0603: module `calc` is private
```

## 极简总结

| | main.rs | lib.rs |
|---|---------|--------|
| 类型 | Binary crate | Library crate |
| 产物 | exe，可运行 | 库，不能 run |
| 对外 | 不能被别人 `use` | 靠 `pub` 暴露，供别人 `use` |
