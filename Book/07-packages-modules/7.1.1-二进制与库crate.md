## 一、Crate 二选一

> Crate 只有两类：
> 1. **Binary crate**（有 `main`，编译出 exe）
> 2. **Library crate**（无 `main`，编译成库，供 `use`）

### 示例 1：Binary crate — 只有 `main.rs`

```
proj/
├ Cargo.toml
└ src/main.rs
```

```rust
fn main() {
    println!("我是二进制程序，运行生成 exe");
}
```

`cargo build` → `target/` 生成 exe。

demo → `only_bin/`

### 示例 2：Library crate — 只有 `lib.rs`

```
mylib/
├ Cargo.toml
└ src/lib.rs
```

```rust
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

`cargo build` → 库文件；**不能 `./` 运行**，只能被别的项目 `use`。

demo → `only_lib/`（包名 `only_lib_demo`）

---

## 二、`lib.rs` = 对外 API 出口（重点）

**`lib.rs` 是整个 lib crate 的根模块。**  
所有想给外部 `use` 的代码，必须在模块树里用 **`pub` 导出**；不 `pub` = 私有。

### Package 同时含 `lib.rs` + `main.rs` + 子模块（最常用）

```
myproj/
├ Cargo.toml
└ src/
   ├ lib.rs
   ├ main.rs
   └ math/
       └ mod.rs
```

#### `src/math/mod.rs`

```rust
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn sub(a: i32, b: i32) -> i32 {
    a - b // 无 pub → 仅 math 内部
}
```

#### `src/lib.rs`（统一收拢、对外暴露）

```rust
pub mod math;

// 可选简写：pub use math::add;
```

| 写法 | 含义 |
|------|------|
| `mod math;` | 引入模块，**私有** → 同包 `main`、外部项目都 `use` 不到 |
| `pub mod math;` | 模块**公开** → `use 包名::math::add` 可用 |

> **`pub mod math;`**：`mod math` 告诉编译器引入模块；**`pub`** 把模块对外开放。

#### `src/main.rs`

```rust
use myproj::math::add; // 包名 = Cargo.toml 的 name

fn main() {
    println!("{}", add(1, 2));
}
```

demo → `bin_plus_lib/`（包名 `bin_plus_lib_demo`，结构同上）

## 三、三种 `pub` 场景（速查）

1. **`mod math;`** — 模块私有，**只有 lib crate 内部**能用；外部 `main`、别的项目不能 `use`
2. **`pub mod math;`** — 模块公开，`use 包名::math` 全项目、外部依赖均可
3. **模块内 `pub fn add()`** — 函数公开；无 `pub` 只能模块内部调用

> bin crate（`main.rs`）与 lib crate（`lib.rs`）是**两个独立 crate**；`main` 只能访问 lib 里 **`pub` 出来的项**。

---

## 四、外部项目引用本库

新项目 `Cargo.toml`：

```toml
[dependencies]
myproj = { path = "../myproj" }
```

新项目代码：

```rust
use myproj::math::add;
```

**能跨项目导入，全靠 `lib.rs` 里的 `pub` 导出链。**

demo → `user_demo/`（`path = "../only_lib"`，调用 `only_lib_demo::add`）

```bash
cd user_demo && cargo run   # 30
```

## 五、反面例子：去掉 `pub` 编译报错

`lib.rs` 写成 `mod math;`（无 `pub`），`main.rs` 里 `use 包名::math::add`：

```text
error[E0603]: module `math` is private
```

```bash
cd bin_plus_lib/compile_fail/private_mod
cargo check   # 预期失败
```

## 极简总结

| | main.rs | lib.rs |
|---|---------|--------|
| 类型 | Binary crate | Library crate |
| 产物 | exe，可运行 | 库，不能 run |
| 对外 | 不能被别人 `use` | 靠 `pub` 暴露，供别人 `use` |
