# 7.1.1 二进制 crate 与库 crate

## 一、核心对照

| | `main.rs` | `lib.rs` |
|---|-----------|----------|
| Crate 类型 | **Binary** | **Library** |
| 产物 | exe，可 `cargo run` | 库，不能 run |
| 对外 | ❌ 不能被别的项目 `use` | ✅ 靠 `pub` 暴露 API |

> 同一 Package 可**同时**有 `main.rs` + `lib.rs` → 既有 exe，又有可复用库。

---

## 二、示例 1：纯 binary（只有 `main.rs`）

```
proj/
├ Cargo.toml
└ src/main.rs
```

```rust
fn main() {
    println!("我是二进制程序，运行生成 exe");
}
```

| 命令 | 结果 |
|------|------|
| `cargo build` | `target/debug/*.exe` |
| `cargo run` | 直接运行 |
| 外部 `use` | ❌ |

demo → `only_bin/`

---

## 三、示例 2：纯 library（只有 `lib.rs`）

```
mylib/
├ Cargo.toml
└ src/lib.rs
```

```rust
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn sub(a: i32, b: i32) -> i32 {
    a - b // 无 pub → 私有
}
```

| 命令 | 结果 |
|------|------|
| `cargo build` | 库文件 |
| `cargo run` | ❌ 无 `main` |

demo → `only_lib/`（包名 `only_lib_demo`）

## 四、示例 3：`lib.rs` + `main.rs` + 子模块（最常用）

**`lib.rs` = lib crate 对外 API 总关口**（不是 Package 出口 → [7.1.5](./7.1.5-lib.rs是Crate出口.md)）：想给外部 `use` 的，必须 `pub` 导出。

```
myproj/
├ Cargo.toml
└ src/
   ├ lib.rs
   ├ main.rs
   └ math/
       └ mod.rs
```

### 三文件分工

| 文件 | 职责 |
|------|------|
| `math/mod.rs` | 业务函数 |
| `lib.rs` | `pub mod math;` 统一收拢导出 |
| `main.rs` | 程序入口，`use 包名::math::add` |

```rust
// math/mod.rs
pub fn add(a: i32, b: i32) -> i32 { a + b }
fn sub(a: i32, b: i32) -> i32 { a - b }

// lib.rs
pub mod math;
// pub use math::add;  // 可选：缩短外部路径

// main.rs
use myproj::math::add;
fn main() { println!("{}", add(1, 2)); }
```

demo → `bin_plus_lib/`（包名 `bin_plus_lib_demo`）

---

## 五、`pub` 三种场景（速查）

| 写法 | 谁能用 |
|------|--------|
| `mod math;` | 仅 **lib crate 内部** |
| `pub mod math;` | 同包 **bin**、**外部依赖** 均可 `use 包名::math` |
| 模块内 `pub fn add()` | 函数公开；无 `pub` 仅模块内部 |

> `main.rs` 与 `lib.rs` 是**两个 crate**；bin 只能访问 lib 里 **`pub` 出来的项**。

## 六、外部项目 `path` 依赖

`user_demo/Cargo.toml`：

```toml
[dependencies]
mylib = { path = "../mylib" }
```

`user_demo/src/main.rs`：

```rust
use mylib::add;
fn main() { println!("{}", add(10, 20)); }
// use mylib::sub;  // ❌ 无 pub
```

跨项目导入，全靠 **A 包 `lib.rs` 里 `pub` 导出链**（见 [7.1.5](./7.1.5-lib.rs是Crate出口.md)）。

demo → `user_demo/`（依赖 `../only_lib`）

## 小结

| 要点 | 一句 |
|------|------|
| bin | `main.rs` → exe |
| lib | `lib.rs` → 库 + `pub` 出口 |
| 同包 | bin 自动依赖 lib，`use 包名::…` |
| 跨包 | `path` / crates.io + lib 侧 `pub` |
