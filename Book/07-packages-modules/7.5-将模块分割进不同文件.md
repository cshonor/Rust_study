## 一句话分清两种写法

| | 写法 1：内嵌（书本示例） | 写法 2：分文件（工程常用） |
|---|-------------------------|---------------------------|
| 语法 | `mod xxx { … 代码 … }` | `mod xxx;` + 独立 `.rs` / 文件夹 |
| 物理 | **全挤在一个 `.rs`** | **文件夹 + 零散 `.rs`** |
| 模块树 | **完全一样** | **完全一样** |

> **逻辑一模一样，只是代码放在磁盘上的方式不同。**

---

## 写法 1：全写在同一个文件里（书本，了解即可）

```rust
// lib.rs 或 main.rs（crate root）内
mod front_of_house {
    mod hosting {
        fn add_to_waitlist() {}
    }
    mod serving {
        fn take_order() {}
    }
}
```

纯 **`mod 名字 { }` 大括号嵌套**，不拆磁盘文件。

→ 餐厅完整树见 [7.2.1](./7.2.1-crate-root与模块树.md)

---

## 写法 2：拆成文件夹 + 零散 `.rs`（日常项目）

```
src/
├ lib.rs              // crate root
└ front_of_house/
   ├ mod.rs           // front_of_house 模块入口
   ├ hosting.rs
   └ serving.rs
```

### `front_of_house/mod.rs`

```rust
// 等价于书本里 front_of_house 内部的 mod hosting { }、mod serving { }
pub mod hosting;
pub mod serving;
```

> `pub mod hosting;` = 书本里 `mod hosting { … }`，只是函数体挪到 **`hosting.rs`**。

### `lib.rs`（crate root）

```rust
// 等价书本最外层 mod front_of_house { … }
pub mod front_of_house;
```

### 两种写法对应的模块树（相同）

```
crate
 └── front_of_house
     ├── hosting
     └── serving
```

---

## 日常工作流程 ↔ 书本语法翻译

| 工程操作 | 书本等价 |
|----------|----------|
| 新建文件夹 `front_of_house/` | `mod front_of_house {` |
| `mod.rs` 里 `pub mod hosting; pub mod serving;` | 括号内的 `mod hosting; mod serving;` |
| `hosting.rs` / `serving.rs` 放代码 | 大括号里的函数体拆出去 |

---

## 调用方式完全一致

```rust
crate::front_of_house::hosting::add_to_waitlist();
// 或 use front_of_house::hosting::add_to_waitlist;
```

---

## `mod xxx;` 文件对应规则（速查）

| 声明位置 | 编译器找的文件 |
|----------|----------------|
| crate root：`mod front_of_house;` | `src/front_of_house.rs` **或** `src/front_of_house/mod.rs` |
| `front_of_house/mod.rs`：`mod hosting;` | `src/front_of_house/hosting.rs` |

> **`front_of_house.rs` 与 `front_of_house/mod.rs` 二选一**，不要同时存在。

---

## 小总结

| 语法 | 含义 |
|------|------|
| `mod xxx { 代码 }` | 内嵌写法（单文件） |
| `mod xxx;` | 分文件写法（工程常用） |

**功能、模块树、路径没有半点区别。**
