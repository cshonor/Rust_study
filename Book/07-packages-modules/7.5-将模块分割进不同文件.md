# 7.5 将模块分割进不同文件

当模块逐渐变大时，可以把模块的定义拆到多个文件里，让结构更清晰；**模块树不变**，调用路径也可以保持不变。

对应 demo 项目：`Book/07-packages-modules/7.5-split-modules-demo/`

main 调 `a.rs` 的两种结构（有 lib / 无 lib）→ [7.1.2 main 调用分文件模块](./7.1.2-main调用分文件模块.md)。

---

## `mod xxx;`：告诉编译器去另一个文件找模块内容

在 crate root（`src/lib.rs` 或 `src/main.rs`）里写：

```rust
mod front_of_house;
```

末尾的分号表示：`front_of_house` 的内容不在花括号里，而在：

- `src/front_of_house.rs`

---

## 继续拆子模块：`mod.rs` 与文件夹

**文件夹 = 模块** 时，需要模块入口文件：

| 布局 | 入口文件 |
|------|----------|
| `src/domain/mod.rs` + `src/domain/*.rs` | `domain/mod.rs` 里 `pub mod email_client;` … |
| `src/front_of_house.rs` + `src/front_of_house/hosting.rs` | 父模块 `.rs` 里 `pub mod hosting;` |

Actix-web 项目完整范例 → [7.1.3 domain/mod.rs](./7.1.3-Actix-web项目目录范例.md#五domainmodrs-是什么不是-mobo)

如果在 `src/front_of_house.rs` 里写：

```rust
pub mod hosting;
```

同样末尾分号表示：`hosting` 的内容在：

- `src/front_of_house/hosting.rs`

---

## demo 结构

```text
src/
  lib.rs
  front_of_house.rs
  front_of_house/
    hosting.rs
```

`lib.rs` 里可以继续保持：

- `pub use crate::front_of_house::hosting;`
- `hosting::add_to_waitlist();`

---

## demo 运行

```bash
cd Book/07-packages-modules/7.5-split-modules-demo
cargo run
```
