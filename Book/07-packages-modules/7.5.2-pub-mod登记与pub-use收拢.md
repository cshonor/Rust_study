# 7.5.2 `pub mod user;` ≠ `use`：登记 vs 收拢

← [7.5.1 mod.rs 里的 mod 与 use](./7.5.1-mod.rs里的mod与use.md) · [7.4.1 use 仅限当前文件](./7.4.1-use作用域仅限当前文件.md)

对应 demo：`7.5-split-modules-demo/src/domain/`

---

## 两句大白话

| 写法 | 干什么 |
|------|--------|
| **`pub mod user;`** | **登记注册** — 把 `user.rs` 挂进模块树，生成路径 `domain::user` |
| **`pub use user::UserEntity;`** | **名字上浮** — 把类型提到 `domain` 出口，外面可 `domain::UserEntity` |

**二者都不能替同目录别的 `.rs` 自动导入名字。**

---

## 目录

```
domain/
├── mod.rs
├── user.rs
└── order.rs
```

---

## 1. `pub mod user;`（登记注册）

写在 `domain/mod.rs`：

```rust
pub mod user;
```

| # | 作用 |
|---|------|
| 1 | **告诉编译器：把同目录 `user.rs` 编入 `domain` 模块树** |
| 2 | `pub` → 外部可通过 `crate::domain::user` 访问该子模块 |
| 3 | **只注册文件、生成路径** — **不会**给同文件夹所有 `.rs` 自动导入名字 |

`order.rs` 想用 `user` 里的类型，**必须自己手写 `use`**：

```rust
use crate::domain::user::UserEntity;
```

> `pub mod user;` = 挂靠文件、建档；**≠** 给兄弟文件发通行证。

---

## 2. `pub use user::UserEntity;`（提升到 domain 出口）

```rust
pub mod user;
pub use user::UserEntity;
```

| | 路径 |
|---|------|
| **原本** | `domain::user::UserEntity` |
| **`pub use` 后** | 名字上浮到 **`domain` 一级** → `use crate::domain::UserEntity;` |

企业项目常用：**内部层级藏起来，对外收拢 API**。

> `pub use` 是把名字挂到**当前模块（domain）的出口**；别的文件仍须自己 `use crate::domain::UserEntity` — 不是全局自动可用（→ [7.4.1](./7.4.1-use作用域仅限当前文件.md)）。

---

## 3. 最大误区纠正

| | |
|---|---|
| ❌ | `pub mod user;` 写完，本文件夹所有 `.rs` 自动能 `use user::…` |
| ✅ | **每个 `.rs` 都是独立作用域** — 谁想用谁自己 `use` |

- **`mod.rs`**：可以 `use self::user::UserEntity;` 或靠 `pub use` 短名
- **`user.rs` / `order.rs`**：不受 `mod.rs` 里 `pub mod` 影响，**不会自动导入**

---

## 极简总结

| # | 规则 |
|---|------|
| 1 | **`pub mod user;`**：挂载 `user.rs` 进模块树（建档） |
| 2 | **`pub use user::X;`**：把 `X` 提到当前 `domain` 模块出口（收拢接口） |
| 3 | 同目录别的 `.rs`，**二者都不能**帮它省掉自己的 `use` |

---

## 实操：`order.rs`

```rust
// 哪怕 mod.rs 写了 pub mod user，这里仍要手写路径
use crate::domain::user::UserEntity;

pub struct OrderEntity;

impl OrderEntity {
    pub fn with_user(_user: &UserEntity) -> Self {
        Self
    }
}
```

若 `mod.rs` 还有 `pub use user::UserEntity`，**order.rs 仍可选择**：

- `use crate::domain::user::UserEntity;`（走子模块路径），或
- `use crate::domain::UserEntity;`（走 `pub use` 上浮后的出口）

两种都对；**都必须在本文件自己写 `use`**。
