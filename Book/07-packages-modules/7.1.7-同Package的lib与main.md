# 7.1.7 同 Package：lib + main = 两个 Crate

← 主笔记：[7.1](./7.1-包和crate.md) · [7.1.1 bin/lib](./7.1.1-二进制与库crate.md) · demo：`bin_plus_lib/` · [7.1.5 lib 出口](./7.1.5-lib.rs是Crate出口.md)

---

## 结论一句话

> **同一个 Package，可以同时存在：1 个 Lib Crate + 1 个默认 Bin Crate（`main.rs`）——两个互相独立、两个不同的 Crate。**

**1 个 Package = 容纳两类 Crate，互不冲突。**

---

## 一、一份项目，两份编译产物

```
demo_package/
├ Cargo.toml
└ src/
   ├ lib.rs    → 【库 crate】→ 库文件，**不出 exe**
   └ main.rs   → 【二进制 crate】→ **exe**
```

| 文件 | Crate 类型 | 产物 | 作用 |
|------|------------|------|------|
| `lib.rs` | Library | `.rlib` 等 | 供外部 / 同包 `main` `use` |
| `main.rs` | Binary | exe | 本地 `cargo run` 跑程序 |

### main 怎么用 lib？

```rust
// main.rs — bin crate
use demo_package::math::add;

fn main() {
    println!("{}", add(1, 2));
}
```

- bin crate **链接**同 Package 的 lib crate
- **形态 1 打包**：最终 **1 个 exe**，lib 代码链进 main 进程（见 [14.3.1](../14-cargo-crates/14.3.1-Workspace与微服务.md)）

### 对外视角

| 谁 | 用什么 |
|----|--------|
| **别的项目** `path` / crates.io 依赖 | 只用 **lib**（`use demo_package::…`） |
| **本地开发** | `cargo run` → **main 编出的 exe** |

demo → `7.1-packages-crates-demo/bin_plus_lib/`

## 二、两个 Crate 隔离，互不隶属

| | lib crate | bin crate（main） |
|---|-----------|-------------------|
| 根文件 | `lib.rs` | `main.rs` |
| 对外暴露 | `pub` 控制 **lib 接口** | **不**对外当库暴露 |
| 外部依赖能拿到 | `lib.rs` 里 `pub` 的项 | ❌ **main 里的函数外部永远拿不到** |

```rust
// lib.rs
pub fn calc() {}

// main.rs
fn run_app() {}   // 仅 bin crate 内部
```

别的 Package 依赖本项目：

```rust
use demo_package::calc;   // ✅
// use demo_package::run_app;  // ❌ 不存在于 lib 导出
```

> **main 是启动壳；lib 是可复用 API。** 业务写 lib，main 只做组装（工业最常用）→ [7.1.3 Actix-web 范例](./7.1.3-Actix-web项目目录范例.md)

## 三、规则巩固

| # | 规则 |
|---|------|
| 1 | 单 Package：**最多 1 个 lib crate**（`src/lib.rs`）→ [7.1.6](./7.1.6-一个Package一个lib.rs.md) |
| 2 | 单 Package：**可 N 个 bin crate**（`main.rs` + `src/bin/*.rs`）→ [7.1.4](./7.1.4-src-bin与多exe.md) |
| 3 | **lib + main 共存**：最常用；逻辑在 lib，入口在 main |

---

## 四、极简总结

| | |
|---|---|
| 有 lib + main | 一个 Package → **两种产物：库 + exe** |
| 引依赖 | 用 **lib** |
| 跑服务 | 用 **main 编出的 exe** |

### 口诀

**lib 对外开门，main 对内开机；同 Package 两 crate，各干各的。**

---

## 与 Workspace 的区别

同 Package 的 lib + main：**一个文件夹里两个 crate**。  
Workspace 多 Package：多个文件夹，每个 Package 仍可各有 lib + main → [14.3](../14-cargo-crates/14.3-Cargo工作空间.md)
