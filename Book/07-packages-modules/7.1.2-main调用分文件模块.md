# 7.1.2 main 如何调用 `a.rs` 里的函数

← 主笔记：[7.1 包与 Crate](./7.1-包和crate.md) · [7.1.1 bin/lib](./7.1.1-二进制与库crate.md) · demo：[7.1-packages-crates-demo](./7.1-packages-crates-demo/) · 分文件细则 → [7.5](./7.5-将模块分割进不同文件.md)

---

## 结论（两种结构，一句话）

| 结构 | 口诀 |
|------|------|
| **有 `lib.rs` + `main.rs`** | **有 lib 过 lib**：在 `lib.rs` 里 `mod` + `pub` 导出，`main` 从包名 `use` |
| **只有 `main.rs`（纯 bin）** | **无 lib 直接 mod 在 main**：在 `main.rs` 里写 `mod a;` |

---

## 结构 1：`lib.rs` + `main.rs` + `a.rs`（推荐）

`main` 要调 `a.rs` 里的函数：**必须先在 `lib.rs` 注册模块并导出**，`main` 再 `use 包名::…` 调用。

```
proj/
├ Cargo.toml
└ src/
   ├ a.rs
   ├ lib.rs
   └ main.rs
```

### `src/a.rs`

```rust
pub fn hello() {
    println!("来自 a.rs");
}
```

### `src/lib.rs`（关键：注册 + 对外开放）

```rust
mod a;              // 注册 a 模块，编译器收录 src/a.rs
pub use a::hello;   // pub：暴露出去，main 才能 use
```

也可写成 `pub mod a;`，则 `main` 里 `use proj::a::hello;`。

### `src/main.rs`

```rust
use proj::hello; // 包名 = Cargo.toml 的 name

fn main() {
    hello();
}
```

> **工业标准写法**：业务代码放 lib 侧模块树，`lib.rs` 统一导出；`main` 只做入口，从包名导入。

demo → `via_lib/`

---

## 结构 2：只有 `main.rs` + `a.rs`（纯二进制）

没有 `lib.rs`，**不经过 lib**，直接在 **bin crate root（`main.rs`）** 注册：

```
proj/
├ Cargo.toml
└ src/
   ├ a.rs
   └ main.rs
```

### `src/main.rs`

```rust
mod a; // bin crate root 里注册 a.rs

fn main() {
    a::hello();
}
```

> `mod a;` 写在 **`main.rs`**（bin 的 crate root），不是 lib。

demo → `via_main_only/`

---

## 关键总结

| # | 规则 |
|---|------|
| 1 | **有 lib.rs**：一律在 lib 里 `mod` + `pub` 导出，`main` 从包名导入 |
| 2 | **无 lib.rs**：直接在 `main.rs` 写 `mod a;`，路径 `a::hello()` |
| 3 | `a.rs` 里函数不加 `pub`，无论怎么 `mod` / `pub use` 都访问不了 |

### 小口诀

**有 lib 过 lib，无 lib 直接 mod 在 main。**

---

## 和 crate 的关系（补一句）

- 结构 1：`lib.rs` 是 **lib crate root**，`main.rs` 是 **bin crate root**——两个 crate，`main` 通过包名访问 lib 里 `pub` 出来的项（见 [7.1.1](./7.1.1-二进制与库crate.md)）。
- 结构 2：只有 **bin crate**，`main.rs` 即 crate root，`mod a;` 挂到 bin 的模块树下。

`mod a;` 与 `src/a.rs` 文件对应规则 → [7.5](./7.5-将模块分割进不同文件.md)。
