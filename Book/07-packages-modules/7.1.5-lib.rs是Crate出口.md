# 7.1.5 `lib.rs` = 库 Crate 的出口（不是 Package 的出口）

← 主笔记：[7.1](./7.1-包和crate.md) · 前置：[7.1.1 bin/lib](./7.1.1-二进制与库crate.md) · Workspace 可见性 → [14.3.1](../14-cargo-crates/14.3.1-Workspace与微服务.md)

---

## 精准结论

> **`lib.rs` = 当前这个「库 Crate」的对外接口总出口，不是 Package 的出口。**

| 层级 | 定义 |
|------|------|
| **Package** | 文件夹 + `Cargo.toml`（打包 / 构建单位） |
| **Library Crate** | 一个 Package **最多 1 份** `src/lib.rs` |
| **`lib.rs`** | 只管 **本 lib crate 内部** 哪些模块/项 `pub` 对外开放 |

---

## 一、一句话理解

`lib.rs` 是 **库箱子的大门**：

- 箱子内部（`order/`、`domain/`、各 `.rs` 子模块）藏在 crate 模块树里；
- 想让 **crate 外面**（同 Package 的 bin、别的 Package、`tests`）拿到函数/结构体 → 必须在 `lib.rs` 用 **`pub` / `pub use` 放行**。

### 示例

```
mylib/                 ← Package
├ Cargo.toml
└ src/
   ├ lib.rs            ← **lib crate** 出入口（不是 Package 出入口）
   └ order/
      └ mod.rs
```

`src/order/mod.rs`：

```rust
pub fn create_order() {}
```

`src/lib.rs`：

```rust
mod order;                       // 收进本 crate，外部不可见
pub use order::create_order;     // 放行：对外暴露
// 或：pub mod order;
```

外部依赖 `mylib` 后：

```rust
use mylib::create_order;
```

**`order/` 文件夹本身不能直接对外**——全靠 `lib.rs` 导出链。

---

## 二、`lib.rs` 只管 Crate，不管 Package 里的二进制

一个 Package 可同时有：

| 文件 | Crate | 作用 |
|------|-------|------|
| `lib.rs` | **Library crate** | 代码导出、可被 `use` |
| `main.rs` | **Binary crate** | 独立 exe，**自有 crate root**，不受 `lib.rs`「管控」 |

同 Package 里 **`main` 要用 lib 代码**：

```rust
use 包名::create_order; // bin crate 去引入同 Package 的 lib crate
```

→ bin 与 lib 是 **两个 crate**；bin 只能看见 lib 里 **`pub` 出来的项**（同 [7.1.1 §五](./7.1.1-二进制与库crate.md)）。

---

## 三、精简总结

| | |
|---|---|
| ✅ | `lib.rs` → **Library Crate 的对外接口清单** |
| ❌ | `lib.rs` **不代表** Package 接口；Package 只是文件夹 + Cargo 配置 |
| 规则 | 不加 `pub` → crate 外不可见；`pub` / `pub use` → 对外开放 |

## 四、联动 Workspace（A lib ← B bin 依赖）

```
workspace/
├ package_a/    lib only（lib.rs）
└ package_b/    bin（main.rs，path 依赖 a）
```

`package_b/Cargo.toml`：

```toml
[dependencies]
package_a = { path = "../package_a" }
```

**B 能用的内容，严格限于 A 的 `lib.rs` 里 `pub` 导出的项**——A 内部未 `pub` 的函数/模块，B **完全看不见**（编译期报错，不是运行时限制）。

demo → [14.3-packaging-demo](../14-cargo-crates/14.3-packaging-demo/)（`common` lib 被 `app_bin` / `feed` / `trade` 链接）

形态 1（链进单 exe）vs 形态 2（多 exe + IPC）→ [14.3.1](../14-cargo-crates/14.3.1-Workspace与微服务.md)

## 五、与 module 的关系（补一句）

- **`domain/mod.rs`** → 收拢 **domain 模块**内部（见 [7.1.3](./7.1.3-Actix-web项目目录范例.md)）
- **`lib.rs`** → 收拢 **整个 lib crate** 对谁公开

三层都要 `pub` 才能从 crate 外 `use 包名::domain::…`。

多库误区 → [7.1.6 一个 Package 一个 lib.rs](./7.1.6-一个Package一个lib.rs.md)

## 小结

| 谁问 | 答 |
|------|-----|
| `lib.rs` 管什么？ | **本 lib crate** 的公开 API |
| Package 有几个 `lib.rs`？ | **最多 1 个** |
| `main.rs` 呢？ | 另一个 **bin crate**，不是 `lib.rs` 的子模块 |
| Workspace B 依赖 A lib？ | 只能 `use` A 的 **`lib.rs` 已 pub 的项** |
