# 7.2.2 结构体与字段私有性

← 主笔记：[7.2 模块与私有性](./7.2-定义模块与控制作用域与私有性.md) · demo：[7.2-modules-privacy-demo](./7.2-modules-privacy-demo/) · 结构体基础 → [5.1](../05-structs/5.1-定义并实例化结构体.md)

---

## 一句话结论

> **结构体、字段默认全私有：同模块内能读写；跨模块必须字段加 `pub` 才能访问、修改。**

---

## 1. 规则拆开讲

Rust **权限最小原则**：

| # | 规则 |
|---|------|
| 1 | **`struct` 无 `pub`** → 结构体类型仅本模块可见 |
| 2 | **字段天生默认私有**，与 `struct` 是否 `pub` **无关** |

```rust
struct User {
    id: u64,
    name: String,
}
```

---

### ① 同一模块内：私有字段随便改

```rust
struct User {
    id: u64,
    name: String,
}

fn test() {
    let mut u = User { id: 1, name: "abc".into() };
    u.id = 999;              // ✅ 同模块
    u.name = "新名字".into();
}
```

> **私有只限制跨模块**；本模块内部无额外限制。

---

### ② 跨模块：字段无 `pub` 不能读写

```rust
pub mod user_mod {
    pub struct User {
        id: u64,      // 私有字段
        name: String,
    }
}

fn outer() {
    let mut u = user_mod::User { id: 1, name: "".into() };
    u.id = 100; // ❌ E0616：字段私有
}
```

字段单独 `pub` 才允许跨模块读写：

```rust
pub struct User {
    pub id: u64,   // 跨模块可读可改
    name: String,  // 仍私有
}
```

---

## 2. 两个高频误区

### 误区 1：`pub struct` ≠ 字段 `pub`

`pub struct User` 只表示 **别的模块能写类型名、能构造**（若字段也可见或可 `new`）；**字段默认仍私有**，不能直接 `u.id = …`。

### 误区 2：封装 — 能读不能改

不公开字段，用 **`impl` 方法** 控制：

```rust
pub struct User {
    id: u64,
}

impl User {
    pub fn set_id(&mut self, val: u64) {
        self.id = val;
    }
    pub fn get_id(&self) -> u64 {
        self.id
    }
}
```

外部：`u.set_id(123)` ✅ · `u.id = 123` ❌

→ 方法语法 [5.3](../05-structs/5.3-方法语法.md)

---

## 3. 与项目分层

| 层 | 做法 |
|----|------|
| **domain 内部** | 同模块私有字段随便改 |
| **routes 外层** | 字段 `pub`，或只暴露 `get` / `set` 方法 |

---

## 4. 精简总结

| 场景 | 私有字段 |
|------|----------|
| **同模块** | 随便读、写、改 |
| **跨模块** | 看不见 / 不能改；**字段 `pub`** 才允许外部读写 |
