## 核心结论（先背）

1. **`fn` 函数指针**：具体类型，**不捕获环境**；实现 `Fn`/`FnMut`/`FnOnce`
2. **闭包 trait**：`Fn` · `FnMut` · `FnOnce` — 由捕获方式决定
3. **返回闭包**：`impl Fn(...)` 或 `Box<dyn Fn(...)>`
4. **高阶函数**：`apply(f, x)` · `map`/`filter` 接收函数或闭包

---

## 一、核心概念

### 函数指针 `fn`

- 类型写作 `fn(i32) -> i32`（小写 `fn`，区别于 trait `Fn`）
- **不捕获环境** — 与闭包本质区别
- 实现了 `Fn`/`FnMut`/`FnOnce` → 多数 `F: Fn` API 也能传函数指针

### 闭包三个 trait

| Trait | 调用 | 捕获 |
|-------|------|------|
| `Fn` | `&self` | 共享借用 / Copy |
| `FnMut` | `&mut self` | 可变借用 |
| `FnOnce` | `self` | 移动捕获（消耗） |

编译器为每个闭包生成唯一匿名类型，自动 impl 满足条件的 `Fn*`。

### 函数/闭包作为返回值

闭包类型匿名 → 不能直接写 `-> Fn(i32) -> i32`。

| 方式 | 特点 |
|------|------|
| `impl Fn(i32) -> i32` | 静态分发，**单一**闭包类型 |
| `Box<dyn Fn(i32) -> i32>` | 动态分发，可返回不同闭包 |

### 高阶函数

接收函数/闭包作为参数：`do_twice(f, arg)`、`iter.map(f)`。

## 二、代码示例

```rust
fn add_one(x: i32) -> i32 { x + 1 }

fn apply(f: fn(i32) -> i32, x: i32) -> i32 {
    f(x)
}

fn make_adder(x: i32) -> impl Fn(i32) -> i32 {
    move |y| x + y
}

fn returns_closure() -> Box<dyn Fn(i32) -> i32> {
    Box::new(|v| v + 1)
}

fn main() {
    println!("{}", apply(add_one, 5));      // 函数指针：6
    let add_two = make_adder(2);
    println!("{}", add_two(3));              // impl Fn：5
}
```

`map(ToString::to_string)` · 构造器作函数 · `Box<dyn Fn>` → demo `main.rs`

## 三、分节详解

### `map` 传函数 vs 闭包

```rust
.map(|i| i.to_string())      // 闭包
.map(ToString::to_string)    // 方法项（函数指针风格）
.map(Wrapper)                // 元组结构体构造器
.map(Status::Value)          // 枚举变体构造器
```

### API 设计建议

- 优先 `F: FnMut(...) -> ...` — 兼收闭包与函数指针
- 仅 FFI 等场景限定 `fn(...) -> ...`

### 返回闭包选型

- 同一逻辑、求性能 → `impl Fn`
- 运行时返回不同闭包 → `Box<dyn Fn>`

## 四、易错点

| # | 易错 | 正确理解 |
|---|------|----------|
| 1 | `fn` 与 `Fn` 混淆 | `fn` = 函数**指针类型**；`Fn` = **trait** |
| 2 | 闭包一定 impl 三个 Fn* | 只 impl **满足捕获方式**的子集 |
| 3 | `impl Fn` 返回多种闭包 | `impl Trait` 只能**单一**具体类型 → 用 `Box<dyn Fn>` |
| 4 | `Box<dyn Fn>` 零开销 | 有 vtable **动态分发**开销 |
| 5 | 闭包捕获与 `'static` | `thread::spawn` 等要求 `Send + 'static`，注意捕获生命周期 |

---

## 五、速记

1. `fn` 不捕获；闭包捕获 → 决定 `Fn*`
2. 高阶：`fn(f: fn(...))` 或 `F: Fn`
3. 返回闭包：`impl Fn`（单态）/ `Box<dyn Fn>`（异构）
4. `map` 可传闭包、函数项、构造器
