## 核心结论（先背）

1. **`fn` 函数指针**：具体类型，**不捕获环境**；实现了 `Fn`/`FnMut`/`FnOnce`
2. **闭包 trait**：`Fn`（`&self`）· `FnMut`（`&mut self`）· `FnOnce`（消耗 `self`）— 由捕获方式决定
3. **API 设计**：优先 `F: Fn...` 泛型（兼收函数指针与闭包）；仅 FFI 等场景限定 `fn`
4. **返回闭包**：匿名类型不能直接写 → `Box<dyn Fn(...)>` 或 `impl Fn(...)`
5. **高阶函数**：`map`/`filter` 等接收函数项或闭包

## 函数指针（function pointer）：`fn`

函数也可以作为参数传递，它的类型写作 `fn(...) -> ...`（小写 `fn`，避免与闭包 trait `Fn` 混淆）。

示例要点：

- `fn` 是**具体类型**，不是 trait
- **函数指针实现了 `Fn`/`FnMut`/`FnOnce`**，因此很多“接收闭包”的 API 也能直接传普通函数
- 设计 API 时通常更倾向于接收泛型 `F: Fn...`，这样既能接收闭包也能接收函数指针；只有在确实只想接收函数指针（例如 FFI 场景）才用 `fn`

---

## `map` 里传“函数”而非闭包

`Iterator::map` 既可以用闭包：

```rust
.map(|i| i.to_string())
```

也可以传函数/方法项：

```rust
.map(ToString::to_string)
```

这里用到了完全限定语法风格的“方法项”写法：指明使用 `ToString` trait 的 `to_string`。

另一个常见技巧是用“构造器函数”：

- 元组结构体构造器（如 `Foo(x)`）在需要 `Fn(T)->U` 的地方可当作函数传递
- 枚举的元组变体构造器（如 `Status::Value`）同理

## 闭包 trait：`Fn` / `FnMut` / `FnOnce`

| Trait | 调用方式 | 典型捕获 |
|-------|----------|----------|
| `Fn` | `&self` | 不可变借用或 Copy |
| `FnMut` | `&mut self` | 可变借用 |
| `FnOnce` | `self`（消耗） | 移动捕获 |

- 闭包具体类型由编译器生成，自动实现满足条件的 `Fn*` trait
- 需要 `'static` 等 bound 时（如 `thread::spawn`），注意捕获的生命周期
- 函数指针**不捕获**，因此同时实现三个 trait

---

## 返回闭包：用 trait object

闭包的具体类型是编译器生成的匿名类型，不能直接写成返回值类型：

```rust
// fn returns_closure() -> Fn(i32) -> i32 { ... } // 不允许
```

解决方案是返回 trait object（DST 放在指针后）：

```rust
fn returns_closure() -> Box<dyn Fn(i32) -> i32> {
    Box::new(|x| x + 1)
}
```
