## 核心结论（先背）

1. **关联类型**：trait 内占位符，每类型**只能 impl 一次**（如 `Iterator::Item`）
2. **泛型 trait**：同一类型可**多次 impl**（不同泛型参数）
3. **默认泛型参数**：`trait Add<Rhs = Self>`
4. **完全限定语法**：`<Type as Trait>::method()` 消歧义
5. **supertrait**：`trait T: Debug` — 实现 T 须先满足父 trait
6. **newtype**：绕过孤儿规则 — **零运行时开销**

---

## 一、核心概念

### 关联类型 vs 泛型 trait

| | 关联类型 | 泛型 trait |
|--|----------|------------|
| 写法 | `type Item;` | `trait Iter<T>` |
| 同一类型 impl 次数 | **一次** | **可多次** |
| 调用处 | 无需再指明 Item | 可能需要类型标注 |
| 适用 | 「每个实现只有一种关联类型」 | 同一类型对不同 T 有不同 impl |

### 默认泛型参数

`trait Add<Rhs = Self>` — 多数 `impl Add for Point` 用默认 RHS；`impl Add<Meters> for Millimeters` 显式指定。

### 完全限定语法（Fully Qualified Syntax）

- 同名方法：`Pilot::fly(&person)` vs `person.fly()`（类型自身方法）
- 无 `self` 的关联函数：`<Dog as Animal>::baby_name()` — 编译器无法从接收者推断

### Supertrait（父 trait）

```rust
trait OutlinePrint: fmt::Display { /* 可用 to_string() */ }
```

只有同时实现 `Display` 的类型才能 impl `OutlinePrint`。

### Newtype 模式

孤儿规则：trait 与类型**至少一个**须在本地 crate。用 `struct Wrapper(Vec<String>)` 包装外部类型 → 可 `impl Display for Wrapper`。

## 二、代码示例

```rust
use std::fmt;

// 关联类型
trait MyIterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}

// 默认泛型参数（运算符重载）
trait Add<Rhs = Self> {
    type Output;
    fn add(self, rhs: Rhs) -> Self::Output;
}

// Newtype + 外部 trait
struct Wrapper(String);
impl fmt::Display for Wrapper {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "Wrapped: {}", self.0)
    }
}

fn main() {
    let w = Wrapper("hello".into());
    println!("{w}");
}
```

消歧义与 supertrait 完整示例 → `19.2-advanced-traits-demo/src/main.rs`（`Pilot`/`Wizard`/`Human`、`OutlinePrint` 等）。

## 三、分节详解

### 关联类型（以 Iterator 为例）

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

为何不用 `Iterator<T>`？关联类型保证**每类型只 impl 一次**，调用 `next()` 无需再选 `T`。

### 运算符重载

`+` 对应 `std::ops::Add`；`Millimeters + Meters` 需 `impl Add<Meters> for Millimeters`。

### Newtype 权衡

- **零运行时开销**（编译期优化，与直接使用内部类型等价）
- 需手动转发 API，或 `Deref` 部分暴露
- `#[repr(transparent)]` 用于 **FFI/ABI 布局**与 C 类型对齐，**不是**为消除性能开销

## 四、易错点

| # | 易错 | 正确理解 |
|---|------|----------|
| 1 | 关联类型与泛型 trait 随意混用 | 关联类型 = 每 impl **一种**；泛型 = 可**多次** impl |
| 2 | 完全限定语法只写 trait 名 | 须 `<Type as Trait>::func()` **同时指定类型和 trait** |
| 3 | newtype 有性能开销 | **零运行时开销**；开销在手动转发 API 的开发成本 |
| 4 | 忘记 supertrait 约束 | `impl OutlinePrint for T` 前须已 `impl Display for T` |
| 5 | `Dog::baby_name()` 调 Animal 默认 | 无 `self` 时须 `<Dog as Animal>::baby_name()` |

---

## 五、速记

1. 关联类型 **一次 impl**；泛型 trait **可多次**
2. `Add<Rhs = Self>` — 默认泛型参数
3. `<T as Tr>::f()` — 完全限定消歧义
4. `trait T: Parent` — supertrait
5. newtype — 孤儿规则 + **零成本**抽象
