## 核心结论（先背）

1. **关联类型**：trait 内占位符，每类型**只能 impl 一次**（如 `Iterator::Item`）
2. **泛型 trait**：同一类型可对不同泛型参数**多次 impl**，调用处可能需类型标注
3. **默认泛型参数**：`trait Add<Rhs = Self>` — 多数 impl 用默认，特殊场景显式指定
4. **完全限定语法**：`<Type as Trait>::method()` — 消歧义
5. **supertrait**：`trait T: Debug` — 实现 T 须先满足父 trait
6. **newtype**：包装外部类型 → 绕过孤儿规则 impl 外部 trait

## 关联类型（Associated Types）

→ **入门（`type` 两种场景 · vs 泛型 Trait · vs struct）**：[10.2.4 type 关键字](../10-generics-traits-lifetimes/10.2.4-type关键字与关联类型.md)

在 trait 内定义一个类型占位符，让实现者在 `impl` 时指定具体类型。

典型例子：`Iterator`（简化）

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

### 为何不用 `Iterator<T>` 这种泛型 trait？

- **泛型参数 trait**：允许同一类型对不同 `T` **多次实现** `Iterator<T>`，调用处可能需要额外类型标注来选中哪一个实现。
- **关联类型 trait**：同一类型对该 trait **只能实现一次**，因此调用 `next()` 时无需再指明 “我要哪个 `T`”。

---

## 默认泛型类型参数 + 运算符重载（Operator Overloading）

很多运算符在 Rust 里通过实现标准库 `std::ops::*` trait 来“重载”。例如 `+` 对应 `Add`：

```rust
trait Add<RHS = Self> {
    type Output;
    fn add(self, rhs: RHS) -> Self::Output;
}
```

这里 `RHS = Self` 就是**默认类型参数**：

- **常见场景**：`Point + Point`，直接 `impl Add for Point`（使用默认 RHS）。
- **自定义 RHS**：`Millimeters + Meters`，写成 `impl Add<Meters> for Millimeters`，在实现里做单位换算。

默认类型参数常见目的：

- **扩展 trait 能力但不破坏旧代码**（新增类型参数并给默认值）。
- **让大多数实现更简洁**，只有少数特殊需求才需要显式指定。

## 完全限定语法（Fully Qualified Syntax）与消歧义

当多个 trait/类型实现了同名方法或关联函数时，需要告诉编译器“你要调用哪个”。

### 同名“方法”（带 `self`）的消歧义

若 `Human` 同时实现 `Pilot::fly`、`Wizard::fly`，并且自身也有 `Human::fly`：

- `person.fly()` 默认调用 **类型自身的方法**
- `Pilot::fly(&person)` / `Wizard::fly(&person)` 指定调用哪个 trait 的实现

### 同名“关联函数”（不带 `self`）的消歧义

当 trait 里的函数没有 `self`（比如 `Animal::baby_name()`），编译器无法从接收者推断实现者类型，需要完全限定语法：

```rust
<Type as Trait>::function(...)
```

例如：

```rust
<Dog as Animal>::baby_name()
```

## 父 trait（Supertrait）

当一个 trait 的默认方法实现需要依赖另一个 trait 的能力时，可声明父 trait 约束：

```rust
trait OutlinePrint: std::fmt::Display {
    fn outline_print(&self) { /* uses to_string() */ }
}
```

含义：**只有同时实现了 `Display` 的类型，才允许实现 `OutlinePrint`**。

## newtype 模式：为外部类型实现外部 trait

孤儿规则（orphan rule）限制：只有当 **trait 或类型至少一个在当前 crate 本地**时，才能 `impl Trait for Type`。

绕过方式：用元组结构体包装外部类型，得到“本地新类型”：

```rust
struct Wrapper(Vec<String>);
```

这样就能：

- `impl Display for Wrapper`（因为 `Wrapper` 是本地类型）
- 内部通过 `self.0` 访问被包装的 `Vec<String>`

权衡：

- newtype **不会引入运行时开销**（编译期可被优化掉）
- 但需要手动“转发”你想暴露的方法；或者实现 `Deref` 让它更像内部类型（是否暴露全部能力取决于你想要的封装边界）
