## 核心结论（先背）

1. **`type` 别名**：同义词，**不**创建新类型
2. **newtype struct**：**创建**新类型 — 类型安全（与别名不同）
3. **`!` never type**：永不返回，可 coerce 为任意类型
4. **DST**：`str`、`[T]`、`dyn Trait` — 须放指针后
5. **`Sized` / `?Sized`**：泛型默认 `T: Sized`
6. **ZST**：大小为 0，不占内存

---

## 一、核心概念

### 类型别名 `type`

`type Kilometers = i32` — 与 `i32` **同一类型**，可相加、可混传，**不**提供单位安全。

用途：缩短冗长签名（`type Thunk = Box<dyn Fn() + Send + 'static>`）、统一 `Result` 错误类型。

### newtype struct（与 19.2 交叉）

`struct Millimeters(u32)` — **新类型**，不能与 `Meters(u32)` 混用。见 19.2 newtype。

### 永不类型 `!`

empty type，无值。用于发散表达式：`panic!()`、无限 `loop {}`、`continue`/`break` 等。

关键：`!` 可 **coerce** 为任意类型 → `match` 分支类型一致。

### DST（动态大小类型）

编译期大小未知：`str`、`[T]`、`dyn Trait`。

**黄金规则**：DST 必须放在指针后 — `&str`、`Box<str>`、`&[T]`、`Box<dyn Trait>`。

### `Sized` / `?Sized`

- 绝大多数类型 `Sized`
- 泛型默认 `T: Sized`
- 接受 DST：`fn f<T: ?Sized>(t: &T)` — 参数通常为引用/指针

### ZST（零大小类型）

`()`、无字段 struct、`[T; 0]` — 大小 0，编译器可优化掉，仍参与类型系统。

## 二、代码示例

```rust
type Kilometers = i32;

fn diverges() -> ! {
    panic!("永远不会返回");
}

fn main() {
    let s: &str = "hello";           // str 是 DST → 用 &str
    let arr: &[i32] = &[1, 2, 3];    // [i32] 是 DST → 用 &[i32]

    let k: Kilometers = 10;
    let x: i32 = 5;
    println!("{}", x + k);           // 别名与 i32 可混算
}
```

`!` 参与 `match`、`continue` · `?Sized` · newtype · `Thunk` 别名 → `19.3-advanced-types-demo/src/main.rs`

## 三、分节详解

### newtype：类型安全 + 抽象

```rust
struct Millimeters(u32);
struct Meters(u32);
// takes_mm(Meters(1))  // E0308 — 不能混用
```

`People(HashMap<...>)` — 隐藏内部表示，只暴露 API。

### `!` 在 match 中

```rust
match result {
    Ok(v) => v,
    Err(_) => panic!("fail"),  // ! coerce 为 u32，与 Ok 分支一致
}
```

### DST 与 trait 对象

`&dyn Greeter` 是胖指针；`accepts_unsized::<str>(s)` 需 `T: ?Sized`。

## 四、易错点

| # | 易错 | 正确理解 |
|---|------|----------|
| 1 | `type` 与 newtype 混淆 | `type` = 同义词；`struct Wrapper(T)` = **新类型** |
| 2 | `!` 用于变量 | `!` 仅作**返回值/发散表达式**，不能 `let x: !` |
| 3 | `Vec<str>` 合法 | DST 不能作 `Vec` 元素 → `Vec<Box<str>>` 或 `Vec<String>` |
| 4 | 忘记 `?Sized` | 泛型默认 `Sized`，接受 `str`/`dyn Trait` 须 `?Sized` |
| 5 | `&str` 与 `str` 混淆 | `str` 是 DST；日常用的是 `&str`（指针+长度） |

---

## 五、速记

1. `type` = 别名；newtype = 新类型
2. `!` = 永不返回，coerce 为任意类型
3. DST 必须 `&` / `Box` / `Rc` 等指针后
4. `T: ?Sized` + `&T` 接受 DST
5. ZST 大小 0，类型系统仍参与
