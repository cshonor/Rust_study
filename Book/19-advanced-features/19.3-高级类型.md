## 核心结论（先背）

1. **newtype struct**：创建**新类型** — 类型安全、封装（与 `type` 别名不同）
2. **`type` 别名**：同义词，**不**创建新类型 — 缩短冗长签名
3. **`!` never type**：永不返回，可 coerce 为任意类型（`panic!`/`loop {}`）
4. **DST**：`str`、`[T]`、`dyn Trait` — 大小未知，**必须**放指针后（`&str`、`Box<dyn T>`）
5. **`Sized` / `?Sized`**：泛型默认 `T: Sized`；接受 DST 写 `T: ?Sized` + 引用/指针参数
6. **ZST**：大小为 0（`()`、无字段 struct）— 不占内存，仍参与类型系统

---

本节介绍 Rust 类型系统里几个常用但容易混淆的点：**newtype 的类型安全/抽象**、**类型别名**、**never type `!`**、**动态大小类型（DST）与 `Sized` / `?Sized`**、**零大小类型（ZST）**。

## 为类型安全与抽象使用 newtype

newtype 本质是“用元组结构体包一层”创建**全新类型**：

- **类型安全**：避免把不同含义但底层同表示的值混用（如 `Millimeters(u32)` vs `Meters(u32)`）。
- **抽象/封装**：隐藏内部表示，只暴露你想提供的 API；比如 `People(HashMap<i32, String>)`，外部无需知道 key 是 `i32`。

这点与类型别名不同：类型别名不会创建新类型。

---

## 类型别名（type alias）是“同义词”

```rust
type Kilometers = i32;
```

`Kilometers` 与 `i32` 是同一类型：能相加、能传参、不会触发“单位混用”的编译错误。

类型别名常用于**缩短冗长类型**，减少重复并提升可读性，例如：

```rust
type Thunk = Box<dyn Fn() + Send + 'static>;
```

也常用于统一 `Result` 的错误类型（如 `std::io::Result<T>`）。

---

## never type：`!`（从不返回）

`!` 是 empty type（无值），用于描述“**永不返回**”的表达式/函数（发散函数），典型来源：

- `continue` / `break`（带值时对应具体类型；不带值/跳转分支常可视为 `!` 参与类型推断）
- `panic!()`
- 不会结束的 `loop { ... }`

关键性质：`!` 可以**强转（coerce）**为任意类型，因此 `match` 中某分支 `panic!()`、`continue` 等不会破坏其它分支的类型一致性要求。

## DST（动态大小类型）与 `Sized`

有些类型在编译期不知道大小：例如 **`str`**（注意：不是 `&str`），以及 trait（用于 trait object 时的 `dyn Trait`）。

黄金规则：**DST 必须放在某种指针后面**：

- `&str` / `Box<str>` / `Rc<str>`
- `&dyn Trait` / `Box<dyn Trait>`

### `Sized` 与 `?Sized`

- `Sized`：编译期已知大小（绝大多数类型）。
- 泛型默认带上 `T: Sized` 约束。

想让泛型接受可能为 DST 的类型，写：

```rust
fn generic<T: ?Sized>(t: &T) { /* ... */ }
```

注意参数通常需要是引用/指针（例如 `&T`），因为 `T` 可能不 `Sized`。

---

## 零大小类型（ZST）

大小为 **0** 的类型，不占用实际内存（编译器可优化掉），但仍参与类型检查与泛型单态化。

| 例子 | 说明 |
|------|------|
| `()` | 单元类型 |
| `struct Empty;` | 无字段结构体 |
| `[T; 0]` | 空数组 |

用途：标记类型（phantom type）、编译期常量占位、某些泛型 API 的“无数据”变体。
