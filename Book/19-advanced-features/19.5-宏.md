## 核心结论（先背）

1. **宏 = 编译期元编程** — 写「生成代码的代码」，**编译期展开**，运行时无宏调用开销
2. **声明宏 `macro_rules!`** — Token 模式匹配 + `$(...),*` 重复
3. **过程宏** — 独立 `proc-macro` crate；`TokenStream → TokenStream`
4. **三类过程宏**：derive · 属性 · 函数式
5. **调试**：`cargo expand` · 优先函数，宏谨慎使用

---

## 一、宏核心总览

### 1. 定义

宏属于**编译期元编程**：在**编译阶段**展开为目标 Rust 代码，运行时无「调用宏」的额外逻辑。

### 2. 两大分类

| 类型 | 语法 | 特点 |
|------|------|------|
| **声明宏** | `macro_rules!` | Token 模式匹配，入门简单 |
| **过程宏** | `#[proc_macro_*]` | 解析 `TokenStream`，能力更强 |

过程宏三子类：

- **派生宏**：`#[derive(MyTrait)]`
- **属性宏**：`#[my_attr]`
- **函数式宏**：`my_macro!()`

### 3. 宏 vs 函数

| 对比项 | 宏 | 普通函数 |
|--------|-----|----------|
| 执行时机 | **编译期展开** | 运行期调用 |
| 参数 | 任意 Token 流、可变结构 | 固定签名、类型、个数 |
| 能力 | 可生成 `struct`/`impl`/代码块 | 仅执行业务逻辑 |
| 调试 | 报错绕、难读 | 栈追踪清晰 |
| 作用域 | **先定义/导入，后使用** | 顺序宽松 |

> **选型**：优先函数；**函数做不到**再考虑宏。

## 二、声明宏 `macro_rules!`

### 1. 核心原理

匹配**源代码 Token 流**（非运行时值），成功后替换为展开代码：

```rust
macro_rules! 宏名 {
    (匹配模式) => { 展开代码 };
    // 多个分支，从上到下匹配
}
```

### 2. 捕获标识 & 重复语法

#### 片段说明符（常用）

| 说明符 | 捕获 |
|--------|------|
| `$var:expr` | 表达式 |
| `$var:ident` | 标识符（变量/函数/类型名） |
| `$var:ty` | 类型 |
| `$var:stmt` | 语句 |
| `$var:tt` | 单个 token 树 |

#### 重复规则

| 写法 | 含义 |
|------|------|
| `$( 内容 ),*` | 0 个或多个，逗号分隔 |
| `$( 内容 ),+` | 1 个或多个 |
| `$( 内容 )?` | 0 或 1 个 |

### 3. 实战：`my_vec!`

```rust
macro_rules! my_vec {
    () => { Vec::new() };
    ( $( $x:expr ),* $(,)? ) => {
        {
            let mut temp_vec = Vec::new();
            $( temp_vec.push($x); )*
            temp_vec
        }
    };
}

// my_vec![]        → []
// my_vec![1, 2, 3] → [1, 2, 3]
```

→ 完整可运行：`19.5-macros-demo/decl_macro_demo/`

### 4. 关键特性

1. **卫生宏（Hygienic）**：宏内局部变量**不污染**外部命名空间
2. 分支**从上到下**匹配，首个成功即展开
3. 宏须在使用**之前**定义或 `use` 导入

## 三、过程宏（Proc-Macro）

### 1. 整体架构

```
TokenStream（输入） → syn 解析 AST → quote 生成 → TokenStream（输出）
```

**工程约束**：

1. 过程宏**必须独立 crate**
2. `Cargo.toml`：`[lib] proc-macro = true`
3. 主项目通过 path/workspace 依赖引入

**标配依赖**：`syn`（解析 AST）+ `quote`（生成 Token）

### 2. 三类过程宏

| 类型 | 写法 | 用途 |
|------|------|------|
| 派生宏 | `#[derive(XXX)]` | 自动生成 trait impl |
| 属性宏 | `#[xxx]` | 修饰/包裹 item |
| 函数式宏 | `xxx!()` | 类似声明宏，能力更强 |

### 3. Workspace 结构（Demo）

```
19.5-macros-demo/
├── Cargo.toml           # workspace
├── decl_macro_demo/     # 声明宏
├── hello_macro/         # trait 定义
├── hello_macro_derive/  # proc-macro 实现
└── pancakes/            # 业务使用 #[derive(HelloMacro)]
```

#### hello_macro — trait 定义

```rust
pub trait HelloMacro {
    fn hello_macro();
}
```

#### hello_macro_derive — 派生宏

```rust
#[proc_macro_derive(HelloMacro)]
pub fn hello_macro_derive(input: TokenStream) -> TokenStream {
    let ast = parse_macro_input!(input as DeriveInput);
    let name = ast.ident;
    quote! {
        impl hello_macro::HelloMacro for #name {
            fn hello_macro() {
                println!("Hello, Macro! My name is {}", stringify!(#name));
            }
        }
    }.into()
}
```

#### pancakes — 使用方

```rust
use hello_macro::HelloMacro;
use hello_macro_derive::HelloMacro;

#[derive(HelloMacro)]
struct Pancakes;

fn main() {
    Pancakes::hello_macro();  // Hello, Macro! My name is Pancakes
}
```

```bash
cargo run -p pancakes
```

### 4. 属性宏 / 函数式宏骨架

```rust
#[proc_macro_attribute]
pub fn my_attr(_attr: TokenStream, item: TokenStream) -> TokenStream {
    item  // 解析 + 改写
}

#[proc_macro]
pub fn my_func_macro(input: TokenStream) -> TokenStream {
    input  // 解析并生成
}
```

→ 见 `hello_macro_derive/src/lib.rs` 底部注释

## 六、高频易错点

| # | 易错 | 正确理解 |
|---|------|----------|
| 1 | 宏分支顺序无关 | **从上到下**，通用分支放最后 |
| 2 | `$(...)` 括号/逗号写错 | 须成对；`,` 位置错误 → 编译失败 |
| 3 | 过程宏写在业务 crate | 须**独立** `proc-macro = true` crate |
| 4 | 宏库与业务循环依赖 | 设计时解耦：trait crate / derive crate / 使用方 |
| 5 | 报错行号不准 | 可能指向**展开后**代码 → 用 `cargo expand` |
| 6 | 依赖宏外变量名 | 卫生宏不保证捕获外部 ident（声明宏内变量不外泄） |
| 7 | 过度使用宏 | 能用函数/trait/泛型就不用宏 |

---

## 七、速记

1. **声明宏**：`macro_rules!` + `$捕获` + `$(...),*`
2. **过程宏**：独立 crate · `syn`+`quote` · 三类 derive/属性/函数式
3. **区分**：简单重复 → 声明宏；复杂 AST/derive → 过程宏
4. **调试**：`cargo expand`
5. **原则**：函数优先，宏兜底
