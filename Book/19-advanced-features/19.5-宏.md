## 核心结论（先背）

1. **宏 = 编译期生成代码**（元编程），非运行期调用
2. **声明宏 `macro_rules!`**：`$(...),*` token 模式匹配
3. **过程宏三类**：derive · 属性 · 函数式 — `TokenStream → TokenStream`
4. **hygiene**：宏内标识符默认不污染外部作用域
5. **调试**：`cargo expand` 查看展开结果

---

## 一、核心概念

### 宏 vs 函数

| | 宏 | 函数 |
|--|-----|------|
| 参数 | 可变 token 结构 | 固定签名 |
| 执行 | **编译期展开** | 运行期调用 |
| 能力 | 可生成 `impl`/定义 | 不能 |
| 代价 | 难写、难调试、错误信息绕 | 类型安全、易读 |

### 声明式宏 `macro_rules!`

模式匹配 **token 结构**（非运行时值）：

- `$( $x:expr ),*` — 零个或多个，逗号分隔
- `$x:expr` / `$i:ident` — 片段捕获类型

### 过程宏（proc-macro）

| 类型 | 挂载 | 示例 |
|------|------|------|
| 派生宏 | `#[derive(T)]` | `#[derive(HelloMacro)]` |
| 属性宏 | `#[attr]` | 自定义属性 |
| 函数式宏 | `mac!(...)` | 类 `println!` |

须**独立 crate**，`[lib] proc-macro = true`。常用 `syn` + `quote`。

### Hygiene（卫生宏）

宏展开时内部绑定的变量/标识符与调用处**隔离**，避免意外捕获外部同名变量（非 100% 覆盖所有场景）。

## 二、代码示例

### 声明式宏

```rust
macro_rules! my_vec {
    () => { Vec::new() };
    ( $( $x:expr ),+ $(,)? ) => {
        {
            let mut temp_vec = Vec::new();
            $( temp_vec.push($x); )+
            temp_vec
        }
    };
}

fn main() {
    let v = my_vec![1, 2, 3];
    println!("{:?}", v);
}
```

→ 完整版：`19.5-macros-demo/decl_macro_demo/src/main.rs`

### 过程宏 derive

`#[derive(HelloMacro)]` 展开为 `impl HelloMacro for TypeName { ... }` — 见 `pancakes` crate + `hello_macro_derive`。

## 三、分节详解

### `macro_rules!` 匹配顺序

规则**从上到下**匹配，**第一个**匹配成功的展开。更具体的规则应放前面。

### 过程宏工作流

```
TokenStream 输入 → syn 解析 AST → quote 生成代码 → TokenStream 输出
```

### 调试

```bash
cargo install cargo-expand
cargo expand -p pancakes
```

## 四、易错点

| # | 易错 | 正确理解 |
|---|------|----------|
| 1 | 宏规则顺序无关 | **从上到下**匹配，顺序很重要 |
| 2 | 过程宏写在普通 crate | 须**独立** proc-macro crate + `proc-macro = true` |
| 3 | 宏错误难读 | 用 **`cargo expand`** 看展开后代码 |
| 4 | 宏 = 运行时调用 | **编译期展开**，增大二进制、影响编译时间 |
| 5 | hygiene 绝对安全 | 复杂宏仍可能有 surprising 捕获/命名行为 |
| 6 | 过度使用宏 | 能用函数/trait 就不用宏（ER Item 28） |

---

## 五、速记

1. `macro_rules!` — token 模式 + `$(...)*` 重复
2. 过程宏 — derive / 属性 / 函数式；独立 crate
3. hygiene — 宏内变量不污染外部
4. 调试 — `cargo expand`
5. 优先函数，宏谨慎使用
