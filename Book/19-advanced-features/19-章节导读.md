## 各节考点速览

### 19.1 不安全 Rust

- **unsafe ≠ 关闭检查**：程序员承诺自行维护安全不变式；借用检查等对 `&T`/`&mut T` 仍有效
- **五大 unsafe 操作**：
  1. 解引用裸指针 `*const T` / `*mut T`
  2. 调用 `unsafe fn` / unsafe 方法
  3. 读写 `static mut`
  4. `unsafe impl` 不安全 trait
  5. 访问 `union` 字段
- **典型场景**：FFI、底层数据结构、安全 API 封装（缩小 unsafe 块）
- → ER：[Item 16 避免 unsafe](../../ER/Chapter-03-Concepts/Item-16-avoid-unsafe.md) · [Item 34 FFI](../../ER/Chapter-06-Beyond-Standard-Rust/Item-34-ffi-boundaries.md)

### 19.2 高级 Trait

- **关联类型 vs 泛型 trait**：关联类型每类型只能 impl 一次；泛型 trait 可多次 impl
- **默认泛型参数**：如 `trait Add<Rhs = Self>`
- **完全限定语法**：`<Type as Trait>::method()` 消歧义
- **supertrait**：`trait T: Debug` — 实现 T 须先实现 Debug
- **newtype**：孤儿规则绕过 — 包装外部类型以 impl 外部 trait
- → 入门：[10.2.4 关联类型](../10-generics-traits-lifetimes/10.2.4-type关键字与关联类型.md)

### 19.3 高级类型

- **类型别名 `type`**：同义词，不创建新类型（vs newtype struct）
- **永不类型 `!`**：发散表达式，可 coerce 为任意类型
- **DST**：`str`、`[T]`、`dyn Trait` — 编译期大小未知，须放指针后
- **`Sized` / `?Sized`**：泛型默认 `T: Sized`；接受 DST 写 `?Sized`
- **ZST**：大小为 0（`()`、空 struct），不占内存但参与类型系统
- → 延伸：[17.2 trait 对象](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md)（DST）

### 19.4 高级函数与闭包

- **`fn` 函数指针**：不捕获环境；实现了 `Fn`/`FnMut`/`FnOnce`
- **闭包 trait**：`Fn`（`&self`）· `FnMut`（`&mut self`）· `FnOnce`（消耗 `self`）
- **返回值**：闭包匿名类型 → `Box<dyn Fn(...)>` 或 `impl Fn`
- **高阶函数**：`map`/`filter` 接收函数或闭包
- → 基础：[13.1 闭包](../13-iterators-closures/13.1-闭包.md)

### 19.5 宏

- **声明宏 `macro_rules!`**：`$(...),*` 重复、token 模式匹配（如 `vec!`）
- **过程宏**：derive / 属性 / 函数式 — `TokenStream → TokenStream`
- **hygiene**：宏内变量不污染外部命名空间
- **调试**：`cargo install cargo-expand` → `cargo expand`
- → ER：[Item 28 宏的使用](../../ER/Chapter-05-Tooling/Item-28-macros-judiciously.md)

1. **循序渐进**：先建立 unsafe 边界意识，再 trait/类型，最后宏

4. **ER 对照**：unsafe → Item 16/34；宏 → Item 28；trait 对象 → Item 12
