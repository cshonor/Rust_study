# 10.2.4 `impl Trait` vs `dyn Trait`（静态 vs 动态）

← [10.2.3 七种写法](./10.2.3-Trait七种写法详解.md) · [10.2.2 impl Trait 全解](./10.2.2-impl-Trait全解.md) · [17.2 trait 对象](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md) · demo：`10.2.4-impl-dyn-demo/`

---

## 核心结论

1. **`impl Trait`**：编译期确定真实类型，**静态派发**，无虚表，**只能单一类型**
2. **`dyn Trait`**：运行期确定真实类型，**动态派发**，带虚表，**能存多种不同类型**

---

## 前置代码（全程复用）

```rust
trait Animal {
    fn cry(&self);
}

struct Dog;
impl Animal for Dog {
    fn cry(&self) { println!("汪汪"); }
}

struct Cat;
impl Animal for Cat {
    fn cry(&self) { println!("喵喵"); }
}
```

---

## 一、入参位置对比

### 1. `impl Animal`（静态）

```rust
fn speak_impl(animal: impl Animal) {
    animal.cry();
}

fn main() {
    speak_impl(Dog);
    speak_impl(Cat);
}
```

- **编译原理**：每调用一次具体类型，就**单态化一份**专用代码  
  `speak_impl(Dog)` → Dog 版 · `speak_impl(Cat)` → Cat 版
- **优点**：运行期无查表，性能高
- **缺点**：**没法用一个集合同时存 Dog + Cat**

### 2. `&dyn Animal`（动态）

```rust
fn speak_dyn(animal: &dyn Animal) {
    animal.cry();
}

fn main() {
    let dog = Dog;
    let cat = Cat;
    speak_dyn(&dog);
    speak_dyn(&cat);

    // Vec 可同时放 Dog、Cat
    let list: Vec<&dyn Animal> = vec![&dog, &cat];
    for item in list {
        item.cry();
    }
}
```

- **原理**：`dyn` 带 **vtable**，运行时查表再调用 `cry`
- **优点**：**异构集合**（插件、多实现业务层）
- **缺点**：轻微运行时开销（HFT 关键链路尽量少用 `dyn`）

---

## 二、返回值位置对比（易错）

### 1. `-> impl Animal`：只能返回**固定一种类型**

```rust
fn get_animal_impl() -> impl Animal {
    Dog
}
```

❌ 分支两种类型：

```rust
fn get(flag: bool) -> impl Animal {
    if flag { Dog } else { Cat } // 编译报错
}
```

### 2. `-> Box<dyn Animal>`：运行期可返回多种类型

```rust
fn get_animal_dyn(flag: bool) -> Box<dyn Animal> {
    if flag {
        Box::new(Dog)
    } else {
        Box::new(Cat)
    }
}
```

> `dyn` 必须依附指针：`Box<dyn T>` / `&dyn T` / `Rc<dyn T>` 等 — **裸 `dyn Animal` 不能单独放栈**。

---

## 三、性能与场景对照

| 写法 | 分发 | 多类型共存 | 内存 | 适用 |
|------|------|------------|------|------|
| **`impl Trait`** | 静态 · 编译期 | 不能 | 无额外 | 迭代器、闭包返回、高性能底层 |
| **`dyn Trait`** | 动态 · 运行期 | 可以 | vtable + 堆 | 插件、异构 `Vec`、多实现业务 |

---

## 四、与泛型的关系

| | 静态 | 动态 |
|--|------|------|
| 写法 | `<T: Animal>` · 入参 `impl Animal` | `&dyn Animal` · `Box<dyn Animal>` |
| 多类型 | 同函数内所有 `T` 必须同类型；`impl` 各参可不同 | **可混合** Dog、Cat |

---

## 五、口诀

1. **类型确定、追求性能** → `impl Trait` / 泛型 `T`
2. **类型不确定、要 Vec 混多种对象** → `dyn` + `Box` / 引用

---

## 六、bonus：`-> impl Fn`（闭包返回）

闭包每种捕获组合对应**不同匿名类型**，无法手写类型名 → 用 **`-> impl Fn`** 隐藏：

```rust
fn make_adder() -> impl Fn(i32) -> i32 {
    |x| x + 5
}

fn main() {
    let add5 = make_adder();
    println!("{}", add5(10)); // 15
}
```

→ 完整 demo：[10.2-traits-demo](./10.2-traits-demo/) 的 `make_adder()`
