> **核心**：`type` = 给现有类型**起别名**，**不创建新类型**（trait 内 `type Item` 除外语义 — 那是**关联类型占位符**）。

---

## 一、场景 1：普通类型别名（最常用）

```rust
type 别名 = 原有类型;
```

作用：简化长类型名、统一命名、提升可读性。

### 基础

```rust
type Num = i32;

let a: Num = 100;
let b: i32 = a; // ✅ 完全等价，可混用
```

### 实战：简化 `Result` / 长类型

```rust
use std::fs::File;
use std::io::Error;

type FileResult = Result<File, Error>;

fn open_file(path: &str) -> FileResult {
    File::open(path)
}
```

### 泛型别名

```rust
type Pair<T> = (T, T);

let p1: Pair<i32> = (1, 2);
let p2: Pair<String> = ("a".into(), "b".into());
```

> **不产生新类型** — 只是「外号」，编译器视为同一类型。

## 二、场景 2：Trait 内 `type` → 关联类型

Iterator 里见到的写法：

```rust
trait MyIterator {
    type Item; // 关联类型占位符
    fn next(&mut self) -> Option<Self::Item>;
}

struct Counter { idx: u32 }

impl MyIterator for Counter {
    type Item = u32; // 实现方指定

    fn next(&mut self) -> Option<Self::Item> {
        if self.idx < 3 {
            let v = self.idx;
            self.idx += 1;
            Some(v)
        } else {
            None
        }
    }
}
```

标准库：

```rust
trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

### 关联类型 vs 泛型 Trait

| | **关联类型 `type Item`** | **泛型 `trait Iter<T>`** |
|--|--------------------------|---------------------------|
| 实现次数 | 每类型**只能 impl 一次** | 同一类型可 impl **多个 T** |
| 谁定类型 | **实现方**写 `type Item = …` | **调用方**指定 `T` |

```rust
trait Iter { type Item; fn next(&mut self) -> Option<Self::Item>; }
trait IterGen<T> { fn next(&mut self) -> Option<T>; }
```

## 三、场景 3：`impl` 块内 `type`

就是场景 2 的落地 — 在 `impl Trait for Type` 里写 `type Item = u32;`，无需单独章节。

## 四、易混：`type` / `struct` / `enum`

| 关键字 | 作用 | 是否新类型 |
|--------|------|------------|
| **`type A = B`** | 别名 | ❌ 与 `B` 完全通用 |
| **`struct NewAge(i32)`** | 新结构体 | ✅ 与 `i32` **不**通用 |
| **`enum`** | 新枚举 | ✅ |

```rust
type Age = i32;
struct NewAge(i32);

let a: Age = 18;
let b: i32 = a; // ✅

let c = NewAge(18);
// let d: i32 = c; // ❌
```

---

## 五、踩坑

1. **别名不能单独 impl 方法** — 只能给**原类型** impl
2. **不能递归别名** — `type A = Vec<A>;` ❌
3. **所有权/借用** — 完全跟随原类型

---

## 六、背诵版

1. **`type X = Y`**：外号，简化长类型，无新类型
2. **trait 内 `type Item`**：关联类型，impl 时指定，Iterator 核心
3. vs **`struct`/`enum`**：后两者定义**全新**类型
