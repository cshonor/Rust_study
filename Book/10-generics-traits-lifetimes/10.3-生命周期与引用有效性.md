## 核心结论

生命周期标记（`'a`、`'static`）**不创建引用**，只约束：**引用在使用期间，指向的数据必须一直有效**。

> 入参、返回值都标 `'a` → 返回的引用**来自**带同一 `'a` 的入参，**不能比该组 `'a` 活得更久**。

---

## 最终三句（唯一结论，可直接复制）

> 与 **rustc / demo §0.8** 一致。详解 [10.3.2](./10.3.2-同a约束与红线.md)

```
【Rust 生命周期 · 同 'a 三条】

1. 无返回值函数
   入参一长一短 + 共用 'a → 永远不报错。
   编译器把 'a 收缩到最短作用域；该块内被借数据都活着。

2. 有返回值函数（返回 &'a T）
   返回值留在最短作用域内 → 不报错；
   返回值跑到最短作用域外 → 报错（悬垂）。

3. 'a 的本质
   'a = 引用的使用边界，不是「两变量必须同时 drop」。
   只要求：在 'a 范围内，所有被引用的数据都活着。
```

**canonical 实测（无返回值，永远 ✅）**：

```rust
fn use_both<'a>(a: &'a i32, b: &'a i32) {}

fn main() {
    let long_live = 100;
    {
        let short_live = 200;
        use_both(&long_live, &short_live);
    }
}
```
