# 10.2.3 Trait 七种写法详解（逐条对比 + 可运行示例）

← [10.2.1 汇总表](./10.2.1-Trait语法速查.md) · [10.2.2 impl Trait 全解](./10.2.2-impl-Trait全解.md) · demo：`10.2.3-trait-syntax-demo/`

> **Trait = 接口**，规定一批方法；**`impl Trait for Type`** = 该类型拥有 trait 里所有函数。

---

## 1. `fn(x: impl Trait)` — 入参 impl

**含义**：任意实现 `Trait` 的类型都能传入；泛型简写；**静态分发**。

> 每个 `impl Trait` **互相独立** — 两个形参可传**两种不同类型**。

```rust
trait Speak { fn hello(&self); }
struct Dog;
struct Cat;
impl Speak for Dog { fn hello(&self) { println!("汪汪"); } }
impl Speak for Cat { fn hello(&self) { println!("喵喵"); } }

fn test(a: impl Speak, b: impl Speak) {
    a.hello();
    b.hello();
}
// test(Dog, Cat); ✅
```

**特点**：简短；**无法**约束两参数同类型。

---

## 2. `fn<T: Trait>(x: T)` — 标准泛型

**`<T: Trait>`**：函数内所有 `T` 必须是**同一具体类型**。

```rust
fn test<T: Speak>(a: T, b: T) {
    a.hello();
    b.hello();
}
// test(Dog, Dog); ✅
// test(Dog, Cat); ❌ T 已定为 Dog
```

| vs §1 | `impl Trait` | 泛型 `T` |
|-------|--------------|----------|
| 多参类型 | 各自独立 | **强制相同** |

---

## 3. `impl A + B` / `T: A + B` — 多 trait 叠加

`+` = 类型**同时**实现 A **和** B，缺一不可。

```rust
trait Run {}
trait Jump {}
struct Rabbit;
impl Run for Rabbit {}
impl Jump for Rabbit {}

fn go(x: impl Run + Jump) {}
// go(Rabbit); ✅
```

---

## 4. `where T: A + B` — 约束拆分

与写在 `<>` 里**功能完全一致**；泛型多、约束长时**可读性更好**。

```rust
fn go<T, U>(a: T, b: U)
where
    T: Run + Jump,
    U: Speak + Run,
{ }
```

---

## 5. `-> impl Trait` — 返回位

**只能返回固定的某一种具体类型**；分支不能返回不同结构体。

```rust
trait Fruit { fn name(&self) -> &str; }
struct Apple;
impl Fruit for Apple { fn name(&self) -> &str { "苹果" } }

fn get_fruit() -> impl Fruit { Apple }

// ❌ if Apple else Banana — 两种类型，编译失败
```

运行期多种类型 → **`Box<dyn Fruit>`** → [17.2](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md)

| vs 泛型 `-> T` | `-> impl Trait` | `-> T` |
|----------------|-----------------|--------|
| 谁定类型 | **实现方** | **调用方** |

---

## 6. `impl<T: Bound> Struct<T> { }` — 条件方法

仅当 **`T` 满足 `Bound`** 时，结构体才有这组方法。

```rust
struct Data<T>(T);
trait Calc {}
impl Calc for i32 {}

impl<T: Calc> Data<T> {
    fn add(&self) {}
}

// Data(1i32).add(); ✅
// Data("abc").add(); ❌ &str 无 Calc
```

---

## 7. `impl<T: Bound> Trait for T` — Blanket impl

**地毯式**：所有满足 `Bound` 的类型**自动**实现目标 trait。

```rust
trait Show { fn print(&self); }
impl<T: std::fmt::Debug> Show for T {
    fn print(&self) { println!("{:?}", self); }
}
// 10.print(); "hi".to_string().print(); ✅
```

标准库：`impl<T: Display> ToString for T { … }`

---

## 一句话总区分

1. 入参省事、可不同类型 → **`impl Trait`**
2. 多参数必须同类型 → **`<T: Trait>`**
3. 同时多个接口 → **`+`**
4. 约束太长 → **`where`**
5. 返回固定单一类型、隐藏实现 → **`-> impl Trait`**
6. 结构体按需挂方法 → **`impl<T: Bound> Struct`**
7. 批量自动 impl → **Blanket impl**

---

## 8. `impl Trait` vs `dyn Trait`（补充）

| | **`impl Trait`** | **`dyn Trait`** |
|--|------------------|-----------------|
| 时机 | 编译期定类型 | 运行期多态 |
| 类型数 | **一种** | **多种** |
| 开销 | 无 vtable | 有 vtable |
| 典型写法 | `-> impl Fruit` | `-> Box<dyn Fruit>` |

```rust
// ✅ impl：函数体内永远只返回 Apple
fn get_fruit() -> impl Fruit { Apple }

// ❌ impl：分支两种结构体 — 编译失败
// fn pick(flag: bool) -> impl Fruit {
//     if flag { Apple } else { Banana }
// }

// ✅ dyn：运行期选 Apple 或 Banana
fn pick(flag: bool) -> Box<dyn Fruit> {
    if flag { Box::new(Apple) } else { Box::new(Banana) }
}
```
