# 10.2.8 虚表 vtable 通俗全解（比喻 · 内存 · 流程）

← [10.2.4 impl vs dyn](./10.2.4-impl与dyn-Trait对比.md) · 阅读 **⑥** · 下一条 **[10.2.6 胖指针](./10.2.6-dyn胖指针与虚表.md)** · demo：`cargo run -- vtable_story`

> 全程避开晦涩术语：**技能清单 · 地址簿 · 胖指针 · 查表调用**。

---

## 一、打比方

`trait Animal` = **技能清单**（规定必须会 `cry()`）。

- `Dog`、`Cat` = 不同**选手**，各自实现了叫唤。

**虚表 (vtable) = 统一的「技能地址簿」**  
簿里只存：**每个选手的 `cry` 函数在内存哪**。

用 `dyn Animal` 调 `cry()` 时：程序不知道当前是狗还是猫 → **先翻地址簿** → 找到对应 `cry` → 再执行。  
这就是 **查虚表**。

---

## 二、静态 vs 动态（impl vs dyn）

```rust
trait Animal { fn cry(&self); }
struct Dog;
struct Cat;
impl Animal for Dog { fn cry(&self) { println!("汪汪"); } }
impl Animal for Cat { fn cry(&self) { println!("喵喵"); } }
```

### 1. `impl Animal` — 编译期锁定，**不用虚表**

```rust
fn call(animal: impl Animal) {
    animal.cry();
}
call(Dog); // 编译器：直接 Dog::cry
call(Cat); // 编译器：直接 Cat::cry
```

**调用地址编译期写死**，运行时零查表 → **静态分发**。

### 2. `dyn Animal` — **必须查虚表**

`&dyn Animal` / `Box<dyn Animal>` = **胖指针**（两个指针绑在一起）：

```
[ 数据指针 ] + [ 虚表指针 ]
```

| 字段 | 指向 |
|------|------|
| **数据指针** | 对象本体（Dog / Cat 实例） |
| **虚表指针** | 该类型对 `Animal` 的 **vtable** |

转成 `dyn` 时**两个地址一次性写好**，不是从数据里再去找虚表 → [10.2.6](./10.2.6-dyn胖指针与虚表.md)

### 虚表长什么样

每个实现 `Animal` 的**类型**有一张表（**类型级**，所有 Dog 实例共用 Dog 那张）：

**Dog 的 vtable**

| 函数 | 地址 |
|------|------|
| `cry` | `Dog::cry` |

**Cat 的 vtable**

| 函数 | 地址 |
|------|------|
| `cry` | `Cat::cry` |

本质：**存函数指针的数组**，Rust 自动生成。

一个类型实现多个 Trait → 多张表 → [10.2.7](./10.2.7-多Trait多虚表.md)

## 三、完整流程：`call_dyn(&dog)` 发生了什么

```rust
fn call_dyn(a: &dyn Animal) {
    a.cry();
}

let dog = Dog;
call_dyn(&dog);
```

| 步骤 | 发生什么 |
|------|----------|
| 1 | `&Dog` → 胖指针：数据 ptr = `dog`；vtable ptr = **Dog 的 Animal 虚表** |
| 2 | 进入 `call_dyn`，执行 `a.cry()` |
| 3 | 用 **虚表指针** 找到 Dog 的 vtable |
| 4 | 在表里查 `cry` 的函数地址 |
| 5 | 跳转执行 `Dog::cry`，数据 ptr 当 `&self` → 打印 `汪汪` |

传 `&Cat` → 虚表指针换成 **Cat 的 vtable** → 执行 `Cat::cry`。

👉 **`dyn Trait` 调方法 = 运行时查虚表**。

---

## 四、为什么 `impl Trait` 不要虚表？

编译期已知具体类型 → 直接绑定 `Dog::cry` 或 `Cat::cry` → **地址提前算好**，运行时零查表。

---

## 五、为什么 `dyn` 能异构 `Vec`？

```rust
let list: Vec<Box<dyn Animal>> = vec![
    Box::new(Dog),
    Box::new(Cat),
];
```

每个元素都是胖指针：

| 元素 | 数据 ptr | vtable ptr |
|------|----------|------------|
| Dog | → Dog 实例 | → Dog 虚表 |
| Cat | → Cat 实例 | → Cat 虚表 |

遍历 `cry()`：各自查自己的虚表 → **运行时多态**。  
`impl Trait` 编译期固定单一类型 → **不能**这样混放。

---

## 六、背诵版

| | **虚表 / dyn** | **impl（无虚表）** |
|--|----------------|---------------------|
| 分发 | 运行时查表 · **动态** | 编译期绑定 · **静态** |
| 结构 | 胖指针（数据 + vtable） | 普通引用/值 |
| 异构 Vec | ✅ | ❌ |
| 开销 | 多一次查表 | 无 vtable |

1. **vtable**：每个 (类型, Trait) 一张函数指针表，Rust 自动生成  
2. **dyn** = 胖指针 + 查虚表 → 灵活、可混类型  
3. **impl** = 编译期定死 → 快、不能异构

---

## 七、一句话收尾

- **`impl Trait`**：编译前就知道是谁 → **直接喊人干活**，不查本子。  
- **`dyn Trait`**：不知道是谁 → **先翻地址簿（虚表）**，再干活。
