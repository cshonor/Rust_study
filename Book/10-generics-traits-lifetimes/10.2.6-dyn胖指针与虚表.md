# 10.2.6 `dyn` 胖指针与虚表（两指针分工 · 误区澄清）

← [10.2.4 impl vs dyn](./10.2.4-impl与dyn-Trait对比.md) · [17.2 trait 对象](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md) · demo：`10.2.4-impl-dyn-demo/`（`cargo run -- vtable`）

---

## 核心结论（先纠正理解）

`&dyn Trait` / `Box<dyn Trait>` 是**胖指针**，固定含**两个独立指针**：

1. **数据指针** → 直接指向**原始实例**（如 `Dog` 对象）
2. **虚表指针** → 直接指向**该类型对这个 Trait 的虚表**

👉 **并列存储，互相不查找、不依赖。**  
**不是**「先拿数据，再顺着数据找虚表」—— **转成 `dyn` 时，两个地址一次性写好**，之后都固定在胖指针里。

---

## 一、构造：`&Dog` → `&dyn Animal`

```rust
trait Animal { fn cry(&self); }
struct Dog;
impl Animal for Dog { fn cry(&self) { println!("汪汪"); } }

let dog = Dog;
let p: &dyn Animal = &dog;
```

| 胖指针字段 | 存入什么 |
|------------|----------|
| 指针 1（数据） | `dog` 实例的内存地址 |
| 指针 2（vtable） | **Dog 对 Animal** 的虚表地址 |

**一步绑定完成**，之后两地址不变。

---

## 二、调用 `p.cry()` 的流程

1. 从胖指针取出**虚表指针** → 定位虚表
2. 在虚表里查 `cry` 的**函数地址**
3. 调用该函数，把**数据指针**作为 `&self` 传入（告诉函数操作哪个对象）

### 比喻：卡片两行地址

| 行 | 含义 |
|----|------|
| 第一行 | **物品在哪**（数据指针 → Dog 本体） |
| 第二行 | **说明书在哪**（vtable → 函数地址表） |

调用时：先看第二行翻说明书找「叫唤」步骤 → 再拿第一行地址操作物品。  
**全程不用物品地址去找说明书。**

---

## 三、误区澄清

### 误区 1：每个实例一张虚表？

❌ 虚表属于**类型**，不属于单个对象。

- 所有 `Dog` 实例 **共用同一张** `Animal` 虚表
- 多少个 `Dog`，虚表仍**一份**；各胖指针的 vtable 字段指向**同一地址**

### 误区 2：多 Trait 怎么办？

```rust
trait Run { fn run(&self); }
impl Run for Dog { fn run(&self) {} }
```

| 转换 | 数据指针 | 虚表指针 |
|------|----------|----------|
| `&dog as &dyn Animal` | → 同一个 `Dog` | → **Animal** 虚表 |
| `&dog as &dyn Run` | → 同一个 `Dog` | → **Run** 虚表 |

换 `dyn Trait` = **只换虚表指针字段**；数据指针始终指向原对象。

---

## 四、与 `&str` 胖指针对比（别混）

| 胖指针 | 两个字段 |
|--------|----------|
| **`&str` / `&[T]`** | ptr + **len**（长度） |
| **`&dyn Trait`** | 数据 ptr + **vtable ptr** |

都是「定长指针壳 + 外部数据」，但第二字段含义不同。

## 六、背诵版

1. 胖指针 = **数据 ptr + vtable ptr**，创建时**同时写好**，互不查找
2. 调方法：**vtable** 找函数，**数据 ptr** 当 `self`
3. 虚表按**类型**共享，不按实例
4. 换 `dyn Trait` → 换 vtable 字段，数据 ptr 不变
