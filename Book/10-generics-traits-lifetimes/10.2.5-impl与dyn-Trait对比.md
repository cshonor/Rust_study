## 核心结论（三句）

1. **`impl Trait`**：编译期定类型 · **静态派发** · 单一类型 · 快
2. **`dyn Trait`**：运行期定类型 · **动态派发** · 异构集合 · 胖指针 + vtable
3. **`dyn Trait` 是 `!Sized`**：本体大小不固定，**不能裸当变量/参数** → 必须 **`&dyn Trait`** 或 **`Box<dyn Trait>`**（类型大小规则，不是语法习惯）

→ DST 总论：[4.3.4 DST 与 Sized](../04-ownership/4.3.4-DST与Sized.md)

## 一、`!Sized` 铁律：为何常见 `&dyn Trait`

### 1. `dyn Trait` = 特征对象（动态分发）

用来实现**动态分发**：不同类型只要实现了同一个 trait，就能统一用 `dyn Trait` 表示，**运行时再确定**调用哪个实现。

### 2. 不定长类型（Sized 问题）

Rust 栈上的变量，**编译期必须确定固定大小**。

`dyn Trait` 可以代表任意实现该 trait 的类型，不同类型大小不一样 → 属于 **`!Sized`（不定长）**。

👉 **规则：`!Sized` 类型不能直接作为局部变量、函数参数、返回值**，必须套一层**固定大小的指针/引用**：

| 合法包装 | 含义 |
|----------|------|
| **`&dyn Trait`** | 栈引用（最常见，零堆分配） |
| **`Box<dyn Trait>`** | 堆装箱 |
| `&mut dyn Trait` | 可变特征对象（如 `FnMut`） |

指针/引用本身大小固定 → 符合栈布局规则。

### 3. 错误 vs 正确（结合 `Fn` 闭包）

```rust
// ❌ dyn Fn() 是 !Sized，不能直接当参数
// fn call(f: dyn Fn()) { f(); }

// ✅ 加 & → &dyn Fn()（日常最常用）
fn call(f: &dyn Fn()) {
    f();
}

fn main() {
    let c = || println!("hello");
    call(&c); // 传引用
}
```

`&` 的作用：把不定长的特征对象包成**固定大小的引用**。

```rust
// ✅ 另一种合法写法：Box<dyn Fn()>（堆分配）
fn call_boxed(f: Box<dyn Fn()>) {
    f();
}

let c = Box::new(|| println!("hello"));
call_boxed(c);
```

### 4. 可变特征对象

需要 **`&mut dyn Trait`**，例如传 `&mut dyn FnMut()`：

```rust
fn call_mut(f: &mut dyn FnMut()) {
    f();
}

let mut c = || println!("mut");
call_mut(&mut c);
```

### 5. 为什么很少直接写裸 `dyn Trait`？

1. **语法硬性限制**：`dyn Trait` 本身 `!Sized`，**不允许裸用**
2. **`&dyn Trait` 开销最小**：栈上引用，不堆分配 → 代码里遍地都是
3. **静态 vs 动态**：
   - 泛型 `<F: Fn()>` / `impl Fn()`：**静态分发**，编译期确定类型
   - `&dyn Fn()`：**动态分发**，运行时查 vtable，支持异构统一传递

### 6. 易混点

- **只有特征对象才写 `dyn`**；普通泛型约束 `<F: Trait>` **不加 `dyn`**，也不强制加引用
- 闭包章节里的 `Fn` / `FnMut` / `FnOnce` bound → [13.1.4 Fn 系列](../../13-iterators-closures/13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)

### 7. 一句话

1. `dyn Trait` 本身大小不固定，Rust **不允许直接使用**
2. **`&dyn Trait` = 给特征对象套一层引用**，变成固定大小，才能当参数/变量
3. 不是习惯，是 **Rust 类型大小规则强制要求**

## 二、`dyn Trait` 四个核心要点（动态派发）

### 1. 运行期确定真实类型（类型擦除）

- 编译期**不知道**具体是 `Dog` 还是 `Cat`，只知道「实现了 `Animal`」
- 具体类型信息**只在运行时**通过 vtable 解析

### 2. 动态派发（dynamic dispatch）

- 调用方法时**不是**编译期直接绑定到 `Dog::cry` / `Cat::cry`
- 运行时再查该对象对应的方法地址 → 有一点点开销

### 3. 胖指针 + 虚表（vtable）

`&dyn Trait` / `Box<dyn Trait>` 是**胖指针**，**两个指针并列、创建时同时写好**：

| 部分 | 指向 |
|------|------|
| **数据指针** | 实际对象（如 `Dog` 实例） |
| **vtable 指针** | 该**类型**对这个 Trait 的虚表 |

调用方法：**vtable 指针**找函数地址，**数据指针**当 `&self` — **不是**从数据里再查 vtable。

→ **通俗全解（比喻·流程·异构 Vec）**：[10.2.6 vtable 通俗全解](./10.2.6-vtable通俗全解.md) · `cargo run -- vtable_story`

### 4. 异构集合（最实用）

`Vec<Box<dyn Animal>>` 可放**任意实现 `Animal` 的不同类型**；  
普通 `Vec<T>` / 泛型只能放**同一种** `T`。

```rust
let animals: Vec<Box<dyn Animal>> = vec![
    Box::new(Cat),
    Box::new(Dog),
];
for a in animals {
    a.cry(); // 运行时查 vtable
}
```

→ 对象安全等细节：[17.2](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md)

## 三、前置代码（demo 复用）

```rust
trait Animal {
    fn cry(&self);
}

struct Dog;
struct Cat;

impl Animal for Dog { fn cry(&self) { println!("汪汪"); } }
impl Animal for Cat { fn cry(&self) { println!("喵喵"); } }
```

---

## 四、入参位置对比

### `impl Animal`（静态）

```rust
fn speak_impl(animal: impl Animal) {
    animal.cry();
}
// speak_impl(Dog); speak_impl(Cat); — 各单态化一份代码
```

- ✅ 无 vtable 开销  
- ❌ **不能** `Vec` 混 Dog + Cat

### `&dyn Animal`（动态）

```rust
fn speak_dyn(animal: &dyn Animal) {
    animal.cry();
}

let dog = Dog;
let cat = Cat;
let list: Vec<&dyn Animal> = vec![&dog, &cat];
```

---

## 五、返回值位置（易错）

| | `-> impl Animal` | `-> Box<dyn Animal>` |
|--|------------------|----------------------|
| 分支多种类型 | ❌ | ✅ |
| 示例 | 固定 `Dog` | `if flag { Dog } else { Cat }` |

> `dyn` 是 **DST**，不能裸放栈 → 必须 `&` / `Box` / `Rc` 等指针。

---

## 六、完整对比表（impl vs dyn）

| 维度 | **`impl Trait`** | **`dyn Trait`** |
|------|------------------|-----------------|
| **分发** | 静态，编译期绑定 | 动态，运行期查 vtable |
| **类型擦除** | ❌ 编译器知道真实类型 | ✅ 编译期不知具体类型 |
| **异构** | ❌ 单一固定类型 | ✅ 多类型共存 |
| **大小** | **Sized**，栈上可直接用 | **DST**，须胖指针包装 |
| **使用位置** | 入参、返回值（常见） | 入参、异构集合、返回值（较少） |
| **二进制体积** | 单态化 → 每种类型一份代码，**可能变大** | 一份动态分发代码，**可能更小** |
| **运行时性能** | 更快（无查表） | 略慢（vtable） |

---

## 七、与泛型 / 口诀

| | 静态 | 动态 |
|--|------|------|
| 写法 | `<T: Animal>` · `impl Animal` | `&dyn Animal` · `Box<dyn Animal>` |

**口诀**：

- **灵活、异构、运行时多态** → `dyn Trait`（胖指针 + vtable）
- **快、静态、单一类型** → `impl Trait`（静态派发）
