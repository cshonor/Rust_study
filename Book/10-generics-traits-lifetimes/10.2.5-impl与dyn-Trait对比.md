## 核心结论（三句）

1. **`impl Trait`**：编译期定类型 · **静态派发** · 单一类型 · 快
2. **`dyn Trait`**：运行期定类型 · **动态派发** · 异构集合 · 胖指针 + vtable
3. **`dyn Trait` 是 `!Sized`**：本体大小不固定，**不能裸当变量/参数** → 必须 **`&dyn Trait`** 或 **`Box<dyn Trait>`**（类型大小规则，不是语法习惯）

→ DST 总论：[4.3.4 DST 与 Sized](../04-ownership/4.3.4-DST与Sized.md) · **静态/动态全流程**：[10.2.6 vtable 通俗全解](./10.2.6-vtable通俗全解.md)

## 一、`!Sized` 铁律：为何常见 `&dyn Trait`

### 1. `dyn Trait` = 特征对象（动态分发）

用来实现**动态分发**：不同类型只要实现了同一个 trait，就能统一用 `dyn Trait` 表示，**运行时再确定**调用哪个实现。

### 2. 不定长类型（Sized 问题）— 根源

`dyn Trait` 能代表**任意实现该 trait 的类型** — `dyn Hello` 可以是 `A`、`B`、某个闭包……

**不同类型内存大小不一样**：

| 类型 | 可能大小 |
|------|----------|
| 结构体 `A` | 8 字节 |
| 结构体 `B` | 16 字节 |
| 某闭包 | 又是一种大小 |

编译阶段编译器**没法算出一个固定大小**来容纳「所有可能的类型」→ `dyn Hello` 标记为 **`!Sized`（不定长）**。

👉 Rust 栈要求变量大小编译期必须确定 → 裸 `dyn T` 不能直接用，只能包 **`&` / `&mut` / `Box`**（指针本身大小固定）。

### 2b. 为什么非要「运行时确定」？

Rust 主打编译期安全，但**有些场景编译期根本猜不到具体类型**，只能运行时判断。

`dyn Trait` 是为这类场景设计的 — **不是设计缺陷，是功能取舍**（统一抽象多种不同类型）。

→ 查表流程：[10.2.6 §一](./10.2.6-vtable通俗全解.md#一两大模式编译期确定-vs-运行期确定)

👉 **规则：`!Sized` 类型不能直接作为局部变量、函数参数、返回值**，必须套一层**固定大小的指针/引用**：

| 合法包装 | 含义 |
|----------|------|
| **`&dyn Trait`** | 栈引用（最常见，零堆分配） |
| **`Box<dyn Trait>`** | 堆装箱 |
| `&mut dyn Trait` | 可变特征对象（如 `FnMut`） |

指针/引用本身大小固定 → 符合栈布局规则。

### 3. 错误 vs 正确（结合 `Fn` 闭包）

```rust
// ❌ dyn Fn() 是 !Sized，不能直接当参数
// fn call(f: dyn Fn()) { f(); }

// ✅ 加 & → &dyn Fn()（日常最常用）
fn call(f: &dyn Fn()) {
    f();
}

fn main() {
    let c = || println!("hello");
    call(&c); // 传引用
}
```

`&` 的作用：把不定长的特征对象包成**固定大小的引用**。

```rust
// ✅ 另一种合法写法：Box<dyn Fn()>（堆分配）
fn call_boxed(f: Box<dyn Fn()>) {
    f();
}

let c = Box::new(|| println!("hello"));
call_boxed(c);
```

### 4. 可变特征对象

需要 **`&mut dyn Trait`**，例如传 `&mut dyn FnMut()`：

```rust
fn call_mut(f: &mut dyn FnMut()) {
    f();
}

let mut c = || println!("mut");
call_mut(&mut c);
```

### 5. 为什么很少直接写裸 `dyn Trait`？

1. **语法硬性限制**：`dyn Trait` 本身 `!Sized`，**不允许裸用**
2. **`&dyn Trait` 开销最小**：栈上引用，不堆分配 → 代码里遍地都是
3. **静态 vs 动态**：
   - 泛型 `<F: Fn()>` / `impl Fn()`：**静态分发**，编译期确定类型
   - `&dyn Fn()`：**动态分发**，运行时查 vtable，支持异构统一传递

### 6. 易混点

- **只有特征对象才写 `dyn`**；普通泛型约束 `<F: Trait>` **不加 `dyn`**，也不强制加引用
- 闭包章节里的 `Fn` / `FnMut` / `FnOnce` bound → [13.1.4 Fn 系列](../../13-iterators-closures/13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)

### 7. 一句话

1. `dyn Trait` 本身大小不固定，Rust **不允许直接使用**
2. **`&dyn Trait` = 给特征对象套一层引用**，变成固定大小，才能当参数/变量
3. 不是习惯，是 **Rust 类型大小规则强制要求**

## 二、为何泛型 / `impl Trait` 做不了「运行时二选一」？

> **核心**：泛型 / `impl Trait` = **编译期单类型绑定**；运行时在同一位置切换 A/B = **类型擦除** → 只能 **`dyn Trait`**。  
> **二者不是二选一，是应对不同场景。**

### 1. 泛型 `<T: Trait>` 做了什么（模板 + 单态化）

1. `fn foo<T: Hello>(t: T)` 只是一份**代码模板**
2. **编译时**，每传入**一种不同具体类型**，编译器**单独生成一份专属函数**：
   - 传 `A` → 生成 `foo` 的 `A` 版本
   - 传 `B` → 生成 `foo` 的 `B` 版本
3. 对**单个函数调用、单个变量**，`T` 在编译期**只能绑定成唯一一个具体类型** — 不能一会儿 A、一会儿 B

### 2. 反例：用 `impl Trait` 做运行时二选一 → 直接报错

```rust
trait Hello { fn say(&self); }
struct A; impl Hello for A { fn say(&self) {} }
struct B; impl Hello for B { fn say(&self) {} }

// ❌ 编译失败
fn get_choose(flag: bool) -> impl Hello {
    if flag {
        A
    } else {
        B // E0308：分支类型不一致
    }
}
```

**报错原因**：

- `flag` 是**运行时**才知道真假，编译期无法判定返回 `A` 还是 `B`
- `-> impl Hello` 要求：**整个函数所有返回路径，必须是同一种具体类型**

### 3. `impl Trait` 的两条硬性规则

| 位置 | 规则 |
|------|------|
| **返回值 `-> impl Trait`** | 所有 `return` / 分支 **必须是同一具体类型**（哪怕都 impl 了该 trait） |
| **参数 `fn f(arg: impl Trait)`** | 等价匿名泛型 — 可**不同调用**分别传 A、B；但**同一次调用、同一变量**只能一种类型 |

### 4. 都实现了同一 Trait，为什么还不能混用？

- **Trait** = **行为约定**（都会 `say`）
- **A、B** = **两个完全不同的实体类型**（内存布局、大小、成员全不同）

Rust 是**静态类型语言**：

> 任意一个变量 / 返回值，**编译期必须确定唯一的具体类型**。

| 机制 | 实际含义 |
|------|----------|
| 泛型 / `impl Trait` | 类型可延后推导，但推导完成后仍是**唯一具体类型** |
| 运行时二选一 | **同一位置**运行时在 A/B 间切换 → 超出泛型/`impl` 能力 |

### 5. `dyn Trait` 如何绕开？（类型擦除）

1. 不管底层是 A 还是 B，对外统一 **`&dyn Hello`** 这一种**外部类型**
2. 内部靠**胖指针 + 虚表**，运行时区分真实类型、调用对应方法
3. 编译期只确认：底层 impl 了 trait，**不必关心具体是 A 还是 B**

```rust
static HELLO_A: A = A;
static HELLO_B: B = B;

// ✅ 编译期只认返回类型 &dyn Hello（单一外部类型）
fn get_choose_dyn(flag: bool) -> &'static dyn Hello {
    if flag { &HELLO_A } else { &HELLO_B }
}
```

堆上所有权版：`Box<dyn Hello>`（见 demo `get_hello`）。

### 6. 容器存多类型（同理）

- `Vec<A>` 只能放 A；`Vec<B>` 只能放 B
- `Vec<impl Hello>` 同时存 A、B → ❌
- `Vec<&dyn Hello>` → ✅ 元素统一为 `&dyn Hello`

```rust
let a = A;
let b = B;
let arr: [&dyn Hello; 2] = [&a, &b];
for item in arr {
    item.say();
}
```

### 7. 分场景选型 + 串联总结

| 场景 | 用什么 | 原因 |
|------|--------|------|
| **编译期就知道是哪一种** | 泛型 / `impl Trait` | 静态分发，**不需要 dyn** |
| **同一容器混放多种类型** | `Vec<&dyn Trait>` / `Vec<Box<dyn Trait>>` | `Vec<T>` 只能一种 `T` |
| **运行时分支返回不同类型** | `Box<dyn Trait>` / `&dyn Trait` | `impl Trait` 分支类型必须一致 |
| **异构闭包 / 插件 / 回调列表** | `Vec<Box<dyn Fn()>>` 等 | 每个闭包是不同匿名类型 |

| 问题 | 答案 |
|------|------|
| **为何不定长？** | `dyn T` 指代所有实现 T 的类型，大小各异 → `!Sized` → 须 `&`/`Box` |
| **为何运行时决议？** | 多类型混存、运行时才知具体类型 → 虚表运行期查表 |
| **和泛型取舍？** | 编译期唯一类型 → 泛型/`impl`；多类型切换/共存 → **`dyn`** |

**三句话：**

1. 泛型 / `impl Trait`：**编译期绑定唯一具体类型**，同一位置不能切换多种类型
2. 运行时二选一、多类型共存：需要**类型擦除** → 只能用 **`dyn Trait`**
3. **都 impl 同一 Trait ≠ 是同一类型** — 静态类型语言的核心边界

→ 查表流程：[10.2.6](./10.2.6-vtable通俗全解.md)

## 三、`dyn Trait` 四个核心要点（动态派发）

### 1. 运行期确定真实类型（类型擦除）

- 编译期**不知道**具体是 `Dog` 还是 `Cat`，只知道「实现了 `Animal`」
- 具体类型信息**只在运行时**通过 vtable 解析

### 2. 动态派发（dynamic dispatch）

- 调用方法时**不是**编译期直接绑定到 `Dog::cry` / `Cat::cry`
- 运行时再查该对象对应的方法地址 → 有一点点开销

### 3. 胖指针 + 虚表（vtable）

`&dyn Trait` / `Box<dyn Trait>` 是**胖指针**，**两个指针并列、创建时同时写好**：

| 部分 | 指向 |
|------|------|
| **数据指针** | 实际对象（如 `Dog` 实例） |
| **vtable 指针** | 该**类型**对这个 Trait 的虚表 |

调用方法：**vtable 指针**找函数地址，**数据指针**当 `&self` — **不是**从数据里再查 vtable。

→ **通俗全解（比喻·流程·异构 Vec）**：[10.2.6 vtable 通俗全解](./10.2.6-vtable通俗全解.md) · `cargo run -- vtable_story`

### 4. 异构集合（最实用）

`Vec<Box<dyn Animal>>` 可放**任意实现 `Animal` 的不同类型**；  
普通 `Vec<T>` / 泛型只能放**同一种** `T`。

```rust
let animals: Vec<Box<dyn Animal>> = vec![
    Box::new(Cat),
    Box::new(Dog),
];
for a in animals {
    a.cry(); // 运行时查 vtable
}
```

→ 对象安全等细节：[17.2](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md)

## 四、前置代码（demo 复用）

```rust
trait Animal {
    fn cry(&self);
}

struct Dog;
struct Cat;

impl Animal for Dog { fn cry(&self) { println!("汪汪"); } }
impl Animal for Cat { fn cry(&self) { println!("喵喵"); } }
```

## 五、入参位置对比

### `impl Animal`（静态）

```rust
fn speak_impl(animal: impl Animal) {
    animal.cry();
}
// speak_impl(Dog); speak_impl(Cat); — 各单态化一份代码
```

- ✅ 无 vtable 开销  
- ❌ **不能** `Vec` 混 Dog + Cat

### `&dyn Animal`（动态）

```rust
fn speak_dyn(animal: &dyn Animal) {
    animal.cry();
}

let dog = Dog;
let cat = Cat;
let list: Vec<&dyn Animal> = vec![&dog, &cat];
```

## 六、返回值位置（易错）

| | `-> impl Animal` | `-> Box<dyn Animal>` |
|--|------------------|----------------------|
| 分支多种类型 | ❌ | ✅ |
| 示例 | 固定 `Dog` | `if flag { Dog } else { Cat }` |

> `dyn` 是 **DST**，不能裸放栈 → 必须 `&` / `Box` / `Rc` 等指针。

---

## 七、完整对比表（impl vs dyn）

| 维度 | **`impl Trait`** | **`dyn Trait`** |
|------|------------------|-----------------|
| **分发** | 静态，编译期绑定 | 动态，运行期查 vtable |
| **类型擦除** | ❌ 编译器知道真实类型 | ✅ 编译期不知具体类型 |
| **异构** | ❌ 单一固定类型 | ✅ 多类型共存 |
| **大小** | **Sized**，栈上可直接用 | **DST**，须胖指针包装 |
| **使用位置** | 入参、返回值（常见） | 入参、异构集合、返回值（较少） |
| **二进制体积** | 单态化 → 每种类型一份代码，**可能变大** | 一份动态分发代码，**可能更小** |
| **运行时性能** | 更快（无查表） | 略慢（vtable） |

---

## 八、与泛型 / 口诀

| | 静态 | 动态 |
|--|------|------|
| 写法 | `<T: Animal>` · `impl Animal` | `&dyn Animal` · `Box<dyn Animal>` |

**口诀**：

- **灵活、异构、运行时多态** → `dyn Trait`（胖指针 + vtable）
- **快、静态、单一类型** → `impl Trait`（静态派发）
