## 核心结论（两句）

1. **`impl Trait`**：编译期定类型 · **静态派发** · 单一类型 · 快
2. **`dyn Trait`**：运行期定类型 · **动态派发** · 异构集合 · 胖指针 + vtable

---

## 一、`dyn Trait` 四个核心要点

### 1. 运行期确定真实类型（类型擦除）

- 编译期**不知道**具体是 `Dog` 还是 `Cat`，只知道「实现了 `Animal`」
- 具体类型信息**只在运行时**通过 vtable 解析

### 2. 动态派发（dynamic dispatch）

- 调用方法时**不是**编译期直接绑定到 `Dog::cry` / `Cat::cry`
- 运行时再查该对象对应的方法地址 → 有一点点开销

### 3. 胖指针 + 虚表（vtable）

`&dyn Trait` / `Box<dyn Trait>` 是**胖指针**，**两个指针并列、创建时同时写好**：

| 部分 | 指向 |
|------|------|
| **数据指针** | 实际对象（如 `Dog` 实例） |
| **vtable 指针** | 该**类型**对这个 Trait 的虚表 |

调用方法：**vtable 指针**找函数地址，**数据指针**当 `&self` — **不是**从数据里再查 vtable。

→ **通俗全解（比喻·流程·异构 Vec）**：[10.2.6 vtable 通俗全解](./10.2.6-vtable通俗全解.md) · `cargo run -- vtable_story`

### 4. 异构集合（最实用）

`Vec<Box<dyn Animal>>` 可放**任意实现 `Animal` 的不同类型**；  
普通 `Vec<T>` / 泛型只能放**同一种** `T`。

```rust
let animals: Vec<Box<dyn Animal>> = vec![
    Box::new(Cat),
    Box::new(Dog),
];
for a in animals {
    a.cry(); // 运行时查 vtable
}
```

→ 对象安全等细节：[17.2](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md)

## 二、前置代码（demo 复用）

```rust
trait Animal {
    fn cry(&self);
}

struct Dog;
struct Cat;

impl Animal for Dog { fn cry(&self) { println!("汪汪"); } }
impl Animal for Cat { fn cry(&self) { println!("喵喵"); } }
```

---

## 三、入参位置对比

### `impl Animal`（静态）

```rust
fn speak_impl(animal: impl Animal) {
    animal.cry();
}
// speak_impl(Dog); speak_impl(Cat); — 各单态化一份代码
```

- ✅ 无 vtable 开销  
- ❌ **不能** `Vec` 混 Dog + Cat

### `&dyn Animal`（动态）

```rust
fn speak_dyn(animal: &dyn Animal) {
    animal.cry();
}

let dog = Dog;
let cat = Cat;
let list: Vec<&dyn Animal> = vec![&dog, &cat];
```

---

## 四、返回值位置（易错）

| | `-> impl Animal` | `-> Box<dyn Animal>` |
|--|------------------|----------------------|
| 分支多种类型 | ❌ | ✅ |
| 示例 | 固定 `Dog` | `if flag { Dog } else { Cat }` |

> `dyn` 是 **DST**，不能裸放栈 → 必须 `&` / `Box` / `Rc` 等指针。

---

## 五、完整对比表（impl vs dyn）

| 维度 | **`impl Trait`** | **`dyn Trait`** |
|------|------------------|-----------------|
| **分发** | 静态，编译期绑定 | 动态，运行期查 vtable |
| **类型擦除** | ❌ 编译器知道真实类型 | ✅ 编译期不知具体类型 |
| **异构** | ❌ 单一固定类型 | ✅ 多类型共存 |
| **大小** | **Sized**，栈上可直接用 | **DST**，须胖指针包装 |
| **使用位置** | 入参、返回值（常见） | 入参、异构集合、返回值（较少） |
| **二进制体积** | 单态化 → 每种类型一份代码，**可能变大** | 一份动态分发代码，**可能更小** |
| **运行时性能** | 更快（无查表） | 略慢（vtable） |

---

## 六、与泛型 / 口诀

| | 静态 | 动态 |
|--|------|------|
| 写法 | `<T: Animal>` · `impl Animal` | `&dyn Animal` · `Box<dyn Animal>` |

**口诀**：

- **灵活、异构、运行时多态** → `dyn Trait`（胖指针 + vtable）
- **快、静态、单一类型** → `impl Trait`（静态派发）
