## 一、参数位置（入参）

```rust
trait Summary {
    fn summarize(&self) -> String;
}

// 写法 1：impl Trait（匿名泛型糖）
pub fn notify(item: &impl Summary) { /* ... */ }

// 写法 2：标准泛型 bound
pub fn notify<T: Summary>(item: &T) { /* ... */ }
```

### 共同点

**静态分发**（编译期单态化，无虚表）— 两种写法在多数场景**等价**，编译器为每个具体类型生成专属代码。

### 关键区别

**1. 多参必须同类型 → 只能用泛型 `T`**

```rust
pub fn two_notify<T: Summary>(item1: &T, item2: &T) {} // ✅ 同类型 T

pub fn two_notify(item1: &impl Summary, item2: &impl Summary) {} // 各参数独立匿名类型，可不同
```

**2. 简洁度**：单参数优先 `impl Trait`，少写 `<T: Summary>`。

## 常见误区：`impl Trait` ≠ trait object

> 入参里的 **`impl Trait` 不是运行时多态**，也**不是** `dyn Trait` 的简写。

### 三条线别混

| 写法 | 分发 | 本质 |
|------|------|------|
| **`T: Trait` / `impl Trait`（入参）** | **静态**（编译期单态化） | 泛型 bound 的两种写法，性能等价 |
| **`&dyn Trait` / `Box<dyn Trait>`** | **动态**（运行期 vtable） | trait object，才是「运行时多态」 |
| **`struct Foo<T: Trait>`** | **静态** | 对泛型参数**提前约定**，凡是用到 `Foo<T>` 处 `T` 必须满足 bound |

```rust
use std::fmt::Display;

// ✅ 静态分发 — 和 fn print<T: Display>(x: T) 等价
fn print_impl(x: impl Display) {
    println!("{x}");
}

// ✅ 动态分发 — trait object，走虚表
fn print_dyn(x: &dyn Display) {
    println!("{x}");
}

fn main() {
    print_impl(42);           // 编译期为 i32 生成专属版本
    print_impl("hello");      // 编译期为 &str 生成专属版本
    print_dyn(&42 as &dyn Display); // 运行期通过 vtable 解析 Display
}
```

### 错在哪里

| 误解 | 正解 |
|------|------|
| 「`impl Trait` 参数 = trait object / 语法糖」 | 入参 `impl Trait` 是 **`<T: Trait>` 的语法糖**，仍是静态分发 |
| 「可以传 i32、String，所以是运行时多态」 | 能传多种类型 ≠ 运行时多态；编译器会为**每个具体类型**分别单态化 |
| 「`T: Trait` 是提前约定，`impl Trait` 是运行时接受」 | **两者都是编译期约束**；差别主要是能否复用同一个 `T`（见 §一） |

### 性能（入参）

| 对比 | 结论 |
|------|------|
| **`T: Trait` vs `impl Trait`（入参）** | **基本相同** — 无 vtable，可内联 |
| **`impl Trait` vs `dyn Trait`（入参）** | **`impl` 通常更快** — `dyn` 多间接调用 + 胖指针 |
| **返回 `-> impl Trait` vs `Box<dyn Trait>`** | 前者零开销抽象；后者堆分配 + 动态派发 |

口诀：**参数/返回用 `impl` → 编译期定类型；要异构集合或擦除类型 → `dyn`**。完整对比 → [10.2.5](./10.2.5-impl与dyn-Trait对比.md)。

## 二、返回位置 `-> impl Trait`（泛型无法等价替代）

> **实现方选定唯一具体类型**，对外隐藏真实类型；编译期确定。

```rust
struct Tweet { /* ... */ }
impl Summary for Tweet { /* ... */ }

fn get_msg() -> impl Summary {
    Tweet { /* ... */ }
}
```

### 硬性规则：只能返回单一固定类型

```rust
// ❌ if → News，else → Tweet，两种实体类型
fn get(flag: bool) -> impl Summary {
    if flag { News { /* ... */ } } else { Tweet { /* ... */ } }
}
```

运行期多种实现 → **`Box<dyn Summary>`** → [17.2](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md)

### 对比泛型返回 `-> T`

```rust
fn gen<T: Summary>() -> T { todo!() }
```

| 写法 | 谁选类型 |
|------|----------|
| **`-> impl Trait`** | **实现方**（函数体内写死） |
| **`-> T`（泛型）** | **调用方**（实参指定 T） |

## 三、两大使用场景

| 位置 | 含义 | 替代 |
|------|------|------|
| **`arg: impl Trait`** | 接收任意实现 trait 的类型 | `<T: Trait>` |
| **`-> impl Trait`** | 返回**单个**实现 trait 的固定类型 | 多类型用 `dyn` |

## 四、高频实用场景

### 1. 返回闭包（类型无法手写）

```rust
fn make_closure() -> impl Fn(i32) -> i32 {
    |x| x + 5
}
```

### 2. 多 trait 约束

```rust
use std::io::{Cursor, Read, Write};

fn io_handle() -> impl Read + Write {
    Cursor::new(vec![])
}
```

## 五、`impl Trait` vs `dyn Trait`

→ **完整对比（入参 · 返回 · 二选一 · Vec · 口诀）**：[10.2.5 impl vs dyn](./10.2.5-impl与dyn-Trait对比.md) · 尤其 [§二 为何 impl 不能二选一](./10.2.5-impl与dyn-Trait对比.md#二为何泛型--impl-trait-做不了运行时二选一) · demo：`10.2.5-impl-dyn-demo/`

| | **`impl Trait`** | **`dyn Trait`** |
|--|------------------|-----------------|
| 分发 | **静态**，编译期 | **动态**，运行期虚表 |
| 类型数 | **单一**固定 | **多种** |
| 开销 | 无 vtable | 有 vtable |

## 六、易错：trait 里不能直接 `fn() -> impl Trait`（简述）

在 **trait 定义**里写：

```rust
trait Factory {
    fn make() -> impl Summary; // ❌ 经典报错（trait 方法返回 impl 受限）
}
```

**原因（考点）**：trait 是**契约**，调用方需要知道返回类型布局；`-> impl Trait` 在 trait 上等于「每个 impl 可以返回不同隐藏类型」，破坏统一接口。

**常见替代**：

| 方案 | 写法 |
|------|------|
| **关联类型** | `type Output; fn make() -> Self::Output;` |
| **泛型方法** | `fn make<T: Summary>() -> T`（由调用方指定，少见） |
| **trait 对象** | `fn make() -> Box<dyn Summary>` |

> Rust 新版本有 **RPITIT**（trait 内返回位置 impl）等演进；学习阶段先记：**trait 签名里别随便写 `-> impl Trait`，用关联类型或 `Box<dyn>`**。
