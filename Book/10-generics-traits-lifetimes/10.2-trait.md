## 定义与实现

```rust
pub trait Summary {
    fn summarize(&self) -> String;
}
```

孤儿规则：只能为**本 crate 的 trait** 或**本 crate 的类型**实现 trait。

默认实现见 demo `Tweet` 使用 `summarize` 默认版。

## 一、函数参数：两种传 Trait 的写法

### 写法 1：`impl Trait`（语法糖）

→ 全解：[10.2.3 impl Trait](./10.2.3-impl-Trait全解.md)

```rust
pub fn notify(item: &impl Summary) {
    println!("Breaking news! {}", item.summarize());
}
```

编译器自动推导隐藏泛型，**每个 `impl Trait` 参数可独立是不同类型**。

### 写法 2：泛型 `T` + trait bound

```rust
pub fn notify<T: Summary>(item: &T) { /* ... */ }
```

**可复用同一个 `T`** — 多参数强制同类型：

```rust
fn notify2<T: Summary>(a: &T, b: &T) {} // a、b 必须同类型 T

// impl 版：a、b 可以是不同具体类型，各自实现 Summary
fn notify2_impl(a: &impl Summary, b: &impl Summary) {}
```

### 多约束：`TraitA + TraitB`

```rust
pub fn notify(item: &impl Summary + Display) {}
pub fn notify<T: Summary + Display>(item: &T) {}
```

`+` = **并且**，同时实现多个 trait。

### 约束太多 → `where` 分句

```rust
fn f<T, U>(t: T, u: U)
where
    T: Display + Clone,
    U: Clone + Debug,
{ }
```

## 二、返回值：`-> impl Trait`

```rust
fn returns_summarizable() -> impl Summary {
    Post {}
}
```

### 硬性限制（易错）

**不能分支返回两种不同具体类型**：

```rust
// ❌ if 返回 Post，else 返回 News — 编译失败
fn bad(flag: bool) -> impl Summary {
    if flag { Post {} } else { News {} }
}
```

## 三、`largest`：三种 trait bound 方案

| 方案 | 签名 | 特点 |
|------|------|------|
| **Copy** | `T: PartialOrd + Copy` → `T` | 简单，大类型拷贝贵 |
| **Clone** | `T: PartialOrd + Clone` → `T` | 堆数据可，有 clone 开销 |
| **引用（推荐）** | `T: PartialOrd` → `&T` | 只借用，工业常用 |

```rust
fn largest_copy<T: PartialOrd + Copy>(list: &[T]) -> T { /* ... */ }
fn largest_clone<T: PartialOrd + Clone>(list: &[T]) -> T { /* list[0].clone() ... */ }
fn largest_ref<T: PartialOrd>(list: &[T]) -> &T { /* ... */ }
```

- **`PartialOrd`**：支持 `>` 比较

## 四、条件 impl + Blanket impl

> 两种写法**长得像**（都有 `impl<T: Bound>`），**作用完全不同** — 一个给**结构体加条件方法**，一个给**所有类型批量 impl trait**。

### 对比表（一眼分清）

| | **普通 impl** | **条件 impl** | **Blanket impl** |
|--|---------------|---------------|------------------|
| **写法** | `impl<T> Struct<T> { }` | `impl<T: Bound> Struct<T> { }` | `impl<T: Bound> Trait for T { }` |
| **作用对象** | 结构体/枚举 | 结构体/枚举 | **trait**（给类型自动加能力） |
| **效果** | 方法**永远存在** | **T 满足 Bound 才有**这批方法 | **所有满足 Bound 的 T** 自动实现 `Trait` |
| **T 不满足时** | — | 调用条件方法 → **编译报错（无此方法）** | 该类型本来就没有这个 trait |
| **典型场景** | `new`、`len` 等通用方法 | 需要 `Clone` 才能 `dup` | `Display` → `ToString` |

→ 七种写法逐条展开：[10.2.2 §6–7](./10.2.2-Trait七种写法详解.md) · demo：`10.2.2-trait-syntax-demo/` §6–7

### 1. 条件 impl：给结构体加「条件方法块」

**写法（两种等价）**

```rust
// 写法 A：bound 写在泛型参数上
impl<T: Bound> Struct<T> {
    fn conditional_method(&self) { /* ... */ }
}

// 写法 B：bound 写在 where 子句（约束多、签名长时更清晰）
impl<T> Struct<T>
where
    T: Bound,
{
    fn conditional_method(&self) { /* ... */ }
}
```

**含义**

- 同一个 `Struct<T>` 可以写**多个 impl 块**
- 无条件块 → 方法永远存在；有条件块 → **T 不满足 Bound 时，编译器认为根本没有这些方法**

```rust
use std::fmt::Display;

struct MyStruct<T> {
    data: T,
}

// 无条件：所有 T 都有 new
impl<T> MyStruct<T> {
    fn new(data: T) -> Self {
        Self { data }
    }
}

// 有条件：仅 T: Display 才有 print_data（where 写法）
impl<T> MyStruct<T>
where
    T: Display,
{
    fn print_data(&self) {
        println!("{}", self.data);
    }
}

// 等价写法：impl<T: Display> MyStruct<T> { fn print_data ... }

// 有条件：仅 T: Clone 才有 dup
impl<T: Clone> MyStruct<T> {
    fn dup(&self) -> Self {
        Self { data: self.data.clone() }
    }
}

fn main() {
    let s = MyStruct::new(42); // i32: Display + Clone
    s.print_data();             // ✅ T: Display → 方法存在
    let _ = s.dup();            // ✅ T: Clone

    // struct NoDisplay;
    // MyStruct::new(NoDisplay).print_data(); // ❌ 无 print_data 方法
}
```

> **逻辑**：只有 `MyStruct<T>` 里的 `T` 实现了 `Display`，`print_data` 才「存在」；否则编译期直接报错 — 不是运行期检查。

**非泛型类型：用不上 impl 上的 `where`**

`where` 搭配泛型约束，是为**有泛型参数**的结构体/枚举/函数提供「条件式实现」。若类型本身不是泛型，字段类型已固定，impl 直接写即可：

```rust
struct Fixed {
    data: i32,
}

// ✅ 普通 impl — data 已是 i32，方法固定基于 i32
impl Fixed {
    fn print_data(&self) {
        println!("{}", self.data);
    }
}

// ❌ 无泛型参数，where 无处约束 — 编译报错
// impl Fixed where i32: Display { ... }
```

| | **泛型 `Struct<T>`** | **非泛型 `Struct`** |
|--|------------------------|---------------------|
| 有无泛型参数 | 有 `T` | 无 |
| 条件 impl | `impl<T: Bound>` 或 `impl<T> where T: Bound` | 不需要；普通 `impl Struct { }` |
| `where` 作用 | 约束 **T** 满足 bound 才启用这批方法 | **不适用**（没有 T 可约束） |

函数参数上的 `where`（如 `fn f<T>(x: T) where T: Display`）同理 — 服务于**泛型**，与非泛型函数无关。

本篇 demo：`MyStruct`（§5 · `dup` / `print_data`）· `Pair<T>` + `cmp_display`（§6）演示同一机制。

### 2. Blanket impl（blanket implementation）

**写法**

```rust
impl<T: Bound> MyTrait for T {
    // 所有满足 Bound 的类型 T，自动获得 MyTrait
}
```

**含义（一句话）**

> **所有实现了 `Bound` 的类型 `T`，都自动实现 `MyTrait`** — 不用给 `i32`、`String`、每个自定义类型逐个写 `impl MyTrait for …`。

也叫 **blanket implementation**（通配 / 地毯式实现）：编译器对**每一个**满足 `T: Bound` 的具体类型，自动生成一份 impl。

**好处**

- **消灭重复**：同一套逻辑只写一次
- **自动扩展**：以后新类型只要 impl 了 `Bound`，就**自动**拥有 `MyTrait`，无需再改 blanket 块
- 标准库大量使用：`Display` → `ToString`、`Borrow<T>` → `Borrow<T>` 等

**完整例子**

```rust
trait PrintMe {
    fn print_me(&self);
}

// 所有实现 Display 的类型，自动有 PrintMe
impl<T: std::fmt::Display> PrintMe for T {
    fn print_me(&self) {
        println!("{}", self);
    }
}

fn main() {
    123.print_me();       // i32: Display → ✅
    "hi".print_me();      // &str: Display → ✅
    // 自定义类型 impl Display 后，也自动能 .print_me()
}
```

> 方法名随意（`print` / `print_me` 都行）；关键是 **impl 块写一次，覆盖所有 `T: Display`**。

**标准库经典**

```rust
impl<T: Display> ToString for T { /* ... */ }

let s: i32 = 10;
s.to_string(); // i32 有 Display → 自动有 ToString，无需手写 impl ToString for i32
```

**注意（初学易混）**

| | **Blanket impl** | **条件 impl** |
|--|------------------|-----------------|
| 写法 | `impl<T: Bound> Trait for T` | `impl<T: Bound> Struct<T>` |
| 给谁加能力 | 给**类型**加一个新 **trait** | 给**结构体**加一批**方法** |
| 例子 | `Display` → `PrintMe` | `MyStruct<T: Clone>` 才有 `dup` |

---

### 3. 一句话

| 写法 | 一句话 |
|------|--------|
| **`impl<T: Bound> Struct<T>`** | 给结构体加条件方法：**T 满足 Bound 才有，否则没有** |
| **`impl<T: Bound> Trait for T`** | 给**所有满足 Bound 的类型**自动 impl trait（blanket） |
