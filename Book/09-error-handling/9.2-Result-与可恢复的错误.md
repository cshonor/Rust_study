# 9.2 Result 与可恢复的错误

← [9.1 panic!](./9.1-panic-与不可恢复的错误.md) · [9.3 panic vs Result](./9.3-使用panic还是不用panic.md) · demo：`9.2-result-demo/`

> 可恢复错误用 **`Result<T, E>`**，调用者决定如何处理 — 不必立刻 panic。

---

## 一、`File::open` 返回类型

```rust
// File::open("hello.txt") → Result<成功类型, 错误类型>
// 完整：Result<std::fs::File, std::io::Error>
```

| 分支 | 含义 |
|------|------|
| **`Ok(File)`** | 打开成功，携带文件句柄 |
| **`Err(io::Error)`** | 失败（不存在 / 权限 / 路径非法等） |

`Result` 标准定义：

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

---

## 二、单层 `match`（最简）

```rust
use std::fs::File;

let res = File::open("hello.txt");
let file = match res {
    Ok(f) => f,
    Err(e) => panic!("打开失败: {e:?}"),
};
```

**`match` 必须穷尽 `Ok` / `Err` 两分支**，编译器强制校验。

---

## 三、嵌套 `match` + `error.kind()`（书本经典）

需求：**不存在 → 自动创建；其余错误 → panic**

```rust
use std::fs::File;
use std::io::ErrorKind;

let file = match File::open("hello.txt") {
    Ok(f) => f,
    Err(e) => match e.kind() {
        ErrorKind::NotFound => match File::create("hello.txt") {
            Ok(new_f) => new_f,
            Err(create_err) => panic!("创建失败: {create_err:?}"),
        },
        other => panic!("未知打开错误: {other:?}"),
    },
};
```

### `e.kind()` 是什么？（先搞清这个）

```rust
Err(e)  // e 的类型是 std::io::Error（具体 IO 失败信息）
e.kind() // 调用方法 → 返回 std::io::ErrorKind（错误「大类」枚举）
```

| 层次 | 类型 | 例子 |
|------|------|------|
| **`io::Error`** | 完整错误（含 OS 原始码、路径等） | `"No such file or directory"` |
| **`.kind()`** | 把上面归类成**跨平台统一枚举** | `ErrorKind::NotFound` |

为什么用 `kind()` 而不是直接比字符串？—— Windows / Linux 报错文案不同，**`ErrorKind::NotFound` 在各平台一致**，适合写 `match`。

```rust
use std::io::{Error, ErrorKind};

fn show(e: Error) {
    match e.kind() {
        ErrorKind::NotFound => println!("文件不存在"),
        ErrorKind::PermissionDenied => println!("没权限"),
        _ => println!("其它 IO 错误: {e:?}"),
    }
}
```

### 关键点

1. **`io::Error.kind() -> ErrorKind`**：系统错误归类为枚举
2. **`ErrorKind` 带 `#[non_exhaustive]`** → 必须写 **`other` 兜底**，不能穷举所有变体

### 常用 `ErrorKind`

| 值 | 含义 |
|----|------|
| `NotFound` | 文件 / 路径不存在 |
| `PermissionDenied` | 无读写权限 |
| `AlreadyExists` | 创建时已存在 |
| `TimedOut` | IO 超时 |
| `UnexpectedEof` | 意外读到 EOF |
| `Other` | 未归类错误 |

## 四、简化：`unwrap_or_else`（减少嵌套）

```rust
let file = File::open("hello.txt").unwrap_or_else(|e| {
    if e.kind() == ErrorKind::NotFound {
        File::create("hello.txt").unwrap()
    } else {
        panic!("打开异常: {e:?}");
    }
});
```

`Err` 时执行闭包 — 生产里比三层 `match` 更常见。

---

## 五、`unwrap` / `expect` / `?` 对比

| 写法 | 行为 | 适用 |
|------|------|------|
| **`match`** | 穷尽分支，完全控制 | 要区分错误类型 |
| **`unwrap()`** | `Err` → panic（固定信息） | demo / 确定不会失败 |
| **`expect("msg")`** | `Err` → panic（自定义前缀） | 同上，信息更清晰 |
| **`?`** | `Err` → 提前返回给调用者 | 函数返回 `Result` 时传播错误 |

```rust
fn read() -> Result<String, io::Error> {
    let mut s = String::new();
    File::open("hello.txt")?.read_to_string(&mut s)?;
    Ok(s)
}
```

→ 何时 panic vs Result：[9.3](./9.3-使用panic还是不用panic.md)
