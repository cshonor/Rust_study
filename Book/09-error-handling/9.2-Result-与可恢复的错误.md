# 9.2 Result 与可恢复的错误

很多错误并不需要让程序直接崩溃（panic）。Rust 用 `Result<T, E>` 表达“可能失败”的返回值，让调用者决定如何处理错误。

对应 demo 项目：`Book/09-error-handling/9.2-result-demo/`

---

## `Result<T, E>`

标准库里的 `Result` 形式是：

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

- `T`：成功时 `Ok(T)` 里的值类型
- `E`：失败时 `Err(E)` 里的错误类型

例如 `File::open("hello.txt")` 的类型是：
`Result<std::fs::File, std::io::Error>`。

---

## 用 `match` 处理 `Result`

最基础的写法是 `match`：

- `Ok(v)`：取出成功值继续执行
- `Err(e)`：按需求处理（比如返回、打印、或 panic）

---

## 匹配不同错误：`io::ErrorKind`

当你想“文件不存在就创建，其他错误就报错”，可以在 `Err(error)` 分支里用 `error.kind()` 判断：

- `ErrorKind::NotFound`：尝试 `File::create`
- 其他：按需要处理（比如直接返回 Err 或 panic）

---

## `unwrap` 与 `expect`（失败时直接 panic 的快捷方式）

- `unwrap()`：`Ok` 返回值；`Err` 就 panic（信息固定）
- `expect("msg")`：`Ok` 返回值；`Err` panic（信息以你给的 msg 开头，更好排查）

一般用于：

- demo / 快速原型
- “这里失败就是 bug”的场景（确保 msg 写清楚）

---

## 传播错误（propagate）

当函数内部没有足够上下文决定怎么处理错误时，可以把 `Err` 返回给调用者：

- match 版：`Err(e) => return Err(e)`
- `?` 版：`File::open(...)?;`（`Err` 会提前返回）

`?` 只能用在返回 `Result`（或实现了 `Try` 的类型）的函数中；`main` 也可以返回 `Result`，这样就能在 `main` 里用 `?`。

---

## demo 运行

```bash
cd Book/09-error-handling/9.2-result-demo

# 文件不存在就创建（或打开），并写入一行
cargo run -- open_or_create

# 读取 username（分别演示 match 版与 ? 版）
cargo run -- read_match
cargo run -- read_q

# 演示 expect/unwrap（会因文件缺失而 panic）
cargo run -- expect
```
