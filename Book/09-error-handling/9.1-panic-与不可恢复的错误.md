## 结论（先看这三句）

1. **默认 `unwind`**：栈回溯 + 逐层 `Drop`；**`abort`**：直接退出、不跑 `Drop`，二进制更小、开销更低。
2. **panic 分主动**（`panic!`）**与被动**（越界、`unwrap` 遇 `None` 等运行时检查）。
3. **HFT 生产 release 常用 `panic = "abort"`** → 完整 profile 对照：[9.1.2](./9.1.2-dev-release与HFT编译配置.md)

## 一、两种 panic 策略

| 策略 | 行为 | 默认 | 特点 |
|------|------|------|------|
| **`unwind`（展开）** | 从当前函数逐层回溯栈，执行各层 **`Drop`** 再退出 | debug / release 默认 | 能析构释放堆资源；开销偏大 |
| **`abort`（终止）** | **SIGABRT** 直接杀进程，**不展开、不跑 Drop** | 需手动配置 | 体积更小、运行更快 |

```toml
# Cargo.toml — release 启用 abort
[profile.release]
panic = "abort"
```

> **HFT 生产标配 `panic = "abort"`**：致命故障直接销毁进程重启，避免 unwind 耗时、残留脏内存导致订单错乱。

### 与 `catch_unwind` 的关系

→ 完整版：[9.1.3 catch_unwind 与 panic 模式](./9.1.3-catch_unwind与panic模式.md)

- **`unwind`**：`catch_unwind` 可拦截栈展开  
- **`abort`**：`catch_unwind` **无效**，进程直接退出

## 二、panic 触发分类

### 1. 主动：`panic!`

```rust
panic!("致命错误：{}", code);
```

手动抛出**不可恢复**错误。

### 2. 被动：运行时逻辑错误

Rust 运行时检查自动 panic，例如：

| 场景 | 示例 |
|------|------|
| 数组越界 | `v[99]` |
| `unwrap` 遇 `None` / `Err` | `opt.unwrap()` |
| 空切片切分 | `split_at` 越界 |
| 整数溢出 | **debug 默认 panic** |

```bash
cargo run -- oob      # Vec 越界
cargo run -- unwrap   # None.unwrap()
```

### release 整数溢出

| 模式 | 溢出行为 |
|------|----------|
| **debug** | panic |
| **release 默认** | **环绕溢出**（wrapping，不 panic） |

如需 release 也 panic：

```toml
[profile.release]
overflow-checks = true
```

## 三、Backtrace 调试

→ 完整用法：[9.1.1 Backtrace 调试](./9.1.1-Backtrace调试.md)（三种场景 · 手动 `std::backtrace` · HFT 生产）

```powershell
$env:RUST_BACKTRACE=1; cargo run -- oob   # 9.1-panic-demo
```

**读栈技巧**：堆栈里**最先出现你自己项目源码文件名的行** ≈ 出错源头。

## demo 与指令

```bash
cd Book/09-error-handling/9.1-panic-demo

cargo run -- panic     # 主动 panic!
cargo run -- oob       # Vec 越界（被动）
cargo run -- unwrap    # None.unwrap()（被动）
cargo run -- catch     # catch_unwind（须 unwind 模式）
cargo run -- help
```

|-----------|----------|
| `panic` | §二 主动 |
| `oob` / `unwrap` | §二 被动 |
| `catch` | §一 `catch_unwind`（abort 模式下无效） |

```powershell
$env:RUST_BACKTRACE=1; cargo run -- oob
```
