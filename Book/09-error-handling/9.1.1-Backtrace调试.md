# 9.1.1 Backtrace 完整调试用法

← 主笔记：[9.1 panic!](./9.1-panic-与不可恢复的错误.md) · demo：`9.1.1-backtrace-demo/`

> 三种场景：**panic 自动堆栈** · **手动打印栈** · **catch 后自定义打印**（HFT 开发/生产选型见 §四）。

---

## 一、环境变量：panic 自动堆栈（日常调试）

### Linux / macOS

```bash
export RUST_BACKTRACE=1    # 精简堆栈
export RUST_BACKTRACE=full # 完整堆栈（推荐调试）
export RUST_BACKTRACE=0    # 关闭
cargo run -- oob
```

### Windows（PowerShell）

```powershell
$env:RUST_BACKTRACE=1
cargo run -- oob
```

| 值 | 效果 |
|----|------|
| **`0`** | 关闭 |
| **`1`** | 精简堆栈 |
| **`full`** | 完整带符号堆栈 |

**release 默认无调试符号 → 看不到行号**。需要 release 出栈：

```toml
[profile.release]
debug = true
```

### 示例：越界自动打栈

```rust
fn main() {
    let arr = vec![1, 2, 3];
    arr[10]; // 越界 panic → 配合 RUST_BACKTRACE 输出堆栈
}
```

## 二、手动打印当前调用栈（不 panic）

### 方案 1：`std::backtrace`（Rust 1.65+，无需第三方）

```rust
use std::backtrace::Backtrace;

fn func_b() {
    let bt = Backtrace::capture();
    println!("当前调用堆栈:\n{bt}");
}

fn func_a() {
    func_b();
}

fn main() {
    func_a();
}
```

| API | 作用 |
|-----|------|
| **`Backtrace::capture()`** | 捕获当前栈（按需解析） |
| **`Backtrace::force_capture()`** | 强制全量捕获 |

手动栈解析常配合：

```bash
RUST_LIB_BACKTRACE=1 cargo run -- manual
```

release 要行号同样设 `[profile.release] debug = true`。

### 方案 2：第三方 `backtrace` crate（老版本 Rust）

```toml
backtrace = "0.3"
```

```rust
use backtrace::Backtrace;

fn test() {
    let bt = Backtrace::new();
    println!("{bt:?}");
}
```

## 三、`catch_unwind` + 自定义打印堆栈

> **`panic = "abort"` 时无效** — 进程直接退出，无法捕获。

```rust
use std::backtrace::Backtrace;
use std::panic::catch_unwind;

fn risky_code() {
    let v = vec![1];
    v[5];
}

fn main() {
    let res = catch_unwind(risky_code);
    if res.is_err() {
        println!("异常，手动堆栈:\n{}", Backtrace::force_capture());
    }
}
```

→ panic 策略见 [9.1 §一](./9.1-panic-与不可恢复的错误.md)。

## 四、HFT 落地规范

| 环境 | 配置 |
|------|------|
| **开发** | `RUST_BACKTRACE=full`，快速定位越界、无锁队列问题 |
| **生产 release** | 见下表 |

```toml
[profile.release]
panic = "abort"         # 禁用栈展开
debug = false           # 减体积；不靠 Rust backtrace 排错
overflow-checks = false
```

生产排查改用 **`perf record -g ./app`** 等系统级采样，而非内置 backtrace。

---

## 五、常见问题

| 现象 | 处理 |
|------|------|
| 堆栈只有地址、无行号 | `debug = true` + `RUST_BACKTRACE=full` |
| `abort` 下 `catch_unwind` 无效 | 设计如此；HFT 生产选型 |
| 手动栈为空/ `<disabled>` | 设 `RUST_LIB_BACKTRACE=1` 或用 `force_capture()` |
