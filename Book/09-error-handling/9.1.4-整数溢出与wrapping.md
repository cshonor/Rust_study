# 9.1.4 整数溢出：wrapping · panic · saturating

← [9.1.2 dev/release 配置](./9.1.2-dev-release与HFT编译配置.md) · demo：`9.1.4-overflow-demo/`

---

## 一、环绕溢出（Wrapping，release 默认）

### 一句话

**release**（`overflow-checks = false`）：无符号/有符号整数加减**不 panic**，按二进制位数**截断绕回**，等价于模 2ⁿ。

**dev**（`overflow-checks = true`）：裸运算溢出 → **panic**。

### 示例：`u8`（0～255，2⁸）

```rust
let mut x: u8 = 0;
x -= 1;
// dev：panic 整数溢出
// release：0 - 1 → 255

let mut y: u8 = 255;
y += 1;
// release：255 + 1 → 0
```

规则：**结果 = 实际值 mod 2^类型位数**

### 有符号 `i32`（补码）

| 运算 | release 环绕结果 |
|------|------------------|
| `-2147483648 - 1` | `2147483647` |
| `2147483647 + 1` | `-2147483648` |

## 二、三种溢出模式

| 模式 | 行为 | 何时出现 |
|------|------|----------|
| **wrapping（环绕）** | 折返，不 panic | **release 裸运算默认** |
| **panicking** | 溢出 panic | **dev 裸运算默认**（`overflow-checks = true`） |
| **saturating（饱和）** | 触界停在最值 | 须显式 API |

```rust
0u8.wrapping_sub(1)    // 255（强制环绕）
0u8.saturating_sub(1)  // 0（饱和，不再减）
255u8.saturating_add(1) // 255
```

→ 不改全局 Cargo，用 **`.wrapping_*` / `.saturating_*` / `.checked_*`** 控制单点行为。

---

## 三、HFT 注意点

1. **订单计数、Tick 序列号严禁依赖默认环绕** — 绕回会造成严重逻辑错误。
2. **推荐** `checked_add` / `checked_sub` → `Option<T>`，溢出为 `None`，走风控：

```rust
let x = 1u32;
match x.checked_add(u32::MAX) {
    Some(v) => { /* 正常 */ }
    None => { /* 溢出 → 告警 / 拒单 */ }
}
```

3. **不要**为 HFT 全局开 `[profile.release] overflow-checks = true`（性能损耗）；只在**关键路径**手动校验。

---

## 四、Cargo 全局：让 release 溢出也 panic

```toml
[profile.release]
overflow-checks = true
```

开启后 release 裸运算溢出与 dev 一致，直接 panic。

→ profile 总览：[9.1.2 §三](./9.1.2-dev-release与HFT编译配置.md)
