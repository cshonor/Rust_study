# 9.1.2 dev / release · panic · 溢出检查（HFT 编译配置）

← 主笔记：[9.1 panic!](./9.1-panic-与不可恢复的错误.md) · [9.1.1 Backtrace](./9.1.1-Backtrace调试.md)

> 一次性讲清：**只有 dev / release 两大 profile** · **`debug` 是配置项不是模式** · panic · overflow · backtrace 搭配。

---

## 先纠正：只有两大类内置 Profile

Cargo **内置两种编译方案**，没有第三种叫「debug」的模式：

```toml
[profile.dev]     # cargo build / cargo run 默认
[profile.release] # cargo build --release 默认
```

**`debug` 不是独立 profile**，而是 profile 里的**配置字段**：`debug = true/false`（是否生成调试符号）。dev / release 里都能改。

| 容易混淆 | 是什么 |
|----------|--------|
| **`dev` / `release`** | Cargo **两种编译 profile**（日常只关心这两个） |
| **`debug = true/false`** | profile 内**开关**，控制符号表；release 也可设 `debug = true` |
| **`cargo debug`** | ❌ **没有这条命令**；只有 `cargo build`（dev）和 `cargo build --release` |

**额外**：还可自定义 `[profile.bench]`、`[profile.test]` 等，极少手写；HFT 日常 **dev + release 足够**。

---

## 结论先说：出厂 dev / release **全是 `panic = "unwind"`**

**不写 `[profile.*]` 的 panic 项 = dev、release 默认都是 unwind。** 只有手动在 `Cargo.toml` 里写 `panic = "abort"` 才会变成终止模式。

```toml
# Cargo 原厂默认（两行等价，通常省略不写）
# [profile.dev]     panic = "unwind"
# [profile.release] panic = "unwind"
```

| Profile | 出厂 panic | 溢出（裸运算） |
|---------|------------|----------------|
| **dev** | **unwind** | 检查开启 → 溢出 **panic** |
| **release** | **unwind** | 检查关闭 → **环绕**，不 panic |

1. **不写配置 → dev / release 全是 unwind**
2. **想要 abort → 必须手动改 `Cargo.toml`**（HFT release 常见）

→ unwind / abort **一行对比 demo**：[9.1.3-catch-unwind-demo](./9.1.3-catch-unwind-demo/)

## 一句话

| 场景 | 配置 |
|------|------|
| **写代码调试** | `dev` + `unwind` + `RUST_BACKTRACE=full` |
| **HFT 上线** | `release` + `panic = "abort"` + `debug = false` |

## 一、两种编译 profile

| 命令 | Profile | 用途 |
|------|---------|------|
| `cargo build` / `cargo run` | **`dev`** | 开发调试 |
| `cargo build --release` | **`release`** | 发行 / 上线 |

### 1. `[profile.dev]`（`cargo build` / `cargo run` 默认）

```toml
[profile.dev]
opt-level = 0           # 无优化
debug = true            # 带符号 → backtrace 显示行号
panic = "unwind"        # 栈展开，可 catch_unwind
overflow-checks = true  # 整数溢出 → panic
```

- 编译快、运行慢，方便断点 / backtrace
- **只开发用，HFT 绝不上线 dev**

### 2. `[profile.release]`（`cargo build --release` 默认）

**出厂默认：**

```toml
[profile.release]
opt-level = 3            # 最高优化
debug = false            # 无符号 → 堆栈只有地址
panic = "unwind"
overflow-checks = false  # 整数溢出 → 环绕 wrapping，不 panic
```

**HFT 上线只改一项（常见）：**

```toml
[profile.release]
panic = "abort"          # ← 关键；其余保持出厂默认即可
```

## 二、panic：`unwind`（出厂默认）vs `abort`（手动改）

### `unwind` 做什么？（dev / release 出厂默认）

panic 触发后：

1. 从出错位置**逐层回溯栈**
2. 栈上变量自动 **`Drop`** 释放资源
3. 可被 **`catch_unwind`** 拦截，程序未必直接退出
4. **子线程 panic** 通常只崩该线程，主线程可继续（abort 则整进程退出）

### `abort`：只有手动配置才有（HFT release 常用）

```toml
[profile.release]
panic = "abort"
```

- 直接 **`abort()`** 杀进程 · **不展开、不 Drop** · **`catch_unwind` 无效**
- **任意线程 panic → 整程序退出**

| | **`unwind`（出厂默认）** | **`abort`（手动改）** |
|--|--------------------------|------------------------|
| **Drop** | 执行 | 不执行 |
| **`catch_unwind`** | ✅ | ❌ |
| **子线程 panic** | 通常只崩该线程 | 整进程退出 |
| **适用** | 开发、普通服务 | **HFT 实盘** |

### HFT 逻辑

致命错误 → **立刻杀进程重启**，远比带病运行、脏内存破坏订单簿稳妥。

→ 细节：[9.1 §一](./9.1-panic-与不可恢复的错误.md) · [9.1.1 §三](./9.1.1-Backtrace调试.md)

## 三、`overflow-checks` 整数溢出

→ 完整说明：[9.1.4 整数溢出与 wrapping](./9.1.4-整数溢出与wrapping.md)（三种模式 · `checked_*` · HFT）

| 配置 | 行为 | 默认出现在 |
|------|------|------------|
| **`true`** | 裸运算溢出 → **panic** | `dev` |
| **`false`** | **环绕 wrapping**，不 panic | `release` |

HFT：**保持 `false`**，关键路径用 **`checked_add/sub`**，勿依赖默认绕回。

```toml
[profile.release]
overflow-checks = true   # 仅当需要 release 也 panic 时
```

## 四、与 Backtrace 搭配

| 场景 | 做法 |
|------|------|
| **日常查 bug** | `RUST_BACKTRACE=full cargo run`（dev + unwind，行号完整） |
| **release 临时排查** | 在 **`[profile.release]`** 里设 `debug = true; panic = "unwind"`（仍是 release profile，不是「debug 模式」）→ `cargo build --release` + `RUST_BACKTRACE=1` |
| **HFT 正式上线** | `release` + `abort` + `debug = false`；排错用 **`perf record -g ./app`**，不用 Rust backtrace |

→ 完整 backtrace 用法：[9.1.1](./9.1.1-Backtrace调试.md)

| `debug` | true | false | false |

## 附录：HFT 项目 `Cargo.toml` 模板（可直接复制）

```toml
[package]
name = "hft_engine"
version = "0.1.0"
edition = "2021"

[dependencies]
# …

# ── 开发：默认 dev profile，无需覆写 ──

# ── 生产 release：HFT 标配 ──
[profile.release]
opt-level = 3
debug = false
overflow-checks = false
panic = "abort"

# ── 可选：release 临时带符号排查（勿上实盘）──
# [profile.release-with-debug]
# inherits = "release"
# debug = true
# panic = "unwind"
# 使用：cargo build --profile release-with-debug
```

```bash
# 开发
RUST_BACKTRACE=full cargo run

# 上线编译
cargo build --release

# 临时 release 排查（自定义 profile 时）
# RUST_BACKTRACE=1 cargo run --profile release-with-debug
```
