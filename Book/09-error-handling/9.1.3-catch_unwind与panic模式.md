# 9.1.3 catch_unwind 与 panic 模式

← [9.1 panic!](./9.1-panic-与不可恢复的错误.md) · [9.1.2 dev/release 配置](./9.1.2-dev-release与HFT编译配置.md) · demo：`9.1.3-catch-unwind-demo/`

---

## 一、核心结论

`std::panic::catch_unwind` **只在 `panic = "unwind"` 下生效**；**`panic = "abort"` 完全失效**。

| 模式 | panic 时 | `catch_unwind` |
|------|----------|----------------|
| **`unwind`**（dev 默认、release 出厂） | 逐层回溯 + 执行 **`Drop`** | ✅ 可拦截，返回 `Result<T, Box<dyn Any + Send>>`，程序继续 |
| **`abort`**（HFT 生产常用） | OS **`abort()`**，不展开、不 Drop | ❌ 无捕获机会，进程直接被杀死 |

---

## 二、unwind 下完整流程

```rust
use std::panic;

fn inner() {
    let _s = String::from("临时资源"); // 实现 Drop
    panic!("出错了");
}

fn main() {
    let ret = panic::catch_unwind(inner);
    if ret.is_err() {
        println!("捕获 panic，程序继续运行");
    }
}
```

**执行顺序（unwind）：**

1. `inner` 触发 panic → 从 `inner` 栈帧向上展开  
2. `_s` 自动 **`Drop`** 释放内存  
3. 展开到 **`catch_unwind` 边界** → 停止回溯  
4. panic 信息装入 `Err`，`main` 继续执行  

> `catch_unwind` = **栈展开的拦截锚点**，依赖 LLVM 异常展开（类似 C++ `eh_frame`）。

---

## 三、为什么 `abort` 下没用？

```toml
[profile.release]
panic = "abort"
```

1. panic → **跳过全部栈展开**，不 Drop、不找捕获锚点  
2. 进程直接系统终止，OS 回收内存  
3. `catch_unwind` 的拦截代码**永远没有执行时机**

## 四、三个拓展

### 1. 子线程 panic

| 模式 | 行为 |
|------|------|
| **unwind** | 子线程 panic 通常**只崩该线程**（运行时隐式 catch），主线程可继续 |
| **abort** | **任意线程 panic → 整进程崩溃** |

### 2. `UnwindSafe` 约束

被 `catch_unwind` 包裹的闭包须满足 **`UnwindSafe`**：panic 中途对象可能半销毁，Rust 防止捕获后拿到残缺状态引发 UB。

### 3. 开发 / 生产对照（HFT）

| 编译配置 | `catch_unwind` | Drop | 场景 |
|----------|----------------|------|------|
| dev (unwind) | ✅ | 执行 | 单测、调试、捕获打日志 |
| release (unwind) | ✅ | 执行 | 普通后端 |
| release (abort) | ❌ | 不执行 | **HFT 实盘、嵌入式** |

→ profile 总表：[9.1.2](./9.1.2-dev-release与HFT编译配置.md)

## 五、release 临时启用 catch_unwind

```toml
[profile.release]
panic = "unwind"
debug = true
```

```bash
cargo build --release
RUST_BACKTRACE=full ./target/release/程序
```

或使用自定义 profile（见 [9.1.2 附录](./9.1.2-dev-release与HFT编译配置.md) 的 `release-with-debug`）。

## demo：unwind 可捕获 vs abort 不可捕获

```bash
cd Book/09-error-handling/9.1.3-catch-unwind-demo

# unwind（dev 默认）→ 捕获成功，程序继续
cargo run -- catch

# abort（本 crate release profile 设了 panic=abort）→ 进程直接退出
cargo run --release -- catch

# release 但改回 unwind 的自定义 profile → 又能捕获
cargo run --profile release-unwind -- catch

# 子线程 panic 对比
cargo run -- thread
```

| 命令 | 预期 |
|------|------|
| `cargo run -- catch` | 打印「捕获成功」+ `main` 继续 |
| `cargo run --release -- catch` | **进程退出**，无「捕获成功」 |
| `cargo run --profile release-unwind -- catch` | 同 dev，捕获成功 |
