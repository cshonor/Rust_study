# JSON、UTF-8 与 GB 编码

**结论**：标准 JSON（RFC 8259）要求文档为 **UTF-8**；Rust `String` 也是 UTF-8，正常 `serde_json` 序列化**无需转码**。仅在与老旧客户端约定 **GB2312/GBK** 时，在**网络字节层**用 `encoding_rs` 互转；Rust 内部 `String` 仍保持 UTF-8。

对应 demo：`Book/03-common-concepts/3.2-json-encoding-demo/`  
Unicode 基础见：`3.2-Unicode与字符串.md`

---

## 核心结论

1. **标准前后端 JSON**：Rust UTF-8 `String` → JSON UTF-8 字节，**直接返回**，零转码。
2. **对方强制 GB2312/GBK**：只在**收发字节**时 UTF-8 ↔ GB 互转；**不要**把 `String` 改成别的编码存储。
3. **最优**：全链路 UTF-8；GB 系字符集小，生僻字/emoji 易丢失。

---

## 一、为什么正常不用转

- JSON 标准：**整个文档必须是 UTF-8**。
- 浏览器 / JS / 现代 API 默认 UTF-8；JS 字符串基于 Unicode，与 UTF-8 JSON 兼容。
- Rust `serde_json::to_string` / `to_vec` 输出的是 **UTF-8 文本字节**。

> 需要 GB 编码的多为老系统、工控、部分桌面客户端，业务上少见。

---

## 二、真遇到 GB2312/GBK

使用 **`encoding_rs`**（`encoding_rs::GBK` 覆盖 GB2312 常用字；也可用 `Encoding::for_label(b"gb2312")`）。

### 依赖

```toml
[dependencies]
encoding_rs = "0.8"
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
```

### 场景 A：UTF-8 字符串 ↔ GB 字节

```rust
use encoding_rs::GBK;

let s = "测试中文内容";
let (gb_bytes, _, had_errors) = GBK.encode(s);
assert!(!had_errors);

let (utf8_cow, _, had_errors) = GBK.decode(&gb_bytes);
assert!(!had_errors);
println!("{}", utf8_cow); // 测试中文内容
```

### 场景 B：JSON 响应体整体转为 GB 字节

**正确流程**：先得到 **UTF-8 JSON 字节** → 按 **UTF-8 文本**理解 → **encode 成 GB 字节**。

```rust
use encoding_rs::GBK;
use serde::Serialize;
use serde_json::to_vec;

#[derive(Serialize)]
struct Resp { name: String, msg: String }

let data = Resp { name: "张三".into(), msg: "后端返回内容".into() };
let json_utf8 = to_vec(&data).unwrap();
let json_str = std::str::from_utf8(&json_utf8).unwrap();

let (json_gb, _, had_errors) = GBK.encode(json_str);
// json_gb 作为 HTTP body 返回；Header: Content-Type: application/json; charset=GBK
```

**常见笔误**：对已是 UTF-8 的 JSON 字节调用 `GBK.decode(...)`——那是 **GB → UTF-8**，方向反了。

### 场景 C：收到 GB 编码的 JSON  body

```rust
let (utf8_cow, _, had_errors) = GBK.decode(&gb_body_bytes);
let resp: Resp = serde_json::from_str(&utf8_cow)?;
```

---

## 三、注意点

1. **字符丢失**：GB2312/GBK 不含生僻字、emoji；`encode` 可能 `had_errors == true` 或替换，需检查并尽量推动 UTF-8。
2. **HTTP Header**：返回 GB body 时必须声明，例如  
   `Content-Type: application/json; charset=GBK`（或与对方约定一致）。
3. **边界**：Rust **`String` / `&str` 始终 UTF-8**；转码只发生在 `&[u8]` / `Vec<u8>` 与外部系统之间。
