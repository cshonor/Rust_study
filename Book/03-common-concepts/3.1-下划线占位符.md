# 下划线占位符 `_`

> **结论**：`_` 是**匿名占位符**——表示「这里有个值，但我不打算用」。用来消掉未使用变量告警，或明确「故意丢弃」返回值。

与 [3.1 变量和可变性](./3.1-变量和可变性.md) 配套；控制流里 `for` 见 [3.5 控制流](./3.5-控制流.md)。

---

## 一句话记住三条

| # | 用法 | 含义 |
|---|------|------|
| 1 | `_` 作绑定名 | 定义了但**不用**；编译器不报 unused |
| 2 | `let _ = x` | **接住**返回值后**直接扔掉**，不存可用变量 |
| 3 | `for _ in 0..n` | 只要循环 **n 次**，不要迭代元素 |

---

## 1. 匿名占位，忽略未使用告警

```rust
let _ = some_fn();  // 必须调用，但不要结果
```

- 若写 `let x = some_fn();` 却从不使用 `x`，会有 **unused variable** 警告。
- `_` **不是**普通变量：不能 `println!("{}", _)`，也不能在后面再引用。

---

## 2. `let _ = x`：接收后丢弃

```rust
let _ = h.await;  // 必须 await 完成；不要 Future 的输出
```

| 写法 | 说明 |
|------|------|
| `h.await` 单独一行 | 仍会求值；`Result` 等有时仍需处理 |
| `let _ = h.await` | 明确：**故意不要**这个结果 |

适合：必须执行副作用 / 必须 poll 完，但返回值无意义或已在别处处理错误。

---

## 3. `for _ in 0..n`：只要次数

```rust
for _ in 0..3 {
    println!("tick");
}
```

- 循环 3 次，**不需要** `0`、`1`、`2` 这些值。
- 若写 `for i in 0..3` 却不用 `i`，会触发 unused；用 `_` 是惯用写法。

---

## 模式匹配里的 `_`（顺带）

在 `match` / 解构里，`_` 同样表示「匹配到了，但不要里面的值」：

```rust
match opt {
    Some(_) => println!("有值"),
    None => {},
}
```

与 `let _` 同一思想：**占位、不绑定可用名字**。

---

## 记忆卡片

| 场景 | 写法 |
|------|------|
| 不要返回值 | `let _ = f();` |
| 只要循环次数 | `for _ in 0..5 {}` |
| match 忽略内容 | `Some(_)` |
| 需要再用这个值 | 用正常名字 `let x = ...`，别用 `_` |
