# 18.1 所有可能会用到模式的位置

模式在 Rust 里出现得很频繁：`match`、`if let`、`while let`、`for`、`let`、函数参数等，本质上都是在做**模式匹配**与**绑定**。

对应 demo：`Book/18-patterns/18.1-pattern-locations-demo/`

---

## `match` 分支

```text
match VALUE {
    PATTERN => EXPRESSION,
    ...
}
```

`match` 必须**穷尽**：所有可能值都要被覆盖；常用 `_` 匹配剩余情况。`_` 不绑定变量。

---

## `if let` / `else if` / `else if let` / `else`

可把一连串条件写成链式，而不必把一切都塞进一个 `match`。注意：`if let Ok(age) = age` 引入的 `age` 只在其块作用域内有效，不能写成 `if let Ok(age) = age && age > 30` 这种合并（除非调整结构）。

**与 `match` 的差异**：`if let` 链**没有**穷尽性检查；漏掉分支时编译器未必能提示。

---

## `while let` 条件循环

只要右侧表达式**持续匹配**模式，就重复执行循环体。常用于“弹出直到 `None`”：

```rust
while let Some(top) = stack.pop() {
    println!("{top}");
}
```

---

## `for` 循环

`for 模式 in 迭代器` 中的“模式”同样可解构，例如 `enumerate()` 产出 `(index, item)`：

```rust
for (index, value) in v.iter().enumerate() {
    // ...
}
```

---

## `let` 语句

更一般的形式是：

```text
let PATTERN = EXPRESSION;
```

`let x = 5` 里的 `x` 也是一种模式（绑定任意值）。元组解构：

```rust
let (x, y, z) = (1, 2, 3);
```

元素个数必须与右侧一致，否则编译错误。可用 `_`、`..` 等忽略部分（见后文“忽略模式中的值”）。

---

## 函数参数（含闭包参数）

参数位置也是模式，例如解构引用后的元组：

```rust
fn print_coordinates(&(x, y): &(i32, i32)) { ... }
```
