# 4.3.3 `str`、`&str` 与字面量

## 两层概念

| 概念 | 含义 |
|------|------|
| **`str`** | DST，不定长 UTF-8，不能 `let x: str` |
| **`&str`** | 胖指针（地址 + 长度），可存变量、可当参数 |
| **`"abc"`** | 字面量进 `.rodata` 只读段 |

```rust
let s = "abc"; // 多为 &'static str
```

### 背诵版：`String` vs `&str` 能不能改？

> **`String` 拥数据，能改内容；`&str` 只引用，不能改本体（最多换指向）。**

| | **`String`** | **`&str`** |
|--|--------------|------------|
| 关系 | 拥有堆数据 | 借用视图 |
| 改内容 | ✅ `push` / `push_str` / `clear` 等 | ❌ `s[0] = ...` 非法 |
| 换指向 | N/A（不是指针变量） | `let mut r: &str = "a"; r = "b";` ✅ |
| 要改原串 | 直接 `mut String` | 需 **`&mut str`**（长度仍不能变）→ [8.2 String](../08-collections/8.2-string.md) |

```rust
let mut s = String::from("abc");
s.push('d'); // ✅ 变长修改

let mut r: &str = "abc";
r = "bcd";   // ✅ 换指向
// r.make_ascii_uppercase(); // ❌ &str 只读

let mut src = String::from("abc");
let m: &mut str = &mut src;
m.make_ascii_uppercase(); // ✅ 通过可变切片改 src 堆内容（ASCII 等）
```

## 两类 `&str`（类型相同，内存不同）

| | ① `&s[0..2]` | ② `"hello"` |
|--|--------------|-------------|
| 数据在哪 | **`String` 的堆** | **`.rodata` 静态段** |
| 生命周期 | 绑 **`s`** | **`'static`** |
| `s.clear()` | `part` 活着 → ❌（见 [4.3.2](./4.3.2-切片与借用规则.md)） | 与 `s` 无关 |

```rust
let mut s = String::from("hello");
let part: &str = &s[0..2]; // 临时借用堆

let lit: &str = "hello";   // 静态只读，程序同寿
```

**HFT**：`let tag: &str = "LOGIN"` 零堆开销；`&buf[..]` 借 `Vec` 缓冲零拷贝。

## 字面量 vs `String::from`

| | `let s = "abc";` | `String::from("abc")` |
|--|------------------|------------------------|
| 类型 | `&'static str` | `String`（拥堆） |
| 堆 | 不拷到堆 | 从静态区拷一份到堆 |
| 可改 | 否 | 是（`push` / `clear`） |

为何不默认把 `"abc"` 变成 `String`？常量只读场景多，隐式堆分配浪费；要可变再显式 `String::from`。

#### 内存布局（极简）

**字面量**：胖指针在栈，字节在 `.rodata`。

```
栈 (lit)                    .rodata
ptr,len ──────────►         a b c   （不 drop）
```

**`String::from`**：栈三元组 + 堆字节（`drop` 释堆）。

## 函数不能返回 `str`

```rust
// ❌ E0277：size not known at compile time
fn bad() -> str { "hello" }

fn good() -> &str { "hello" }                    // 常 &'static str
fn owned() -> String { String::from("hello") }   // 拥所有权
```

验证：`rustc --edition 2021 compile_fail/return_str_dst.rs`。

| 返回 | 场景 |
|------|------|
| `&str` | 字面量或 `&s[0..2]`（绑 `s`） |
| `String` | 调用方独立持有、可改 |
| `str` | **非法** |

---

## 函数参数：优先 `&str`

```rust
fn first_word(s: &str) -> &str { /* ... */ }

first_word(&st);       // &String → &str（deref）
first_word(&st[..]);
first_word("lit");
```

详见 [ER Item 05](../../ER/Chapter-01-Types/Item-05-type-conversions.md)。

## 小结

| 要点 | 一句 |
|------|------|
| `str` / `&str` | DST 本体 vs 引用包装 |
| 两类 `&str` | 堆切片 vs 静态字面量 |
| 返回值 | 不能 `-> str`；要实体用 `String` |
