# 4.3.3 `str`、`&str` 与字面量

## 两层概念

| 概念 | 含义 |
|------|------|
| **`str`** | DST，不定长 UTF-8，不能 `let x: str` |
| **`&str`** | 胖指针（地址 + 长度），可存变量、可当参数 |
| **`"abc"`** | 字面量进 `.rodata` 只读段 |

```rust
let s = "abc"; // 多为 &'static str
```

### 背诵版：`String` vs `&str` 能不能改？

> **`String` 拥数据，能改内容；`&str` 只引用，不能改本体（最多换指向）。**

| | **`String`** | **`&str`** |
|--|--------------|------------|
| 关系 | 拥有堆数据 | 借用视图 |
| 改内容 | ✅ `push` / `push_str` / `clear` 等 | ❌ `s[0] = ...` 非法 |
| 换指向 | N/A（不是指针变量） | `let mut r: &str = "a"; r = "b";` ✅ |
| 要改原串 | 直接 `mut String` | 需 **`&mut str`**（长度仍不能变）→ [8.2 String](../08-collections/8.2-string.md) |

```rust
let mut s = String::from("abc");
s.push('d'); // ✅ 变长修改

let mut r: &str = "abc";
r = "bcd";   // ✅ 换指向
// r.make_ascii_uppercase(); // ❌ &str 只读

let mut src = String::from("abc");
let m: &mut str = &mut src;
m.make_ascii_uppercase(); // ✅ 通过可变切片改 src 堆内容（ASCII 等）
```

### `let mut` 变量：`&str` 换指向（不改只读区内容）

`&str` = **指针 + 长度**。`s = "你好"` 只改变量里存的地址/长度，**不碰**原先 `"hello"` 在 `.rodata` 里的字节。

```rust
let mut s: &str = "hello";
println!("{s}"); // 指向静态区 "hello"

s = "你好";      // ✅ 换指向另一段静态 UTF-8
println!("{s}"); // "hello" 仍在只读段，内容未改

// s[0] = 'a';   // ❌ 不能改指向内存的内容
```

结构体字段同理 → [5.1.3 字段所有权](../05-structs/5.1.3-字段所有权与引用.md)。

```rust
struct User { name: &'static str }

let mut user = User { name: "zhangsan" };
user.name = "lisi"; // ✅ 只换 name 的指针；不改 "zhangsan" 字节
```

从 `String` 借出的 `&str`：`let r: &st` 只读；`r = "abc"` 若 `r` 是 `mut &str` 则换指向，**不影响** `st` 堆上数据。

## UTF-8：`str` / `&str` / `String` 全是 UTF-8

Rust **强制**字符串语义为 UTF-8 字节序列（`&[u8]` 无此约束，HFT 二进制用 `&[u8]`）。

切片按**字节**下标，边界必须在 **UTF-8 字符边界**上，否则 **运行时 panic**（一般不是编译期报错）：

```rust
let text = "你好"; // 各 3 字节，共 6 字节
// let sub = &text[0..2]; // ❌ 运行 panic：切在「你」的中间
let sub = &text[0..3];    // ✅ 完整一个「你」
```

| | `&str` | `&[u8]` |
|--|--------|---------|
| 编码 | 必须合法 UTF-8 | 任意字节 |
| 切片 | 须在字符边界 | 任意切 |

安全切分：用 `.chars()`、`.char_indices()` 或 crate `unicode-segmentation`（ grapheme 更复杂时）。

## 两类 `&str`（类型相同，内存不同）

| | ① `&s[0..2]` | ② `"hello"` |
|--|--------------|-------------|
| 数据在哪 | **`String` 的堆** | **`.rodata` 静态段** |
| 生命周期 | 绑 **`s`** | **`'static`** |
| `s.clear()` | `part` 活着 → ❌（见 [4.3.2](./4.3.2-切片与借用规则.md)） | 与 `s` 无关 |

```rust
let mut s = String::from("hello");
let part: &str = &s[0..2]; // 临时借用堆

let lit: &str = "hello";   // 静态只读，程序同寿
```

**HFT**：`let tag: &str = "LOGIN"` 零堆开销；`&buf[..]` 借 `Vec` 缓冲零拷贝。

## 字面量 vs `String::from`

| | `let s = "abc";` | `String::from("abc")` |
|--|------------------|------------------------|
| 类型 | `&'static str` | `String`（拥堆） |
| 堆 | 不拷到堆 | 从静态区拷一份到堆 |
| 可改 | 否 | 是（`push` / `clear`） |

为何不默认把 `"abc"` 变成 `String`？常量只读场景多，隐式堆分配浪费；要可变再显式 `String::from`。

#### 内存布局（极简）

**字面量**：胖指针在栈，字节在 `.rodata`。

```
栈 (lit)                    .rodata
ptr,len ──────────►         a b c   （不 drop）
```

**`String::from`**：栈三元组 + 堆字节（`drop` 释堆）。

## 函数不能返回 `str`

```rust
// ❌ E0277：size not known at compile time
fn bad() -> str { "hello" }

fn good() -> &str { "hello" }                    // 常 &'static str
fn owned() -> String { String::from("hello") }   // 拥所有权
```

验证：`rustc --edition 2021 compile_fail/return_str_dst.rs`。

| 返回 | 场景 |
|------|------|
| `&str` | 字面量或 `&s[0..2]`（绑 `s`） |
| `String` | 调用方独立持有、可改 |
| `str` | **非法** |

---

## 函数参数：优先 `&str`

```rust
fn first_word(s: &str) -> &str { /* ... */ }

first_word(&st);       // &String → &str（deref）
first_word(&st[..]);
first_word("lit");
```

详见 [ER Item 05](../../ER/Chapter-01-Types/Item-05-type-conversions/README.md)。

## 小结

| 要点 | 一句 |
|------|------|
| `str` / `&str` | DST 本体 vs 引用包装 |
| 两类 `&str` | 堆切片 vs 静态字面量 |
| 返回值 | 不能 `-> str`；要实体用 `String` |
