# 4.3.2 切片与借用规则（`&` / `&mut`）

← 主笔记：[4.3 切片](./4.3-切片slice.md) · 通用规则：[4.2 引用与借用](./4.2-引用与借用.md) · demo：[4.3-slices-demo](./4.3-slices-demo/)

本章只讲：**切片场景下，不可变借用和可变借用怎么互斥**（`first_word` / `clear` 那条线）。

---

## 两条铁则（复习）

| 借用 | 数量 | 能做什么 |
|------|------|----------|
| **`&T` 不可变** | 可多个 | 只读 |
| **`&mut T` 可变** | 同时最多一个 | 独占写 |

> **同一时刻**：不能既有存活的 **`&`**，又申请 **`&mut`** 指向同一块数据。

完整条文、NLL、`&mut` 互斥小例 → [4.2](./4.2-引用与借用.md)。

---

## 案例：`word: &str` 挡住 `s.clear()`

```rust
fn first_word(s: &String) -> &str { /* &s[0..i] */ }

let mut s = String::from("hello world");
let word = first_word(&s); // 不可变借用 s 的堆字节
s.clear();                 // ❌ E0502
println!("{word}");
```

### `clear()` 在要什么

`fn clear(&mut self)` → `s.clear()` = 向 `s` 申请**整段堆数据的独占可变借用**。

### 报错链（一步步）

1. `word` 活着 → 占用 `s` 堆内存的**不可变借用**；
2. `clear()` 要 **`&mut s`**；
3. 不可变借用未结束 → 编译器拒绝。

### 何时能 `clear`

```rust
let mut s = String::from("hello world");
{
    let word = first_word(&s);
} // word 出作用域，不可变借用结束
s.clear(); // ✓
```

验证：`rustc --edition 2021 compile_fail/slice_blocks_clear.rs`。

---

## 关键：`&str` 借的是堆字节，不是「不算借用」

```rust
let mut s = String::from("hello world");
let word: &str = &s[0..5];
```

- `s`：栈上 ptr/len/cap，**字节在堆**；
- `word`：**胖指针指向 `s` 在堆里的那一段**。

借用检查看**谁在用这块堆内存**，不看表面类型：

| 写法 | 阻碍 `clear`？ |
|------|----------------|
| `let r: &String = &s` | ✅ |
| `let word: &str = &s[..]` | ✅ **一样** |
| `let idx: usize = 5` | ❌ 不拦（见 [4.3.1](./4.3.1-usize下标与悬空索引.md)） |

> **`&str` 切片 ≈ 给 `s` 挂上不可变借用**；`word` 活着就拿不到 `&mut` 去 `clear`。

若 `word` 仍指着堆里的旧内容就清空 → `word` 会悬空 → 编译器提前禁止。

---

## `&String` vs `&str`（效果）

```rust
let r: &String = &s;      // 借整个串对象
let word: &str = &s[..];   // 借堆里一段字节
```

对 **`clear`** 而言：**二者都会锁 `s` 的堆数据**，只要还有一个不可变借用存活，`clear` 就失败。

---

## 与 `usize` 的对比（一句话）

`idx: usize` **不产生借用** → `clear` 能通过 → 后面 `&s[0..idx]` 可能运行时越界。  
`word: &str` **产生借用** → `clear` 编译失败 → 从根上避免悬空切片。

---

## 小结

| 要点 | 一句 |
|------|------|
| 铁则 | 有存活 `&` 就不能再 `&mut` 同数据 |
| `clear` | 要 `&mut self`，与 `&str` 切片冲突 |
| `&str` | 借堆字节，和 `&String` 一样挡 `clear` |
| `usize` | 不借，规则管不到 |
