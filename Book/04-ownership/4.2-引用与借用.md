# 4.2 引用与借用

用**引用** (`&`) 使用值但不获取所有权，称为**借用**（borrowing）。

对应 demo 项目：`Book/04-ownership/4.2-references-demo/`

## 有 `&` vs 无 `&`：move 还是借用（总览）

**分界**：赋值/传参右边**不带 `&`** → 所有权转移（move，对 `String` 等）；**带 `&` / `&mut`** → 只造引用，原变量仍拥有数据。

### 赋值

```rust
let a = String::from("hello");
let b = a;    // 无 &：move，a 失效
// let r = &a;  // 若 b=a 之后，a 已不能再用

let a = String::from("hello");
let b = &a;   // 有 &：b 借用 a，a 仍有效
println!("{a}");
```

| 右边 | 效果 | 原变量 |
|------|------|--------|
| `a`（无 `&`） | 移交所有权 | 失效（`String` 等） |
| `&a` | 不可变借用 | 仍可用 |
| `&mut a` | 可变借用（`a` 须 `mut`） | 仍拥有；独占修改权 |

字面量：`let s = "abc"` 等价 `let s: &str = "abc"`，**天生是引用**，不是 `String`。

> `i32` 等 `Copy` 类型赋值是**拷贝**而非 move → [4.1 什么是所有权](./4.1-什么是所有权.md) §Copy / Move / clone **对照表**。

### 函数传参（同一套逻辑）

```rust
fn take(_s: String) {}       // 调用 take(s) → s 移入函数，外面 s 失效
fn borrow(_s: &String) {}     // 调用 borrow(&s) → 只借，外面 s 仍有效
fn touch(_s: &mut String) {}  // 调用 touch(&mut s) → 可变借，可改内容，仍不拿走所有权

let mut s = String::from("a");
take(s);
// borrow(&s); touch(&mut s);
```

### 两类变量（记忆）

1. **所有者**：`String`、`Vec`、`String::from(...)` — 手里有所有权；无 `&` 的赋值/传参会 move。
2. **引用**：`&String`、`&str`、`&[u8]` — 只借用；永远不单独拥有那份堆数据。

`&mut` 仍是借用，但能改原数据；同一时刻只能有一个活跃 `&mut`（互斥见下文）。Move 详解见 [4.1](./4.1-什么是所有权.md) §移动。

## 引用语法

```rust
let s1 = String::from("hello");
let len = calculate_length(&s1);  // &s1 创建引用
println!("'{}' 长度 {}", s1, len);  // s1 仍有效

fn calculate_length(s: &String) -> usize {
    s.len()
}  // s 离开作用域，但不丢弃指向的数据（没有所有权）
```

- `&s1`：指向 s1 的引用，不拥有数据
- `*` 解引用：见第 8、15 章

### 引用赋给引用：`&T` 本身是 `Copy`

```rust
let src = String::from("abc");
let r1: &String = &src;
let r2 = r1;        // 复制栈上的指针值，不是 Move src
println!("{r1} {r2}"); // r1、r2 都有效，同时不可变借用 src
```

- **`&T` 实现了 `Copy`**：赋值只多拷一个指针，**不转移** `src` 的所有权。
- 效果：多个不可变引用**共存**，指向同一数据源（借用规则仍禁止在有 `&` 时 `&mut`）。

**三类赋值对照**

| 写法 | 行为 | `src` / 原变量 |
|------|------|----------------|
| `let b = src;`（`String`） | Move | `src` 失效 |
| `let r = &src;` | 新建借用 | `src` 仍拥有 |
| `let r2 = r;`（`r: &String`） | **Copy 指针**，多一个 `&` | `src` 不变 |

**`&mut` 赋值对比**：`&mut T` **也实现 `Copy`**（同样只拷指针），但**同一时刻只能有一个活跃的 `&mut`**，不能有两个可变引用同时写：

```rust
let mut s = String::from("a");
let m1 = &mut s;
// let m2 = m1;   // 若 m1 仍活跃，再 let m2 = m1 会复制指针——但第二行 &mut 会与规则 1 冲突
// 正确：在一个作用域里只保留一个 &mut，或用 `{}` 结束 m1 后再借
```

不可变 `let r2 = r1` 允许多份；可变借用靠**互斥规则**限制，不是靠「引用不能 Copy」。

### 两个 `&`：造引用（表达式）vs 引用类型（标注）

| 位置 | 写法 | 含义 |
|------|------|------|
| **变量/实参前** | `&s`、`&mut s` | **取值运算符**，生成引用实例 |
| **冒号后** | `: &String`、`: &str` | **类型**，表示变量/形参存的是引用 |

```rust
let s = String::from("abc");   // s：拥有所有权
let r = &s;                    // 表达式 &s → 造出 &String
let r: &String = &s;           // 左：类型 &String；右：&s 赋值
```

函数同理：形参 `arg: &str` 是**类型声明**；调用 `func(&st)` 里 `&st` 是**生成引用实参**。

### 三行对照（一眼分清）

```rust
let r = &s;                 // ① 变量前的 &：造引用（表达式）
let r: &String = &s;          // ② 冒号后的 &：引用类型 + 右侧再造引用
let lit = "hello";            // ③ 字面量：等价 let lit: &str = "hello"; 编译器隐式当借用
```

**口诀**：变量前的 `&` 造引用；冒号后的 `&` 是类型；双引号字面量自动变成 `&str`（静态区 `str` 不能直接当变量类型，只能借）。

`str` 是 DST 本体；`&str` 是对 `str` 的借用，可存变量、可做参数。切片场景见 [4.3.3 str 与 &str](./4.3.3-str与str类型.md)；借用互斥见 [4.3.2](./4.3.2-切片与借用规则.md)。

## 不可变引用

默认引用是**不可变**的，不能通过引用修改数据：

```rust
fn change(some_string: &String) {
    some_string.push_str(", world");  // ❌ 错误
}
// cannot borrow `*some_string` as mutable, as it is behind a `&` reference
```

---

## 可变引用 &mut

要修改借用来的值，使用**可变引用**：

```rust
let mut s = String::from("hello");
change(&mut s);

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}
```

- 变量需为 `mut`
- 传参用 `&mut s`
- 函数签名为 `&mut String`

## 借用规则

### 规则 1：同一时间只能有一个可变引用

```rust
let r1 = &mut s;
let r2 = &mut s;  // ❌ 错误: cannot borrow `s` as mutable more than once
```

- 避免**数据竞争**（多指针写、无同步）
- 可用大括号限定作用域，错开可变借用：

```rust
{
    let r1 = &mut s;
}  // r1 离开作用域
let r2 = &mut s;  // ✓
```

### 规则 2：不可变与可变引用不能同时存在

```rust
let r1 = &s;
let r2 = &s;
let r3 = &mut s;  // ❌ 错误: cannot borrow as mutable (已有不可变引用)
```

- 多个不可变引用可以共存（`let r2 = r1` 复制指针也算多份 `&`）
- 只要还有**存活的**不可变借用，就不能再拿 `&mut`

#### 案例：`s.clear()` 需要 `&mut self`（与 [4.3.2 切片与借用规则](./4.3.2-切片与借用规则.md) 同源）

`clear` 签名：`fn clear(&mut self)` → 调用 `s.clear()` 等于向 `s` 申请**可变借用**。

```rust
let mut s = String::from("hello world");
let word = first_word(&s); // word: &str，占用 s 的不可变借用
s.clear();                 // ❌ E0502：word 还活着，不能再 &mut s
println!("{word}");
```

报错链：`word` 存活 → 不可变借用未结束 → `clear` 要的 `&mut` 被拒绝。

**何时能 `clear`？** 不可变借用先结束，例如：

```rust
let mut s = String::from("hello world");
{
    let word = first_word(&s);
} // word 出作用域，不可变借用失效
s.clear(); // ✓
```

本地验证：`rustc --edition 2021 compile_fail/slice_blocks_clear.rs`（见 `4.3-slices-demo`）。

**对比 `usize` 下标**（4.3）：`let idx = first_word_usize(&s)` 返回纯数字，**不产生借用** → `s.clear()` 能通过编译，但 `&s[0..idx]` 可能运行时越界——借用规则管不到数字。

#### `&mut` 互相冲突（补充小例）

```rust
let mut s = String::from("a");
let m1 = &mut s;
let m2 = &mut s;  // ❌ 规则 1：两个同时活跃的 &mut

let r = &s;
let m = &mut s;   // ❌ 规则 2：已有不可变 r 时不能 &mut
```

### 引用的作用域与 NLL

引用的作用域到**最后一次使用**为止（非词法作用域 NLL）：

```rust
let r1 = &s;
let r2 = &s;
println!("{} {}", r1, r2);  // r1、r2 最后一次使用
let r3 = &mut s;  // ✓ r1、r2 已“结束”，不重叠
```

## 悬垂引用

Rust 保证引用不会指向已释放的内存（悬垂指针）：

```rust
fn dangle() -> &String {  // ❌ 错误
    let s = String::from("hello");
    &s  // s 离开作用域被释放，返回的引用无效
}
```

- 错误：`missing lifetime specifier`，返回值借用的数据不存在
- 显式生命周期时，编译器用 `'a` 校验——局部数据无法满足约束（见 [10.3](Book/10-generics-traits-lifetimes/10.3-生命周期与引用有效性.md)）

- 正确做法：返回拥有所有权的值，如 `String`：

```rust
fn no_dangle() -> String {
    let s = String::from("hello");
    s  // 转移所有权
}
```

---

## 引用规则小结

1. **任意时刻**：要么只有一个可变引用，要么只有多个不可变引用
2. **引用必须有效**：不能悬垂

## 与所有权的对比

| 方式 | 所有权 | 原变量 |
|------|--------|--------|
| 传值 | 移动 | 失效 |
| 不可变引用 `&T` | 借用 | 仍可用 |
| 可变引用 `&mut T` | 借用并可能修改 | 仍可用 |
