# 4.3 切片 Slice

**Slice**（`&str` / `&[T]`）：只存**起始指针 + 长度**，不持有底层数据所有权，借用原容器。

对应 demo：[4.3-slices-demo](./4.3-slices-demo/)（含 **HFT 零拷贝 `&[u8]` 报文分段**）

---

## 一、核心本质

解决痛点：单纯返回 `usize` 下标与原数据脱钩，原数据修改/清空后下标失效；**`&str` / `&[T]` 绑定生命周期，借用检查在编译期拦截非法修改**。

---

## 二、旧方案缺陷：返回 `usize` 下标

```rust
fn first_word_index(s: &String) -> usize {
    let bytes = s.as_bytes();
    for (i, &b) in bytes.iter().enumerate() {
        if b == b' ' {
            return i;
        }
    }
    s.len()
}
```

```rust
let mut s = String::from("hello world");
let idx = first_word_index(&s);
s.clear(); // 清空原字符串
// idx 仍是 5，但字符串已空 —— 悬空索引，编译器查不出（The Book 4.3 原话）
```

### 悬空索引：官方原话拆解

#### 1. 为啥 `idx: usize = 5` 叫裸数字

1. **`usize` 只是栈上的普通整数**：`5` 不存指针、不绑定 `s`、不带生命周期，和原 `String` **完全割裂**。
2. **借用检查只管 `&` 引用，不管普通数字**：生命周期只约束引用类型；`usize` 是值类型，编译器**不会追踪**「这个 5 对应哪段字符串、字符串有没有被 `clear()`」。

> **编译器查不出隐患的根源**：`idx` 不是借用，规则管不到它。

**澄清：数字 5 本身不是「悬空索引」**

- `idx: usize = 5` 只是栈上的值，**不是引用、没有生命周期**，变量本身合法。
- **悬空的是「使用行为」**：在 `s.clear()` 之后写 `&s[0..idx]`，用数字去访问已失效的容器内存（类似 C 里 `pos=5` 后 `memset` 清空再 `arr[pos]`）。

```c
char arr[] = "hello world";
int pos = 5;
memset(arr, 0, sizeof(arr)); // 清空
// pos 仍是 5；arr[5] 越界 —— 数字无罪，下标访问才出事
```

> **单纯数字不算悬空；拿数字访问销毁后的数据，才是悬空索引。** `&str` 从类型上禁止在源数据被可变改动后继续借用。

#### 2. 什么是悬空索引

- 当初 `5` 表示：原串第 5 字节处是空格；
- `s.clear()` 后堆上内容清空、`len() == 0`；
- **数字仍是 5，但原语义已失效**；再写 `&s[0..idx]` → 越界，这就是**悬空索引**；
- **`s.clear()` 与 `idx` 并存时编译可通过**，往往要到**运行时**切片才 panic。

#### 3. 返回 `&str` 为何不会悬空

```rust
let mut s = String::from("hello world");
let word: &str = first_word(&s);
s.clear(); // ❌ 编译错误
```

`&str` = 指针 + 长度 + **借用**：`word` 存活期间 `s` 不能可变借用（`clear` 是可变修改），从根上杜绝悬空。

#### 4. 一句话

| | `usize` 裸下标 | `&str` / `&[u8]` |
|---|----------------|------------------|
| 关系 | 脱离原数据 | 绑定原内存 |
| 编译 | 常放行 | 非法修改直接报错 |
| 风险 | 悬空索引、运行翻车 | 编译期拦截 |

#### 5. HFT：`Vec` 报文缓冲区

```rust
// ❌ 坏：裸 usize 分段下标
let mut buf: Vec<u8> = receive_udp();
let split_pos: usize = 16;
buf.clear(); // split_pos 仍是 16，后续按下标解析 → 悬空、越界

// ✅ 好：&[u8] 子切片
let part: &[u8] = &buf[0..16];
// buf.clear(); // 若 part 仍存活 → 编译错误
```

**本地对比编译**（见 demo [README](./4.3-slices-demo/README.md)）：

```bash
cargo test -p slices-4-3-demo dangling_usize_panics_at_runtime   # usize 路径：运行 panic
rustc --edition 2021 compile_fail/slice_blocks_clear.rs         # 切片路径：编译失败
```

| 问题 | 说明 |
|------|------|
| 裸数字 | `usize` 与 `String` 无绑定 |
| 悬空索引 | 原数据销毁/修改后下标仍“合法” |
| 借用规则 | 无法介入，只能运行时踩坑 |

### 对照表：`usize` vs 切片（复习）

| 对比 | `usize` 裸下标 | `&str` / `&[u8]` 切片 |
|------|----------------|----------------------|
| 本质 | 普通整数 | 绑定原容器的引用 |
| 原数据被改 | 数字不变，编译器不管 | 有切片存活时禁止可变修改 |
| 出错时机 | 往往**运行时**越界 | 往往**编译期**拒绝 |

**`as_bytes()`**：遍历工具，算出位置 `i`；旧方案返回 `i`，新方案返回 `&s[0..i]`。

## 三、字符串切片 `&str`

### Range（左闭右开，不含结束下标）

| 写法 | 等价 | 说明 |
|------|------|------|
| `[0..5]` | 0～4 | 常规区间 |
| `[..2]` | `[0..2]` | 左边界可省略 |
| `[3..]` | `[3..len]` | 右边界可省略 |
| `[..]` | `[0..len]` | 全切片 |

```rust
let s = String::from("hello world");
let hello = &s[0..5];
let world = &s[6..11];
```

⚠️ **切片边界必须落在 UTF-8 字符字节边界**，不能从多字节字符中间切，否则 **panic**（`&s[0..1]` 切到多字节字符中间）。

### 切片版 `first_word`

```rust
fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}
```

借用规则生效：

```rust
let mut s = String::from("hello world");
let w = first_word(&s);
s.clear(); // ❌ 编译错误：w 持有 s 的不可变借用
println!("{w}");
```

`clear()` = `&mut self`：只要 `w: &str` 还活着，就不能再对 `s` 可变借用。详见 [4.2 规则 2](./4.2-引用与借用.md#规则-2不可变与可变引用不能同时存在)。

**让 `clear` 通过**：先结束不可变借用——

```rust
let mut s = String::from("hello world");
{
    let w = first_word(&s);
} // w 销毁，不可变借用结束
s.clear(); // ✓
```

或根本不保留 `w`，且无其它 `&s` 存活时再 `clear`。

### `&str` 借用的是 `String` 的堆字节（关键一句）

> **`&str` 切片 ≈ 借用原 `String` 底层堆内存；只要 `word` 活着，就等于 `s` 上挂着一份不可变借用**——哪怕类型是 `&str` 而不是 `&String`，照样挡住 `clear()` 拿 `&mut`。

```rust
let mut s = String::from("hello world");
let word: &str = &s[0..5];
// s：栈上 ptr/len/cap；word：胖指针指向 s 在堆里的那一段字节
```

借用检查看的是**谁在用这块堆内存**，不看你是 `&String` 还是 `&str`：

| 借用 | 锁什么 | 阻碍 `clear`？ |
|------|--------|----------------|
| `let r: &String = &s` | 整个字符串对象 | ✅ |
| `let word: &str = &s[..]` | 堆上的一段字节 | ✅ 一样 |
| `let idx: usize = 5` | 无（纯数字） | ❌ 不拦 `clear`，但可能悬空访问 |

`s.clear()` 需要 `&mut self` 独占整段堆数据；若 `word` 仍指向堆里旧内容，清空后 `word` 会悬空 → 编译器禁止在 `word` 存活时发放 `&mut`。

`usize` 不产生借用标记，规则管不到数字 → `clear` 能通过，隐患留在后面的 `&s[0..idx]`（运行时）。

## 四、`str`、`&str` 与字符串字面量

### `&` = 借用标记（读切片前先记）

只要类型/形参前面有 **`&`** → 不拿走所有权，只借用原数据。

| 类型 | 含义 |
|------|------|
| `String` | 拥有堆上数据，`drop` 时释放 |
| `&String` | 借用整个 `String`（只读，除非 `&mut`） |
| `&str` | 借用一段 UTF-8 字符视图（切片） |
| `&[u8]` | 借用一段字节视图（HFT 报文常用） |

**不带 `&` = 转移/获得所有权；带 `&` = 借用。**

函数形参写 `&T`，调用时传 `&变量`：

```rust
fn peek(s: &String) { /* 只读借用，不能 drop s，不能 s.clear()（无 &mut） */ }

let st = String::from("abc");
peek(&st); // st 仍拥有数据，函数结束借用失效
```

`let s = "hello";`：`"hello"` 是静态区里的 `str`，`s` 的类型是 **`&str`**（多为 `&'static str`），本质也是**借用静态内存**；字面量处编译器自动补上 `&`。

「变量前的 `&` vs 冒号后的 `&`、字面量自动 `&str`」见 [4.2 引用与借用](./4.2-引用与借用.md) §两个 `&`。  
完整互斥规则（`&` / `&mut`）也在 4.2。

### 传值 vs 不可变借用 vs 可变借用（三行对照）

```rust
fn by_value(s: String) {}           // 调用方失去所有权（移动进函数）
fn by_ref(s: &str) {}                // 只读借用，可传 &String、&st[..]、"lit"
fn by_mut_ref(s: &mut String) {}     // 可变借用，可 push/clear；同时只能有一个 &mut

let st = String::from("abc");
by_value(st);              // st 之后不可用
// by_ref(&st); by_mut_ref(&mut st);  // st 仍归调用方所有
```

| 方式 | 形参 | 调用 | 原变量 |
|------|------|------|--------|
| 传值 | `String` | `by_value(st)` | 移动后失效 |
| 不可变借用 | `&str` / `&[u8]` | `by_ref(&st)` / `by_ref(&buf[..])` | 仍可用 |
| 可变借用 | `&mut String` | `by_mut_ref(&mut st)` | 仍拥有，但被独占修改权 |

HFT：`fn parse(pkt: &[u8])` 即**只借用报文缓冲区**，字段用子切片 `&pkt[a..b]`，零拷贝、低延迟。

### 两层概念

| 概念 | 含义 |
|------|------|
| **`str`** | 不定长 UTF-8 字节序列（**DST**），不能 `let x: str` 单独放栈/堆变量 |
| **`&str`** | 指向 `str` 的**借用**（指针 + 长度），可以当变量类型 |
| **`"abc"`** | **字符串字面量**：编译期放进程序**只读静态区**（`.rodata`），运行期不 drop |

```rust
let s = "abc"; // s: &str，实际多为 &'static str，指向静态只读内存
```

### 两种来源的 `&str`（易混点）

**来源 1：字面量 → `&'static str`**

```rust
let s: &str = "hello";
// 静态区与程序同寿，不会被 clear/drop，这类 &str 始终有效
```

**来源 2：从 `String` 切片 → 临时借用**

```rust
let st = String::from("xxx");
let s: &str = &st[..]; // 借用 st 管理的堆内存，生命周期跟 st 走
// st 离开作用域后，这个 &str 不能再使用
```

对比 demo：`cargo run` §6，或 `rustc --edition 2021 compile_fail/part_outlives_string.rs`（编译失败）。

### 字面量 `&str` vs `String::from`（不会默认变成 String）

| | `let s = "abc";` | `let s = String::from("abc");` |
|--|------------------|--------------------------------|
| 本质 | `&'static str`，借用 `.rodata` | 堆上**拥有所有权**的 `String` |
| 堆分配 | 无（不拷贝字节到堆） | 从静态区**拷贝** `"abc"` 到堆 |
| 可修改 | 否（只读段，`push` 编译失败） | 是（`push` / `clear` 等） |
| 释放 | 与程序同寿，无 `drop` | 变量出作用域 `drop` 堆内存 |

**为何不默认把 `"abc"` 变成 `String`？** 大量常量若每次都隐式堆分配+拷贝，浪费内存与 CPU；只读文案/协议常量用 `&'static str` 零开销；需要可变或拥有所有权时再显式 `String::from`（Rust：不偷偷堆分配）。

```rust
let s = "abc";                      // 等价 let s: &'static str = "abc";
let mut h = String::from("abc");    // 堆拷贝，可 h.push('d')
```

#### 内存布局（极简）

**`let lit = "abc";` — `lit: &'static str`（胖指针在栈，数据在只读段）**

```
栈 (lit)                    .rodata（程序镜像，只读）
┌─────────────┐             ┌───┬───┬───┐
│ ptr ───────────────►      │ a │ b │ c │
│ len: 3      │             └───┴───┴───┘
└─────────────┘             不 drop、不扩容
```

**`let mut s = String::from("abc");` — `String` 三字段在栈，字节在堆**

```
栈 (s: String)              堆
┌─────────────┐             ┌───┬───┬───┬─── … ───┐
│ ptr ───────────────►      │ a │ b │ c │  (cap≥len) │
│ len: 3      │             └───┴───┴───┴───────────┘
│ cap: ≥3     │             drop(s) 时释放这块堆
└─────────────┘
     ↑ 来源：编译期从 .rodata 拷贝 "abc" 到堆
```

**HFT**：固定指令 `let cmd: &str = "LOGIN"` → 静态 `&str`，零堆开销；动态拼报文 `String::new()` / `Vec<u8>` → 堆或缓冲池扩容。

### 函数不能返回 `str`（只能 `&str` 或 `String`）

**Rust 不能写 `fn f() -> str`**：`str` 是 **DST**，运行时长度不定，栈帧大小须编译期确定，**不能作为返回值、不能单独存变量**；原生 `str` 永远要套引用 `&str`（胖指针：地址 + 长度）。

```rust
// ❌ 编译报错（unsized return type）
fn bad() -> str { "hello" }

// ✓ 返回借用
fn good() -> &str { "hello" }           // let s = good();  // &'static str

// ✓ 返回所有权
fn owned() -> String { String::from("hello") }  // 堆上实体，可 move / push
```

与 `let s = "abc"` 同一逻辑：字面量是静态区 `str` 本体，变量里只能接 **`&str` 引用**，不会拿到裸 `str`。

| 返回类型 | 含义 | 典型场景 |
|----------|------|----------|
| `&str` | 借用一段 UTF-8 视图 | 字面量 → `&'static str`；`&s[0..2]` → 绑定 `s` 生命周期 |
| `String` | 堆上拥有所有权 | 调用方独立持有、可修改、可 move |
| `str` | **非法** | — |

```rust
fn get_static() -> &str { "abcd" }              // res: &'static str

fn get_part(s: &String) -> &str { &s[0..2] }    // r 绑定 st，st drop 后 r 失效
let st = String::from("hello");
let r = get_part(&st);
```

验证：`rustc --edition 2021 compile_fail/return_str_dst.rs`（见 demo README）。

## 五、函数参数：优先 `&str`（Deref 强制转换）

```rust
fn first_word(s: &str) -> &str { /* ... */ }
```

三种传参均合法：

```rust
let st = String::from("xx");
first_word(&st);       // &String → &str（deref coercion）
first_word(&st[..]);
first_word("aaa bbb"); // 字面量 &str
```

工程习惯：**接收字符串优先 `&str`**，比 `&String` 更通用。详见 [ER Item 05](../../ER/Chapter-01-Types/Item-05-type-conversions.md)。

## 六、数组切片 `&[T]`

```rust
let arr = [1, 2, 3, 4, 5];
let sl: &[i32] = &arr[1..3]; // [2, 3]
```

- 与 `&str` 相同结构：指针 + 长度。
- `Vec<T>` 等容器同样支持 `&vec[..]`（第 8 章）。

## 七、类型速查

| 类型 | 含义 |
|------|------|
| `&str` | 字符串切片（UTF-8 字节序列视图） |
| `&[T]` | 任意 `T` 的连续切片 |
| `start..end` | Range，左闭右开 |

---

## 八、易错点

| 陷阱 | 后果 |
|------|------|
| 非 UTF-8 边界切片 | 运行时 panic |
| 返回 `usize` 索引 | 与原数据生命周期脱钩 |
| 切片存活时修改原容器 | 编译错误（预期行为） |
| 把 `&String` 当最佳参数 | 可用但不如 `&str` 通用 |
| 混淆静态/堆 `&str` | 字面量 `&'static str` vs `String` 切片临时借用 |

---

## 九、HFT 视角（零拷贝 `&[u8]`）

1. **行情二进制报文**常用 `&[u8]`：解析 UDP/共享内存缓冲区时**不拷贝**整包，只对字段做子切片，降低延迟。
2. **避免裸 `usize` 分段下标**；用切片绑定缓冲区生命周期，编译期防越界与悬空。
3. **解析函数统一 `&[u8]`**：兼容静态数组、`Vec`、mmap 映射区。

实战 demo：`4.3-slices-demo` 中 `parse_tick()` —— 模拟 `[magic:2][price:8][qty:4]` 定长头；另有 **`Vec<u8>` 缓冲区** 与裸 `usize` 下标对比见 demo §5。

## 十、Vec 报文缓冲区：`usize` vs `&[u8]`（巩固）

```rust
let mut buf: Vec<u8> = receive_udp_packet();
let off = 10usize;           // 裸下标：buf 被覆写/truncate 后 off 仍「存在」
let hdr = &buf[0..14];       // 切片：hdr 存活期间 buf 不能可变借用（clear/resize 等）
```

- **`Vec` 与静态数组一样**：`&buf[..]` 得到 `&[u8]`，不拷贝整包。
- **HFT 写法**：解析函数入参 `pkt: &[u8]`，返回的字段视图仍是 `&[u8]` 子切片，绑定 `Vec` 或 mmap 区生命周期。

## 十一、记忆卡片（汇总）

| 要点 | 一句 |
|------|------|
| `usize` | 只是数；用其下标访问已清空数据 → 悬空**访问** |
| `&str`（堆） | 借用 `String`，有活跃切片则不能 `clear` |
| `&str`（字面量） | `&'static str`，指向只读静态区 |
| `str` vs `&str` | DST 本体 vs 引用包装；**函数不能 `-> str`** |
| Slice | 指针 + 长度，不拥有数据 |
| API | 字符串参数优先 `&str`；二进制优先 `&[u8]` |
| HFT | 用 `&[u8]` 零拷贝子切片，不用裸 `usize` 偏移 |
