# 4.3 切片 Slice

**Slice**（`&str` / `&[T]`）：只存**起始指针 + 长度**，不持有底层数据所有权，借用原容器。

对应 demo：[4.3-slices-demo](./4.3-slices-demo/)（含 **HFT 零拷贝 `&[u8]` 报文分段**）

---

## 一、核心本质

解决痛点：单纯返回 `usize` 下标与原数据脱钩，原数据修改/清空后下标失效；**`&str` / `&[T]` 绑定生命周期，借用检查在编译期拦截非法修改**。

---

## 二、旧方案缺陷：返回 `usize` 下标

```rust
fn first_word_index(s: &String) -> usize {
    let bytes = s.as_bytes();
    for (i, &b) in bytes.iter().enumerate() {
        if b == b' ' {
            return i;
        }
    }
    s.len()
}
```

```rust
let mut s = String::from("hello world");
let idx = first_word_index(&s);
s.clear(); // 清空原字符串
// idx 仍是 5，但字符串已空 —— 悬空索引，编译器查不出
```

| 问题 | 说明 |
|------|------|
| 裸数字 | `usize` 与 `String` 无绑定 |
| 悬空索引 | 原数据销毁/修改后下标仍“合法” |
| 借用规则 | 无法介入，只能运行时踩坑 |

### 大白话：`usize` 裸数字 vs `&str` 切片

**旧版返回 `usize`：** `i = 5` 只是栈上的一个整数，表示「空格在第 5 个字节处」。它**不知道**自己对应哪段 `String`，也不管 `s` 后来被 `clear()` 还是扩容——借用检查**管不到这个数**。

```rust
let mut s = String::from("hello world");
let idx = first_word_index(&s); // idx = 5，孤零零的数字
s.clear();                      // s 变成 ""，idx 仍是 5
// 若再写 &s[0..idx] → 越界；编译器不会在 clear 时报错
```

**新版返回 `&str`：** `&s[0..i]` 不是数字，是**指向 `s` 堆内存的借用**（指针 + 长度）。只要 `word: &str` 还活着，`s` 就不能再被可变借用（例如 `clear()`），**编译期**拦住下标失效。

```rust
let mut s = String::from("hello world");
let word = first_word(&s); // word 借用了 s
s.clear();                 // ❌ 编译错误：不可变借用与可变借用冲突
```

| 对比 | `usize` 裸下标 | `&str` / `&[u8]` 切片 |
|------|----------------|----------------------|
| 本质 | 普通整数 | 绑定原容器的引用 |
| 原数据被改 | 数字不变，编译器不管 | 有切片存活时禁止可变修改 |
| 出错时机 | 往往**运行时**越界 | 往往**编译期**拒绝 |

**`as_bytes()` 做什么？** `s.as_bytes()` 得到 `&[u8]`，用来**遍历**找空格位置 `i`；旧方案把 `i` 当返回值，新方案用 `i` 生成 `&s[0..i]` 再返回切片。

## 三、字符串切片 `&str`

### Range（左闭右开，不含结束下标）

| 写法 | 等价 | 说明 |
|------|------|------|
| `[0..5]` | 0～4 | 常规区间 |
| `[..2]` | `[0..2]` | 左边界可省略 |
| `[3..]` | `[3..len]` | 右边界可省略 |
| `[..]` | `[0..len]` | 全切片 |

```rust
let s = String::from("hello world");
let hello = &s[0..5];
let world = &s[6..11];
```

⚠️ **切片边界必须落在 UTF-8 字符字节边界**，不能从多字节字符中间切，否则 **panic**（`&s[0..1]` 切到多字节字符中间）。

### 切片版 `first_word`

```rust
fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}
```

借用规则生效：

```rust
let mut s = String::from("hello world");
let w = first_word(&s);
s.clear(); // ❌ 编译错误：w 持有 s 的不可变借用
println!("{w}");
```

## 四、字符串字面量天生 `&str`

```rust
let s = "abc"; // &str，指向只读段
```

字面量在程序只读区，`&str` 不可通过安全代码修改。

---

## 五、函数参数：优先 `&str`（Deref 强制转换）

```rust
fn first_word(s: &str) -> &str { /* ... */ }
```

三种传参均合法：

```rust
let st = String::from("xx");
first_word(&st);       // &String → &str（deref coercion）
first_word(&st[..]);
first_word("aaa bbb"); // 字面量 &str
```

工程习惯：**接收字符串优先 `&str`**，比 `&String` 更通用。详见 [ER Item 05](../../ER/Chapter-01-Types/Item-05-type-conversions.md)。

---

## 六、数组切片 `&[T]`

```rust
let arr = [1, 2, 3, 4, 5];
let sl: &[i32] = &arr[1..3]; // [2, 3]
```

- 与 `&str` 相同结构：指针 + 长度。
- `Vec<T>` 等容器同样支持 `&vec[..]`（第 8 章）。

---

## 七、类型速查

| 类型 | 含义 |
|------|------|
| `&str` | 字符串切片（UTF-8 字节序列视图） |
| `&[T]` | 任意 `T` 的连续切片 |
| `start..end` | Range，左闭右开 |

---

## 八、易错点

| 陷阱 | 后果 |
|------|------|
| 非 UTF-8 边界切片 | 运行时 panic |
| 返回 `usize` 索引 | 与原数据生命周期脱钩 |
| 切片存活时修改原容器 | 编译错误（预期行为） |
| 把 `&String` 当最佳参数 | 可用但不如 `&str` 通用 |

---

## 九、HFT 视角（零拷贝 `&[u8]`）

1. **行情二进制报文**常用 `&[u8]`：解析 UDP/共享内存缓冲区时**不拷贝**整包，只对字段做子切片，降低延迟。
2. **避免裸 `usize` 分段下标**；用切片绑定缓冲区生命周期，编译期防越界与悬空。
3. **解析函数统一 `&[u8]`**：兼容静态数组、`Vec`、mmap 映射区。

实战 demo：`4.3-slices-demo` 中 `parse_tick()` —— 模拟 `[magic:2][price:8][qty:4]` 定长头；另有 **`Vec<u8>` 缓冲区** 与裸 `usize` 下标对比见 demo §5。

---

## 十、Vec 报文缓冲区：`usize` vs `&[u8]`（巩固）

```rust
let mut buf: Vec<u8> = receive_udp_packet();
let off = 10usize;           // 裸下标：buf 被覆写/truncate 后 off 仍「存在」
let hdr = &buf[0..14];       // 切片：hdr 存活期间 buf 不能可变借用（clear/resize 等）
```

- **`Vec` 与静态数组一样**：`&buf[..]` 得到 `&[u8]`，不拷贝整包。
- **HFT 写法**：解析函数入参 `pkt: &[u8]`，返回的字段视图仍是 `&[u8]` 子切片，绑定 `Vec` 或 mmap 区生命周期。

---

## 小结

| 要点 | 一句 |
|------|------|
| Slice | 指针 + 长度，不拥有数据 |
| 绑定生命周期 | 有切片则原数据不可变借用冲突 |
| API | 字符串参数优先 `&str`；二进制优先 `&[u8]` |
| HFT | 零拷贝子切片解析报文字段 |
