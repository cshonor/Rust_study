# 5.1 定义并实例化结构体

结构体（`struct`）与元组类似：都能把多个值组合在一起；不同的是结构体**为每一部分命名字段**，不依赖位置顺序，更清晰、更灵活。

对应 demo 项目：`Book/05-structs/5.1-structs-demo/`

---

## 定义结构体

使用 `struct` + 结构体名 + 花括号定义字段（field）：

```rust
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
```

---

## 实例化结构体

创建实例：`结构体名 { 字段: 值, ... }`（字段顺序不要求与定义一致）：

```rust
let user1 = User {
    email: String::from("someone@example.com"),
    username: String::from("someusername123"),
    active: true,
    sign_in_count: 1,
};
```

访问字段：点语法 `user1.email`。

---

## 修改字段

要修改字段，**整个实例**必须是可变的（不能只让某个字段可变）：

```rust
let mut user1 = User {
    email: String::from("someone@example.com"),
    username: String::from("someusername123"),
    active: true,
    sign_in_count: 1,
};

user1.email = String::from("anotheremail@example.com");
```

---

## 函数返回结构体

可以在函数末尾构造结构体作为最后一个表达式隐式返回：

```rust
fn build_user(email: String, username: String) -> User {
    User {
        email: email,
        username: username,
        active: true,
        sign_in_count: 1,
    }
}
```

### 字段初始化简写（field init shorthand）

当变量名与字段名相同，可简写：

```rust
fn build_user(email: String, username: String) -> User {
    User {
        email,
        username,
        active: true,
        sign_in_count: 1,
    }
}
```

---

## 结构体更新语法（struct update syntax）

用旧实例的大部分字段创建新实例：

```rust
let user2 = User {
    email: String::from("another@example.com"),
    ..user1
};
```

**注意（与所有权相关）**：
- `..user1` 会把未显式赋值的字段从 `user1` **移动/拷贝**到新实例
- **`String` 会被移动**（不实现 `Copy`），所以 `user1` 可能在这之后不可用
- `bool`、`u64` 等 **Copy 类型会被拷贝**，不影响旧实例继续使用  

这与第 4 章的「移动 / Copy」规则一致。

---

## 元组结构体（tuple struct）

像元组一样按位置存字段，但有自己的类型名（更强类型区分）：

```rust
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

let black = Color(0, 0, 0);
let origin = Point(0, 0, 0);
```

- `Color` 与 `Point` 是不同类型，即使字段完全相同
- 访问：`black.0`、`black.1`、`black.2`

---

## 类单元结构体（unit-like struct）

没有任何字段，像 `()` 一样：

```rust
struct AlwaysEqual;

let subject = AlwaysEqual;
```

常用于：需要在某类型上实现 trait，但不需要存储数据（第 10 章详述 trait）。

---

## 结构体字段所有权

结构体字段常用 `String`（拥有所有权）而不是 `&str`（引用）：

```rust
struct User {
    username: String,
    email: String,
}
```

如果结构体想存引用（如 `&str`），需要显式生命周期参数（第 10 章再讲）。否则会报：
`missing lifetime specifier`。
