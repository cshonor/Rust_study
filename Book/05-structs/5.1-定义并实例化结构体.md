# 5.1 定义并实例化结构体

结构体（`struct`）与元组类似：都能把多个值组合在一起；不同的是结构体**为每一部分命名字段**，不依赖位置顺序，更清晰、更灵活。

对应 demo 项目：`Book/05-structs/5.1-structs-demo/`

---

## 定义结构体

使用 `struct` + 结构体名 + 花括号定义字段（field）：

```rust
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
```

---

## 实例化结构体

创建实例：`结构体名 { 字段: 值, ... }`（字段顺序不要求与定义一致）：

```rust
let user1 = User {
    email: String::from("someone@example.com"),
    username: String::from("someusername123"),
    active: true,
    sign_in_count: 1,
};
```

访问字段：点语法 `user1.email`。

---

## 修改字段（承袭可变性）

> **要改任意字段：整个实例必须 `let mut`；不能在 `struct` 定义里给单个字段写 `mut`。**

可变性是**变量绑定**的属性，不是字段本身的属性：

| 绑定 | 效果 |
|------|------|
| `let user1 = User { ... }` | 全部字段不可改，任何 `user1.xxx = ...` 报错 |
| `let mut user1 = User { ... }` | 所有字段都可改 |

```rust
let mut user1 = User {
    email: String::from("someone@example.com"),
    username: String::from("someusername123"),
    active: true,
    sign_in_count: 1,
};

user1.email = String::from("anotheremail@example.com"); // ✓ 改单个字段
// struct User { mut email: String }  // ❌ 非法语法
```

**口诀**：字段没有自己的 mut，跟着实例的 `mut` 走。

### 实例不可变时仍想改某一字段？

| 方案 | 场景 |
|------|------|
| **构造新实例** | `let u2 = User { email: new, ..u1 }`（见下节） |
| **内部可变性** | `Cell<T>`（Copy）、`RefCell<T>`（非 Copy）包字段，实例本身不必 `mut`（第 15 章智能指针） |
| **字段级 `&mut` 借用** | `let mut u = ...; let e = &mut u.email; *e = new.into();` 与 `&u.username` 可并存（不同字段不冲突） |

---

## 原地改字段 vs `..` 更新（所有权）

两种方式都能「换 email」，但机制不同：

| | **`user1.email = new`**（原地） | **`let u2 = User { email: new, ..user1 }`** |
|--|-----------------------------------|---------------------------------------------|
| 前提 | `let mut user1` | `user1` 可读即可（不必 mut） |
| 实例 | **同一个** `user1` | **新建** `u2` |
| 原 `user1` | 仍在，只改了字段 | 非 Copy 字段可能被 **move**，部分失效 |
| `String` 字段 | 旧 `String` **drop**，赋入新值 | 显式写的 `email` 是新串；`..` 可能 **move** `username` 等 |
| 典型用途 | 就地更新状态 | 函数式「基于旧值生成新值」、保留旧副本（用 `clone`） |

```rust
let mut user1 = User { /* ... */ };
user1.email = String::from("in-place@x.com"); // 同一实例，旧 email 被 drop

let user2 = User {
    email: String::from("new@x.com"),
    ..user1   // username 等从 user1 move 到 user2；user1 部分失效
};
```

- **`let mut` + `.字段=`**：不新建 struct，不涉及 `..` 的部分移动（除非你把某字段 move 出去赋给别的变量）。
- **更新语法**：总是**新 struct**；`..` 按字段 Copy/Move → 见下节。

## 函数返回结构体

可以在函数末尾构造结构体作为最后一个表达式隐式返回：

```rust
fn build_user(email: String, username: String) -> User {
    User {
        email: email,
        username: username,
        active: true,
        sign_in_count: 1,
    }
}
```

### 字段初始化简写（field init shorthand）

当变量名与字段名相同，可简写：

```rust
fn build_user(email: String, username: String) -> User {
    User {
        email,
        username,
        active: true,
        sign_in_count: 1,
    }
}
```

## 结构体更新语法（`..orig`）

用旧实例的大部分字段创建新实例；**自定义字段写在前，`..源` 固定在最后**。

```rust
let user2 = User {
    email: String::from("another@example.com"), // 显式覆盖，不从 .. 走
    ..user1
};
```

### `..user1` = 按字段展开，遵循 Copy / Move

等价手写：`active: user1.active, username: user1.username, ...`，每条赋值仍走第 4 章规则：

| 字段类型 | `..user1` 时 |
|----------|----------------|
| `bool`、`u64` 等 **Copy** | **拷贝**值 → `user1` 上该字段仍可用 |
| **`String` 等非 Copy** | **Move** 所有权 → `user1` 上该字段失效 |

**部分移动（partial move）**：只要有一个非 Copy 字段被 `..` 移走，**整个 `user1` 不能整体再用**（如 `println!("{user1:?}")` 报错），但 **Copy 字段仍可单独访问**：

```rust
// println!("{:?}", user1);     // ❌ partial moved
println!("{}", user1.active);    // ✅ Copy 字段
println!("{}", user1.sign_in_count);
// println!("{}", user1.username); // ❌ 已 move
```

显式写在前面的字段（如 `email`）**永远不会**被 `..` 移动。

### 三种写法对比（保留原实例）

| 写法 | 非 Copy 字段 | 原实例 `user1` |
|------|--------------|----------------|
| `..user1` | Move | 部分失效 |
| `username: user1.username.clone(), ..user1` | 手动 clone 后，其余仍可能 move | 若 `..` 不再碰 `username` 可保留更多 |
| 全 Copy 的 struct + `..i1` | 全拷贝 | 完好 |
| `..user1.clone()`（需 `#[derive(Clone)]`） | **Clone** 剩余字段 | **完好**（堆上多一份） |

```rust
// 保留 user1：对单个字段先 clone，或整体 clone 再 ..
let user2 = User {
    email: String::from("new@xxx.com"),
    username: user1.username.clone(),
    ..user1
};

#[derive(Clone)]
struct User { /* ... */ }
let user3 = User {
    email: String::from("new@xxx.com"),
    ..user1.clone()   // 剩余字段从克隆体取，user1 不 move
};
```

`#[derive(Copy, Clone)] struct Info { a: u32, b: bool }` → `let i2 = Info { a: 99, ..i1 };` 后 `i1` 仍完整可用。

### 整体 move vs 部分 move vs 克隆式更新

| 场景 | 写法 | 结果 |
|------|------|------|
| **整体 move** | `let u2 = u1;` | `u1` 完全失效 |
| **部分 move** | `let u2 = User { email: new, ..u1 }` | `u1` 只能访问未移动的 Copy 字段 |
| **克隆式更新** | `let u2 = User { ..u1.clone() }` | `u1` 保留；非 Copy 字段被 **clone** 进 `u2` |

与 [4.1 Copy / Move](../04-ownership/4.1-什么是所有权.md) 一致。可跑 demo：`5.1-structs-demo` §3、§6。

## 元组结构体（tuple struct）

像元组一样按位置存字段，但有自己的类型名（更强类型区分）：

```rust
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

let black = Color(0, 0, 0);
let origin = Point(0, 0, 0);
```

- `Color` 与 `Point` 是不同类型，即使字段完全相同
- 访问：`black.0`、`black.1`、`black.2`

---

## 类单元结构体（unit-like struct）

没有任何字段，像 `()` 一样：

```rust
struct AlwaysEqual;

let subject = AlwaysEqual;
```

常用于：需要在某类型上实现 trait，但不需要存储数据（第 10 章详述 trait）。

## 结构体字段所有权

结构体字段常用 `String`（拥有所有权）而不是 `&str`（引用）：

```rust
struct User {
    username: String,
    email: String,
}
```

如果结构体想存引用（如 `&str`），需要显式生命周期参数（第 10 章再讲）。否则会报：
`missing lifetime specifier`。

### `let mut` + 字段是 `&str`：换指向，不改只读内存

> **`mut` 管的是「字段里存什么值/引用」，不管「引用指向的那块内存能不能写」。**

**字段 `&'static str`（字面量）**

```rust
struct User {
    name: &'static str,
    num: u32,
}

let mut u = User { name: "zhangsan", num: 10 };
u.name = "lisi"; // ✅ 合法：把字段里的指针改指向另一段静态只读串
// 没有改写 "zhangsan" 在 .rodata 里的字节
```

| | `&'static str` 字段 + `mut` 实例 | `String` 字段 + `mut` 实例 |
|--|----------------------------------|------------------------------|
| `u.name = 新值` | **换引用指向**（合法） | **换堆所有权** / 新 `String` |
| 改指向内存里的字节 | ❌ 静态区不可写 | ✅ `u.name.push('x')` 等 |
| 误区 | `mut` ≠ 能改 `"abc"` 本体 | 堆上可原地改 |

```rust
let mut s = "abc"; // s: &str
// s[0] = b'd';    // ❌ 不能改只读区内容
```

**字段借外部 `String` 的 `&str`（非 `'static`）**

```rust
struct User { name: &'str }

let mut outer = String::from("123");
let mut u = User { name: &outer };
u.name = &outer[1..]; // ✅ 仍只是换指针；不能通过 u.name 改 outer 里字符
```

生命周期、`outer` drop 后 `u.name` 失效 → 第 10 章。静态/堆 `&str` 区别 → [4.3.3](../04-ownership/4.3.3-str与str类型.md)。
