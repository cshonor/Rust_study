# 5.1.5 结构体变量：栈上实体 vs 引用 vs `Box`

← 主笔记：[5.1](./5.1-定义并实例化结构体.md) · 栈/堆：[4.1](../04-ownership/4.1-什么是所有权.md) · 方法 `&self`：[5.3](../5.3-方法语法.md)

**结论**：`let r = Rect { ... }` 时，**`r` 是结构体实体（栈上整块字段）**，不是指针；要指针才写 `&`。

---

## 1. 全栈字段的例子

```rust
#[derive(Debug, Copy, Clone)]
struct Rect { w: u32, h: u32 }

let r1 = Rect { w: 10, h: 20 };
```

- **`r1`**：栈上直接放 `w`、`h` 两个 `u32`，**整块 struct 实体**
- 无额外堆分配（字段全是栈类型时）

```rust
let r2 = &r1; // r2: &Rect —— 这才是指针（栈上存地址+元数据）
```

---

## 2. 三种形态

| 写法 | 变量是什么 | 典型内存 |
|------|------------|----------|
| `let a = Rect { ... }` | **实体**，拥有值 | 字段在栈（全栈类型时） |
| `let a = &Rect { ... }` | **不可变引用**（指针） | 栈上放 `&`，指向别处 |
| `let a = Box::new(Rect { ... })` | **堆指针**（拥有堆数据） | 栈上 `Box` 指针，实体在堆 |

> **口诀**：不加 `&` → 实体；加 `&` → 引用；`Box` → 栈指针 + 堆上本体。

---

## 2.1 三种内存布局（图式）

**铁律**：**变量绑定的 struct 本体在栈**；有没有堆，看字段里有没有 `String` / `Vec` / `Box` 等。

### ① 全基础字段 `Rect` —— 整 struct 在栈，无堆

```rust
struct Rect { w: u32, h: u32 }
let r = Rect { w: 10, h: 20 };
```

```
栈 (r: Rect)
┌────┬────┐
│ w  │ h  │   8 字节，无指针、无堆
└────┴────┘
```

`let r2 = r1;`：Copy 时栈上整份拷贝；非 Copy 时 move 栈上 8 字节。

### ② 含 `String` 的 `User` —— 壳在栈，字节在堆

```rust
struct User { name: String }
let u = User { name: String::from("张三") };
```

```
栈 (u: User)                 堆
┌─────────────────┐          ┌──────────────┐
│ name.ptr ───────────────►  │ 张 三 (UTF-8) │
│ name.len        │          └──────────────┘
│ name.cap        │
└─────────────────┘
     ↑ String 三字段在栈（常见 24 字节，平台相关）
```

- **struct 容器在栈**；**字符串内容在堆**
- `let u2 = u1;`：move 栈上 `User` + **堆所有权**交给 `u2`（不是只拷指针）

持久化 struct 常见这种形态 → [5.4](./5.4-结构体与持久化.md)。

### ③ `Box<Rect>` —— 栈上只有指针，Rect 在堆

```rust
let r = Box::new(Rect { w: 1, h: 2 });
```

```
栈 (r: Box<Rect>)            堆
┌──────────┐                 ┌────┬────┐
│ ptr ───────────────►       │ w  │ h  │
└──────────┘                 └────┴────┘
```

即使 `Rect` 本身很小，也**整颗**放在堆上。

---

## 2.2 对照表

| 类型 | struct 在哪 | 数据在哪 | 堆？ |
|------|-------------|----------|------|
| `Rect { u32, u32 }` | 栈 | 全在栈 | 无 |
| `User { String }` | 栈（壳） | 字符在堆 | 有 |
| `Box<Rect>` | 栈（指针） | Rect 在堆 | 有 |

**极简**：普通值在栈；`String`/`Vec` **内容**在堆；**struct 变量本身在栈**，内部字段决定要不要堆。

---

## 3. 与 `&self` 串起来

```rust
impl Rect {
    fn get_w(&self) -> u32 { self.w }
}
let r = Rect { w: 5, h: 8 };
r.get_w(); // 编译器：(&r).get_w()
```

- **`r` 本身不是指针**
- **`&self` 是借用的指针**；调用时编译器自动 `&r`

---

## 4. 为何会有 move

```rust
let r1 = Rect { w: 1, h: 2 };
let r2 = r1; // 整块实体 move 到 r2（非 Copy 时 r1 失效）
// println!("{:?}", r1); // ❌
```

变量存**实体**才会「整份移走、旧变量失效」。若 `r1` 只是 `&Rect` 指针，Copy 指针后两个引用共存，不会出现这种 move。

---

## 5. 对照 C

| | C | Rust |
|--|---|------|
| 栈上实体 | `Rect r = {10, 20};` | `let r = Rect { w: 10, h: 20 };` |
| 指针 | `Rect *p = &r;` | `let p: &Rect = &r;` |
| 堆上实体 | `Rect *p = malloc(...);` | `let b = Box::new(Rect { ... });` |
| move 语义 | 赋值常是浅拷指针 | 非 `Copy` 类型默认 **move** 实体 |

C 里 `Rect r2 = r1;` 常是**逐字段拷贝**（浅拷）；Rust 对含 `String` 的 struct 是 **move 堆所有权**，不是 C 式双指针。

---

## 小结

| 要点 | 一句 |
|------|------|
| 普通 `let r = ...` | 实体，不是指针 |
| `&r` | 引用 / 指针 |
| `Box` | 拥有堆上实体 |
| move | 因变量存实体（或实体内的堆字段） |
