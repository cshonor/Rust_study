# 5.1.4 `Debug` / `Display` 与 Trait 静态约束

← 主笔记：[5.1](./5.1-定义并实例化结构体.md) · Trait 总览：[10.2 trait](../10-generics-traits-lifetimes/10.2-trait.md) · 可 derive 列表：[附录 C.3](../appendix/C.3-可derive的Trait.md)

**结论**：Rust **不是 Python 鸭子类型**——能否 `{}` / `{:?}` 打印、能否调某方法，**编译期**看是否 **impl 对应 Trait**。

---

## `{}` vs `{:?}`

| 格式 | Trait | 谁实现 |
|------|-------|--------|
| **`{}`** | **`Display`** | **手写** `impl`（面向用户的展示格式） |
| **`{:?}`** | **`Debug`** | 常 **`#[derive(Debug)]`**（调试输出） |

```rust
struct User { name: String }
let u = User { name: "aa".into() };

// println!("{}", u);   // ❌ 未实现 Display
// println!("{:?}", u);  // ❌ 未实现 Debug

#[derive(Debug)]
struct User { name: String }
println!("{:?}", u);    // ✅

// println!("{}", u);   // 仍 ❌，Display 不能 derive
```

### 手写 `Display`

```rust
use std::fmt;

impl fmt::Display for User {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "用户名：{}", self.name)
    }
}
// 此后 println!("{}", u); ✅
```

官方刻意分开：**Debug** 给程序员；**Display** 给最终用户，格式由你定。

---

## `#[derive(Debug)]` 是什么？

**语法糖**：编译器自动帮你写 `impl Debug for User { ... }`。  
本质仍是**显式实现 trait**，不是运行时「碰巧能打印」。

- 不加 derive → 没 `Debug` → `{:?}` **编译报错**
- 加 derive → 满足约束 → **编译通过**

更多可 derive 的 trait → [附录 C.3](../appendix/C.3-可derive的Trait.md)。

## Rust ≠ Python 鸭子类型

| | **Python 鸭子** | **Rust Trait** |
|--|-----------------|----------------|
| 检查时机 | **运行时**找方法 | **编译期**查 impl |
| 约束 | 「有同名方法就行」 | **必须 `impl Trait`** |
| 失败 | `AttributeError` 运行时崩 | **编译错误** |

```python
# Python：运行时才知 x 有没有 hello
def test(x):
    x.hello()
```

```rust
trait Hello { fn hello(&self); }
struct B;
// fn test<T: Hello>(x: T) { x.hello(); }
// test(B);  // ❌ 编译期：B 未 impl Hello
```

> **Rust**：有 impl = 能用；无 impl = 写代码阶段就拦。  
> **Python**：跑起来才知道有没有那个方法。

Wasm 等同理：trait 在编译期校验完， wasm 里不会出现「动态找不到方法」那类错误（对比 JS/Python）。

Trait 泛型约束、`impl Trait` → [10.2](../10-generics-traits-lifetimes/10.2-trait.md)。

## 小结

| 要点 | 一句 |
|------|------|
| `{:?}` | 要 `Debug`，常 `derive` |
| `{}` | 要 `Display`，手写 |
| 鸭子类型 | Rust 无；静态 Trait 绑定 |
| derive | 自动 `impl`，仍是显式约束 |
