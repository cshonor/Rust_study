# 5.1.2 结构体更新语法（`..orig`）

← 主笔记：[5.1](./5.1-定义并实例化结构体.md) · 所有权：[4.1 Copy/Move](../04-ownership/4.1-什么是所有权.md) · demo：[5.1-structs-demo](./5.1-structs-demo/)

---

## 语法

**自定义字段在前，`..源` 固定在最后**：

```rust
let user2 = User {
    email: String::from("another@example.com"),
    ..user1
};
```

显式写的字段（如 `email`）**不会被 `..` 移动**。

---

## `..user1` = 按字段 Copy / Move

| 字段类型 | `..user1` 时 |
|----------|----------------|
| **Copy**（`bool`、`u64`） | 拷贝 → `user1` 上仍可用 |
| **非 Copy**（`String`） | **Move** → `user1` 上该字段失效 |

**部分移动**：任一非 Copy 字段被 `..` 移走 → **不能整体再用 `user1`**，Copy 字段仍可单独访问：

```rust
// println!("{:?}", user1);        // ❌
println!("{}", user1.active);      // ✅
// println!("{}", user1.username); // ❌ moved
```

---

## 保留原实例

| 写法 | 原 `user1` |
|------|------------|
| `..user1` | 部分失效 |
| `username: user1.username.clone(), ..user1` | 视剩余字段而定 |
| `..user1.clone()`（需 `Clone`） | **完好** |
| 全 `Copy` struct + `..i1` | **完好** |

```rust
#[derive(Clone)]
struct User { /* ... */ }

let user3 = User {
    email: String::from("new@xxx.com"),
    ..user1.clone()
};
```

---

## 三种 move 对照

| 场景 | 写法 | 结果 |
|------|------|------|
| 整体 move | `let u2 = u1;` | `u1` 完全失效 |
| 部分 move | `User { email: new, ..u1 }` | 只剩 Copy 字段可读 |
| 克隆式更新 | `User { ..u1.clone() }` | `u1` 保留 |

demo：`cargo run` §3 `..clone()`，§4 部分 move，§5 全 Copy `Info`。

---

## 小结

| 要点 | 一句 |
|------|------|
| `..` | 逐字段赋值，走 Copy/Move |
| partial move | 一个 `String` 被移走，整体不能 debug 打印 |
| 保留原值 | `clone()` 或全 Copy 字段 |
