## 承袭可变性

> **要改任意字段：整个实例必须 `let mut`；不能在 `struct` 定义里给单个字段写 `mut`。**

可变性是**变量绑定**的属性，不是字段本身的属性：

| 绑定 | 效果 |
|------|------|
| `let user1 = User { ... }` | 全部字段不可改 |
| `let mut user1 = User { ... }` | 所有字段都可改 |

```rust
let mut user1 = User {
    email: String::from("someone@example.com"),
    username: String::from("someusername123"),
    active: true,
    sign_in_count: 1,
};

user1.email = String::from("anotheremail@example.com"); // ✓
// struct User { mut email: String }  // ❌ 非法
```

**口诀**：字段没有自己的 mut，跟着实例的 `mut` 走。

### 实例不可变时仍想改字段？

| 方案 | 场景 |
|------|------|
| **构造新实例** | `let u2 = User { email: new, ..u1 }` → [5.1.2](./5.1.2-结构体更新语法.md) |
| **内部可变性** | `Cell` / `RefCell` 包字段（第 15 章） |
| **字段级借用** | `let e = &mut u.email;` 与 `&u.username` 可并存 |

---

## 原地改字段 vs `..` 更新

| | **`user1.email = new`** | **`let u2 = User { email: new, ..user1 }`** |
|--|-------------------------|---------------------------------------------|
| 前提 | `let mut user1` | `user1` 可读即可 |
| 实例 | **同一个** | **新建** `u2` |
| 原 `user1` | 仍在，只改字段 | 可能被 **部分 move** |

```rust
let mut user1 = User { /* ... */ };
user1.email = String::from("in-place@x.com");

let user2 = User {
    email: String::from("new@x.com"),
    ..user1
};
```

`..` 的细节 → [5.1.2 结构体更新语法](./5.1.2-结构体更新语法.md)。demo §2 原地改，§4 部分 move。

---

## 小结

| 要点 | 一句 |
|------|------|
| `mut` | 整实例可变，不能 per-field |
| 原地 `.=` | 同一 struct，旧 `String` drop |
| `..` 更新 | 新 struct，可能 partial move |
