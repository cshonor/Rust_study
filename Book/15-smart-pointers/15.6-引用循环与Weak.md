## 核心结论（先背）

1. **`Rc` 强计数 > 0 就不释放**；两个 `Rc` **互指** → 计数永 ≥ 1 → **内存泄漏**（仍 memory safe，无 UAF）。
2. **`Weak<T>` 不增加强计数**，只做观察；需 `upgrade()` → `Option<Rc<T>>`。
3. **规则**：正向/所有权边用 **`Rc`**，回边/父指针用 **`Weak`**。

---

## 一、为什么会出现循环引用？

`Rc` 靠 **strong_count** 管理堆内存：

- **> 0** → 数据保留  
- **= 0** → `Drop` 释放  

若 **A 强引用 B 且 B 强引用 A**，变量出作用域后各自 `-1`，但**内部仍互相持有**，计数永远 ≥ 1 → **泄漏**。

Rust **允许泄漏**（不算 UB），但不会自动帮你破环。

---

## 二、实战：双向节点（经典环）

```rust
use std::rc::Rc;
use std::cell::RefCell;

#[derive(Debug)]
struct Node {
    val: i32,
    prev: Option<Rc<RefCell<Node>>>,
}

let a = Rc::new(RefCell::new(Node { val: 1, prev: None }));
let b = Rc::new(RefCell::new(Node {
    val: 2,
    prev: Some(Rc::clone(&a)), // B → A，a 计数 2
}));

a.borrow_mut().prev = Some(Rc::clone(&b)); // A → B，b 计数 2 → 成环！
```

### 计数拆解

| 步骤 | a 强计数 | b 强计数 |
|------|----------|----------|
| 创建 `a` | 1 | — |
| 创建 `b`（`prev` 指 `a`） | **2** | 1 |
| `a.prev = Some(b)` | 2 | **2** |
| 变量 `a`、`b` 出作用域各 -1 | **1** | **1** |
| 结果 | 永不归 0 → **泄漏** | |

→ demo：`cargo run -- node` §泄漏

## 三、解法：`Weak<T>` 弱引用

| | `Rc`（强） | `Weak`（弱） |
|---|-----------|--------------|
| 计数 | `strong_count +1` | `weak_count +1`，**不影响释放** |
| 访问数据 | 直接 `Deref` | 须 **`upgrade()`** → `Option<Rc<T>>` |
| 释放判定 | strong = 0 即释堆 | weak 可残留，upgrade 得 `None` |

### 修正：前驱用 `Weak`

```rust
use std::rc::{Rc, Weak};

struct Node {
    val: i32,
    prev: Option<Weak<RefCell<Node>>>, // 弱引用，不增 strong
}

let a = Rc::new(RefCell::new(Node { val: 1, prev: None }));
let b = Rc::new(RefCell::new(Node {
    val: 2,
    prev: Some(Rc::downgrade(&a)),
}));

a.borrow_mut().prev = Some(Rc::downgrade(&b)); // A 弱指 B，不成环

// 出作用域：a、b strong 归 0 → 正常释放
```

### 常用 API

- `Rc::downgrade(&rc)` → `Weak<T>`
- `weak.upgrade()` → `Some(Rc)` 或 `None`

→ demo：`cargo run -- node` §Weak · `cargo run` §Weak 树（`weak_tree.rs`）

## 五、整体脉络（串起 Ch.15）

```text
Box<T>           独占，不能共享
Rc<T>            共享，默认只读，strong_count
Rc<Cell<T>>      共享 + 改 Copy 小值
Rc<RefCell<T>>   共享 + 改复杂值（单线程标准）
Rc 互指          strong 不归零 → 泄漏
Weak<T>          回边/父指针，不增 strong
Arc + Mutex      多线程版（Ch.16）
```

| # | 知识点 | 一句 |
|---|--------|------|
| 1 | `Box` | 独占 |
| 2 | `Rc` | 共享只读 + 强计数 |
| 3 | `Cell`/`RefCell` | 内部可变性 |
| 4 | 循环 `Rc` | 泄漏 |
| 5 | `Weak` | 破环 |

---

## 六、提醒

1. **`Rc` / `Cell` / `RefCell` / `Weak` 均单线程**；多线程 → **`Arc` + `Mutex`/`RwLock`**。
2. **RefCell 规则在运行时** —— 编译不报错，写代码须自觉保证 borrow 不冲突。
