## 核心结论（先背）

1. **`Rc` 不能代替 `Box`  everywhere** —— 单线程、多所有者、默认只读，三者缺一则换方案。
2. **多线程共享** → **`Arc<T>`**（原子计数）。
3. **共享可变** → **`Rc<RefCell<T>>`**（运行时 borrow）。
4. **互相 `Rc` 指向** → 计数永不为 0 → **泄漏** → 用 **`Weak<T>`** 打破。

---

## 一、为何不全用 `Rc`？三大限制

### 1. 单线程专用

`Rc` 的计数增减**非原子** → **不能**跨线程 `Send` 共享。

| 场景 | 用 |
|------|-----|
| 单线程多所有者 | `Rc<T>` |
| 多线程多所有者 | `Arc<T>` → [16 章](../16-fearless-concurrency/) |

### 2. 默认不可变（只读共享）

`Rc<T>` 只提供 **`&T`** 级别的共享访问，不能直接 `*rc = new_val`。

**单线程共享可变**：

```rust
use std::cell::RefCell;
use std::rc::Rc;

let data = Rc::new(RefCell::new(0));
*data.borrow_mut() += 1;
```

→ 完整 walkthrough：[15.5 Rc+Cell/RefCell](./15.5-RefCell与内部可变性.md) · 循环/Weak：[15.6](./15.6-引用循环与Weak.md)

### 3. 循环引用 → 内存泄漏

```text
A --Rc--> B
B --Rc--> A   // strong_count 永 ≥ 1，堆永不释放
```

**修法**：回边、父指针用 **`Weak<T>`** → [15.6](./15.6-引用循环与Weak.md) · demo：`15.6-reference-cycle-demo/`

## 二、`Box` move 失败 vs `Rc` clone 成功（对照）

### `Box`：第二次所有者 = 编译错误

```rust
let a = Box::new(list);
let b = Cons(3, Box::new(a)); // a 被 move
// let c = Cons(4, Rc::clone(&a)); // ❌ a 已失效
```

### `Rc`：多次 clone 同一堆块

```rust
let a = Rc::new(list);
let b = Cons(3, Rc::clone(&a)); // a 仍有效
let c = Cons(4, Rc::clone(&a)); // 计数 +1
```

## 三、与 `Box` / `RefCell` / `Arc` 选型（单线程视角）

| 需求 | 类型 |
|------|------|
| 唯一所有者、递归节点 | `Box<T>` |
| 多所有者、只读 | `Rc<T>` |
| 多所有者、可变（单线程） | `Rc<RefCell<T>>` |
| 多所有者、多线程 | `Arc<T>` |
| 多线程 + 可变 | `Arc<Mutex<T>>` 等 → [16.3](../16-fearless-concurrency/16.3-共享状态并发.md) |

→ [15.5 §一 如何选择](./15.5-RefCell与内部可变性.md#box--rc--refcell-如何选择单线程)

## 四、完整对照表

| 特性 | `Box<T>` | `Rc<T>` | `Arc<T>` |
|------|----------|---------|----------|
| 所有者 | 1 | 多 | 多 |
| 线程 | move 可跨线程 | 单线程 | 多线程 |
| 默认可变 | 是（独占时） | 否 | 否 |
| 计数 | 无 | 强/弱 | 原子强/弱 |
| clone 语义 | move（非 Copy） | 指针 +1 | 指针 +1 |

---

## 五、一句话总结

1. **`Box` 独占，`Rc` 共享** —— 你的直觉完全正确。
2. **`Rc::clone`** 不是深拷贝，是 **+1**。
3. **单线程 + 只读** 用 `Rc`；要可变加 `RefCell`；要跨线程换 `Arc`；要破环用 `Weak`。
