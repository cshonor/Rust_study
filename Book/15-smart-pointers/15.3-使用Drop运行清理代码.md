# 15.3 使用 `Drop` trait 运行清理代码

对应 demo 项目：`Book/15-smart-pointers/15.3-drop-demo/`

`Drop` 允许在值**离开作用域**时执行自定义清理逻辑（释放堆内存、关闭文件/网络连接、释放锁等）。`Drop` 在 prelude 中，通常无需 `use`。

---

## 实现方式

实现 `Drop` trait，提供：

```rust
fn drop(&mut self) { /* ... */ }
```

编译器会在值不再使用时**自动**插入对 `drop` 的调用，无需手动调用。

---

## 丢弃顺序

同一作用域内多个变量按**与创建相反**的顺序丢弃（后创建的先 `drop`）。示例：先创建 `c` 再创建 `d`，结束时先 `drop(d)` 再 `drop(c)`。

---

## 不能显式调用 `Drop::drop`

不能写 `value.drop()`：Rust 会报 **explicit destructor calls not allowed**。原因是作用域结束时仍会再自动 `drop` 一次，会导致**重复释放**（double free）风险。

---

## 提前释放：`std::mem::drop`

若需在作用域结束前**强制**结束值的生命周期，应使用 **`std::mem::drop(value)`**（也在 prelude）。它会获取所有权并立即丢弃该值（从而运行 `Drop`）。

这与 `Drop::drop` 方法不同：`mem::drop` 是标准库函数，通过移动值来触发析构。

---

## 与所有权的关系

借用检查保证“仍在使用”的值不会被提前销毁；`drop` 只会在值不再被需要时运行**一次**。
