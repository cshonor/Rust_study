# 15.1 使用 `Box<T>` 指向堆上的数据

对应 demo 项目：`Book/15-smart-pointers/15.1-box-demo/`

`Box<T>` 是最直接的智能指针：把值放在**堆**上，栈上只保留指向堆数据的指针。

---

## `Box<T>` 的典型用途

1. **编译期未知大小、却要在需要“确切大小”的上下文中使用**（例如递归类型里用间接层打破无限大小）。
2. **大量数据**：转移所有权时只移动小的指针，避免在栈上整块拷贝大结构（堆上数据不随 `Box` 移动而逐字节拷贝）。
3. **只关心实现了某 trait，不关心具体类型**：trait 对象（如 `Box<dyn Trait>`），第 17 章会展开。

除“数据在堆上”外，`Box` 本身没有额外运行时开销；也没有像 `Rc`/`Arc` 那样的引用计数等功能。

---

## 基本用法

```rust
let b = Box::new(5);
println!("b = {}", b); // 自动解引用，像用栈上值一样用
```

`b` 离开作用域时，`Drop` 会释放堆内存。

单独把一个小 `i32` 放进 `Box` 在工程里不常见，栈上默认存放即可；`Box` 的价值更多体现在**间接**与**递归类型**。

---

## 递归类型与“无限大小”

Rust 要在编译期知道类型大小。若枚举变体里**直接**嵌套同类型：

```rust
// 不能编译：recursive type has infinite size
enum List {
    Cons(i32, List),
    Nil,
}
```

`Cons` 需要 `i32 + sizeof(List)`，而 `List` 又依赖 `Cons`，大小无法有限计算。

---

## cons list（教学用）

源于 Lisp 的 **cons**：把“一个头元素”和“剩余列表”接在一起；终止符常写作 **Nil**。在 Rust 里日常列表更常用 `Vec<T>`，这里用 cons list 只是为了演示**递归枚举 + 间接**。

## 用 `Box<List>` 打破递归（间接）

`Box<T>` 的大小固定（指针宽度），因此：

```rust
enum List {
    Cons(i32, Box<List>),
    Nil,
}
```

`Cons` 里是 `i32 + 指针`，不再无限嵌套同类型在字段里的“展开”，编译器可以算出 `List` 的大小。语义上仍是链表，实现上是**堆上的下一结点**。

---

## 智能指针：`Deref` 与 `Drop`

- **`Deref`**：让 `Box<T>` 在多数场景下可以像 `&T` 一样用（解引用 coercion）。
- **`Drop`**：`Box` 被 drop 时释放堆上数据。

后续 `Rc`、`Arc` 等也会围绕这些 trait 展开。
