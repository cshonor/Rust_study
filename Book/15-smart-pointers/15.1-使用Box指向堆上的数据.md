# 15.1 使用 `Box<T>` 指向堆上的数据

## 一、`Box<T>` 本质

**最简单智能指针**：

| 位置 | 存什么 |
|------|--------|
| **栈** | `Box` 实例（一个指针宽） |
| **堆** | `T` 的真实数据 |

> 无引用计数、无额外运行时开销；自动 **`Deref`** + **`Drop`**。

### 三大使用场景

1. **递归类型** — 打破「无限尺寸」；编译期须确定 `sizeof`，同类型直接嵌套算不出大小，`Box` 用固定大小指针做间接层
2. **大数据上堆** — move 只拷贝栈上指针，避免整块大结构栈复制
3. **`Box<dyn Trait>`** — trait 对象，抹平不同实现类型的尺寸差 → [17.2](../17-oop/17.2-为使用不同类型的值而设计的trait对象.md)

## 二、基础语法与内存模型

```rust
let b = Box::new(100);
println!("{}", b); // Deref 自动解引用
```

离开作用域 → **`Drop`** → 自动释放堆内存。

> 小类型（`i32`/`bool`）**不必**无脑 `Box`；栈上更高效。`Box` 优势在**间接存储**与**递归**。

```rust
let box_num = Box::new(99);
assert_eq!(*box_num, 99);
```

---

## 三、递归类型报错 & Cons 链表

### 错误写法（编译失败）

```rust
enum List {
    Cons(i32, List), // List 嵌 List → sizeof 无限递归
    Nil,
}
```

`Cons = i32 + List`，`List` 又含 `Cons` → 编译器算不出固定大小。

### `Box` 修复

```rust
enum List {
    Cons(i32, Box<List>), // 固定指针宽 → 尺寸可确定
    Nil,
}

use List::{Cons, Nil};
let list = Cons(1, Box::new(Cons(2, Box::new(Nil))));
```

内存：每个 `Cons` 栈上 `i32 + 指针`；下一结点数据在**堆**。

> 工程列表优先 **`Vec<T>`**；Cons 链表用于学习递归 + `Box`。

## 四、`Deref` 与 `Drop`（智能指针基石）

| Trait | 作用 |
|-------|------|
| **`Deref`** | `Box<T>` 隐式当 `&T` 用，可直接读写 |
| **`Drop`** | 生命周期结束释放堆内存 |

→ **`Deref`**：解引用 · **`Drop`**：销毁清理 → [15.3 Drop](./15.3-使用Drop运行清理代码.md) · [15.3.1](./15.3.1-Drop顺序与进阶场景.md)

---

## 五、易错点

1. **`Box<T>` 拥有堆数据所有权** — 不是借用 `&T`
2. **move `Box`** — 只移动栈上指针，堆数据不逐字节拷贝
3. 业务列表用 **`Vec`**，Cons 仅教学
4. **`Box::new` 必在堆分配**，无法强制放栈
