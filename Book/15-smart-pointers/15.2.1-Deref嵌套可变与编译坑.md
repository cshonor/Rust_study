## 一、多层嵌套 coercion 链

编译器在**需要 `&U` 的地方**对 `&T` 反复应用 `deref()`，直到类型匹配或失败。

### 典型链：`&MyBox<String>` → `&str`

```rust
use std::ops::Deref;

struct MyBox<T>(T);

impl<T> Deref for MyBox<T> {
    type Target = T;
    fn deref(&self) -> &T { &self.0 }
}

fn takes_str(s: &str) {}

let m = MyBox(String::from("hi"));
takes_str(&m);
// 展开等价思路：
// &MyBox<String>  --MyBox::deref-->  &String
//                 --String::deref-->  &str   ✓ 匹配
```

### 再套一层：`Box<MyBox<String>>`

```rust
let nested = Box::new(MyBox(String::from("deep")));
takes_str(&nested);
// &Box<MyBox<String>> → &MyBox<String> → &String → &str
```

### 手动 vs 自动

```rust
// 自动（日常写法）
takes_str(&nested);

// 手动（理解用，等价）
takes_str(&nested.as_ref().0.as_str());
// 或
takes_str(&(*(*nested).deref()).deref());
```

**底层视角**：每一步仍是「取地址 + 读指针」，coercion **不分配、不拷贝字符串内容**，只调整**引用类型**的编译期视图。

→ demo：`cargo run -- nested`

## 二、`DerefMut` 完整规则

| # | 条件 | 转换 | 说明 |
|---|------|------|------|
| 1 | `T: Deref<Target = U>` | `&T` → `&U` | 只读链 |
| 2 | `T: DerefMut<Target = U>` | `&mut T` → `&mut U` | 可变链，可改内部 |
| 3 | `T: Deref<Target = U>` | `&mut T` → `&U` | 可变借用**降级**为不可变视图 |
| ✗ | 仅 `Deref`、无 `DerefMut` | `&T` ↛ `&mut U` | **不可**从共享引用 coerce 出可变 |

### 示例：`MyBox` + `DerefMut`

```rust
use std::ops::{Deref, DerefMut};

struct MyBox<T>(T);

impl<T> Deref for MyBox<T> {
    type Target = T;
    fn deref(&self) -> &T { &self.0 }
}

impl<T> DerefMut for MyBox<T> {
    fn deref_mut(&mut self) -> &mut T { &mut self.0 }
}

fn main() {
    let mut b = MyBox(vec![1, 2, 3]);
    b.push(4);              // 方法调用：&mut MyBox → &mut Vec（DerefMut）
    (*b)[0] = 99;           // 手动可变解引用
    println!("{:?}", *b);   // [99, 2, 3, 4]
}
```

### `Box<T>` 的特殊点

```rust
let b = Box::new(String::from("own"));
let s: String = *b; // move 出堆上 String；b 此后不可用（除非 T: Copy）
```

`*` 对 `Box` 是**取值/移出**，不是普通「解引用看一眼」；这是 `Box` 作为**唯一所有权容器**的语义，不是 `Deref` trait 本身规定的。

→ demo：`cargo run -- mut`

## 三、方法解析与自动解引用（三层机制）

Rust 方法调用 `x.foo()` 时编译器依次尝试：

1. **自动解引用**：`&x` → `&mut x` → `x`（配合 `Deref`/`DerefMut` 链）
2. **自动引用**：给 `self` 参数自动加 `&` / `&mut`
3. **trait 解析**：找匹配的 `impl`

因此：

```rust
let b = MyBox(String::from("abc"));
b.len();           // MyBox → &String → str::len
b.push_str("!");   // 需要 &mut String；MyBox 无 DerefMut 则编译失败
```

## 四、常见编译报错（踩坑表）

| 报错 / 现象 | 原因 | 修法 |
|-------------|------|------|
| `E0614: cannot be dereferenced` | 类型未实现 `Deref` | 实现 `Deref`，或改用 `.as_ref()` / 显式字段 |
| `cannot borrow as mutable` | 只有 `Deref`，却要 `&mut` coercion | 实现 `DerefMut`，或 `&mut inner` 显式访问 |
| `cannot assign to ... which is behind a &` | 通过 `&T` coercion 来的引用不能写 | 要 `&mut` + `DerefMut` |
| `expected &str, found &MyBox<String>` | **未**发生 coercion（如直接赋值给非引用参数） | 传参/方法上下文才会 coerce；或 `.deref()` / `&**x` |
| 类型推断「一层套一层」 | 滥用 `Deref` 导致隐式转换过深 | 业务 struct 别 `Deref`；用显式 API |
| 校验被绕过 | `Deref` 到整个 `HashMap` 等 | [LeakyDb 反例](../../ER/ER-demos/item-05-06-newtype/src/over_deref.rs) |

### 反例 1：缺 `DerefMut`

```rust
// struct ROBox<T>(T);
// impl Deref for ROBox<T> { ... }
// let mut b = ROBox(vec![1]);
// b.push(2);  // ❌ cannot borrow as mutable
```

### 反例 2：期望可变却只拿到共享 coercion

```rust

// let b = MyBox(vec![1, 2]);
// set_first(&b);  // ❌ expected &mut [i32], found &MyBox<...>
// set_first(&mut b); // ✅ 需 &mut + DerefMut
```

// db.insert(id, bad);  // 绕过 insert() 里的校验

## 五、与 `AsRef` / `Borrow` 选型

| 需求 | 推荐 |
|------|------|
| 智能指针透明当内部类型用 | `Deref` / `DerefMut` |
| API 收多种借用视图 | `AsRef<T>` + 显式调用 |
| `HashMap` 键 `&str` 查 `String` | `Borrow`（标准库容器） |
| 领域类型防 bypass | **不要** `Deref` 到整个内部容器 |

---

## 六、一句话总结

1. **多层 coercion**：编译期链式 `deref`，`&Box<MyBox<String>>` 一路到 `&str`，零运行时成本。
2. **`DerefMut`**：`&mut T` → `&mut U`；仅有 `Deref` 时 **`&mut T` 最多降到 `&U`**，不能升可变。
3. **方法调用** = 自动解引用 + 自动引用 + trait 解析，三者叠加。
4. **编译报错**多半是没 `DerefMut`、在非 coercion 上下文用、或滥用 `Deref`。
5. 业务封装用**显式方法**；透明指针用 **`Deref`**。
