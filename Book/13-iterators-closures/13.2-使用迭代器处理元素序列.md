## 一、迭代器核心概念

迭代器：遍历集合序列的抽象，**帮开发者管理游标与终止条件**。

> **惰性（Lazy）**：只定义遍历规则，**不消费元素、不执行计算** — 直到调用**消费适配器**触发 `next()` 才开始遍历。

### 核心：`Iterator` trait

```rust
pub trait Iterator {
    type Item; // 关联类型：每次产出的元素类型
    fn next(&mut self) -> Option<Self::Item>;
}
```

| `next` 返回 | 含义 |
|-------------|------|
| **`Some(item)`** | 取到元素，游标后移 |
| **`None`** | 遍历结束 |

- 修改内部状态 → 手动调 `next` 时迭代器须 **`mut`**
- **只实现 `next`** → 自动拥有 `map` / `filter` / `zip` / `sum` 等默认方法

```rust
let arr = [1, 2, 3];
let mut iter = arr.iter();
assert_eq!(iter.next(), Some(&1));
assert_eq!(iter.next(), Some(&2));
assert_eq!(iter.next(), Some(&3));
assert_eq!(iter.next(), None);
```

## 二、三种生成方式（以 `Vec` 为例）

| 方法 | 元素类型 | 所有权 | 原集合之后 |
|------|----------|--------|------------|
| **`.iter()`** | `&T` | 不可变借用 | ✅ 仍可用 |
| **`.iter_mut()`** | `&mut T` | 可变借用 | 遍历中不能再用原集合 |
| **`.into_iter()`** | `T` | **move** | ❌ 原 `Vec` 失效 |

```rust
let mut v = vec![10, 20, 30];
let _it1 = v.iter();
let _it2 = v.iter_mut();
let _it3 = v.into_iter();
// v 已被 move，此处不可用
```

> **`for x in v`** 等价 **`for x in v.into_iter()`**（按 `IntoIterator` 消费集合）。

→ 与 [8.3.1 zip/collect](../08-collections/8.3.1-zip-collect与HashMap.md) 中 `iter` vs `into_iter` 同一套所有权逻辑。

## 三、两大类方法

### 1. 消费适配器（Consuming）

**拿走迭代器所有权**，内部反复 `next`，**用完即废**。

常用：`sum` · `collect` · `count` · `max` · `min` · `for` 循环

```rust
let nums = vec![1, 2, 3];
let total: i32 = nums.iter().sum();
```

- **`collect()`**：按**目标类型**收进 `Vec` / `HashSet` / … → 须标注 `Vec<_>` 等

### 2. 迭代器适配器（Iterator Adaptors）

**返回新迭代器**，仍惰性；**必须再跟消费适配器**才求值。

| API | 作用 |
|-----|------|
| **`map(\|x\| …)`** | 逐个映射 |
| **`filter(\|x\| bool)`** | 保留 `true` |
| **`skip(n)`** | 跳过前 n 个 |
| **`take(n)`** | 只取前 n 个 |
| **`zip(other)`** | 两迭代器配对 |

```rust
let src = vec![1, 2, 3, 4, 5];
let res: Vec<_> = src
    .iter()
    .map(|n| n * 2)
    .filter(|&&val| val > 4)
    .collect();
// [6, 8, 10] — 只有 collect 才真正遍历
```

#### `filter` + 闭包捕获环境

→ [13.1 闭包](./13.1-闭包.md)

```rust
let limit = 5;
let nums = vec![2, 4, 6, 8];
let out: Vec<_> = nums.iter().filter(|&&x| x > limit).collect();
// [6, 8]
```

## 四、自定义迭代器：`Counter`（1～5）

```rust
struct Counter { count: u32 }

impl Iterator for Counter {
    type Item = u32;
    fn next(&mut self) -> Option<Self::Item> {
        self.count += 1;
        if self.count <= 5 {
            Some(self.count)
        } else {
            None
        }
    }
}

let total: u32 = Counter::new().map(|x| x * 2).sum();
```

链式示例（demo 测试）：

```rust
Counter::new()
    .zip(Counter::new().skip(1))
    .map(|(a, b)| a * b)
    .filter(|x| x % 3 == 0)
    .sum(); // 18
```

---

## 五、易错点

1. **只写 `map().filter()` 不 `collect`/`sum`** → **完全不执行**
2. **`into_iter()` move 集合**；`iter` / `iter_mut` 只借用
3. 手动 `next` → 迭代器要 **`mut`**；`for` 自动管理
4. **`collect` 须标注目标类型**（`Vec<_>`）
