## 一、概念

**迭代器**：实现了 `Iterator` trait 的类型，**逐个产生序列元素**。

- 是 Rust 处理集合的**零成本抽象**
- 核心：**只定义规则，不立即执行（惰性）**

---

## 二、要点总览（先背这几句）

- **惰性**：`map`/`filter` 只存规则；`collect`/`sum`/`for` 才真正跑
- **`Iterator` trait**：只必须实现 **`next` + 关联类型 `Item`**
- **适配器**：返回**新迭代器**，仍然惰性（`map`/`filter`/`zip`…）
- **消费器**：**拿走迭代器、用完作废**（`collect`/`sum`/`count`/`for`…）
- **`iter` / `iter_mut` / `into_iter`**：对应 **`&T` / `&mut T` / `T`**，所有权不同
- **返回迭代器**：
  - 单一类型：`impl Iterator<Item = T>`
  - 分支异构：`Box<dyn Iterator<Item = T>>`

---

## 三、惰性（Lazy）——最关键

**定义迭代器 ≠ 执行**；**只有消费时才计算**。

```rust
let v = vec![1, 2, 3];
let it = v.iter().map(|x| x * 2);
// ⚠️ 惰性：此时什么都不算，只存「乘 2」规则

// 到这里才真正遍历、计算
let res: Vec<_> = it.collect();
```

- **适配器**（`map`/`filter`/`skip`）：**不干活，只包一层规则**
- **消费器**（`collect`/`sum`/`for`）：**触发 `next()`，真正执行**

→ Filter 伪实现 · 更多例子：[13.2.1 §一–§三](./13.2.1-迭代器精讲.md) · **分步打印 demo**：[13.2.4](./13.2.4-惰性演示-适配器与消费器.md)（`cargo run -- lazy`）

## 四、`Iterator` Trait（极简）

```rust
pub trait Iterator {
    type Item;                // 关联类型：元素类型
    fn next(&mut self) -> Option<Self::Item>;
}
```

- **唯一必须实现：`next()`**
- `next()`：每次调用**推进状态**，返回 `Some(值)` 或 `None`（结束）
- 所有其他方法（`map`/`filter`/`fold`…）都是**默认实现，基于 `next`**

→ `next` 须 `mut` · `Item` 与 iter 三件套：[13.2.1 §二](./13.2.1-迭代器精讲.md)

## 五、适配器 vs 消费器（对照表）

### 1）迭代器适配器（返回新迭代器，惰性）

| 方法 | 作用 |
|------|------|
| `map(f)` | 每个元素应用 `f`，映射新值 |
| `filter(f)` | 保留 `f` 返回 `true` 的元素 |
| `zip(it2)` | 与另一个迭代器配对 |
| `take(n)` | 只取前 `n` 个 |
| `enumerate()` | 产出 `(索引, 值)` |

### 2）消费器（拿走迭代器，立即执行）

| 方法 | 作用 |
|------|------|
| `collect::<C>()` | 收集到集合 `C`（`Vec`/`HashSet`…） |
| `sum()` | 求和 |
| `count()` | 计数 |
| `for x in it` | 循环遍历（本质：反复 `next`） |
| `fold(init, f)` | 自定义累加 |

**一句话：适配器 = 链式加工；消费器 = 收尾出结果。**

→ 详解：[13.2.5 适配器与消费器](./13.2.5-适配器与消费器详解.md) · 分步 demo：[13.2.4](./13.2.4-惰性演示-适配器与消费器.md)

## 六、`iter` / `iter_mut` / `into_iter`（所有权核心）

| 方法 | 元素类型 | 所有权 | 原集合还能用？ |
|------|----------|--------|----------------|
| `iter()` | `&T` | 不可变借用 | ✅ 可以继续用 |
| `iter_mut()` | `&mut T` | 可变借用 | ✅ 可以继续用（但不能同时借用） |
| `into_iter()` | `T` | **拿走所有权** | ❌ 原集合失效 |

```rust
let v = vec![1, 2, 3];

// iter：只读借用
for x in v.iter() {
    println!("{}", x);
}
println!("{:?}", v); // ✅ 还能用

// into_iter：拿走所有权
for x in v.into_iter() {
    println!("{}", x);
}
// println!("{:?}", v); // ❌ 报错，所有权已转移
```

> **`for x in v`** 默认等价 **`for x in v.into_iter()`**（move 所有权）。

→ 签名 · 踩坑 · `IntoIterator`：[13.2.2](./13.2.2-三种迭代生成方式.md) · 结构体类型：[13.2.3](./13.2.3-迭代器结构体与生命周期.md)

## 七、返回迭代器怎么写

### 1）单一类型（推荐：`impl Trait`）

```rust
fn double_it(v: &[i32]) -> impl Iterator<Item = i32> {
    v.iter().map(|x| x * 2)
}
```

- 优点：**静态分发、零成本**
- 适用：返回**确定、单一**迭代器类型

### 2）分支 / 多种迭代器（必须 `Box<dyn …>`）

```rust
fn make_it(flag: bool) -> Box<dyn Iterator<Item = i32>> {
    if flag {
        Box::new((1..=5).into_iter())
    } else {
        Box::new(vec![10, 20, 30].into_iter())
    }
}
```

- 适用：**不同分支返回不同迭代器类型**
- 代价：**堆分配、动态分发**（小开销）

→ [10.2.5 impl vs dyn](../10-generics-traits-lifetimes/10.2.5-impl与dyn-Trait对比.md) · demo `get_iter` / `get_dyn`

## 八、一句话总结（背下来）

- **惰性**：`map`/`filter` 只记规则，**`collect`/`for` 才跑**
- **`Iterator`**：只有 **`next` + `Item`** 是必须的
- **适配器**：返回新迭代器、**不消费**
- **消费器**：**拿走迭代器、算出结果**
- **三迭代方法**：**`iter` = `&`、`iter_mut` = `&mut`、`into_iter` = 拿走**
- **返回迭代器**：单一用 **`impl`**，异构用 **`Box<dyn>`**
