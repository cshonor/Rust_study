## 一、核心定义

闭包是**可存入变量、可作为函数参数的匿名函数**。与普通 **`fn`** 的核心区别：**能捕获定义处外层作用域的变量** → 行为延后执行、惰性求值。

| | **`fn`** | **闭包** |
|--|----------|----------|
| 捕获环境 | ❌ | ✅ |
| 类型 | 函数指针 / 具名 | **匿名类型**，编译器推导 |
| 参数/返回类型 | 通常须标注 | **可选**，上下文推断 |

### 基础语法

→ 四段式演变详解：[13.1.1](./13.1.1-闭包四段式语法.md)

```rust
let add = |x: i32| -> i32 { x + 1 }; // 完整标注
let add = |x| x + 1;                 // 最简：单行可省略 {}
add(5);
```

1. 参数：`|参数列表|``（不是 `()`）
2. 体：单语句可省略 `{}`；多语句必须 `{}`
3. **类型可选** — 不像 `fn` 强制标注

### 类型锁定（易错）

→ 完整讲解 · 速查表：[13.1.3 类型推导易错点](./13.1.3-闭包类型推导易错点.md)

同一闭包**首次调用推导参数类型后永久锁定**；**只定义不调用**且无标注 → `E0282`：

```rust
let cl = |x| x;
cl(5);      // 推导 x: i32
// cl("abc"); // ❌ 编译报错
```

## 二、业务场景：惰性 + 缓存（Cacher）

### 1. 提前存变量 — 浪费

```rust
let res = simulated_expensive_calc(10); // 立刻算，无用分支也白跑
if flag { println!("{}", res); }
```

### 2. 闭包存行为 — 惰性

```rust
let calc = |n| simulated_expensive_calc(n); // 只存逻辑
if flag { calc(10); }                          // 需要才算
```

### 3. Cacher — 泛型 + `Fn` trait

闭包是**匿名类型**，结构体字段不能直接写类型 → **`T: Fn(...)`** 约束。

#### 简易版：只缓存**一次**结果

```rust
struct Cacher<T>
where
    T: Fn(u32) -> u32,
{
    calc: T,
    val: Option<u32>,
}
```

> **缺陷**：只缓存第一次入参；换参数仍返回旧值 → demo `CacherOnce`。

#### 通用版：`HashMap` 多参数缓存

```rust
struct Cacher<T, Arg, Ret>
where
    T: Fn(Arg) -> Ret,
    Arg: Eq + Hash,
{
    calc: T,
    cache: HashMap<Arg, Ret>,
}
```

→ demo `CacherMap`。

## 三、三大捕获 Trait：`Fn` / `FnMut` / `FnOnce`

→ 完整讲解 · 推导规则 · 传参铁律 · 速查表：[13.1.4 Fn / FnMut / FnOnce](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)

由闭包**如何使用捕获变量**自动决定 impl 哪几个；**所有闭包至少实现 `FnOnce`** — 不是三个都自动 impl。

→ 完整讲解：[13.1.4 Fn / FnMut / FnOnce](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)

| Trait | 捕获 | 闭包实际 impl |
|-------|------|---------------|
| **`Fn`** | 只读 `&T` | Fn + FnMut + FnOnce |
| **`FnMut`** | `&mut T` | FnMut + FnOnce（**无 Fn**） |
| **`FnOnce`** | move / 消耗 | 至少 FnOnce；消耗时往往**仅** FnOnce |

**层级**：`Fn ⊂ FnMut ⊂ FnOnce`（继承 · bound 由严到宽）→ [13.1.4 §二](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#二trait-继承约束由严到宽)

## 四、`move` 闭包

→ 详解 · 默认捕获 vs `move` · 线程：[13.1.5 move 闭包](./13.1.5-move闭包详解.md)

```rust
move |args| { /* ... */ }
```

**强制**把捕获变量 **move 进闭包**，绑定闭包自身生命周期。

### 经典：跨线程

```rust
use std::thread;

let s = String::from("hello");
let handle = thread::spawn(move || println!("{s}"));
handle.join().unwrap();
// s 已 move，此处不可用（非 Copy 类型）
```

| 类型 | `move` 后原变量 |
|------|-----------------|
| **`Copy`**（`i32`） | 仍可用（拷贝） |
| **非 `Copy`**（`String`） | 失效 |

## 五、易错点汇总

1. 闭包类型**不能显式书写** → 编译器**自动 impl** `Fn` 系列；API 用 bound 限定 → [13.1.4 §三](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#三编译器自动-impl底层逻辑)
2. **只定义不调用** → `E0282`；**首次调用锁定**类型 → [13.1.3](./13.1.3-闭包类型推导易错点.md)
3. `CacherOnce` 只缓存单参数；多参数要 `HashMap`
4. **`move` = 强制所有权**，不是借用 → [13.1.5](./13.1.5-move闭包详解.md)
5. **`FnMut` 不能降级为 `Fn`** 传参 → [13.1.4](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)
