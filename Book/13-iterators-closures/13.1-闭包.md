## 一、核心定义

闭包是**可存入变量、可作为函数参数的匿名函数**。与普通 **`fn`** 的核心区别：**能捕获定义处外层作用域的变量** → 行为延后执行、惰性求值。

| | **`fn`** | **闭包** |
|--|----------|----------|
| 捕获环境 | ❌ | ✅ |
| 类型 | 函数指针 / 具名 | **匿名类型**，编译器推导 |
| 参数/返回类型 | 通常须标注 | **可选**，上下文推断 |

### 基础语法

→ 四段式演变详解：[13.1.1](./13.1.1-闭包四段式语法.md)

```rust
let add = |x: i32| -> i32 { x + 1 }; // 完整标注
let add = |x| x + 1;                 // 最简：单行可省略 {}
add(5);
```

1. 参数：`|参数列表|``（不是 `()`）
2. 体：单语句可省略 `{}`；多语句必须 `{}`
3. **类型可选** — 不像 `fn` 强制标注

### 类型锁定（易错）

同一闭包**首次调用推导参数类型后永久锁定**：

```rust
let cl = |x| x;
cl(5);      // 推导 x: i32
// cl("abc"); // ❌ 编译报错
```

## 二、业务场景：惰性 + 缓存（Cacher）

### 1. 提前存变量 — 浪费

```rust
let res = simulated_expensive_calc(10); // 立刻算，无用分支也白跑
if flag { println!("{}", res); }
```

### 2. 闭包存行为 — 惰性

```rust
let calc = |n| simulated_expensive_calc(n); // 只存逻辑
if flag { calc(10); }                          // 需要才算
```

### 3. Cacher — 泛型 + `Fn` trait

闭包是**匿名类型**，结构体字段不能直接写类型 → **`T: Fn(...)`** 约束。

#### 简易版：只缓存**一次**结果

```rust
struct Cacher<T>
where
    T: Fn(u32) -> u32,
{
    calc: T,
    val: Option<u32>,
}
```

> **缺陷**：只缓存第一次入参；换参数仍返回旧值 → demo `CacherOnce`。

#### 通用版：`HashMap` 多参数缓存

```rust
struct Cacher<T, Arg, Ret>
where
    T: Fn(Arg) -> Ret,
    Arg: Eq + Hash,
{
    calc: T,
    cache: HashMap<Arg, Ret>,
}
```

→ demo `CacherMap`。

## 三、三大捕获 Trait：`Fn` / `FnMut` / `FnOnce`

由闭包**如何使用捕获变量**自动决定；**所有闭包至少实现 `FnOnce`**。

| Trait | 捕获方式 | 调用次数 | 场景 |
|-------|----------|----------|------|
| **`Fn`** | 不可变借用 `&T` | 多次 | 只读外部变量 |
| **`FnMut`** | 可变借用 `&mut T` | 多次 | 修改捕获变量 |
| **`FnOnce`** | move 拿走 `T` | **一次** | 消耗捕获值 |

### 自动推导

1. 只读 → `Fn` + `FnMut` + `FnOnce`
2. 修改 → `FnMut` + `FnOnce`
3. 消耗所有权 → **仅** `FnOnce`

### 传参约束（易错）

需要 **`FnMut`** 的参数**不能**传只满足 **`Fn`** 的闭包 — trait 不能降级。

## 四、`move` 闭包

```rust
move |args| { /* ... */ }
```

**强制**把捕获变量 **move 进闭包**，绑定闭包自身生命周期。

### 经典：跨线程

```rust
use std::thread;

let s = String::from("hello");
let handle = thread::spawn(move || println!("{s}"));
handle.join().unwrap();
// s 已 move，此处不可用（非 Copy 类型）
```

| 类型 | `move` 后原变量 |
|------|-----------------|
| **`Copy`**（`i32`） | 仍可用（拷贝） |
| **非 `Copy`**（`String`） | 失效 |

---

## 五、易错点汇总

1. 闭包类型**不能显式书写** → 泛型 + `Fn` trait
2. **首次调用锁定**参数类型
3. `CacherOnce` 只缓存单参数；多参数要 `HashMap`
4. **`move` = 强制所有权**，不是借用
5. **`FnMut` 不能降级为 `Fn`** 传参
