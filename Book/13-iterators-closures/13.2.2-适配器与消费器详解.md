## 一、通俗比喻

把**原始集合（`Vec`/数组）** 比作**原材料仓库**，**迭代器**就是一条**加工流水线**：

| 角色 | 比喻 | Rust 里 |
|------|------|---------|
| **适配器** | 流水线上的**加工工位**（打磨、筛选、裁切） | `map` / `filter` / `skip` / `take` … |
| **消费器** | **启动开关 + 成品收纳箱** | `collect` / `sum` / `for` / `count` … |

- **适配器**：只**加装加工规则**，流水线搭完，**不开机、不处理原料**
- **消费器**：按下开关，原料逐个加工，产出成品；流水线走完就**空了**

---

## 二、迭代器适配器（Adapter）

### 定义

对现有迭代器**再包装、追加处理逻辑**，**返回一个新迭代器**。

### 核心特征

1. **惰性执行**：只记录「要做什么」，**不会立刻遍历、计算元素**
2. **链式包装**：每加一个适配器 = 多包一层规则（底层仍是「等 `next` 才动」）
3. **支持链式调用**：`map` → `filter` → `skip` … 组成复杂流程
4. **本质**：修改**产出元素的规则**，不触发遍历

### 常见适配器

| 方法 | 作用 |
|------|------|
| `map(f)` | 对每个元素做转换 |
| `filter(f)` | 按条件过滤 |
| `skip(n)` | 跳过前 `n` 个 |
| `take(n)` | 只取前 `n` 个 |
| `zip(it2)` | 与另一迭代器两两配对 |
| `enumerate()` | 产出 `(索引, 元素)` |

### 代码印证（只搭流水线，不运行）

```rust
let v = vec![1, 2, 3, 4, 5];
let pipeline = v.iter()
    .map(|x| x * 2)
    .filter(|&x| x > 5);
// 👉 仅搭建规则，map/filter 内部逻辑一次都没跑

println!("流水线搭建完成，还没有开始加工！");
```

→ demo：`cargo run -- lazy` 里 §2 之前会先打印「规则组装完成，未执行」

## 三、迭代器消费器（Consumer）

### 定义

**启动迭代器、触发遍历**，逐个调用 `next()`，执行所有适配器规则，最终产出结果。

### 核心特征

1. **触发执行**：让前面所有适配器的逻辑**真正运行**
2. **消耗迭代器**：游标走到末尾，**彻底失效**（不能指望再产出元素）
3. **负责收尾**：把元素转成集合、数字、打印等**最终结果**

### 常见消费器

| 方法 / 写法 | 作用 |
|-------------|------|
| `collect()` | 收集到 `Vec` / `HashSet` 等（最常用） |
| `sum()` | 数值求和 |
| `count()` | 统计剩余元素个数 |
| `fold(init, f)` | 自定义累加 / 聚合 |
| `for x in iter` | 逐个遍历（语法糖，底层反复 `next()`） |
| `while let Some(x) = iter.next()` | **手动**逐个 `next()`，最底层、最灵活的消费 |

### 代码印证（按下开关，流水线开工）

```rust
let pipeline = v.iter()
    .map(|x| {
        println!("执行 map：{} * 2", x);
        x * 2
    })
    .filter(|&x| {
        println!("执行 filter：判断 {} > 5", x);
        x > 5
    });

println!("流水线搭建完成，准备启动...");

let result: Vec<_> = pipeline.collect(); // 消费器：从这里才跑
println!("最终成品：{:?}", result);    // [6, 8, 10]
```

**典型输出顺序**：先「搭建完成」→ 再逐条 map/filter 日志 → 最后成品。

→ 完整分步输出：[13.2.4 §二](./13.2.4-惰性演示-适配器与消费器.md#二链式适配器map--filter)

## 四、`while let Some(x) = iter.next()`（手动消费）

**一句话：手动、逐个地消费迭代器**，直到 `next()` 返回 `None` 为止。它是 Rust 的**模式匹配循环语法糖**，专门用来遍历迭代器或处理 `Option`。

### 1. 先回忆：`next()`

所有迭代器都有核心方法：

```rust
fn next(&mut self) -> Option<Item>
```

- **有元素** → `Some(元素)`
- **没元素了** → `None`

→ trait 定义：[13.2 hub §四](./13.2-使用迭代器处理元素序列.md#四iterator-trait极简)

### 2. 本质 = `loop` + `match` + `break`

```rust
while let Some(x) = iter.next() {
    // 使用 x
}
```

**完全等价于**：

```rust
loop {
    match iter.next() {
        Some(x) => {
            // 使用 x
        }
        None => break,
    }
}
```

`while let` 把「无限循环 + 匹配 + 中断」收成一行。

### 3. 适配器 + `while let`（完整可运行）

```rust
fn main() {
    let v = vec![1, 2, 3, 4, 5];

    // 适配器：只搭流水线，不执行
    let mut iter = v.iter()
        .map(|x| x * 2)
        .filter(|&x| x > 5);

    // 消费器：每次 next() 驱动整条链前进一步
    while let Some(x) = iter.next() {
        println!("拿到元素：{}", x);
    }
}
```

输出：`6`、`8`、`10`（各一行）。

**关系：**

- `map` / `filter` → **适配器**，惰性，返回新迭代器
- `while let … next()` → **消费器**，每次调用 `next()` 才跑流水线
- 与 `for`、`collect()`、`sum()` 同属消费器，只是**更底层、更灵活**

### 4. `while let` vs `for`（效果相同）

```rust
// A：手动
while let Some(x) = iter.next() {
    println!("{}", x);
}

// B：语法糖（自动 next + 遇 None 退出）
for x in iter {
    println!("{}", x);
}
```

- **`for`**：更常用；`for x in v` 还涉及 `IntoIterator` → [13.2.5 §二](./13.2.5-三种迭代生成方式.md#二for-x-in-v--into_iterintointoiterator)
- **`while let`**：可**控制每次何时 `next()`**，中途做别的事、提前 `break`

→ 三种消费器同链逐步对比：[13.2.4 §五](./13.2.4-惰性演示-适配器与消费器.md#五三种消费器同链对比while-let--for--collect)（`cargo run -- lazy` §5）

## 五、两大核心区别（对照表）

| 对比项 | **适配器** | **消费器** |
|--------|------------|------------|
| 核心行为 | 追加规则，返回**新迭代器** | 启动遍历，产出**最终结果** |
| 执行时机 | 惰性，**定义时不运行** | **调用时**执行所有前置规则 |
| 迭代器状态 | 可继续链式拼接 | 被**耗尽**，不能再次产出 |
| 链中位置 | **中间**，可无限拼接 | **最后一步**，收尾 |

## 六、两个关键补充

### 1. 链式执行顺序

**书写顺序 = 元素处理顺序**：

```rust
v.iter().map(...).filter(...).collect()
// 原始元素 → map 转换 → filter 过滤 → collect 收集
```

### 2. 迭代器只能消费一次

消费后游标在末尾，再次 `collect` 得到**空结果**（变量若仍持有已耗尽的迭代器）：

```rust
let mut iter = v.iter().map(|x| x * 2);

let res1: Vec<_> = (&mut iter).collect();
println!("第一次: {:?}", res1); // [2, 4, 6]

let res2: Vec<_> = iter.collect();
println!("第二次: {:?}", res2); // [] — 已空
```

> 第一次若写 `iter.collect()` 会直接 **move** 走迭代器；演示「二次 collect」用 `&mut iter` → [13.2.4 §五](./13.2.4-惰性演示-适配器与消费器.md#五消费后不能复用)

## 七、一句话总结

1. **适配器**：搭加工流水线，只定规则，**不开工**
2. **消费器**：启动流水线，加工原料，**出成品**（`for` / `collect` / **`while let … next()`**）
3. 写代码：**适配器中间串联，消费器最后收尾**

→ Filter 底层怎么包一层规则：[13.2.1 §三 Filter 伪实现](./13.2.1-迭代器精讲.md#三适配器惰性--消费器触发执行)
