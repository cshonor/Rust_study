## 表格 + 方法签名

| 方法 | 产出 `Item` | 借用/所有权 | 调用后原 `Vec` | 签名本质 |
|------|-------------|-------------|----------------|----------|
| **`.iter()`** | `&T` | 不可变借用 `&self` | ✅ 完整可用 | `fn iter(&self) -> Iter<'_, T>` |
| **`.iter_mut()`** | `&mut T` | 可变借用 `&mut self`（独占） | 迭代器存活期间被锁住；销毁后恢复 | `fn iter_mut(&mut self) -> IterMut<'_, T>` |
| **`.into_iter()`** | `T` | **move `self`**，消费容器 | ❌ 所有权移走，变量失效 | `fn into_iter(self) -> IntoIter<T>`（`IntoIterator`） |

---

## 一、逐段实例 + 踩坑

```rust
let mut v = vec![10, 20, 30];

// 1. iter：不可变借用
{
    let val: Vec<&i32> = v.iter().collect();
    dbg!(val); // [&10, &20, &30]
} // val 释放后，才能对 v 再 mut 借用
println!("{:?}", &v); // ✅

// 2. iter_mut：独占可变借用
{
    let mut refs: Vec<&mut i32> = v.iter_mut().collect();
    for x in refs {
        *x *= 2;
    }
} // 迭代器/引用 vec 销毁后 v 恢复
dbg!(&v); // [20, 40, 60]

// 3. into_iter：move
let owned: Vec<i32> = v.into_iter().collect();
dbg!(owned);
// println!("{:?}", &v); // ❌ E0382：v 已 move
```

### 底层原因

| 方法 | 本质 |
|------|------|
| `iter(&self)` | **栈上 Vec 仍在**，只借元素地址 |
| `iter_mut(&mut self)` | **可变独占** — 同一时刻不能另有 `&v` / 第二个 `iter_mut` |
| `into_iter(self)` | **传值 move** — 控制权交给 `IntoIter`，原变量作废 |

> **踩坑**：`v.iter().collect::<Vec<&i32>>()` 得到的引用存活期间，**不能**再 `v.iter_mut()` — 须用 `{}` 缩小 `val` 作用域。

## 二、`for x in v` ⇔ `into_iter()`（`IntoIterator`）

`for A in B` 语法糖 ≈：

```rust
let mut iter = B.into_iter();
while let Some(x) = iter.next() { /* 使用 x */ }
```

→ 完整拆解（等价写法 · 适配器链 · vs `for`）：[13.2.2 §四](./13.2.2-适配器与消费器详解.md#四while-let-somex--iternext手动消费) · 同链三种消费器对比：[13.2.4 §五](./13.2.4-惰性演示-适配器与消费器.md#五三种消费器同链对比while-let--for--collect)

### 标准库三套 `IntoIterator`（决定 `for` 行为）

| 写法 | 实现 | `Item` | 等价 |
|------|------|--------|------|
| `for x in v` | `IntoIterator for Vec<T>` | `T` | `v.into_iter()` — **move，v 报废** |
| `for x in &v` | `IntoIterator for &Vec<T>` | `&T` | `v.iter()` — 只读 |
| `for x in &mut v` | `IntoIterator for &mut Vec<T>` | `&mut T` | `v.iter_mut()` — 原地改 |

```rust
let v = vec![1, 2];
for _ in v {}           // v move
// println!("{v:?}");  // ❌

let v = vec![1, 2];
for x in &v { println!("{x}"); }
println!("{v:?}");     // ✅

let mut v = vec![1, 2];
for x in &mut v { *x += 1; }
// v = [2, 3]
```

**口诀**：`for v` 拿走所有权 · `for &v` 只读 · `for &mut v` 可变改。

## 三、与 zip / collect 的所有权

`collect` 按**目标类型**决定收的是引用还是值：

```rust
let v = vec![1, 2, 3];

let ref_vec: Vec<&i32> = v.iter().collect();   // v 仍可用
let own_vec: Vec<i32> = v.into_iter().collect(); // v 报废

let a = vec![1, 2];
let b = vec![10, 20];
let z: Vec<(&i32, &i32)> = a.iter().zip(b.iter()).collect();
// a、b 仍可用
```

→ 详解：[8.3.1 zip/collect](../08-collections/8.3.1-zip-collect与HashMap.md)

## 四、高频易错总结

1. **只读、保留原数组** → `.iter()` / `for x in &v`
2. **原地改元素、保留原数组** → `.iter_mut()` / `for x in &mut v`
3. **要元素所有权、不再用原数组** → `.into_iter()` / `for x in v`
4. **既要遍历又要保留原 Vec 的所有权** → `for x in &v`，或 `v.clone().into_iter()` 克隆再消耗

## 五、自测（3 道改错题）

1. ```rust
   let mut v = vec![1, 2, 3];
   let r: Vec<&i32> = v.iter().collect();
   for x in v.iter_mut() { *x += 1; }
   ```
2. ```rust
   let v = vec![1, 2];
   for x in v { println!("{x}"); }
   v.push(3);
   ```
3. ```rust
   let v = vec![1, 2, 3];
   let a: Vec<i32> = v.iter().collect();
   ```

<details>
<summary>参考答案</summary>

1. **`r` 持有 `&v` 的引用**，与 `v.iter_mut()` **互斥** — 给 `r` 加 `{}` 块提前 drop，或先改再 collect。  
2. **`for x in v` move 了 v**，后面 `v.push` → **E0382** — 改用 `for x in &v` 或 `for x in v.clone()`。  
3. **`v.iter()` 产出 `&i32`**，不能 `collect` 成 `Vec<i32>` — 改为 `Vec<&i32>`，或 `v.into_iter().collect()` / `.copied()`。

</details>
