## 1. 三个结构体是什么

```rust
// v.iter() 返回
pub struct Iter<'a, T>;
// v.iter_mut() 返回
pub struct IterMut<'a, T>;
// v.into_iter() 返回
pub struct IntoIter<T>;
```

| 类型 | 含义 |
|------|------|
| **`Iter<'_, T>`** | 匿名生命周期 `'_` = 借用 `Vec` 的时长；内部指针指向堆数组，**只借不拥有** |
| **`IterMut<'_, T>`** | 可变借用指针，**独占** |
| **`IntoIter<T>`** | **接管堆数组所有权**，原 `Vec` 作废 |

| 调用 | 返回类型 | `Item` |
|------|----------|--------|
| `v.iter()` | `Iter<'_, T>` | `&T` |
| `v.iter_mut()` | `IterMut<'_, T>` | `&mut T` |
| `v.into_iter()` | `IntoIter<T>` | `T` |

方法签名：

```rust
fn iter(&self) -> Iter<'_, T>;
fn iter_mut(&mut self) -> IterMut<'_, T>;
fn into_iter(self) -> IntoIter<T>; // IntoIterator trait
```

## 2. 编译报错：`iter` + `iter_mut` 同时存活

```rust
let mut v = vec![10, 20, 30];
let _it1 = v.iter();     // 不可变借用 &v
let _it2 = v.iter_mut(); // ❌ 已有不可变借用，不能再可变借用
let _it3 = v.into_iter();
```

**Rust 借用规则**：同一数据上，**不可变引用与可变引用不能重叠**（`_it1` 存活期间锁住了 `v`）。

### 修正：作用域分开，迭代器销毁 = 借用释放

```rust
let mut v = vec![10, 20, 30];
{ let _it1 = v.iter(); }      // 结束 → 释放不可变借用
{ let _it2 = v.iter_mut(); }   // OK
let _it3 = v.into_iter();     // move v
// 此后不能访问 v
```

## 3. `for in` 与 `IntoIterator`（三句话）

```rust
let v = vec![1, 2, 3];

// 1. for x in v → into_iter，move，v 失效
for _ in v {}
// println!("{v:?}"); // ❌

// 2. for x in &v → 等价 iter，只读，v 保留
let v2 = vec![1, 2];
for _ in &v2 {}
println!("{v2:?}"); // ✅

// 3. for x in &mut v → 等价 iter_mut，原地改
let mut v3 = vec![1, 2];
for x in &mut v3 { *x += 1; }
```

> `for B in A` ≈ `A.into_iter()` 再循环 `next()`。  
> 标准库为 **`Vec<T>` / `&Vec<T>` / `&mut Vec<T>`** 各实现一套 `IntoIterator`，故三种 `for` 效果不同。

→ 更多实例：[13.2.5 §二](./13.2.5-三种迭代生成方式.md)

## 4. `collect` 巩固所有权

```rust
let v = vec![1, 2, 3];

let r1: Vec<&i32> = v.iter().collect(); // 借用，v 仍可用
let r2: Vec<i32> = v.into_iter().collect(); // move 元素，v 报废
// println!("{v:?}"); // ❌
```

| 链 | 收进 | 原 `Vec` |
|----|------|----------|
| `v.iter().collect()` | `Vec<&T>` | ✅ |
| `v.into_iter().collect()` | `Vec<T>` | ❌ |

## 5. `'_` 匿名生命周期

`Iter<'_, T>` = `Iter<'a, T>` 简写，`'_` 由编译器推导：

- 迭代器**不能活得比被借用的 `Vec` 更久**
- `Vec` 销毁 → 基于它的 `Iter` / `IterMut` **立刻失效**

```rust
let r;
{
    let v = vec![1, 2];
    let it = v.iter();
    r = it.next(); // ❌ it 借 v，v 在本块结束销毁 — 不能把引用带出作用域
}
```

（正确做法：在 `v` 存活期间用完迭代器，或 `collect` 成 owned 数据。）

→ 系统学习：[10.3.7 匿名生命周期 `'_`](../10-generics-traits-lifetimes/10.3.7-匿名生命周期与下划线.md) · [10.3 索引](../10-generics-traits-lifetimes/10.3-生命周期与引用有效性.md)

## 6. 实战：什么时候用 `Iter` vs `IntoIter`

| 场景 | 选用 | 原因 |
|------|------|------|
| 只读扫描、统计、打印 | **`iter()`** / `for x in &v` | 保留 `Vec`，可多次遍历 |
| 原地改元素 | **`iter_mut()`** / `for x in &mut v` | 不分配新 `Vec` |
| 过滤/transform 成新集合且**不再需要原 Vec** | **`into_iter()`** | 避免 clone；元素 move 进新容器 |
| 函数参数 `fn f(v: Vec<T>)` 内部消费 | **`into_iter()`** | 已有所有权，直接消耗 |
| 还要用原 `Vec` 做别的事 | **`iter()`** | 或 `v.clone().into_iter()` 克隆再消耗 |
| 与另一集合 `zip` 只读配对 | **双方 `iter()`** | 两个源都保留 |

**示例（书版 `shoes_in_my_size`）**：

```rust
pub fn shoes_in_my_size(shoes: Vec<Shoe>, size: u32) -> Vec<Shoe> {
    shoes.into_iter().filter(|s| s.size == size).collect()
    // 调用方交出 shoes 所有权，函数内 filter 后 collect 新 Vec — 典型 IntoIter
}
```

只读查大小、原列表还要展示 → 用 `shoes.iter().filter(...)`。
