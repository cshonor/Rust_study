# 13.1.3 `move` 闭包：强制捕获所有权

> **`move` 管捕获规则（所有权进闭包）；`Fn` 系列管怎么用捕获 — 二者互不绑定** → [13.1.4 §二](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#二捕获规则-vs-fn-trait分开看)

## 一、默认捕获（不写 `move` = 借用）

闭包**默认不写 `move` = 借用捕获**（不拿所有权）：

| 闭包内用法 | 捕获方式 | 典型 trait |
|------------|----------|------------|
| 只读 | `&T` 不可变借用 | `Fn` |
| 修改 | `&mut T` 可变借用 | `FnMut` |

> 「消耗捕获 → FnOnce」看**调用时是否销毁/转移**，不是默认借用路径 — 详见 [13.1.4](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)。

```rust
let s = String::from("hello");
let f = || println!("{}", s);
// ✅ 只读 → 捕获 &s，s 还能继续用
f();
println!("{}", s); // 正常
```

---

## 二、`move` 关键字：强制所有权进闭包

→ **move 不直接等于 FnOnce** — 只读 move 仍可能是 `Fn`：[13.1.4 §二.2A](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#2-加-move所有权进闭包--仍不直接等于-fnonce)

```rust
let s = String::from("hello");
let f = move || println!("{}", s);
// ✅ 强制把 s 的所有权 move 进闭包
f();
// println!("{}", s); // ❌ E0382：所有权已转移
```

| 类型 | `move` 后原变量 |
|------|-----------------|
| **`Copy`**（`i32`） | 仍可用（拷贝进闭包，原值还在） |
| **非 `Copy`**（`String`） | **失效**，不能再读 |

> **`move` = 强制捕获所有权**，不管闭包体内看起来像在「借用」。

---

## 三、什么时候必须用 `move`？

| 场景 | 原因 |
|------|------|
| **跨线程** `thread::spawn` | 线程可能比栈上变量活得更久，必须拥有数据 |
| **返回闭包** | 闭包要带走的变量不能借外层栈 |
| **`FnOnce` / 消耗捕获** | 如 `move \|\| v` 把 `Vec` 整个拿走 |

### 经典：线程必须 `move`

```rust
use std::thread;

fn main() {
    let v = vec![1, 2, 3];

    // ❌ 不 move：闭包只借 &v，主线程结束 v 可能已销毁 — 编译器通常直接报错
    // let h = thread::spawn(|| println!("{:?}", v));

    // ✅ move：v 所有权进线程，生命周期独立
    let h = thread::spawn(move || println!("{:?}", v));
    h.join().unwrap();
}
```

## 四、`move` 与 Fn trait

- `move || { ... }` **常见**为 **`FnOnce`**（所有权被拿走，语义上可能只跑一次）
- 若捕获的是 **`Copy`** 且闭包内只读，仍可能同时满足 **`Fn`**

```rust
let v = vec![1, 2, 3];
let consume = move || v; // FnOnce：返回 v，消耗捕获
assert_eq!(consume(), vec![1, 2, 3]);
// consume(); // ❌ 第二次调用可能报错（FnOnce）
```

→ Fn 系列详解：[13.1.4 Fn / FnMut / FnOnce](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)

---

## 五、小结（直接背）

| 要点 | 一句话 |
|------|--------|
| 类型推导 | 不标注时**至少调用一次**，否则 E0282 |
| `move` | **强制所有权**进闭包，不是「尽量借用」 |
| 跨线程 | 几乎总是 `thread::spawn(move \|\| ...)` |
| vs 默认捕获 | 只读用默认；要独立生命周期 → `move` |
