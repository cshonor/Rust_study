## 四段式对照（`add` 为例）

```rust
fn add(x: i32) -> i32 { x + 1 }           // 普通 fn 对照

let add = |x: i32| -> i32 { x + 1 };      // 1. 完整：参数 + 返回 + {}
let add = |x: i32| { x + 1 };             // 2. 省略 ->，保留参数类型
let add = |x: i32| x + 1;                  // 3. 单表达式，省略 {}
let add = |x| x + 1;                      // 4. 最简：全靠推断

add(5); // 6
```

---

## 一、省略规则（核心考点）

### 1. 参数类型可省

`|x|` 不写 `x: i32` — **第一次调用实参确定类型后永久固定**：

```rust
let add = |x| x + 1;
add(5i32);   // x 锁定 i32
// add(5u64); // ❌ 编译报错
```

### 2. 返回值 `-> T` 可省

| 形式 | 规则 |
|------|------|
| **单表达式** `\|x\| x + 1` | 不用写 `->`，返回自动推导 |
| **多语句 `{}`** | 须显式 `-> T`，否则编译报错 |

```rust
let add = |x: i32| -> i32 {
    let tmp = x + 2;
    tmp - 1
};
```

### 3. 单表达式可省 `{}`

- **一条表达式**：`|x| x + 1`
- **多条语句 / 带分号语句**：必须 `{}`

---

## 二、闭包 vs `fn`

| | **`fn`** | **闭包** |
|--|----------|----------|
| 类型标注 | 签名须全标注 | 可推断 |
| 捕获环境 | ❌ | ✅ |

```rust
let base = 10;
let add = |x| base + x; // 捕获外层 base，fn 做不到
println!("{}", add(5)); // 15
```

## 三、易错点

→ 完整讲解 · 速查表：[13.1.2 类型推导易错点](./13.1.2-闭包类型推导易错点.md)

**只定义不调用** — 编译器无法推导类型：

```rust
// let add = |x| x + 1; // ❌ 未调用，类型未知
let add = |x| x + 1;
add(5i32); // ✅ 至少调用一次（或写全类型标注）
```

或写全标注：`let add = |x: i32| -> i32 { x + 1 };`

## 五、参数 vs 捕获（两套独立能力）

> **无参闭包照样能捕获外部变量** — **有没有形参**与**抓不抓外部变量**是两回事，互不干扰。

### 分清两个概念

| | **闭包形参 `\|n\|`** | **外部捕获** |
|--|----------------------|--------------|
| 写在哪 | `\|\|` **里面** | `\|\| { … }` **体内**直接用外层已有变量 |
| 谁提供 | 调用时**外部传入** | 从**当前作用域抓进来** |
| 与无参关系 | 无参 = 形参列表为空 | **与有没有形参无关**，无参也能捕获 |

---

### 无参闭包 + 外部捕获（三种常见）

#### ① 不可变捕获（只读 → `Fn`）

```rust
let num = 100;
let cb = || println!("{}", num); // 无参，捕获 num

cb();
cb(); // 可多次调用
```

#### ② 可变捕获（修改外部 → `FnMut`）

```rust
let mut num = 0;
let mut cb = || num += 1; // 无参，直接改外面的 num

cb();
println!("{}", num); // 1
```

#### ③ `move` 捕获（所有权进闭包，无参也生效）

```rust
let s = String::from("hello");
let cb = move || println!("{}", s); // 无参 + move

cb();
// println!("{}", s); // ❌ 外层 s 已 move（非 Copy）
```

→ 捕获规则 · Fn trait：[13.1.3](./13.1.3-move闭包详解.md) · [13.1.4](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md)

---

### 有参闭包：参数 + 捕获可同时存在

有参数只是**额外接收调用方传入的值**，照样捕获外部变量：

```rust
let base = 10;
let cb = |n: i32| base + n; // 形参 n + 捕获 base

println!("{}", cb(5)); // 15
```

### 只有参数、无捕获

```rust
let add = |n: i32| n + 1;
println!("{}", add(2)); // 3
println!("{}", add(5)); // 6
```

### 只有捕获、无参数

```rust
let base = 10;
let add = || base + 1;
println!("{}", add()); // 11
```

---

### 一句话

- **有没有参数，不影响抓取外部变量** — 无参闭包就是 `|| { 直接用外面的变量 }`，编译器按借用 / `move` 规则自动处理捕获。
- **有参数**：动态入参、逻辑更复用；**无参数**：依赖固定环境。

> 闭包**形参** ≠ **捕获** — Fn trait 只看捕获怎么用 → [13.1.4 §三.4](./13.1.4-Fn-FnMut-FnOnce三大捕获trait.md#4-其他误区与-capture-无关)
