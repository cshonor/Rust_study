# 14.3 Cargo 工作空间（Workspace）

> **一句话：Workspace = 一个大文件夹统一管好多个独立 Package**；HFT、大型量化项目标配。

回顾：单个 Package 最多 **1 个 lib**、若干 bin；**Workspace 可装 N 个 Package**。

## 一、三层级（再也不混）

| 层级 | 是什么 | 例子 |
|------|--------|------|
| **Workspace** | 最外层大容器；**无 `src/`**，只收纳多个 Package | `hft_workspace/` |
| **Package** | 子项目；各自 `Cargo.toml` + `src/` | `engine/`、`gateway/` |
| **Crate** | Package 内部编译单元 | `engine/src/lib.rs`（lib）· `gateway/src/main.rs`（bin） |

```
Workspace（hft_workspace/）
├ Cargo.toml          ← [workspace] members = [...]
├ Cargo.lock          ← 全工作区唯一一份
├ engine/             ← Package1
│  ├ Cargo.toml
│  └ src/lib.rs       ← lib crate
├ gateway/            ← Package3
│  ├ Cargo.toml
│  └ src/main.rs      ← bin crate
└ tools/              ← Package4
   ├ Cargo.toml
   └ src/bin/replay.rs
```

## 二、HFT 目录样板

```
hft_workspace/        ← 工作区根（无 src！）
 ├ Cargo.toml         ← 工作区总配置
 ├ Cargo.lock
 ├ engine/            ← Package1：交易引擎（lib）
 ├ market_data/       ← Package2：行情（lib）
 ├ gateway/           ← Package3：券商网关（bin exe）
 └ tools/             ← Package4：辅助 CLI（bin/*.rs）
```

### 顶层 `Cargo.toml`

```toml
[workspace]
members = [
    "engine",
    "market_data",
    "gateway",
    "tools",
]
resolver = "2"
```

写上 `members`，Cargo 认定这些文件夹都是受管控的子 Package。

---

## 三、核心三大好处（HFT 向）

### 1. 子 Package 之间轻松互相依赖

`gateway/Cargo.toml`：

```toml
[dependencies]
engine = { path = "../engine" }
market_data = { path = "../market_data" }
```

`gateway/src/main.rs`：

```rust
use engine::on_tick;
use market_data::decode;

fn main() {
    let tick = decode(b"BID 100");
    on_tick(tick);
}
```

### 2. 统一版本、共用一份 `Cargo.lock`

- 全工作区**只生成一个 `Cargo.lock`**
- 各成员依赖同一外部 crate 时解析为**同一版本**（HFT 对依赖一致性要求高）
- 各成员仍须在**自己的** `Cargo.toml` 声明依赖后才能 `use`

### 3. 批量编译 / 测试 / 指定运行

在**工作区根目录**：

| 命令 | 作用 |
|------|------|
| `cargo build` | 编译全部 member |
| `cargo run -p gateway` | 只运行 `gateway` 这个 Package 的默认 bin |
| `cargo test` | 跑全工作区测试 |
| `cargo test -p engine` | 只测 `engine` |

产物在 **`workspace-root/target/`**（成员目录下通常不再各自一份 `target/`）。

## 四、和 `src/bin/` 的区别（重点）

| | `src/bin/`（单 Package） | Workspace（多 Package） |
|---|--------------------------|-------------------------|
| 边界 | 同一项目内多个 exe，**共用一套 lib** | 引擎 / 行情 / 网关 **独立 Package** |
| 部署 | 多工具同仓库 | **进程可分开部署**（低延迟隔离） |
| 适用 | 迁移脚本、报表等小工具 | **HFT 多组件拆分** |

→ 详见 [7.1.4](../07-packages-modules/7.1.4-src-bin与多exe.md)

## 五、The Book 原版要点（demo `14.3-workspace-demo`）

```toml
[workspace]
members = ["adder", "add-one", "add-two"]
```

- `adder` 的 `main` 通过 `path` 依赖 `add-one`、`add-two`
- crate 名 `add-one` → 代码里 `add_one::…`（连字符变下划线）

```bash
cd Book/14-cargo-crates/14.3-workspace-demo
cargo build
cargo run -p adder
cargo test -p add-one
```

---

## 六、发布

向 crates.io 发布时，**每个 Package 单独** `cargo publish`；没有一次发布整个 workspace 的 `--all`。

---

## 七、怎么选（口诀）

| 场景 | 结构 |
|------|------|
| 小项目、单体 web | 单个 Package + `src/main.rs` |
| 同项目多 CLI 小工具 | 单 Package + `src/bin/*.rs` |
| 中大型、HFT、多组件拆分 | **Workspace + 多个子 Package** |
