## 一、核心规则：`cargo test` 两套参数，`--` 做分隔

```bash
cargo test [cargo 自身参数] -- [传给测试程序的参数]
# -- 左边：cargo 管理编译、筛选用例
# -- 右边：测试运行时配置（线程、打印输出、ignored）
```

---

## 二、控制测试并发（多线程 / 串行）

### 规则

`cargo test` **默认多线程并行**跑所有用例。

缺点：多个测试操作**同一文件、全局环境变量**时，会互相篡改数据、随机报错。

### 命令：指定单线程串行

```bash
# --test-threads 属于测试二进制参数，放 -- 右侧
cargo test -- --test-threads=1
```

- `=1`：只用 1 个线程，从上到下串行跑
- 也可写 `--test-threads=4` 指定 4 线程

### demo 代码（文件竞争）

```rust
#[cfg(test)]
mod tests {
    #[test]
    fn test_a() {
        std::fs::write("tmp.txt", "A").unwrap();
    }
    #[test]
    fn test_b() {
        let s = std::fs::read_to_string("tmp.txt").unwrap();
        println!("{}", s);
    }
}
```

> **并行**：a、b 随机先后，文件内容错乱；**串行** `--test-threads=1` 顺序执行无异常。

demo 里 `test_file_a` / `test_file_b` 默认 `#[ignore]`，避免 `cargo test` 随机失败；串行演示：

```bash
cargo test file -- --test-threads=1 --include-ignored
```

## 三、控制控制台打印：`--show-output`

### 默认行为

- **测试通过**：自动吞噬 `println!`，控制台看不见
- **测试失败**：自动打出所有 `println!`

### 通过的测试也要打印

```bash
cargo test -- --show-output
```

```rust
#[test]
fn show_print() {
    println!("我是测试内部打印内容");
    assert_eq!(1, 1);
}
```

不加 `--show-output`：无打印；加参数：控制台输出字符串。

## 四、按函数名筛选（只跑部分测试）

`cargo test 关键字`：**cargo 侧参数，写在 `--` 左边**，匹配测试函数名（**模糊匹配，包含即执行**）。

### ① 精准匹配单个函数

```bash
cargo test one_hundred
```

### ② 模糊匹配名字含 `add` 的所有测试

```bash
cargo test add
# 匹配 add_two、add_two_and_two 等
```

> 测试全路径形如 `tests::add_two_and_two`，搜索匹配**全名子串**。

```rust
#[test]
fn add_two() { assert_eq!(2 + 2, 4); }
#[test]
fn add_two_and_two() { assert_eq!(2 + 2, 4); }
#[test]
fn one_hundred() { assert_eq!(100, 100); }
```

- `cargo test add` → 前两个
- `cargo test one_hundred` → 只第三个

## 五、`#[ignore]` 忽略耗时测试

```rust
#[test]
#[ignore] // 默认跳过
fn expensive_test() {
    std::thread::sleep(std::time::Duration::from_secs(2));
    assert!(true);
}
```

### 三条常用命令

| 场景 | 命令 |
|------|------|
| 默认（跳过 `#[ignore]`） | `cargo test` |
| **只**跑被 ignore 的 | `cargo test -- --ignored` |
| 普通 + ignore **全跑** | `cargo test -- --include-ignored` |

---

## 六、速查命令清单

| 需求 | 命令 |
|------|------|
| 单线程串行 | `cargo test -- --test-threads=1` |
| 通过用例也打印 println | `cargo test -- --show-output` |
| 只跑名字带 add 的测试 | `cargo test add` |
| 只跑被 `#[ignore]` 的测试 | `cargo test -- --ignored` |
| 全部测试（普通 + 忽略） | `cargo test -- --include-ignored` |

## demo 实操（复制即跑）

在 `11.2-control-tests-demo/` 目录下：

```bash
# 默认：5 通过 + 3 ignored（含 2 个文件竞争用例）
cargo test

# 串行文件竞争演示
cargo test file -- --test-threads=1 --include-ignored

# 看 println
cargo test show_print -- --show-output

# 名字过滤
cargo test add
cargo test one_hundred

# ignore 相关
cargo test -- --ignored          # 只跑 expensive_test（约 2s）
cargo test -- --include-ignored  # 全部 4 个 + ignore
```

| 测试函数 | 演示点 |
|----------|--------|
| `test_file_a` / `test_file_b` | §二 并行 vs 串行 · 共享 `tmp.txt`（默认 ignore） |
| `show_print` | §三 `--show-output` |
| `add_two` / `add_two_and_two` / `one_hundred` | §四 名字过滤 |
| `expensive_test` | §五 `#[ignore]` |
