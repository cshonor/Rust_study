## 一、测试三段式（AAA：Arrange · Act · Assert）

所有测试函数固定三步：

| 步骤 | 英文 | 做什么 |
|------|------|--------|
| 1 | **Arrange** | 构造数据、初始化对象 |
| 2 | **Act** | 调用待测业务代码 |
| 3 | **Assert** | 校验输出是否符合预期 |

```rust
#[test]
fn larger_can_hold_smaller() {
    // Arrange
    let larger = Rectangle { width: 8, height: 7 };
    let smaller = Rectangle { width: 5, height: 1 };
    // Act + Assert（简单场景可合并）
    assert!(larger.can_hold(&smaller));
}
```

## 二、测试模块与注解：`#[cfg(test)]` / `#[test]`

### 基础模板

```rust
#[cfg(test)] // 条件编译：仅 cargo test 时编译，cargo build 正式打包忽略
mod tests {
    use super::*; // 引入当前文件上层所有代码

    #[test] // 标记为测试用例；独立线程运行；panic → 失败
    fn it_works() {
        assert_eq!(2 + 2, 4);
    }
}
```

### 注解说明

| 注解 | 作用 |
|------|------|
| **`#[cfg(test)]`** | 单元测试标配；减小正式产物体积、缩短编译 |
| **`#[test]`** | 测试器自动执行；**每个用例独立线程**，互不干扰 |
| **panic** | 测试函数 panic → 该用例**失败**；无 panic → **通过** |

→ 拆碎讲 `cfg` vs `test`、模拟输入：[11.1.1](./11.1.1-cfg-test与模拟输入.md)

## 三、三大断言宏

### 1. `assert!(布尔表达式)`

表达式必须为 `true`，否则 panic 失败。

```rust
assert!(big.can_hold(&small));
assert!(!small.can_hold(&big));
```

### 2. `assert_eq!(预期, 实际)`

左右**相等**；失败时自动打印两边值。

> 约束：类型须实现 **`PartialEq + Debug`**；自定义 struct 加 `#[derive(PartialEq, Debug)]`。

```rust
assert_eq!(add_two(2), 4);
```

### 3. `assert_ne!(A, B)`

左右**不相等**（与 `assert_eq!` 相反）。

```rust
assert_ne!(add_two(2), 5);
```

### 4. 自定义报错（`format!` 占位）

三个断言宏都支持追加文案，`{}` 填充变量：

```rust
assert!(
    res.contains("Carol"),
    "名字匹配失败，实际字符串：{}",
    res
);
```

## 四、预期 panic：`#[should_panic]`

校验**入参非法时必须 panic** 才算通过。

### 基础：只要 panic 就过

```rust
#[test]
#[should_panic]
fn test_overflow() {
    Guess::new(200); // 限制 ≤100，传 200 必 panic
}
```

### 精准匹配（推荐）：`expected = "关键字"`

限定 panic 信息须包含指定字符串，避免**非预期 panic 误通过**：

```rust
#[test]
#[should_panic(expected = "less than or equal to 100")]
fn test_overflow() {
    Guess::new(200);
}
```

## 五、返回 `Result` 的测试

测试函数签名可为 `-> Result<(), E>`：`Err` = 失败，`Ok(())` = 通过。

优势：测试内部可直接用 **`?`** 传播错误。

> ⚠️ **不能与 `#[should_panic]` 共用**，二选一。

```rust
#[test]
fn test_with_result() -> Result<(), String> {
    if add_two(2) == 4 {
        Ok(())
    } else {
        Err("计算错误".to_string())
    }
}
```

## 六、常用 `cargo test` 指令

```bash
cargo test                          # 全部用例
cargo test it_adds_two              # 只跑函数名匹配的用例
cargo test -- --test-threads=1      # 单线程串行（排查并发问题）
cargo test -- --nocapture           # 显示 println! 输出
```

→ 更多过滤、忽略、并行控制见 [11.2](./11.2-控制测试如何运行.md)。
