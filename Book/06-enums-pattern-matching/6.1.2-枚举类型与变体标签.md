# 6.1.2 枚举类型 vs 变体标签

← 主笔记：[6.1 定义枚举](./6.1-定义枚举.md) · 四种变体 → [Message 详解](./6.1-定义枚举.md#message-四种变体完整详解4-种形态全覆盖) · `match` → [6.2](./6.2-match.md)

对应 demo：`6.1-enums-demo/` · 预期编译失败：`compile_fail/variant_not_a_type.rs`

---

## 一句话

> **变量的类型永远是外层枚举名（如 `IpAddr`）；`V4`/`V6` 只是变体标签，不能用来标注变量类型。**

```rust
let ip: IpAddr = IpAddr::V4(192, 168, 1, 1); // ✅

// let ip: V4 = IpAddr::V4(192, 168, 1, 1);  // ❌ E0412：V4 不是类型
```

---

## 一、三点拆分（对标 `Option`）

### 1. 变量类型永远 = 枚举本名

```rust
enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String),
    Unknown,
}

let a: IpAddr = IpAddr::V4(127, 0, 0, 1);
let b: IpAddr = IpAddr::V6("::1".into());
let c: IpAddr = IpAddr::Unknown;
// 左边统一是 IpAddr，不管右边哪个变体
```

#### 对标 `Option<T>`

```rust
let x: Option<i32> = Some(5);  // 类型 Option<i32>，不是 Some
let y: Option<i32> = None;       // 类型 Option<i32>，不是 None
```

> **`V4`/`V6` ≈ `Some`；`Unknown` ≈ `None`** — 全是**变体构造器/标签，不是类型**。

### 2. `IpAddr::V4(数值)` 是什么？

**`IpAddr::V4` = 构造器**：用 V4 标签 + 括号数据，生成一个 **`IpAddr` 实例**。

| | 角色 |
|---|------|
| `IpAddr` | 大类型（整体箱子） |
| `V4` | 分类标签 + 携带括号内数据 |

类比：`Option::Some(5)` → 产出 `Option<i32>` 的值。

### 3. 不能把 `V4` 单独当类型

Rust **没有**「把枚举变体拆成独立类型」的语法。要取 V4 里的数据，只能 **`match` 解构**：

```rust
match a {
    IpAddr::V4(n1, n2, n3, n4) => println!("{}", n1),
    IpAddr::V6(s) => println!("{}", s),
    IpAddr::Unknown => {}
}
```

---

## 二、为什么 `Some` 可省略 `Option::`，`V4` 必须写 `IpAddr::`？

| | 原因 |
|---|------|
| **`Some` / `None`** | 标准库 **prelude 预导入** `use Option::{Some, None}`，全局可用 |
| **`V4` / `V6`** | 自定义 enum，**无自动导入**，须全称 `IpAddr::V4` |

手动导入后可省略前缀：

```rust
use IpAddr::{V4, V6, Unknown};

let ip: IpAddr = V4(192, 168, 1, 1); // 等价 IpAddr::V4(...)
```

---

## 三、总结

| # | 规则 |
|---|------|
| 1 | **标注变量类型**：永远写外层枚举（`Option<i32>` / `IpAddr`） |
| 2 | **`Some`、`None`、`V4`、`V6`**：变体标签，**不能**当类型写在冒号后 |
| 3 | **`Some(5)` / `V4(1,2,3,4)`**：构造实例；产物类型是外层 enum |
