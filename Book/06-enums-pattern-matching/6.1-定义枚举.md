# 6.1 定义枚举

对应 demo 项目：`Book/06-enums-pattern-matching/6.1-enums-demo/`

枚举（`enum`）适合表示“一个值只能是多种可能之一”的场景，例如 IP 地址：要么 IPv4 要么 IPv6。

---

## 枚举与成员（variants）

```rust
enum IpAddrKind {
    V4,
    V6,
}
```

创建枚举值：

```rust
let four = IpAddrKind::V4;
let six = IpAddrKind::V6;
```

成员在枚举的命名空间中，用 `::` 访问。

---

## 用枚举 + 结构体存数据（先把“类型”和“数据”分开）

```rust
enum IpAddrKind { V4, V6 }

struct IpAddr {
    kind: IpAddrKind,
    address: String,
}
```

---

## 直接把数据放到枚举成员里（更简洁）

```rust
enum IpAddr {
    V4(String),
    V6(String),
}
```

更进一步：不同成员可存不同类型/数量的数据：

```rust
enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String),
}
```

---

## 标准库的 IpAddr

标准库也提供 `std::net::IpAddr`，其成员内嵌不同结构体（概念相同，只是实现更完整）：

```rust
enum IpAddr {
    V4(Ipv4Addr),
    V6(Ipv6Addr),
}
```

---

## 枚举成员可包含多种形态的数据

```rust
enum Message {
    Quit,
    Move { x: i32, y: i32 },   // 匿名结构体
    Write(String),             // 单值
    ChangeColor(i32, i32, i32) // 多值
}
```

枚举也可以用 `impl` 定义方法：

```rust
impl Message {
    fn call(&self) {
        // ...
    }
}
```

---

## Option：用类型系统表达“有值/无值”

Rust 没有 null，用 `Option<T>` 表示：

```rust
enum Option<T> {
    Some(T),
    None,
}
```

`Option` 与 `T` 是不同类型，编译器会强制你处理 `None` 情况，避免“把空值当成有效值使用”的 bug。
