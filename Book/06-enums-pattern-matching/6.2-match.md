# 6.2 match 控制流运算符

对应 demo 项目：`Book/06-enums-pattern-matching/6.2-match-demo/`

`match` 会把一个值与一系列**模式（pattern）**比较，执行第一个匹配成功的分支。Rust 会在编译期检查 **match 必须穷尽（exhaustive）**：所有可能情况都要处理。

---

## 基本结构

```rust
match value {
    pattern1 => expr1,
    pattern2 => expr2,
    // ...
}
```

- `match` 后面是一个表达式（不要求是 bool，和 `if` 不同）
- 每个分支用 `=>` 连接「模式」与「要执行的表达式」
- 分支之间用逗号分隔
- 分支代码多行时可用 `{ ... }`

---

## 枚举 + match：硬币示例

```rust
enum Coin { Penny, Nickel, Dime, Quarter }

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}
```

### 分支里执行多行代码

```rust
Coin::Penny => {
    println!("Lucky penny!");
    1
}
```

---

## 绑定值的模式（从成员里取数据）

```rust
enum Coin {
    Quarter(UsState),
    // ...
}

match coin {
    Coin::Quarter(state) => {
        println!("State quarter from {:?}!", state);
        25
    }
    // ...
}
```

`state` 会绑定到 `Quarter(...)` 内部的值。

---

## 匹配 Option<T>

```rust
fn plus_one(x: Option<i32>) -> Option<i32> {
    match x {
        None => None,
        Some(i) => Some(i + 1),
    }
}
```

---

## match 必须穷尽

```rust
match x {
    Some(i) => Some(i + 1),
    // ❌ 少了 None 分支会报错：non-exhaustive patterns
}
```

---

## 通配模式与 `_`

### 使用变量接住“其他情况”

```rust
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    other => move_player(other),
}
```

### 不需要值时用 `_`

```rust
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    _ => reroll(),
}
```

也可以什么都不做：`_ => ()`
