## 一、核心区分：`str` 和 `String`

### 1. `str`（字符串切片）

#### 本质

`str` 是**不定长类型**（DST），Rust 里**没有独立栈实例**，**永远只能以借用形式出现**（几乎都是 `&str`）。

```rust
// let s: str = "abc"; // ❌ 编译报错：大小未知，不能单独放变量里
let s: &str = "abc";   // ✅ 借用：栈上 ptr + len，指向那段 UTF-8 数据
```

语义：代表「一串 **UTF-8 字符序列**」，是**底层原始字符串数据**——没有 `ptr/len/cap` 这种管理头，**不负责分配/释放内存**。

#### 内存位置与特性

| 数据来源 | 数据在哪 | 能否改 |
|----------|----------|--------|
| 字面量 `"hello"` | **程序只读静态段**（常为 `&'static str`） | ❌ 全程只读 |
| `&some_string` | 指向 **`String` 的堆** | 通过 `&str` 只读；见下文 `&mut str` |

```text
&str 在栈上:
  ptr ──►  [ UTF-8 字节 ... ]   ← 只读段 或 String 的堆
  len        （字节数，不是“几个字”）
```

#### 日常写法

```rust
let s: &str = "hello world"; // 字面量 ≈ &'static str，轻量、不分配堆
```

---

### 2. `String`（可变字符串结构体）

#### 本质

`String` 是标准库里的 **`struct`**：**拥有所有权**的堆字符串容器，负责分配、扩容、释放。

```rust
// 简化理解（真实源码更复杂）
struct String {
    ptr: *mut u8,  // 指向堆上 UTF-8 数据
    len: usize,    // 已用字节数
    cap: usize,    // 堆容量
}
```

#### 内存位置与特性

| 在哪 | 存什么 |
|------|--------|
| **栈** | `ptr`、`len`、`cap` |
| **堆** | 真实 UTF-8 字节 |

- **可修改、可扩容、可追加**（`push`、`push_str` 等）
- 变量出 `{}` → **drop** → 自动释放堆内存（RAII）

#### 日常创建与使用

```rust
let mut s = String::from("hello");
s.push_str(" world");
println!("{}", s); // hello world
```

---

### 对照一眼（好记）

| | `&str`（`str` 的借用） | `String` |
|--|------------------------|----------|
| 角色 | **视图 / 借用** | **容器 / 拥有** |
| 数据 | 只读段 或 别人的堆 | **自己的堆** |
| 改内容 | 默认 ❌ | ✅ |
| 扩容 | ❌ | ✅ |
| 变量写法 | 只能 `&str` | `let s: String` |

**类比（和 `Vec` 对齐记忆）**

- `str` ≈ 连续字节内容（纯数据，不定长）
- `String` ≈ `Vec<u8>` 那一套（带长度、容量、可动态改）

## 二、常见疑问

### 问题 1：`str` 才是“字符串本身”，`String` 是包装？

✅ **可以这么记。**

- **`str`**：纯字符数据（字符串**本体**），不定长、通常**无所有权**、以只读借用为主；
- **`String`**：**管理堆上那段数据的容器**，管分配、扩容、释放，并提供读写 API。

关系：`String` 在堆上存数据；`&str` 是「指向某段 `str` 数据」的胖指针。

---

### 问题 2：`str` 是不是只读、不能改？

**分形态看：**

| 形态 | 能否改 | 能否扩容 |
|------|--------|----------|
| `&str`（字面量等） | ❌ 只读 | ❌ |
| `&mut str`（从 `String` 借出） | ✅ **原地改已有字节** | ❌ 不能加长 |
| `String` | ✅ | ✅ |

**1）字面量 `&'static str`（最常见）**

存在只读段，**不能改**。

**2）从 `String` 借的 `&mut str`（少见但重要）**

容量由底层 `String` 的 `cap` 固定，只能改**已有长度内**的字节，不能 `push` 加长：

```rust
let mut s = String::from("abc");
let s_mut: &mut str = &mut s;
s_mut.make_ascii_uppercase(); // 原地改，不扩容
println!("{}", s_mut); // ABC
```

**总结**

- 日常说的「字符串切片」= **`&str`，只读**；
- 要**追加、拼接、变长** → 必须用 **`String`**（或先 `String` 再借 `&mut str` 做原地小改）。

---

### 问题 3：口语里「字符串变量」指谁？

| 场景 | 用什么 |
|------|--------|
| 固定文案、字面量、函数**只读**参数 | **`&str`**（轻、不拷堆） |
| 要拼接、修改、`format!`、跨作用域**拥有** | **`String`** |

口语「定义一个字符串变量」在业务代码里**多半指 `String`**；`str` / `&str` 更多出现在**参数、临时借用**。

函数签名习惯：`fn foo(s: &str)` —— 字面量和 `&String` 都能传（自动解引用）。

## 三、互相转换

### `String` → `&str`（借用，最高频）

```rust
let s = String::from("rust");
let s_ref: &str = &s; // 不拷贝，只借视图
```

### `&str` → `String`（拷到堆，拿所有权）

```rust
let s_ref: &str = "rust";
let s = s_ref.to_string();
// 或
let s = String::from(s_ref);
```

| 写法 | 说明 |
|------|------|
| `&s` | 借，零拷贝 |
| `String::from` / `to_string` | 新分配堆 + 拷贝 |
| `format!(...)` | 动态拼接，得到 `String` |

赋值 `String` 会 **move**；需要保留旧值要先 `clone`。

---

## 四、一句话背诵版

1. **`str`**：字符串**原始数据**，不定长，变量里只能是 **`&str`**；字面量在只读段，默认**不可改**。
2. **`String`**：管堆上字符串的**结构体**，**拥有所有权**，可改、可扩容。
3. **固定文本**用 `&str`，**动态操作**用 `String`。
4. **`&str`** = 借来用；**`String`** = 自己拥有这块数据。

**比喻**：`String` = 自己的储物箱（堆）；`&str` = 查看通行证（只看不拿箱）。

## 五、UTF-8 实操

`String` / `&str` 内容都是 **UTF-8**；长度、切片按**字节**，不能当「第几个汉字」直接 `[i]`。

### 新建与更新

```rust
let mut s = String::new();
let s = String::from("initial contents");

s.push_str("bar");
s.push('l');

let s1 = String::from("Hello, ");
let s2 = String::from("world!");
let s3 = s1 + &s2; // s1 被 move
```

### 为什么不能用 `s[i]`？

```rust
// let h = s[0]; // ❌
```

内部类似 `Vec<u8>`；「第 i 个」是字节还是 `char` 语义不清，且不能保证 O(1) 随机访问。

### 字符串 slice（字节 range）

```rust
let hello = "Здравствуйте";
let s = &hello[0..4]; // "Зд"（边界须在 UTF-8 字符上，否则 panic）
```

### 遍历

```rust
for c in "नमस्ते".chars() { /* Unicode 标量 */ }
for b in "नमस्ते".bytes() { /* 原始字节 */ }
```
