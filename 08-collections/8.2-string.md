# 8.2 使用字符串存储 UTF-8 编码的文本

对应 demo 项目：`08-collections/8.2-string-demo/`

Rust 的字符串之所以容易“卡住”，主要因为：**UTF-8**、字符串的内部表示更复杂、以及 Rust 倾向于在编译期暴露潜在错误。

---

## 什么是字符串？

Rust 核心语言里只有一种字符串类型：`str`（通常以借用形式出现：`&str`）。

标准库提供 `String`：
- 可增长（growable）
- 可变（mutable）
- 有所有权（owned）
- UTF-8 编码

常见术语“字符串”通常同时指 `String` 与 `&str`。

另外还有 `OsString/OsStr/CString/CStr` 等，用于不同场景（本节不展开）。

---

## 新建 String

### 空字符串

```rust
let mut s = String::new();
```

### 从字面量创建

```rust
let s = "initial contents".to_string();
let s = String::from("initial contents");
```

`String` 是 UTF-8，可存任何合法 UTF-8 文本（多语言/emoji 等）。

---

## 更新 String

### push_str / push

```rust
let mut s = String::from("foo");
s.push_str("bar"); // 追加 &str，不获取所有权

let mut s = String::from("lo");
s.push('l'); // 追加单个 char
```

### `+` 拼接 vs `format!`

```rust
let s1 = String::from("Hello, ");
let s2 = String::from("world!");
let s3 = s1 + &s2; // s1 被 move；s2 仍可用（&String -> &str 会发生 deref coercion）
```

多段拼接推荐：

```rust
let s = format!("{}-{}-{}", s1, s2, s3); // 不会拿走所有权
```

---

## 为什么 String 不能用索引？

```rust
let s = String::from("hello");
let h = s[0]; // ❌ 编译错误
```

原因：
- `String` 内部是 `Vec<u8>`（UTF-8 字节序列）
- “第 i 个”含义不明确：字节？Unicode 标量值（char）？字形簇（人眼的字母）？
- 对字符串按“字符索引”无法保证 \(O(1)\)（需要从头扫描确认边界）

---

## 字符串 slice（用 range 明确字节范围）

```rust
let hello = "Здравствуйте";
let s = &hello[0..4]; // "Зд"（每个字母 2 字节）
```

注意：range 必须落在 UTF-8 字符边界上，否则运行时 panic。

---

## 遍历字符串

### 按 char（Unicode 标量值）

```rust
for c in "नमस्ते".chars() {
    println!("{c}");
}
```

### 按字节

```rust
for b in "नमस्ते".bytes() {
    println!("{b}");
}
```

字形簇（grapheme cluster）标准库不提供，需第三方 crate。

---

## 小结

- `String`/`&str` 都是 UTF-8
- 不能用 `s[i]` 索引字符串
- 用 `&s[a..b]`、`.chars()`、`.bytes()` 按需选择视角
