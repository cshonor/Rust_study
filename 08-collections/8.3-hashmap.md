# 8.3 哈希 map 储存键值对（HashMap<K, V>）

对应 demo 项目：`08-collections/8.3-hashmap-demo/`

`HashMap<K, V>` 用键 `K` 映射到值 `V`，常用于“用键查找数据”的场景（如队伍分数、词频统计）。

> 需要 `use std::collections::HashMap;`，它不在 prelude 中。

---

## 新建与插入

```rust
use std::collections::HashMap;

let mut scores = HashMap::new();
scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Yellow"), 50);
```

- 键和值都存到堆上
- 所有键类型相同、所有值类型相同（同质）

---

## zip + collect 创建 HashMap

```rust
use std::collections::HashMap;

let teams = vec![String::from("Blue"), String::from("Yellow")];
let initial_scores = vec![10, 50];

let scores: HashMap<_, _> = teams.iter().zip(initial_scores.iter()).collect();
```

`collect()` 目标类型不明确，所以需要 `HashMap<_, _>` 指定。

---

## 所有权：插入会 move

```rust
let field_name = String::from("Favorite color");
let field_value = String::from("Blue");

let mut map = HashMap::new();
map.insert(field_name, field_value);
// field_name / field_value 此后不可用（已 move）
```

Copy 类型（如 i32）会拷贝进 map；`String` 会 move 进 map。

---

## 访问：get 返回 Option<&V>

```rust
let team_name = String::from("Blue");
let score = scores.get(&team_name); // Option<&i32>
```

---

## 遍历

```rust
for (key, value) in &scores {
    println!("{key}: {value}");
}
```

打印顺序不保证（哈希表无序）。

---

## 更新策略

### 覆盖

```rust
scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Blue"), 25); // 覆盖为 25
```

### 只在不存在时插入：entry / or_insert

```rust
scores.entry(String::from("Yellow")).or_insert(50);
scores.entry(String::from("Blue")).or_insert(50); // Blue 已存在，不改变
```

`or_insert` 返回 `&mut V`，不存在就插入并返回新值引用。

### 基于旧值更新：词频统计

```rust
for word in text.split_whitespace() {
    let count = map.entry(word).or_insert(0);
    *count += 1;
}
```

## 哈希函数（Hasher）

标准库默认 hasher 偏安全（抗 DoS），不一定最快。若性能瓶颈明显，可用实现 `BuildHasher` 的类型替换（crates.io 有现成实现）。
