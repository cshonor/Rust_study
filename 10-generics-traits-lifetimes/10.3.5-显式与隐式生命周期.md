## 一、核心对照

不写 `'a` 时编译器**隐式划分区间**，校验逻辑相同；手写 `'a` = **手动命名分组**。

| | 隐式 | 显式 |
|---|------|------|
| 适用 | 单引用入参+返回、简单借用 | 多引用入参、结构体持引用 |
| 例子 | `fn f(x: &i32) -> &i32` | `fn longest<'a>(...) -> &'a str` |

```rust
fn get_data(data: &i32) -> &i32 { data }
// 等价于
fn get_data_explicit<'a>(data: &'a i32) -> &'a i32 { data }
```

```rust
fn same(a: &i32, b: &i32) {}           // 各自独立
fn same_group<'a>(a: &'a i32, b: &'a i32) {} // 同组，取最短
```

## 二、何时手写

| 情况 | 原因 |
|------|------|
| 结构体 / 枚举含引用 | **强制** |
| 多个引用入参需分组 | 编译器无法自动绑定返回值 |
| 复杂返回 / 特殊约束 | 需明确 |

---

## 三、省略三条规则

1. 每个引用参数有输入生命周期；
2. **仅一个**输入生命周期 → 赋给**所有**输出；
3. 方法有 `&self` → 输出默认跟 `self`。

> 多个引用入参 + 返回引用 → **必须手写**（如 [longest](./10.3.4-longest与get_first.md#二longest)）。
