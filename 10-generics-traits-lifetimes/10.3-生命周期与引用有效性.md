# 10.3 生命周期与引用有效性

对应 demo 项目：`10-generics-traits-lifetimes/10.3-lifetimes-demo/`

生命周期（lifetime）描述**引用在多长的作用域内有效**。多数时候编译器可推断；当多个引用之间的关系不明确时，需要用**泛型生命周期参数**把它们的关系写进函数/结构体签名，让借用检查器能证明“引用一定有效”。

---

## 生命周期避免悬垂引用

下面代码会产生悬垂引用，因此不能编译（示例保留为概念）：  

```rust
// let r;
// {
//     let x = 5;
//     r = &x;
// }
// println!("{}", r); // x 已离开作用域
```

借用检查器会拒绝：被引用者必须活得比引用更久。

---

## 函数中的泛型生命周期：longest

没有标注时，返回值借用自哪个参数不明确：

```rust
// fn longest(x: &str, y: &str) -> &str { ... } // ❌ missing lifetime specifier
```

用同一个生命周期 `'a` 关联输入与输出：

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

含义：返回引用的生命周期 = `x` 与 `y` 中**较短**的那个（重叠部分）。

---

## 返回引用必须来自参数

如果返回引用指向函数内部创建的数据，会变成悬垂引用（不能靠标注修复）：

```rust
// fn bad<'a>() -> &'a str {
//     let s = String::from("hello");
//     s.as_str() // ❌ s 离开作用域
// }
```

应返回拥有所有权的值（如 `String`），或引用来自参数。

---

## 结构体中的引用需要生命周期

```rust
struct ImportantExcerpt<'a> {
    part: &'a str,
}
```

含义：`ImportantExcerpt` 的实例不能比 `part` 指向的数据活得更久。

---

## 生命周期省略规则（Lifetime Elision）

编译器用三条规则在常见场景省略标注：

1. 每个引用参数都有自己的输入生命周期
2. 只有一个输入生命周期时，赋给所有输出生命周期
3. 方法有 `&self` / `&mut self` 时，输出生命周期默认取 self 的生命周期

因此 `fn first_word(s: &str) -> &str` 可省略标注。

---

## 'static 生命周期

`'static` 表示引用可活到整个程序结束。字符串字面量都是 `'static`：

```rust
let s: &'static str = "I have a static lifetime.";
```

不要用 `'static` 逃避错误；通常应修复真实的生命周期不匹配/悬垂引用问题。

---

## 结合泛型 + trait bound + 生命周期

```rust
use std::fmt::Display;

fn longest_with_an_announcement<'a, T>(x: &'a str, y: &'a str, ann: T) -> &'a str
where
    T: Display,
{
    println!("Announcement! {}", ann);
    if x.len() > y.len() { x } else { y }
}
```
