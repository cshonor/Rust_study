# 10.3 生命周期与引用有效性

对应 demo 项目：`10-generics-traits-lifetimes/10.3-lifetimes-demo/`  
所有权背景（栈/堆、字面量、无 drop）：[4.1-什么是所有权.md](../04-ownership/4.1-什么是所有权.md)

---

## 核心结论（先记这句）

生命周期标记（如 `'a`、`'static`）**不会创建新引用**，而是给引用**绑定「指向的数据能活多久」的约束**——**不是**描述引用变量本身活多久。

> 入参标 `'a`、返回值也标 `'a`  
> → 返回的引用**一定来自**带同一 `'a` 的入参（或与之同寿命的数据），**不能比入参活得更久**。

---

## 一、两个概念：变量生命周期 vs 生命周期标注

### 1. 普通变量的「生命周期」（客观时长）

从**声明**到**离开作用域被销毁**，这段实际存活时间就是它的真实生命周期——运行时客观行为。

```rust
fn main() {
    let num = 10;
    let r = &num;
} // num、r 一同销毁
```

### 2. 生命周期标注 `'a`（编译器的约束标签）

`'a` **不是**描述变量本身活多久，而是**给引用加的一套规则约定**：

> 所有标了同一个 `'a` 的引用，它们**指向的数据**，存活时间必须**互相兼容**（取最短上限）。

它是编译期的**语法约束**，用来排查悬垂引用，**不改变**变量本身的销毁规则。

---

## 二、生命周期标注 vs 作用域：两套独立规则

| 规则 | 管什么 |
|------|--------|
| 生命周期标签（`'a`、`'static`） | **被指向的数据**能存活多久 |
| 代码块 `{}` | **引用变量本身**什么时候销毁 |

```rust
fn main() {
    let s: &'static str = "test";
} // 变量 s（引用本身）消失；"test" 仍在只读段
```

两者独立：`s` 可以消失，但 `'static` 数据仍在；`r` 和 `num` 可以同生共死，由 `'a` 保证引用不越界。

---

## 三、类型拆解：`&'static str`

| 部分 | 含义 |
|------|------|
| `&` | **引用**——借用，不持有所有权 |
| `'static` | **生命周期标注**——指向的数据与程序同生共死 |
| `str` | 被引用的原生字符串类型 |

> 不存在单独的 `'static str`，一定是 **`&'static str`**。

```rust
let s: &'static str = "abc";
// s 是栈上局部引用，指向只读数据段的 "abc"
// 'static 修饰的是 "abc" 这份数据，不是 s 本身
```

**合法返回示例**：

```rust
fn get_str() -> &'static str {
    let s = "hello";
    s // 数据在只读段，局部 s 消失后数据仍在
}
```

**非法返回对比**：

```rust
fn bad() -> &str {
    let st = String::from("hi");
    &st // ❌ st 出函数 drop
}
```

## 四、`'a` 到底约束什么

```rust
fn borrow<'a>(data: &'a i32) -> &'a i32 {
    data
}

fn main() {
    let num = 10;
    let r = borrow(&num);
} // r、num 一同销毁——引用不能比数据活得更久
```

**函数签名解读**：

| 部分 | 含义 |
|------|------|
| `data: &'a i32` | 入参引用指向的数据打上标签 `'a` |
| `-> &'a i32` | 返回引用指向的数据也必须是 `'a` 那一份 |

### 违反约束 → 编译报错

```rust
fn bad<'a>() -> &'a i32 {
    let tmp = 20;
    &tmp // ❌ tmp 出函数销毁，无入参提供 'a 数据源
}
```

### 三条约束规则（好记）

1. `'a` 是**时长分组标签**，给「引用能合法使用的最长时间」分组——**不是变量绑定标签**；
2. 同标签 `'a` 的引用，数据源**存活时长须互相兼容**（可以是不同变量，不要求是同一个）；
3. 引用**可用时长 ≤ 原始数据的生命周期**（整组 `'a` 取**最短**数据源为上限）。

### 核心疑问速答

- **`'a` 是什么？** **时长分组**——同组引用的**最晚可用时间**必须一致（取最短数据源为准）。
- **引用变量会被销毁吗？** **会。** `'a` 只管：在使用引用期间，**指向的数据必须还活着**。

---

## 五、同 `'a` 约束什么：最晚可用时间

**核心**：`'a` 是**时长分组标签**，不是**变量绑定标签**。  
同 `'a` ≠ 必须指向同一个变量——强制的是：**组内所有引用的「最晚可用时间」必须是同一个节点**。

### 什么是「最晚可用时间」

对一个**引用**来说：

> **最晚可用时间** = 它指向的数据**被销毁的那一刻**。

数据一旦销毁，引用就变成悬垂指针，Rust 禁止。  
生命周期 `'a` 的作用：**所有标了 `'a` 的引用，最晚可用时间必须相同**；编译器以**最短**的那个为准。

**为什么 `'a` 写在引用上，却要管数据？**

1. `'a` 标在引用类型上（`&'a i32`），约束的是引用；
2. 但引用能不能用，**完全依赖指向的数据**；
3. 所以约束引用 = 保证：**引用活着的整个期间，数据一定还活着**。

### 一句话定规则

同一组 `'a` 的所有引用：

> **它们指向的数据，最晚销毁时间必须一致**——不能有的先消失、有的后消失。

| 情况 | 数据销毁 | 共用 `'a` |
|------|----------|-----------|
| 合规 | `a`、`b` 在 `main` 末尾**同时**销毁 | ✅ |
| 违规 | `inner` 内层先销毁，`outer` 外层后销毁 | ❌ |

```rust
fn test<'a>(x: &'a i32, y: &'a i32) {}

fn main() {
    let a = 1;
    let b = 2;
    test(&a, &b); // ✅ 同时销毁
}

fn main_bad() {
    let outer = 1;
    {
        let inner = 2;
        // test(&outer, &inner); // ❌ inner 先销毁
    }
}
```

### 易混淆：约束的是数据，不是引用变量

不是要求**引用变量本身**同时销毁，而是：

👉 **引用所指向的数据源**，必须**同一时刻**销毁（最晚销毁时间一致）。

- 引用变量是栈上普通变量，出 `{}` 该怎么销毁就怎么销毁，`'a` **不管它**；
- `'a` 只管：在引用还能用的区间里，**数据源不能提前消失**。

一长一短的数据源 → 用 **`'a` / `'b` 不同标签**，各自按自己的销毁时间约束（见场景 3）。

### 场景 1：同代码块 — 最晚可用时间相同（可共用 `'a`）

```rust
fn cmp<'a>(x: &'a i32, y: &'a i32) -> &'a i32 {
    if *x > *y { x } else { y }
}

fn main() {
    let v1 = 10;
    let v2 = 20;
    cmp(&v1, &v2);
} // ← v1、v2 在这里一起销毁
```

| 引用 | 指向 | 最晚可用时间 |
|------|------|--------------|
| `&v1` | `v1` | `main` 块末尾 |
| `&v2` | `v2` | `main` 块末尾 |

二者**完全一致** → 可打同一个 `'a`，编译器放行。  
`v1` 先定义、`v2` 后定义，**不改变「一起销毁」**。

#### 声明先后 ≠ 生命周期长短

Rust 看**销毁时机**，不看定义顺序。同一块内变量**统一在块末尾回收**；栈上销毁顺序 LIFO（`v2` → `v1`）只是回收瞬间，**不影响** `'a`——二者最晚可用节点都是块末尾。

### 场景 2：嵌套块 — 最晚可用时间不同（不能共用 `'a`）

```rust
fn print_two<'a>(x: &'a i32, y: &'a i32) {}

fn main() {
    let outer = 100;
    {
        let inner = 200;
        // print_two(&outer, &inner); // ❌
    } // ← inner 在这里销毁
} // outer 在这里才销毁
```

| 引用 | 指向 | 最晚可用时间 |
|------|------|--------------|
| `&outer` | `outer` | **外层** `}` |
| `&inner` | `inner` | **内层** `}` |

👉 **两个引用的最晚可用时间不一样！**  
函数签名却强制归入同一组 `'a` → 承诺与事实冲突 → 编译失败。

### 场景 3：不同标签 — 各自独立的最晚可用时间

```rust
fn two_group<'a, 'b>(x: &'a i32, y: &'b i32) {}

fn main() {
    let outer = 100;
    {
        let inner = 200;
        two_group(&outer, &inner); // ✅
    }
}
```

- `'a` → `&outer` 的最晚可用时间
- `'b` → `&inner` 的最晚可用时间  
两组独立，互不干涉。

### 也合法：同 `'a`，指向同一变量（只是特例）

```rust
fn take<'a>(a: &'a i32, b: &'a i32) {}

fn main() {
    let num = 99;
    take(&num, &num); // ✅ 合法，但不是 'a 的硬性要求
}
```

### 三个独立标签 `'a` / `'b` / `'c`

```rust
fn triple<'a, 'b, 'c>(x: &'a i32, y: &'b i32, z: &'c i32) {
    println!("{} {} {}", x, y, z);
}
```

### 返回值标签与入参对应

**只返回其中一个**——谁作返回值，复用谁的标签：

```rust
fn pick_one<'a, 'b>(x: &'a i32, y: &'b i32) -> &'a i32 { x }
```

**返回多个引用**——元组每个位置分别对应：

```rust
fn return_two<'a, 'b>(x: &'a i32, y: &'b i32) -> (&'a i32, &'b i32) {
    (x, y)
}
```

**入参共用 `'a`，返回任意一个**：

```rust
fn pick_any<'a>(x: &'a i32, y: &'a i32) -> &'a i32 {
    if *x > *y { x } else { y }
}
```

### 三条铁律 + 误区纠正

| | |
|---|---|
| 共用同一 `'a` | 所有引用**最晚可用时间须一致**；变量可同可不同 |
| 最晚可用时间不一还共用 `'a` | **编译报错** |
| 想互不约束 | 用不同标签 `'a`/`'b`/`'c` |

❌ 同 `'a` → 必须同一变量  
✅ 同 `'a` → 归为**同一时长组**，变量可以相同也可以不同

**五条总结**：

1. **最晚可用时间** = 引用指向的数据**被销毁的那一刻**（= 最晚销毁时间）；
2. 同写 `'a` = 组内数据源**最晚销毁时间必须一致**；
3. 同一大代码块：同时销毁 → 可共用 `'a`；
4. 嵌套块：一先一后销毁 → 不能共用 `'a`（用 `'a`/`'b` 分开）；
5. `'a` 约束**数据源**存活匹配，不管引用变量本身何时销毁。

## 六、为什么需要生命周期？

Rust 要杜绝**悬垂引用**。编译器必须知道引用**能活多久**；用 `'a`、`'b` 等当「时间标签」。

```rust
// let r;
// {
//     let x = 5;
//     r = &x;
// }
// println!("{}", r); // ❌ x 已离开作用域
```

被引用者必须活得比引用更久。

## 七、最简示例：`get_first` 与 `longest`

### 无生命周期标注（编译失败）

```rust
// fn get_first(s: &str) -> &str {
//     &s[0..1]   // ❌ missing lifetime specifier
// }
```

### 加上 `'a`（正确）

```rust
fn get_first<'a>(s: &'a str) -> &'a str {
    &s[0..1]
}
```

1. 所有 `'a` **共享同一存活约束**
2. `s: &'a str`：入参引用存活时间为 `'a`
3. 返回 `&'a str`：返回值**来自 `s` 内部**，存活 ≤ `s` 的存活

### `longest`：两个入参同一 `'a`

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

- 两者须**能重叠到同一 `'a`**（实际有效长度取**较短**那个）
- 返回 `'a`：结果**要么是 `x` 要么是 `y`**

## 八、结构体中的引用

```rust
struct ImportantExcerpt<'a> {
    part: &'a str,
}
```

`ImportantExcerpt` 的实例不能比 `part` 指向的数据活得更久。

## 九、生命周期省略规则（Lifetime Elision）

1. 每个引用参数都有自己的输入生命周期
2. 只有一个输入生命周期时，赋给所有输出生命周期
3. 方法有 `&self` / `&mut self` 时，输出生命周期默认取 self 的生命周期

因此 `fn first_word(s: &str) -> &str` 可省略标注。

---

## 十、`'static` 生命周期

`'static` 表示引用指向的数据可活到整个程序结束。字符串字面量都是 `'static`：

```rust
let s: &'static str = "I have a static lifetime.";
```

- 字面量本体在**只读数据段**，程序全程存在
- 局部引用变量 `s` 出 `{}` 仍消失，但指向的数据不受影响
- 不要用 `'static` 逃避错误；通常应修复真实的生命周期不匹配

## 十一、结合泛型 + trait bound + 生命周期

```rust
use std::fmt::Display;

fn longest_with_an_announcement<'a, T>(x: &'a str, y: &'a str, ann: T) -> &'a str
where
    T: Display,
{
    println!("Announcement! {}", ann);
    if x.len() > y.len() { x } else { y }
}
```
