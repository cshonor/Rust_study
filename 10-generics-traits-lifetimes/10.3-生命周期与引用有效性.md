# 10.3 生命周期与引用有效性

对应 demo 项目：`10-generics-traits-lifetimes/10.3-lifetimes-demo/`

---

## 核心结论（先记这句）

生命周期标记（如 `'a`）**不会创建新引用**，而是给引用**绑定「能活多久」的约束**。

> 入参标 `'a`、返回值也标 `'a`  
> → 返回的引用**一定来自**带同一 `'a` 的入参（或与之同寿命的数据），**不能比入参活得更久**。

人话：**返回的引用跟着传入的那份数据同生共死**——不是「原封不动返回同一个变量」，而是**存活时间被绑在一起**。

---

## 为什么需要生命周期？

Rust 要杜绝**悬垂引用**（引用指向已释放内存）。编译器必须知道引用**能活多久**；引用本身没有名字，所以用 `'a`、`'b` 等**生命周期参数**当「时间标签」。

---

下面代码会产生悬垂引用，因此不能编译（示例保留为概念）：  

```rust
// let r;
// {
//     let x = 5;
//     r = &x;
// }
// println!("{}", r); // x 已离开作用域
```

借用检查器会拒绝：被引用者必须活得比引用更久。

---

## 最简示例：`get_first` 与 `longest`

### 无生命周期标注（编译失败）

```rust
// fn get_first(s: &str) -> &str {
//     &s[0..1]   // ❌ missing lifetime specifier
// }
```

编译器不知道：返回的引用依附于谁、能活多久。

### 加上 `'a`（正确）

```rust
fn get_first<'a>(s: &'a str) -> &'a str {
    &s[0..1]   // 演示：仅当首字节是合法 UTF-8 边界时安全；中文等请用 chars()
}
```

含义：

1. 所有 `'a` **共享同一存活约束**
2. `s: &'a str`：入参引用存活时间为 `'a`
3. 返回 `&'a str`：返回值**来自 `s` 内部**，存活 ≤ `s` 的存活

### `longer` / `longest`：两个入参同一 `'a`

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

- `a`、`b` 都标 `'a`：两者生命周期须**能重叠到同一 `'a`**（实际有效长度取**较短**那个）
- 返回 `'a`：结果**要么是 `x` 要么是 `y`**，不会凭空造引用

---

## 反例：没有入参就造不出 `&'a`

```rust
// fn bad<'a>() -> &'a str {
//     let s = String::from("hello");
//     &s   // ❌ s 函数结束即销毁
// }
```

无带 `'a` 的入参时，无法合法返回 `&'a T`。侧面说明：**带生命周期的返回引用必须来自同标签的入参（或 `'static` 等特例）**。

## 结构体中的引用需要生命周期

```rust
struct ImportantExcerpt<'a> {
    part: &'a str,
}
```

含义：`ImportantExcerpt` 的实例不能比 `part` 指向的数据活得更久。

## 生命周期省略规则（Lifetime Elision）

编译器用三条规则在常见场景省略标注：

1. 每个引用参数都有自己的输入生命周期
2. 只有一个输入生命周期时，赋给所有输出生命周期
3. 方法有 `&self` / `&mut self` 时，输出生命周期默认取 self 的生命周期

因此 `fn first_word(s: &str) -> &str` 可省略标注。

---

## 'static 生命周期

`'static` 表示引用可活到整个程序结束。字符串字面量都是 `'static`：

```rust
let s: &'static str = "I have a static lifetime.";
```

不要用 `'static` 逃避错误；通常应修复真实的生命周期不匹配/悬垂引用问题。

---

## 结合泛型 + trait bound + 生命周期

```rust
use std::fmt::Display;

fn longest_with_an_announcement<'a, T>(x: &'a str, y: &'a str, ann: T) -> &'a str
where
    T: Display,
{
    println!("Announcement! {}", ann);
    if x.len() > y.len() { x } else { y }
}
```

## 记忆卡片

| 要点 | 一句话 |
|------|--------|
| `'a` 是什么 | 生命周期**标签**，不是新引用 |
| `&'a` 入 + `&'a` 出 | 返回引用**源自**入参，**同寿命绑定** |
| 为何存在 | 编译期禁止悬垂引用 |
| 不能做什么 | 无入参时不能合法返回 `&'a`；不能返回指向函数局部值的引用 |
