# 9.1 panic! 与不可恢复的错误

当程序遇到“无法或不应该继续”的情况时，可以使用 `panic!`。触发 panic 后，程序会打印错误信息（可选 backtrace），然后退出。

对应 demo 项目：`09-error-handling/9.1-panic-demo/`

---

## panic 时：展开（unwind）还是终止（abort）

- **默认行为：展开（unwind）**：回溯栈并清理遇到的各层函数栈数据，再退出
- **可选：终止（abort）**：直接退出，不做展开清理（最终内存由操作系统回收）

如果你希望最终二进制更小，可以在 `Cargo.toml` 配置 profile（示例）：

```toml
[profile.release]
panic = "abort"
```

---

## 两类常见 panic

- **显式 panic**：直接调用 `panic!("...")`
- **由 bug 导致的 panic**：例如 `Vec` 越界索引 `v[99]`

---

## backtrace（定位问题关键）

当 panic 发生在标准库/依赖库里时，报错位置可能指向库源码；这时看 backtrace，找到**第一处出现你自己源码文件的那一行**，通常就是排查起点。

Windows PowerShell 下开启 backtrace（示例）：

```powershell
$env:RUST_BACKTRACE=1
cargo run -- oob
```

---

## demo 运行

```bash
cd 09-error-handling/9.1-panic-demo

# 触发显式 panic
cargo run -- panic

# 触发 Vec 越界 panic
cargo run -- oob
```
